Per eseguire la migrazione dei timer EJB

Se si arresta un'istanza di GlassFish Server, viene interrotta anche l'esecuzione dei timer di tecnologia EJB (Enterprise JavaBeans) installati nell'istanza. Se l'arresto dell'istanza è anomalo o imprevisto, per abilitare l'esecuzione dei timer quando l'istanza è arrestata è necessario eseguirne la migrazione in un'istanza server in esecuzione nel cluster.

  1. Espandere il nodo Cluster nell'albero di navigazione.

  2. Selezionare il cluster nel nodo Cluster dell'albero di navigazione.

    Viene visualizzata la pagina Informazioni generali per il cluster.

  3. Nella pagina Informazioni generali fare clic su Esegui migrazione timer EJB.

    Viene visualizzata la pagina Esegui migrazione timer EJB.

    Nota:

    Se tutte le istanze nel cluster sono in esecuzione, il pulsante Esegui migrazione timer EJB è disattivato.

    Per assistere l'utente durante la migrazione dei timer, la pagina Esegui migrazione timer EJB contiene una tabella di timer EJB con le informazioni riportate di seguito.

    Nome server

    Lista delle istanze di GlassFish Server nel cluster.

    Conteggio timer

    Il numero di timer persistenti in ciascuna istanza.

  4. Nella pagina Esegui migrazione timer EJB selezionare le istanze di origine e di destinazione per la migrazione dei timer.

    1. Nell'elenco a discesa Origine selezionare l'istanza arrestata da cui eseguire la migrazione dei timer.

    2. Nell'elenco a discesa Destinazione selezionare l'istanza in esecuzione in cui eseguire la migrazione dei timer.

    3. Fare clic su OK.

Se l'istanza di origine è in esecuzione oppure l'istanza di destinazione non è in esecuzione, nell'Console di amministrazione viene visualizzato un messaggio di errore.
