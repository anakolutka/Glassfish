重新部署应用程序或模块

使用“重新部署应用程序或模块”页面可以重新部署先前部署的应用程序。

“重新部署应用程序或模块”页面包含以下选项：

位置

您要重新部署的应用程序新归档的位置。

以下选项指明了可以访问归档文件的位置，以及归档文件是一个文件还是一个目录。

要上载到服务器的打包文件

归档文件位于客户机中的文件或可从客户机访问的文件中。

客户机是可通过浏览器查看管理控制台的主机。

可从 Enterprise Server 访问的本地打包文件或目录

归档文件是位于服务器计算机上的文件，或者是展开目录中的未打包应用程序。

服务器计算机是运行 Enterprise Server 域管理服务器的主机。

应用程序名称

显示要重新部署的应用程序名称的只读字段。

保留会话

如果选中“保留会话”复选框，则系统将保留您要重新部署的应用程序的活动会话，然后在重新部署完成之后恢复这些会话。

如果无法保留或存储应用程序的任何活动会话，则在重新部署完成时将无会话可用。但是，重新部署将继续，并且记录一个警告。

为了保留活动会话，Enterprise Server 会对会话进行序列化，并将其保存在内存中。为了恢复这些会话，新近重新部署的应用程序的类加载器将对先前保存的所有会话进行反序列化。

预编译 JSP

如果选中了此选项，将对 JavaServer Pages (JSP) 文件进行预编译。如果禁用此选项，则在运行时首次访问 JSP 文件时对其进行编译。默认情况下禁用此选项。

运行验证器

如果选中此选项，则将在部署前运行验证器。如果验证失败，则部署将不会执行。验证器将检查文件的结构和内容。大型应用程序的检验通常会很耗费时间。默认情况下禁用此选项。
