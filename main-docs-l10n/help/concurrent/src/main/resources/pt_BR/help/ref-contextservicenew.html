Novo Serviço de Contexto

Use a página Novo Serviço de Contexto para criar um recurso de serviço de contexto.

A página Novo Serviço de Contexto contém as opções a seguir.

Nome JNDI

Um nome JNDI exclusivo que identifica o recurso. Por convenção, o nome começa com um indicador de tipo de recurso e uma barra, tais como concurrent/, jdbc/ ou jms/.

Informações de Contexto

Os contextos a serem propagados para threads:, Classloader; JNDI, Segurança ou WorkArea. A propagação de contexto pode ser ativada ou desativada. Se a opção estiver ativada, os contextos selecionados serão propagados. Se esta opção estiver desativada, nenhum dos contextos será propagado, mesmo que eles estejam selecionados. O valor default é Ativado, com todos os quatro contextos selecionados.

Descrição

Uma descrição deste recurso.

Status

O status do recurso. O recurso pode ser ativado ou desativado. O valor default é Ativado.

Propriedades Adicionais

Propriedades adicionais do recurso. O GlassFish Server não define propriedades adicionais para este tipo de recurso. Além disso, este tipo de recurso não usa atualmente propriedades adicionais.

Alvos

(Esta opção só será exibida se os clusters ou instâncias standalone tiverem sido criados no domínio.)

Clusters e instâncias standalone do recurso. Mova os alvos desejados para a coluna Alvos Selecionados utilizando os botões Adicionar e Adicionar Tudo. Mova os alvos desnecessários para a coluna Alvos Disponíveis utilizando os botões Remover e Remover Tudo.
