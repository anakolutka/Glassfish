So wird der Grenzwert für die Anzahl von rotierten Logdateien geändert

Wenn GlassFish Server eine Logdatei rotiert, wird eine neue, leere Datei mit dem Namen server.log erstellt und die alte Datei in server.log_date umbenannt, wobei mit date Datum und Uhrzeit des Rotierens der Datei angegeben wird.

Standardmäßig wird durch GlassFish Server die Anzahl der rotierten Logdateien auf 10 begrenzt. Wenn die maximale Anzahl erreicht ist, wird die älteste Logdatei gelöscht, wenn GlassFish Server die nächste Logdatei rotiert.

Gegebenenfalls können Sie die maximale Anzahl der rotierten Logdateien ändern. Eine neu festgelegte Begrenzung gilt für rotierte Logdateien für sowohl Zugriffs- als auch Server-Logging.

  1. Legen Sie die Systemeigenschaft com.sun.enterprise.server.logging.max_history_files auf die maximale Anzahl von rotierten Logdateien fest, die beibehalten werden sollen.

    Weitere Informationen zum Festlegen einer Systemeigenschaft finden Sie unter So werden Systemeigenschaften verwaltet.

    Die Systemeigenschaft com.sun.enterprise.server.logging.max_history_files hat das folgende Behavior:

    • Wenn die Eigenschaft nicht festgelegt ist, speichert GlassFish Server maximal 10 rotierte Logdateien.

    • Wenn für die Eigenschaft ein ungültiger Wert oder Null festgelegt ist, speichert GlassFish Server maximal 10 rotierte Logdateien.

    • Wenn die Eigenschaft auf 0 gesetzt ist, speichert GlassFish Server keine rotierten Logdateien.
