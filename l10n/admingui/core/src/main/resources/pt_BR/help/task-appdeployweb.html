Para implantar um aplicativo da Web

  1. Na árvore de navegação, selecione o nó Aplicativos.

    A página Aplicativos é exibida.

  2. Na tabela Aplicativos Implantados, clique no botão Implantar.

    A página Implantar Aplicativos ou Módulos é exibida.

  3. Especifique o local do arquivo compacto que contém o aplicativo a ser implantado.
    • Se o arquivo compacto residir em ou for acessado do computador cliente, selecione a opção Arquivo Empacotador a Ser Carregado no Servidor.

      O computador cliente é o host em que você visualiza o Console de Administração por meio de um navegador.

      Clique em Procurar para localizar o arquivo ou digite o caminho completo para o arquivo.

    • Se o arquivo compacto residir no computador servidor ou se for um aplicativo não empacotado localizado em um diretório destacado, selecione a opção Arquivo Empacotado Local ou Diretório Acessado no Enterprise Server.

      O computador servidor é o host que executa o servidor de administração de domínio (DAS).

      Clique em Procurar Arquivos para localizar um arquivo ou clique em Procurar Pastas para localizar um diretório. Como alternativa, digite o caminho completo para o arquivo ou diretório. A implantação a partir de um diretório avançado destina-se a desenvolvedores avançados e não é recomendada para ambientes de produção.

  4. Na lista suspensa Tipo, selecione Aplicativo da Web.

    Se você tiver especificado um Arquivo Empacotador a Ser Carregado no Servidor ou um Arquivo Empacotado Local Acessado do Enterprise Server, o tipo do aplicativo será detectado automaticamente. Se você tiver especificado um Diretório Acessado do Enterprise Server, você deve especificar o tipo manualmente.

  5. No campo Raiz de Contexto, digite uma sequência que identifica o aplicativo da Web.

    Na URL do aplicativo Web, a raiz de contexto aparece imediatamente após o número da porta (http://host:port/ context-root/...). A raiz de contexto deve iniciar com uma barra; por exemplo, /hello.

  6. No campo Nome do Aplicativo, preserve o nome padrão, que é o nome do diretório ou o prefixo do nome do arquivo, ou digite outro nome.

    O nome padrão será exibido se você optar por carregar um arquivo. O nome do aplicativo deve ser exclusivo.

  7. Na lista suspensa Servidores Virtuais, selecione o servidor virtual a ser associado ao aplicativo.
  8. Marque a caixa de seleção Satatus Ativado para ativar o aplicativo.

    Por padrão, a opção está ativada. Esta opção está desativada, o aplicativo não está disponível.

  9. Marque a caixa de seleção Precompilar JSPs para precompilar arquivos JavaServer PagesTM (JSPTM).

    Se esta opção for desativada, os arquivos JSP serão compilados no tempo de execução, quando forem acessados pela primeira vez. Por padrão, a opção está desativada.

  10. Marque a caixa de seleçao Executar Verificador para executar o verificador antes da implantação.

    Se esta opção estiver ativada e se a verificação falhar, a implantação não será realizada. O verificador examina a estrutura e o conteúdo do arquivo. A verificação de aplicativos grandes geralmente consome muito tempo. Por padrão, a opção está desativada.

  11. No campo Bibliotecas, digite uma lista separada por vírgulas dos arquivos JAR da biblioteca, específicos deste módulo ou aplicativo.

    Os caminhos podem ser absolutos ou relativos. Um caminho relativo é relativo aodomain-dir/lib/applibs Se o caminho for absoluto, o caminho deve ser acessível ao servidor de administração de domínio (DAS), o que significa que ele deve estar em domain-dir As bibliotecas são disponibilizadas para o aplicativo na ordem em que elas são especificadas.

  12. No campo Descrição, digite uma descrição para o aplicativo.
  13. Clique em OK.
