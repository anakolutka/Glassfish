Crear un módulo de auditoría

La auditoría es el modo en que GlassFish Server registra ciertos eventos, como errores o violaciones de seguridad. GlassFish Server registra todos los eventos de autenticación en archivos log. Un log de acceso completo ofrece una pista secuencial de los eventos de acceso de GlassFish Server.

GlassFish Server ofrece un sencillo módulo de auditoría por defecto; para obtener más información, consulte Utilizar el módulo de auditoría por defecto.

  1. En el árbol de navegación, amplíe el nodo Configuración.

  2. Seleccione la instancia que va a configurar:

    • Para configurar una instancia determinada, amplíe el nodo de configuración de la instancia. Por ejemplo, para la instancia por defecto server, amplíe el nodo server-config.

    • Para configurar los valores por defecto para todas las instancias, amplíe el nodo default-config.

  3. En el nodo Configuración, amplíe el nodo Seguridad.

  4. En el nodo Seguridad, seleccione el nodo Módulos de Auditoría.

    Se abrirá la página Módulos de Auditoría.

  5. En la página Módulos de Auditoría, haga clic en Nuevo.

    Se abrirá la página Nuevo Módulo de Auditoría.

  6. En la página Nuevo Módulo de Auditoría, en el campo Nombre, escriba el nombre del módulo de auditoría.

  7. En el campo Nombre de Clase, escriba el nombre completo de la clase que implanta este módulo.

    El nombre de clase para el módulo de auditoría por defecto es com.sun.enterprise.security.Audit.

  8. En la sección Propiedades Adicionales, especifique las propiedades adicionales.

    Para agregar una propiedad, haga clic en el botón Agregar Propiedad. En la fila en blanco que aparece, introduzca el nombre de propiedad en el campo Nombre y escriba el valor de propiedad en el campo Valor.

    Para ver una descripción de las propiedades adicionales disponibles para los módulos de auditoría, consulte Propiedades específicas de módulos de auditoría.

  9. Haga clic en Aceptar.
