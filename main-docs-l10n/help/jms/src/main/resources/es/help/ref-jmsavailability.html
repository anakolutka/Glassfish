Disponibilidad de JMS

Utilice la página Disponibilidad de JMS para configurar el cluster de broker Cola de mensajes para atender el cluster de GlassFish Server.

Atención:

Antes de usar la página Disponibilidad de JMS para volver a configurar un cluster existente, realice los procedimientos específicos para la migración a otro tipo de cluster de broker, como se describe en la sección sobre migración entre tipos de clusters de broker convencionales embebidos o locales en Oracle GlassFish Server High Availability Administration Guide. Si no realiza estos procedimientos especiales, se pueden perder o corromper datos o incluso hacer que la configuración sea inservible, dependiendo de las operaciones de JMS realizadas en el cluster existente.

La página Disponibilidad de JMS contiene las siguientes opciones.

Nota:

Si se están ejecutando instancias que utilizan la configuración, la página Disponibilidad de JMS mostrará una advertencia, definirá todas las opciones en sólo lectura y desactivará el botón Guardar.
Tipo de servicio JMS

Tipo de broker (llamado host JMS en GlassFish Server) que se debe usar en el cluster de broker. El tipo seleccionado determinará los tipos de clusters de broker disponibles. Los tipos de broker disponibles son los siguientes:

Embebido

GlassFish Server gestiona el ciclo de vida y la configuración del broker. Cada instancia en cluster de GlassFish la sirve un broker que se ejecuta en la misma JVM que la instancia.

Si la opción Tipo de servicio JMS se define en Embebido, sólo se soportarán los clusters de broker Cola de mensajes convencionales. Se mostrarán las opciones de configuración del cluster convencional.

Local

GlassFish Server gestiona el ciclo de vida y la configuración del broker. Cada instancia en cluster de GlassFish la sirve un broker que se ejecuta en una JVM distinta situada en el mismo host que la instancia.

Si la opción Tipo de servicio JMS se define en Local, se soportarán los clusters de broker Cola de mensajes convencionales y también los mejorados (con alta disponibilidad). Se mostrarán las opciones para seleccionar el tipo de cluster y para la configuración del mismo.

Remoto

La configuración y el ciclo de vida del broker y del cluster de broker se gestionan a través de las herramientas administrativas de Cola de mensajes.

Si la opción Tipo de servicio JMS se define en Remoto, no se mostrarán otras opciones.

Tipo de cluster de JMS

Tipo de cluster de broker Cola de mensajes que se debe usar para el cluster de GlassFish Server.

Si la opción Tipo de cluster de JMS se define en Convencional, se mostrarán las opciones siguientes.

Tipo de almacén de configuración de JMS

Tipo de almacén de datos que se va a usar para los datos de configuración del cluster convencional. Si esta opción se define como Broker maestro, se designará un broker del cluster como broker maestro que será responsable del almacenamiento de los datos de configuración. Si esta opción se define en Base de datos compartida, los datos de configuración se almacenarán en un almacén de datos JDBC accesible para todos los brokers. En este caso, se mostrarán las opciones relacionadas con la base de datos.

Tipo de almacén de mensajes de JMS

Tipo de almacén de datos que va a usar cada broker para almacenar los datos de sus mensajes. Si se define esta opción en Archivo, cada broker almacenará los datos correspondientes en un almacén de datos basado en archivos. Si se define esta opción en JDBC, cada broker almacenará los datos correspondientes en un almacén de datos JDBC.

Si la opción Tipo de cluster de JMS se define en Mejorado (HA), se mostrarán las opciones relacionadas con las bases de datos.

Proveedor de base de datos
URL de base de datos
Usuario de base de datos

Proveedor de la base de datos, URL de acceso y usuario de la base de datos JDBC que se van a utilizar en cualquiera de estas situaciones:

  • Cuando el tipo de almacén de configuración de un cluster convencional se ha definido en Base de datos compartida.

  • Cuando el tipo de almacén de mensajes de un cluster convencional se ha definido en JDBC.

  • Cuando el tipo de cluster se ha definido en Mejorado (HA).

Para obtener información sobre proveedores soportados y los formatos de URL de acceso de cada proveedor, consulte la sección sobre persistencia basada en JDBC en Oracle GlassFish Server Message Queue Administration Guide.

Autenticación de base de datos
Contraseña de base de datos
Alias de contraseña

Información de contraseña para el usuario de la base de datos JDBC especificado en Usuario de base de datos.

Si la opción Autenticación de base de datos está definida en Contraseña, se mostrará la opción Contraseña de base de datos. Si la opción Autenticación de base de datos está definida en Alias de contraseña, se mostrará la opción Alias de contraseña.

Propiedades Adicionales

Propiedades adicionales para la disponibilidad de JMS.

GlassFish Server no predefine las propiedades adicionales de disponibilidad de JMS.
