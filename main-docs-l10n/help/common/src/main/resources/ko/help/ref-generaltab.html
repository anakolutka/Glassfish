일반 정보

GlassFish Server은(는) 설치 시 서버라는 한 개의 애플리케이션 서버 인스턴스를 생성합니다. 원하는 경우 서버 인스턴스를 삭제하고 다른 이름의 새 인스턴스를 생성할 수 있습니다.

각 GlassFish Server 인스턴스는 고유한 Java 구성, Java 리소스, 응용 프로그램 배치 영역 및 서버 구성 설정을 사용합니다. 한 애플리케이션 서버 인스턴스가 변경되면 다른 애플리케이션 서버 인스턴스도 영향을 받습니다. 한 관리 도메인 내에는 한 개의 애플리케이션 서버 인스턴스가 있을 수 있습니다.

대부분의 사용자에게는 애플리케이션 서버 인스턴스가 하나만 있으면 됩니다. 하지만 환경에 따라 애플리케이션 서버 인스턴스를 추가로 하나 이상 생성해야 할 수도 있습니다. 예를 들어, 개발 환경에서 여러 애플리케이션 서버 인스턴스를 사용하여 다른 GlassFish Server 구성을 테스트하거나 여러 응용 프로그램 배치를 비교 및 테스트할 수 있습니다. 애플리케이션 서버 인스턴스는 쉽게 추가하거나 삭제할 수 있으므로 이를 통해 개발 도중 테스트할 임시 "Sandbox" 영역을 생성할 수 있습니다.

또한 애플리케이션 서버 인스턴스마다 가상 서버를 생성할 수도 있습니다. 설치된 하나의 애플리케이션 서버 인스턴스 내에 회사 또는 개인 도메인 이름, IP 주소 및 일부 관리 기능을 제공할 수 있습니다. 사용자의 경우 이는 하드웨어 및 기본 서버 유지 관리 없이 자체 웹 서버를 보유하고 있는 것과 마찬가지입니다. 이 가상 서버는 애플리케이션 서버 인스턴스를 확장하지 않습니다. 가상 서버에 대한 자세한 내용은 JVM 일반 설정을 참조하십시오.

운영 배치에서 대부분의 경우 가상 서버를 여러 애플리케이션 서버 인스턴스 대신 사용할 수 있습니다. 그러나 가상 서버가 사용자의 요구를 충족하지 못할 경우 애플리케이션 서버 인스턴스를 여러 개 사용할 수도 있습니다.

GlassFish Server 인스턴스는 자동으로 시작되지 않습니다. 인스턴스는 한 번 시작되면 정지할 때까지 실행됩니다. 애플리케이션 서버 인스턴스를 정지하면 새 접속이 허용되지 않고 처리되지 않은 접속이 모두 완료될 때까지 기다립니다. 시스템이 손상되거나 오프라인인 경우 서버가 종료되고 서버에서 제공하던 요청이 손실될 수 있습니다.

[일반 정보] 페이지에는 다음 정보가 포함되어 있습니다.

정지 단추

GlassFish Server를 정지하려면 [정지] 단추를 누릅니다.

재시작 단추

GlassFish Server를 재시작하려면 [재시작] 단추를 누릅니다.

로그 파일 보기 단추

GlassFish Server 인스턴스나 클러스터에 대한 로그 파일을 보려면 [로그 파일 보기] 단추를 누릅니다.

로그 교체 단추

관리 서버(서버)의 로그 파일을 교체하려면 [로그 교체] 단추를 누릅니다.

트랜잭션 복구 단추

페이지에서 관리 서버(서버트랜잭션 복구)에 대한 트랜잭션을 복구하려면 [트랜잭션 복구] 단추를 누릅니다.

보안 관리 단추

[보안 관리] 단추를 누르면 보안 관리 페이지에서 보안 관리가 사용 또는 사용 안함으로 설정됩니다.

이름

현재 서버의 이름입니다.

상태

서버 인스턴스의 현재 상태입니다. 서버는 정지됨, 시작됨 또는 실행 중 상태일 수 있습니다.

작동 시간

서버 인스턴스가 연속으로 실행 중인 시간(시:분)을 나타내며, 읽기 전용입니다.

HTTP 로드 밸런서

서버 인스턴스에 대해 로드 밸런서가 사용으로 설정된 경우, 로드 밸런서의 이름입니다.

JVM

JVM 보고서를 누르면 별도의 창이 열리고 요약 보고서, 메모리 관리 및 불필요한 정보 수집 보고서, 클래스 로딩 보고서 및 현재 스레드 덤프를 포함한 Java Virtual Machine에 대한 보고서가 표시됩니다.

구성 디렉토리

domain.xml과 같은 구성 파일이 포함된 호스트 시스템의 디렉토리입니다.

설치된 버전

제품 버전 식별자입니다.

보안 관리

보안 관리의 현재 상태입니다. 가능한 값은 [사용] 및 [사용 안함]입니다.

디버그

디버깅이 사용으로 설정되었는지 여부를 식별합니다. 디버깅이 사용으로 설정된 경우 포트 번호도 표시됩니다.

HTTP 포트

HTTP 요청 및 응답에 대해 현재 구성된 포트입니다.

IIOP 포트

IIOP 요청 및 응답에 대해 현재 구성된 포트입니다.
