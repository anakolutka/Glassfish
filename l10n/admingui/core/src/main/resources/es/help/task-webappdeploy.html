Cómo implementar una aplicación web

  1. En el árbol de navegación, expanda el nodo Aplicaciones.
  2. En el nodo Aplicaciones, seleccione el nodo Aplicaciones web.

    Se abrirá la página Aplicaciones web.

  3. En la tabla Aplicaciones web implementadas, haga clic en el botón Implementar.

    Se abrirá la página Implementar aplicaciones y módulos empresariales.

  4. En la lista desplegable Tipo, seleccione la aplicación web (.war).
  5. Especifique la ubicación del archivo de almacenamiento donde se encuentra la aplicación que se va a implementar.
    • Si el archivo de almacenamiento es un archivo que reside en la máquina cliente o al que se puede acceder desde la máquina cliente, seleccione la opción Archivo empaquetado que se cargará en el servidor.

      La máquina cliente es el host en el que está viendo la Consola de administración por medio de un navegador.

      Haga clic en Examinar para buscar el archivo o bien escriba la ruta completa hasta el archivo.

    • Si el archivo de almacenamiento es un archivo que reside en el equipo servidor o es una aplicación descomprimida desde un directorio desplegado, seleccione la opción Archivo empaquetado local o Directorio accesible desde el servidor de aplicaciones.

      El equipo servidor es el host que se está ejecutando en el servidor de administración del dominio (DAS).

      Escriba el nombre de la ruta completa del archivo o directorio. Implementar desde un directorio desplegado es para desarrolladores avanzados y no es recomendable para entornos de producción.

  6. En el campo Nombre de la aplicación, puede mantener el nombre predeterminado, que es el prefijo del nombre del archivo, o bien escribir un nombre distinto.

    El nombre predeterminado aparece si decide cargar un archivo. El nombre de la aplicación debe ser único.

  7. En el campo Raíz de contexto, introduzca la cadena que identifica la aplicación web.

    En la URL de la aplicación web, la raíz de contexto aparece justo después del número de puerto (http://host: port/context-root/...). La raíz de contexto debe comenzar por una barra diagonal, por ejemplo, /hello.

  8. Para deshabilitar la aplicación de manera que no esté disponible, anule la selección de la casilla de verificación Habilitado.

    De manera predeterminada, la aplicación presenta el estado Habilitado una vez implementada.

  9. Marque la casilla de verificación Verificador de ejecución activado para verificar la estructura y los contenidos del archivo antes de la implementación.

    La verificación de aplicaciones de gran tamaño puede llevar mucho tiempo. Verifique el archivo sólo si piensa que puede estar dañado o que no es transferible.

  10. Marque la casilla de verificación Precompilar JSP activado para precompilar archivos JavaServer PagesTM (JSPTM).

    Si no marca esta casilla, los archivos JSP se compilarán durante el tiempo de ejecución cuando se acceda a ellos por primera vez. Como la compilación requiere mucho tiempo, en un entorno de producción es mejor marcar esta casilla.

  11. En el campo Bibliotecas, enumere los archivos de biblioteca JAR que requiera la aplicación.
  12. En el campo Descripción, introduzca una descripción de esta aplicación web.
  13. Haga clic en Aceptar.
