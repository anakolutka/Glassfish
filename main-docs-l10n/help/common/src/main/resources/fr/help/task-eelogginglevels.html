Configuration de niveaux de journalisation

  1. Dans l'arborescence de navigation, développez le noeud Configurations.

  2. Sous le noeud Configuration, cliquez sur la configuration d'instance de serveur ou de cluster pour laquelle vous souhaitez définir les paramètres du journaliseur.

    La page Configuration apparaît.

  3. Sur la page Configuration, cliquez sur Paramètres du journaliseur.

    La page Paramètres du journaliseur de la cible de configuration sélectionnée apparaît.

  4. Sur la page Paramètres du journaliseur, cliquez sur l'onglet Niveaux de journalisation.

    La page Niveaux de journalisation de module apparaît.

  5. Sur la page Niveaux de journalisation de module, modifiez les niveaux de journalisation des modules comme suit.

    • Pour modifier le niveau de journalisation d'un module individuel, sélectionnez une nouvelle valeur dans la liste déroulante située en face du module dont vous voulez modifier le niveau de journalisation.

    • Pour modifier le niveau de journalisation d'un groupe de modules, procédez comme suit :

      1. Cochez les cases en regard des noms des modules dont vous voulez modifier le niveau de journalisation.

      2. Sélectionnez une valeur dans la liste déroulante en haut de la table.

      3. Cliquez sur le bouton Modifier le niveau.

    Les messages correspondant au niveau sélectionné ou à un niveau supérieur apparaissent dans le journal. Choisissez l'une des valeurs suivantes (classées dans l'ordre décroissant) :

    • SEVERE

    • WARNING

    • INFO

    • CONFIG

    • FINE

    • FINER

    • FINEST

    • OFF

    Pour en savoir plus sur la signification des niveaux de journalisation, reportez-vous à Niveaux de journalisation de module. Le niveau par défaut est INFO. Par conséquent, le journal contiendra par défaut des messages des niveaux INFO, WARNING et SEVERE.

  6. Cliquez sur Enregistrer.

    Les modifications apportées aux paramètres de niveau de journalisation prennent effet immédiatement. Elles sont également enregistrées en vue d'une utilisation au redémarrage du serveur.
