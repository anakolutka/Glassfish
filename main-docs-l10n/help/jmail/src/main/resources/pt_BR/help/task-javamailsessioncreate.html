Para Criar uma Sessão JavaMail

  1. Na árvore de navegação, expanda o nó Recursos.

  2. No nó Recursos, selecione Sessões JavaMail.

    A página Sessões JavaMail é aberta.

  3. Na tabela Sessões, clique em Novo.

    A página Nova sessão JavaMail será aberta.

  4. No campo Nome JNDI, digite um nome exclusivo para o recurso de sessão de correio.

    Use o prefixo de subcontexto de nomeação mail/para recursos do JavaMail. Por exemplo, mail/MySession. O nome deve conter somente caracteres alfanuméricos, sublinhado, traço ou ponto.

  5. No campo Host de Correio, digite o nome do host do servidor de correio default.

    Os métodos de conexão dos objetos Armazenamento e Transporte usam esse valor se não é fornecida uma propriedade de host específica do protocolo. O nome deve ser resolúvel para um nome de host real.

  6. No campo Usuário Default, digite o nome do usuário a ser fornecido durante a conexão com um servidor de correio.

    Os métodos de conexão dos objetos Armazenamento e Transporte usam esse valor se não é fornecida uma propriedade de nome de usuário específica do protocolo. O nome de usuário deve conter somente caracteres alfanuméricos, sublinhado, traço ou ponto.

  7. No campo Endereço Default do Remetente, digite o endereço de e-mail do usuário default.

    O formato é

Nome do usuário@host.domain.

No campo Descrição, digite uma descrição para a sessão de correio.

Marque a caixa de seleção Status Ativado para ativar a sessão de correio.

O valor default é Ativado.

No campo Protocolo do Armazenamento, defina o serviço de protocolo do armazenamento a ser utilizado.

O serviço de protocolo do armazenamento conecta-se a um servidor de correio, recupera mensagens e as salva em uma ou mais pastas. Os valores permitidos são imap, pop3, imaps e pop3s. O valor default é imap.

No campo Classe de Protocolo do Armazenamento, especifique a classe de implementação do provedor de serviço para armazenamento.

Os valores permitidos são:

  • com.sun.mail.imap.IMAPStore

  • com.sun.mail.pop3.POP3Store

  • com.sun.mail.imap.IMAPSSLStore

  • com.sun.mail.pop3.POP3SSLStore

O valor default é com.sun.mail.imap.IMAPStore.

No campo Protocolo do Transporte, especifique o método de comunicação de transporte.

O serviço de protocolo de transporte envia mensagens. Os valores permitidos são smtp e smtps. Por default, o Protocolo do Transporte é smtp.

No campo Classe do Protocolo de Transporte, especifique a classe de implementação do provedor de serviço para transporte.

O valor default é com.sun.mail.smtp.SMTPTransport.

Marque a caixa de seleção Depuração para ativar a saída de depuração extra, incluindo um rastreamento de protocolo, para a sessão de correio.

Se o nível de log JavaMail estiver definido como BOM ou melhor, a saída de depuração será gerada e incluída no arquivo de log do sistema. Para obter informações sobre a definição do nível de log, consulte Para Configurar Níveis de Log. Por default, essa opção está desativada.

Na tabela Propriedades Adicionais, especifique propriedades adicionais requeridas por aplicações, como uma propriedade de nome de usuário ou de host específica do protocolo.

Para obter uma lista das propriedades disponíveis, consulte a documentação da API JavaMail (http://java.sun.com/products/javamail/javadocs/index.html). Propriedades adicionais devem começar com mail- e devem usar hífens, não pontos, como separadores. Por exemplo, mail-debug é correto, mas mail.debug não. O back-end do GlassFish Server converte os hífens nos pontos esperados pela JavaMail API.

Para adicionar uma propriedade, clique no botão Adicionar Propriedade. Na linha em branco que for exibida, digite o nome da propriedade no campo Nome e digite o valor da propriedade no campo Valor.

Na lista de alvos disponíveis, selecione o alvo ou alvos dos quais configurar a sessão de e-mail e clique em Adicionar.

Os alvos podem ser clusters ou instâncias standalone do servidor. Se um alvo não for selecionado, a sessão é configurado com a instância default do servidor server.

Clique em OK.
