配置常规日志记录设置

  1. 在导航树中，展开“配置”节点。
  2. 在“配置”节点下，选择“日志程序设置”节点。

    此时将打开“日志程序设置”页面。

  3. 选中“启用写入系统日志”复选框，以便除了服务器日志以外，还将日志记录输出发送到 syslog 工具。

    此功能仅在 Solaris 和 Linux 系统上可用。默认情况下禁用此选项。

  4. 选中“启用记录到控制台中”复选框以了解情况

    默认情况下禁用此选项。

  5. 在“日志文件”字段中，键入服务器日志文件的替代名称或位置。

    默认位置为 glassfish/domains/domain1/logs/server.log。

  6. 在“日志处理程序”字段中，键入自定义日志处理程序的绝对类名。

    自定义日志处理程序可让您将日志发送到除了 server.log 或 syslog 以外的其他目标。自定义处理程序必须扩展 java.util.logging.Handler 类（JSR 047 兼容 API）。将处理程序类置于 Enterprise Server 类路径中，以便在服务器启动过程中安装该处理程序。

  7. 在“文件轮转限制”字段中，键入在轮转之前，服务器日志文件必须达到的大小（以字节为单位）。

    服务器日志达到指定大小时，Enterprise Server 将创建一个新的名为 server.log 的空文件，并将旧文件重命名为 server.log_date，其中 date 是轮转文件的日期和时间。默认值为 2 兆字节。该限制的最小值为 500 千字节；如果指定更低的值，则要在达到 500 千字节时该文件才轮转。要关闭日志文件轮转，请将该值设置为 0。

  8. 在“文件轮转时间限制”字段中，键入服务器日志必须轮转之前的分钟数。

    默认值为 0，表示文件达到“文件轮转限制”字段中指定的大小时进行轮转。

    注 - 如果指定一分钟或若干分钟，则时间限制将优先于“文件轮转限制”字段指定的大小限制。

  9. 单击“保存”。
  10. 停止并重新启动 Enterprise Server。
