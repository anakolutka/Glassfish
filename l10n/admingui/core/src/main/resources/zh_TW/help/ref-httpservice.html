HTTP 服務

[HTTP 服務] 頁面包含下列選項：

存取記錄

若已選取該選項，針對尚未指定此特性的所有虛擬伺服器子元素，啟用存取記錄功能。如果未選取，則會針對尚未指定此特性的所有虛擬伺服器子元素，停用存取記錄功能。

存取記錄緩衝區大小

存取記錄緩衝區的大小 (以位元組為單位)，或未緩衝存取記錄小於或等於零的值。預設值為 4096。

存取記錄寫入間隔

將記錄寫入磁碟之前經過的秒數。當緩衝區已滿或間隔過期時，存取記錄會被寫入。如果值小於或等於零，則即使緩衝區未滿，也會一直寫入緩衝區。這意味著每次存取伺服器時，會將記錄訊息直接儲存至檔案中。

其他特性

HTTP 服務的其他特性。

您可以使用下列特性：

connectionTimeout

指定 HTTP 偵聽程式在針對要顯示的請求 URI 行接受連線之後等待的毫秒數。

預設值為 12,000 (12 秒)。

maxKeepAliveRequests

指定在伺服器關閉連線之前可以透過管線傳送的 HTTP 請求的最大數目。將該特性設定為 1 可停用 HTTP/1.0 持續作用、HTTP/1.1 持續作用以及管線操作。

預設值為 1000。

traceEnabled

如果設定為 True，則啟用 TRACE 方法。將該特性設定為 false 可減少跨站點程序檔攻擊對 Enterprise Server 的影響。

如需有關 TRACE 方法的更多資訊，請參閱 RFC 2616: Hypertext Transfer Protocol -- HTTP/1.1。

預設值為 True。
