透過 JDBC 技術將應用程式連線至資料庫

JavaTM 資料庫連結 (JDBCTM) 技術提供關聯式資料庫的連結。在 Enterprise Server 中，可以透過將 JDBC 資源 (資料來源) 增加至網域來配置資料庫連結。資料來源是 Java Platform, Enterprise Edition (Java EE 平台) 配置資料庫連結的標準方法。每個資料來源都包含一個資料庫連線池。應用程式會在 Java Naming and Directory InterfaceTM (J.N.D.I.) API 樹狀結構或本機應用程式環境中查找資料來源，然後使用 getConnection 方法擷取資料庫連線。資料來源及其連線池提供了有助於保持系統執行的連線管理程序。

透過 JDBC 技術將應用程式連線至資料庫包括下列作業：

  • 設定資料庫存取

  • 整合 JDBC 驅動程式

  • 建立 JDBC 資源

  • 編輯 JDBC 資源

  • 刪除 JDBC 資源

  • 建立 JDBC 連線池

  • 編輯 JDBC 連線池一般設定

  • 編輯 JDBC 連線池進階屬性

  • 驗證連線池設定

  • 編輯 JDBC 連線池特性

  • 刪除 JDBC 連線池
