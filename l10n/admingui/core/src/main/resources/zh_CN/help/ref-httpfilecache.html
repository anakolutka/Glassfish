HTTP 文件高速缓存

HTTP 文件高速缓存存储静态内容，以便 Enterprise Server 可以快速处理对静态内容的请求。

“HTTP 文件高速缓存”页面包含以下选项：

全局启用

如果选择此选项，则将启用全局文件高速缓存。默认情况下，此选项未选中。

最长生存期

有效高速缓存条目的最长生存期（以秒为单位）。

该参数用于控制文件经过高速缓存之后，使用高速缓存信息的时间。时间长于最长生存期的条目将用同一文件的新条目替代。如果您的内容很少发生更改，请增大该值以改善性能。

最优的最长生存期取决于现有文件是否被频繁修改。例如，如果以固定的时间间隔一天修改文件四次，则可以考虑将最长生存期设置为 21600 秒（6 小时）。否则，可以考虑将最长生存期设置为您希望在上一版本的文件修改后，处理该文件的最长时间。

最大文件计数

文件高速缓存中可以存储的文件的最大数量。

如果该值过大，服务器将高速缓存较少使用的文件，这样会浪费内存。如果该值过小，将无法体现高速缓存的优点。尝试为该属性设置不同的值，以寻找适合特定应用程序的最佳解决方案。

散列初始大小

散列容器的初始数量。

散列容器的初始数量会影响内存使用和搜索时间。

中等文件大小限制

可以存储为内存映射文件的高速缓存文件的最大大小（以字节为单位）。

中等文件的内容通过将文件映射到虚拟内存中来进行高速缓存。

最优限制是略高于 VM 页面大小的值。

中等文件大小

仅适用于 UNIX® 平台和 Linux 平台：高速缓存为内存映射文件的所有文件的总大小（以字节为单位）。

小文件大小限制

可以读入内存的文件的最大大小。

小文件的内容通过分配堆空间并将文件读入该堆空间来进行高速缓存。

最优限制是略低于 VM 页面大小的值。

小文件大小

要读入内存的文件的总大小（以字节为单位）。

启用文件高速缓存

确定如果文件大小小于中等文件大小限制，是启用还是禁用文件内容的高速缓存。

打开

启用文件内容的高速缓存。

关闭

禁用文件内容的高速缓存。
