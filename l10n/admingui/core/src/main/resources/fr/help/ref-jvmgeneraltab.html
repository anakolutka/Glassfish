Page des paramètres généraux JVM

La machine virtuelle Java (JVM) est incluse dans le logiciel Java Platform Standard Edition (J2SETM), nécessaire à l'exécution d'Enterprise Server. La saisie de paramètres JVM incorrects empêchant l'exécution du serveur, vous devez être prudent(e) lors de la modification de ces paramètres.

Accueil Java

Nom du répertoire d'installation du logiciel Java SE, nécessaire à l'exécution d'Enterprise Server. Pour vérifier si la version du logiciel Java SE spécifié est prise en charge par cette version, consultez les Sun GlassFish Enterprise Server v3 Prelude Release Notes. Si vous saisissez un nom de répertoire inexistant ou le répertoire d'une version non prise en charge du logiciel Java SE, Enterprise Server ne démarrera pas.

Options Javac

Options de ligne de commande pour le compilateur de langage de programmation Java. Enterprise Server exécute le compilateur lorsque des composants Enterprise JavaBeansTM (EJBTM) sont déployés.

Débogage

Active le débogage avec JPDA (Architecture de débogueur de plate-forme Java).

L'architecture JPDA est utilisée par les développeurs d'applications. Pour plus d'informations, consultez le Sun GlassFish Enterprise Server v3 Prelude Developer’s Guide.

Options de débogage

Indique les options JPDA transmises à la JVM quand le débogage est activé.

Options de compilation RMI

Options de ligne de commande pour le compilateur rmic. Enterprise Server exécute le compilateur rmic lorsque des composants EJB sont déployés.

Préprocesseur du bytecode

Noms des classes de prétraitement du bytecode. Séparez les noms de classe par des virgules. Chaque classe doit implémenter l'interface com.sun.appserv.BytecodePreprocessor. Les classes sont appelées par ordre d'énumération.
