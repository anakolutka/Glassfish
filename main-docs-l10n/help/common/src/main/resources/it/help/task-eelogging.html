Per configurare le impostazioni generali di log

  1. Espandere il nodo Configurazioni nell'albero di navigazione.

  2. Nel nodo Configurazioni fare clic sulla configurazione dell'istanza server o del cluster per cui si desidera configurare le impostazioni del logger.

    Viene visualizzata la pagina Configurazione.

  3. Nella pagina Configurazione fare clic su Impostazioni logger.

    Viene visualizzata la pagina Impostazioni logger per la destinazione della configurazione selezionata.

  4. Selezionare la casella di controllo Abilitato dell'opzione Scrivi nel log di sistema per inviare l'output di log alla funzione syslog oltre che al log del server.

    Questa funzione è disponibile solo nei sistemi Solaris e Linux. Questa opzione è disabilitata per impostazione predefinita.

  5. Selezionare la casella di controllo Abilitato dell'opzione Log nella console per inviare l'output di log alla console del sistema.

  6. Selezionare la casella di controllo Abilitato dell'opzione Rotazione a cambio data per eseguire la rotazione dei file di log del server ogni giorno a mezzanotte.

  7. Selezionare la casella di controllo Abilitato dell'opzione Modalità a più righe per formattare i messaggi di log secondo modalità che ne facilitino la lettura. Il corpo dei messaggi di log viene indentato e inizia su una nuova riga.

  8. Nell'elenco a discesa Formato di log console selezionare il formato per il log nella console. L'opzione predefinita è ODL (Oracle Diagnostics Logging). L'altra opzione disponibile è ULF (UniformLogFormatter).

  9. Nell'elenco a discesa Formato di log LogFile selezionare il formato per il log nel file di log del server. L'opzione predefinita è ODL.

  10. Nel campo Limite di rotazione file digitare la dimensione in byte che il file di log del server deve raggiungere prima che ne venga eseguita la rotazione.

    Quando il log del server raggiunge la dimensione specificata, GlassFish Server crea un nuovo file vuoto denominato server.log e rinomina il file precedente server.log_data, dove data corrisponde alla data e all'ora di rotazione del file. Il valore predefinito è 2 megabyte. Il valore limite minimo è 500 KB; se si specifica un valore inferiore, il file viene ruotato quando la dimensione raggiunge 500 KB. Per disattivare la rotazione del file di log impostare il valore su 0.

  11. Nel campo Limite di tempo rotazione file digitare il numero di minuti trascorso il quale si desidera eseguire la rotazione del log del server.

    Il valore predefinito è 0 e indica che il file viene sottoposto a rotazione quando raggiunge la dimensione specificata nel campo Limite di rotazione file.

    Nota:

    Se si specificano uno o più minuti, il limite di tempo ha la precedenza rispetto al limite di dimensione specificato nel campo Limite di rotazione file.

  12. Nel campo Frequenza flush digitare il numero massimo di messaggi scrivibili contemporaneamente dalla coda al log del server.

    Il valore predefinito è 1.

  13. Nel campo Numero massimo di file di cronologia digitare il numero massimo di file di log che GlassFish Server conserva prima di eliminare il file meno recente. Se si imposta questo valore su zero (0), GlassFish Server non elimina alcun file di log meno recente.

  14. Nel campo File di log digitare una posizione alternativa o un nome alternativo per il file di log del server.

    La posizione predefinita è ${com.sun.aas.instanceRoot}/logs/server.log.

  15. Nel campo Handler di log digitare il nome classe assoluto di un handler di log personalizzato.

    Un handler di log personalizzato consente di inviare i log a una destinazione diversa da server.log o syslog. L'handler personalizzato deve estendere la classe java.util.logging.ConsoleHandler (un'interfaccia API conforme allo standard JSR 047). Inserire la classe handler nel classpath di GlassFish Server, in modo che l'handler venga installato durante l'avvio del server.

  16. Fare clic su Salva.

    Nota:

    A seconda della configurazione per cui è stato abilitato il log, potrebbe essere necessario riavviare il server.

    Il riavvio del server potrebbe inoltre essere necessario in caso di modifica del nome del file di log rispetto all'impostazione predefinita. Se è necessario riavviare il server, verrà visualizzato un messaggio Riavvio necessario nell'angolo superiore destro di Console di amministrazione.
