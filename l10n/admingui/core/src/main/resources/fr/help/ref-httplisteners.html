Modules d'écoute HTTP

Chaque module d'écoute HTTP doit posséder sa propre combinaison numéro de port/adresse IP. Par exemple, un module d'écoute HTTP peut écouter sur toutes les adresses IP configurées sur un port donné pour une machine en spécifiant l'adresse IP 0.0.0.0. De même, le module d'écoute HTTP peut spécifier une adresse IP unique pour chaque module, mais utiliser le même port.

Un module d'écoute HTTP étant une combinaison adresse IP/numéro de port, plusieurs modules d'écoute HTTP peuvent avoir la même adresse IP et des numéros de port différents (par exemple, 1.1.1.1:8081 et 1.1.1.1:8082), ou des adresses IP distinctes et le même numéro de port (par exemple, 1.1.1.1:8081 et 1.2.3.4:8081, si votre machine est configurée pour répondre à ces deux adresses).

Toutefois, lorsqu'un module d'écoute HTTP utilise l'adresse IP 0.0.0.0, écoutant sur toutes les adresses IP d'un port, vous ne pouvez pas créer de module d'écoute HTTP pour d'autres adresses IP écoutant sur le même port pour une adresse IP spécifique. Par exemple, si un module d'écoute HTTP utilise la combinaison 0.0.0.0:8080 (toutes les adresses IP sur le port 8080), un autre module d'écoute HTTP ne peut pas utiliser 1.2.3.4:8080.

Le système exécutant Application Server n'ayant généralement accès qu'à une seule adresse IP, les modules d'écoute HTTP utilisent en générale l'adresse IP 0.0.0.0 et des numéros de port différents, chaque numéro servant un objectif distinct. Si le système a accès à plusieurs adresses IP, chacune d'elles doit avoir un objectif distinct.
