Editar una configuración de seguridad de mensajes

Para activar la seguridad de los mensajes para puntos finales de servicios web desplegados en GlassFish Server, debe especificar un proveedor para usar por defecto en el servidor. Si activa un proveedor por defecto para la seguridad de los mensajes, también tendrá que activar proveedores que utilizarán los clientes de los servicios web desplegados en GlassFish Server. Encontrará información sobre la activación de los proveedores usados por clientes en Activar la seguridad de mensajes para clientes de aplicación.

Para activar la seguridad de los mensajes para llamadas de servicios web que se originan en puntos finales desplegados, debe especificar un proveedor de cliente por defecto. Si ha activado un proveedor de cliente por defecto para GlassFish Server, debe asegurarse que los servicios llamados desde los puntos finales desplegados en GlassFish Server están configurados de forma compatible para la seguridad de la capa de mensajes.

  1. En el árbol de navegación, amplíe el nodo Configuración.

  2. Amplíe el nodo Seguridad.

  3. Seleccione el nodo Seguridad de Mensajes.

    Se abrirá la página Configuraciones de Seguridad de Mensajes.

  4. En la página Configuraciones de Seguridad de Mensajes, haga clic en el nombre de la capa de autenticación de la configuración que está editando.

    Por defecto, la única configuración de seguridad de mensajes configurada es SOAP.

    Se abrirá la página Editar Configuración de Seguridad de Mensajes.

  5. En la página Editar Configuración de Seguridad de Mensajes, en la lista desplegable Proveedores por Defecto, seleccione un proveedor de servidor.

    El proveedor de servidor por defecto, también llamado proveedor por defecto, identifica el proveedor de servidor llamado para una aplicación que no identifica un proveedor de servidor específico. Las opciones disponibles son ServerProvider, XWS_ServerProvider o ninguno. Por defecto no hay ningún proveedor seleccionado, por tanto GlassFish Server no llamará a un proveedor de seguridad de mensajes para el servidor. Generalmente, debe seleccionar ServerProvider como proveedor por defecto.

  6. En la lista desplegable Proveedor de Cliente por Defecto, seleccione un proveedor de cliente.

    El proveedor de cliente por defecto identifica el proveedor de cliente llamado para una aplicación que no identifica un proveedor de cliente específico. Las opciones disponibles son ClientProvider, XWS_ClientProvider o ninguno. Por defecto no hay ningún proveedor seleccionado, por tanto GlassFish Server no llamará a un proveedor de seguridad de mensajes para clientes. Generalmente, no debe seleccionar ninguno como proveedor de clientes por defecto. Seleccione ClientProvider para activar un proveedor por defecto y una política de protección de mensajes para aplicar a las llamadas a servicios web que se originan en los puntos finales de servicios web desplegados en GlassFish Server.

  7. Haga clic en Guardar.

  8. Para obtener instrucciones sobre cómo modificar las políticas de protección de mensajes de cliente activado o proveedores de servidores, consulte Editar una configuración de proveedor de seguridad de mensajes.
