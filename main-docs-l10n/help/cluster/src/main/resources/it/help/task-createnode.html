Per creare un nodo

Un nodo rappresenta un host in cui è installato il software di GlassFish Server. È necessario un nodo per ogni host in cui risiedono le istanze di GlassFish Server.

  1. Selezionare il nodo Nodi nell'albero di navigazione.

    Viene visualizzata la pagina Nodi.

  2. Nella pagina Nodi fare clic su Nuovo.

    Viene visualizzata la pagina Nuovo nodo.

  3. Nel campo Nome digitare il nome scelto per il nodo.

  4. Nell'elenco a discesa Tipo selezionare il tipo di nodo da creare.

    • Per abilitare il supporto delle comunicazioni sul protocollo remoto DCOM (Distributed Component Object Model) da parte del nodo, selezionare DCOM.

    • Per creare un nodo per il supporto della comunicazione su SSH (Secure Shell), selezionare SSH.

    • Per creare un nodo che non supporta le comunicazioni remote, selezionare CONFIG.

  5. Nel campo Host nodo immettere il nome dell'host rappresentato dal nodo.

    Questo campo può essere obbligatorio a seconda del tipo del nodo:

    • se il nodo è di tipo CONFIG, è possibile omettere il nome dell'host. Il nome dell'host può essere determinato al momento della creazione delle istanze che risiedono nel nodo.

    • Se il nodo è di tipo DCOM o SSH, è necessario specificare il nome dell'host. In caso contrario si verifica un errore.

  6. Nel campo Directory nodo digitare il percorso nella directory destinata a contenere le istanze di GlassFish Server create nel nodo.

    Se il nodo è di tipo CONFIG, lasciare vuoto questo campo. La directory del nodo può essere determinata al momento della creazione delle istanze che risiedono nel nodo.

    Se si specifica un percorso relativo, il percorso è relativo rispetto alla directory as-install, dove as-install è la directory di installazione di base del software di GlassFish Server nell'host.

  7. Nel campo directory di installazione digitare il percorso completo della directory padre della directory di installazione di base del software di GlassFish Server nell'host, ad esempio /export/glassfish3/.

    Se il nodo è di tipo CONFIG, lasciare vuoto questo campo. La directory di installazione può essere determinata al momento della creazione delle istanze che risiedono nel nodo.

  8. Completare la creazione del nodo come richiesto in base al tipo di nodo.

    1. Se il nodo è di tipo CONFIG, fare clic su Salva.

    2. Se il nodo è di tipo DCOM, attenersi alle istruzioni fornite in Per completare la creazione di un nodo DCOM.

    3. Se il nodo è di tipo SSH, attenersi alle istruzioni fornite in Per completare la creazione di un nodo SSH.
