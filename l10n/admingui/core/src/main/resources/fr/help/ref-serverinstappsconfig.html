Configuration des applications

Sur la page Configuration des applications, vous pouvez définir les propriétés de configuration des applications qui permettent de s'assurer que les modifications apportées aux applications déployées sont détectées et que les classes modifiées sont rechargées.

Recharger

Si cette option est activée, les classes modifiées sont rechargées dynamiquement. Le serveur recherche régulièrement la présence de modifications dans le répertoire de déploiement et redéploie l'application, automatiquement et dynamiquement, avec ces modifications. Cette fonction est utile dans un environnement de développement, car elle permet de tester rapidement les modifications du code. Cependant, dans un environnement de production, le rechargement dynamique peut diminuer les performances. De plus, à chaque rechargement, les sessions ouvertes durant cette période de transition ne sont pas valides. Le client doit redémarrer la session.

Intervalle d'interrogation de rechargement

Définit la fréquence à laquelle le serveur doit vérifier les éventuelles modifications dans les applications déployées.

Délai d'expiration de session d'administration

Définit le nombre de minutes d'inactivité devant s'écouler avant que la Console d’administration n'arrive à expiration et exige une reconnexion.

Déploiement automatique

Si cette option est activée, les applications placées dans le répertoire de déploiement automatique sont déployées automatiquement.

Intervalle d'interrogation de déploiement automatique

Définit la fréquence à laquelle le serveur doit vérifier la présence d'archives d'application ou de module dans le répertoire de déploiement automatique. La modification de l'intervalle d'interrogation n'affecte pas le temps de déploiement d'une application ou d'un module.

Délai d'expiration de la tentative de déploiement automatique

Définit le nombre de secondes pendant lesquelles la taille d'un fichier partiellement copié peut demeurer inchangée avant qu'une erreur ne se produise. La valeur par défaut est 30.

Lorsqu'un fichier est copié lentement dans le répertoire de déploiement automatique, il peut s'afficher avant d'être copié en entier, entraînant l'échec de la tentative de déploiement automatique de l'application. Dans ce cas, Enterprise Server tente de déployer l'application à nouveau. Si la taille du fichier partiellement copié demeure inchangée pendant le délai spécifié, une erreur se produit.

Répertoire de déploiement automatique

Identifie le répertoire dans lequel vous développez votre application de sorte que vous n'ayez pas besoin de copier les fichiers dans le répertoire de déploiement automatique par défaut.

Vérificateur

Si cette option est activée, le vérificateur est exécuté avant le déploiement. Le vérificateur examine la structure et le contenu du fichier. En général, la vérification de grosses applications prend du temps.

Précompiler

Si cette option est activée, les fichiers JavaServer PagesTM (JSPTM) sont précompilés. Si cette option est désactivée, les fichiers JSP sont compilés au moment de l'exécution lors de leur premier accès.

Autres propriétés
