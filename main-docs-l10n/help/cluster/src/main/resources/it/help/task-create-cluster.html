Per creare un cluster

Per cluster si intende una raccolta denominata di istanze di GlassFish Server che condividono le stesse applicazioni, risorse e informazioni di configurazione.

Un cluster fornisce alta disponibilità grazie a scalabilità, bilanciamento del carico e protezione contro gli errori. Quando si verifica un errore in un'istanza o in un host di un cluster, GlassFish Server rileva l'errore, reindirizza le richieste dall'istanza in stato di errore ad altre istanze nel cluster e ripristina la condizione della sessione utente. Poiché tutte le istanze presenti nel cluster dispongono delle stesse applicazioni e delle stesse risorse, un'istanza può eseguire il failover verso qualsiasi altra istanza nel cluster.

GlassFish Server consente di amministrare tutte le istanze in un cluster come unità singola da un solo host, indipendentemente dal fatto che le istanze risiedano sullo stesso host o su host diversi.

Prima di iniziare

Se si prevede di aggiungere istanze di GlassFish Server al cluster in fase di creazione, assicurarsi che i nodi in cui dovranno risiedere le istanze esistano già.

  1. Selezionare il nodo Cluster nell'albero di navigazione.

    Viene visualizzata la pagina Cluster.

  2. Nella pagina Cluster fare clic su Nuovo.

    Viene visualizzata la pagina Nuovo cluster.

  3. Nel campo Nome cluster digitare un nome per il cluster.

    Il nome deve soddisfare i requisiti riportati di seguito.

    • Il nome può contenere solo caratteri ASCII.

    • Il nome deve iniziare con una lettera, un numero o un carattere di sottolineatura.

    • Il nome può contenere solo i caratteri seguenti:

      • Lettere minuscole

      • Lettere maiuscole

      • Numeri

      • Trattino

      • Punto

      • Carattere di sottolineatura

    • Il nome deve essere univoco nel dominio e non deve essere il nome di un altro cluster, di una configurazione denominata, di un'istanza di GlassFish Server o di un nodo.

    • Il nome non deve essere domain, server né un'altra parola chiave riservata di GlassFish Server.

  4. Nell'elenco a discesa Configurazione selezionare la configurazione denominata che verrà utilizzata dal cluster.

    Nota:

    Se si seleziona default-config, è necessario specificare nel passo successivo che la configurazione non è condivisa. La configurazione default-config non può essere condivisa.

  5. Specificare se la configurazione che verrà utilizzata dal cluster è condivisa.

    • Se la configurazione non è condivisa, lasciare selezionata l'opzione Crea una copia della configurazione selezionata.

      La configurazione selezionata viene copiata al momento della creazione del cluster. Alla copia della configurazione viene assegnato il nome nome-cluster-config, dove nome-cluster è il nome del cluster che si sta creando.

    • Se la configurazione è condivisa, selezionare l'opzione Crea riferimento alla configurazione selezionata.

      Il cluster utilizzerà la configurazione denominata esistente specificata.

  6. Aggiungere le istanze di GlassFish Server al cluster.

    Nota:

    Per creare un'istanza in un nodo di tipo CONFIG che rappresenta un host remoto, è necessario utilizzare il comando secondario create-local-instance(1). Per questa operazione non è possibile utilizzare Console di amministrazione.

    È inoltre possibile aggiungere le istanze dopo la creazione del cluster, come descritto in Per aggiungere un'istanza di GlassFish Server a un cluster.

    Aggiungere ogni istanza attenendosi alla procedura riportata di seguito.

    1. Nell'area Istanze server da creare, fare clic su Aggiungi.

      Viene aggiunta una riga alla tabella delle istanze nel cluster.

    2. Nel campo Nome istanza, digitare un nome per l'istanza.

    3. Nel campo Peso, digitare un numero intero che rappresenta il peso, o fattore ponderale, per il bilanciamento del carico dell'istanza.

      Il fattore ponderale per il bilanciamento del carico determina la proporzione di tutte le richieste inviate al cluster che l'istanza dovrebbe elaborare.

      Ad esempio, in un cluster a due istanze può essere necessaria un'istanza per elaborare una di quattro richieste e un'altra istanza per elaborarne tre di quattro. In questa situazione, impostare su 1 il peso dell'istanza che elabora una delle quattro richieste e su 3 il peso dell'altra istanza. Se si preferisce utilizzare valori percentuali, impostare i valori di peso delle istanze rispettivamente su 25 e 75.

      Il valore di peso predefinito è 100.

    4. Nell'elenco a discesa Nodo selezionare il nodo in cui deve risiedere l'istanza.

  7. Fare clic su OK.

    Viene visualizzata di nuovo la pagina Cluster. Il cluster viene aggiunto alla tabella dei cluster della pagina.
