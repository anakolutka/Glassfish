もし、あなたが開発者で、Java EE 5 互換のアプリケーションサーバを実装した コミュニティの一員になるのに興味があるのでしたら、Project GlassFish に参加してはどうでしょう。

GlassFish コミュニティの参加者には私たちの Wiki の People's Page にできるだけ登録するようお願いしています。 あと、もう少し見栄えのよいページもあって、 こちらでは開発者にスポットライトを あて、拡大していこうとしています。

コードを理解する

アプリケーションサーバを開発するのは大変なことです。以下が、 この様な大きさのプロジェクト開発を組織するのに役立つリンクです。

  • GlassFish プロジェクトはモジュールに分割されています
  • 毎プロモーションビルド更新されるjavadoc を読んで下さい
  • 実装の詳細については他の GlassFish 開発者に メーリングリストで聞いて下さい。 メーリングリストを見るにはログインするのを忘れないように。
  • モジュールオーナーのリストをチェックして下さい
  • GlassFish の一部である関連テクノロジー もご覧下さい

ビルドする

GlassFish のビルドインフラは maven と ant を使用します。現在、私たちはこのインフラの ant への依存を減らすよう開発者に協力してもらっています。

  • クイックスタートガイド  はサーバーをビルドするのに必要な手順を簡単に載せています
  • 完全な手順 が GlassFish のビルドのしかたを詳細に載せています。

テストする

GlassFish には現在いくつかのテストがあります。QuickLook テスト は短時間で実行でき、アプリケーションサーバの広範な機能セットを カバーしています。開発者テストはモジュール別に用意されています。 それらのテストに関しては各モジュールに詳細な情報があります。

詳細については GlassFish Quality Portal をご覧下さい。

  • nightly promoted ビルドをダウンロード
  • クイックルックテスト を動かす
  • CATfish programに参加
  • バグを報告

コードを貢献する

GlassFish プロジェクトへの貢献のしかたは沢山あります: バグ修正、機能拡張、新機能、もしくはモジュールまるごとなど。

  • 貢献してもらえる方には全員、Contributor Agreementにサイン、返送してもらう必要があります。
  • 役割、責任に関しての詳細情報は governance policy をご覧下さい。
  • ワークスペースにチェックインされるコードは全て コーディング規約 に従う必要があります。
  • もしまだ developer ロールをお持ちでなければ、パッチ提供プロセス をご覧下さい。
  • 開発者がワークスペースを更新する際は、コミット手順に従う必要があります。

サンプルの貢献

GlassFish または Java EE 5 テクノロジーの機能や側面をデモする高品質のサンプルは歓迎です。 GlassFish Samples Project に参加して、あなたの貢献で違いを見せて下さい。

このページ、inviting samples contributions を見て下さい。 あと、アプリケーションのベストプラクティスを貢献するために、Java Blueprints project も見て下さい。

全ての貢献は Contributor Agreement にサインし返送する必要があります。

ドキュメント

高品質の製品を提供するには、ドキュメントサポートが必要です。 ドキュメントセット内のドラフト数冊は、レビューできます。

ドラフト状態の本に関する詳細は ドキュメント ページをご覧下さい。 また、様々な例などは growing list of tech tips and blogs をご覧下さい。どんなフィードバックも歓迎します。
