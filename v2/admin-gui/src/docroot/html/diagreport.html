To generate a diagnostic report

This page will generate a diagnostic report based on your preferences set in the Diagnostic Service page. To specify data you want to collect and include in the diagnostic report, see To configure the diagnostic service.

  1. Select Domain.
  2. Select Diagnostics tab.
  3. Define the report settings as follows:

    • Select the target for which you need the report from the following options:

      Entire domain

      Cluster

      Instance

      Node Agent

    • Description: Provide a unique description of this report.

    • Bug IDs: If you know the bug number related to the problem you are facing, enter it here.

    • Server Log Start Date: Enter a date in MM/DD/YY format.

      The date from which server.logfiles (if log rotation is enabled) for server instances are captured. If the log start date is not specified, number of entries from server.log file as specified by max-no-of-entries matching min-log-level in diagnostic service are collected.

    • Server Log End Date: Enter a date in MM/DD/YY format

      If specified, takes precedence over max-no-of-entries from diagnostic service configuration. The log end date must be specified if the log start date is specified If the log end date is not specified, max-no-of-entries from diagnostic service is used to limit the server.log content being captured.

  4. Click Generate Report.
  5. You will be prompted to save the generated report.
