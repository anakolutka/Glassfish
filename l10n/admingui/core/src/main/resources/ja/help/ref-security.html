セキュリティー

セキュリティーマネージャー

このオプションを選択すると、ドメインのセキュリティーマネージャーが有効になります。

有効にすると、Enterprise Server の JVM 設定に JVM オプション -Djava.security.manager が追加されます。「JVM オプションを設定する」を参照してください。この変更を有効にするには、サーバーを再起動する必要があります。

すべてのアプリケーションについて正しいアクセス権を与えていることを確認してください。セキュリティーマネージャーを無効にすると、パフォーマンスを向上させることができます

このオプションは、デフォルトでは無効になっています。

監査ログ

有効な場合、サーバーは「監査モジュール」設定で指定されたすべての監査モジュールのロードおよび実行を行います。無効な場合、サーバーは監査モジュールにアクセスしません。このオプションは、デフォルトでは無効になっています。

デフォルトレルム

サーバーが認証用に使用する有効な (デフォルト) レルム。アプリケーションでは、配備記述子によって別のレルムが指定されないかぎり、このレルムが使用されます。設定されているすべてのレルムがこのリストに表示されます。デフォルト値は「ファイル」です。

デフォルトの主体

デフォルトのユーザー名を指定します。サーバーでは、他の主体が指定されていないときにこのユーザー名が使用されます。このフィールドに値を入力する場合は、「デフォルト主体のパスワード」フィールドに対応するパスワードを入力します。

この属性は通常のサーバー動作には不要です。

デフォルト主体のパスワード

「デフォルト主体」フィールドで指定したデフォルト主体のパスワード。この属性は通常のサーバー動作には不要です。

監査モジュール

コンマで区切られている、監査モジュールプロバイダのクラスのリスト。ここで表示されるモジュールは事前に設定しておく必要があります。監査ログが有効な場合、このフィールドで監査モジュールが表示される必要があります。デフォルトで、サーバーは default という名前の監査モジュールを使用します。新しい監査モジュールを作成する方法については、「監査モジュールを作成する」を参照してください。

デフォルトのロールマッピングの主体

このオプションを選択すると、デフォルトのロールマッピングの主体がアプリケーション固有のマッピングを持たないアプリケーションに適用されます。

マッピングされた主体のクラス

java.security のカスタム実装を指定します。デフォルトのロールマッピングの主体で使用されている Principal クラスです。
