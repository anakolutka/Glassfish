Sintassi degli indirizzi dello schema MQ e del servizio MQ

La sintassi completa per l'indirizzo di un servizio di messaggi è

scheme://address-syntax

dove lo schema e la sintassi-indirizzo vengono descritti nella tabella riportata di seguito.

I valori Schema MQ e Servizio MQ sono quelli riportati nelle prime due colonne della tabella seguente.

Nome schema Servizio di connessione Descrizione Sintassi indirizzo
mq jms e ssljms Il runtime del client Message Queue stabilisce una connessione al port mapper MQ sull'host e sulla porta specificati. Il port mapper restituisce una lista di porte di servizio di connessione definite in modo dinamico, quindi il runtime del client Message Queue stabilisce una connessione alla porta host del servizio di connessione specificato. [nome host][:porta][/nome servizio]

Le impostazioni predefinite sono: nome host = localhost, porta = 7676, nome servizio = jms

Le impostazioni predefinite vengono applicate solo al servizio di connessione jms. Per il servizio di connessione ssljms è necessario specificare tutte le variabili.

Esempio: mq:MyHost:7677/ssljms
mqtcp jms Per stabilire una connessione, il runtime del client Message Queue esegue una connessione TCP all'host e alla porta specificati ignorando il port mapper MQ. nome host:porta/jms

Esempio: mqtcp:localhost:7676/jms
mqssl ssljms Per stabilire una connessione, il runtime del client Message Queue esegue una connessione SSL sicura all'host e alla porta specificati evitando il port mapper MQ. nomeHost:porta/ssljms

Esempio: mqssl:localhost:7676/ssljms
http httpjms Il runtime del client Message Queue esegue una connessione HTTP a un servlet di tunnel Message Queue nell'URL specificato. Il broker deve essere configurato per l'accesso al servlet di tunnel HTTP, come descritto nella Administrator's Guide di Message Queue. nome host:porta/radice contesto/tunnel

Se più istanze di broker utilizzano lo stesso servlet di tunnel, la sintassi per la connessione a un'istanza di broker specifica (anziché a una selezionata in modo casuale) è: http://nome host:porta/radice contesto/tunnel?nome server=nome host:nome istanza
https httpsjms IL runtime del client Message Queue esegue una connessione sicura HTTPS all'URL del servlet di tunnel Message Queue specificato. Il broker deve essere configurato per l'accesso al servlet di tunnel HTTPS, come descritto nella Administrator's Guide di Message Queue. nome host:porta/radice contesto/tunnel

Se più istanze di broker utilizzano lo stesso servlet di tunnel, la sintassi per la connessione a un'istanza di broker specifica (anziché a una selezionata in modo casuale) è: http://nome host:porta/radice contesto/tunnel?nome server=nome host:nome istanza
