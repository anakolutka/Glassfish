Définition des attributs de domaine

  1. Dans l'arborescence de navigation, sélectionnez le noeud Domaine.

    La page Attributs de domaine apparaît.

  2. Dans le champ Charger la console après le démarrage de DAS, sélectionnez l'une des options suivantes dans la liste déroulante :

    En fonction de l'utilisation

    Charge la Console d'administration après le serveur d'administration de domaine (DAS) si la console a été utilisée dans les dernières 24 heures ou si des instances de serveur autres que celles du serveur DAS existent. Il s'agit de la valeur par défaut.

    Toujours

    Charge toujours la Console d'administration après le serveur DAS. La console est chargée une fois la séquence de démarrage principale de GlassFish Server terminée.

    Jamais

    Ne charge jamais la Console d'administration après le serveur DAS. La console est chargée une fois l'accès à celle-ci demandé.

  3. Dans le champ Racine application, saisissez le chemin complet du répertoire dans lequel les applications déployées résident.

    La valeur par défaut est domain-dir/applications.

  4. Dans le champ Racine journal, saisissez le chemin complet du répertoire dans lequel les fichiers journaux du serveur sont conservés.

    La valeur par défaut est domain-dir/logs.

  5. Pour utiliser l'environnement local par défaut de l'hôte, ne renseignez pas le champ Environnement local.

    Un environnement local est un identificateur qui spécifie une combinaison particulière de langue et de région. Par exemple, l'environnement local pour l'anglais des Etats-Unis est en_US et pour le japonais, ja_JP. Pour utiliser un environnement local non anglophone, GlassFish Server doit être localisé, c'est-à-dire adapté de l'anglais en une autre langue.

  6. Cliquez sur Enregistrer.

  7. Redémarrez le serveur.
