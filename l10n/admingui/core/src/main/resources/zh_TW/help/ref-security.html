安全性

安全性管理員

若選取此選項，則會啟用網域的安全性管理員。

啟用之後，JVM 選項 -Djava.security.manager 會加入 Enterprise Server 的 JVM 設定。請參閱配置 JVM 選項。您必須重新啟動伺服器才能啟用此變更。

確保所有應用程式都已授予正確的權限。 您可以關閉安全性管理員以提昇效能。

此選項預設為停用。

稽核記錄

如果已啟用該功能，則伺服器將載入並執行所有在 [稽核模組] 設定中指定的稽核模組。如果已停用該功能，則伺服器不會存取稽核模組。此選項預設為停用。

預設範圍

伺服器用於認證的使用中 (預設) 範圍。除非應用程式的部署描述元指定了不同的範圍，否則應用程式會使用此範圍。所有已配置的範圍都會顯示在清單中。預設值為 File。

預設主體

指定預設使用者名稱。若未提供任何其他主體，伺服器會使用此使用者名稱。如果在此欄位中輸入值，請在 [預設主體密碼] 欄位中輸入對應的密碼。

該屬性對於一般伺服器作業並非必要。

預設主體密碼

在 [預設主體] 欄位中指定的預設主體之密碼。該屬性對於一般伺服器作業並非必要。

稽核模組

稽核模組提供者類別清單，由逗號分隔。此處列出的模組必須已經過配置。如果已啟用稽核記錄功能，則該欄位必須列出稽核模組。依預設，伺服器使用名為 default 的稽核模組。如需有關建立新稽核模組的資訊，請參閱建立稽核模組。

啟用預設主體至角色的對映

若選取此選項，即可將預設的主體至角色對映，套用於不具有應用程式專屬對映的應用程式。

對映的主體類別

指定用於預設主體至角色對映的 java.security。Principal 類別之自訂實作。

  • 授予對管理工具的存取權限
