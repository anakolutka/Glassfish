응용 프로그램 구성 페이지

응용 프로그램 구성 페이지에서는 배포된 응용 프로그램에 대한 변경 사항을 감지하고 수정된 클래스를 다시 로드하도록 도와 주는 등록 정보를 설정할 수 있습니다.

다시 로드

이 옵션을 사용하도록 설정된 경우 수정된 클래스가 동적으로 다시 로드됩니다. 서버는 배포 디렉토리에서 변경 사항을 정기적으로 확인하고 변경 사항과 함께 동적으로(자동으로) 응용 프로그램을 재배포합니다. 이 기능은 코드 변경을 빠르게 테스트할 수 있기 때문에 개발 환경에서 유용합니다. 그러나 프로덕션 환경에서는 동적 다시 로드가 성능을 저하시킬 수 있습니다. 또한 다시 로드가 수행될 때마다 해당 전송 시간의 세션이 무효화됩니다. 클라이언트가 세션을 다시 시작해야 합니다.

다시 로드 폴링 간격

서버에서 배포된 응용 프로그램의 변경 사항을 확인하는 빈도를 식별합니다.

관리 세션 시간 초과

관리 콘솔의 시간이 초과되어 다시 로그인하기까지의 비활성 시간(분)을 식별합니다.

자동 배포

이 옵션을 사용하도록 설정된 경우 자동 배포 디렉토리의 응용 프로그램이 자동으로 배포됩니다.

자동 배포 폴링 간격

서버가 자동 배포 디렉토리에 응용 프로그램 또는 모듈 아카이브가 있는지 확인하는 빈도를 지정합니다. 폴링 간격을 변경해도 응용 프로그램 또는 모듈 배포에 걸리는 시간은 영향을 받지 않습니다.

자동 배포 재시도 시간 초과

부분적으로 복사된 파일이 오류 발생 없이 크기가 변경되지 않고 그대로 유지될 수 있는 시간(초)을 지정합니다. 기본값은 30입니다.

파일이 자동 배포 디렉토리로 느리게 복사되는 경우 파일이 완전히 복사되기 전에 표시될 수 있습니다. 따라서 응용 프로그램을 자동으로 배포하려고 하면 실패합니다. 응용 프로그램의 자동 배포 시도가 이런 이유로 실패하면 Enterprise Server는 응용 프로그램을 다시 배포하려고 합니다. 부분적으로 복사된 파일의 크기가 지정된 기간 동안 변경되지 않고 유지되면 오류가 발생합니다.

자동 배포 디렉토리

파일을 기본 자동 배포 디렉토리로 복사할 필요가 없도록 응용 프로그램을 구축하는 디렉토리를 식별합니다.

검증자

이 옵션을 사용하도록 설정된 경우 검증자가 배포 전에 실행됩니다. 검증자는 파일의 구조와 내용을 검사합니다. 용량이 큰 응용 프로그램의 검증은 시간이 소요될 수 있습니다.

사전 컴파일

이 옵션을 사용하도록 설정된 경우 JSPTM(JavaServer PagesTM) 파일이 사전 컴파일됩니다. 이 옵션을 사용하지 않도록 설정된 경우 JSP 파일은 런타임에 처음으로 액세스될 때 컴파일됩니다.
