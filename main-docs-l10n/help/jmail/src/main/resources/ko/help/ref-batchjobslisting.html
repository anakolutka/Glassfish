일괄 처리 작업 실행

GlassFish Server에서는 일괄 처리 작업의 일정 잡기 및 실행을 위해 일괄 처리 런타임을 제공합니다. 일괄 처리 작업은 대개 여러 단계가 포함된, 오랫동안 실행되는 대량 작업입니다. 일괄 처리 응용 프로그램에서는 일괄 처리 런타임에 작업을 제출하고, 각 단계를 실행하는 방법 및 시기에 대한 지침을 제공합니다. 일괄 처리 런타임은 각 단계를 처리하고 작업에 대한 정보를 작업 저장소에 저장합니다. GlassFish Server에서 작업 저장소는 데이터베이스입니다.

[일괄 처리 작업 실행] 페이지를 사용하여 일괄 처리 작업에 대해 기록된 실행의 세부 정보를 볼 수 있습니다.

[일괄 처리 작업 실행] 페이지에는 다음 정보가 포함되어 있습니다.

인스턴스 이름
클러스터 이름

구성이 적용되는 GlassFish 클러스터 또는 서버 인스턴스 이름입니다.

일괄 처리 작업

인스턴스 또는 클러스터에 대해 기록된 일괄 처리 작업 실행 목록입니다. 각 실행에 대해 다음 정보가 표시됩니다.

실행 ID

일괄 처리 작업 실행에 지정된 ID입니다. 실행 ID를 누르면 해당 실행에 대한 [일괄 처리 작업 실행 세부 정보] 페이지가 열립니다. 새 실행은 작업이 처음 시작될 때, 그리고 기존 실행이 재시작될 때마다 생성됩니다.

작업 이름

작업의 이름입니다.

일괄 처리 상태

일괄 처리 런타임에 의해 설정된 실행 상태입니다.

종료 상태

해당 작업에 대한 작업 XML 또는 일괄 처리 응용 프로그램에 의해 설정된 실행 상태입니다. 종료 상태가 명시적으로 대체되지 않는 한 기본적으로 종료 상태와 일괄 처리 상태는 동일합니다.

인스턴스 ID

일괄 처리 작업의 인스턴스에 지정된 ID입니다. 일괄 처리 작업의 인스턴스에는 실행이 여러 개 포함될 수 있습니다.

시작 시간

실행이 시작된 시간입니다.

종료 시간

작업이 종료된 시간입니다.
