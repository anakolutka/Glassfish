Création d'un hôte JMS

  1. Dans l'arborescence de navigation, développez le noeud Configuration.

  2. Sélectionnez l'instance à configurer :

    • Pour configurer une instance spécifique, sélectionnez le noeud de configuration de l'instance. Par exemple, pour l'instance par défaut, server, sélectionnez le noeud server-config.

    • Pour configurer les paramètres par défaut des futures instances utilisant une copie de default-config, sélectionnez le noeud default-config.

  3. Sous le noeud Configuration, développez le noeud Service JMS.

  4. Sous le noeud Service JMS, sélectionnez le noeud Hôtes JMS.

    La page Hôtes JMS apparaît.

  5. Sur la page Hôtes JMS, cliquez sur Nouveau.

    La page Nouvel hôte JMS apparaît.

  6. Dans le champ Nom, saisissez le nom de l'hôte.

    Exemple :

    NewJmsHost

  7. Dans le champ Hôte, saisissez le nom ou l'adresse IP du système sur lequel l'hôte JMS sera exécuté (localhost ou le nom du système local ou distant).

  8. Dans le champ Port, indiquez le numéro de port du service JMS.

    Ne modifiez ce champ que si le service JMS à utiliser n'est pas exécuté sur le port par défaut. Le port par défaut est 7676.

  9. Dans les champs Nom utilisateur de l'administrateur et Mot de passe de l'administrateur, saisissez le nom utilisateur et le mot de passe du broker Message Queue.

    Ceux-ci diffèrent du nom utilisateur et du mot de passe GlassFish Server. Modifiez ces champs uniquement si les valeurs du broker Message Queue ont été modifiées à l'aide de la commande as-install-parent/mq/bin/imqusermgr. Les valeurs par défaut sont admin et admin.

  10. Cliquez sur OK.
