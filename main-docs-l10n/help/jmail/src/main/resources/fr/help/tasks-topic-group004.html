Configuration de l'exécution de batch

  1. Dans l'arborescence de navigation, sélectionnez le cluster ou l'instance de serveur GlassFish Server dont vous voulez visualiser les exécutions de traitement batch.

    La page Informations générales apparaît.

  2. Sélectionnez l'onglet Batch.

    La page Exécutions de traitement batch apparaît.

  3. Cliquez sur l'onglet Configuration.

    La page Configuration d'exécution de batch apparaît. L'exécution utilise un service exécuteur géré et une source de données pour exécuter les traitements batch.

  4. Dans la liste déroulante Nom de recherche de service exécuteur, sélectionnez le service exécuteur géré à utiliser pour fournir des threads aux traitements batch.

    Le service exécuteur géré par défaut est concurrent/__defaultManagedExecutorService. Le service exécuteur géré peut être modifié une fois qu'un traitement batch a été soumis sur l'exécution de batch sans incidence sur l'exécution du travail.

  5. Dans la liste déroulante Nom de recherche de source de données, sélectionnez la source de données à utiliser pour stocker les informations relatives aux travaux.

    La source de données par défaut est jdbc/__TimerPool. Ne modifiez pas la source de données une fois le premier traitement batch soumis à l'exécution de batch pour exécution. Si la source de données doit être modifiée, arrêtez et redémarrez le domaine, puis appliquez la modification avant le démarrage ou le redémarrage de tout travail. Cependant, une fois que la source de données a été modifiée, les informations stockées dans la source de données précédente deviennent inaccessibles.

  6. Cliquez sur Enregistrer.
