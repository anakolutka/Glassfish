Para editar la configuración HTTP para un protocolo

  1. En el árbol de navegación, amplíe el nodo Configuración.

  2. En el nodo Configuración, amplíe el nodo Configuración de Red.

  3. En el nodo Configuración de Red, seleccione el nodo Protocolos.

    Se abrirá la página Protocolos.

  4. En la página Protocolos, haga clic en el nombre del protocolo que está editando.

    Se abrirá la página Editar Protocolo.

  5. En la página Editar Protocolo, haga clic en el separador HTTP.

    Se abrirá la página HTTP.

  6. En el campo Nombre de Servidor, introduzca el nombre del host que se va a utilizar en las URL que el servidor envía al cliente.

    Este nombre es el nombre de alias si el servidor utiliza un alias. Si el servidor no utiliza un alias, deje vacío este campo.

  7. En la lista desplegable Servidor Virtual por Defecto, seleccione el servidor virtual asociado al protocolo.

    Utilice la página Servidores Virtuales para definir los servidores virtuales.

  8. En el campo Puerto de Redireccionamientos, introduzca el valor del puerto que se utilizará para redireccionar una solicitud a otro puerto.

    GlassFish Server redirige automáticamente la solicitud si se dan estas dos condiciones:

    • Este protocolo soporta otras solicitudes que no son SSL.

    • Se recibe una solicitud para la que una restricción de seguridad coincidente requiere transporte SSL.

    Si no se especifica un puerto de redireccionamientos, GlassFish Server utiliza el número de puerto especificado en la solicitud original.

  9. En el campo Número Máximo de Conexiones, introduzca el número máximo de conexiones persistentes que se van a mantener en el modo de mantenimiento de conexiones.

    El subsistema de mantenimiento de conexiones sondea periódicamente las conexiones inactivas.

  10. En el campo Timeout, introduzca el tiempo máximo en segundos que una conexión puede considerarse inactiva y conservarse en estado de mantenimiento de conexiones.

    Un valor de 0 o menos significa que las conexiones del mantenimiento de conexiones ese mantienen abiertas de forma indefinida. El valor por defecto es 30 segundos.

  11. Active la casilla de control de activación de timeout de carga para que la conexión de un servlet que lea bytes lentamente se cierre tras alcanzarse el límite indicado en Timeout de carga de conexión.

    Si esta opción se encuentra desactivada, las conexiones del servlet no sufren un timeout. Esta opción se activada por defecto.

  12. En el campo Timeout de Carga de Conexión, introduzca el timeout de carga en milisegundos.

    Este campo únicamente se aplica si se ha activado la casilla de control de desactivación de Timeout de Carga. El valor por defecto es 300000 milisegundos.

  13. En el campo Timeout de Solicitud, introduzca el número de segundos que deben transcurrir antes de que la solicitud sufra un timeout.

    Si la solicitud no se procesa antes de alcanzarse el valor de timeout, se ignorará la solicitud. El valor por defecto es 30 segundos.

  14. En el campo Tamaño de Buffer de Envío, introduzca el tamaño en bytes del buffer de envío.

    El valor por defecto es 8192 bytes.

  15. En el campo Longitud de Buffer de Cabecera, introduzca el tamaño en bytes del buffer que utilizan los threads de procesamiento de solicitudes para leer los datos de las solicitudes.

    El valor por defecto es 8192 bytes.

  16. En el campo Tamaño Máximo de Publicación, especifique el tamaño máximo en bytes de las acciones POST.

    El valor por defecto es 2097152 bytes.

  17. En el campo Codificación de URI, escriba el nombre del juego de caracteres utilizado para descodificar los URI de solicitudes recibidos.

    El valor debe ser un nombre de juego de caracteres IANA válido. El valor por defecto es UTF-8.

  18. En el campo Versión, introduzca la versión del protocolo HTTP utilizado.

    El valor por defecto es HTTP/1.1.

  19. En la lista desplegable Compresión, seleccione la compresión GZIP HTTP/1.1 para ahorrar ancho de banda de servidor.

    Las opciones disponibles son:

    on

    Comprime datos.

    off

    Desactiva la compresión.

    force

    Fuerza la compresión de datos en todos los casos.

    El valor por defecto es off.

  20. En el campo Tipos MIME Comprimibles, escriba una lista separada por comas de los tipos MIME para los que se utiliza la compresión HTTP.

    El valor por defecto es text/html,text/xml,text/plain.

  21. En el campo Tamaño Mínimo de Compresión, escriba el tamaño mínimo de un archivo cuando se le aplica compresión.

    Este valor debe configurarse si Compresión se define en on o force. El valor por defecto es 2048 bytes.

  22. En el campo Agentes de Usuario sin Compresión, escriba una lista separada por comas de expresiones regulares que coinciden con agentes de usuario de clientes HTTP para los que no se debe usar compresión.

    Por defecto, este valor es una cadena vacía.

  23. En el campo Agente de Usuario Restringido, escriba una lista de agentes de usuario restringidos a los que se les aplica compresión HTTP.

    Si no se ha especificado ningún agente de usuario, se aplica la compresión HTTP a todos los agentes de usuario. Por defecto, no hay agentes de usuario especificados.

  24. En el campo Tipo de Respuesta por Defecto, escriba una cadena que especifique el tipo de respuesta por defecto.

    El formato es una cadena separada por punto y coma que incluye el tipo de contenido, la codificación, el idioma y el juego de caracteres. El valor por defecto es text/html; charset=iso-8859-1.

  25. En el campo Tipo de Respuesta Forzada, escriba una cadena que especifique el tipo de solicitud utilizado en caso de que no haya ninguna asignación MIME disponible que coincida con la extensión del archivo.

    El formato es una cadena separada por punto y coma que incluye el tipo de contenido, la codificación, el idioma y el juego de caracteres. El valor por defecto es text/html; charset=iso-8859-1.

  26. En el campo Adaptador, escriba el nombre de clase del adaptador de recursos estático.

    El valor por defecto es com.sun.grizzly.tcp.StaticResourcesAdapter.

  27. Active la casilla de control de activación de soporte de Comet para activar el soporte de Comet del protocolo.

    Esta opción se activada por defecto.

  28. Active la casilla de control de activación de búsqueda de DNS para activar la búsqueda en el sistema de nombres de dominio (DNS).

    Esta opción se activada por defecto.

  29. Active la casilla de control de activación de soporte de RCM para activar la Gestión de Configuración de Recursos (RCM).

    Esta opción se activada por defecto.

  30. Active la casilla de control de activación de rastreo para activar la operación TRACE.

    Si esta opción se encuentra desactivada, GlassFish Server es menos susceptible de sufrir ataques de script entre sitios. Esta opción se encuentra activada por defecto.

  31. Active la casilla de control Traspaso de Autenticación para indicar que el listener de red que utiliza este protocolo recibe tráfico desde un servidor proxy con terminación SSL.

    Esta opción se activada por defecto.

  32. Active la casilla de control de activación de fragmentación para activar la fragmentación de la respuesta HTTP.

    Esta opción se encuentra activada por defecto.

  33. Active la casilla de control de activación de XPowered By para indicar a GlassFish Server que utilice las cabeceras de X-Powered-By conforme a las especificaciones de Java Servlet 3.0 y Java Server Pages 2.0.

    Esta opción se encuentra activada por defecto.

  34. Haga clic en Guardar.

    Sugerencia:

    Para volver a la configuración por defecto, haga clic en el botón Cargar Valores por Defecto y, a continuación, en Guardar.
