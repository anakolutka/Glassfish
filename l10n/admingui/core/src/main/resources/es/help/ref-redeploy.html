Volver a implementar aplicaciones o módulos

Utilice la página Volver a implementar aplicaciones o módulos para implementar de nuevo una aplicación previamente implementada.

La página Volver a implementar aplicaciones o módulos contiene las siguientes opciones.

Ubicación

Ubicación del nuevo archivo de almacenamiento para la aplicación que está volviendo a implementar.

Las siguientes opciones especifican desde dónde se puede acceder al archivo de almacenamiento y si es un archivo o directorio.

Archivo empaquetado que se cargará en el servidor

El archivo de almacenamiento se encuentra en un archivo que reside en la máquina cliente o al que se puede acceder desde la máquina cliente.

La máquina cliente es el host en el que está viendo la Consola de administración por medio de un navegador.

Directorio o archivo empaquetado local al que se puede acceder desde Servidor empresarial

El archivo de almacenamiento es un archivo que reside en la máquina servidor o es una aplicación desempaquetada en un directorio de despliegue.

El equipo servidor es el sistema anfitrión que está ejecutando el servidor de administración del dominio de Servidor empresarial.

Nombre de la aplicación

Campo de sólo lectura que muestra el nombre de la aplicación que está volviendo a implementar.

Conservar sesiones

Si está marcada la casilla de verificación Conservar sesiones, se conservarán las sesiones activas de la aplicación que está reimplementando y, a continuación, se restaurarán cuando la nueva implementación se haya completado.

Si en alguna sesión activa se produce algún fallo en la preservación o restauración de la aplicación, no habrá disponible ninguna sesión cuando se haya completado la nueva implementación. Sin embargo, la reimplementación continuará y se registrará una advertencia.

Para preservar las sesiones activas, Servidor empresarial serializa las sesiones y las guarda en memoria. Para restaurar las sesiones, el cargador de clases de la aplicación que se acaba de volver a implementar deserializa cualquier sesión que se hubiera guardado previamente.

Precompilar JSP

Si está seleccionada esta opción, los archivos JavaServer Pages ( JSP) se precompilarán. Si se deshabilita esta opción, los archivos JSP se compilarán en el tiempo de ejecución cuando se acceda a los mismos por primera vez. Esta opción se encuentra deshabilitada de forma predeterminada.

Ejecutar verificador

Si esta opción está seleccionada, el verificador se ejecutará antes de la implementación. Si se produce un error en la verificación, no se realiza la implementación. El verificador examinará la estructura y contenido del archivo. La verificación de aplicaciones de gran tamaño tiende a llevar mucho tiempo. Esta opción se encuentra deshabilitada de forma predeterminada.
