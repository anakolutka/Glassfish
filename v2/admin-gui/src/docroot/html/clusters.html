Working With Clusters

In the Clusters page, you can do the following operations:

  • View the following information about existing clusters:

    Name

    The name that was assigned to the cluster when the cluster was created. To configure a cluster, click the name of the cluster. For more information, see To Configure a Cluster.

    HTTP Load Balancer

    The name of the load balancer that performs HTTP load balancing for the cluster. A load balancer is assigned to a cluster during the configuration of the load balancer. For more information, see To Manage Load Balancer Targets. To change load balancer device settings, click the name of the load balancer. For more information, see To Change Load Balancer Device Settings.

    Configuration

    The configuration that was selected for the cluster when the cluster was created. To modify a configuration, click the name of the configuration. For more information, see Configuration.

    Instances

    The name and status of each server instance in the cluster. To configure a server instance, click the name of the server instance. For more information, see To Configure Server Instances in a Cluster.

  • Create a cluster by clicking New. The New Cluster page appears where you can create the cluster. For more information, see To Create a Cluster.

  • Start a cluster by clicking the check box next to the required cluster and clicking Start.

    Note - Before you start a cluster, ensure that the node agents are running. To start a node agent, use the adadmin command start-node-agent.

  • Stop a cluster by clicking the check box next to the required cluster and clicking Stop.

  • Remove a cluster by clicking the check box next to the required cluster and clicking Delete.
