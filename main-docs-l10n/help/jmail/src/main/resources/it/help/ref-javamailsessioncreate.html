Nuova sessione JavaMail

Utilizzare la pagina Nuova sessione JavaMail per creare una nuova risorsa sessione JavaMail.

La pagina Nuova sessione JavaMail contiene le opzioni riportate di seguito.

Nome JNDI

Il nome univoco che identifica la sessione di posta. Utilizzare il prefisso del contesto secondario di denominazione mail/ per le risorse JavaMail. Ad esempio: mail/MySession. Il nome deve contenere solo caratteri alfanumerici, caratteri di sottolineatura, lineette e punti.

Host posta

Il nome host del server di posta predefinito. Se non viene fornita una proprietà host specifica per un protocollo, i metodi di connessione degli oggetti Store e Transport utilizzano questo valore. Il nome deve essere risolvibile in un nome host effettivo.

Utente predefinito

Il nome utente da fornire durante la connessione a un server di posta. Se non viene fornita una proprietà nome utente specifica per un protocollo, i metodi di connessione degli oggetti Store e Transport utilizzano questo valore. Il nome utente deve contenere solo caratteri alfanumerici, caratteri di sottolineatura, lineette e punti.

Indirizzo mittente predefinito

L'indirizzo di posta elettronica dell'utente predefinito. Il formato da usare è nome utente@host.dominio.

Descrizione

La descrizione della sessione di posta corrente.

Stato

Lo stato della sessione di posta. La sessione di posta può essere abilitata o disabilitata. Lo stato predefinito è Abilitato.

Protocollo di memorizzazione

Il servizio del protocollo di memorizzazione che consente la connessione a un server di posta, il recupero dei messaggi e il salvataggio dei messaggi in una o più cartelle. I valori consentiti sono imap, pop3, imaps e pop3s. Il valore predefinito è imap.

Classe protocollo di memorizzazione

La classe di implementazione del provider di servizi per la memorizzazione. Di seguito sono riportati i valori consentiti.

  • com.sun.mail.imap.IMAPStore

  • com.sun.mail.pop3.POP3Store

  • com.sun.mail.imap.IMAPSSLStore

  • com.sun.mail.pop3.POP3SSLStore

Il valore predefinito è com.sun.mail.imap.IMAPStore.

Protocollo di trasporto

Il servizio del protocollo di trasporto che invia messaggi. I valori consentiti sono smtp e smtps. Il valore predefinito è smtp.

Classe protocollo di trasporto

La classe di implementazione del provider di servizi per il trasporto. Di seguito sono riportati i valori consentiti.

  • com.sun.mail.smtp.SMTPTransport

  • com.sun.mail.smtp.SMTPSSLTransport

Il valore predefinito è com.sun.mail.smtp.SMTPTransport.

Debug

Se questa opzione è selezionata, il debug per la risorsa corrente è abilitato. Se il livello di log JavaMail è impostato su FINE o FINER, l'output di debug viene generato e incluso nel file di log di sistema. Per informazioni sull'impostazione dei livelli del log, vedere Per configurare i livelli di log.

Proprietà aggiuntive

Proprietà aggiuntive per la sessione JavaMail. Per la lista delle proprietà disponibili, consultare la documentazione dell'interfaccia API JavaMail (http://java.sun.com/products/javamail/javadocs/index.html). Le proprietà aggiuntive devono iniziare con mail- e utilizzare i trattini, e non i punti, come separatori. Ad esempio, mail-debug è una proprietà aggiuntiva corretta, mentre mail.debug non lo è. Il processo backend di GlassFish Server converte i trattini nei punti previsti dall'interfaccia API JavaMail.

Destinazioni

I cluster e le istanze standalone per la sessione JavaMail. Per spostare le destinazioni desiderate nella colonna Destinazioni selezionate, utilizzare i pulsanti Aggiungi e Aggiungi tutto. Per spostare le destinazioni non necessarie nella colonna Destinazioni disponibili, utilizzare i pulsanti Rimuovi e Rimuovi tutto.

Questa opzione viene visualizzata solo se nel dominio sono stati creati cluster o istanze standalone.
