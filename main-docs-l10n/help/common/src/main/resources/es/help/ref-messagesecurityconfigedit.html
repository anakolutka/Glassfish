Editar configuración de seguridad de mensajes

Consulte la página Editar configuración de seguridad de mensajes para activar los proveedores de la seguridad de mensajes.

Para activar la seguridad de los mensajes para puntos finales de servicios web desplegados en GlassFish Server, debe especificar un proveedor para usar por defecto en el servidor. Si activa un proveedor por defecto para la seguridad de los mensajes, también tendrá que activar proveedores que utilizarán los clientes de los servicios web desplegados en GlassFish Server. Encontrará información sobre la activación de los proveedores usados por clientes en Activar la seguridad de mensajes para clientes de aplicación.

Para activar la seguridad de los mensajes para llamadas de servicios web que se originan en puntos finales desplegados, debe especificar un proveedor de cliente por defecto. Si ha activado un proveedor de cliente por defecto para GlassFish Server, debe asegurarse que los servicios llamados desde los puntos finales desplegados en GlassFish Server están configurados de forma compatible para la seguridad de la capa de mensajes.

La página Editar configuración de seguridad de mensajes contiene las siguientes opciones.

Nombre de Configuración

Nombre de la configuración a la que se aplican los valores de esta página. Este campo es de sólo lectura.

Capa de autenticación

Capa del mensaje en la que se lleva a cabo la autenticación. Éste es un campo de sólo lectura. El valor puede ser SOAP (para configuración de seguridad de mensajes por defecto) o HttpServlet.

Proveedor por defecto

Proveedor de servidor por defecto, si lo hubiera. El proveedor de servidor por defecto, también llamado proveedor por defecto, identifica el proveedor de servidor llamado para una aplicación que no identifica un proveedor de servidor específico. Las opciones disponibles son ServerProvider, XWS_ServerProvider o ninguno. Por defecto no hay ningún proveedor seleccionado, por tanto GlassFish Server no llamará a un proveedor de seguridad de mensajes para el servidor. Generalmente, debe seleccionar ServerProvider como proveedor por defecto.

Proveedor de cliente por defecto

Proveedor de cliente por defecto, si lo hubiera. El proveedor de cliente por defecto identifica el proveedor de cliente llamado para una aplicación que no identifica un proveedor de cliente específico. Las opciones disponibles son ClientProvider, XWS_ClientProvider o ninguno. Por defecto no hay ningún proveedor seleccionado, por tanto GlassFish Server no llamará a un proveedor de seguridad de mensajes para clientes. Generalmente, no debe seleccionar ninguno como proveedor de clientes por defecto. Seleccione ClientProvider para activar un proveedor por defecto y una política de protección de mensajes para aplicar a las llamadas a servicios web que se originan en los puntos finales de servicios web desplegados en GlassFish Server.

Comandos asadmin Relacionados

Utilice el comando set(1) para modificar la configuración de seguridad de mensajes.

  • Cómo especificar el proveedor de servidor por defecto:

    asadmin set --user admin-user --port admin-port 
server-config.security-service.message-security-config.SOAP.
default_provider=ServerProvider

  • Cómo especificar el proveedor de cliente por defecto:

    asadmin set --user admin-user --port admin-port 
server-config.security-service.message-security-config.SOAP.
default_client_provider=ClientProvider
