Para editar atributos avanzados del conjunto de conexiones JDBC

  1. En el árbol de navegación, expanda el nodo Recursos.
  2. En el nodo Recursos, seleccione el nodo Conjuntos de conexiones.

    Se abrirá la página Conjuntos de conexiones.

  3. En la página Conjuntos de conexiones, haga clic en el nombre del conjunto que desea editar.

    Se abrirá la página Editar conjunto de conexiones.

  4. En la página Editar conjunto de conexiones, haga clic en la ficha Avanzado.

    Se abrirá la página Editar atributos avanzados del conjunto de conexiones.

    Nota – El campo Nombre es un campo de sólo lectura. Sólo puede especificar el nombre del conjunto de conexiones al crear un nuevo conjunto de conexiones JDBC.

  5. En el campo Tiempo de espera de instrucción, introduzca el periodo de tiempo, en segundos, tras el que finalizarán las consultas que duren demasiado.

    Servidor empresarial establecerá QueryTimeout en las instrucciones creadas. Establezca el valor en -1 para deshabilitar esta opción. El valor predeterminado es -1.

  6. Seleccione la opción Ajustar objetos JDBC activados para que las aplicaciones obtengan objetos JDBC ajustados de Statement, PreparedStatement , CallableStatement, ResultSet, DatabaseMetaData.

    El valor predeterminado es Falso.

  7. En el campo Validar al menos una vez, introduzca el tiempo, en segundos, tras el cual se valida una conexión al menos una vez.

    Esto ayudará a reducir el número de solicitudes de validación para una conexión. Establezca el valor en 0 para deshabilitar la validación de una conexión. El valor predeterminado es 0.

  8. En el campo Tiempo de espera de pérdidas, introduzca el tiempo, en segundos, para realizar un seguimiento de las pérdidas de conexión de un conjunto de conexiones.

    Establezca el valor en 0 para deshabilitar el seguimiento de las pérdidas de conexión. El valor predeterminado es 0.

  9. Seleccione la opción Reclamo de pérdidas activado para restaurar las conexiones perdidas en el conjunto una vez haya finalizado el seguimiento de conexiones de pérdidas.
  10. En el campo Intentos de creación, introduzca el número de intentos que se realizarán si se produjera un error al crear una conexión nueva.

    Establezca el valor en 0 para que no se intente crear de nuevo una conexión. El valor predeterminado es 0.

  11. En el Intervalo de reintentos, introduzca el intervalo, en segundos, que hay entre dos intentos de creación de una conexión.

    El valor predeterminado es 10. Este atributo sólo se utiliza si el valor de Intentos de creación es mayor de 0.

  12. Selecciona la opción Asociación no activa habilitada para asociar de manera inactiva las conexiones cuando se desarrolla una operación en ellas.

    Las conexiones se disocian cuando la trasacción se completa y un método de componentes finaliza, lo que ayuda a reutilizar las conexiones físicas. Esta opción se encuentra deshabilitada de forma predeterminada.

  13. Seleccione la opción Alistamiento de conexiones no activas habilitado para alistar un recurso a la transacción sólo cuando se esté utilizando en ese instante por un método.
  14. Seleccione la opción Asociación con subprocesos activada para crear asociaciones entre las conexiones y los subprocesos.

    Cuando otra conexión necesita el mismo subproceso, puede volver a utilizar la conexión que ya está asociada con ese subproceso, sin necesidad de incurrir en la sobrecarga de coger otra conexión del conjunto.

    Esta opción se encuentra deshabilitada de forma predeterminada.

  15. Seleccione la opción Correspondencia de conexiones activada para habilitar la correspondencia de conexiones para el conjunto.

    Esta opción se puede deshabilitar si el administrador sabe que las conexiones del conjunto siempre serán homogéneas y, por tanto, para una conexión recogida del conjunto no se necesita establecer correspondencia mediante el adaptador de recursos. Esta opción se encuentra deshabilitada de forma predeterminada.

  16. En el campo Uso máximo de conexión, introduzca el número de veces que una conexión debe ser reutilizada por el conjunto.

    Una vez que una conexión se reutilice el número de veces especificado, se cerrará. Esto es útil, por ejemplo, para evitar pérdidas por instrucción. Establezca el valor en 0 para deshabilitar este atributo. El valor predeterminado es 0.
