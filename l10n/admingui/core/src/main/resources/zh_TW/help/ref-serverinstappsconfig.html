應用程式配置

使用 [應用程式配置] 頁面可設定特性，確保會偵測到對已部署應用程式所做的變更，並會重新載入修改的類別。

重新載入

若啟用此選項，則會動態重新載入修改的類別。伺服器會定期檢查部署目錄中的變更，然後自動且動態地重新載入已有變更的應用程式。動態重新載入在開發環境中非常有用，因為它能快速測試程式碼變更。但在生產環境中，動態重新載入可能會使效能降低。另外，每當重新載入完成時，轉換時間內的階段作業都會變得無效。用戶端必須重新啟動階段作業。

重新載入輪詢間隔

決定伺服器檢查已部署應用程式有無變更的頻率。

管理階段作業逾時

決定管理主控台逾時而必須重新登入之前，需等候的未使用分鐘數。

自動部署

若啟用此選項，自動部署目錄中的應用程式將會自動部署。

自動部署輪詢間隔

指定伺服器檢查自動部署目錄中有無應用程式或模組歸檔的頻率。變更輪詢間隔不會影響部署應用程式或模組所用的時間。

自動部署重試逾時

指定發生錯誤之前，部分複製的檔案可維持大小不變的秒數。預設值為 30。

若檔案緩慢地複製到自動部署目錄，則在複製整個檔案之前可能會出現該檔案。因此，嘗試自動部署應用程式會失敗。若嘗試自動部署應用程式因為此原因而失敗，Enterprise Server 會重新嘗試部署該應用程式。若部分複製的檔案在指定的時段內大小維持不變，即會出現錯誤。

自動部署目錄

指定您建立應用程式的目錄，如此一來即不需要將檔案複製到預設的自動部署目錄中。

檢驗器

若啟用此選項，則會在部署前執行檢驗器。檢驗器將檢查檔案的結構和內容。檢驗大型應用程式通常非常耗時。

預編譯

若啟用此選項，則會預編譯 JavaServer PagesTM (JSPTM) 檔案。若停用此選項，則首次存取 JSP 檔案時會於執行階段編譯這些檔案。

其他特性

指定其他特性。
