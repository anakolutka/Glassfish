配置 JVM 常规设置

Java 虚拟机 (Java Virtual Machine, JVM) 包含在 Java Platform, Standard Edition（Java SE 平台）中，该平台是 Enterprise Server 所需的软件。不正确的 JVM 设置将导致服务器无法运行，因此在更改这些设置时应当谨慎。

  1. 在导航树中，选择 "Application Server" 节点。
  2. 在“常规信息”页面上，单击“JVM 设置”选项卡。

    此时将打开“JVM 常规设置”页面。

  3. 在“Java 主目录”字段中，键入 Java SE 软件的安装目录的名称。

    Enterprise Server 依赖于 Java SE 平台。要验证此发行版是否支持您指定的 Java SE 软件版本，请参见《Sun GlassFish Enterprise Server v3 Prelude Release Notes 》。如果输入的目录名称不存在或输入不受支持的 Java SE 软件版本的目录，将无法启动 Enterprise Server。

  4. 在“Javac 选项”字段中，键入 Java 编程语言编译器的命令行选项。

    部署 Enterprise JavaBeansTM (EJBTM) 组件后，Enterprise Server 将运行编译器。

  5. 选中“启用调试”复选框，以便使用 Java 平台调试器体系结构 (Java Platform Debugger Architecture, JPDA) 启用调试。

    JPDA 供应用程序开发者使用。有关更多信息，请参见《Sun GlassFish Enterprise Server v3 Prelude Developer’s Guide 》。

  6. 在“调试选项”字段中，键入要传递给 JVM 的 JPDA 选项。

    仅当在上一步中启用调试时，此字段才适用。

  7. 在“RMI 编译选项”字段中，键入 rmic 编译器的命令行选项。

    部署 EJB 组件后，Enterprise Server 将运行 rmic 编译器。

  8. 在“字节码预处理程序”字段中，键入类名列表。

    使用逗号分隔类名。每个类都必须实现 com.sun.appserv.BytecodePreprocessor 接口。将按指定顺序调用这些类。

  9. 单击“保存”。
  10. 重新启动服务器。
