Modifier un connecteur JMX

La page Modifier un connecteur JMX permet de modifier le connecteur JMX conforme à la JSR 360.

Elle contient les options ci-après.

Nom de configuration

Nom de la configuration à laquelle s'appliquent les paramètres de cette page. Ce champ est en lecture seule.

Nom

Nom du connecteur JMX. Ce champ est en lecture seule.

Protocole JMX

Nom du protocole que ce connecteur JMX prend en charge. Ce champ est en lecture seule.

JMXServiceURL

URL du service JMX. Ce champ est en lecture seule.

Sécurité

Si la case Activé est cochée, la communication JMX est cryptée. Cette option est désactivée par défaut.

Adresse

Adresse IP du service de dénomination auprès duquel le stub de serveur de connecteur JMX est inscrit. Il ne s'agit pas du port du socket de serveur qui effectue la véritable communication JMX. Il s'agit de l'adresse de l'interface réseau à partir de laquelle le registre RMI est démarré. Si votre système est doté de plusieurs interfaces réseau, modifiez cette valeur afin de sélectionner une seule interface. La valeur par défaut est 0.0.0.0.

Port

Numéro du port sur lequel le service de dénomination (registre RMI) écoute les connexions client RMI. La seule fonction de ce service de dénomination consiste à télécharger les stubs RMI. Si le port par défaut est occupé, un port libre est utilisé à la place. Les valeurs valides sont comprises entre 1 et 65535. Sous les systèmes UNIX, la création de sockets qui écoutent sur les ports 1 à 1024 exige des privilèges de superutilisateur. La valeur par défaut est 8686.

Nom de domaine

Nom qui représente le domaine administratif spécial. Toutes les authentifications sont gérées par ce domaine. La valeur par défaut est admin-realm.

Autres propriétés

Propriétés supplémentaires pour le connecteur JMX. Plusieurs propriétés sont indiquées par défaut.
