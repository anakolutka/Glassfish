Para Editar um Módulo do Ciclo de Vida

  1. Na árvore de navegação, selecione o nó Módulos de Ciclo de Vida.

    A página Módulos de Ciclo de Vida será aberta.

  2. Na página Módulos de Ciclo de Vida, clique no nome do módulo de ciclo de vida a ser editado.

    A página Editar Módulo de Ciclo de Vida será aberta.

  3. No campo Nome da Classe, digite o nome totalmente qualificado do arquivo de classe do módulo de ciclo de vida.

    A classe deve implementar a interface com.sun.appserv.server.LifecycleListener.

  4. No campo Classpath, digite o classpath do módulo do ciclo de vida.

    O classpath especifica onde o módulo do ciclo de vida está localizado. O valor default é domain-dir/applications. Se o módulo já estiver neste diretório (isto é, no classpath do servidor), esse campo poderá ser deixado em branco.

  5. No campo Ordem de Carga, digite um valor especificando a ordem na qual esse módulo do ciclo de vida deve ser carregado na inicialização.

    Os módulos com valores inteiros menores de ordem de carga são carregados antes. O valor pode variar de 101 até o MAXINT do sistema operacional. Valores de 1 a 100 são reservados.

  6. No campo Descrição, digite uma descrição do módulo de ciclo de vida.

  7. Marque a caixa de seleção Status Ativado para ativar o módulo de ciclo de vida.

    Por default, a opção está ativada.

  8. Marque a caixa de seleção Inicialização de Instância de Prevenção no Caso de Falha ao Carregar para especificar que o servidor seja submetido a shutdown em caso de falha ao carregar o módulo do ciclo de vida.

    Por default, essa opção está desativada.

  9. A guia Alvos aparece se clusters ou instâncias de servidores standalone, além da instância default do servidor (server), existirem. Selecione a guia Alvos para exibir os alvos nos quais o módulo de ciclo de vida foi ativado. Consulte Para Exibir Alvos da Aplicação para obter instruções sobre como ativar ou desativar um módulo de ciclo de vida em um ou vários alvos.

  10. Clique em Salvar.
