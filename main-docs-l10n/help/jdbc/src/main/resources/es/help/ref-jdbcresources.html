Recursos JDBC

Un recurso (origen de datos) JNDI (Interfaz de Directorios y Nomenclatura de Java proporciona a las aplicaciones los medios de conexión a una base de datos. Normalmente, el administrador crea un recurso JDBC para cada base de datos a la que acceden las aplicaciones desplegadas en un dominio; sin embargo, se puede crear más de un recurso JDBC para una base de datos.

Las aplicaciones obtienen una conexión con la base de datos a partir de un pool de conexiones buscando un origen de datos en el árbol API de JNDI (Interfaz de Directorios y Nomenclatura de Java) y, posteriormente, solicitando una conexión. El pool de conexiones asociado al origen de datos proporciona la conexión a la aplicación.

Utilice la página Recursos JDBC para configurar los recursos de JDBC.

Para cada recurso, aparece la siguiente información.

Nombre JNDI

Nombre único que identifica el recurso JDBC.

Activado

True si el recurso está activado o false, si no lo está.

Pool de conexiones

Pool de conexiones JDBC asociado al recurso.

Descripción

Breve descripción del recurso.

La tabla Recursos también contiene las siguientes opciones.

Nuevo

Botón para crear un nuevo recurso.

Suprimir

Botón para suprimir uno o varios recursos seleccionados.

Activar

Botón para activar uno o varios recursos seleccionados.

Desactivar

Botón para desactivar uno o varios recursos seleccionados.
