Si vous êtes développeur et que vous souhaitez faire partie de la communauté ayant implémenté le premier serveur d'application compatible Java EE 5, alors rejoignez le projet GlassFish.

Nous encourageons les membres de la communauté GlassFish à s'enregistrer sur la page des membres autonome de notre site wiki. Nous proposons également une autre page, conviviale et en constante évolution, utilisée pour présenter les développeurs.

Compréhension du code

Le développement d'un serveur d'application est un processus complexe. Voici une liste des liens nous aidant à organiser la mise en œuvre d'un projet d'une telle envergure.

  • Le projet GlassFish se divise en modules
  • Consultez les documents Java mis à jour pour chaque version promue
  • Discutez avec d'autres développeurs de GlassFish à propos des détails d'implémentation sur nos listes de diffusion. N'oubliez pas de vous identifier pour pouvoir visualiser toutes les listes de diffusion disponibles.
  • Consultez la liste des propriétaires de module
  • Découvrez les technologies associées faisant partie du projet GlassFish

Construction

L'infrastructure de construction de GlassFish utilise Maven et Ant. Nous recherchons actuellement des développeurs pouvant nous aider à améliorer l'infrastructure de construction de manière à être moins dépendant de Ant.

  • Le guide de démarrage rapide propose une courte liste d'étapes nécessaires à la construction du serveur
  • Les instructions complètes présentent en détails la procédure de construction de GlassFish

Test

GlassFish propose plusieurs tests de différents types. Les tests QuickLook sont rapides et couvrent un vaste ensemble de fonctionnalités dans le serveur d'application. Des tests développeur sont également disponibles pour les différents modules. Chaque module comporte de plus amples informations sur le mode d'exécution de ces tests.

Pour plus d'informations, reportez-vous au portail qualité de GlassFish.

  • Téléchargez un nightly build ou une version promue
  • Exécutez les tests QuickLook
  • Rejoignez le programme CATfish
  • Signalez un bogue

Participation au code

Vous pouvez participer au projet GlassFish de diverses manières : en apportant des résolutions de bogues, des améliorations, de nouvelles fonctionnalités, voire même des modules complets.

  • Tous les intervenants doivent signer et retourner un accord de participation
  • Consultez notre politique de gouvernance pour obtenir de plus amples informations sur les rôles et responsabilités
  • Le code extrait dans l'espace de travail doit respecter les conventions de code
  • Si vous ne disposez pas encore de rôle développeur, reportez-vous au processus de transfert de patch
  • Les développeurs doivent respecter les procédures de validation lors de la mise à jour de l'espace de travail

Participation aux exemples

Nous sommes toujours heureux de recevoir des exemples de bonne qualité démontrant une fonctionnalité ou un aspect de GlassFish ou de la technologie Java EE 5. Rejoignez le projet d'exemples de GlassFish et observez les effets positifs de votre participation.

Consultez cette page pour découvrir les besoins d'exemple dans différents domaines Reportez-vous également au projet Java Blueprints pour observer les diverses contributions à une meilleure utilisation des applications.

Documentation
