Nueva escucha HTTP (paso 1 de 2)

La Nueva escucha HTTP (paso 1 de 2) contiene las siguientes opciones:

Nombre

Un identificador exclusivo para la nueva escucha.

Escucha

Introduzca una dirección IP válida para el servidor.

Seguridad

Disponible en clústeres y perfiles empresariales. Crea una escucha segura.

Dirección de red

Introduzca 0.0.0.0 para la dirección de red si desea que la escucha atienda a todas las direcciones IP del servidor mediante un valor de puerto exclusivo.

Puerto de escucha

Introduzca un valor de puerto de escucha exclusivo si el campo Dirección de red es 0.0.0.0 o el valor de puerto deseado si utiliza otra dirección IP.

Servidor virtual predeterminado

Seleccione un servidor virtual en la lista desplegable.

Nombre de servidor

Nombre de host que se debe utilizar en las URL que el servidor envía al cliente. Este nombre es el alias si el servidor hace uso de un nombre de alias. Si el servidor no utiliza un alias, deje el campo en blanco.

Redireccionar puerto

Introduzca un valor de puerto para redireccionar la solicitud a otro puerto. Servidor empresarial redirecciona automáticamente la solicitud si se dan las dos condiciones siguientes:

  • La escucha admite otras solicitudes que no son SSL.

  • Se recibe una solicitud para la que una restricción de seguridad coincidente requiere transporte SSL.

    Servidor empresarial utiliza el número de puerto especificado en la solicitud original de forma predeterminada.

Subprocesos del aceptador

Número de subprocesos del aceptador. El valor predeterminado es 1.

Funciona con

Desactive la casilla para deshabilitar la inclusión del encabezado X-Powered-By: Servlet/2.4 en encabezados de respuesta HTTP generados por servlet.

La especificación Java Servlet 2.4 define este encabezado, que los contenedores pueden agregar a respuestas generadas por Servlet. De modo similar, la especificación JavaServer PagesTM (JSPTM) 2.0 define un encabezado X-Powered-By: JSP/2.0 para que se agregue (de forma opcional) a respuestas que utilizan la tecnología JSP. La inclusión del encabezado X-Powered-By: JSP/2.0 está activada de forma predeterminada para las aplicaciones web. El objetivo de estos encabezados es permitir a los administradores de sitios web recabar datos estadísticos sobre el uso de Servlet y la tecnología JSP.

Para obtener más información sobre cómo habilitar y deshabilitar el encabezado X-Powered-By para páginas JSP, consulte el Apéndice A, Deployment Descriptor Files de Sun GlassFish Enterprise Server v3 Prelude Application Deployment Guide.

Los entornos de producción pueden decidir omitir la generación de encabezados X-Powered-By para ocultar su tecnología subyacente.

Bloqueo

Permite habilitar el bloqueo de escuchas y puertos externos.
