環境服務

環境服務主要用於建立動態代理物件，這些物件可擷取容器的環境，而且可讓應用程式在稍後於該環境內執行。系統會將容器的環境傳輸至執行工作的繫線。

使用「環境服務」頁面可組態環境服務資源。

對於每個環境服務資源，均提供以下資訊。

JNDI 名稱

可識別資源的唯一名稱。

邏輯 JNDI 名稱

資源的邏輯 JNDI 名稱。請參閱 Logical JNDI Names 以瞭解詳細資訊。

已啟用

(網域中有預設伺服器執行處理 server 存在時才會顯示此欄。)

如果已啟用資源則為勾選記號，如果停用資源則為打叉記號 (X)。

狀態

(已在網域中建立叢集或獨立執行處理時才會顯示此欄。)

設定為資源目標的叢集和獨立執行處理數目，以及資源中啟用的目標數。例如，若為「已在 2 個 (共 4 個) 目標上啟用」，表示資源的目標有四個叢集和獨立執行處理，而且已在這四個目標的其中兩個啟用資源。

環境資訊

傳輸至繫線的容器環境。

描述

資源的描述。

「環境服務」表格也包含下列選項。

新建

建立新資源的按鈕。

刪除

刪除一或多個所選取資源的按鈕。

啟用

啟用一或多個所選取資源的按鈕。

停用

停用一或多個所選取資源的按鈕。
