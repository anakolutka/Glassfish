Configurer les paramètres généraux du profileur JVM

La machine virtuelle Java (JVM) est incluse dans le logiciel Java Platform Standard Edition (J2SETM), nécessaire à l'exécution d'Enterprise Server. La saisie de paramètres JVM incorrects empêchant l'exécution du serveur, vous devez être prudent(e) lors de la modification de ces paramètres.

  1. Dans l'arborescence de navigation, sélectionnez le nœud Application Server.
  2. Dans la page Informations générales, cliquez sur l'onglet Paramètres JVM.

    La page Paramètres JVM s'ouvre.

  3. Dans le champ Accueil Java, saisissez le nom du répertoire d'installation du logiciel Java SE.

    Enterprise Server fonctionne avec la plate-forme Java SE. Pour vérifier que la version du logiciel Java SE spécifié est prise en charge par cette version, consultez les Sun GlassFish Enterprise Server v3 Prelude Release Notes . Si vous saisissez un nom de répertoire inexistant ou le répertoire d'une version non prise en charge du logiciel Java SE, Enterprise Server ne démarrera pas.

  4. Dans le champ Options Javac, saisissez les options de ligne de commande du compilateur du langage de programmation Java.

    Enterprise Server exécute le compilateur lorsque des composants Enterprise JavaBeansTM (EJBTM) sont déployés.

  5. Cochez la case Débogage activé pour activer le débogage avec JPDA (Java Platform Debugger Architecture).

    L'architecture JPDA est utilisée par les développeurs d'applications. Pour plus d'informations, consultez le Sun GlassFish Enterprise Server v3 Prelude Developer’s Guide .

  6. Dans le champ Options de débogage, saisissez les options JPDA à transmettre à la JVM.

    Ce champ ne s'applique que si le débogage a été activé à l'étape précédente.

  7. Dans le champ Options de compilation RMI, saisissez les options de ligne de commande du compilateur rmic.

    Enterprise Server exécute le compilateur rmic lorsque des composants EJB sont déployés.

  8. Dans le champ Préprocesseur du bytecode, saisissez la liste des noms de classe.

    Séparez les noms de classe par des virgules. Chaque classe doit implémenter l'interface com.sun.appserv.BytecodePreprocessor. Les classes sont appelées par ordre d'énumération.

  9. Cliquez sur Enregistrer.
  10. Redémarrez le serveur.
