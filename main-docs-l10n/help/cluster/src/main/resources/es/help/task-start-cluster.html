Para iniciar un cluster

Al iniciar un cluster se inician todas las instancias de GlassFish Server del cluster que no estén aún en ejecución.

Antes de Empezar

Asegúrese de que se cumplan los siguientes requisitos previos:

  • Todos los nodos en los que residen las instancias del cluster están activados para la comunicación remota.

  • El usuario del Servidor de Administración de Dominio (DAS) puede usar el protocolo remoto de Distributed Component Object Model (DCOM) o Secure Shell (SSH) para conectarse al host para cualquier nodo en el que residan las instancias del cluster.

Si no se cumplen algunos de estos requisitos previos, inicie el cluster iniciando cada instancia de forma individual desde la línea de comandos mediante el subcomando start-local-instance(1).

  1. En el árbol de navegación, seleccione el nodo Clusters.

    Se abrirá la página Clusters.

  2. En la página Clusters, active la casilla de control que aparece junto al nombre del cluster.

  3. Haga clic en Iniciar Cluster.

    Se le pedirá que confirme que desea iniciar el cluster.

  4. Confirme que desea iniciar el cluster.

    Se inicia el cluster. El estado de cada instancia del cluster se actualiza en la página Clusters cuando se inicia la instancia.
