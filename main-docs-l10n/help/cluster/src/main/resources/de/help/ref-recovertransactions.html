Transaktionen wiederherstellen

Auf der Seite "Transaktionen wiederherstellen" können Sie Transaktionen für eine GlassFish Server-Instanz wiederherstellen.

Wenn die Instanz ausgeführt wird, enthält die Seite "Transaktionen wiederherstellen" nur den Text Recovery is performed by the running server itself (Die Wiederherstellung wird vom Server selbst durchgeführt) und die Schaltfläche "OK".

Wenn die Instanz nicht ausgeführt wird, enthält die Seite "Transaktionen wiederherstellen" die folgenden Optionen.

Instanzname

Der Name, der der Instanz bei der Erstellung zugewiesen wurde. Dieses Feld ist schreibgeschützt.

Zielserver

Eine weitere Instanz, die die Transaktionen in der aktuellen Instanz wiederherstellen kann.

Transaktionslogverzeichnis

Das Verzeichnis, in dem Transaktionslogs gespeichert sind. Standardmäßig handelt es sich um /logs/tx unter dem Domainverzeichnis.

Transaktionslogs werden im Unterverzeichnis tx des Verzeichnisses gespeichert, das durch die Einstellung "Transaktionslogverzeichnis" des Transaktionsservice für die Clusterkonfiguration angegeben ist. Der Standardwert für "Transaktionslogverzeichnis" ist das Verzeichnis, das auf der Seite "Domainattribute" im Feld "Log-Root" angegeben ist, das sich im Knoten GlassFish Server in der Registerkarte "Erweitert" befindet.

Verwandte Aufgaben

  • So werden Transaktionen für eine geclusterte GlassFish Server-Instanz wiederhergestellt

  • So werden Transaktionen für eine Standalone-GlassFish Server-Instanz wiederhergestellt

Entsprechende asadmin-Unterbefehle

recover-transactions(1)
