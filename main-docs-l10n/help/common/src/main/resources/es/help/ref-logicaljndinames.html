Nombres de JNDI lógicos

Cuando cree un recurso, se especifica un único nombre de JNDI para el recurso. Las aplicaciones utilizan este nombre para acceder al recurso.

El estándar de Java EE especifica que se deben poner a disposición de las aplicaciones determinados recursos por defecto y define nombres concretos de JNDI para estos recursos por defecto. GlassFish Server pone estos nombres a disposición de las aplicaciones a través del uso de nombres de JNDI lógicos, que asignan nombres de JNDI estándar de Java EE a determinados recursos de GlassFish Server de la siguiente forma:

java:comp/DefaultContextService

Este nombre estándar de Java EE se asigna al recurso concurrent/__defaultContextService.

java:comp/DefaultManagedThreadFactory

Este nombre estándar de Java EE se asigna al recurso concurrent/__defaultManagedThreadFactory.

java:comp/DefaultManagedExecutorService

Este nombre estándar de Java EE se asigna al recurso concurrent/__defaultManagedExecutorService.

java:comp/DefaultManagedScheduledExecutorService

Este nombre estándar de Java EE se asigna al recurso concurrent/__defaultManagedScheduledExecutorService.

java:comp/DefaultDataSource

Este nombre estándar de Java EE se asigna al recurso jdbc/__default.

java:comp/DefaultJMSConnectionFactory

Este nombre estándar de Java EE se asigna al recurso jms/__defaultConnectionFactory.
