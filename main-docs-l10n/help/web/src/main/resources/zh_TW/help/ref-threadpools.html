繫線集區

使用「繫線集區」頁面可組態繫線集區。

使用繫線集區可將服務限制在特定數目的並行運作繫線。

當您第一次安裝 GlassFish Server 時，預設已存在兩個繫線集區。名稱為 http-thread-pool 的繫線集區是為供網路監聽程式使用而組態，而 thread-pool-1 是為供 ORB 針對 RMI/IIOP 請求使用而組態。您可能需要建立具有不同設定的繫線集區，以供特定種類應用程式和特定硬體系統使用，或視您的需要調校預設繫線集區。

Java 虛擬機器 (JVM) 可以支援一次執行多個繫線。為了提昇效能，GlassFish Server 可維護一個或多個繫線集區。您可以將特定的繫線集區指定給特定的服務。

一個繫線集區可供多個服務使用。請求繫線會處理使用者請求。伺服器收到請求時，它會將該請求指定給繫線集區中的空閑繫線。繫線會執行從屬端請求，並傳回結果。例如，如果請求需要使用目前被佔用的系統資源，則此繫線將等待，直至此資源可用時，才允許請求使用此資源。

指定保留給請求的最大繫線數和最小繫線數。可以在這兩個值之間，動態調整繫線集區。指定的最小繫線集區大小，將要求伺服器配置至少這個數量的保留繫線給請求。該數目可以增加到所指定的最大繫線集區大小。

增加處理作業可用的繫線數目，可讓處理作業同時回應更多的請求。

對於每個繫線集區，均提供以下資訊。

繫線集區 ID

繫線集區的名稱。

最大繫線集區大小

繫線集區中繫線的最大數目。

最小繫線集區大小

繫線集區中繫線的最小數目。這些繫線是在實例化繫線集區時建立的。

佇列最大大小

在有繫線可處理訊息之前，可排入佇列的最大訊息數目。

閒置繫線逾時

從集區中移除閒置繫線之前經過的時間 (以秒為單位)。

繫線集區表也包含下列選項。

新建

建立新繫線集區的按鈕。

刪除

刪除一或多個選取繫線集區的按鈕。
