Per usare il modulo di audit predefinito

Il modulo di audit default registra le richieste di autenticazione e autorizzazione nel file di log del server. Per informazioni sulla modifica della posizione del file di log, vedere Per configurare le impostazioni generali di log.

Le voci del log di autenticazione includono le informazioni seguenti:

  • I nomi degli utenti che hanno tentato l'autenticazione

  • Il realm che ha elaborato la richiesta di accesso

  • Il componente URI o EJB del modulo Web richiesto

  • L'esito positivo o negativo delle richieste

Indipendentemente dall'abilitazione del log di audit, GlassFish Server registra tutti gli eventi di autenticazione negata.

Le voci del log di autorizzazione includono le informazioni seguenti:

  • I nomi degli eventuali utenti autenticati

  • Il componente URI o EJB Web richiesto

  • L'esito positivo o negativo delle richieste

  1. Espandere il nodo Configurazione nell'albero di navigazione.

  2. Selezionare l'istanza da configurare:

    • Per configurare un'istanza particolare, espandere il nodo di configurazione specifico dell'istanza. Ad esempio, per l'istanza predefinita server, espandere il nodo server-config.

    • Per configurare le impostazioni predefinite per tutte le istanze, espandere il nodo default-config.

  3. Nel nodo Configurazione selezionare il nodo Sicurezza.

    Viene visualizzata la pagina Sicurezza.

  4. Nell'elenco a discesa Moduli di audit selezionare il modulo di audit default.

  5. Fare clic su Salva.
