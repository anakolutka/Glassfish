Pools de Conexiones

Un pool de conexiones JDBC contiene un grupo de conexiones JDBC que se crean una vez registrado el pool de conexiones. Los pools de conexiones utilizan un controlador JDBC para crear conexiones de bases de datos físicas. La aplicación pide prestada una conexión al pool, la utiliza y, a continuación, se la devuelve al pool cerrándola. Para crear un recurso JDBC, especifique el pool de conexiones asociado. Varios recursos JDBC pueden especificar un único pool de conexiones.

Un pool de conexiones JDBC es un grupo de conexiones reutilizables para una determinada base de datos. Como la creación de cada nueva conexión física lleva mucho tiempo, el servidor mantiene un pool de conexiones disponibles para mejorar el rendimiento. Cuando una aplicación solicita una conexión, obtiene una del pool. Cuando una aplicación cierra una conexión, ésta se devuelve al pool.

Cuando se crea un pool de conexiones con la Consola de administración, está definiendo los aspectos de una conexión con una base de datos específica.

Utilice la página Pools de conexiones JDBC para configurar pools de conexiones JDBC.

Para cada pool de conexiones se proporciona la siguiente información.

Nombre de pool

Nombre del pool de conexiones.

Tipo de recurso

Tipo de recurso del pool de conexiones.

Nombre de clase

Nombre de clase específico del proveedor que implanta las API javax.sql.XADataSource, java.sql.ConnectionPoolDataSource, javax.sql.DataSource o java.sql.Driver.

Descripción

Breve descripción del pool de conexiones.

La tabla Pools también contiene las siguientes opciones.

Nuevo

Botón para crear un nuevo pool de conexiones.

Suprimir

Botón para suprimir uno o más pools de conexiones seleccionados.
