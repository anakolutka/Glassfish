Instancias de servidor en cluster

Utilice la página Instancias de servidor en cluster para crear y gestionar instancias de GlassFish Server en el cluster actual.

El campo Nombre de cluster es un campo de sólo lectura que muestra el nombre del cluster actual.

La página Instancias de servidor en cluster muestra una lista de las instancias que son miembros del cluster actual. Para cada instancia se proporciona la información siguiente.

Nombre

Nombre que se asignó a la instancia en el momento de su creación. Al hacer clic en el nombre se abrirá la página Información general de la instancia.

Peso de equilibrador de carga

Entero que representa el peso de equilibrio de carga de la instancia.

El peso de equilibrio de carga determina la proporción de todas las solicitudes con respecto al cluster que debe procesar la instancia.

Por ejemplo, en un cluster de dos instancias, puede necesitar que una instancia procese una solicitud de cuatro, y que otra procese las otras tres. En este caso, configure el peso de la instancia que debe procesar una de las cuatro solicitudes como 1 y el peso de la otra instancia como 3. Si prefiere usar porcentajes, defina los pesos de las instancias como 25 y 75, respectivamente.

El peso por defecto es 100.

Configuración

Configuración a la que hace referencia la instancia. Al hacer clic en la configuración se abrirá la página Configuraciones para la instancia.

Nodo

Nombre del nodo en el que reside la instancia. Al hacer clic en el nombre se abrirá la página Editar nodo del nodo.

Estado

Indicación de si la instancia se está ejecutando, si es necesario reiniciar para que se actualice con cambios de configuración o si no se está ejecutando. Este campo es de sólo lectura.

La tabla Instancias de servidor también contiene las opciones siguientes.

Nuevo

Botón para agregar una instancia al cluster.

Suprimir

Botón para eliminar una o varias instancias seleccionadas del cluster.

Atención:

Si utiliza un cluster Java Message Service (JMS) con un broker maestro, no suprima la instancia asociada al broker maestro. Si esta instancia se debe suprimir, utilice el subcomando change-master-broker(1) para asignar el broker maestro a una instancia diferente.
Iniciar

Botón para iniciar una o varias instancias seleccionadas en el cluster.

Parar

Botón para parar una o varias instancias seleccionadas en el cluster.
