Anwendungen

Die Seite "Anwendungen" zeigt eine Liste von Anwendungen an, die im GlassFish Server eingesetzt werden. Sie können die bereits bereitgestellten Anwendungen anzeigen und verwalten, und Sie können weitere Anwendungen bereitstellen.

Für jede Anwendung werden die folgenden Informationen zur Verfügung gestellt.

Name

Der Name der Anwendung.

Der Name kann eine optionale Versions-ID enthalten, die dem Namen folgt und von diesem durch einen Doppelpunkt (:) getrennt ist. Weitere Informationen zu Modul- und Anwendungsversionen finden Sie unter Module and Application Versions im Oracle GlassFish Server Application Deployment Guide.

Deployment-Reihenfolge

Die Deployment-Reihenfolge der Anwendung.

Anwendungen mit einer niedrigeren Nummer werden beim Hochfahren des Servers zuerst geladen. Eine Anwendung mit einer Deployment-Reihenfolge von 102 wird vor einer Anwendung mit einer Deployment-Reihenfolge von 110 geladen. Wenn beim Deployment einer Anwendung keine Deployment-Reihenfolge angegeben wird, wird die Standard-Deployment-Reihenfolge von 100 zugewiesen. Wenn zwei Anwendungen dieselbe Deployment-Reihenfolge haben, wird die Anwendung, die zuerst bereitgestellt wurde, zuerst geladen. Die Angabe einer Deployment-Reihenfolge ist sinnvoll, wenn die Anwendung Abhängigkeiten hat und in einer bestimmten Reihenfolge geladen werden muss.

Aktiviert oder Status

Ein Häkchen, wenn die Anwendung aktiviert ist. Ein X, wenn die Anwendung deaktiviert und nur die standardmäßige Serverinstanz server vorhanden ist.

Wenn Cluster oder eigenständige Serverinstanzen vorhanden sind, aktivieren Sie "Aktiviert auf Zielen", um die Ziele anzuzeigen, auf denen die Anwendung bereitgestellt wird.

Engines

Die von der Anwendung verwendeten Containertypen. Es stehen folgende Containertypen zur Verfügung:

  • web

  • webservices

  • ejb

  • connector

  • appclient

  • weld (der Container für Contexts und Dependency Injection für Anwendungen der Java EE-Plattform)

Aktion

Links zu den Aktionen, die Sie mit der Komponente ausführen können, nachdem sie bereitgestellt wurde. "Erneut bereitstellen" und "Erneut laden" für alle Komponenten, "Starten" für Webanwendungen und Anwendungsclients und "Client-Stubs herunterladen" für Anwendungsclients.

Die Tabelle "Bereitgestellte Anwendungen" enthält darüber hinaus die folgenden Optionen.

Bereitstellen

Schaltfläche zum Bereitstellen einer Anwendung.

Deployment aufheben

Schaltfläche, um das Deployment einer oder mehrerer ausgewählter Anwendungen aufzuheben.

Aktivieren

Schaltfläche, um eine oder mehrere ausgewählte Anwendungen zu aktivieren, wenn nur die standardmäßige Serverinstanz server vorhanden ist.

Deaktivieren

Schaltfläche, um eine oder mehrere ausgewählte Anwendungen zu deaktivieren, wenn nur die standardmäßige Serverinstanz server vorhanden ist.

Filter

Dropdown-Liste, die Anwendungen nach Engine filtert.
