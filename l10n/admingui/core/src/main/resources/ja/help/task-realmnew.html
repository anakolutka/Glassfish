レルムを作成する

Enterprise Server には、次の 3 つのレルムが事前に設定されています。file、certificate、および admin-realm です。さらに、ldap、JDBC、solaris、またはカスタムレルムも作成できます。通常、サーバーには各タイプのレルムが 1 つずつありますが、Enterprise Server では、次の 2 つの file レルムがあります。file および admin-realm です。これらは、異なる 2 つの目的に使用される同じタイプの 2 つのレルムです。また、システムの各仮想サーバーで異なる証明書データベースを持つこともできます。

  1. ナビゲーションツリーで、「設定」ノードを展開します。
  2. ドメインがクラスタをサポートするように設定されている場合は、変更する設定を選択します。

    たとえば、クラスタ プロファイルや エンタープライズ プロファイルで作成されるドメインなどです。

  3. 「セキュリティー」ノードを開きます。
  4. 「レルム」ノードを選択します。
  5. 「レルム」ページで、「新規」をクリックします。

    「新しいレルム」ページが表示されます。

  6. 「名前」フィールドにレルムの名前を入力します。
  7. 作成されたレルムのクラス名を指定します。

    選択可能なクラス名を次の表に示します。

    レルム名

    クラス名

    file

    com.sun.enterprise.security.auth.realm.file.FileRealm

    certificate

    com.sun.enterprise.security.auth.realm.certificate.CertificateRealm

    ldap

    com.sun.enterprise.security.auth.realm.ldap.LDAPRealm

    JDBC

    com.sun.enterprise.security.auth.realm.jdbc.JDBCRealm

    solaris

    com.sun.enterprise.security.auth.realm.solaris.SolarisRealm

    custom

    ログインレルムクラスの名前
  8. 必要に応じて、レルムの追加プロパティーを追加します。
    1. 「プロパティーを追加」をクリックします。
    2. 「名前」フィールドに、プロパティーの名前を入力します。

      • ファイルレルムプロパティーの詳細については、「FileRealm クラスに固有のプロパティー」を参照してください。

      • 証明書レルムプロパティーの詳細については、「CertificateRealm クラスに固有のプロパティー」を参照してください。

    3. 「値」フィールドに、プロパティーの値を入力します。
  9. 「了解 (OK)」をクリックします。
