Editar Nó

Utilize a página Editar nó para atualizar os dados de configuração de um nó do GlassFish Server. Você também poderá usar essa página para alterar o tipo de um nó de CONFIG para DCOM ou SSH para ativar comunicação remota para o nó.

Se você alterar o tipo de nó de CONFIG para DCOM, os valores default serão aplicados se qualquer um dos campos a seguir ficarem vazios:

  • Domínio do Windows

  • Usuário do Windows

Se o tipo de nó for alterado de CONFIG para SSH , os valores default serão aplicados se nenhum dos campos a seguir for deixado em branco:

  • Porta SSH

  • Nome de Usuário

  • Arquivo de Chave

As opções que a página Editar Nó contêm dependem do tipo de nó.

Opções para Todos os Tipos de Nós

As opções a seguir estão disponíveis para todos os tipos de nós:

Nome

Nome designado ao nó quando o nó foi criado. Este campo é somente para leitura.

Tipo

Lista drop-down para especificar o tipo do nó, que é dos seguintes tipos:

CONFIG

O nó não oferece suporte à comunicação remota. Se CONFIG for selecionado, nenhuma opção de configuração de DCOM ou SSH será exibida. Se o tipo atual de nó for SSH e você selecionar CONFIG, DCOM ou SSH, os dados de configuração serão removidos da configuração do nó ao salvar as alterações.

DCOM

O nó suporta comunicação pelo protocolo remoto do DCOM (Distributed Component Object Model). O protocolo DCOM está disponível somente nos sistemas do Windows.

SSH

O nó oferece suporte a comunicações com shell seguro (SSH).

Host do Nó

Nome do host que o nó representará depois que o nó for atualizado. Se este campo é obrigatório ou não, depende do tipo de nó:

  • Se o tipo de nó for CONFIG, o nome do host pode ser omitido. O nome do host pode ser determinado quando as instâncias que residem no nó forem criadas.

  • Se o tipo de nó for DCOM ou SSH, o nome do host deverá ser especificado. Caso contrário, ocorre um erro.

Diretório do Nó

O caminho para o diretório que contém as instâncias GlassFish Server que são criadas no nó. Se um caminho relativo for especificado, o caminho é relativo ao diretório as-install, no qual as-install corresponde ao diretório de instalação de base do software GlassFish Server no host.

Se este campo for deixado em branco, a configuração do nó será atualizada da seguinte forma:

  • Se o tipo do nó for CONFIG, nenhum diretório das instâncias é especificado para o nó na configuração do servidor de administração de domínios (DAS). O diretório do nó pode ser determinado quando as instâncias que residem no nó forem criadas.

  • Se o tipo do nó for DCOM, o default é as-install\nodes em que as-install é o diretório base de instalação do software GlassFish Server no host.

  • Se o tipo do nó for SSH, o diretório do nó será as-install/nodes, no qual as-install é o diretório de instalação de base do software GlassFish Server no host.

Diretório de Instalação

O caminho completo do pai no diretório de instalação base do software GlassFish Server no host, por exemplo, /export/glassfish3/.

Se este campo for deixado em branco, a configuração do nó será atualizada da seguinte forma:

  • Se o tipo do nó for CONFIG, nenhum diretório de instalação é especificado para o nó na configuração do DAS. O diretório de instalação pode ser determinado quando as instâncias que residem no nó forem criadas.

  • Se o tipo de nó for DCOM ou SSH, o diretório de instalação será o pai do diretório de instalação base do software GlassFish Server para o DAS. Essa definição é útil só se o GlassFish Server estiver instalado no mesmo local em todos as máquinas.

Opções Adicionais para Nós de DCOM

As opções a seguir só estão disponíveis se o campo Tipo for definido como DCOM:

Forçar

Se a opção Ativado estiver selecionada, o nó é criado na configuração DAS mesmo se a validação dos parâmetros do nó falhar. Para validar os parâmetros de um nó, será necessário que o DAS possa entrar com contato com o host por meio de DCOM.

Validar DCOM

Se a opção Ativado for selecionada, o GlassFish Server irá testar a conexão do host do nó pelo DCOM antes de atualizar o nó. Como parte do teste, o GlassFish Server copia um script denominado delete_me.bat para o host.

Se você selecionar essa opção, o campo Diretório de Teste Remoto é ativado para possibilitar o fornecimento do diretório ao qual o GlassFish Server deverá copiar o script.

Diretório de Teste Remoto

A pasta no host remoto no qual o GlassFish Server deverá criar o arquivo delete_me.bat. O usuário que a opção Nome de Usuário do Windows especifica deve ter acesso para gravação a esta pasta. Caso contrário, ocorre um erro. O valor default é C:\.

Domínio do Windows

O nome do domínio do Wondows que contém o usuário que a opção Nome do Usuário do Windows especifica. O valor default é o nome do host no qual o Console de Administração está sendo executado.

Nome de Usuário do Windows

O usuário neste host do nó que irá executar o processo para conectar-se ao host por meio de DCOM. O default é o usuário que está executando o processo DAS. Se a opção Host do Nó estiver definida para o domínio localhost-, a opção Nome do Usuário do Windows será ingorada.

Autenticação do Usuário do Windows

Lista drop-down para especificar como o usuário do Windows será autenticado ao efetuar log-in no host desse nó. Os possíveis métodos de autenticação são os seguintes:

Senha

O usuário do Windows será autenticado por meio de uma senha que é armazenada na configuração DAS.

Cuidado:

Esta senha é armazenada em forma de texto criptografado na configuração do DAS. Para aumentar a segurança, use um alias de senha.

Se você selecionar esta opção, o campo Senha do Usuário do Windows será ativado para possibilitar que você forneça a senha.

Alias de Senha

O usuário do Windows será autenticado por meio de um alias de senha. Se selecionar esta opção, selecione o alias de senha na lista drop-down Alias de senha.

Senha do Usuário do Windows

A senha que o usuário do Windows irá usar ao efetuar log-in nesse host do nó.

Alias de Senha

Lista drop-down de aliases de senha definidos para este domínio.

Opções Adicionais para Nós de SSH

As opções a seguir só ficam disponíveis se o campo Tipo for definido como SSH:

Forçar

Se a opção Ativo estiver selecionada, o nó será atualizado na configuração do servidor de administração de domínios (DAS) mesmo se a validação dos parâmetros do nó falhar. Para validar os parâmetros de um nó, o DAS deve ser capaz de contatar o host do nó por meio de SSH.

Porta SSH

A porta para utilizar para conexões SSH para este host do nó.

Se o tipo de nó for alterado de CONFIG para SSH e este campo for deixado vazio, a opção Porta SSH será definida como 22.

Se a opção Host do Nó estiver definida para o domínio localhost-, a opção Porta SSH será ignorada.

Nome de Usuário SSH

Usuário deste host do nó que deve executar o processo para conexão com o host através de SSH.

Se o tipo de nó for alterado de CONFIG para SSH e este campo for deixado vazio, a opção Nome de Usuário será definida como o usuário que estiver em execução no processo DAS.

Se a opção Host do Nó estiver definida para o domínio localhost-, a opção Usuário SSH será ignorada.

Autenticação de Usuário SSH

Lista drop-down para especificar como o usuário SSH é autenticado ao efetuar log-in neste host do nó. Os possíveis métodos de autenticação são os seguintes:

Arquivo de Chave

O usuário SSH é autenticado através de um arquivo de chave privada SSH. Se selecionar esta opção, especifique um caminho absoluto para o arquivo no campo Arquivo de Chave.

Senha

O usuário SSH é autenticado através de uma senha que é armazenada na configuração do DAS.

Cuidado:

Esta senha é armazenada em forma de texto criptografado na configuração do DAS. Para maior segurança, utilize um arquivo de chave ou um alias de senha.

Se você selecionar esta opção, o campo Senha do Usuário do SSH será ativado para possibilitar que você forneça a senha.

Alias de Senha

O usuário SSH é autenticado através de um alias de senha. Se selecionar esta opção, selecione o alias de senha na lista drop-down Alias de senha.

Configurar SSH

Se a opção Ativado estiver selecionada, GlassFish Server define uma chave de SSH no host do nó. O GlassFish Server configura a chave de SSH copiando arquivos de chave existentes do host DAS ou, se necessário, gerando os arquivos antes de copiá-los. Essa opção só fica disponível se a opção Autenticação do Usuário de SSH for definida como Arquivo de Chave.

Gerar Chave

Se a opção Ativado for selecionada, GlassFish Server irá gerar os arquivos de chave de SSH antes de copiar arquivos para o host do nó, caso os arquivos não existam. Se os arquivos de chave do SSH existirem, GlassFish Server não irá gerar os arquivos, mesmo que a opção Ativado esteja selecionada.

Observação:

Se os arquivos de chave do SSH não existirem, a opção Ativado deverá ser selecionada. Caso contrário, ocorrerá um erro quando você tentar salvar suas alterações.
Senha de Usuário SSH

A senha que o usuário SSH utilizará ao efetuar log-in neste host do nó.

Alias de Senha

Lista drop-down de aliases de senha definidos para este domínio.

Arquivo de Chave Pública

O caminho absoluto para o arquivo de chave pública de SSH para o usuário que a opção Nome de Usuário de SSH especifica. O conteúdo do arquivo de chave pública é acrescentado ao arquivo .ssh/authorized_keys no host do nó. Se o arquivo .ssh/authorized_keys não existir no host, o GlassFish Server irá criá-lo.

Arquivo de Chave

Caminho absoluto para o arquivo da chave privada de SSH para o usuário que a opção Nome de Usuário especifica. Este arquivo é utilizado para autenticação do daemon sshd no host do nó.

Observação:

O GlassFish Server também suporta autenticação por senha por meio da entrada AS_ADMIN_SSHPASSWORD no arquivo de senhas. O arquivo de senhas é especificado na opção -paswordfiledo utilitário asadmin(1).

Se o arquivo de chave privada de SSH for protegido por uma frase-senha o arquivo de senhas deverá conter a entrada AS_ADMIN_SSHKEYPASSPHRASE.

O caminho para o arquivo de chave deve ser acessível pelo DAS e o arquivo de chave poderá ser lido pelo DAS.

O default dependerá se o tipo do nó foi alterado de CONFIG para SSH para ativar a comunicação SSH do nó:

Se o tipo de nó for alterado de CONFIG para SSH e este campo for deixado vazio, a opção Arquivo de Chave será definida como um arquivo de chave no diretório .ssh do usuário. Se forem encontrados vários arquivos de chave, o GlassFish Server usa a seguinte ordem de preferência:

  1. id_rsa

  2. id_dsa

  3. identity
