Atributos de dominio

Utilice la página Atributos de Dominio para definir atributos de dominio avanzados de GlassFish Server.

La página Atributos de Dominio contiene las siguientes opciones.

Cargar Consola después del Inicio de DAS

Determina cuándo se debe cargar Consola de administración:

Basado en Uso

Carga Consola de administración después del servidor de administración de dominio (DAS) si la consola se ha utilizado en las últimas 24 horas o si las demás instancias del servidor existen de forma independiente al DAS. Éste es el valor por defecto.

Siempre

Siempre carga Consola de administración después del DAS. La consola se cargará después de que haya finalizado la secuencia de inicio principal de GlassFish Server.

Nunca

No carga nunca Consola de administración después del DAS. La consola se carga cuando se solicita acceso a ella.

Raíz de Aplicación

Ruta de acceso completa del directorio donde residen las aplicaciones desplegadas. El valor por defecto es domain-dir/applications.

Raíz de Log

Identifica el lugar en el que se guardan los archivos log del servidor. El valor por defecto es domain-dir/logs.

Configuración Regional

Si se deja este campo en blanco, se utilizará la configuración regional por defecto del host.

La configuración regional es un identificador que especifica una combinación concreta de idioma y región. Por ejemplo, la configuración regional para inglés de Estados Unidos es en_US y para japonés es ja_JP. Para utilizar una configuración regional que no sea del idioma inglés, GlassFish Server tiene que estar localizado, un proceso que incluye traducir el inglés a otro idioma.
