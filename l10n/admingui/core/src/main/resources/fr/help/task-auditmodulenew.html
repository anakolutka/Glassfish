Créer un module d'audit

Enterprise Server fournit un simple module d'audit par défaut ; pour plus d'informations, consultez Utiliser le module d'audit par défaut .

  1. Développez le nœud Configurations.
  2. Sélectionnez l'instance à configurer :
    • Pour configurer une instance particulière, développez le nœud de configuration de l'instance. Par exemple, pour l'instance par défaut, server, développez le nœud server-config.
    • Pour configurer les paramètres par défaut de toutes les instances, développez le noeud default-config.
  3. Développez le nœud Sécurité.
  4. Sélectionnez le nœud Modules d'audit.
  5. Sur la page Modules d'audit, cliquez sur Nouveau.
  6. Sur la page Créer un module d'audit, entrez les informations suivantes :

    • Nom – Nom utilisé pour identifier le module d'audit.

    • Nom de classe – Nom complet de la classe qui implémente le module. Le nom de classe du module d'audit par défaut est com.sun.enterprise.security.Audit.

  7. Pour ajouter des propriétés JVM à ce module, cliquez sur Ajouter une propriété. Spécifiez un nom et une valeur pour chaque propriété.

    Les propriétés valides incluent auditOn, qui détermine si cette classe d'implémentation doit être activée ou non. Les valeurs valides sont vrai et faux.

  8. Cliquez sur OK pour sauvegarder, ou cliquez sur Annuler pour quitter sans sauvegarder.
