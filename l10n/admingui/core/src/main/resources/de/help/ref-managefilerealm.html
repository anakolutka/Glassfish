Dateibenutzer

Verwalten Sie Benutzer des file-Bereichs über die Administration Console. Benutzer und Gruppen im file-Bereich sind in der Schlüsseldatei aufgeführt, deren Speicherort durch die Eigenschaft file angegeben wird.

Hinweis – Es ist auch möglich, diese Schritte zum Hinzufügen von Benutzern zu einem Dateibereich (einschließlich admin-realm) zu verwenden. Ersetzen Sie einfach den Namen des Zielbereichs des file-Bereichs, der in diesem Abschnitt referenziert wird.

Ein Benutzer im file-Bereich kann zu einer Java-Gruppe (einer Benutzerkategorie, die durch gemeinsame Merkmale klassifiziert ist) gehören. So können Kunden einer eCommerce-Anwendung z. B. zur Gruppe CUSTOMER gehören, die Großkunden würden jedoch der Gruppe PREFERRED angehören. Das Kategorisieren von Benutzern in Gruppen vereinfacht die Zugangssteuerung bei einer großen Benutzeranzahl.

Nach der Installation von Enterprise Server ist der während der Installation eingegebene Benutzer der einzige Benutzer. Dieser Benutzer ist standardmäßig Mitglied der Gruppe asadmin im Bereich admin-realm und damit berechtigt, Änderungen an Enterprise Server vorzunehmen. Benutzer, die dieser Gruppe zugewiesen sind, verfügen über Administratorrechte und können damit auf die asadmin-Tools und die Administration Console zugreifen.
