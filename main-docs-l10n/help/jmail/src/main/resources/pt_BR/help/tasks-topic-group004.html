Para Configurar o Tempo de Execução em Batch

  1. Na árvore de navegação, selecione a instância de cluster ou servidor do GlassFish Server cujas execuções de job em batch você deseja exibir.

    A página Informações Gerais será aberta.

  2. Selecione a guia Batch.

    A página Execuções de Job em Batch é aberta.

  3. Clique na guia Configuração.

    A página Configuração do Tempo de Execução em Batch é aberta. O runtime usa um serviço executor gerenciado e uma origem de dados para executar jobs em batch.

  4. Na lista drop-down Nome de Pesquisa do Serviço Executor, selecione o executor gerenciado a ser usado para fornecer threads aos jobs em batch.

    O serviço executor gerenciado default é concurrent/__defaultManagedExecutorService. O serviço executor gerenciado pode ser alterado após um job em batch ter sido submetido ao tempo de execução em lote sem causar impacto sobre a execução do job.

  5. Na lista drop-down Nome de Pesquisa da Origem de Dados, selecione a origem de dados a ser usada para armazenar informações sobre o job.

    A origem de dados default é jdbc/__TimerPool. Não altere a origem de dados após o primeiro job em batch ter sido submetido à execução do tempo de execução em batch. Se a origem de dados tiver de ser alterada, interrompa e reinicie o domínio e, em seguida, faça a alteração antes de qualquer job ser inicializado ou reinicializado. No entanto, depois que a origem de dados tiver sido alterada, as informações armazenadas na origem de dados anterior ficarão inacessíveis.

  6. Clique em Salvar.
