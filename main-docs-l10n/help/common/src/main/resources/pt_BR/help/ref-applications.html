Aplicações

A página Aplicações exibe uma lista das aplicações que estão implantadas no GlassFish Server. É possível exibir e gerenciar as aplicações implantadas, e você pode implantar mais aplicações.

Para cada aplicação, as informações a seguir são mostradas.

Nome

O nome da aplicação.

O nome pode incluir um identificador de versão opcional, que segue o nome e é separado do nome por dois-pontos (:). Para obter mais informações sobre versões modulares ou da aplicação, consulte Module and Application Versions em Oracle GlassFish Server Application Deployment Guide.

Ordem de Implantação

A ordem de implantação da aplicação.

Aplicações com um número mais baixo são carregadas primeiro na inicialização do servidor. Um aplicação com uma ordem de implantação de 102 é carregada antes de uma de 110. Se uma ordem de implantação não for especificada no tempo uma aplicação será implantada, a ordem de implantação default de 100 será designada. Se duas aplicações tiverem a mesma ordem de implantação, a aplicação que foi implantada primeiro será carregada primeiro. Especificar uma ordem de implantação é útil se a aplicação tiver dependências e deve ser carregada em uma determinada ordem.

Ativado ou Status

Uma marca de seleção se a aplicação for ativada ou um X se a aplicação for desativada, se existir somente a instância do servidor default, server.

Se existirem clusters ou outras instâncias de servidores standalone, selecione Ativado em Alvos para exibir os alvos nos quais a aplicação está implantada.

Mecanismos

Os tipos de contêineres usados pela aplicação. Os tipos de contêineres podem ser:

  • web

  • webservices

  • ejb

  • connector

  • appclient

  • weld (o contêiner para Contextos e Injeção de Dependência para aplicações da Plataforma Java EE)

Ação

Links para as ações que você pode realizar no componente depois de implantá-lo: Reimplantar e Recarregar para todos os componentes, Acionar para aplicações Web e clientes de aplicações e Fazer download de stubs para clientes da aplicação;

A tabela Aplicações Implantadas também contém as opções a seguir.

Implantar

Botão para implantar uma aplicação.

Cancelar Implantação

Botão para cancelar a implantação de uma ou mais aplicações selecionadas.

Ativar

Botão para ativar uma ou mais aplicações selecionadas, somente se a instância default do servidor, server, existir.

Desativar

Botão para desativar um ou mais aplicações selecionadas, somente se a instância default do servidor, server, existir.

Filtro

Lista drop-down que filtra as aplicações por mecanismo.
