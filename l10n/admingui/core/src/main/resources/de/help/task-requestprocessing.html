So konfigurieren Sie die Verarbeitung von HTTP-Dienstanforderungen

  1. Erweitern Sie in der Navigationsstruktur den Konfigurationsknoten.
  2. Wählen Sie den Knoten für den HTTP-Dienst aus.
  3. Wählen Sie auf der Seite "HTTP-Dienst" die Registerkarte "Anforderungsabwicklung".

    Die Seite "Anforderungsabwicklung" wird geöffnet.

  4. Geben Sie im Feld "Anzahl der Threads" die maximale Anzahl der Threads zur Anforderungsabwicklung an.

    Der Standardwert lautet 20.

  5. Geben Sie im Feld "Ursprüngliche Anzahl der Threads" die Anzahl der Threads zur Anforderungsabwicklung an, die beim Starten des Servers verfügbar sind.

    Der Vorgabewert ist 2.

  6. Geben Sie im Feld "Thread-Inkrement" die Anzahl der Threads für die Anforderungsabwicklung an, die hinzugefügt werden, wenn die Anzahl der Anforderungen den Wert für "Ursprüngliche Anzahl der Threads" übersteigt.

    Der Vorgabewert ist 1.

  7. Geben Sie im Feld "Zeitüberschreitung für Anforderung" die Dauer (in Sekunden) bis zur Zeitüberschreitung für die Anforderung an.

    Der Standardwert ist 30.

  8. Geben Sie im Feld "Pufferlänge" die Größe (in Byte) des Puffers an, der von den Threads für die Anforderungsabwicklung zum Lesen der Anforderungsdaten verwendet wird.

    Der Standardwert ist 8192.

  9. Klicken Sie auf "Speichern".
