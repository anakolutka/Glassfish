Neuer Netzwerk-Listener

Auf der Seite "Neuer Netzwerk-Listener" können Sie einen neuen Netzwerk-Listener erstellen.

Wenn Sie einen Netzwerk-Listener erstellen, können Sie gleichzeitig auch ein vom Netzwerk-Listener zu verwendendes Protokoll erstellen. Alternativ können Sie ein Protokoll verwenden, das Sie zuvor erstellt haben. In der Regel liegt eine 1:1-Beziehung zwischen einem Netzwerk-Listener und dem zugehörigen Protokoll vor, dies ist jedoch nicht obligatorisch.

Die Seite "Neuer Netzwerk-Listener" enthält die folgenden Optionen.

Name der Konfiguration

Der Name der Konfiguration, für die die Einstellungen auf dieser Seite angewendet werden. Dieses Feld ist schreibgeschützt.

Name

Ein eindeutiger Listener-Name. Der Name eines Netzwerk-Listeners darf nicht mit einer Ziffer beginnen.

Protokoll

Das mit dem Netzwerk-Listener verbundene Protokoll. Die folgenden Optionen geben an, ob ein Protokoll erstellt oder ein vorhandenes Protokoll verwendet werden soll.

Neues Protokoll erstellen

Für den Netzwerk-Listener wird ein neues Protokoll mit Standardeinstellungen erstellt. Als Protokollname wird standardmäßig der Name des Netzwerk-Listener mit dem Zusatz -protocol verwendet, dieser Name kann jedoch geändert werden.

Virtueller Server

Der mit einem neuen Protokoll zu verbindende virtuelle Server.

Vorhandenes Protokoll verwenden

Das ausgewählte Protokoll wird verwendet. Das Protokoll legt fest, ob die Sicherheit für den Netzwerk-Listener aktiviert ist.

Status

Der Status des Netzwerk-Listeners. Der Listener kann aktiviert oder deaktiviert werden. Diese Option ist standardmäßig aktiviert. Wenn der Listener deaktiviert ist, führen alle Versuche, eine Verbindung zum Listener herzustellen, zu einer Socket-Ausnahme (java.net.ConnectException).

In Versionen vor GlassFish Server 9.1 wurde bei einem Listener, dessen Attribut "Aktiviert" auf "False" gesetzt war, für alle an den Listener gesendeten Anforderungen der Antwortcode 404 zurückgegeben. Um dieses Behavior in der aktuellen Version von GlassFish Server zu erzielen, setzen Sie den Status des Listeners auf "Aktiviert" und den Status aller verbundenen virtuellen Server auf "Deaktiviert". Die mit einem virtuellen Server verbundenen Listener werden in der Dropdown-Liste "Netzwerk-Listener" des Servers aufgeführt.

Sicherheit

Bei aktivierter Option ist für das vom Netzwerk-Listener verwendete Protokoll Sicherheit aktiviert. Das Feld "Sicherheit" ist ein schreibgeschütztes Feld.

Wenn Sie den admin-listener für den Domain Administration Server (DAS) konfigurieren, der die Konfiguration server-config verwendet, ist diese Option schreibgeschützt. Um die Sicherheit für diesen Listener zu aktivieren, müssen Sie die sichere Administration aktivieren. Siehe Sichere Administration.

JK-Listener

Bei aktivierter Option unterstützt der Listener das Apache-Protokoll mod-jk. Das Protokoll mod-jk ist ein proprietäres Kommunikations-/Netzwerkprotokoll zwischen dem Apache-Webserver httpd und einer Backend-Instanz von GlassFish Server. Wenn der Listener dieses Protokoll unterstützt, wird davon ausgegangen, dass httpd den gesamten statischen Content verarbeitet, während alle Anforderungen von dynamischen Ressourcen (d.h. Servlets und JSPs) an den mod-jk-fähigen Listener von GlassFish Server weitergeleitet werden, wobei das Protokoll mod-jk verwendet wird. Diese Option ist standardmäßig deaktiviert.

Port

Die Nummer des Ports, auf dem der Netzwerk-Listener horcht. Gültige Werte sind 1 bis 65535. Bei einem UNIX-System sind zum Erstellen von Sockets, die auf den Ports 1 bis 1024 horchen, Superuser-Berechtigungen erforderlich.

Adresse

Die IP-Adresse, auf der der Netzwerk-Listener horcht. Die Adresse kann als Punktepaar oder in IPv6-Notation angegeben werden. Beim Wert any (für INADDR_ANY) wird auf allen IP-Adressen gehorcht. Auch die Angabe eines Hostnamens ist möglich.

Threadpool

Der mit dem Netzwerk-Listener verbundene Threadpool. Normalerweise wählen Sie einen der beiden Threadpools, die bei der Installation von GlassFish Server konfiguriert werden.

Transport

Der Transporttyp für den Netzwerk-Listener. Standardmäßig steht tcp als einzige Option zur Auswahl.
