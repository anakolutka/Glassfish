Outil de mise à niveau : présentation

Utilisez l'outil de mise à niveau pour transférer les informations de configuration à partir d'une installation de serveur d'application 8.x ou 9 vers une installation de serveur d'application 9.1. Les informations transférées incluent des données sur les applications déployées, le domaine du fichier, les certificats de sécurité et d'autres ressources et paramètres de configuration du serveur.

Le tableau suivant présente les mises à niveau du serveur d'application prises en charge, où PE signifie Platform Edition et EE signifie Enterprise Edition.

L'outil détecte automatiquement la version de l'installation de serveur source spécifiée.

Remarque : avant de lancer le processus de mise à niveau, assurez-vous que le serveur source (le serveur à partir duquel vous effectuez la mise à niveau) et le serveur cible (le serveur auquel vous effectuez la mise à niveau) sont arrêtés.

Terminologie de mise à niveau

Vous trouverez ci-dessous une liste des termes importants associés au processus de mise à niveau :

  • Serveur source : l'installation à partir de laquelle vous effectuez la mise à niveau vers la nouvelle version.

  • Serveur cible : l'installation vers laquelle vous effectuez la mise à niveau.

  • Racine de domaines : répertoire où les domaines sont créés. Ce répertoire est, par défaut, l'emplacement spécifié comme AS_DEF_DOMAINS_PATH dans le répertoire de domaine asenv.conf

  • ou domain-dir : répertoire (dans la racine de domaines) correspondant à un domaine spécifique. Toutes les données de configuration et autres relatives au domaine sont contenues dans ce répertoire.

  • Racine de l'installation : répertoire vers lequel le serveur d'application est installé.

Si la case Mettre à niveau a été cochée lors du processus d'installation du serveur d'application, l'écran Assistant de mise à niveau s'affiche automatiquement une fois l'installation terminée. Le processus de mise à niveau démarre.
