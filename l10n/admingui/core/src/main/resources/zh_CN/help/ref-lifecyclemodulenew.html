新建生命周期模块

使用“新建生命周期模块”页面可以定义新的生命周期模块。

“新建生命周期模块”页面包含以下选项：

名称

生命周期模块名称。

类名

生命周期模块类文件的全限定名称。此类必须实现 com.sun.appserv.server.LifecycleListener 接口。

类路径

生命周期模块的类路径。类路径指定生命周期模块的位置。默认值为 domain-dir/applications。如果模块已经在此目录中（即，在服务器类路径中），则此字段可以留为空白。

加载顺序

启动时装入此生命周期模块的顺序。加载顺序值为较小整数的模块会较早装入。值的范围在 101 和操作系统的 MAXINT 之间。1 和 100 之间的值为保留值。

状态

如果选中此选项，则将启用生命周期模块。默认情况下启用此选项。

加载失败时

如果选中此选项，则在生命周期模块无法装入时将关闭服务器。默认情况下禁用此选项。

虚拟服务器

要与生命周期模块关联的虚拟服务器。默认情况下，不选择任何虚拟服务器。

说明

生命周期模块说明。

其他属性

生命周期模块的其他属性。Enterprise Server 不定义生命周期模块的任何其他属性。
