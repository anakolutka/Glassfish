升级工具：概述

使用升级工具可以将配置信息从 Application Server 8.x 或 Application Server 9 安装传输到 Application Server 9.1 安装。传输的信息包括关于已部署应用程序、文件领域、安全性证书以及其他资源和服务器配置设置的数据。

下表显示了受支持的 Application Server 升级，其中 PE 表示 Platform Edition，EE 表示 Enterprise Edition。

源安装
 目标安装 (9.1)
7.0 PE
 不支持
7.1 SE
 不支持
7.1 EE
 不支持
8.0 PE
 支持（支持从 8.0 PE 域升级到 9.1 开发者域）
8.1 PE
 支持（支持从 8.1 PE 域升级到 9.1 开发者域）
8.1 EE
 支持（支持从 8.1 EE 域升级到 9.1 企业域）
8.2 PE
 支持（支持从 8.2 PE 域升级到 9.1 开发者域）
8.2 EE
 支持（支持从 8.2 EE 域升级到 9.1 企业域）
9.0 PE
 支持（支持从 9.0 PE 域升级到 9.1 开发者域）

该工具自动检测指定源服务器安装的版本。

注意：在开始升级过程之前，请确保源服务器（要进行升级的服务器）和目标服务器（要升级到的服务器）都已停止。

升级术语

以下是与升级过程有关的重要术语：

  • 源服务器：要升级到新版本的安装。

  • 目标服务器：要升级到的安装。

  • 域根目录：在其中创建域的目录。默认情况下，此目录是在 asenv.conf 中指定为 AS_DEF_DOMAINS_PATH 的 位置

  • 域目录或 domain-dir：与特定域对应的目录（在域根目录内）。与该域相关的所有配置及其他数据都存在于此目录中。

  • 安装根目录：Application Server 的安装目录。

如果在 Application Server 安装过程中选中了“升级”复选框，则完成安装后将会自动显示“升级向导”屏幕。升级过程开始。
