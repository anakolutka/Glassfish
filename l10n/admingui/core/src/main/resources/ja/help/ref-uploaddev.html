アプリケーションまたはモジュールの配備

アプリケーションを配備するには、「アプリケーションまたはモジュールを配備」ページを使用します。

「アプリケーションまたはモジュールを配備」ページには、すべてのアプリケーションで次のオプションが表示されます。追加のオプションは、アプリケーションの「位置」フィールドを指定したあとに表示されます。

位置

配備するアプリケーションのアーカイブの場所。

次のオプションは、アーカイブにアクセス可能な場所とアーカイブがファイルであるかディレクトリであるかを指定します。

サーバーにアップロードされるパッケージファイル

このアーカイブは、クライアントマシンにあるファイル、またはクライアントマシンからアクセス可能なファイルです。

クライアントマシンとは、ブラウザを介して 管理コンソール を表示しているホストです。

Enterprise Server からアクセス可能なローカルのパッケージファイルまたはディレクトリ

このアーカイブは、サーバーマシンにあるファイル、または展開されたディレクトリにあるパッケージ化されていないアプリケーションです。

サーバーマシンは、Enterprise Server のドメイン管理サーバーを実行しているホストです。

タイプ

アプリケーションのタイプ。次のタイプを指定できます。

  • Web アプリケーション

  • エンタープライズアプリケーション

  • アプリケーションクライアント

  • コネクタモジュール

  • EJB JAR

  • Ruby アプリケーション

  • その他

各アプリケーションのタイプの追加オプションについては、次の節で説明します。

  • 「Web アプリケーションの追加オプション」

  • 「エンタープライズアプリケーションの追加オプション」

  • 「Ruby アプリケーションの追加オプション」

Web アプリケーションの追加オプション

アプリケーションのタイプが Web アプリケーションの場合、次のオプションが表示されます。

コンテキストルート

アプリケーションのパス。Web アプリケーションの URL では、コンテキストルートはポート番号の直後に続きます (http://host:port /context-root/...)。たとえば /hello のように、コンテキストルートがスラッシュで始まるようにしてください。

アプリケーション名

アプリケーションの名前。

Virtual Servers

このアプリケーションと関連付けられている仮想サーバー。

ステータス

このオプションを有効にすると、アプリケーションが有効になります。このオプションはデフォルトで有効になっています。

JSP をプリコンパイル

このオプションが有効な場合、JavaServer PagesTM (JSPTM) ファイルがプリコンパイルされます。このオプションが無効な場合、JSP ファイルは初めてアクセスされた実行時にコンパイルされます。このオプションは、デフォルトでは無効になっています。

ベリファイアを実行

このオプションが有効な場合、配備の前にベリファイアが実行されます。検証が失敗すると、配備は実行されません。ベリファイアはファイルの構造とコンテンツを調べます。大きなアプリケーションの検証は時間がかかる可能性があります。このオプションは、デフォルトでは無効になっています。

ライブラリ

このモジュールまたはアプリケーションに固有のライブラリ JAR ファイルのコンマ区切りのリスト。絶対パスと相対パスのどちらでも指定できます。相対パスは、domain-dir/lib/applibs からの相対で指定します。絶対パスの場合は、ドメイン管理サーバー (DAS) がパスにアクセスできる必要があります。つまり、domain-dir 以下にある必要があります。ライブラリは、指定した順にアプリケーションで利用可能になります。

説明

アプリケーションの説明。

Ruby アプリケーションの追加オプション

アプリケーションのタイプが Ruby アプリケーションの場合、次のオプションが表示されます。

アプリケーション名

アプリケーションの名前。

コンテキストルート

アプリケーションのパス。Web アプリケーションの URL では、コンテキストルートはポート番号の直後に続きます (http://host:port /context-root/...)。たとえば /hello のように、コンテキストルートがスラッシュで始まるようにしてください。

ステータス

このオプションを有効にすると、アプリケーションが有効になります。このオプションはデフォルトで有効になっています。

JRuby ホーム

JRuby コンテナがインストールされるディレクトリのパス。デフォルトの場所は as-install/jruby です。このフィールドに値を指定すると、Ruby コンテナに設定されている「JRuby ホーム」の値はアプリケーションの値で上書きされます。

ランタイム

開始する JRuby ランタイムの初期数。指定する値は、「Minimum Runtime」フィールドの値以上で、「Maximum Runtime」フィールドの値未満にする必要があります。デフォルト値は 1 です。このフィールドに値を指定すると、Ruby コンテナで設定された「ランタイム」の値はアプリケーションの値で上書きされます。

Minimum Runtime

プール内の JRuby ランタイムの最小数。指定する値は、「ランタイム」フィールドの値以下で、「Maximum Runtime」フィールドの値未満にする必要があります。デフォルト値は 1 です。このフィールドに値を指定すると、Ruby コンテナで設定された「Minimum Runtime」の値はアプリケーションの値で上書きされます。

Maximum Runtime

プール内の JRuby ランタイムの最大数。指定する値は、「Minimum Runtime」フィールドの値以上で、「ランタイム」フィールドの値以上にする必要があります。デフォルト値は 2 です。このフィールドに値を指定すると、Ruby コンテナで設定された「Maximum Runtime」の値はアプリケーションの値で上書きされます。

フレームワーク

アプリケーションを配備するフレームワーク。指定できる値は、rails、merb、sinatra、および auto-detection です。デフォルトは rails です。このプロパティーに auto-detection 以外の値を設定すると、通常の (時間がかかる) 自動検出プロセスが省略され、強制的に指定したフレームワークで配備を行います。配備したアプリケーションが指定したフレームワーク用に記述されていない場合は、エラーが発生します。

Rack 環境

JRuby フレームワーク (Rails、Merb など) を実行する環境。指定できる値は、production、development、および test です。デフォルトは development です。

マルチスレッドに対して安全

アプリケーションを実行するフレームワークのスレッドセーフ設定。このプロパティーは、自動検出されたユーザー指定の起動スクリプトを使用して起動されたアプリケーションに影響します。次の値を指定できます。

true

フレームワークがスレッドセーフで、プールの作成を必要としないことを指定します。

false

アプリケーションがアプリケーションインスタンスのプールを使用して起動され、アプリケーションの各インスタンスは一度に 1 つのスレッドからアクセスされることを指定します。

default

Enterprise Server は、自動的にスレッドセーフを検出しようとします。この値はデフォルトです。

このプロパティーは、スレッドセーフを自動的に判定できない、起動中のフレームワークにのみ影響します。この値を true に設定すると、自動検出された Rails 2.1.x アプリケーションはスレッドセーフモードで起動されず、これを使用してスレッドセーフなフレームワークを強制的にプールモードで開始することもできません。
