このプロモートされるビルドには、クラスタリング機能と、WSIT を使用した Microsoft の相互運用性が含まれています。

プロモートされるビルドは週に 1 回実行される GlassFish のビルドで、クィックルックテスト、CTS テストのサブセット (344 項目の J2EE 1.4 テストの代表的なコレクション)、SQE テストのサブセット (合計 22 項目) を含む最低レベルのテストに通過しています。クィックルック、CTS スモーク、SQE スモークのすべてのテストを通過しないかぎり、ビルドはプロモートされません。

注: このリストは、さまざまなプラットフォームのファイルを提供しています。自分のプラットフォームに適したファイルを選択するようにしてください。以下にリストされているファイルをよく確認して必要なものを選択し、リンクをクリックしてダウンロードします。ブラウザによっては、右クリックまたは CTRL キーを押しながらクリックしてこれらのファイルをコンピュータに保存する必要のあるものもあります。

GlassFish コードの大部分は、Common Development and Distribution License (CDDL) v1.0 および GNU General Public License (GPL) v2 で構成されるデュアルライセンスのもとで入手できます。次のページには、GlassFish のコンポーネントと、対象となるライセンスについての詳細を記載します。

GlassFish のアンバンドルと設定方法

GlassFish をインストールして設定するには、システムに JDK 5 または JDK 6 がインストールされている必要があります。設定の処理は、Ant (1.6.5) に基づきます。バンドルには、Application Server 用の Java EE 5 アプリケーションの開発を促すためにタスクが拡張された Ant ディストリビューションが含まれています。

  1. バンドルの 1 つをディスクにダウンロードし、JAVA_HOME をシステムにインストールされている JDK に設定します。
  2. 次を実行します。 
       % java -Xmx256m -jar filename.jar
    このコマンドは、GlassFish をアンバンドルし、'glassfish' という名前のディレクトリの下に新しいディレクトリ構造を作成します。
  3.  % cd glassfish
  4. UNIX(tm) から派生したオペレーティングシステムのマシンを使用している場合は、GlassFish バンドルに含まれている Ant バイナリの実行権を設定します。 
    % chmod -R +x lib/ant/bin
    % lib/ant/bin/ant -f setup.xml
    または、Windows の場合は次のように設定します。
    % lib\ant\bin\ant -f setup.xml

クラスタリングがサポートされた GlassFish V2 インストールの場合、上記のインストールの手順 4 を変更します。

% lib/ant/bin/ant -f setup-cluster.xml
または、Windows の場合は次のように設定します。
% lib\ant\bin\ant -f setup-cluster.xml

GlassFish を起動するには、『クイックスタートガイド』を参照してください。

GlassFish V2 でクラスタを作成後に負荷分散を行う方法については、「GlassFish V2 でのクラスタ/負荷分散の構成」を参照してください。
