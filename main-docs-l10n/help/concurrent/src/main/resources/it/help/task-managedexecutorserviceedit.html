Per modificare un servizio esecutore gestito

  1. Espandere il nodo Risorse nell'albero di navigazione.

  2. Nel nodo Risorse espandere il nodo Risorse concorrenti.

  3. Nel nodo Risorse concorrenti selezionare Servizi esecutore gestiti.

    Viene visualizzata la pagina Servizi esecutore gestiti.

  4. Nella pagina Servizi esecutore gestiti fare clic sul nome della risorsa da modificare.

    Viene visualizzata la pagina Modifica servizio esecutore gestito.

    Nota:

    Il campo Nome JNDI è di sola lettura. È possibile specificare il nome JNDI solo quando si crea una nuova risorsa. Anche il campo del nome JNDI logico è di sola lettura.

  5. Selezionare la casella di controllo Abilitato dell'opzione Informazioni contesto per propagare i contesti ai thread, quindi selezionare i contesti che si desidera propagare.

    Questa opzione è abilitata e tutti i contesti sono selezionati per impostazione predefinita. Per selezionare più contesti, fare clic su di essi tenendo premuto il tasto Maiusc o Ctrl. Se questa opzione è disabilitata, i contesti non vengono propagati, anche se sono selezionati.

  6. Selezionare la casella di controllo Abilitato dell'opzione Stato per abilitare la risorsa.

    Questa opzione è abilitata per impostazione predefinita.

  7. Nel campo Priorità thread digitare la priorità da assegnare ai thread creati.

    Il valore predefinito è 5.

  8. Selezionare la casella di controllo Abilitato dell'opzione Task con tempi di esecuzione lunghi per utilizzare la risorsa per l'esecuzione di task con tempi di esecuzione lunghi.

    Se è abilitata, i task con tempi di esecuzione lunghi non vengono segnalati come bloccati. Questa opzione è disabilitata per impostazione predefinita.

  9. Nel campo Sospendi dopo digitare il numero di secondi in cui i task possono essere eseguiti prima di essere considerati senza risposta.

    Il valore predefinito è 0 e indica che i task non vengono mai considerati senza risposta.

  10. Nel campo Ordine di distribuzione digitare un numero per specificare l'ordine di distribuzione della risorsa.

    Le risorse con i numeri più bassi vengono caricate per prime all'avvio del server. Il valore predefinito è 100.

  11. Nel campo Descrizione digitare una breve descrizione della risorsa.

    La descrizione è un valore stringa con lunghezza limitata a 250 caratteri.

  12. Nel campo Dimensione di base digitare il numero di thread da conservare in un pool di thread, anche se sono inattivi.

    Quando viene inviato un nuovo task e il numero di thread in esecuzione è minore della dimensione di base, per gestire la richiesta viene creato un nuovo thread. Il valore predefinito è 0 e indica che nuovi thread non vengono mai creati a meno che la coda dei task non sia piena o la risorsa stia utilizzando l'accodamento diretto. L'accodamento diretto si verifica quando la capacità della coda dei task è 0 oppure quando è 2147483647 e la dimensione di base è 0.

  13. Nel campo Dimensione massima pool digitare il numero massimo di thread che un pool di thread può contenere.

    Il valore predefinito è 2147483647 e indica che il pool di thread è essenzialmente senza limiti e può contenere qualsiasi numero di thread.

  14. Nel campo Controllo attività digitare il numero di secondi in cui i thread possono rimanere inattivi quando il numero di thread è maggiore della dimensione di base.

    Il valore predefinito è 60.

  15. Nel campo Durata thread digitare il numero di secondi in cui i thread possono rimane in un pool di thread prima di essere rimossi, indipendentemente dal fatto che il numero di thread sia maggiore della dimensione di base o che i thread siano inattivi.

    Il valore predefinito è 0 e indica che i thread non vengono mai rimossi.

  16. Nel campo Capacità coda di task digitare il numero di task inviati che è possibile memorizzare nella coda di task in attesa di esecuzione.

    Il valore predefinito è 2147483647 e indica che la coda di task è essenzialmente senza limiti e può memorizzare qualsiasi numero di task inviati.

  17. Nella sezione Proprietà aggiuntive, non aggiungere ulteriori proprietà.

    GlassFish Server non definisce alcuna proprietà aggiuntiva per questo tipo di risorsa. Inoltre, questo tipo di risorsa non utilizza attualmente alcuna proprietà aggiuntiva.

  18. Fare clic su Salva.
