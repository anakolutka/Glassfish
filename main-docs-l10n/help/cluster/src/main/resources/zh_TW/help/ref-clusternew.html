新建叢集

使用「新建叢集」頁面可建立叢集。

「新建叢集」頁面包含下列選項。

叢集名稱

叢集的名稱。

名稱必須符合下列需求：

  • 名稱只能包含 ASCII 字元。

  • 名稱必須以字母、數字或底線為開頭。

  • 名稱只能包含下列字元：

    • 小寫字母

    • 大寫字母

    • 數字

    • 連字符

    • 小數點號

    • 底線

  • 名稱在網域中必須是唯一的，且不得為另一個叢集、已命名組態、GlassFish Server 執行處理或節點的名稱。

  • 名稱不得為 domain、server 或 GlassFish Server 所保留的任何其他關鍵字。

組態

叢集可以使用之現有已命名組態的下拉式清單。叢集需要已命名組態，該組態會定義增加至叢集之所有執行處理的組態。您必須從清單中選取一個組態。建立叢集時，叢集將使用所選取的組態。

複製選取的組態

如果選取此選項，就會在建立叢集時複製選取的組態。為組態副本指定的名稱是 cluster-name-config，其中 cluster-name 是在「叢集名稱」欄位中鍵入的叢集名稱。

如果已從「組態」下拉式清單中選取 default-config，則必須選取此選項。default-config 組態僅可複製，無法參照。

參照選取的組態

如果選取此選項，叢集將會使用指定的現有已命名組態。

如果已從「組態」下拉式清單中選取 default-config，則不能選取此選項。default-config 組態無法參照，僅可複製。

訊息佇列叢集組態類型

使用此選項可指定要使用於新 GlassFish Server 叢集的 Message Queue 代理程式叢集類型。依預設，GlassFish Server 會使用含有主代理程式的傳統 Message Queue 代理程式叢集，代理程式會內嵌於 GlassFish Server 叢集中的每個執行處理中。

如果此選項設為「自訂」，就會顯示可組態 Message Queue 代理程式叢集以用於 GlassFish Server 叢集的其他選項。請參閱自訂 Message Queue 代理程式叢集的選項。

要建立的伺服器執行處理

即將在建立叢集時建立的 GlassFish Server 執行處理清單。對於每個執行處理，均提供以下資訊：

執行處理名稱

執行處理的名稱。

加權

一個整數，代表執行處理的負載平衡加權。

負載平衡加權可決定送至叢集供執行處理處理之所有請求的比例。

例如，在有兩個執行處理的叢集中，您可能需要讓其中一個執行處理處理每四個請求中的一個請求，而讓另一個執行處理處理每四個請求中的三個請求。在這種情況下，可將處理每四個請求中之一個請求的執行處理加權設定為 1，將另一個執行處理加權設定為 3。如果您要使用百分比，請將這兩個執行處理的加權分別設定為 25 與 75。

預設加權為 100。

節點

可存在執行處理之現有節點的下拉式清單。您必須從清單中選取一個節點。建立叢集時，執行處理將位於選取的節點上。

要建立的伺服器執行處理表也包含下列選項。

新建

建立執行處理的按鈕。按一下此按鈕即可在要建立的伺服器執行處理表中增加一列。

刪除

刪除執行處理的按鈕。

自訂 Message Queue 代理程式叢集的選項

當「訊息佇列叢集組態類型」設為「自訂」時，就會顯示可組態 Message Queue 代理程式叢集以用於 GlassFish Server 叢集的其他選項。

JMS 服務類型

要用於代理程式叢集中的代理程式類型 (在 GlassFish Server 中稱為 JMS 主機)。選擇的類型會決定可用的代理程式叢集類型。代理程式類型如下：

內嵌

代理程式組態與生命週期由 GlassFish Server 所管理。每個 GlassFish 叢集執行處理均由與執行處理在同一個 JVM 中執行的代理程式提供服務。

如果「JMS 服務類型」選項設為「內嵌」，則僅支援傳統的 Message Queue 代理程式叢集。系統會顯示組態傳統叢集的選項。

本機

代理程式組態與生命週期由 GlassFish Server 所管理。每個 GlassFish 叢集執行處理均由與執行處理在同一主機的不同 JVM 中執行的代理程式提供服務。

如果「JMS 服務類型」選項設為「本機」，則支援傳統與增強的 (高可用性) Message Queue 代理程式叢集。系統會顯示選取叢集類型與組態叢集的選項。

遠端

代理程式與代理程式叢集的組態和生命週期是使用 Message Queue 管理工具所管理的。

如果「JMS 服務類型」選項設為「遠端」，則不會顯示其他選項。

JMS 叢集類型

要使用於 GlassFish Server 叢集的 Message Queue 代理程式叢集類型。

如果「JMS 叢集類型」選項設為「傳統」，會顯示下列選項。

JMS 組態存放區類型

要用於傳統叢集之組態資料的資料存放區類型。如果此選項設為「主代理程式」，會將叢集中的某個代理程式指定為主代理程式，並由其儲存組態資料。如果此選項設為「共用的資料庫」，則組態資料會儲存在所有代理程式可存取的 JDBC 資料存放區中。在此情況下，會顯示與資料庫相關的選項。

JMS 訊息存放區類型

每個代理程式用來儲存其訊息資料的資料存放區類型。如果此選項設為「檔案」，每個代理程式會將其訊息資料儲存在檔案型的資料存放區。如果此選項設為 JDBC，每個代理程式會將其訊息資料儲存在 JDBC 資料存放區。

如果「JMS 叢集類型」選項設為「增強 (HA)」，會顯示與資料庫相關的選項。

資料庫供應商
資料庫 URL
資料庫使用者

要在以下任一情形使用之 JDBC 資料庫的資料庫供應商、存取 URL 以及使用者：

  • 當傳統叢集的組態存放區類型設為「共用資料庫」時

  • 當傳統叢集的訊息存放區類型設為 JDBC 時

  • 當叢集類型設為「增強 (HA)」時

如需有關支援供應商及各供應商存取 URL 格式的相關資訊，請參閱 Oracle GlassFish Server Message Queue Administration Guide 中的 JDBC-Based Persistence。

資料庫認證
資料庫密碼
密碼別名

在「資料庫使用者」中指定的 JDBC 資料庫使用者的密碼資訊。

如果「資料庫認證」設為「密碼」，會顯示「資料庫密碼」選項。如果「資料庫認證」設為「密碼別名」，會顯示「密碼別名」選項。

JMS 叢集特性

組態代理程式所用的一或多個 Message Queue 代理程式特性清單。此清單是以冒號分隔 (:)，格式為：

prop1Name=prop1Value:prop2Name=prop2Value:...

如果代理程式特性名稱包含點，請在點前面加上兩個反斜線 (\\)；例如，若要包含 imq.system.max_count 特性，請在清單中指定 imq\\.system\\.max_count。
