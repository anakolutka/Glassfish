編輯連線池進階屬性標籤

為了協助診斷連線錯誤並增強易用性，特地提供進階屬性，用於在建立連線池時配置該池。

名稱

識別連線池名稱的唯讀欄位。

敘述逾時

在終止執行時間過長的查詢之前，需等待的時間長度 (以秒為單位)。Enterprise Server 將對建立的敘述設定「查詢逾時」。預設值 -1 表示未啟用該屬性。

包裝 JDBC 物件

當此欄位設定為 True 時，應用程式將會取得 Statement、PreparedStatement、CallableStatement、ResultSet、DatabaseMetaData 的包裝 JDBC 物件。預設值為 false。

最多驗證一次

最多對連線進行一次驗證之前的時間 (以秒為單位)。這有助於減少連線發出的驗證請求數目。預設值 0 表示未啟用連線驗證。

錯誤逾時

追蹤連線池中連線錯誤的時間 (以秒為單位)。預設值 0 表示停用連線錯誤追蹤功能。如果啟用連線錯誤追蹤功能，則可以在 [監視資源] 標籤中取得連線錯誤次數的統計資料。若要檢視此標籤，請移至 [Application Server] -> [監視] -> [資源]。

錯誤收回

若啟用該選項，則在錯誤連線追蹤完成之後，會將錯誤的連線復原至連線池。

建立重試嘗試

在建立新連線失敗之後，將進行的嘗試次數。預設值 0 表示不會重新嘗試建立連線。

重試間隔

兩次嘗試建立連線之間的時間間隔 (以秒為單位)。預設值為 10 秒。僅當 [建立重試嘗試] 的值大於 0 時，才使用該屬性。

惰性關聯

若啟用此選項，對連線執行作業時，這些連線便會呈現惰性關聯。此外，當作業事件完成以及元件方法結束時，這些連線便會取消關聯，從而可以重複使用實體連線。預設值為停用。

惰性連線登記

若啟用此選項，則作業事件所使用的資源僅會在實際用於方法中時才會登記。

與執行緒建立關聯

若啟用此選項，將建立連線與執行緒之間的關聯，這樣當相同的執行緒需要連線時，便可重複使用已與該執行緒相關聯的連線，而省下從池中取得連線的經常性耗用時間。預設值為停用。

比對連線

若啟用此選項，符合池的連線會開啟/關閉。可以在以下情況下停用此選項：管理員知道池中的連線始終為同質連線，因此不需要資源介面對從池中挑選的連線進行比對。預設值為停用。

連線的最多使用次數

池應重複使用連線的次數。當連線的重複使用次數達到指定的次數後，將關閉該連線。該選項非常有用，例如可以避免敘述錯誤。預設值 0 表示未啟用該屬性。
