So konfigurieren Sie die Batchlaufzeit

  1. Wählen Sie in der Navigationsbaumstruktur das GlassFish Server-Cluster oder die -Serverinstanz, für die bzw. für das Sie die Batchjobausführungen anzeigen möchten.

    Die Seite "Allgemeine Informationen" wird geöffnet.

  2. Wählen Sie die Registerkarte "Batch".

    Die Seite "Batchjobausführungen" wird geöffnet.

  3. Klicken Sie auf die Registerkarte "Konfiguration".

    Die Seite "Konfiguration der Batchlaufzeit" wird geöffnet. Die Laufzeit verwendet einen Managed Executor-Service sowie eine Datenquelle zur Ausführung von Batchjobs.

  4. Wählen Sie aus der Dropdown-Liste "Lookup-Name des Executor-Service" den Managed Executor-Service, der Threads für Batchjobs bereitstellen soll.

    Der Managed Executor-Standardservice lautet concurrent/__defaultManagedExecutorService. Der Managed Executor-Service kann nach Weiterleitung eines Batchjobs an die Batchlaufzeit geändert werden, ohne die Ausführung des Jobs zu beeinträchtigen.

  5. Wählen Sie aus der Dropdown-Liste "Lookup-Name der Datenquelle" die Datenquelle, in der Jobinformationen gespeichert werden sollen.

    Die Standarddatenquelle lautet jdbc/__TimerPool. Sobald der erste Batchjob an die Batchlaufzeit zur Ausführung weitergeleitet wurde, dürfen Sie die Datenquelle nicht mehr ändern. Wenn die Datenquelle geändert werden muss, stoppen Sie die Domain, und starten Sie sie anschließend neu, und nehmen Sie dann die notwendigen Änderungen vor, bevor Sie Jobs starten bzw. neu starten. Sobald die Datenquelle jedoch geändert wurde, kann auf Informationen in der vorherigen Datenquelle nicht mehr zugegriffen werden.

  6. Klicken Sie auf "Speichern".
