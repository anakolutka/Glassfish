도메인 속성을 설정하는 방법

  1. 탐색 트리에서 [도메인] 노드를 선택합니다.

    [도메인 속성] 페이지가 열립니다.

  2. [DAS 시작 후 콘솔 로드] 필드의 드롭다운 목록에서 다음 옵션 중 하나를 선택합니다.

    사용량 기반

    콘솔이 지난 24시간 내에 사용되었거나 다른 서버 인스턴스가 DAS(도메인 관리 서버)와 별개로 존재하는 경우 DAS 시작 후 관리 콘솔이 로드됩니다. 이 값은 기본값입니다.

    항상

    DAS 시작 후 관리 콘솔이 항상 로드됩니다. 콘솔은 기본 GlassFish Server 시작 시퀀스가 완료된 후에 로드됩니다.

    사용 안함

    DAS 시작 후 관리 콘솔이 로드되지 않습니다. 콘솔은 액세스가 요청될 때 로드됩니다.

  3. 응용 프로그램 루트 필드에 배치된 응용 프로그램이 있는 디렉토리의 전체 경로를 입력합니다.

    기본값은 domain-dir/applications입니다.

  4. [로그 루트] 필드에 서버 로그 파일을 저장할 디렉토리의 전체 경로를 입력합니다.

    기본값은 domain-dir/logs입니다.

  5. 호스트의 기본 로케일을 사용하려면 로케일 필드를 비워 둡니다.

    로케일은 언어 및 지역의 특정 조합을 지정하는 식별자입니다. 예를 들어 미국 영어의 로케일은 en_US이고 일본어의 로케일은 ja_JP입니다. 영어가 아닌 다른 로케일을 사용하려면 GlassFish Server를 현지화(영어를 다른 언어로 번역하는 작업이 포함된 프로세스)해야 합니다.

  6. [저장]을 누릅니다.

  7. 서버를 재시작합니다.

관련 항목

  • DAS에 대한 일반 정보를 보는 방법

  • DAS에 대한 로그 파일을 보는 방법

  • 표시된 로그 레코드를 필터링하는 방법

  • 응용 프로그램 구성을 설정하는 방법

  • 로그를 수집 및 다운로드하는 방법

  • 관리자 비밀번호를 변경하는 방법

  • 시스템 속성을 관리하는 방법

  • 응용 프로그램 모니터링 데이터를 보는 방법

  • 서버 모니터링 데이터를 보는 방법

  • 리소스 모니터링 데이터를 보는 방법
