Redéploiement d'applications ou de modules

Pour redéployer une application qui a déjà été déployée, utilisez la page Redéploiement d'applications ou de modules.

La page Redéploiement d'applications ou de modules contient les options suivantes :

Nom

Champ en lecture seule indiquant le nom de l'application que vous redéployez.

Conserver les sessions

Si la case Conserver les sessions est sélectionnée, les sessions actives de l'application que vous redéployez sont conservées, puis restaurées une fois le redéploiement terminé.

Si la conservation ou la restauration d'une session active de l'application échoue, aucune des sessions ne sera disponible une fois le redéploiement terminé. Toutefois, le redéploiement se poursuit et un avertissement est enregistré.

Pour conserver les sessions actives, Enterprise Server sérialise les sessions et les enregistre dans la mémoire. Pour restaurer les sessions, le chargeur de classe de l'application qui vient d'être redéployée désérialise les sessions qui ont été enregistrées.

Lieu

Le lieu de la nouvelle archive de l'application que vous redéployez.

Les options suivantes indiquent les points d'accès à l'archive et précisent si cette dernière est un fichier ou un répertoire.

Fichier empaqueté à télécharger sur le serveur

L'archive est contenue dans un fichier stocké sur la machine client ou accessible depuis cette même machine.

L'ordinateur client est l'hôte sur lequel vous visualisez la Console d?administration via un navigateur.

Fichier empaqueté ou répertoire local accessible depuis le serveur d'application

L'archive est un fichier stocké sur la machine serveur ou une application non empaquetée contenue dans un répertoire éclaté.

Le serveur est l'hôte qui exécute le serveur d'administration du domaine Application Server.
