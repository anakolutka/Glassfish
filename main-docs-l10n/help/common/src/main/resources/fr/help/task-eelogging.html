Configuration de paramètres de journalisation généraux

  1. Dans l'arborescence de navigation, développez le noeud Configurations.

  2. Sous le noeud Configurations, cliquez sur la configuration d'instance de serveur ou de cluster pour laquelle vous souhaitez définir les paramètres du journaliseur.

    La page Configuration apparaît.

  3. Sur la page Configuration, cliquez sur Paramètres du journaliseur.

    La page Paramètres du journaliseur de la cible de configuration sélectionnée apparaît.

  4. Cochez la case Ecriture dans le journal système activée pour envoyer la sortie de journalisation à l'utilitaire syslog en plus du journal du serveur.

    Cette fonctionnalité est disponible sur les systèmes Solaris et Linux uniquement. Cette option est désactivée par défaut.

  5. Cochez la case Journalisation dans la console activée pour envoyer la sortie de journalisation à la console système.

  6. Cochez la case Modification de la date de rotation activée pour changer de fichier journal du serveur chaque jour à minuit.

  7. Cochez la case Mode multiligne activé pour mettre en forme les messages de journalisation afin d'en faciliter la lecture. Le corps des messages de journalisation est mis en retrait et commence sur une nouvelle ligne.

  8. Dans la liste déroulante Format de journalisation de console, sélectionnez le format de journalisation dans la console. L'option par défaut est ODL (Oracle Diagnostics Logging). L'autre option est ULF (UniformLogFormatter).

  9. Dans la liste déroulante Format de journalisation LogFile, sélectionnez le format de journalisation dans le fichier journal du serveur. L'option par défaut est ODL.

  10. Dans le champ Limite de rotation des fichiers, saisissez la taille en octets que le fichier journal du serveur doit atteindre avant d'être déplacé.

    Lorsque le journal du serveur atteint la taille indiquée, GlassFish Server crée un fichier vide nommé server.log et renomme l'ancien fichier server.log_date, où date indique la date et l'heure auxquelles le fichier est déplacé. La valeur par défaut est 2 Mo. La valeur minimale est 500 ko. Si vous spécifiez une valeur inférieure, le fichier est déplacé lorsqu'il atteint 500 ko. Pour désactiver la rotation du fichier journal, spécifiez la valeur 0.

  11. Dans le champ Temps limite de rotation des fichiers, saisissez le nombre de minutes au bout duquel le journal du serveur doit être déplacé.

    La valeur par défaut est 0 ; cela signifie que le fichier est déplacé lorsqu'il atteint la taille spécifiée dans le champ Limite de rotation des fichiers.

    Remarque:

    Si vous spécifiez une ou plusieurs minutes, le temps limite remplace la taille limite indiquée par le champ Limite de rotation des fichiers.

  12. Dans le champ Fréquence de vidage, saisissez le nombre maximal de messages à écrire simultanément de la file d'attente vers le journal du serveur.

    La valeur par défaut est 1.

  13. Dans le champ Nombre maximal de fichiers d'historique, saisissez le nombre maximal de fichiers journaux que GlassFish Server doit conserver avant de supprimer le fichier le plus ancien. Si vous définissez cette valeur sur zéro (0), GlassFish Server ne supprime pas les anciens fichiers journaux.

  14. Dans le champ Fichier journal, saisissez un autre nom ou emplacement pour le fichier journal du serveur.

    L'emplacement par défaut est ${com.sun.aas.instanceRoot}/logs/server.log.

  15. Dans le champ Gestionnaire du journal, saisissez le nom de classe absolu d'un gestionnaire de journal personnalisé.

    Un gestionnaire de journal personnalisé permet d'envoyer des journaux vers une destination autre que server.log ou syslog. Celui-ci doit étendre la classe java.util.logging.ConsoleHandler (API compatible JSR 047). Ajoutez la classe du gestionnaire au classpath de GlassFish Server de façon à installer le gestionnaire au démarrage du serveur.

  16. Cliquez sur Enregistrer.

    Remarque:

    Selon la configuration pour laquelle la journalisation est activée, vous devrez peut-être redémarrer le serveur.

    Un redémarrage du serveur peut également être requis si le nom du fichier journal est modifié par rapport à la valeur par défaut. Un message Redémarrage requis apparaît dans l'angle supérieur droit de la Console d'administration s'il est nécessaire de redémarrer le serveur.
