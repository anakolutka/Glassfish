Per impostare l'intervallo dei punti chiave

Le operazioni punti chiave determinano la compressione del file di log delle transazioni. Per intervallo punti chiave si intende il numero di transazioni tra un'operazione punto chiave e l'altra nel log. Le operazioni punti chiave possono ridurre la dimensione dei file di log delle transazioni. Con un valore di intervallo punti chiave più elevato i file di log delle transazioni assumono dimensioni maggiori, ma il minor numero delle operazioni punti chiave può garantire prestazioni migliori. Con un valore di intervallo punti chiave più basso, ad esempio 256, si ottengono file di log di dimensioni minori con prestazioni di livello leggermente inferiore a causa della maggiore frequenza di operazioni punti chiave.

  1. Espandere il nodo Configurazione nell'albero di navigazione.

  2. Selezionare l'istanza da configurare:

    • Per configurare un'istanza particolare, selezionare il nodo di configurazione dell'istanza. Ad esempio, per l'istanza predefinita server, selezionare il nodo server-config.

    • Per configurare le impostazioni predefinite per tutte le istanze, selezionare il nodo default-config.

  3. Selezionare il nodo Servizio transazioni.

    Viene visualizzata la pagina Servizio transazioni.

  4. Nel campo Intervallo punti chiave digitare il numero di transazioni tra un'operazione punto chiave e l'altra.

    Il valore predefinito è 65.536.

  5. Fare clic su Salva.
