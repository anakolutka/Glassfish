Asignaciones de Seguridad de Trabajo

Utilice la página Asignaciones de Seguridad de Trabajo para configurar las asignaciones de seguridad de trabajo para los adaptadores de recursos.

Una asignación de seguridad de trabajo asigna la identidad del emisor de la llamada del trabajo enviado por el principal del EIS o un grupo de usuarios del EIS del adaptador de recursos a un principal o grupo de usuarios adecuado del dominio de segurida de GlassFish Server. Se puede asociar una o más asignaciones de seguridad de trabajo a un adaptador de recursos. La configuración de la asignación de seguridad de trabajo del conector soporta el uso del carácter asterisco como comodín (*) para hacer referencia a todos los usuarios o todos los grupos de usuarios.

Una asignación de seguridad de trabajo es diferente de una asignación de seguridad para un pool de conexiones de conectores, el cual asigna el principal recibido durante la autenticación EJB o de servlet a las credenciales aceptadas por el EIS. Consulte Asignaciones de seguridad del pool de conexiones del conector para obtener más información sobre las asignaciones de seguridad relativas a pools de conexiones de conectores.

El sistema de información de empresa (EIS) es cualquier sistema que almacene los datos de una organización. Puede ser un mainframe, un sistema de mensajería, un sistema de base de datos o una aplicación.

Para cada asignación de seguridad de trabajo, se proporciona la siguiente información.

Nombre

Nombre de la asignación de seguridad de trabajo.

La tabla Asignaciones de seguridad de trabajo también contiene las siguientes opciones.

Nuevo

Botón para crear una nueva asignación seguridad de trabajo.

Suprimir

Botón para suprimir una o más asignaciones de seguridad de trabajo seleccionadas.
