批处理运行时配置

GlassFish Server 提供了批处理运行时用于批处理作业的调度和执行。批处理作业通常是长时间运行的、批量执行的任务，其中包含一系列步骤。批处理应用程序将作业提交给批处理运行时，并提供有关执行步骤的方法和时间的指示。批处理运行时处理步骤并将有关作业的信息存储在作业资料档案库中。在 GlassFish Server 中，作业资料档案库是数据库。

使用“批处理运行时配置”页可以查看和更改批处理运行时的配置。批处理运行时使用托管执行程序服务和数据源来执行批处理作业。托管执行程序服务向作业提供线程，而数据源则会存储作业信息。批处理运行时配置数据存储在 domain.xml 的 config 元素中。

“批处理运行时配置”页包含以下信息。

实例名称
集群名称

该配置应用到的 GlassFish 集群或服务器实例的名称。这是只读字段。

执行程序服务查找名称

向批处理作业提供线程的托管执行程序服务的 JNDI 查找名称。默认托管执行程序服务为 concurrent/__defaultManagedExecutorService。

在批处理作业已提交到批处理运行时之后，可以更改托管执行程序服务而不会影响作业的执行。

数据源查找名称

存储有关当前和过去批处理作业的信息的数据源的 JNDI 查找名称。默认数据源为 jdbc/__TimerPool。

在第一个批处理作业已提交到批处理运行时以供执行后，请勿更改数据源。如果必须更改数据源，请停止并重新启动域，然后在任何作业启动或重新启动之前进行更改。但是，一旦更改了数据源，存储在以前数据源中的信息将不可访问。
