配置 HTTP 文件高速缓存

HTTP 文件高速缓存存储静态内容，以便 Enterprise Server 可以快速处理对静态内容的请求。

  1. 在导航树中，展开“配置”节点。
  2. 在“配置”节点下，选择“HTTP 服务”节点。
  3. 从“HTTP 服务”页面中，选择“HTTP 文件高速缓存”选项卡。
  4. 选中“全局启用”复选框以启用全局文件高速缓存。

    默认情况下禁用此选项。

  5. 在“最长生存期”字段中，键入有效高速缓存条目的最长生存期（以秒为单位）。

    该参数用于控制文件经过高速缓存之后，使用高速缓存信息的时间。时间长于最长生存期的条目将用同一文件的新条目替代。如果您的内容很少发生更改，请增大该值以改善性能。

    最优的最长生存期取决于现有文件是否被频繁修改。例如，如果以固定的时间间隔一天修改文件四次，则可以考虑将最长生存期设置为 21,600 秒（6 小时）。否则，可以考虑将最长生存期设置为您希望在上一版本的文件修改后，处理该文件的最长时间。

    默认值为 30。

  6. 在“最大文件计数”字段中，键入文件高速缓存中可以存储的文件的最大数量。

    如果该值过大，服务器将高速缓存较少使用的文件，这样会浪费内存。如果该值过小，将无法体现高速缓存的优点。尝试为该属性设置不同的值，以寻找适合特定应用程序的最佳解决方案。默认值为 1024。

  7. 在“散列初始大小”字段中，键入散列容器的初始数量。

    散列容器的初始数量会影响内存使用和搜索时间。默认值为 0。

  8. 在“中等文件大小限制”字段中，键入可以存储为内存映射文件的高速缓存文件的最大大小（以字节为单位）。

    中等文件的内容通过将文件映射到虚拟内存中来进行高速缓存。最优限制是略高于 VM 页面大小的值。默认值为 537,600。

  9. （仅 UNIX® 平台和 Linux 平台）在“中等文件大小”字段中，键入高速缓存为内存映射文件的所有文件的总大小（以字节为单位）。

    默认值为 10,485,760。

  10. 在“小文件大小限制”字段中，键入可以读入内存的文件的最大大小（以字节为单位）。

    小文件的内容通过分配堆空间并将文件读入该堆空间来进行高速缓存。最优限制是略低于 VM 页面大小的值。默认值为 2048。

  11. 在“小文件大小”字段中，键入要读入内存的文件的总大小（以字节为单位）。

    默认值为 1,048,576。

  12. 设置“启用文件高速缓存”选项，以便在文件大小小于中等文件大小限制时启用或禁用文件内容的高速缓存。
    • 要启用高速缓存，请将此选项设置为“打开”。
    • 要禁用高速缓存，请将此选项设置为“关闭”。
  13. 单击“保存”。

    提示 – 要返回到默认设置，请单击“装入默认值”按钮，然后单击“保存”。
