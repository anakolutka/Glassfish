Cómo configurar las propiedades del administrador

El administrador de sesión proporciona los medios para configurar cómo se crean y destruyen las sesiones, la ubicación en la que se almacena el estado de sesión y el número máximo de sesiones disponibles.

  1. En el árbol de navegación, expanda el nodo Configuración.
  2. En el nodo Configuración, seleccione el nodo Contenedor web.

    Se abrirá la página Contenedor web.

  3. En la página Contenedor web, haga clic en la ficha Propiedades del administrador.
  4. En el campo Intervalo de cierre , introduzca el número de segundos que deben transcurrir antes de que los datos de sesión inactiva se eliminen del almacén.
  5. En el campo Número máximo de sesiones, introduzca el número máximo de sesiones permitidas.
  6. En el campo Nombre de archivo de sesión, introduzca el nombre del archivo que contiene los datos de sesión.
  7. En el campo Nombre de clase del generador de Id. de sesión, especifique una clase personalizada para generar Id. de sesión exclusivos.

    Esta clase debe estar en la ruta de clase de Servidor empresarial. Sólo se permite una clase de generador de Id. de sesión por instancia del servidor. Todas las instancias de un clúster deben utilizar el mismo generador de Id. de sesión para evitar un conflicto de claves en la sesión. Las clases del generador de Id. de sesión personalizadas deben implementar la interfaz com.sun.enterprise.util.uuid.UuidGenerator: 

    package com.sun.enterprise.util.uuid;

public interface UuidGenerator {

    public String generateUuid();
    public String generateUuid(Object obj);  //obj is the session object
}
  8. (Opcional) En la sección Propiedades adicionales, especifique las propiedades adicionales.
  9. Haga clic en "Guardar".
