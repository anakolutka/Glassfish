应用程序配置

使用“应用程序配置”页面可以设置一些属性，通过这些属性可帮助确保能够检测到对部署的应用程序所做的更改并重新装入已修改的类。

重新装入

如果启用了此选项，则动态重新装入已修改的类。服务器将定期检查部署目录中的更改，并自动使用这些更改动态重新部署应用程序。这在开发环境中很有用，因为它允许快速测试代码更改。但在生产环境中，动态重新装入可能会使性能降低。此外，每当完成重新装入时，当时的会话都将无效。客户机必须重新启动该会话。

重新装入轮询时间间隔

确定服务器检查已部署的应用程序中的更改的频率。

管理会话超时

确定在管理控制台超时从而您必须再次登录之前处于非活动状态的分钟数。

自动部署

如果启用了此选项，将自动部署自动部署目录中的应用程序。

自动部署轮询时间间隔

指定服务器在自动部署目录中检查应用程序或模块归档的频率。更改轮询时间间隔不会影响部署应用程序或模块所需的时间。

自动部署重试超时

指定部分复制的文件在发生错误前可以保持大小不变的秒数。默认值为 30。

如果以很慢的速度将文件复制到自动部署目录中，则在复制整个文件之前可以显示该文件。因此，自动部署应用程序的尝试将失败。如果自动部署应用程序的尝试因此而失败，Enterprise Server 将尝试再次部署应用程序。如果部分复制的文件在指定的时间段内保持大小不变，将会发生错误。

自动部署目录

确定用来生成应用程序的目录，这样就不必将文件复制到默认的自动部署目录。

验证器

如果启用了此选项，将在部署前运行验证器。验证器将检查文件的结构和内容。大型应用程序的检验通常会很耗费时间。

预编译

如果启用了此选项，将对 JavaServer PagesTM ( JSPTM) 文件进行预编译。如果禁用此选项，则在运行时首次访问 JSP 文件时对其进行编译。

其他属性
