Ficha Editar atributos avanzados del conjunto de conexiones

Para ayudar a diagnosticar pérdidas de conexión y a mejorar la facilidad de uso, se ofrecen atributos avanzados para configurar un conjunto de conexiones durante su creación.

Nombre

Campo de sólo lectura que identifica el nombre del conjunto de conexiones.

Tiempo de espera de instrucción

Tiempo de espera, en segundos, tras el que finalizarán las consultas cuya ejecución se prolonga de modo inusual.Servidor empresarial establecerá "QueryTimeout" en las instrucciones creadas. El valor predeterminado -1 significa que el atributo no está activado.

Ajustar objetos JDBC

Cuando este campo se establece en "true" (verdadero), las aplicaciones obtendrán objetos JDBC ajustados para Statement, PreparedStatement, CallableStatement, ResultSet y DatabaseMetaData. El valor predeterminado es Falso.

Validar máximo una vez

Tiempo en segundos tras el que una conexión se valida como máximo una vez. Esto ayudará a reducir el número de solicitudes de validación para una conexión. El valor predeterminado 0 significa que la validación de conexión no está activada.

Tiempo de espera de pérdidas

Tiempo en segundos para realizar un seguimiento de las pérdidas de conexión en un conjunto de conexiones. El valor predeterminado 0 significa que el seguimiento de pérdidas de conexión está desactivado. Si el seguimiento de pérdidas de conexión está activado, puede obtener estadísticas sobre el número de pérdidas de conexión en la ficha Recursos de supervisión. Para ver esta ficha, vaya a Application Server ->Supervisión-> Recursos.

Reclamación de pérdidas

Si esta opción está habilitada, las conexiones perdidas se restablecerán en el conjunto tras completar un seguimiento de las pérdidas de conexión.

Intentos de creación

Número de intentos que se realizarán si se produce un fallo en la creación de una nueva conexión. El valor predeterminado 0 significa que no se realizarán intentos para volver a crear la conexión.

Intervalo de reintentos

Intervalo de tiempo, en segundos, entre dos intentos para crear una conexión. El valor predeterminado es 10 segundos. Este atributo se utiliza sólo si el valor de Intentos de creación es mayor que 0.

Asociación mínima

Si esta opción está habilitada, las conexiones estarán ligeramente asociadas cuando se realice una operación con ellas. Además, se disocian cuando la transacción se completa y un método de componentes finaliza, lo que ayuda a reutilizar las conexiones físicas. Esta opción está deshabilitada de forma predeterminada.

Registro de conexiones mínimo

Si esta opción está deshabilitada, sólo se da de alta un recurso en la transacción cuando se utiliza realmente en un método.

Asociar con subproceso

Si esta opción está deshabilitada, se realiza una asociación entre una conexión y el subproceso de forma que cuando el mismo subproceso necesite una conexión, se pueda utilizar la conexión ya asociada a ese subproceso y, por tanto, no se incurra en una sobrecarga a la hora de obtener una conexión del conjunto. El valor predeterminado está desactivado.

Coincidir conexiones

Si esta opción está habilitada, la coincidencia de conexión del conjunto se activa/desactiva. Se puede deshabilitar si el administrador sabe que las conexiones del conjunto siempre serán homogéneas y, por tanto, para una conexión recogida del conjunto no se necesita establecer una correspondencia mediante el adaptador de recursos. Esta opción está deshabilitada de forma predeterminada.

Uso máximo de conexión

Número de veces que una conexión se puede reutilizar en el conjunto. Una vez que una conexión se reutilice el número de veces especificado, se cerrará. Esto es útil, por ejemplo, para evitar pérdidas por instrucción. El valor predeterminado 0 significa que el atributo no está activado.
