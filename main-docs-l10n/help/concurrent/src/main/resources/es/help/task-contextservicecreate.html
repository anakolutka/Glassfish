Para crear un servicio de contexto

Un servicio de contexto se utiliza para crear objetos de proxy dinámicos que capturan el contexto de un contenedor y permiten ejecutar las aplicaciones en ese contexto posteriormente. El contexto del contenedor se propaga al thread que ejecuta la tarea.

En GlassFish Server, puede configurar un recurso de servicio de contexto y ponerlo a disposición de componentes de aplicación, como servlets y EJB.

  1. En el árbol de navegación, amplíe el nodo Recursos.

  2. En el nodo Recursos, amplíe el nodo Recursos simultáneos.

  3. En el nodo Recursos simultáneos, seleccione Servicios de contexto.

    Se abrirá la página Servicios de contexto.

  4. En la página Servicios de contexto, haga clic en Nuevo.

    Se abrirá la página Nuevo servicio de contexto.

  5. En el campo Nombre de JNDI, introduzca un nombre único para identificar el recurso.

    El nombre de JNDI se registra en el servicio de nomenclatura de JNDI y lo utilizan las aplicaciones para consultar el recurso. Utilice el formato concurrent/nombre. Por ejemplo, concurrent/miServicioContextual. El nombre puede contener únicamente caracteres alfanuméricos, guiones bajos, rayas y puntos.

  6. Active la casilla de control Información de contexto activada para propagar contextos a los threads y seleccione los contextos que desea propagar.

    Esta opción está activada y todos los contextos están seleccionados por defecto. Utilice la tecla Mayús o la tecla Ctrl y haga clic para seleccionar varios contextos. Si esta opción está desactivada, los contextos no se propagan, aunque estén seleccionados.

  7. En el campo Descripción, introduzca una breve descripción del recurso.

    Esta descripción es un valor de cadena y puede incluir un máximo de 250 caracteres.

  8. Active la casilla de control Estado activado para activar el recurso.

    Esta opción está activada por defecto, lo que significa que el recurso está disponible inmediatamente después de crearlo.

  9. En la sección Propiedades adicionales, no agregue ninguna propiedad adicional.

    GlassFish Server no define ninguna propiedad adicional para este tipo de recurso. Además, este tipo de recurso no utiliza actualmente ninguna propiedad adicional.

  10. Haga clic en Aceptar.
