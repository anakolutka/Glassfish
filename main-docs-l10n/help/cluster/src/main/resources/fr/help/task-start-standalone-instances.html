Démarrage d'instances GlassFish Server autonomes

Avant de commencer

Assurez-vous que les prérequis suivants sont respectés :

  • Tous les noeuds sur lesquels résident les instances sont activés pour la communication à distance.

  • L'utilisateur du serveur d'administration de domaine (DAS) peut utiliser le protocole DCOM (Distributed Component Object Model) ou SSH (shell sécurisé) pour se connecter à l'hôte pour n'importe quel noeud dans lequel résident les instances dans le cluster.

Si l'un de ces prérequis n'est pas respecté, démarrez chaque instance séparément à partir de la ligne de commande à l'aide de la sous-commande start-local-instance(1).

  1. Dans l'arborescence de navigation, sélectionnez le noeud Instances autonomes.

    La page Instances de serveur autonomes apparaît.

  2. Pour chaque instance à démarrer, cochez la case en regard de l'instance dans la table Instances de serveur.

  3. Cliquez sur Démarrer.

    Vous êtes invité à confirmer que vous souhaitez démarrer les instances.

  4. Confirmez le démarrage des instances.

    Le statut de chaque instance est mis à jour dans la table Instances de serveur au démarrage de l'instance.
