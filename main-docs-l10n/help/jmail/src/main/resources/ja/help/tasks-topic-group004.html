バッチ・ランタイムを構成するには

  1. ナビゲーション・ツリーで、バッチ・ジョブ実行を表示するGlassFish Serverのクラスタまたはサーバー・インスタンスを選択します。

    「一般情報」ページが開きます。

  2. バッチ・タブを選択します。

    バッチ・ジョブ実行ページが表示されます。

  3. 「構成」タブをクリックします。

    バッチ・ランタイムの構成ページが表示されます。ランタイムでは、管理対象エグゼキュータ・サービスおよびデータ・ソースを使用して、バッチ・ジョブが実行されます。

  4. エグゼキュータ・サービス・ルックアップ名ドロップダウン・リストから、バッチ・ジョブへのスレッドの提供に使用する管理対象エグゼキュータ・サービスを選択します。

    デフォルトの管理対象エグゼキュータ・サービスは、concurrent/__defaultManagedExecutorServiceです。管理対象エグゼキュータ・サービスは、バッチ・ジョブがバッチ・ランタイムに送信された後に、ジョブの実行に影響を与えずに変更できます。

  5. データ・ソース・ルックアップ名ドロップダウン・リストから、ジョブ情報の格納に使用するデータ・ソースを選択します。

    デフォルトのデータ・ソースは、jdbc/__TimerPoolです。最初のバッチ・ジョブが実行のバッチ・ランタイムに送信された後に、データ・ソースを変更しないでください。データ・ソースを変更する必要がある場合は、ドメインを停止して再起動し、ジョブが開始または再開される前に変更を行ってください。ただし、データ・ソースが変更されると、前のデータ・ソースに格納されている情報にアクセスできなくなります。

  6. 「保存」をクリックします。
