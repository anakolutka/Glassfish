Mappe di sicurezza lavoro

Utilizzare la pagina Mappe di sicurezza lavoro per configurare le mappe di sicurezza lavoro per i Resource Adapter.

Una mappa di sicurezza lavoro del connettore esegue il mapping tra l'identità del chiamante del lavoro inviato da un principal o un gruppo di utenti EIS del Resource Adapter e un principal o un gruppo di utenti adeguato nel dominio di sicurezza di GlassFish Server. A un Resource Adapter è possibile associare una o più mappe di sicurezza lavoro. La configurazione della mappa di sicurezza lavoro del connettore supporta l'uso del carattere jolly asterisco (*) per indicare tutti gli utenti o tutti i gruppi.

Una mappa di sicurezza lavoro è diversa da una mappa di sicurezza per un connection pool del connettore, che esegue il mapping tra il principal ricevuto durante l'autenticazione EJB o del servlet e le credenziali accettate dal sistema EIS. Per informazioni sulle mappe di sicurezza per i connection pool del connettore, vedere Mappe di sicurezza connection pool connettore.

Per sistemi EIS (Enterprise Information Software) si intendono tutti i sistemi che consentono la gestione dei dati di un'organizzazione. Può essere costituito da un mainframe, un sistema di messaggistica, un sistema di database o un'applicazione.

Per ogni mappa di sicurezza lavoro vengono fornite le informazioni riportate di seguito.

Nome

Il nome della mappa di sicurezza lavoro.

La tabella Mappe di sicurezza lavoro contiene inoltre le opzioni riportate di seguito.

Nuovo

Pulsante che consente di creare una nuova mappa di sicurezza lavoro.

Elimina

Pulsante che consente di eliminare una o più mappe di sicurezza lavoro selezionate.
