Cómo establecer la configuración general de JVM

La máquina virtual de JavaTM (JVM) permite ejecutar código de bytes de Java en un equipo físico. Al ajustar los valores de configuración de JVM, mejora el rendimiento y se evitan errores de asignación de memoria.

La máquina virtual de Java se incluye en la plataforma Java, Standard Edition (plataforma Java SE), que necesita Servidor empresarial. La introducción de valores incorrectos para la configuración de JVM hará que no funcione el servidor, por lo que tiene que ir con cuidado a la hora de cambiar estos valores.

Nota - Los términos “máquina virtual de Java” y “JVM” hacen referencia a una máquina virtual para la plataforma Java.

  1. En el árbol de navegación, seleccione el nodo Configuración.
  2. En el nodo Configuración, seleccione el nodo Configuración de JVM.

    Se abrirá la página Configuración general de JVM.

  3. En la página Configuración general de JVM, en el campo Inicio de Java, escriba el nombre del directorio de instalación del software Java SE.

    Servidor empresarial está basado en la plataforma Java SE. Para comprobar que la versión del software de JAVA SE que ha especificado es compatible con esta versión, consulte las Sun GlassFish Enterprise Server v3 Preview Release Notes . Si introduce un nombre de directorio no existente o un directorio para una versión no compatible del software de Java SE, entonces Servidor empresarial no se podrá iniciar.

  4. En el campo Opciones de Javac, introduzca las opciones de línea de comandos del compilador del lenguaje de programación Java.

    Servidor empresarial ejecuta el compilador cuando se implementan los componentes de Enterprise JavaBeansTM (EJBTM).

  5. Seleccione la casilla de verificación Depuración habilitada para habilitar la depuración con Java Platform Debugger Architecture (JPDA).

    JPDA lo utilizan los desarrolladores de aplicaciones. Para obtener más información, consulte la Sun GlassFish Enterprise Server v3 Preview Application Development Guide .

  6. En el campo Opciones de depuración, introduzca las opciones de JPDA que van a pasar a la JVM.

    Este campo sólo será aplicable si se ha habilitado la depuración en el paso anterior.

  7. En el campo Opciones de compilación RMI, introduzca las opciones de línea de comandos del compilador rmic.

    Servidor empresarial ejecuta el compilador rmic cuando se implementan componentes EJB.

  8. En el campo Preprocesador de código byte, introduzca una lista de nombres de clase.

    Utilice comas para separar los nombres de clase. Cada clase debe implementar la interfaz com.sun.appserv.BytecodePreprocessor . Las clases se llaman en el orden especificado.

  9. (Opcional) En la sección Propiedades adicionales, especifique las propiedades adicionales.

    Para añadir una propiedad, haga clic en el botón Agregar propiedad. En la fila vacía escriba el nombre de la propiedad en el campo Nombre y el valor de la propiedad en el campo Valor.

    Servidor empresarial no define ninguna propiedad adicional para JVM.

  10. Haga clic en Guardar.
  11. Reinicie el servidor.
