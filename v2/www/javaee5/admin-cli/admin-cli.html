Welcome to the Admin Command Line Interface (CLI) home page.  Admin CLI is a client side tool to administer, configure and monitor GlassFish.  This tool is called "asadmin" and it resides in publish/glassfish/bin directory after invoking "maven configure-runtime".

In addition to asadmin, asant is also considered a part of Admin CLI.  asant is an extention to Ant tool developed by the Apache Software Foundation (see http://ant.apache.org/).  asant supports additional built-in tasks that interact with the GlassFish administration utility.

This page is dedicated to dicussing development, testing and other idea relating to Admin CLI.

asadmin
Admin CLI code resides in the module: glassfish/admin-cli.
There are two submodules in admin-cli and they are:  commands and framework.

The commands module contains the code for the CLI commands and framework module contains the code for the CLI framework.  The CLI framework is compliant to Sun's Command Line Interface Paradigm (CLIP)
To checkout and build just the admin-cli, use the following command:

 maven -Dmodules=admin-cli checkout bootstrap build 

please see Build GlassFish for additional details on building GlassFish.
After building this module, admin-cli.jar is created in publish/glassfish/lib directory.
Further details about commands and framework implementation will be added to this section.

asant
asant code resides in the module: glassfish/appserv-common.
This module contains many common code used by deployment, security, config and etc.   The directory path that focuses on asant is in: glassfish/appserv-common/src/java/org.
To checkout and build just  appserv-common, use the following command:

    maven -Dmodules=appserv-common checkout bootstrap build

After buildling this module, sun-appserv-ant.jar is created in publish/glassfish/lib directory.

Unit/Acceptance Tests

Besides executing the QuickLook tests, you may also want to execute the unit tests.
There are 137 tests written for the admin-cli/framework classes.  To execute the unit tests, you must have junit installed or have access to junit.jar file.
Here are the steps to execute junit tests for admin-cli/framework module:
1.  edit frameworkbuild.properties in admin-cli/framework:
- make sure javac.source is equal to 1.5
- set junit.jar to the location where you have junit.jar installed
2. execute the following command in admin-cli/framework:
ant -f frameworkbuild.xml -Dcut=AllTest test


There are 97 tests written for the admin-cli/commands classes. 
Execute the following command in admin-cli/commands to run the junit tests:
ant -f commandsbuild.xml -Dcut=AllTest test
