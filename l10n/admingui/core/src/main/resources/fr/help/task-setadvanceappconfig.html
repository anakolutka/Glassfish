Définir la configuration avancée des applications

Sur la page Configuration des applications, vous pouvez définir les propriétés de configuration des applications qui permettent de s'assurer que les modifications apportées aux applications déployées sont détectées et que les classes modifiées sont rechargées.

  1. Dans l'arborescence de navigation, sélectionnez le nœud Application Server.
  2. Dans la page Informations générales, cliquez sur l'onglet Avancé.

    La page Configuration des applications s'ouvre.

  3. Cochez la case Rechargement activé pour autoriser le rechargement dynamique des applications.

    Lorsque le rechargement dynamique est activé, il n'est pas nécessaire de redéployer une application ou un module lorsque vous changez son code ou ses descripteurs de déploiement. Il suffit de copier les fichiers JSPTM (JavaServer PagesTM) ou de classe modifiés dans le répertoire de déploiement pour l'application ou le module. Le serveur recherche régulièrement la présence de modifications dans le répertoire de déploiement et redéploie l'application, automatiquement et dynamiquement, avec ces modifications. Cette fonction est utile dans un environnement de développement, car elle permet de tester rapidement les modifications du code. Cependant, dans un environnement de production, le rechargement dynamique peut diminuer les performances. De plus, à chaque rechargement, les sessions ouvertes durant cette période de transition ne sont pas valides. Le client doit redémarrer la session.

    Cette option est activée par défaut.

  4. Dans le champ Intervalle d'interrogation de rechargement, saisissez la durée, en secondes, de l'intervalle selon lequel les modifications du code doivent être vérifiées dans les applications et les modules et selon lequel ces derniers doivent être rechargés dynamiquement.

    La valeur par défaut est 2.

  5. Dans le champ Délai d'expiration de session d'administration, saisissez le nombre de minutes d'inactivité après lesquelles la session de la Console d’administration doit expirer.

    La valeur par défaut est 60.

  6. Cochez la case Déploiement automatique activé pour activer le déploiement automatique.

    Le déploiement automatique implique la copie d'un fichier d'application ou de module (JAR, WAR, RAR ou EAR) dans un répertoire spécial, dans lequel il est automatiquement déployé par Application Server. Cette option est désactivée par défaut.

  7. Dans le champ Intervalle d'interrogation de déploiement automatique, saisissez la durée, en secondes, de l'intervalle selon lequel les modifications du code doivent être vérifiées dans les applications et les modules et selon lequel ces derniers doivent être rechargés dynamiquement.

    La valeur par défaut est 2. La modification de l'intervalle d'interrogation n'affecte pas le temps de déploiement d'une application ou d'un module.

  8. Dans le champ Délai d'expiration de la tentative de déploiement automatique, saisissez le nombre de secondes durant lesquelles la taille d'un fichier partiellement copié peut demeurer inchangée avant qu'une erreur ne se produise.

    La valeur par défaut est 30.

  9. Dans le champ Répertoire de déploiement automatique, saisissez le chemin complet du répertoire à surveiller pour le déploiement automatique des applications.
  10. Cochez la case Vérificateur activé pour vérifier les fichiers du descripteur de déploiement.
  11. Cochez la case Précompilation activée pour compiler préalablement les fichiers JSP.
  12. (Facultatif) Dans la section Autres propriétés, spécifiez d'autres propriétés.
  13. Cliquez sur Enregistrer.
  14. Redémarrez le serveur.
