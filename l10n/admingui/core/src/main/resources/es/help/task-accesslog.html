Para configurar el registro de acceso del servicio HTTP

Servidor empresarial utiliza el registro de acceso del servicio HTTP para habilitar y configurar la rotación de los registros de acceso de los servidores virtuales. Estos registros se encuentran en el directorio domain_root_dir/domain_dir /logs/access y tienen los siguientes nombres: virtual_server_name_access_log.yyyyMMdd- HHhmmmsss.txt

  1. En el árbol de navegación, expanda el nodo Configuración.
  2. En el nodo Configuración, seleccione el nodo Servicio HTTP.
  3. En la página Servicio HTTP, haga clic en la ficha Registro de acceso.

    Se abrirá la página Registro de acceso.

  4. Marque la casilla de verificación Rotación activada para habilitar la rotación del archivo de registro.
  5. En la lista desplegable Directiva de rotación, seleccione una directiva de rotación.

    La única directiva de rotación disponible es time.

  6. En el campo Intervalo de rotación, introduzca el número de minutos entre las rotaciones del registro de acceso.

    Este campo sólo es válido si la directiva de rotación es "tiempo". El valor predeterminado es 1.440 minutos (24 horas).

    Asegúrese de que el sufijo de rotación contiene los valores necesarios para crear un sufijo único para cada rotación de registro. Por ejemplo, si el registro de acceso rota más de una vez al día, se incluye la hora del día en el sufijo de rotación.

    Es necesario un único sufijo para asegurar que el archivo de registro de acceso rota cada vez que transcurre el intervalo de rotación . Si el sufijo de rotación del nuevo archivo de registro de acceso es el mismo que el del archivo de registro de acceso existente, el archivo no ha rotado.

  7. En el campo Sufijo de rotación, introduzca el valor de la cadena del sufijo que se va a agregar al final del nombre del archivo de registro tras la rotación.

    El valor predeterminado es yyyyMMdd-HH'h'mm'm'ss's'.

  8. En el campo Formato, introduzca el valor de la cadena para especificar las propiedades siguientes del registro de acceso:

    • Información que se capturará en el registro de acceso

    • Orden en el que se capturará la información

    El valor predeterminado es el siguiente:

    %client.name% %auth-user-name% %datetime% %request% %status% %response.length%

    Los valores siguientes están disponibles para su inclusión en la cadena de formato:

    %client.name%

    Nombre de host del cliente

    %client.dns%

    DNS de cliente

    %datetime%

    Fecha del sistema

    %request%

    Línea de solicitud HTTP completa

    %status%

    Estado

    %response.length%

    Longitud del contenido de respuesta

    %header.referer%

    Encabezado del referenciador

    %header.user-agent%

    Encabezado del agente del usuario

    %http-method%

    Método HTTP

    %http-uri%

    URI de HTTP

    %query-str%

    Cadena de consulta HTTP

    %http-version%

    Versión de protocolo HTTP

    %header.accept%

    Encabezado de aceptación

    %header.date%

    Encabezado de fecha

    %header.if-mod-since%

    Encabezado si modificados desde

    %header.auth%

    Encabezado de autorización

    %header.any%

    Cualquier valor de encabezado HTTP válido definido en RFC 2616 (cualquiera también es un valor de encabezado válido; se especifica como variable aquí)

    %auth-user-name%

    Nombre de usuario autorizado

    %cookie.value%

    Valor de una cookie

    %vs.id%

    Id. de servidor virtual

  9. Haga clic en "Guardar".
