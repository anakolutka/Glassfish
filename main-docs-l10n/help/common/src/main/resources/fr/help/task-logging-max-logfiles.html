Modification du nombre limite de fichiers journaux déplacés

Lorsque GlassFish Server change de fichier journal, il crée un fichier vide nommé server.log et renomme l'ancien fichier server.log_date, où date indique la date et l'heure auxquelles le fichier est déplacé.

Par défaut, GlassFish Server limite le nombre de fichiers journaux déplacés à 10. Lorsque cette limite est atteinte, le fichier journal le plus ancien est supprimé au prochain changement de fichier journal de GlassFish Server.

Si nécessaire, vous pouvez modifier le nombre limite de fichiers journaux déplacés. Toute nouvelle limite définie s'applique aux fichiers journaux déplacés pour la journalisation des accès et le journal du serveur.

  1. Définissez la propriété système com.sun.enterprise.server.logging.max_history_files sur le nombre maximal de fichiers journaux déplacés à conserver.

    Pour plus d'informations sur la définition d'une propriété système, reportez-vous à Gestion des propriétés système.

    Le comportement de la propriété système com.sun.enterprise.server.logging.max_history_files est le suivant :

    • Si la propriété n'est pas définie, GlassFish Server conserve jusqu'à 10 fichiers journaux déplacés.

    • Si la propriété est définie sur un nombre non valide ou NULL, GlassFish Server conserve jusqu'à 10 fichiers journaux déplacés.

    • Si la propriété est définie sur 0, GlassFish Server ne conserve aucun fichier journal déplacé.
