Información general

La página Información general contiene las opciones siguientes.

Iniciar cluster

Botón para iniciar el cluster. Si todas las instancias en el cluster están en ejecución, este botón está desactivado.

Parar cluster

Botón para parar el cluster. Si todas las instancias en el cluster están paradas, este botón está desactivado.

Migrar temporizadores EJB

Botón para migrar temporizadores de tecnología Enterprise JavaBeans (EJB) desde una instancia de GlassFish Server parada a una instancia en ejecución en el cluster. Al hacer clic en este botón se abrirá la página Migrar temporizadores EJB. Si todas las instancias en el cluster están en ejecución, este botón está desactivado.

Rotar log

Botón para rotar el archivo log para el cluster. Cuando se rota el archivo log, se cambia el nombre del archivo por un nombre con registro de hora con el formato server.log_date-and-time y se crea un archivo log vacío para los nuevos mensajes de log. Los cambios se aplican de forma dinámica. No es necesario reiniciar el servidor.

Nombre de cluster

Nombre que se asignó al cluster en el momento de su creación. Este campo es de sólo lectura.

JVM

Si hace clic en Informe de JVM, se abrirá una ventana independiente que muestra informes sobre Java Virtual Machine, incluyendo un informe resumido, un informe sobre gestión de la memoria y recolección de basura, otro informe sobre carga de clase y un volcado de thread actual.

Configuración

Configuración que se seleccionó para el cluster en el momento de su creación. Al hacer clic en la configuración se abrirá la página Configuraciones del cluster. Este campo es de sólo lectura.

GMS

Si esta opción está activada, se inicia el servicio de gestión de grupos (GMS) en cada instancia de GlassFish Server del cluster y en el servidor de administración de dominio (DAS). DAS participa en cada cluster para el que se ha activado GMS. GMS proporciona supervisión del cluster, pertenencia al cluster y servicios de comunicación para los grupos. Esta opción se encuentra activada por defecto.

Puerto de multidifusión

Número del puerto de comunicación en el que GMS recibe los eventos de grupo. Esta opción debe especificar un número de puerto válido entre 2048 y 49151. El valor por defecto se genera automáticamente dentro de este rango.

Dirección de multidifusión

Dirección en la que GMS recibe los eventos de grupo. Esta opción debe especificar una dirección de multidifusión en el rango de 224.0.0.0 a 239.255.255.255. El valor por defecto es 228.9.XX. YY, donde XX e YY son valores independientes generados automáticamente entre 0 y 255.

Dirección de interfaz de enlace

Dirección IP (Internet Protocol) de la interfaz de red a la que se enlaza GMS. Esta opción debe especificar la dirección IP de una interfaz de red local. Los valores por defecto son direcciones de interfaz de red públicas.

En una máquina multiorigen, esta opción configura la interfaz de red usada para GMS. Un equipo multiorigen posee dos o más interfaces de red.

Para especificar una dirección que sea válida para todas las instancias de GlassFish Server del cluster, utilice una propiedad del sistema para definir la dirección de forma individual para cada instancia.

Por ejemplo, utilice la página Propiedades del sistema de cluster para crear la propiedad del sistema GMS-BIND-INTERFACE-ADDRESS-cluster-name. A continuación, defina la opción Dirección de enlace en esta página como ${GMS-BIND-INTERFACE-ADDRESS-cluster-name} para especificar la propiedad del sistema. Por último, para cada instancia del cluster, defina la propiedad del sistema GMS-BIND-INTERFACE-ADDRESS-cluster-name en la dirección de interfaz de red necesaria de la máquina de la instancia.

Estado

Indicación del número de instancias del cluster que están en ejecución y que no lo están. Este campo es de sólo lectura.
