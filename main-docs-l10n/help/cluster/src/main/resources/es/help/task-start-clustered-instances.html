Para iniciar instancias de GlassFish Server en cluster

Antes de Empezar

Asegúrese de que se cumplan los siguientes requisitos previos:

  • Todos los nodos en los que residen las instancias están activados para la comunicación remota.

  • El usuario del servidor de administración de dominio (DAS) puede usar el protocolo remoto de Distributed Component Object Model (DCOM) o Secure Shell (SSH) para conectar al host para cualquier nodo en el que residan las instancias.

Si no se cumplen algunos de estos requisitos previos, inicie cada instancia de forma individual desde la línea de comandos mediante el subcomando start-local-instance(1).

  1. En el árbol de navegación, amplíe el nodo Clusters.

  2. En el nodo Clusters ampliado, seleccione el nodo para el cluster.

    Se abre la página Información General para el cluster.

  3. En la página Información General, haga clic en el separador Instancias.

    Se abrirá la página Instancias del Servidor en Cluster para el cluster.

  4. Para cada instancia que vaya a iniciar, active la casilla de control que aparece junto a la instancia en la tabla Instancias del Servidor.

  5. Haga clic en Iniciar.

    Se le pedirá que confirme que desea iniciar las instancias.

  6. Confirme que desea iniciar las instancias.

    El estado de cada instancia se actualiza en la tabla Instancias del Servidor cuando se inicia la instancia.
