감사 모듈을 만드는 방법

Enterprise Server에서는 간단한 기본 감사 모듈을 제공합니다. 자세한 내용은 기본 감사 모듈을 사용하는 방법 을 참조하십시오.

  1. 구성 노드를 확장합니다.
  2. 구성할 인스턴스를 선택합니다.
    • 특정 인스턴스를 구성하려면 해당 인스턴스 구성 노드를 확장합니다. 예를 들어, 기본 인스턴스 server의 경우 server-config 노드를 확장합니다.
    • 모든 인스턴스의 기본 설정을 구성하려면 default-config 노드를 확장합니다.
  3. 보안 노드를 확장합니다.
  4. 감사 모듈 노드를 선택합니다.
  5. 감사 모듈 페이지에서 새로 만들기를 누릅니다.
  6. 감사 모듈 만들기 페이지에서 다음 정보를 입력합니다.

    • 이름 – 이 감사 모듈을 식별하는 데 사용할 이름을 입력합니다.

    • 클래스 이름 – 이 모듈을 구현하는 정규화된 클래스 이름을 입력합니다. 기본 감사 모듈의 클래스 이름은 com.sun.enterprise.security.Audit입니다.

  7. 이 모듈에 JVM 등록 정보를 추가하려면 등록 정보 추가를 누릅니다. 각 등록 정보에 대해 이름 및 값을 지정합니다.

    유효한 등록 정보에는 이 구현 클래스를 활성화할지 여부를 지정하는 auditOn이 포함되어 있습니다. 유효한 값은 true 및 false입니다.

  8. 확인을 눌러 항목을 저장하거나 취소를 눌러 저장하지 않고 종료합니다.
참조

  • 감사 모듈을 편집하는 방법

  • 감사 모듈을 삭제하는 방법

  • 감사 로깅을 활성화하거나 비활성화하는 방법

  • 활성 감사 모듈을 설정하는 방법

  • 기본 감사 모듈을 사용하는 방법
