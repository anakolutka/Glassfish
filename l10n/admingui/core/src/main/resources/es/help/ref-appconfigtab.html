Página Configuración de aplicaciones

Utilice la página Configuración de aplicaciones para establecer las propiedades que permiten garantizar que los cambios realizados en las aplicaciones implementadas se detecten y las clases modificadas se vuelvan a cargar.

Recargar

Si esta opción está habilitada, las clases modificadas se recargarán de forma dinámica. El servidor comprueba periódicamente los cambios realizados en el directorio de implementación y vuelve a implementar la aplicación de forma automática y dinámica con los cambios. Esto es útil en un entorno de desarrollo ya que permite poner a prueba los cambios realizados en el código de forma rápida. En cambio, en un entorno de producción la recarga dinámica puede hacer que disminuya el rendimiento. Además, cuando ha finalizado la recarga, las sesiones del periodo de transición pierden su validez. El cliente debe reiniciar la sesión.

Intervalo de sondeo de recarga

Identifica con qué frecuencia debe comprobar el servidor los cambios en las aplicaciones implementadas.

Tiempo de espera de sesión de administración

Identifica el número de minutos de inactividad antes de que se agote el tiempo de espera de Consola de administración y se deba iniciar una sesión de nuevo.

Implementación automática

Si esta opción está habilitada, las aplicaciones del directorio de implementación automática se implementan automáticamente.

Intervalo de sondeo de implementación automática

Especifica con qué frecuencia debe comprobar el servidor el directorio de implementación automática de la aplicación o los archivos del módulo. Si cambia el intervalo de sondeo no quedará afectado el tiempo que se tarda en implementar una aplicación o un módulo.

Tiempo de espera de reintento de implementación automática

Especifica el número de segundos que el tamaño de un archivo copiado parcialmente puede permanecer sin cambios antes de que se produzca un error. La opción predeterminada es 30.

Si se copia un archivo lentamente en el directorio de implementación automática, éste puede aparecer antes de que se copie el archivo completo. Como resultado, se produce un error al intentar implementar la aplicación de forma automática. Si se produce un error al intentar implementar una aplicación de forma automática por este motivo, Servidor empresarial intentará implementar de nuevo la aplicación. Si el tamaño del archivo copiado parcialmente permanece sin cambios durante el período de tiempo especificado, se producirá un error.

Directorio de implementación automática

Identifica el directorio en el que se crea la aplicación, por lo que no tendrá que copiar los archivos en el directorio de implementación automática predeterminado.

Verificador

Si se habilita esta opción, el verificador se ejecutará antes de la implementación. El verificador examinará la estructura y contenido del archivo. La verificación de aplicaciones de gran tamaño tiende a llevar mucho tiempo.

Precompilar

Si está habilitada esta opción, los archivos JavaServer PagesTM (JSPTM) se precompilarán. Si se deshabilita esta opción, los archivos JSP se compilarán en el tiempo de ejecución cuando se acceda a los mismos por primera vez.
