HTTP 서비스 요청 처리를 구성하는 방법

  1. 탐색 트리에서 구성 노드를 선택합니다.
  2. HTTP 서비스 노드를 선택합니다.
  3. HTTP 서비스 페이지에서 요청 처리 탭을 선택합니다.

    요청 처리 페이지가 열립니다.

  4. 스레드 수 필드에 요청 처리 스레드의 최대 수를 입력합니다.

    기본값은 20입니다.

  5. 초기 스레드 수 필드에 서버가 시작할 때 사용할 수 있는 요청 처리 스레드 수를 입력합니다.

    기본값은 2입니다.

  6. 스레드 증분 필드에 요청 수가 초기 스레드 수를 초과했을 때 추가될 요청 처리 스레드 수를 입력합니다.

    기본값은 1입니다.

  7. 요청 시간 초과 필드에 요청 시간이 초과되기 전까지의 시간(초)을 입력합니다.

    기본값은 30입니다.

  8. 버퍼 길이 필드에 요청 처리 스레드에서 요청 데이터를 읽을 때 사용할 버퍼의 크기(바이트)를 입력합니다.

    기본값은 8192입니다.

  9. 저장을 누릅니다.
참조

  • HTTP 서비스 액세스 로그를 구성하는 방법

  • HTTP 서비스 연결 유지를 구성하는 방법

  • HTTP 서비스 연결 풀을 구성하는 방법

  • HTTP 프로토콜을 구성하는 방법

  • HTTP 파일 캐시를 구성하는 방법
