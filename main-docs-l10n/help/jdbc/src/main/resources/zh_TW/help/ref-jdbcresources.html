JDBC 資源

Java Database Connectivity (JDBC) 資源 (資料來源) 提供應用程式連線至資料庫的方法。管理員一般會為每個資料庫 (由建置於網域的應用程式所存取) 建立一個 JDBC 資源，但是您可為一個資料庫建立多個 JDBC 資源。

應用程式會在 Java Naming and Directory Interface (JNDI) API 樹狀結構上查找資料來源並請求連線，以便從連線集區取得資料庫連線。與資料來源相關聯的連線集區可提供與應用程式的連線。

使用「JDBC 資源」頁面可組態 JDBC 資源。

對於每個資源，都將提供以下資訊。

JNDI 名稱

可識別 JDBC 資源的唯一名稱。

邏輯 JNDI 名稱

資源的邏輯 JNDI 名稱。請參閱 Logical JNDI Names 以瞭解詳細資訊。

已啟用

(網域中有預設伺服器執行處理 server 存在時才會顯示此欄。)

如果已啟用資源則為勾選記號，如果停用資源則為打叉記號 (X)。

狀態

(已在網域中建立叢集或獨立執行處理時才會顯示此欄。)

設定為資源目標的叢集和獨立執行處理數目，以及資源中啟用的目標數。例如，若為「已在 2 個 (共 4 個) 目標上啟用」，表示資源的目標有四個叢集和獨立執行處理，而且已在這四個目標的其中兩個啟用資源。

連線集區

與資源關聯的 JDBC 連線集區。

描述

資源的簡短描述。

資源表也包含下列選項。

新建

建立新資源的按鈕。

刪除

刪除一或多個所選取資源的按鈕。

啟用

啟用一或多個所選取資源的按鈕。

停用

停用一或多個所選取資源的按鈕。
