バッチ・ランタイムの構成

GlassFish Serverは、バッチ・ジョブのスケジュールと実行用にバッチ・ランタイムを提供します。バッチ・ジョブは一般的に、長期実行のバルク指向のタスクで、一連のステップが含まれます。バッチ・アプリケーションはバッチ・ランタイムにジョブを送信し、ステップの実行方法やタイミングを指示します。バッチ・ランタイムはステップを処理し、ジョブ・リポジトリのジョブに関する情報を格納します。GlassFish Serverでは、ジョブ・リポジトリはデータベースです。

バッチ・ランタイムの構成を表示および変更するには、バッチ・ランタイムの構成ページを使用します。バッチ・ランタイムでは、管理対象エグゼキュータ・サービスおよびデータ・ソースを使用して、バッチ・ジョブが実行されます。管理対象エグゼキュータ・サービスはジョブにスレッドを提供し、データ・ソースにジョブ情報が格納されます。バッチ実行時の構成データは、domain.xmlのconfig要素に格納されます。

バッチ・ランタイムの構成ページには、次の情報が表示されます。

インスタンス名
クラスタ名

構成が適用されるGlassFishクラスタまたはサーバー・インスタンスの名前。このフィールドは読取り専用です。

エグゼキュータ・サービス・ルックアップ名

バッチ・ジョブにスレッドを提供する管理対象エグゼキュータ・サービスのJNDIルックアップ名。デフォルトの管理対象エグゼキュータ・サービスは、concurrent/__defaultManagedExecutorServiceです。

管理対象エグゼキュータ・サービスは、バッチ・ジョブがバッチ・ランタイムに送信された後に、ジョブの実行に影響を与えずに変更できます。

データ・ソース・ルックアップ名

現在と過去のバッチ・ジョブに関する情報を格納するデータ・ソースのJNDIルックアップ名。デフォルトのデータ・ソースは、jdbc/__TimerPoolです。

最初のバッチ・ジョブが実行のバッチ・ランタイムに送信された後に、データ・ソースを変更しないでください。データ・ソースを変更する必要がある場合は、ドメインを停止して再起動し、ジョブが開始または再開される前に変更を行ってください。ただし、データ・ソースが変更されると、前のデータ・ソースに格納されている情報にアクセスできなくなります。
