編輯 HTTP 偵聽程式

  1. 在瀏覽樹狀結構中，展開 [配置] 節點。
  2. 在 [配置] 節點下，展開 [HTTP 服務] 節點。
  3. 在 [HTTP 服務] 節點下，選取 [HTTP 偵聽程式] 節點。
  4. 在 [HTTP 偵聽程式] 頁面上，按一下您要編輯的偵聽程式名稱。

    如果要編輯名為 admin-listener 的特定 HTTP 偵聽程式，必須重新啟動伺服器才能使變更生效。管理主控台無法指出在此情形下必須重新啟動的情況。

  5. 選取 [啟用偵聽程式] 核取方塊，在每次重新啟動伺服器時啟用此偵聽程式。
  6. 選取 [啟用安全性] 核取方塊，確保此偵聽程式的安全。
  7. 在 [網路位址] 欄位中，鍵入伺服器的 IP 位址。

    若偵聽程式將偵聽該伺服器的所有 IP 位址，請在 [網路位址] 欄位中鍵入 0.0.0.0。

  8. 在 [偵聽程式連接埠] 欄位中，鍵入此偵聽程式的連接埠值。
    • 若在步驟 7 中鍵入 0.0.0.0，請確認此伺服器沒有任何 IP 位址正在使用該偵聽程式連接埠值。
    • 否則，請確認所指定的 IP 位址未使用該偵聽程式連接埠值。
  9. 從 [預設虛擬伺服器] 下拉式清單中，選取一部虛擬伺服器。
  10. 在 [伺服器名稱] 欄位中，鍵入要在伺服器傳送給用戶端的 URL 中使用的主機名稱。

    如果伺服器使用別名，則該名稱應為此別名。如果伺服器未使用別名，請將該欄位保留為空。

  11. 若要將請求重新導向至其他連接埠，請在 [重新導向連接埠] 欄位中鍵入一個值。

    如果滿足以下兩個條件，Enterprise Server 將自動重新導向請求：

    • 該偵聽程式支援非 SSL 請求。

    • 接收了符合安全性限制需要 SSL 傳輸的請求。

      依預設，Enterprise Server 使用原始請求中指定的連接埠號。

  12. 在 [接受器執行緒] 欄位中，鍵入此偵聽程式的執行緒數目。
  13. 取消選取 [啟用 Powered By] 核取方塊，在 Servlet 產生的 HTTP 回應標頭中停用 X-Powered-By: Servlet/2.4 標頭。

    Java Servlet 2.4 規格中定義了該標頭，其容器可以增加到 Servlet 產生的回應。類似的情況，JavaServer Pages (JSP) 2.0 規格中定義了一個 X-Powered-By: JSP/2.0 標頭，而該標頭將會增加到使用 JSP 技術的回應 (此為選用步驟)。依預設，將為 Web 應用程式啟用 X-Powered-By: JSP/2.0 標頭。這些標頭的目標是協助網站管理員收集有關使用 Servlet 和 JSP 技術的統計資料。

    如需有關啟用和停用 JSP 檔案的 X-Powered-By 標頭之資訊，請參閱「Sun GlassFish Enterprise Server v3 Prelude Application Deployment Guide」中的附錄 A「Deployment Descriptor Files」。

    依據生產環境的不同，可以省略 X-Powered-By 標頭的產生，以隱藏基礎技術。

  14. 選取 [區段化已啟用] 核取方塊，啟用此偵聽程式和外部連接埠的區段化功能。
  15. (可選擇) 在 [其他特性] 區段中，指定其他特性。
  16. 按一下 [儲存]。
