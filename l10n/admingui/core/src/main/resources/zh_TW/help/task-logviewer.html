檢視伺服器記錄

  1. 在瀏覽樹狀結構中，選取 [Enterprise Server] 節點。

    此時將開啟 [一般資訊] 頁面。

  2. 在 [一般資訊] 頁面上，按一下 [檢視記錄檔] 按鈕。

    此時將開啟 [記錄檢視器] 視窗。

  3. 使用 [搜尋條件] 區域自訂和篩選記錄檢視器。
    • 按一下 [搜尋條件] 區域中的 [進階搜尋] 以進一步細化記錄檢視器的搜尋條件。
    • 在 [時間戳記] 欄位中：
      • 若要檢視最近訊息，請選取 [最近] (預設)。
      • 若要僅檢視特定時段內的訊息，請選取 [指定範圍]，然後在所顯示的 [從] 和 [到] 欄位中鍵入日期和時間值。

        對於時間值，必須採用以下語法格式 (SSS 表示毫秒)：

        hh:mm:ss.SSS

        例如︰

        17:10:00.000

        如果 [從] 欄位中的時間值遲於 [到] 欄位中的時間值，將顯示錯誤訊息。

    • 從 [記錄層級] 下拉式清單中，選取記錄層級，依記錄層級篩選訊息。選取標有 [請勿包含更多嚴重訊息] 的核取方塊，以僅顯示所選層級的訊息。

      依預設，將顯示伺服器記錄中選取記錄層級和更高層級的所有訊息。

      為確保要檢視的訊息都顯示在伺服器記錄中，請先在 [模組記錄層級] 頁面上設定適當的記錄層級。請參閱配置記錄層級。

      如果選擇基於記錄層級篩選記錄訊息，則僅顯示符合指定篩選條件的訊息。不過，這種篩選不影響那些記錄到伺服器記錄中的訊息。

  4. 如果您在 [搜尋條件] 區域中按下了 [進階搜尋]，請執行下列步驟。
    1. 從 [記錄程式] 下拉式清單中，選取一個或多個名稱空間，依模組進行篩選。

      如果要選擇多個名稱空間，請在按住 Shift 鍵或 Ctrl 鍵的同時以滑鼠點選。

      選取較高層級的名稱空間時，也會連帶選取下層的所有名稱空間。例如，選取 [javax.enterprise.system] 的同時也就選取了該名稱空間下所有模組的記錄程式︰ javax.enterprise.system.core、javax.enterprise.system.tools.admin 等等。

    2. 在 [自訂記錄程式] 欄位中，一行鍵入一個記錄程式名稱，以從特定應用程式專屬的記錄程式檢視訊息。

      如果應用程式具有多個模組，則可以檢視其中任意一個模組或者全部模組。例如，假設應用程式具有使用以下名稱的記錄程式：

      com.mycompany.myapp.module1
com.mycompany.myapp.module2
com.mycompany.myapp.module3

      若要檢視應用程式中所有模組的訊息，請鍵入 com.mycompany.myapp。若要僅檢視 module2 的訊息，請鍵入 com.mycompany.myapp.module2。

      如果指定了一個或多個自訂記錄程式，則僅當在 [記錄程式] 區域中明確指定 Enterprise Server 模組時，才會顯示這些模組的訊息。

    3. 在 「名稱-值」對欄位中，鍵入鍵值名稱和值，以供該執行緒檢視特定執行緒的輸出。

      金鑰名稱為 _ThreadID。例如︰

      _ThreadID=13

      假設 com.mycompany.myapp.module2 在多個執行緒中執行。若要細化記錄檢視器以僅顯示單一執行緒的輸出，請在 [自訂記錄程式] 欄位中指定該模組的記錄程式，然後在該欄位中指定執行緒 ID。

    4. 從 [顯示] 下拉式清單中，選取一次要檢視的訊息數目值。

      顯示的預設訊息數目為 40。其他選項包括 100、250 和 1000。

      若要檢視堆疊追蹤，請取消選取 [限制過長訊息] 核取方塊。依預設，堆疊追蹤不在檢視器中顯示；若要檢視它們，請按一下訊息的 [(details)] 連結。

    5. 按一下 [基本搜尋] 以隱藏 [進階選項] 區域。
  5. 在 [搜尋結果] 區域中，自訂記錄訊息的顯示。

    [搜尋結果] 區域透過在 [記錄設定] 和 [記錄層級] 頁面中指定的設定，顯示伺服器記錄中的 40 個最新的項目。

    • 按一下欄中的箭頭可以用不同方式排序訊息。
    • 按一下標有 [(details)] 的連結可檢視任何訊息的格式化版本。

      將顯示標有 [記錄項目詳細資訊] 的視窗，其中包含訊息的格式化版本。

    • 按一下表格頂部和底部的按鈕可以檢視記錄檔中較早或較晚的項目。
