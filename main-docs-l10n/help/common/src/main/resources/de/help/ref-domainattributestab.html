Domainattribute

Verwenden Sie die Seite "Domainattribute", um die erweiterten Domainattribute für GlassFish Server festzulegen.

Die Seite "Domainattribute" umfasst die folgenden Optionen.

Konsole nach Hochfahren des DAS laden

Legt fest, wann die Administrationskonsole hochgeladen werden soll:

Verwendungsbasiert

Lädt die Administrationskonsole nach dem Domain Administration Server (DAS), wenn die Konsole innerhalb der letzten 24 Stunden verwendet wurde oder wenn neben dem DAS noch andere Serverinstanzen vorhanden sind. Dies ist die Standardeinstellung.

Immer

Lädt die Administrationskonsole immer nach dem DAS. Die Konsole wird nach Abschluss der Hauptstartsequenz von GlassFish Server geladen.

Nie

Lädt die Administrationskonsole nie nach dem DAS. Die Konsole wird geladen, wenn der Zugriff darauf angefordert wird.

Anwendungs-Root

Der vollständige Pfad des Verzeichnisses, in dem sich bereitgestellte Anwendungen befinden. Der Standardwert ist domain-dir/applications.

Log-Root

Gibt an, wo die Serverlogdateien gespeichert sind. Der Standardwert istdomain-dir/logs.

Gebietsschema

Wenn Sie dieses Feld leer lassen, wird das Standardgebietsschema des Hosts verwendet.

Ein Gebietsschema ist eine ID, die eine bestimmte Kombination aus Sprache und Region angibt. Beispiel: Das Gebietsschema für Englisch (USA) ist en_US, das Gebietsschema für Japanisch ist ja_JP. Um ein nicht englisches Gebietsschema zu verwenden, muss GlassFish Server lokalisiert, d.h. in eine andere Sprache übersetzt werden.

Verwandte Aufgaben

  • So wird die Anwendungskonfiguration festgelegt

  • So werden die Domainattribute festgelegt
