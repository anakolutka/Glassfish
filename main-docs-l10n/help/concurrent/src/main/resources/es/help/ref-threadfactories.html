Fábricas de threads gestionados

Las aplicaciones utilizan las fábricas de threads gestionados para crear threads gestionados a petición. El contenedor inicia y gestiona los threads. El contexto del contenedor se propaga al thread que ejecuta la tarea.

Utilice la página Fábricas de threads gestionadas para configurar recursos de la fábrica de threads gestionada.

En cada recurso de la fábrica de threads gestionada, aparece la siguiente información.

Nombre JNDI

Nombre único que identifica el recurso.

Nombre de JNDI lógico

Nombre de JNDI lógico para el recurso. Consulte Nombres de JNDI lógicos para obtener más información.

Activado

(Esta columna sólo se muestra si la instancia del servidor por defecto, server, existe en el dominio).

Una marca de control si el recurso está activado, o una X si está desactivado.

Estado

(Esta columna sólo se muestra si en el dominio se ha creado algún cluster o instancia independiente).

Número de clusters e instancias independientes a los que se dirige el recurso y el número de estos destinos en los que el recurso está activado. Por ejemplo, "Activado en 2 de 4 de destinos" significa que el recurso se dirige a cuatro clusters e instancias independientes y que está activado en dos de estos cuatro destinos.

Información de contexto

Contextos de contenedor propagados a los threads.

Prioridad de threads

Prioridad asignada a los threads.

Descripción

Descripción del recurso.

La tabla Fábricas de threads gestionadas también contiene las siguientes opciones.

Nuevo

Botón para crear un nuevo recurso.

Suprimir

Botón para suprimir uno o varios recursos seleccionados.

Activar

Botón para activar uno o varios recursos seleccionados.

Desactivar

Botón para desactivar uno o varios recursos seleccionados.
