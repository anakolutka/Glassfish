Configurações do Agente de Log

Use a página Configurações do Agente de Log para configurar o log do Enterprise Server.

A página Configurações do Agente de Log contém as seguintes opções.

Gravar no Log do Sistema

Se esta opção estiver selecionada, a saída do log será enviada ao recurso syslog além do log do servidor. Esse recurso está disponível apenas nos sistemas Solaris e Linux. Por padrão, a opção está desativada.

Log no Console

Se essa opção for selecionada, NÃO FAÇO IDÉIA do que acontecerá. Por padrão, a opção está desativada.

Arquivo de Log

Um nome alternativo ou local do arquivo do log do servidor. O local padrão é glassfish/domains/domain1/logs/server.log.

Manipulador de Logs

O nome da classe absoluta de um manipulador de logs personalizado. Um manipulador de logs personalizado permite que você envie logs para um destino diferente de server.log ou syslog O manipulador personalizado deve estender a classe java.util.logging.Handler (uma API compatível com JSR 047). Coloque a classe do manipulador no caminho da classe do Enterprise Server, para que o manipulador seja instalado durante a inicialização do servidor.

Limite de Rotação de Arquivo

O tamanho em bytes que o arquivo do log do servidor deve alcançar antes de ser rotacionado. Quando o log do servidor alcança o tamanho especificado, o Enterprise Server cria um arquivo novo, vazio, chamado server.log e renomeia o arquivo antigo para server.log_date, onde date é a data e a hora em que o arquivo foi rotacionado. O valor padrão é 2 megabytes. O valor mínimo para o limite é de 500 quilobytes; se você especificar um valor menor, o arquivo será rotacionado quando alcançar 500 KB. Para desativar a rotação de arquivos de log, defina o valor para 0.

Limite de Tempo de Rotação de Arquivo

O número de minutos após o qual o log do servidor deve ser rotacionado. O valor padrão é 0, o que significa que o arquivo é rotacionado quando ele alcança o tamanho especificado no campo Limite de Rotação de Arquivo. Se você especificar um ou mais minutos, o limite de tempo tem precedência sobre o limite de tamanho especificado pelo campo Limite de Rotação de Arquivo.
