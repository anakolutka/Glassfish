Configuration d'exécution de batch

GlassFish Server fournit une exécution de batch pour la planification et l'exécution de traitements batch. Les traitements batch sont généralement des tâches à longue durée d'exécution en masse contenant une série d'étapes. Les applications de traitement batch soumettent des travaux à l'exécution de batch et fournissent des instructions sur la méthode et le moment d'exécution des étapes. L'exécution de batch traite les étapes et stocke les informations relatives aux travaux dans un référentiel de travaux. Dans GlassFish Server, le référentiel de travaux est une base de données.

Utilisez la page Configuration d'exécution de batch pour visualiser et modifier la configuration de l'exécution de batch. Celle-ci utilise un service exécuteur géré et une source de données pour exécuter les traitements batch. Le service exécuteur géré fournit des threads aux travaux et la source de données stocke les informations des travaux. Les données de configuration de l'exécution du batch sont stockées dans l'élément config du fichier domain.xml.

La page Configuration d'exécution de batch contient les informations ci-après.

Nom d'instance
Nom du cluster

Nom de l'instance de serveur et du cluster GlassFish auquel s'applique la configuration. Ce champ est en lecture seule.

Nom de recherche de service exécuteur

Nom de recherche JNDI du service exécuteur géré qui fournit des threads aux traitements batch. Le service exécuteur géré par défaut est concurrent/__defaultManagedExecutorService.

Le service exécuteur géré peut être modifié une fois qu'un traitement batch a été soumis sur l'exécution de batch sans incidence sur l'exécution du travail.

Nom de recherche de source de données

Nom de recherche JNDI de la source de données qui stocke les informations relatives aux traitements batch en cours et passés. La source de données par défaut est jdbc/__TimerPool.

Ne modifiez pas la source de données une fois le premier traitement batch soumis à l'exécution de batch pour exécution. Si la source de données doit être modifiée, arrêtez et redémarrez le domaine, puis appliquez la modification avant le démarrage ou le redémarrage de tout travail. Cependant, une fois que la source de données a été modifiée, les informations stockées dans la source de données précédente deviennent inaccessibles.
