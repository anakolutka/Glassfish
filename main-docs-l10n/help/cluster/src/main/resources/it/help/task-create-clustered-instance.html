Per aggiungere un'istanza di GlassFish Server a un cluster

Le istanze di GlassFish Server costituiscono la base per la distribuzione di un'applicazione. Un'istanza in cluster è un membro del cluster. Ogni istanza appartiene a un solo dominio e dispone di una struttura di directory, di una configurazione e di applicazioni distribuite proprie. Ogni istanza include inoltre il contenitore Web Java Platform, Enterprise Edition (Java EE) e il contenitore di tecnologia EJB (Enterprise JavaBeans). Ogni nuova istanza deve contenere un riferimento a un nodo che definisce l'host in cui risiederà l'istanza.

Nota:

Per creare un'istanza in un nodo di tipo CONFIG che rappresenta un host remoto, è necessario utilizzare il comando secondario create-local-instance(1). Per questa operazione non è possibile utilizzare Console di amministrazione.
Prima di iniziare

Verificare l'esistenza degli elementi riportati di seguito.

  • Il cluster al quale aggiungere l'istanza. Per le necessarie istruzioni, vedere Per creare un cluster.

  • Il nodo in cui deve risiedere l'istanza. Per le necessarie istruzioni, vedere Per creare un nodo.

  1. Espandere il nodo Cluster nell'albero di navigazione.

  2. Selezionare il nodo per il cluster sotto il nodo Cluster espanso.

    Viene visualizzata la pagina Informazioni generali per il cluster.

  3. Nella pagina Informazioni generali, fare clic sulla scheda Istanze.

    Viene visualizzata la pagina Istanze server in cluster per il cluster.

  4. Nella pagina Istanze server in cluster, fare clic su Nuovo.

    Viene visualizzata la pagina Nuova istanza server in cluster.

  5. Nel campo Nome digitare il nome per l'istanza.

    Il nome deve soddisfare i requisiti riportati di seguito.

    • Il nome può contenere solo caratteri ASCII.

    • Il nome deve iniziare con una lettera, un numero o un carattere di sottolineatura.

    • Il nome può contenere solo i caratteri seguenti:

      • Lettere minuscole

      • Lettere maiuscole

      • Numeri

      • Trattino

      • Punto

      • Carattere di sottolineatura

    • Il nome deve essere univoco nel dominio e non deve essere il nome di un'altra istanza di GlassFish Server, di un cluster, di una configurazione denominata o di un nodo.

    • Il nome non deve essere domain, server né un'altra parola chiave riservata di GlassFish Server.

  6. Nell'elenco a discesa Nodo selezionare il nodo in cui risiederà l'istanza.

  7. Fare clic su OK.
