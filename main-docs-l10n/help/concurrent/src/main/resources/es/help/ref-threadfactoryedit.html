Editar fábrica de threads gestionada

Utilice la página Editar fábrica de threads gestionada para editar un recurso de la fábrica de threads gestionada.

La página Editar fábrica de threads gestionada contiene las siguientes opciones.

Cargar Valores por Defecto

Botón para restaurar configuración que tiene valores por defecto a sus valores por defecto. Las configuraciones que no tienen valores por defecto no se modifican.

Nombre JNDI

Nombre único que identifica el recurso. El nombre JNDI es un campo de sólo lectura. Sólo puede especificar el nombre JNDI al crear un nuevo recurso.

Nombre de JNDI lógico

Nombre de JNDI lógico para el recurso. Se trata de un nombre estándar de Java EE a través del que se puede acceder al recurso. El nombre de JNDI lógico es un campo de sólo lectura. Consulte Nombres de JNDI lógicos para obtener más información.

Información de contexto

Contextos que se propagarán a los threads: Classloader, JNDI, seguridad o área de trabajo. La propagación de contexto se puede activar o desactivar. Si está activada, se propagan los contextos seleccionados. Si está desactivada, no se propaga ninguno de los contextos, aunque estén seleccionados. El valor por defecto es Activado, con los cuatro contextos seleccionados.

Estado

Indica si el recurso está disponible en el tiempo de ejecución.

Si sólo existe la instancia de servidor por defecto, server en el dominio, la disponibilidad en tiempo de ejecución del recurso puede estar activada o desactivada. La disponibilidad en tiempo de ejecución está activa por defecto.

Si se ha creado algún cluster o alguna instancia independiente en el dominio, utilice el separador Destino para controlar la disponibilidad del recurso en tiempo de ejecución en sus destinos.

Prioridad de threads

Prioridad que se asignará a los threads creados. El valor por defecto es 5.

Orden de despliegue

Orden de despliegue de este recurso. Los recursos con los números más bajos se cargan primero al iniciar el servidor. El valor por defecto es 100.

Descripción

Descripción de este recurso.

Propiedades Adicionales

Propiedades adicionales del recurso. GlassFish Server no define ninguna propiedad adicional para este tipo de recurso. Además, este tipo de recurso no utiliza actualmente ninguna propiedad adicional.
