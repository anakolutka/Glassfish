연결 풀 고급 속성 편집

연결 풀을 만들 때 연결 누수를 진단하고 사용 편의성을 향상시키기 위해 연결 풀을 구성하는 데 사용되는 고급 속성이 제공됩니다.

이름

연결 풀의 이름을 식별하는 읽기 전용 필드입니다.

명령문 시간 초과

비정상적으로 길게 실행되는 쿼리를 종료하기 전까지의 시간 길이(초)입니다. Enterprise Server에서는 명령문을 만들 때 "QueryTimeout"을 설정합니다. –1 값은 옵션이 비활성화되었음을 의미합니다. 기본값은 –1입니다.

JDBC 객체 래핑

이 필드를 true로 설정할 경우 응용 프로그램이 Statement, PreparedStatement, CallableStatement, ResultSet, DatabaseMetaData에 대해 래핑된 JDBC 객체를 가져옵니다기본값은 false입니다.

최대 1회 검증 기간

연결이 최대 한 번 검증되기까지의 시간(초)입니다. 연결의 검증 요청 수를 줄이는 데 도움이 됩니다. 0 값은 연결 검증이 비활성화되었음을 의미합니다. 기본값은 0입니다.

누수 시간 초과

연결 풀의 연결 누수를 추적하는 시간(초)입니다. 기본값 0은 연결 누수 추적이 비활성화됨을 의미합니다. 연결 누수 추적이 활성화되면 자원 모니터링 탭에서 연결 누수의 수에 대한 통계를 얻을 수 있습니다. 이 탭을 보려면 Application Server -> 모니터링 -> 자원으로 이동하십시오.

누수 재생

이 옵션을 사용하도록 설정된 경우 누수된 연결이 누수 연결 추적 완료 후 풀로 복원됩니다.

만들기 재시도 횟수

새 연결 만들기에 실패한 경우 재시도하는 횟수입니다. 0 값은 연결 만들기를 다시 시도하지 않음을 의미합니다. 기본값은 0입니다.

재시도 간격

두 연결 만들기 시도 간의 간격(초)입니다. 기본값은 10입니다. 이 속성은 만들기 재시도 횟수 값이 0보다 클 경우에만 사용됩니다.

지연 연관

사용하도록 설정된 경우 작업이 해당 연결에서 수행될 때 느리게 연결됩니다. 또한 트랜잭션이 완료되고 구성 요소 메소드가 종료되면 연관 해제되어 물리적 연결의 재사용을 돕습니다. 이 옵션은 기본적으로 비활성화됩니다.

지연 연결 참여

사용하도록 설정된 경우 실제로 자원이 메소드에 사용되는 경우에만 자원을 트랜잭션에 참여시킵니다.

스레드와 연결

사용하도록 설정된 경우 연결을 스레드와 연관시켜 연결에서 같은 스레드가 필요할 때 해당 스레드에 이미 연관된 연결을 재사용함으로써 풀에서 연결을 가져올 때 수반되는 오버헤드를 줄입니다. 이 옵션은 기본적으로 비활성화됩니다.

연결 일치

사용하도록 설정된 경우 풀에 대한 연결 일치를 설정하거나 해제합니다. 풀의 연결이 항상 같은 종류이며 따라서 풀에서 선택된 연결을 자원 어댑터와 일치시킬 필요가 없음을 관리자가 알고 있을 때 사용하지 않도록 설정할 수 있습니다. 이 옵션은 기본적으로 비활성화됩니다.

최대 연결 사용자 수

풀에서 재사용할 연결 횟수입니다. 연결을 지정된 횟수만큼 재사용한 다음에는 해당 연결이 닫힙니다. 예를 들어 명령문 누수를 피하려고 할 때 유용합니다. 0 값은 이 속성이 비활성화되었음을 의미합니다. 기본값은 0입니다.
