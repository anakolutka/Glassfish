JVM 常规设置

JavaTM 虚拟机 (Java Virtual Machine, JVMTM) 使 Java 字节码可以在物理计算机上运行。调整 JVM 设置可以提高性能和避免出现内存分配错误。

Java 虚拟机包含在 Java Platform, Standard Edition（Java SE 平台）中，该平台是 Enterprise Server 所需要的。不正确的 JVM 设置将导致服务器无法运行，因此在更改这些设置时应当谨慎。

注 - 术语“Java 虚拟机”和 "JVM" 表示用于 Java 平台的虚拟机。

“JVM 常规设置”页面包含以下选项：

Java 主目录

Java SE 软件（该软件是运行 Enterprise Server 所需的软件）的安装目录的名称。要验证此发行版是否支持您指定的 Java SE 软件版本，请参见《Sun GlassFish Enterprise Server v3 Release Notes》。如果输入的目录名称不存在或输入不受支持的 Java SE 软件版本的目录，将无法启动 Enterprise Server。

Javac 选项

Java 编程语言编译器的命令行选项。部署 Enterprise JavaBeans TM (EJBTM) 组件后，Enterprise Server 将运行编译器。

调试

使用 Java 平台调试器体系结构 (Java Platform Debugger Architecture, JPDA) 启用调试。

JPDA 供应用程序开发者使用。有关更多信息，请参见《Sun GlassFish Enterprise Server v3 Preview Application Development Guide》。

调试选项

指定启用调试时传递到 JVM 的 JPDA 选项。

RMI 编译选项

rmic 编译器的命令行选项。部署 EJB 组件后，Enterprise Server 将运行 rmic 编译器。

字节码预处理程序

用于字节码预处理的类的名称。使用逗号分隔类名。每个类都必须实现 com.sun.appserv.BytecodePreprocessor 接口。将按指定顺序调用这些类。

其他属性

JVM 的其他属性。Enterprise Server 不定义 JVM 的任何其他属性。
