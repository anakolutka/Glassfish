Configurazione runtime batch

GlassFish Server fornisce un ambiente runtime batch per la pianificazione e l'esecuzione di processi batch. I processi batch sono in genere task con tempi di esecuzione lunghi, orientati a blocchi, che contengono una serie di passi. Le applicazioni batch inviano i processi al runtime batch e forniscono istruzioni sulla modalità e sui tempi di esecuzione dei passi. Il runtime batch elabora i passi e memorizza le informazioni sui processi in un repository di processi. In GlassFish Server, il repository di processi è un database.

Utilizzare la pagina Configurazione runtime batch per visualizzare e modificare la configurazione del runtime batch. Per eseguire i processi batch, il runtime batch utilizza un servizio esecutore gestito e un'origine dati. Il servizio esecutore gestito fornisce i thread ai processi e nell'origine dati vengono memorizzate le informazioni sui processi. I dati di configurazione del runtime batch sono memorizzati nell'elemento config del file domain.xml.

La pagina Configurazione runtime batch contiene le informazioni riportate di seguito.

Nome istanza
Nome cluster

Il nome del cluster o dell'istanza server di GlassFish cui si applica la configurazione. Questo campo è di sola lettura.

Nome ricerca servizio esecutore

Il nome della ricerca JNDI del servizio esecutore gestito che fornisce i thread ai processi batch. Il servizio esecutore gestito predefinito è concurrent/__defaultManagedExecutorService.

Dopo l'invio di un processo batch al runtime batch, è possibile modificare il servizio esecutore gestito senza alcun impatto sull'esecuzione del processo.

Nome ricerca origine dati

Il nome della ricerca JNDI dell'origine dati che memorizza le informazioni sui processi batch correnti e precedenti. L'origine dati predefinita è jdbc/__TimerPool.

Non modificare l'origine dati dopo che il primo processo batch è stato inviato al runtime batch per l'esecuzione. Se è necessario modificare l'origine dati, arrestare e riavviare il dominio, quindi apportare la modifica prima dell'avvio o riavvio di qualsiasi processo. Tuttavia, una volta modificata l'origine dati, le informazioni memorizzate nell'origine dati precedente diventano inaccessibili.
