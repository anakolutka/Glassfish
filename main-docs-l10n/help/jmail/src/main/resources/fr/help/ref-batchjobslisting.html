Exécutions de traitement batch

GlassFish Server fournit une exécution de batch pour la planification et l'exécution de traitements batch. Les traitements batch sont généralement des tâches à longue durée d'exécution en masse contenant une série d'étapes. Les applications de traitement batch soumettent des travaux à l'exécution de batch et fournissent des instructions sur la méthode et le moment d'exécution des étapes. L'exécution de batch traite les étapes et stocke les informations relatives aux travaux dans un référentiel de travaux. Dans GlassFish Server, le référentiel de travaux est une base de données.

Utilisez la page Exécutions de traitement batch pour visualiser les détails concernant les exécutions enregistrées des traitements batch.

Elle contient les informations ci-après.

Nom d'instance
Nom du cluster

Nom de l'instance de serveur et du cluster GlassFish auquel s'applique la configuration.

Traitements batch

Liste des exécutions de traitement batch enregistrées pour le cluster ou l'instance. Pour chaque exécution, les informations suivantes sont affichées :

ID d'exécution

ID affecté à l'exécution du traitement batch. Cliquez sur l'ID d'exécution pour ouvrir la page Détails d'exécution de traitement batch pour l'exécution. Une exécution est créée la première fois qu'un travail est démarré et chaque fois que l'exécution existante est redémarrée.

Nom de travail

Nom du travail.

Statut de batch

Statut de l'exécution, tel que défini par l'exécution de batch.

Statut de sortie

Statut de l'exécution, tel que défini par le fichier XML du travail ou par l'application de traitement batch. Par défaut, le statut de sortie et le statut de batch sont identiques, sauf si le statut de sortie est explicitement remplacé.

ID d'instance

ID affecté à l'instance du traitement batch. L'instance d'un traitement batch peut disposer de plusieurs exécutions.

Heure de début

Heure de début de l'exécution.

Heure de fin

Heure de fin de l'exécution.
