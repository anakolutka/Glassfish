So konfigurieren Sie ein HTTP-Zielgerät für die Verwendung von SSL

Wenn Sie das HTTP-Zielgerät zur Verwendung von SSL konfigurieren, ermöglichen Sie Enterprise Server, einen bestimmten Port mit einem Serverzertifikat zu verbinden.

  1. Erweitern Sie in der Navigationsstruktur den Konfigurationsknoten.
  2. Erweitern Sie unter dem Konfigurationsknoten den Knoten "HTTP-Dienst".
  3. Wählen Sie unter dem Knoten "HTTP-Dienst" den Knoten "HTTP-Zielgeräte" aus.
  4. Klicken Sie auf der Seite "HTTP-Zielgeräte" auf den Namen des zu konfigurierenden HTTP-Zielgeräts.
  5. Klicken Sie auf der Seite "HTTP-Zielgerät bearbeiten" auf die Registerkarte "SSL".
  6. Aktivieren Sie das Kontrollkästchen "Client-Authentifizierung", um Clients dazu zu zwingen, sich bei Verwendung dieses Zielgeräts beim Server zu authentifizieren.
  7. Geben in das Feld "Zertifikatspseudonym" den Namen eines vorhandenen Serverschlüsselpaars und -zertifikats ein.
  8. Aktivieren Sie das Kontrollkästchen "SSL3 aktiviert", um das SSL3-Sicherheitsprotokoll auf dem Zielgerät zu aktivieren.
  9. Aktivieren Sie das Kontrollkästchen "SSL2 aktiviert", um das SSL2-Sicherheitsprotokoll auf dem Zielgerät zu aktivieren.
  10. Aktivieren Sie das Kontrollkästchen "TLS aktiviert", um das TLS-Sicherheitsprotokoll auf dem Zielgerät zu aktivieren.
  11. Wählen Sie im Bereich "Ciphersuites" die von den Protokollen verwendeten verfügbaren Ciphersuites aus, und klicken Sie anschließend auf die Schaltfläche "Hinzufügen".

    Um alle Ciphersuites zu aktivieren, klicken Sie auf "Alle hinzufügen".

  12. Klicken Sie auf "Speichern".
  13. Starten Sie den Server neu.
