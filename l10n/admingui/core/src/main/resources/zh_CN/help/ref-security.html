安全性

安全管理器

如果选中此选项，则将启用域的安全管理器。

如果启用了安全管理器，将会向 Enterprise Server 的 JVM 设置中添加 JVM 选项 -Djava.security.manager。请参见配置 JVM 选项。必须重新启动服务器，才能使此更改生效。

请确保您已针对所有应用程序授予了正确的权限。您可以通过关闭安全管理器来提高性能。

默认情况下禁用此选项。

审计日志记录

如果启用了此设置，服务器将装入并运行在“审计模块”设置中指定的所有审计模块。否则，服务器将不访问审计模块。默认情况下禁用此选项。

默认领域

服务器用来进行验证的活动（默认）领域。如果应用程序的部署描述符没有指定其他的领域，则应用程序使用此领域。列表中将显示所有已配置的领域。默认值为 File。

默认主体

指定默认用户名。如果没有提供其他的主体，则服务器使用此用户名。如果您在此字段中输入了一个值，则请在“默认主体密码”字段中输入一个相应的密码。

正常服务器操作不需要此属性。

默认主体密码

在“默认主体”字段中指定的默认主体的密码。正常服务器操作不需要此属性。

审计模块

以逗号分隔的审计模块提供者类列表。此处列出的模块必须是已配置的模块。如果启用了“审计日志记录”，则此字段必须列出审计模块。默认情况下，服务器使用名为 default 的审计模块。有关创建新审计模块的信息，请参见创建审计模块。

启用主体到角色的默认映射

如果选中了此选项，则将对不具有特定于应用程序的映射的应用程序应用主体到角色的默认映射。

映射主体类

指定主体到角色的默认映射中所使用的 java.security.Principal 类的自定义实现。
