HTTP 수신기를 편집하는 방법

  1. 탐색 트리에서 구성 노드를 선택합니다.
  2. 구성 노드에서 HTTP 서비스 노드를 확장합니다.
  3. HTTP 서비스 노드에서 HTTP 수신기 노드를 선택합니다.
  4. HTTP 수신기 페이지에서 편집할 수신기의 이름을 누릅니다.

    admin-listener라는 특수 HTTP 수신기를 편집하는 경우 변경 사항을 적용하려면 서버를 다시 시작해야 합니다. 관리 콘솔에서는 이 경우에 재시작이 필요하다는 것을 식별하지 못합니다.

  5. 서버가 다시 시작할 때마다 이 수신기를 활성화하려면 수신기 사용 가능 확인란을 선택합니다.
  6. 이 수신기를 안전하게 하려면 보안 활성화 확인란을 선택합니다.
  7. 네트워크 주소 필드에 서버의 IP 주소를 입력합니다.

    수신기가 서버에 대한 서버의 전체 IP 주소를 수신하는 경우 네트워크 주소 필드에 0.0.0.0을 입력합니다.

  8. 수신기 포트 필드에 이 수신기의 포트 값을 입력합니다.
    • 단계 7에서 0.0.0.0을 입력한 경우 수신기 포트 값이 이 서버의 IP 주소에서 사용되고 있지 않은지 확인합니다.
    • 그렇지 않은 경우 수신기 포트 값이 지정한 IP 주소에서 사용되고 있지 않은지 확인합니다.
  9. 기본 가상 서버 드롭다운 목록에서 가상 서버를 선택합니다.
  10. 서버 이름 필드에 서버가 클라이언트에 전송하는 URL에서 사용할 호스트 이름을 입력합니다.

    서버가 별칭을 사용하는 경우 이 이름은 별칭 이름입니다. 서버가 별칭을 사용하지 않는 경우에는 이 필드를 비워 두십시오.

  11. 요청을 다른 포트로 리디렉션하려면 포트 리디렉션 필드에 값을 입력합니다.

    다음 두 조건이 존재하면 Enterprise Server는 자동으로 요청을 리디렉션합니다.

    • 이 수신기가 비-SSL 요청을 지원하는 경우

    • SSL 전송이 필요한 일치하는 보안 제약 조건에 대한 요청이 수신된 경우

      기본적으로 Enterprise Server는 원래 요청에 지정된 포트 번호를 사용합니다.

  12. 억셉터 스레드에 이 수신기의 스레드 수를 입력합니다.
  13. 서블릿 생성 HTTP 응답 헤더에서 X-Powered-By: Servlet/2.4 헤더 포함을 비활성화하려면 제공 사용 가능 확인란의 선택을 취소합니다.

    Java Servlet 2.4 사양은 컨테이너가 서블릿 생성 응답에 추가할 수 있는 이 헤더를 정의합니다. 마찬가지로 JavaServer Pages(JSP) 2.0 사양은 JSP 기술을 사용하는 응답에 X-Powered-By: JSP/2.0 헤더가 추가되도록(선택 사항에 따라) 정의합니다. 웹 응용 프로그램의 경우 X-Powered-By: JSP/2.0 헤더의 포함이 기본적으로 활성화되어 있습니다. 이 헤더의 목적은 서블릿 및 JSP 기술의 사용에 대한 통계 데이터를 수집하는 웹 사이트 관리자에게 도움을 주기 위한 것입니다.

    JSP 페이지에 대한 X-Powered-By 헤더의 활성화 및 비활성화에 대한 정보는 Sun GlassFish Enterprise Server v3 Prelude Application Deployment Guide 의 부록 A, Deployment Descriptor Files를 참조하십시오.

    프로덕션 환경에서는 기본 기술을 숨기기 위해 X-Powered-By 헤더의 생성을 생략할 수 있습니다.

  14. 이 수신기 및 외부 포트에 대해 차단을 활성화하려면 차단 사용 가능 확인란을 선택합니다.
  15. (옵션) 추가 등록 정보 섹션에서 추가 등록 정보를 지정합니다.
  16. 저장을 누릅니다.
