Configurer les paramètres du chemin d'accès à la JVM

Le chemin de classe est la liste des fichiers JAR dans laquelle l'environnement d'exécution Java recherche des classes et d'autres ressources.

  1. Dans l'arborescence de navigation, sélectionnez le noeud Configuration.
  2. Sous le nœud Configuration, sélectionnez le nœud Paramètres JVM.

    La page Paramètres JVM s'ouvre.

  3. Dans la page Paramètres généraux JVM, cliquez sur l'onglet Paramètres du chemin.

    La page Paramètres de classpath JVM s'ouvre.

  4. Cochez la case Ignorer le classpath de l'environnement pour ignorer la variable d'environnement CLASSPATH.

    La variable d'environnement CLASSPATH est utile pour les didacticiels de base relatifs à la programmation, mais déconseillée pour les environnements d'entreprise.

  5. Dans le champ Classpath du système, saisissez le chemin d'accès des classes sur votre système.
  6. Dans le champ Préfixe du chemin de classe, saisissez les chemins complets des fichiers JAR pour lesquels vous voulez ajouter un préfixe au chemin de classe du serveur.

    Pour ajouter un préfixe à plusieurs fichiers, saisissez un chemin par ligne.

  7. Dans le champ Suffixe du chemin de classe, saisissez les chemins complets des fichiers JAR pour lesquels vous voulez ajouter un suffixe au chemin de classe du serveur.

    Par exemple, vous devez spécifier le fichier JAR d'un pilote de base de données. Voir To Integrate a JDBC Driver.

    Pour ajouter un suffixe à plusieurs fichiers, saisissez un chemin par ligne.

  8. Dans le champ Préfixe du chemin de bibliothèque native, saisissez les chemins complets des fichiers JAR pour lesquels vous voulez ajouter un préfixe au chemin de bibliothèque native.

    Pour ajouter un préfixe à plusieurs fichiers, saisissez un chemin par ligne.

  9. Dans le champ Suffixe du chemin de bibliothèque native, saisissez les chemins complets des fichiers JAR pour lesquels vous voulez ajouter un suffixe au chemin de bibliothèque native.

    Pour ajouter un suffixe à plusieurs fichiers, saisissez un chemin par ligne.

    Le chemin de la bibliothèque native est une concaténation des chemins suivants :

    • Chemin relatif du serveur pour ses bibliothèques natives partagées

    • Chemin de la bibliothèque native dans l'environnement d'exécution standard

    • Paramètre de l'environnement shell (LD_LIBRARY_PATH sous UNIX®)

    • Tout chemin indiqué sur la page Paramètres du profileur JVM

  10. Cliquez sur Enregistrer.
  11. Redémarrez le serveur.
