Promoted builds are weekly builds of GlassFish which have passed a minimum level of testing including Quicklook and SQE functional tests.

Carefully review the files listed below to select the one you want, then click the link to download.  Some browsers may require you to right-click/CTRL-click to save these files to your computer.


This build (b38) removes the (already deprecated) support for specifying clear text passwords on asadmin command lines or their environment. In other words, options --adminpassword (domain-admin-password) and --password (admin-password) are now prohibited and their use will generate appropriate errors. It is encouraged to use "asadmin login" command *once* to "login" to a domain to be configured/managed and then you don't have to specify either the admin user name or admin password for domain/server management. The password is stored in an encoded format in a file called .asadminpass in your home directory. The idea is similar to what the "cvs login" command achieves for CVS. If you don't change the password across GlassFish build downloads, you'd need to do this only once (not every time you install!) and it will be picked up automatically. See https://glassfish.dev.java.net/javaee5/admin-infra/subprojects/asadminlogin/index.html for details.

Instructions to unbundle and configure GlassFish

GlasssFish installation requires JDK 5.

  1. Download one of the bundles
  2. Run: 
    java -Xmx256m -jar filename.jar
  3. cd glassfish
  4. Run: 
    ant -f setup.xml

Ant is included in the GlassFish bundle that was downloaded.

To get started with GlassFish, see the quick start guide.
