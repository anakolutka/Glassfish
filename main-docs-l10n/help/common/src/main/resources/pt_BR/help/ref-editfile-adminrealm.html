Propriedades Específicas para a Classe FileRealm

As propriedades a seguir são necessárias a um realm file.

Contexto JAAS

O contexto JAAS (Java Authentication and Authorization Service) (o identificador do módulo de log-in para usar este realm). O único valor válido é fileRealm.

Arquivo de Chave

Caminho completo e nome do arquivo em que o servidor armazenará todas as informações de usuário, grupo e senha para esse realm. O valor default é domain-dir/config/keyfile para o realm file e domain-dir/config/admin-keyfile para o realm admin-realm.

O arquivo de chave para o realm file inicialmente é vazio, dessa forma, os usuários devem ser adicionados na Usuários do Arquivo página antes que o realm file seja usado.

O arquivo de chaves do realm admin-realm inicialmente contém o nome de usuário do administrador, a senha do administrador em um formato criptografado e o grupo ao qual este usuário pertence (asadmin, por default). Para obter mais informações sobre como adicionar usuários ao admin-realm, consultePara Conceder Acesso às Ferramentas de Administração.

Observação:

Os usuários do grupo asadmin no admin-realm são autorizados a usar o Console de Administração e as ferramentas de asadmin. Adicione, a esse grupo, somente usuários com privilégios administrativos de servidor.

A propriedade opcional a seguir está disponível no realm file.

Designar Grupos

Uma lista de nomes de grupos, separados por vírgulas. Todos os clientes que apresentarem certificados válidos serão designados a esses grupos; por exemplo, funcionário, gerente, onde esses forem os nomes de grupos de usuários.
