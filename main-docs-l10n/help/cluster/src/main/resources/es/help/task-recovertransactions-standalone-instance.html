Recuperar transacciones para una instancia de GlassFish Server independiente

  1. En el árbol de navegación, seleccione el nodo Instancias Independientes.

    Se abrirá la página Instancias de Servidor Independientes.

  2. Seleccione la instancia para la que se deben recuperar transacciones.

    Se abrirá la página Información General para la instancia.

  3. En la página Información General, haga clic en Recuperar Transacciones.

    Se abrirá la página Recuperar Transacciones.

    El contenido de la página depende de si la instancia se está ejecutando.

    • Si la instancia se está ejecutando, la página Recuperar transacciones sólo contiene el texto La recuperación la realiza el propio servidor en ejecución y el botón Aceptar.

    • Si la instancia no se está ejecutando, la página Recuperar Transacciones contiene opciones adicionales.

  4. En la lista desplegable Servidor de Destino, seleccione otra instancia que pueda recuperar transacciones en la instancia actual.

    Para una instancia independiente, seleccione la instancia denominada server, que representa el servidor de administración de dominio (DAS).

  5. En el campo Directorio de Log de Transacciones, escriba el directorio donde se guardarán los logs de transacciones.

    El valor por defecto es /logs/tx en el directorio de dominio. Los logs de transacciones se guardan en el subdirectorio tx del directorio especificado por la configuración de Ubicación de log de transacciones del servicio de transacción para la configuración del cluster. El valor por defecto de Ubicación de log de transacciones es el directorio especificado en el campo Raíz de log de la página Atributos de dominio, que está ubicada en el separador Avanzado del nodo GlassFish Server.

  6. Haga clic en Aceptar.
