Création d'un module d'audit

L'audit est la manière dont GlassFish Server consigne les événements importants, comme les erreurs et les brèches de sécurité. GlassFish Server consigne tous les événements d'authentification dans des fichiers journaux. Un journal d'accès complet fournit un suivi chronologique des événements d'accès dans GlassFish Server.

GlassFish Server fournit un simple module d'audit par défaut. Pour plus d'informations, reportez-vous à Utilisation du module d'audit par défaut.

  1. Dans l'arborescence de navigation, développez le noeud Configuration.

  2. Sélectionnez l'instance à configurer :

    • Pour configurer une instance spécifique, développez le noeud de configuration de l'instance. Par exemple, pour l'instance par défaut server, développez le noeud server-config.

    • Pour configurer les paramètres par défaut pour toutes les instances, développez le noeud default-config.

  3. Sous le noeud Configuration, développez le noeud Sécurité.

  4. Sous le noeud Sécurité, sélectionnez le noeud Modules d'audit.

    La page Modules d'audit apparaît.

  5. Sur la page Modules d'audit, cliquez sur Nouveau.

    La page Nouveau module d'audit apparaît.

  6. Sur la page Nouveau module d'audit, dans le champ Nom, saisissez le nom du module d'audit.

  7. Dans le champ Nom de classe, saisissez le nom complet de la classe qui implémente ce module.

    Le nom de classe du module d'audit par défaut est com.sun.enterprise.security.Audit.

  8. Dans la section Autres propriétés, spécifiez les propriétés supplémentaires.

    Pour ajouter une propriété, cliquez sur le bouton Ajouter une propriété. Sur la ligne vide qui apparaît, saisissez le nom de la propriété dans le champ Nom et la valeur de la propriété dans le champ Valeur.

    Pour obtenir une description de la propriété supplémentaire disponible dans les modules d'audit, reportez-vous à Propriétés propres aux modules d'audit.

  9. Cliquez sur OK.
