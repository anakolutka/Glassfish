Availability Service

From this page, you can configure the availability service for a server instance.

The configuration attributes that you must set depend on the type of storage that you are using for session state data. Application Server provides the following types of storage for session state data:

  • In-memory replication on other servers in the cluster

  • High-availability database (HADB)

For both types of storage of for session state data, set the following attributes:

Availability Service

Select this check box to enable availability service at the instance level.

MQ Store Pool Name

Type the JNDI name of the JDBC Resource used by the MQ broker cluster when saving persistent JMS messages and other broker cluster configuration information.

If you are using HADB to store session state data, you must additionally set the following attributes:

Note - If you are using in-memory replication on other servers in the cluster to store session state data, these attributes are ignored.

HA Store Name

Type the name of the HA session store.

HA Agent Hosts

Provide a comma-separated list of host names or IP addresses of the servers hosting the HA agents.

HA Agent Port

Type the port number used by the HA agents.

HA Agent Password

Type the password for the HA agent.

Auto Manage Store

Check this box to match the lifecycle of the highly available store with the lifecycle of the highly available cluster.

Store Health Check

Check this box to enable periodic detection of store service availability.

HA Store Health Check

Specify, in seconds, how frequently the health of the HA store is to be checked. The default value is 5.

If necessary, add or delete any additional properties. You can add or delete properties for either type of storage for session state data.
