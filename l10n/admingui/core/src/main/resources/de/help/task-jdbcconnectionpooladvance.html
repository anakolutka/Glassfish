So bearbeiten Sie die erweiterten Attribute eines JDBC-Verbindungspools

  1. Erweitern Sie in der Navigationsstruktur den Ressourcenknoten.
  2. Wählen Sie unter dem Ressourcenknoten den Knoten für Verbindungspools aus.

    Die Seite "Verbindungspool" wird geöffnet.

  3. Klicken Sie auf der Seite "Verbindungspools" auf den Namen des zu bearbeitenden Pools.

    Die Seite "Verbindungspool bearbeiten" wird geöffnet.

  4. Klicken Sie auf der Seite "Verbindungspool bearbeiten" auf die Registerkarte "Erweitert".

    Die Seite "Erweiterte Attribute des Verbindungspools bearbeiten" wird geöffnet.

    Hinweis – Das Feld "Name" ist ein schreibgeschütztes Feld. Sie können den Verbindungspoolnamen nur angeben, wenn Sie einen neuen JDBC-Verbindungspool erstellen.

  5. Geben Sie im Feld "Zeitüberschreitung bei Anweisung" die Dauer (in Sekunden) an, nach der abnormal lang dauernde Anfragen beendet werden.

    Enterprise Server legt für die erstellten Anweisungen QueryTimeout fest. Legen Sie den Wert auf -1 fest, um diese Option zu deaktivieren. Der Standardwert ist -1.

  6. Aktivieren Sie die Option "Umbrechen von JDBC-Objekten aktiviert", sodass die Anwendungen umgebrochene JDBC-Objekte für Statement, PreparedStatement, CallableStatement, ResultSet, DatabaseMetaData erhalten.

    Der Standardwert lautet "False".

  7. Geben Sie in das Feld "Höchstens einmal überprüfen" die Dauer (in Sekunden) ein, nach der eine Verbindung höchstens einmal überprüft wird.

    Durch diese Einstellung wird die Anzahl der Überprüfungsanforderungen einer Verbindung reduziert. Setzen Sie den Wert auf 0, um die Verbindungsüberprüfung zu deaktivieren. Der Standardwert ist 0.

  8. Geben Sie in das Feld "Zeitüberschreitung bei Leck" die Dauer (in Sekunden) ein, die zum Ermitteln von Verbindungslecks in einem Verbindungspool erforderlich ist.

    Setzen Sie den Wert auf 0, um die Ermittlung von Verbindungslecks zu deaktivieren. Der Standardwert ist 0.

  9. Aktivieren Sie die Option "Verbindungsleck-Wiederherstellung aktiviert", um beschädigte Verbindungen im Pool wiederherzustellen, wenn die Ermittlung von Verbindungslecks abgeschlossen ist.
  10. Geben Sie in das Feld "Herstellungs-Neuversuche" die Anzahl der Versuche ein, die bei Fehlern zum Herstellen einer Verbindung unternommen werden.

    Setzen Sie den Wert auf 0, wenn keine Versuche zur erneuten Herstellung der Verbindung unternommen werden sollen. Der Standardwert ist 0.

  11. Geben Sie in das Feld "Intervall für Neuversuche" das Intervall in Sekunden zwischen zwei Verbindungsversuchen ein.

    Der Standardwert ist 10. Dieses Attribut wird nur verwendet, wenn der Wert für Herstellungs-Neuversuche größer als 0 ist.

  12. Aktivieren Sie die Option "Lazy-Verknüpfung aktiviert", um Verbindungen verzögert zu verknüpfen, wenn ein Vorgang in Bezug auf die Verbindungen ausgeführt wird.

    Die Verknüpfung wird wieder aufgehoben, wenn die Transaktion abgeschlossen ist und eine Komponentenmethode beendet wird. Auf diese Weise wird die Wiederverwendung der physischen Verbindungen gefördert. Diese Option ist standardmäßig deaktiviert.

  13. Wählen Sie die Option "Eintragen von Lazy-Verbindung aktiviert", um eine Ressource für die Transaktion nur dann einzutragen, wenn sie tatsächlich in einer Methode verwendet wird.
  14. Wählen Sie die Option "Verknüpfen mit Thread aktiviert", um Verknüpfungen zwischen Verbindungen und Threads zu erstellen.

    Wenn derselbe Thread eine weitere Verbindung benötigt, kann er die bereits mit diesem Thread verknüpfte Verbindung wiederverwenden, wodurch die zusätzliche Belastung durch das Abrufen einer weiteren Verbindung aus dem Pool vermieden wird.

    Diese Option ist standardmäßig deaktiviert.

  15. Wählen Sie die Option "Abgleichen von Verbindungen aktiviert", um den Verbindungsabgleich für den Pool zu aktivieren.

    Diese Option kann deaktiviert werden, wenn der Administrator weiß, dass die Verbindungen im Pool immer homogen sind und eine aus dem Pool abgerufene Verbindung nicht mit dem Ressourcenadapter abgeglichen werden muss. Diese Option ist standardmäßig deaktiviert.

  16. Geben Sie in das Feld "Maximale Verbindungsnutzung" einen Wert dafür ein, wie oft eine Verbindung durch den Pool wiederverwendet werden soll.

    Nachdem eine Verbindung so häufig wie angegeben wiederverwendet wurde, wird sie geschlossen. Diese Einstellung eignet sich beispielsweise zur Vermeidung von Anweisungslecks. Setzen Sie den Wert auf 0, um dieses Attribut zu deaktivieren. Der Standardwert ist 0.
