Batchjobausführungen

GlassFish Server bietet eine Batchlaufzeit für die Planung und Ausführung von Batchjobs. Batchjobs haben in der Regel eine lange Ausführungszeit und sind auf die Massenverarbeitung von Tasks ausgerichtet, die eine Reihe von Schritten enthalten. Batchanwendungen leiten Jobs an die Batchlaufzeit weiter und liefern Anweisungen, wie und wann die Schritte ausgeführt werden sollen. Die Batchlaufzeit verarbeitet die Schritte und speichert Informationen zu den Jobs in einem Job-Repository. In GlassFish Server ist das Job-Repository eine Datenbank.

Auf der Seite "Batchjobausführungen" können Sie Details zu aufgezeichneten Ausführungen von Batchjobs anzeigen.

Die Seite "Batchjobausführungen" enthält die folgenden Informationen.

Instanzname
Clustername

Der Name des GlassFish-Clusters oder der GlassFish-Serverinstanz, auf das bzw. auf die sich die Konfiguration bezieht.

Batchjobs

Eine Liste der aufgezeichneten Batchjobausführungen für die Instanz oder das Cluster. Für jede Ausführung werden folgende Informationen angezeigt:

Ausführungs-ID

Die ID, die der Ausführung des Batchjobs zugewiesen ist. Wenn Sie auf die Ausführungs-ID klicken, wird die Seite "Details zur Batchjobausführung" für die jeweilige Ausführung angezeigt. Beim erstmaligen Starten eines Jobs und bei jedem Neustart der vorhandenen Ausführung wird eine neue Ausführung erstellt.

Jobname

Der Name des Jobs.

Batchstatus

Der von der Batchlaufzeit festgelegte Status der Ausführung.

Beendigungsstatus

Der Status der Ausführung, der von der Job-XML für den Job oder von der Batchanwendung festgelegt wurde. Standardmäßig sind Beendigungsstatus und Batchstatus identisch, es sei denn, der Beendigungsstatus wird explizit überschrieben.

Instanz-ID

Die ID, die der Instanz des Batchjobs zugewiesen ist. Die Instanz eines Batchjobs kann mehrere Ausführungen haben.

Startzeit

Der Zeitpunkt, an dem die Ausführung gestartet wurde.

Endzeit

Der Zeitpunkt, an dem die Ausführung abgeschlossen wurde.
