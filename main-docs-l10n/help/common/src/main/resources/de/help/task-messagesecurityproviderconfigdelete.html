So wird eine Nachrichtensicherheitsproviderkonfiguration gelöscht

Bevor Sie beginnen

Es ist nicht möglich, Nachrichtensicherheitsprovider für die standardmäßige Konfiguration server-config zu erstellen oder zu löschen. Die einzigen Nachrichtensicherheitsprovider, die für die standardmäßige Konfiguration server-config unterstützt werden, sind SOAP und HttpServlet. Da diese beiden Nachrichtenprovider bei der Installation von GlassFish Server standardmäßig erstellt werden, wird die Schaltfläche "Neu" auf der Seite "Nachrichtensicherheitskonfigurationen" für die Standardkonfiguration server-config ausgegraut.

  1. Blenden Sie in der Navigationsbaumstruktur den Knoten "Konfiguration" ein.

  2. Blenden Sie den Knoten "Sicherheit" ein.

  3. Wählen Sie den Knoten "Nachrichtensicherheit".

    Die Seite "Nachrichtensicherheitskonfigurationen" wird geöffnet.

  4. Wählen Sie auf der Seite "Nachrichtensicherheitskonfigurationen" SOAP.

    Die Seite "Neue Nachrichtensicherheitskonfiguration bearbeiten" wird geöffnet.

  5. Klicken Sie auf der Seite "Nachrichtensicherheitskonfiguration bearbeiten" auf die Registerkarte "Provider".

    Die Seite "Providerkonfiguration bearbeiten" wird geöffnet.

  6. Aktivieren Sie auf der Seite "Nachrichtensicherheitskonfigurationen" das Kontrollkästchen links von der Providerkonfiguration, die Sie löschen.

  7. Klicken Sie auf "Löschen".
