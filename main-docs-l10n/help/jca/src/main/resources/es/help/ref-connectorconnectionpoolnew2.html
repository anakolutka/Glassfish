Nuevo pool de conexiones del conector (paso 2 de 2).

Utilice el nuevo pool de conexiones del conector (paso 2 de 2) para crear un nuevo pool de conexiones del conector.

La página Nuevo pool de conexiones del conector (paso 2 de 2) contiene las siguientes opciones.

Ping

Si esta opción está activada, se hace ping en el pool de conexiones durante la creación o la nueva configuración a fin de identificar y advertir si hay valores erróneos para sus atributos. Esta opción se activada por defecto.

Descripción

Descripción del pool de conexiones de conector.

Tamaño de pool inicial y mínimo:

Número mínimo de conexiones del pool de conexiones de conector. El valor por defecto es 8.

Tamaño de Pool Máximo

Número máximo de conexiones del pool de conexiones de conector. El valor por defecto es 32.

Cantidad de Cambio de Tamaño del Pool

Número de conexiones que se pueden eliminar cuando se agota el timeout de inactividad del pool. El valor por defecto es 2.

Timeout de Inactividad

Tiempo máximo, en segundos, que una conexión puede permanecer inactiva en el pool. El valor por defecto es 300.

Tiempo de Espera Máximo

Cantidad de tiempo que la aplicación que solicita una conexión puede esperar para obtener un mensaje de timeout de conexión. El valor por defecto es 60000.

Validación de la conexión

Si la casilla de control Necesaria está activada, las conexiones se validarán antes de proporcionárselas a la aplicación. Si falla la validación de un recurso, éste se destruye, y se crea y se devuelve un recurso nuevo. Esta opción se activada por defecto.

Ante cualquier fallo

Si se activa la casilla de control Cerrar Todas las Conexiones, GlassFish Server cerrará todas las conexiones del pool y las restablecerá aunque tan sólo falle una. Esta opción se activada por defecto.

Soporte de Transacción

El soporte de transacción seleccionado sustituye el atributo de soporte de transacción en el adaptador de recursos asociado al pool de conexiones de una forma descendente. Dicho de otro modo, puede soportar un nivel de transacción inferior o idéntico al que se especifica en el adaptador de recursos, pero no puede indicar un nivel superior.

El valor puede ser cualquiera de los siguientes:

XATransaction

El adaptador de recursos soporta transacciones JTA y transacciones locales del gestor de recursos mediante la implantación de las interfaces LocalTransaction y XAResource. Las transacciones XA se controlan y coordinan a través de un gestor de transacciones externo en un gestor de recursos.

LocalTransaction

El adaptador de recursos soporta transacciones locales mediante la implantación de la interfaz LocalTransaction. Las transacciones locales se gestionan de manera interna en un gestor de recursos y no involucran a gestores de transacciones externos.

NoTransaction

El adaptador de recursos no soporta transacciones JTA o transacciones locales del gestor de recursos y no implanta las interfaces XAResource o LocalTransaction.

Propiedades Adicionales

Propiedades adicionales para el pool de conexiones de conector. Las propiedades disponibles dependen del adaptador de recursos para el pool de conexiones de conector. Los pares nombre-valor especificados en esta tabla se pueden usar para sustituir los valores por defecto de las propiedades definidas por el proveedor del adaptador de recursos.

Si especifica un nombre, pero no un valor para una propiedad, éste se elimina de la tabla al finalizar la creación del pool de conexiones. Si no desea definir un valor para una propiedad ahora, pero desea que la propiedad permanezca en la tabla para que el valor se pueda definir más adelante, introduzca un juego de paréntesis vacío en el campo Valor:

()
