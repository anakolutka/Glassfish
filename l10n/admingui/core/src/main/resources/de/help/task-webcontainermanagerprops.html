So konfigurieren Sie Manager-Eigenschaften

Mit dem Sitzungsmanager können Sie konfigurieren, wie Sitzungen erstellt und zerstört werden, wo der Sitzungsstatus gespeichert wird und wie viele Sitzungen maximal zulässig sind.

  1. Erweitern Sie in der Navigationsstruktur den Konfigurationsknoten.
  2. Wählen Sie unter dem Konfigurationsknoten den Webcontainerknoten aus.

    Die Seite "Webcontainer" wird geöffnet.

  3. Klicken Sie auf der Seite "Webcontainer" auf die Registerkarte "Manager-Eigenschaften".
  4. Im Feld "Reap-Intervall" geben Sie die Dauer (in Sekunden) ein, bevor die Daten der inaktiven Sitzung aus dem Speicher gelöscht werden.
  5. Geben Sie im Feld "Maximale Sitzungen" die maximal zulässige Anzahl an Sitzungen an.
  6. Geben Sie im Feld "Sitzungsdateiname" den Namen für die Datei mit den Sitzungsdaten ein.
  7. Geben Sie im Feld "Klassenname für Sitzungs-ID-Generator" eine benutzerdefinierte Klasse für das Generieren eindeutiger Sitzungs-IDs an.

    Die Klasse muss sich im Enterprise Server-Klassenpfad befinden. Es ist nur jeweils eine einzige Klasse für den Sitzungs-ID-Generator pro Serverinstanz erlaubt. Sämtliche Instanzen in einem Cluster müssen denselben Sitzungs-ID-Generator aufweisen, um einen Sitzungsschlüsselkonflikt zu vermeiden. Benutzerdefinierte Sitzungs-ID-Generatorklassen müssen die Schnittstelle com.sun.enterprise.util.uuid.UuidGenerator implementieren: 

    package com.sun.enterprise.util.uuid;

public interface UuidGenerator {

    public String generateUuid();
    public String generateUuid(Object obj);  //obj ist das Sitzungsobjekt
}
  8. (Optional) Geben Sie im Abschnitt "Weitere Eigenschaften" weitere Eigenschaften an.
  9. Klicken Sie auf "Speichern".
