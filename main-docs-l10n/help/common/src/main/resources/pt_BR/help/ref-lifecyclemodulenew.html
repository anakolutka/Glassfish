Novo Módulo do Ciclo de Vida

Use a página Novo Módulo do Ciclo de Vida para definir um novo módulo do ciclo de vida.

A página Novo Módulo do Ciclo de Vida contém estas opções.

Nome

O nome do módulo do ciclo de vida.

Nome da Classe

O nome totalmente qualificado do arquivo de classes do módulo do ciclo de vida. A classe deve implementar a interface com.sun.appserv.server.LifecycleListener.

Classpath

O classpath do módulo do ciclo de vida. O classpath especifica onde o módulo do ciclo de vida está localizado. O valor default é domain-dir/applications. Se o módulo já estiver neste diretório (isto é, no classpath do servidor), esse campo poderá ser deixado em branco.

Ordem de Carga

A ordem na qual este módulo do ciclo de vida deve ser carregado na inicialização. Os módulos com valores inteiros menores de ordem de carga são carregados antes. O valor pode variar de 101 até o MAXINT do sistema operacional. Valores de 1 a 100 são reservados.

Descrição

Uma descrição do módulo do ciclo de vida.

Status

Se esta opção for selecionada, o módulo do ciclo de vida será ativado. Por default, a opção está ativada.

No Caso de Falha ao Carregar

Se esta opção estiver selecionada, o servidor será submetido a shutdown se o módulo do ciclo de vida não for carregado. Por default, essa opção está desativada.

Alvos

Clusters e instâncias standalone do módulo do ciclo de vida. Mova os alvos desejados para a coluna Alvos Selecionados utilizando os botões Adicionar e Adicionar Tudo. Mova os alvos desnecessários para a coluna Alvos Disponíveis utilizando os botões Remover e Remover Tudo.

Esta opção é exibida somente se os clusters ou instâncias standalone tiverem sido criados no domínio.
