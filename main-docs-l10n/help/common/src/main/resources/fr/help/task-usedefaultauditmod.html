Utilisation du module d'audit par défaut

Le module d'audit default consigne les demandes d'authentification et d'autorisation dans le fichier journal du serveur. Pour plus d'informations sur la modification de l'emplacement du fichier journal, reportez-vous à Configuration de paramètres de journalisation généraux.

Les entrées de journal relatives à l'authentification comprennent les informations suivantes :

  • Noms des utilisateurs qui ont tenté de s'authentifier

  • Domaine ayant traité la demande d'accès

  • Composant EJB ou URI de module Web demandé

  • Succès ou échec de la demande

Que la journalisation d'audit soit activée ou non, GlassFish Server journalise tous les événements d'authentification refusée.

Les entrées de journal relatives aux autorisations comprennent les informations suivantes :

  • Noms des utilisateurs authentifiés, le cas échéant

  • Composant EJB ou URI Web demandé

  • Succès ou échec des demandes

  1. Dans l'arborescence de navigation, développez le noeud Configuration.

  2. Sélectionnez l'instance à configurer :

    • Pour configurer une instance spécifique, développez le noeud de configuration de l'instance. Par exemple, pour l'instance par défaut server, développez le noeud server-config.

    • Pour configurer les paramètres par défaut pour toutes les instances, développez le noeud default-config.

  3. Sous le noeud Configuration, sélectionnez le noeud Sécurité.

    La page Sécurité apparaît.

  4. Dans la liste déroulante Modules d'audit, sélectionnez le module d'audit nommé default.

  5. Cliquez sur Enregistrer.
