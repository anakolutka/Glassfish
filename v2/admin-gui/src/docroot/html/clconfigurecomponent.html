Configuration

A JBI component must be enabled before you can specify configuration properties. The properties available to configure a component vary, depending on the component. Consult the documentation for the JBI component for a description of the configuration properties available.

The Configuration tab presents a View Instances drop down selection of all the started instances where the component is installed. You can select one instance at a time to view the component's configurable properties currently in use for that instance. When you change any property value and click the Save button, the properties are saved to the selected instance or instances in the Update Instance(s) multiple selection list. By default, the viewed instance is selected for update. You can save the properties to a different instance by selecting it instead of the viewed instance. You can save the properties to multiple (or all) instances by using the Shift or Control keys when selecting.

Note - An instance is either a stand-alone server instance, such as server or a cluster instance. A target is either a stand-alone server or a cluster. JBI components can only be installed to zero or more targets, while the configuration of component properties is performed on one or more instances.
