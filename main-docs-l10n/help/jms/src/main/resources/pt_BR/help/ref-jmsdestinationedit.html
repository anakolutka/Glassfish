Editar Recurso de Destino JMS

Utilize a página Editar Recurso de Destino JMS para modificar as definições de um recurso de destino JMS.

A página Editar Recurso de Destino JMS contém as opções a seguir.

Carregar Defaults

Botão para restaurar definições para seus valores default. Definições que não tiveram seus valores default alterados.

Nome JNDI

Um nome exclusivo que identifica o recurso de destino. O nome JNDI é um campo somente para leitura. Você pode especificar o nome JNDI somente quando cria um novo recurso de destino JMS.

Nome de Destino Físico

O nome do destino físico associado a esse recurso.

O GlassFish Server cria o destino físico automaticamente quando necessário e o deleta quando você deleta o recurso de destino.

Tipo de Recurso

O tipo do recurso de destino. As escolhas disponíveis são javax.jms.Topic e javax.jms.Queue.

Ordem de Implantação

A ordem de implantação do recurso de destino.

Descrição

Uma descrição do recurso de destino.

Status

Se o recurso está disponível no runtime.

Se apenas a instância do servidor default, server, existir no domínio, a disponibilidade de runtime do recurso poderá ser ativada ou desativada. A disponibilidade de runtime está ativada por default.

Se clusters ou instâncias standalone tiverem sido criados no domínio, utilize a guia Alvo para controlar a disponibilidade de runtime do recurso em seus alvos.

Propriedades Adicionais

Propriedades adicionais para o recurso de destino. O GlassFish Server não define propriedades adicionais para recursos do destino JMS.
