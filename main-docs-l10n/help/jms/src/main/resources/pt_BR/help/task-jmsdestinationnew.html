Para Criar um Recurso de Destino JMS

Quando você cria um recurso de destino JMS, o GlassFish Server cria um recurso de objeto de admin para o recurso de destino. Você pode exibir esse recurso no nó Conectores.

  1. Na árvore de navegação, expanda o nó Recursos.

  2. No nó Recursos, expanda o nó Recursos JMS.

  3. No nó Recursos JMS, selecione o nó Recursos de Destino JMS.

    A página Recursos de Destino JMS é aberta.

  4. Na página Recursos de Destino JMS, clique em Novo.

    A página Novo Recurso de Destino JMS é aberta.

  5. No campo Nome JNDI, digite o nome JNDI do recurso.

    Recomenda-se usar o prefixo de subcontexto de nomeação jms/ para recursos do JMS. Por exemplo: jms/Queue.

  6. No campo Nome do Destino Físico, digite o nome do destino físico associado a esse recurso.

    O GlassFish Server cria um destino físico automaticamente quando necessário e o deleta quando você deleta o recurso de destino.

  7. Na lista drop-down Tipo de Recurso, selecione javax.jms.Topic ou javax.jms.Queue.

  8. No campo Descrição, digite uma descrição do recurso de destino.

  9. Marque a caixa de seleção Status Ativado para ativar o recurso durante o runtime.

    Por default, a opção está ativada.

  10. Na seção Propriedades Adicionais, especifique propriedades adicionais do servidor de destino.

    O GlassFish Server não define quaisquer propriedades adicionais para recursos de destino JMS.

    Para adicionar uma propriedade, clique no botão Adicionar Propriedade. Na linha em branco que for exibida, digite o nome da propriedade no campo Nome e digite o valor da propriedade no campo Valor.

  11. Na área Alvos, faça o seguinte:

    1. Na coluna Disponível, selecione o alvo ou os alvos aos quais as aplicações que usam o recurso serão implantadas.

      Os alvos disponíveis incluem os clusters disponíveis e as instâncias do servidor, bem como a instância do servidor default, server.

    2. Clique em Adicionar para mover o alvo para a coluna Selecionado.

  12. Clique em OK.
