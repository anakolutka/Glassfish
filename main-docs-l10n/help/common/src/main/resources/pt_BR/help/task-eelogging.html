Para Configurar Definições Gerais de Log

  1. Na árvore de navegação, expanda o nó Configurações.

  2. No nó Configurações, clique na configuração de cluster ou instância de servidor na qual deseja configurar as Definições do Logger.

    A página Configuração será aberta.

  3. Na página Configuração, clique em Definições do Logger.

    A página Definições do Logger da configuração selecionada será aberta.

  4. Marque a caixa de seleção Gravar no Log do Sistema Ativado para enviar a saída de log para o recurso syslog, além do log do servidor.

    Esse recurso está disponível apenas nos sistemas Solaris e Linux. Por default, essa opção está desativada.

  5. Marque a caixa de seleção Registrar na Console para enviar a saída do registro para a console do sistema.

  6. Marque a caixa de seleção Rotação na Alteração da Data para rotacionar os arquivos de log do servidor todo dia à meia-noite.

  7. Marque a caixa de seleção Modo de Várias Linhas Ativado para formatar mensagens de log de forma que elas se tornem mais fáceis de ler. O corpo da mensagem de log é recuado e começa em uma nova linha.

  8. Na lista drop-down Formato de Log da Console, selecione o formato de log para a console. A opção default é ODL (Oracle Diagnostics Logging). A outra opção é ULF (UniformLogFormatter).

  9. Da lista drop-down Formato de Registro do Arquivo de Log, selecione o formato de log para o arquivo de log do servidor. A opção default é ODL.

  10. No campo Limite de Rotação de Arquivo, digite o tamanho em bytes que o arquivo de log do servidor deve alcançar antes de ser rotacionado.

    Quando o log do servidor alcançar o tamanho especificado, o GlassFish Server cria um novo arquivo vazio denominadoserver.log e renomeia o arquivo antigo server.log_data, onde data é a data e a hora em que o arquivo foi rotacionado. O valor default é 2 megabytes. O valor mínimo para o limite é de 500 quilobytes; se você especificar um valor menor, o arquivo será rotacionado quando alcançar 500 KB. Para desativar a rotação de arquivos de log, defina o valor para 0.

  11. No campo Limite de Tempo de Rotação de Arquivo, digite o número de minutos após o qual o log do servidor deve ser rotacionado.

    O valor default é 0, o que significa que o arquivo é rotacionado quando ele alcança o tamanho especificado no campo Limite de Rotação de Arquivo.

    Observação:

    Se você especificar um ou mais minutos, o limite de tempo tem precedência sobre o limite de tamanho especificado pelo campo Limite de Rotação de Arquivo.

  12. No campo Frequência de Esvaziamento, digite o número máximo de mensagens a serem escritas de cada vez a partir da fila para o log do servidor.

    O valor default é 1.

  13. No campo Máximo de Arquivos de Histórico, digite o número máximo de arquivos de log que o GlassFish Server deve manter antes de deletar o arquivo mais antigo. Se você definir este valor como zero (0), o GlassFish Server não deletará os antigos arquivos.

  14. No campo Arquivo de Log, digite um nome alternativo ou local para o arquivo de log do servidor.

    A localização default é ${com.sun.aas.instanceRoot}/logs/server.log.

  15. No campo Handler de Log, digite o nome da classe absoluta de um handler de log personalizado.

    Um handler de log personalizado permite que você envie logs para um destino diferente de server.log ou syslog O handler personalizado deve estender a classe java.util.logging.ConsoleHandler (uma API compatível com JSR 047). Coloque a classe do handler no caminho da classe do GlassFish Server, para que o handler seja instalado durante a inicialização do servidor.

  16. Clique em Salvar.

    Observação:

    Dependendo da configuração para a qual o log foi ativado, pode ser necessário reiniciar o servidor.

    Também pode ser necessário reiniciar o servidor se o nome do arquivo de log estiver alterado na definição default. Será exibida a mensagem Reinicialização necessária no canto superior direito do Console de Administração caso seja necessário reiniciar o servidor.
