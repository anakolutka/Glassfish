編輯 HTTP 偵聽程式標籤

編輯 HTTP 偵聽程式通訊端，使 Enterprise Server 可快速處理 HTTP 請求。

名稱

偵聽程式的唯讀且唯一的識別碼。

偵聽程式

輸入伺服器的有效 IP 位址。

安全性

可用於叢集和企業設定檔。建立安全的偵聽程式。

網路位址

若希望偵聽程式偵聽該伺服器的所有 IP 位址，請在 [網路位址] 輸入 0.0.0.0，並使用唯一的連接埠值。

偵聽程式連接埠

若 [網路位址] 欄位為 0.0.0.0，請輸入唯一的偵聽程式連接埠值。或是，若要使用其他的 IP 位址，請輸入所需的連接埠值。

預設虛擬伺服器

從下拉式清單中選擇虛擬伺服器。

伺服器名稱

要在伺服器傳送給用戶端的 URL 中使用的主機名稱。如果伺服器使用別名，則該名稱應為此別名。如果伺服器未使用別名，請將該欄位保留為空。

重新導向連接埠

輸入連接埠值，將請求重新導向至其他連接埠。如果滿足以下兩個條件，Enterprise Server 將自動重新導向請求：

  • 該偵聽程式支援非 SSL 請求。

  • 接收了符合安全性限制需要 SSL 傳輸的請求。

    依預設，Enterprise Server 使用原始請求中指定的連接埠號。

接收器執行緒

接收器執行緒的數目。預設值為 1。

提供者

取消核取以在 Servlet 產生的 HTTP 回應標頭中停用 X-Powered-By: Servlet/2.4 標頭。

Java Servlet 2.4 規格中定義了該標頭，其容器可以增加到 Servlet 產生的回應。類似地，JavaServer Pages (JSP) 2.0 規格中定義了一個 X-Powered-By: JSP/2.0 標頭，該標頭將增加到使用 JSP 技術的回應 (可選擇)。依預設，將為 Web 應用程式啟用 X-Powered-By: JSP/2.0 標頭。這些標頭的目標是協助網站管理員收集有關使用 Servlet 和 JSP 技術的統計資料。

如需有關啟用與停用 JSP 頁面 X-Powered-By 標頭的資訊，請參閱「GlassFish v3 Application Server Application Deployment Guide」中的附錄 A「Deployment Descriptor Files」。

依據生產環境的不同，可以省略 X-Powered-By 標頭的產生，以隱藏基礎技術。

區段化

啟用偵聽程式和外部連接埠的區段化功能。

其他特性
