Nuevo listener de red

Utilice la página Nuevo Listener de Red para crear un nuevo listener de red.

Cuando se crea un listener de red, es posible crear simultáneamente un protocolo que utilizará el listener de red o utilizar un protocolo que cree previamente. Normalmente, existe una relación de uno a uno entre un listener de red y su protocolo asociado, pero no es necesario que sea así.

La página Nuevo Listener de Red contiene las siguientes opciones.

Nombre de Configuración

Nombre de la configuración a la que se aplican los valores de esta página. Este campo es de sólo lectura.

Nombre

Nombre único de listener. Un nombre de listener de red no puede empezar con un número.

Protocolo

El protocolo asociado al listener de red. Las siguientes opciones especifican si se debe crear un protocolo o utilizar uno existente.

Crear un Protocolo Nuevo

Se crea un nuevo protocolo con valores de configuración por defecto para el listener de red. El nombre por defecto del protocolo es el nombre del listener de red con la terminación -protocol agregada, pero es posible editar este valor.

Servidor Virtual

En el caso de un nuevo protocolo, el servidor virtual que se asociará con el protocolo.

Usar un Protocolo Existente

Se utiliza el protocolo seleccionado. El protocolo determina si se ha activado la seguridad para el listener de red.

Estado

Estado del listener de red. El listener puede estar activado o desactivado. Esta opción se encuentra activada por defecto. Si el listener se encuentra desactivado, cualquier intento de establecer una conexión con él tiene como resultado una excepción de socket (java.net.ConnectException).

En las versiones anteriores a la 9.1 de GlassFish Server, un listener cuyo atributo activado se hubiera definido como false devolvía un código de respuesta 404 para cualquier solicitud que se le enviaba. Para lograr este comportamiento en la versión actual de GlassFish Server, defina el estado del listener en activado y defina cada estado de servidor virtual asociado como desactivado. Un servidor virtual incluye sus listeners asociados en la lista desplegable Listeners de Red.

Seguridad

Cuando esta opción está seleccionada, la seguridad se encuentra activada para el protocolo utilizado por el listener de red. El campo Seguridad es un campo de sólo lectura.

Si está configurando el admin-listener para el servidor de administración de dominio (DAS), que usa la configuración server-config, esta opción será de sólo lectura. Para activar la seguridad de este listener, debe activar primero la administración segura. Consulte la sección Administración Segura.

Listener JK

Si esta opción se encuentra seleccionada, el listener soporta el protocolo mod-jk de Apache. El protocolo mod-jk es un protocolo de red/comunicación de propiedad entre httpd de Apache (servidor web) y una instancia back-end de GlassFish Server. Si el listener soporta este protocolo, se supone que httpd maneja todo el contenido estático, mientras que cualquier solicitud de recursos dinámicos (es decir, servlets y JSP) se direccionan al listener de red con el protocolo mod jk activado de GlassFish Server a través del protocolomod-jk. Esta opción se activada por defecto.

Puerto

Número de puerto en el que recibirá el listener. Los valores válidos se encuentran entre 1 y 65535. En un sistema UNIX, la creación de sockets que reciben en los puertos 1-1024 requiere privilegios de superusuario.

Dirección

Dirección IP en la que actuará el listener de red. La dirección se puede especificar en par punteado o notación IPv6. Puede ser cualquiera (para INADDR_ANY) para recibir en todas las direcciones IP. Puede ser un nombre de host.

Pool de Threads

Pool de threads asociado al listener de red. Normalmente, puede seleccionar uno de los dos pools de threads que están configurados cuando se instala GlassFish Server.

Transporte

Tipo de transporte del listener de red. Por defecto, la única alternativa es tcp.
