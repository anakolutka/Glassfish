Per configurare il servizio timer EJB

Il servizio timer è un servizio di notifica persistente e transazionale che viene fornito dal contenitore di bean enterprise e viene utilizzato per pianificare notifiche o eventi utilizzati dai bean enterprise. Tutti i bean enterprise, a eccezione dei bean di sessione con conservazione dello stato, possono ricevere notifiche dal servizio timer. I timer persistenti impostati dal servizio non vengono eliminati quando il server viene chiuso o riavviato.

  1. Espandere il nodo Configurazione nell'albero di navigazione.

  2. Selezionare il nodo Contenitore EJB.

    Viene visualizzata la pagina Enterprise Java Bean (EJB).

  3. Fare clic sulla scheda Servizio timer EJB.

    Viene visualizzata la pagina Servizio timer EJB.

  4. Nel campo Intervallo minimo di consegna digitare il numero minimo di millisecondi consentito prima della scadenza del timer successivo per un determinato timer.

    L'impostazione di questo intervallo su un valore troppo basso può causare il sovraccarico del server. Il valore predefinito è 1000.

  5. Nel campo N. massimo ripetizioni consegna digitare il numero massimo di tentativi eseguiti dal servizio timer per notificare la scadenza di un timer dovuta a un'eccezione o al rollback.

    Il valore predefinito è 1.

  6. Nel campo Intervallo tra tentativi di consegna digitare l'intervallo, in millisecondi, tra i tentativi di consegna.

    Il valore predefinito è 5000.

  7. Nel campo Origine dati timer digitare il nome JNDI della risorsa JDBC che verrà utilizzata come origine dati del timer.

    Se specificato, questo valore sostituisce il valore predefinito jdbc/__TimerPool per l'applicazione di sistema del servizio timer.

  8. Fare clic su Salva.

    Suggerimento:

    Per ripristinare le impostazioni predefinite, fare clic sul pulsante Carica impostazioni predefinite, quindi su Salva.

  9. Riavviare GlassFish Server.
