批处理作业执行

GlassFish Server 提供了批处理运行时用于批处理作业的调度和执行。批处理作业通常是长时间运行的、批量执行的任务，其中包含一系列步骤。批处理应用程序将作业提交给批处理运行时，并提供有关执行步骤的方法和时间的指示。批处理运行时处理步骤并将有关作业的信息存储在作业资料档案库中。在 GlassFish Server 中，作业资料档案库是数据库。

使用“批处理作业执行”页可以查看有关记录的批处理作业执行的详细信息。

“批处理作业执行”页包含以下信息。

实例名称
集群名称

该配置应用到的 GlassFish 集群或服务器实例的名称。

批处理作业

实例或集群的已记录批处理作业执行的列表。对于每个执行，都将显示以下信息：

执行 ID

分配给批处理作业执行的 ID。单击执行 ID 可打开该执行的“批处理作业执行详细信息”页。新执行在作业首次启动以及现有执行每次重新启动时创建。

作业名称

作业的名称。

批处理状态

批处理运行时所设置的执行的状态。

退出状态

由作业 XML 为作业设置的执行状态或者由批处理应用程序设置的执行状态。默认情况下，退出状态和批处理状态相同，除非显式覆盖了退出状态。

实例 ID

分配给批处理作业实例的 ID。一个批处理作业的实例可以有多个执行。

开始时间

开始执行的时间。

结束时间

结束执行的时间。
