Per modificare una risorsa Connection Factory JMS

  1. Espandere il nodo Risorse nell'albero di navigazione.

  2. Nel nodo Risorse espandere il nodo Risorse JMS.

  3. Nel nodo Risorse JMS selezionare il nodo Connection Factory JMS.

    Viene visualizzata la pagina Connection Factory JMS.

  4. Nella pagina Connection Factory JMS fare clic sul nome del Connection Factory da modificare.

    Viene visualizzata la pagina Modifica Connection Factory JMS.

  5. Nel campo Descrizione digitare una descrizione del Connection Factory.

  6. Selezionare la casella di controllo Abilitato dell'opzione Stato per abilitare la risorsa in fase di esecuzione.

    Questa opzione è abilitata per impostazione predefinita.

  7. Nel campo Dimensione pool iniziale e minima specificare il numero minimo di connessioni per il pool.

    Il valore predefinito è 8.

  8. Nel campo Dimensione massima pool specificare il numero massimo di connessioni per il pool.

    Il valore predefinito è 32.

  9. Nel campo Quantità di ridimensionamento pool specificare il numero di connessioni da rimuovere allo scadere del timeout di inattività del pool.

    Il valore predefinito è 2.

  10. Nel campo Timeout inattività specificare il numero massimo di secondi durante i quali una connessione può rimanere inattiva nel pool.

    Il valore predefinito è 300.

  11. Nel campo Tempo massimo di attesa specificare il tempo di attesa per la richiesta di connessione di un'applicazione prima che si verifichi il timeout della connessione.

    Il valore predefinito è 60000.

  12. Selezionare la casella di controllo In caso di qualsiasi errore: Chiudi tutte le connessioni in modo che GlassFish Server chiuda e ristabilisca tutte le connessioni del pool in caso di errore di una sola connessione.

    Se non si seleziona la casella di controllo, le singole connessioni vengono ristabilite solo quando vengono utilizzate.

  13. Per una risorsa Connection Factory JMS, specificare il valore Supporto transazioni come riportato di seguito.

    • Specificare XATransaction (valore predefinito) per una risorsa utilizzabile per le transazioni che implicano l'uso di più risorse nell'ambito della transazione.

      Ad esempio, questa risorsa in abbinamento a una risorsa JDBC, a una risorsa connettore o a un'altra risorsa Connection Factory JMS. Questo valore offre la massima flessibilità. Una risorsa configurata come XATransaction verrà utilizzata nelle operazioni di commit a due fasi.

    • Specificare LocalTransaction per una risorsa può essere utilizzata per le transazioni che implicano l'uso di una sola risorsa nell'ambito della transazione oppure come ultimo agente in una transazione distribuita che prevede l'uso di più risorse XA.

      Questo valore offre prestazioni decisamente migliori. Una risorsa configurata come LocalTransaction non verrà utilizzata nelle operazioni di commit a due fasi.

    • Specificare NoTransaction per una risorsa che non può mai essere utilizzata nelle transazioni.

      Questa impostazione viene utilizzata raramente nelle applicazioni JMS.

  14. Selezionare la casella di controllo Convalida connessione per abilitare la convalida della connessione.

    Questa opzione è disabilitata per impostazione predefinita.

  15. Nella tabella Proprietà aggiuntive specificare le proprietà aggiuntive per il Connection Factory.

    Per informazioni sulle proprietà disponibili, vedere Proprietà specifiche dei connection factory JMS.

    • Per aggiungere una proprietà, fare clic sul pulsante Aggiungi proprietà. Nella riga vuota visualizzata, digitare il nome della proprietà nel campo Nome e il valore della proprietà nel campo Valore.

    • Per modificare una proprietà, è necessario modificare il valore del campo Valore della proprietà.

    • Per eliminare una proprietà, selezionare la casella di controllo a sinistra del campo Nome corrispondente alla proprietà da eliminare, quindi fare clic sul pulsante Elimina proprietà.

  16. Facoltativamente, fare clic sulla scheda Destinazioni per accedere alla pagina Destinazioni risorse Connection Factory JMS. In questa pagina effettuare le operazioni riportate di seguito.

    1. Fare clic su Gestisci destinazioni per visualizzare la pagina Gestisci destinazioni risorse.

      In questa pagina spostare le destinazioni dalla colonna Disponibili alla colonna Selezionate e viceversa. Assicurarsi di spostare nella colonna Selezionate la destinazione o le destinazioni in cui verranno distribuite le applicazioni che utilizzano la risorsa. Le destinazioni disponibili includono tutti i cluster e le istanze server disponibili, oltre all'istanza server predefinita server. Fare clic su OK per salvare le modifiche.

    2. Selezionare la casella di controllo per una destinazione, quindi fare clic su Abilita o Disabilita per abilitare o disabilitare la risorsa per la destinazione.

  17. Fare clic su Salva.

    Suggerimento:

    Per ripristinare le impostazioni predefinite, fare clic sul pulsante Carica impostazioni predefinite, quindi su Salva.
