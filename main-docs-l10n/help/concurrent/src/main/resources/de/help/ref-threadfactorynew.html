Neue verwaltete Thread-Factory

Auf der Seite "Neue verwaltete Thread-Factory" können Sie eine verwaltete Thread-Factory-Ressource erstellen.

Die Seite "Neue verwaltete Thread-Factory" enthält die folgenden Optionen.

JNDI-Name

Ein eindeutiger JNDI-Name, der die Ressource identifiziert. Konventionsgemäß beginnt der Name mit einem Indikator des Ressourcentyps gefolgt von einem Schrägstrich. Beispiel: concurrent/, jdbc/ oder jms/.

Kontextinformation

Die an Threads zu propagierenden Kontexte: Class Loader, JNDI, Sicherheit oder WorkArea. Die Kontextpropagierung kann aktiviert oder deaktiviert werden. Ist die Option aktiviert, werden die gewählten Kontexte propagiert. Ist die Option deaktiviert, wird keiner der Kontexte propagiert, auch wenn sie ausgewählt sind. Standardmäßig ist die Option aktiviert, und alle vier Kontexte sind gewählt.

Status

Der Status der Ressource. Die Ressource kann aktiviert oder deaktiviert sein. Der Standardwert ist "Aktiviert".

Threadpriorität

Die Priorität, die erstellten Threads zugewiesen werden soll. Der Standardwert ist 5.

Beschreibung

Eine Beschreibung dieser Ressource.

Weitere Eigenschaften

Weitere Eigenschaften für die externe Ressource. In GlassFish Server werden keine weiteren Eigenschaften für diesen Ressourcentyp definiert. Darüber hinaus verwendet dieser Ressourcentyp derzeit keine weiteren Eigenschaften.

Ziele

(Diese Option wird angezeigt, wenn in der Domain Cluster oder Standalone-Instanzen erstellt wurden.)

Cluster und Standalone-Instanzen für die Ressource. Mithilfe der Schaltflächen "Hinzufügen" und "Alle hinzufügen" verschieben Sie die benötigten Ziele in die Spalte "Ausgewählte Ziele". Verschieben Sie die nicht benötigten Ziele in die Spalte "Verfügbare Ziele" mithilfe der Schaltflächen "Entfernen" und "Alle entfernen".

Verwandte Aufgaben

  • So wird eine verwaltete Thread-Factory bearbeitet

  • So wird eine verwaltete Thread-Factory gelöscht
