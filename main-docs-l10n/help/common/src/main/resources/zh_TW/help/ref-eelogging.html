日誌設定

使用「日誌設定」頁面可組態所選取 GlassFish Server 執行處理或叢集的日誌。

「日誌設定」頁面包含下列選項。

組態名稱

套用此頁面上之設定的組態名稱。本欄位是唯讀的。

寫入系統日誌

啟用時，除了伺服器日誌外，還會將記錄日誌輸出傳送到 syslog 功能。此功能僅適用於 Solaris 和 Linux 系統。此選項預設為停用。

記錄至主控台

啟用時，會將記錄日誌輸出傳送至系統主控台。

日期變更時循環

啟用時，GlassFish Server 會於每天午夜循環伺服器日誌檔。

多行模式

啟用時，系統會格式化日誌訊息，使其較易於閱讀。日誌訊息內文會縮排，並從新的一行開始。

主控台記錄日誌格式

記錄至主控台的記錄日誌格式。可用的選項包括 ULF (UniformLogFormatter) 和 ODL (Oracle 診斷記錄日誌)。預設值為 ODL。

日誌檔記錄日誌格式

記錄至伺服器日誌檔的記錄日誌格式。可用的選項為 ULF 和 ODL。預設值為 ODL。

檔案循環限制

伺服器日誌檔必須達到此位元組大小後，才會循環。當伺服器日誌達到指定大小時，GlassFish Server 會新建一個名為 server.log 的空檔案，並將舊檔案重新命名為 server.log_date，其中 date 為循環檔案的日期和時間。預設值為 2 百萬位元組。最小限定值為 500 KB；如果您指定了較低的值，則要在達到 500 KB 時該檔案才循環。若要關閉日誌檔循環，請將該值設定為 0。

檔案循環時間限制

伺服器日誌必須循環之前所等候的分鐘數。預設值為 0，這表示當檔案達到「檔案循環限制」欄位中指定的大小時即循環。如果指定一或多分鐘，此時間限制會優先於「檔案循環限制」欄位中指定的大小限制。

清除頻率

一次可從佇列寫入到伺服器日誌的訊息最大數目。預設值為 1。

歷程記錄檔案上限

GlassFish Server 刪除最舊的檔案之前可保留的日誌檔數目上限。如果您將值設為零 (0)，GlassFish Server 便不會刪除任何舊的日誌檔。

日誌檔

伺服器日誌檔的替代名稱或位置。預設位置為 domain-dir/logs/server.log。

日誌處理程式

自訂日誌處理程式的絕對類別名稱。自訂日誌處理程式可讓您將日誌傳送到 server.log 或 syslog 以外的目的地。自訂記錄程式必須延伸 java.util.logging.Handler 類別 (與 JSR 047 相容的 API)。請將處理程式類別置於 GlassFish Server 類別路徑中，以便在伺服器啟動期間安裝該處理程式。
