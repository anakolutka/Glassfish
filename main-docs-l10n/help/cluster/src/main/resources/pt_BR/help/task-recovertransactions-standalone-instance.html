Para Recuperar Transações de uma Instância de GlassFish Server Standalone

  1. Na árvore de navegação, selecione o nó Instâncias Standalone.

    A página Instância de Servidor Standalone será aberta.

  2. Selecione a instância para a qual é necessária a recuperação de transações.

    A página Informações Gerais da instância se abre.

  3. Na página Informações Gerais, clique em Recuperar Transações.

    A página Recuperar Transações se abre.

    O conteúdo da página depende se a instância está ou não em execução.

    • Se a instância estiver em execução, a página Recuperar Transações contém somente o texto A recuperação é realizada pelo próprio servidor em execução e o botão OK.

    • Se a instância não estiver em execução, a página Recuperar Transações contém opções adicionais.

  4. Na lista drop-down Servidor de Destino, selecione outra instância que pode recuperar as transações na instância atual.

    Selecione, para uma instância standalone, a instância com o nome server, que representa o servidor de administração de domínios (DAS).

  5. No campo Diretório de Log de Transações, digite o diretório em que os logs de transação são armazenados.

    O default é /logs/tx no diretório de domínio. Os logs de transação são mantidos no subdiretório tx do diretório especificado pela definição do Localização do Log de Transações do Serviço de Transação na configuração do cluster. O valor default do Localização do Log de Transações é o diretório especificado pelo campo Raiz de Log da página Atributos do Domínio, que está localizado na guia Avançado do nó do GlassFish Server.

  6. Clique em OK.
