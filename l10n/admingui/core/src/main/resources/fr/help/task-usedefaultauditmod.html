Utiliser le module d'audit par défaut

Les entrées relatives à l'authentification comprennent les informations suivantes :

  • Nom des utilisateurs ayant tenté de s'authentifier.

  • Domaine ayant traité la requête d'accès.

  • Module Web URI ou composant EJB requis.

  • Réussite ou échec de la requête.

Que la fonction de journalisation d'audit soit activée ou non, le Enterprise Server consigne tous les événements d'authentification refusée.

Les entrées relatives aux autorisations comprennent les informations suivantes :

  • Nom des utilisateurs authentifiés, le cas échéant.

  • Module Web URI ou composant EJB requis.

  • Réussite ou échec des requêtes.

  1. Dans l'arborescence de navigation, développez le nœud Configurations.
  2. Sélectionnez l'instance à configurer :
    • Pour configurer une instance particulière, développez le nœud de configuration de l'instance. Par exemple, pour l'instance par défaut, server, développez le nœud server-config.
    • Pour configurer les paramètres par défaut de toutes les instances, développez le noeud default-config.
  3. Sous le nœud Configuration, développez le nœud Sécurité.
  4. Sous le nœud Securité, développez le nœud Modules d'audit.
  5. Sous le nœud Modules d'audit, sélectionnez le nœud default.

    La page Modification du module d'audit s'ouvre.

  6. Définissez la valeur de la propriété auditOn sur true (vrai).
  7. Cliquez sur Appliquer.
  8. Redémarrez le Enterprise Server si « Redémarrage requis » s'affiche dans la console.
