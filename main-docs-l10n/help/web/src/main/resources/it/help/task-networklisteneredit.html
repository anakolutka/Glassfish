Per modificare un listener di rete

  1. Espandere il nodo Configurazione nell'albero di navigazione.

  2. Nel nodo Configurazione espandere il nodo Configurazione di rete.

  3. Nel nodo Configurazione di rete selezionare il nodo Listener di rete.

    Viene visualizzata la pagina Listener di rete.

  4. Nella pagina Listener di rete fare clic sul nome del listener di rete da modificare.

    Viene visualizzata la pagina Modifica listener di rete.

  5. Selezionare il protocollo utilizzato dal listener di rete per modificare le proprie impostazioni. Ad esempio, è possibile abilitare la sicurezza. Non è possibile scegliere un protocollo diverso.

  6. Selezionare o deselezionare la casella di controllo Abilitato dell'opzione Stato per modificare lo stato del listener di rete.

    Il listener può essere abilitato o disabilitato. Questa opzione è abilitata per impostazione predefinita. Se il listener è disabilitato, qualsiasi tentativo di connessione al listener genera un'eccezione socket (java.net.ConnectException).

  7. La casella di controllo di sola lettura Sicurezza abilitata indica se la sicurezza è abilitata per il protocollo utilizzato dal listener di rete.

    Se si sta configurando admin-listener per il server di amministrazione del dominio (DAS), che utilizza la configurazione server-config, questa opzione è di sola lettura. Per abilitare la sicurezza per questo listener, è necessario abilitare l'amministrazione sicura. Vedere Amministrazione sicura.

  8. Selezionare la casella di controllo Abilitato dell'opzione Listener JK per impostare nel listener il supporto del protocollo mod-jk di Apache.

    Il protocollo mod-jk è un protocollo di comunicazione/rete proprietario tra httpd (Web server) di Apache e un'istanza backend di &ProductName. Se il listener supporta il protocollo, tutto il contenuto statico deve essere gestito mediante httpd, mentre le richieste di risorse dinamiche (ovvero servlet e pagine JSP) vengono reindirizzate al listener di rete abilitato mod-jk di GlassFish Server tramite il protocollo mod-jk. Questa opzione è disabilitata per impostazione predefinita.

  9. Nel campo Porta digitare il numero della porta di ascolto del listener di rete.

    I valori validi sono compresi tra 1 e 65535. In un sistema UNIX, per creare socket in ascolto sulle porte comprese tra 1 e 1024 sono necessari i privilegi di utente privilegiato.

  10. Nel campo Indirizzo digitare l'indirizzo IP di ascolto per il listener di rete.

    L'indirizzo può essere specificato utilizzando la notazione per indirizzi IPv6 o con separazione mediante coppie di punti. È possibile specificare any (per INADDR_ANY) per impostare l'ascolto su tutti gli indirizzi IP. Può essere costituito da un nome host.

  11. Nell'elenco a discesa Pool di thread selezionare il pool di thread da associare al listener di rete.

  12. Fare clic su Salva.

    Suggerimento:

    Per ripristinare le impostazioni predefinite, fare clic sul pulsante Carica impostazioni predefinite, quindi su Salva.
Passi successivi

È possibile utilizzare le schede SSL, HTTP e Cache di file per modificare le impostazioni del protocollo associato al listener di rete. La scheda SSL risulta utile solo se è stata abilitata la sicurezza per il protocollo.
