서버 로그를 보는 방법

  1. 탐색 트리에서 Enterprise Server 노드를 선택합니다.

    일반 정보 페이지가 열립니다.

  2. 일반 정보 페이지에서 로그 파일 보기 버튼을 누릅니다.

    로그 뷰어 창이 열립니다.

  3. 검색 기준 영역을 사용하여 로그 뷰어를 사용자 정의하고 필터링합니다.
    • 검색 기준 영역에서 고급 검색을 눌러 로그 뷰어를 구체화합니다.
    • 타임스탬프 필드에서 다음을 수행합니다.
      • 가장 최근 메시지를 보려면 가장 최근(기본값)을 선택합니다.
      • 특정 기간의 메시지만 보려면 특정 범위를 선택하고 표시되는 시작 및 끝 필드에 날짜와 시간 값을 입력합니다.

        시간 값의 경우 구문은 다음 형식을 따라야 합니다. 여기서 SSS는 밀리초를 나타냅니다.

        hh:mm:ss.SSS

        예를 들면 다음과 같습니다.

        17:10:00.000

        시작 값이 끝 값보다 이후일 경우 오류 메시지가 표시됩니다.

    • 로그 수준 드롭다운 목록에서 로그 수준별로 메시지를 필터링할 로그 수준을 선택합니다. 선택한 수준의 메시지만 표시하려면 "더 심각한 메시지는 포함되지 않습니다." 확인란을 선택합니다.

      기본적으로 선택한 로그 수준과 더 심각한 수준에서 서버 로그에 표시되는 모든 메시지가 표시됩니다.

      확인할 메시지를 서버 로그에 표시하려면 먼저 모듈 로그 수준 페이지에서 적절한 로그 수준을 설정합니다. 로그 수준을 구성하는 방법을 참조하십시오.

      로그 수준을 기준으로 로그 메시지를 필터링하도록 선택한 경우 지정한 필터 기준에 맞는 메시지만 표시됩니다. 그러나 이 필터링은 서버 로그에 어떤 메시지를 기록할지에는 영향을 주지 않습니다.

  4. 검색 기준 영역에서 고급 검색을 누른 경우 다음 단계를 수행합니다.
    1. 로거 드롭다운 목록에서 모듈별로 필터링할 하나 이상의 이름 공간을 선택합니다.

      Shift 키나 Ctrl 키를 사용하면 이름 공간을 여러 개 선택할 수 있습니다.

      더 높은 수준의 이름 공간을 선택하면 그 아래에 있는 모든 이름 공간이 선택됩니다. 예를 들어, javax.enterprise.system을 선택하면 해당 이름 공간 아래의 모든 모듈(javax.enterprise.system.core, javax.enterprise.system.tools.admin 등)에 대한 로거도 선택됩니다.

    2. 사용자 정의 로거 필드에서 한 줄에 하나씩 로거 이름을 입력하여 특정 응용 프로그램에 관련된 로거의 메시지를 봅니다.

      응용 프로그램에 여러 모듈이 있을 경우 그 중 일부 또는 모두를 확인할 수 있습니다. 예를 들어 응용 프로그램에 다음과 같은 이름의 로거가 있다고 가정합니다.

      com.mycompany.myapp.module1
com.mycompany.myapp.module2
com.mycompany.myapp.module3

      응용 프로그램에 있는 모든 모듈의 메시지를 확인하려면 com.mycompany.myapp를 입력합니다. module2의 메시지만 확인하려면 com.mycompany.myapp.module2를 입력합니다.

      사용자 정의 로거를 하나 이상 지정한 경우 해당 로거를 로거 영역에 명시적으로 지정해야만 Enterprise Server 모듈의 메시지가 표시됩니다.

    3. 이름-값 쌍 필드에서 특정 스레드에 대한 키 이름과 값을 입력하여 해당 스레드의 출력을 봅니다.

      키 이름은 _ThreadID입니다. 예를 들면 다음과 같습니다.

      _ThreadID=13

      com.mycompany.myapp.module2가 여러 스레드에서 실행되는 것으로 가정합니다. 단일 스레드의 출력만 표시하도록 로그 뷰어를 구체화하려면 사용자 정의 로거 필드에 해당 모듈의 로거를 지정한 다음 이 필드에 스레드 아이디를 지정합니다.

    4. 표시 드롭다운 목록에서 드롭다운 목록의 값을 선택하여 한 번에 해당 개수의 메시지를 봅니다.

      표시되는 기본 메시지 수는 40입니다. 기타 선택 항목은 100, 250 및 1000입니다.

      스택 추적을 보려면 “과도하게 긴 메시지 제한” 확인란을 선택 취소합니다. 기본적으로 스택 추적은 뷰어에 표시되지 않습니다. 스택 추적을 보려면 메시지의 (세부 사항) 링크를 누릅니다.

    5. 고급 옵션 영역을 숨기려면 기본 검색을 누릅니다.
  5. 검색 결과 영역에서 로그 메시지 표시를 사용자 정의합니다.

    검색 결과 영역에는 로깅 설정 및 로그 수준 페이지에서 지정한 설정으로 서버 로그에 있는 40개의 최근 항목이 표시됩니다.

    • 메시지를 다른 방식으로 정렬하려면 열에 있는 화살표를 누릅니다.
    • 레이블이 (details)인 링크를 누르면 서식이 지정된 버전의 메시지를 볼 수 있습니다.

      로그 항목 세부 정보 창이 서식이 지정된 메시지 버전으로 표시됩니다.

    • 로그 파일의 이전 또는 이후 항목을 보려면 테이블의 맨 위와 맨 아래에 있는 버튼을 누릅니다.

  • Enterprise Server에 대한 일반 정보를 보는 방법

  • 고급 응용 프로그램 구성을 설정하는 방법

  • 고급 도메인 속성을 설정하는 방법

  • 관리자 비밀번호를 변경하는 방법

  • 시스템 등록 정보를 관리하는 방법

  • 응용 프로그램 모니터링 데이터를 보는 방법

  • 서버 모니터링 데이터를 보는 방법
