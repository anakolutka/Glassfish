復原獨立 GlassFish Server 執行處理的作業事件

  1. 在瀏覽樹狀結構中，選取「獨立執行處理」節點。

    此時將開啟「獨立伺服器執行處理」頁面。

  2. 選取需要復原其作業事件的執行處理。

    此時將開啟執行處理的「一般資訊」頁面。

  3. 在「一般資訊」頁面上，按一下「復原作業事件」。

    「復原作業事件」頁面會隨即開啟。

    此頁面的內容取決於執行處理是否正在執行中。

    • 如果執行處理在執行中，「復原作業事件」頁面只包含文字「復原是由正在執行的伺服器自行執行」與「確定」按鈕。

    • 如果執行處理不在執行中，「復原作業事件」頁面會包含其他選項。

  4. 從「目的地伺服器」下拉式清單中，選取能夠復原目前執行處理之作業事件的其他執行處理。

    對於獨立執行處理，請選取名為 server 的執行處理，其代表網域管理伺服器 (DAS)。

  5. 在「作業事件日誌目錄」欄位中，鍵入儲存作業事件日誌的目錄。

    預設是位於網域目錄底下的 /logs/tx。作業事件日誌保存在 tx 中，它是叢集組態之作業事件服務「作業事件日誌位置」設定所指定目錄的子目錄。「作業事件日誌位置」預設值為「網域屬性」頁面上的「日誌檔根目錄」欄位所指定的目錄，該頁面位於「GlassFish Server」節點的「進階」標籤之下。

  6. 按一下「確定」。
