Applications

La page Applications présente la liste des applications déployées sur GlassFish Server. Vous pouvez afficher et gérer les applications qui sont déjà déployées et vous pouvez déployer d'autres applications.

Pour chaque application, les informations suivantes sont fournies :

Nom

Nom de l'application.

Le nom peut être suivi d'un identificateur de version facultatif dont il est séparé par un signe deux-points (:). Pour plus d'informations sur les versions des modules et des applications, reportez-vous à Module and Application Versions dans le manuel Oracle GlassFish Server Application Deployment Guide.

Ordre de déploiement

Ordre de déploiement de l'application.

Les applications ayant le numéro le plus faible sont chargées en premier au démarrage du serveur. Une application ayant un ordre de déploiement de 102 est chargée avant une application ayant un ordre de déploiement de 110. Si l'ordre de déploiement n'est pas indiqué au moment où une application est déployée, l'ordre de déploiement par défaut de 100 est affecté. Si deux applications ont le même ordre de déploiement, l'application qui a été déployée en premier est chargée en premier. La spécification d'un ordre de déploiement s'avère utile si l'application a des dépendances et doit être chargée dans un certain ordre.

Activé ou statut

Coche si l'application est activée ou X si l'application est désactivée (si seule l'instance de serveur par défaut server existe).

S'il existe des clusters ou d'autres instances de serveur autonomes, sélectionnez Activé sur les cibles pour afficher les cibles sur lesquelles l'application est déployée.

Moteurs

Types de conteneurs utilisés par l'application. Voici les différents types de conteneur :

  • web

  • webservices

  • ejb

  • connector

  • appclient

  • weld (conteneur Contexts and Dependency Injection des applications Java EE Platform)

Action

Liens vers les actions que vous pouvez effectuer sur le composant après l'avoir déployé : Redéploiement et Rechargement pour tous les composants, Lancement pour les applications Web et les clients d'application et Téléchargement des stubs client pour les clients d'application.

La table Applications déployées contient également les options ci-après.

Déployer

Bouton permettant de déployer une application.

Annuler le déploiement

Bouton permettant d'annuler le déploiement des applications sélectionnées.

Activer

Bouton permettant d'activer les applications sélectionnées, si seule l'instance de serveur par défaut, server, existe.

Désactiver

Bouton permettant de désactiver les applications sélectionnées, si seule l'instance de serveur par défaut, server, existe.

Filtre

Liste déroulante qui filtre les applications par moteur.
