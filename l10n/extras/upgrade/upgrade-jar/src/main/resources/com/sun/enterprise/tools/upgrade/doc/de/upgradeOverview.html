Das Upgrade-Tool: Ein Überblick

Mit dem Upgrade-Tool können Sie Konfigurationsinformationen von einer Application Server 8.x- oder einer Application Server 9-Installation auf eine Application Server 9.1-Installation übertragen. Die übertragenen Informationen beinhalten Daten über eingesetzte Anwendungen, den Dateibereich, Sicherheitszertifikate und andere Ressourcen- und Serverkonfigurationseinstellungen.

Das Tool erkennt automatisch die Version der angegebenen Quellserver-Installation.

Hinweis: Stellen Sie vor Beginn des Upgrade-Vorgangs sicher, dass sowohl der Quellserver (der Server, von dem aktualisiert wird) als auch der Zielserver (der Server, auf den aktualisiert wird) angehalten wurden.

Upgrade-Terminologie

Im Folgenden sind wichtige Begriffe im Zusammenhang mit dem Upgrade-Prozess aufgeführt:

  • Quellserver: die Installation, welche auf die neue Version aktualisiert wird.

  • Zielserver: die Installation, auf die Sie aktualisieren.

  • Domänen-Root: das Verzeichnis, in dem die Domänen erstellt werden. Der Speicherort dieses Verzeichnisses wird standardmäßig angegeben als AS_DEF_DOMAINS_PATH in asenv.conf

  • Domänenverzeichnis oder domain-dir: das Verzeichnis (innerhalb von Domänen-Root) für eine spezifische Domäne. Alle Konfigurations- und anderen Daten für die Domäne werden in diesem Verzeichnis gespeichert.

  • Installations-Root: das Verzeichnis, in dem Application Server installiert ist.

Wenn das Upgrade-Kontrollkästchen während der Application Server-Installation aktiviert wurde, wird der Upgrade-Assistent nach dem Abschluss der Installation automatisch angezeigt. Der Upgrade-Prozess beginnt.
