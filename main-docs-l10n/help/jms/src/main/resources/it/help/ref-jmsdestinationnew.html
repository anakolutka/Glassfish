Nuova risorsa di destinazione JMS

Utilizzare la pagina Nuova risorsa di destinazione JMS per creare una nuova risorsa di destinazione JMS.

La pagina Nuova risorsa di destinazione JMS contiene le opzioni riportate di seguito.

Nome JNDI

Il nome univoco che identifica la risorsa di destinazione.

Si consiglia di utilizzare il prefisso del contesto secondario di denominazione jms/ per le risorse JMS. Ad esempio: jms/Queue.

Nome destinazione fisica

Il nome della destinazione fisica da associare a questa risorsa.

GlassFish Server crea in modo automatico la destinazione fisica quando è necessaria e la elimina quando si elimina la risorsa di destinazione.

Tipo di risorsa

Il tipo di risorsa di destinazione. Le scelte disponibili sono javax.jms.Topic e javax.jms.Queue.

Descrizione

La descrizione della risorsa di destinazione.

Stato

Lo stato della risorsa di destinazione. La risorsa di destinazione può essere abilitata o disabilitata. Questa opzione è abilitata per impostazione predefinita.

Proprietà aggiuntive

Proprietà aggiuntive per la risorsa di destinazione. GlassFish Server non definisce alcuna proprietà aggiuntiva per le risorse di destinazione JMS.

Destinazioni

I cluster e le istanze standalone per la risorsa di destinazione. Per spostare le destinazioni desiderate nella colonna Destinazioni selezionate, utilizzare i pulsanti Aggiungi e Aggiungi tutto. Per spostare le destinazioni non necessarie nella colonna Destinazioni disponibili, utilizzare i pulsanti Rimuovi e Rimuovi tutto.

Questa opzione viene visualizzata solo se nel dominio sono stati creati cluster o istanze standalone.
