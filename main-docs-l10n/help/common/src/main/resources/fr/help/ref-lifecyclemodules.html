Modules de cycle de vie

La page Modules de cycle de vie permet de configurer les modules de cycle de vie.

Un module de cycle de vie accomplit certaines tâches lorsqu'il est déclenché par des événements dans le cycle de vie du serveur. Les événements de serveur agissant comme des déclencheurs potentiels sont les suivants : l'initialisation, le démarrage, le fait d'être prêt pour les demandes de service et l'arrêt. Les modules de cycle de vie ne font pas partie de la spécification Java EE, mais ils optimisent le fonctionnement de GlassFish Server.

Pour chaque module de cycle de vie, les informations ci-après sont fournies.

Nom

Nom du module de cycle de vie.

Activé ou statut

Coche si l'application est activée ou X si l'application est désactivée (si seule l'instance de serveur par défaut server existe).

S'il existe des clusters ou d'autres instances de serveur autonomes, sélectionnez Activé sur toutes les cibles pour afficher les cibles sur lesquelles l'application est déployée.

Ordre de chargement

Valeur indiquant l'ordre relatif dans lequel le module de cycle de vie doit être chargé.

La table Modules de cycle de vie contient également les options ci-après.

Nouveau

Bouton permettant de créer un module de cycle de vie.

Supprimer

Bouton permettant de supprimer les modules de cycle de vie sélectionnés.

Activer

Bouton permettant d'activer les modules de cycle de vie sélectionnés. Présent si seule l'instance de serveur par défaut, server, existe.

Désactiver

Bouton permettant de désactiver les modules de cycle de vie sélectionnés. Présent si seule l'instance de serveur par défaut, server, existe.
