Modification d'une application

Pour modifier une application existante, utilisez la page Modification d'une application.

La page Modification d'une application contient les options suivantes :

Nom

Champ en lecture seule indiquant le nom de l'application que vous modifiez.

Serveurs virtuels

Les serveurs virtuels associés à cette application.

Racine contexte

Pour une application Web, spécifie le chemin de l'application. Dans l'URL de l'application Web, la racine contexte suit immédiatement le numéro de port (http://host: port/context-root/...). La racine contexte doit commencer par une barre oblique, par exemple /hello.

Description

Une courte description de l'application.

Statut

Si la case Activé est cochée, l'application est activée. Cette option est sélectionnée par défaut.

Java Web Start

Si la case Activé est cochée, l'application utilise le logiciel Java Web Start. Java Web Start permet de déployer des applications Java sans navigateur à exécuter sur une machine virtuelle Java.

Directory Deployed (Déployé par répertoire)

Si cette case est cochée, l'application est déployée à partir d'un répertoire.

Lieu

Champ en lecture seule affichant l'emplacement de l'application déployée.

Bibliothèques

Champ en lecture seule affichant les fichiers JAR de la bibliothèque requis par l'application.

Sub Modules (Sous-modules)

Tableau affichant les noms des sous-modules de l'application et leurs moteurs.
