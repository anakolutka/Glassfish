ロガーの設定

「ロガーの設定」ページを使用して、Enterprise Server のログ機能を設定します。

「ロガーの設定」ページには、次のオプションが含まれます。

システムログに書き込み

このオプションを選択すると、ログ出力がサーバーログに加え、syslog 機能にも送信されます。この機能は Solaris および Linux システムのみで使用できます。このオプションは、デフォルトでは無効になっています。

コンソールにログを記録

このオプションを選択した場合は、結果を予想できません。このオプションは、デフォルトでは無効になっています。

ログファイル

サーバーログファイルの別名または場所。デフォルトの場所は glassfish/domains/domain1/logs/server.log です。

「ログハンドラ」

カスタムログハンドラの絶対クラス名。カスタムログハンドラを使用すると、ログを server.log または syslog 以外の宛先に送信できます。カスタムハンドラを使用する場合は、クラス java.util.logging.Handler (JSR 047 準拠の API) を拡張する必要があります。ハンドラクラスを Enterprise Server のクラスパスに置き、サーバーの起動時にハンドラがインストールされるようにします。

「ファイルローテーション制限」

サーバーログファイルがローテーションされるサイズ (バイト単位)。サーバーログが指定したサイズに達すると、Enterprise Server は server.log という名前の新しい空のファイルを作成し、古いファイルの名前を server.log_date に変更します。date は、ファイルがローテーションされた日時を表します。デフォルト値は 2M バイトです。制限の最小値は 500K バイトです。それより小さな値を指定すると、ファイルは 500K バイトに達した時点でローテーションされます。ログファイルのローテーションをオフにするには、この値を 0 に設定します。

「ファイルローテーションの制限時間」

サーバーログがローテーションされるまでの時間 (分単位)。デフォルト値は 0 です。これは、「ファイルローテーション制限」フィールドで指定したサイズに達したときに、ファイルがローテーションされることを意味しています。1 分以上の値を指定すると、「ファイルローテーション制限」フィールドで指定したサイズ制限よりも、時間による制限が優先されます。
