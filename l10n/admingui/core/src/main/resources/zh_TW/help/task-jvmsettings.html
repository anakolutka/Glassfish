配置 JVM 一般設定

Java 虛擬機器 (JVM) 包含在 Java Platform, Standard Edition (Java SE 平台) 中，Enterprise Server 需要此軟體。錯誤的 JVM 設定會導致伺服器無法執行，因此，變更這些設定時應謹慎。

  1. 在瀏覽樹狀結構中，選取 [Application Server] 節點。
  2. 在 [一般資訊] 頁面上，按一下 [JVM 設定] 標籤。

    [JVM 一般設定] 頁面會隨即開啟。

  3. 在 [Java 首頁] 欄位中，鍵入 Java SE 軟體的安裝目錄名稱。

    Enterprise Server 依賴於 Java SE 平台。若要驗證此發行版本是否支援所指定的 Java SE 軟體版本，請參閱「Sun GlassFish Enterprise Server v3 Prelude Release Notes」。若輸入不存在的目錄名稱或不受支援的 Java SE 軟體版本的目錄名稱，則 Enterprise Server 將無法啟動。

  4. 在 [Javac 選項] 欄位中，鍵入 Java 程式語言編譯器的指令行選項。

    部署 Enterprise JavaBeansTM (EJBTM) 元件後，Enterprise Server 會執行編譯器。

  5. 選取 [啟用除錯] 核取方塊，啟用 Java 平台除錯程式架構 (Java Platform Debugger Architecture, JPDA) 的除錯功能。

    JPDA 由應用程式開發者使用。如需更多資訊，請參閱「Sun GlassFish Enterprise Server v3 Prelude Developer’s Guide」。

  6. 在 [除錯選項] 欄位中，鍵入要傳遞至 JVM 的 JPDA 選項。

    只有在上述步驟中啟用除錯功能時，才可使用此欄位。

  7. 在 [RMI 編譯選項] 欄位中，鍵入 rmic 編譯器的指令行選項。

    部署 EJB 元件後，Enterprise Server 將執行 rmic 編譯器。

  8. 在 [位元碼預處理程式] 欄位中，鍵入類別名稱的清單。

    使用逗號分隔類別名稱。每個類別都必須實作 com.sun.appserv.BytecodePreprocessor 介面。將按指定次序呼叫這些類別。

  9. 按一下 [儲存]。
  10. 重新啟動伺服器。
