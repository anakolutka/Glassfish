So wird der Connector-Service konfiguriert

  1. Blenden Sie in der Navigationsbaumstruktur den Knoten "Konfiguration" ein.

  2. Wählen Sie unter dem Konfigurationsknoten den Knoten "Connector-Service".

    Die Seite "Connector-Service" wird geöffnet.

  3. Geben Sie im Feld "Timeout beim Herunterfahren" die maximale Anzahl Sekunden ein, die während des Herunterfahrens von GlassFish Server, auf die Ausführung der ResourceAdapter.stop-Methode einer Connector-Modulinstanz gewartet wird.

    Ressourcenadapter, deren Herunterfahren länger dauert, werden ignoriert, und das Herunterfahren von GlassFish Server wird fortgesetzt. Der Standardwert ist 30 Sekunden.

  4. Wählen Sie in der Dropdown-Liste "Policy zum Laden von Connector-Klassen" die zu verwendende Policy.

    • Wählen Sie derived, um anzugeben, dass die Ressourcenadapter entsprechend den Verweisen einer Anwendung auf beliebige Ressourcen des Ressourcenadapters bereitgestellt werden.

      Diese Policy ist die Standardeinstellung.

    • Wählen Sie global, um anzugeben, dass alle Ressourcenadapter für alle Anwendungen sichtbar sind.

    Diese Policy gilt nicht für den vorinstallierten JMS-Systemressourcenadapter, jmsra, der immer für alle Anwendungen sichtbar ist.

  5. Klicken Sie auf "Speichern".

    Tipp:

    Um zu den Standardeinstellungen zurückzukehren, klicken Sie auf die Schaltfläche "Standardwerte laden" und anschließend auf "Speichern".
