批次工作執行

GlassFish Server 提供批次工作排程與執行的批次執行階段。批次工作通常是長時間執行，且包含一系列步驟的大量執行導向工作。批次應用程式會提交工作給批次執行階段，並提供步驟執行方式與步驟執行時間的指示。批次執行階段會處理步驟，並將與工作有關的資訊儲存在工作儲存區域中。在 GlassFish Server 中，工作儲存區域就是資料庫。

您可以使用「批次工作執行」頁面來檢視與記錄之批次工作執行有關的詳細資訊。

「批次工作執行」頁面包含以下資訊。

執行處理名稱
叢集名稱

要套用組態之 GlassFish 叢集或伺服器執行處理的名稱。

批次工作

執行處理或叢集之記錄的批次工作執行清單。每個執行均會顯示以下資訊：

執行 ID

指定給批次工作之執行的 ID。按一下執行 ID 可開啟執行的「批次工作執行詳細資訊」頁面。在工作第一次啟動時及每次重新啟動現有的執行時，都會建立新的執行。

工作名稱

工作的名稱。

批次狀態

批次執行階段設定的執行狀態。

結束狀態

由工作的工作 XML 或批次應用程式設定的執行狀態。根據預設，除非已明確置換結束狀態，否則結束狀態應與批次狀態相同。

執行處理 ID

指定給批次工作的執行處理 ID。批次工作的執行處理可以擁有多個執行。

起始時間

執行的起始時間。

結束時間

執行的完成時間。
