HTTP 檔案快取

HTTP 檔案快取可儲存靜態內容，讓 Enterprise Server 可快速處理靜態內容的請求。

[HTTP 檔案快取] 頁面包含下列選項：

全域啟用

若選取此選項，則會啟用全域檔案快取。此選項預設為取消選取。

最長存在時間

有效快取項目的最長存在時間 (以秒為單位) 。

該參數可控制在快取檔案之後，快取的資訊的使用時間。對於超過最長存在時間的項目，將由同一檔案的新項目取代。如果內容不經常變更，請增大該值以提昇效能。

理想的最長存在時間取決於是否會定期修改現有檔案。例如，若每天依固定間隔修改檔案四次，請考慮將最長存在時間設定為 21600 秒 (6 小時)。否則，請考慮將最長存在時間設定為檔案修改後，您願意提供舊版內容檔案的最長時間。

最大檔案計數

檔案快取中可儲存的最大檔案數。

如果值過大，則伺服器會快取那些消耗記憶體且不太需要的檔案。如果值過小，則會失去快取的優點。請針對該屬性嘗試使用不同的值，以便找到用於特定應用程式的最佳解決方案。

雜湊初始大小

雜湊儲存區的初始數目。

雜湊儲存區的初始數目會影響記憶體使用量和搜尋時間。

中型檔案大小限制

可儲存為記憶體對映檔案的快取檔案大小 (以位元組為單位) 上限。

將檔案對映到虛擬記憶體，以快取中型檔案的內容。

最佳上限值略高於 VM 頁面大小。

中型檔案大小

僅限 UNIX® 平台和 Linux 平台：快取為記憶體對映檔案的所有檔案總大小 (以位元組為單位)。

小型檔案大小限制

可讀取到記憶體的檔案大小上限。

配置堆疊儲存區空間並將檔案讀取到該空間，以快取小型檔案的內容。

最佳上限值略低於 VM 頁面大小。

小型檔案大小

要讀取到記憶體的檔案總大小 (以位元組為單位)。

啟用檔案快取

決定若檔案大小低於中型檔案大小的限制，則是啟用檔案內容的快取功能還是停用。

開

啟用檔案內容的快取功能。

關

停用檔案內容的快取功能。
