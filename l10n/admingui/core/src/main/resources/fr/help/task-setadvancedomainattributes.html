Définir les attributs avancés du domaine

  1. Dans l'arborescence de navigation, sélectionnez le nœud Application Server.

    La page Informations générales s'affiche.

  2. Dans la page Informations générales, cliquez sur l'onglet Avancé.

    La page Configuration des applications s'ouvre.

  3. Dans la page Configuration des applications, cliquez sur l'onglet Attributs de domaine.

    La page Attributs de domaine s'ouvre.

  4. Dans le champ Racine application, saisissez le chemin complet du répertoire dans lequel les applications déployées résident.

    La valeur par défaut est domain-dir/applications.

  5. Dans le champ Racine journal, saisissez le chemin complet du répertoire dans lequel les fichiers journaux du serveur sont conservés.

    La valeur par défaut est domain-dir/logs.

  6. Pour utiliser les paramètres locaux de l'hôte, ne renseignez pas le champ Paramètres locaux.

    Un environnement linguistique est un identifiant qui spécifie une combinaison particulière de langues et de régions. Par exemple, l'environnement linguistique pour l'anglais des États-Unis est en_US et pour le japonais, ja_JP. Pour utiliser un environnement linguistique non anglophone, Enterprise Server doit être localisé, c'est-à-dire adapté de l'anglais en une autre langue.

  7. Cliquez sur Enregistrer.
  8. Redémarrez le serveur.
