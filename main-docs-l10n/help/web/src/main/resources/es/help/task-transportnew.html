Para crear un transporte

  1. En el árbol de navegación, amplíe el nodo Configuración.

  2. En el nodo Configuración, amplíe el nodo Configuración de Red.

  3. En el nodo Configuración de Red, seleccione el nodo Transportes.

    Se abrirá la página Transportes.

  4. En la página Transportes, haga clic en Nuevo.

    Se abrirá la página Nuevo Transporte.

  5. En el campo Nombre, introduzca un nombre único para el transporte.

  6. En el campo Nombre de Clase, escriba el nombre de clase de la implantación del transporte.

    El valor por defecto es com.sun.grizzly.TCPSelectorHandler.

  7. En el campo Manejador de Clave de Selección, escriba el nombre de clase del manejador de clave de selección.

    Un manejador de clave de selección es una clase abstracta que gestiona el ciclo de vida de las claves de selección. Si no se especifica ningún valor, se utiliza una implantación por defecto.

  8. En la lista desplegable Tipo de Buffer de Bytes, seleccione el tipo de ByteBuffer que se va a utilizar.

    Las opciones disponibles son HEAP y DIRECT. El valor por defecto es HEAP.

  9. En el campo Threads de Responsable de Aceptación, escriba el número de procesadores del equipo.

    Un valor de -1 indica que Grizzly calculará por su cuenta el número de threads del dispositivo de aceptación según el número de procesadores de la máquina. El valor por defecto es 1.

    Para definir el número de threads de procesamiento de solicitudes, defina el valor del Tamaño Máximo del Pool de Threads del pool de threads utilizado por un listener de red que emplea este transporte.

  10. En el campo Número Máximo de Conexiones, escriba el número máximo de conexiones pendientes en un listener de red que utiliza este transporte.

    El valor por defecto es 4096.

  11. En el campo Tamaño de Buffer, escriba el tamaño (en bytes) del buffer que se proporciona para los flujos de entrada creados por un listener de red que utiliza este transporte.

    El valor por defecto es 8192.

  12. En el campo Timeout de Clave Inactiva, escriba el número de segundos después de los cuales se cancelará una clave inactiva y se cerrará el canal.

    El valor por defecto es 30.

  13. En el campo Timeout de Lectura, introduzca el número de milisegundos que GlassFish Server esperará durante la fase de análisis de cabecera y cuerpo de una operación de lectura.

    El valor por defecto es 30000.

  14. En el campo Timeout de Sondeo de Selector, escriba el número de milisegundos tras los cuales un selector NIO se bloqueará a la espera de eventos (solicitudes del usuario).

    El valor por defecto es 1.000.

  15. En el campo Timeout de Escritura, introduzca el número de milisegundos que GlassFish Server espera antes de considerar que el cliente remoto está desconectado al escribir la respuesta.

    El valor por defecto es 30000.

  16. Active la casilla de control de activación de configuración de pantalla para que la configuración interna de Grizzly se descargue en los logs del servidor.

    Esta opción puede proporcionar información útil para la depuración. Esta opción se activada por defecto.

  17. Active la casilla de control de activación de investigación para que la información de la respuesta/solicitud se vuelque en el log del servidor.

    Puede que esta opción proporcione información útil para realizar la depuración, pero reduce significativamente el rendimiento. Esta opción se activada por defecto.

  18. Active la casilla de control TCP sin Retraso para activar la opción TCP_NODELAY (también llamada algoritmo de Nagle).

    Esta opción se encuentra activada por defecto.

  19. Haga clic en Aceptar.
