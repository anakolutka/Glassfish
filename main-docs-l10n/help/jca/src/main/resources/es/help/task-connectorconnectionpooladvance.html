Para editar atributos avanzados del pool de conexiones de conector

  1. En el árbol de navegación, amplíe el nodo Recursos.

  2. Amplíe el nodo Conectores.

  3. En el nodo Configuración, seleccione el nodo Pools de conexiones de conector.

    Se abrirá la página Pools de conexiones de conectores.

  4. En la página Pool de conexiones de conectores, haga clic en el nombre del pool de conexiones de conector que desee editar.

    Se abrirá la página Editar pool de conexiones del conector.

  5. En la página Editar Pool de Conexiones de Conector, haga clic en el separador Avanzado.

    Se abrirá la página Editar atributos avanzados del pool de conexiones de conector

  6. En el campo Validar al menos una vez, introduzca el tiempo, en segundos, tras el cual se valida una conexión al menos una vez.

    Si se define este atributo con un valor apropiado se minimiza el número de solicitudes de validación por conexión. El valor por defecto es 0, lo que significa que el atributo no está activado.

  7. En el campo Timeout de pérdida, introduzca el tiempo, en segundos, para realizar un seguimiento de las pérdidas de conexión de un pool de conexiones.

    El valor 0 indica que el seguimiento de pérdidas de conexión está desactivado. El valor por defecto es 0.

  8. Active la casilla de control Recuperación de pérdida para restaurar las conexiones perdidas en el pool una vez haya finalizado el seguimiento de pérdidas de conexión.

    Esta opción se activada por defecto.

  9. En el campo Reintentos de creación, introduzca el número de intentos que se realizarán si se produjera un fallo al crear una conexión nueva.

    El valor de 0 significa que no se realizará ningún intento para crear la conexión de nuevo. El valor por defecto es 0.

  10. En el campo Intervalo de reintentos, introduzca el intervalo, en segundos, que hay entre dos intentos de creación de una conexión.

    Este atributo se utiliza sólo si el valor de Intentos de creación es mayor que 0. El valor por defecto es 10 segundos.

  11. Active la casilla de control Pool para activar pool de conexiones para el pool.

    Esta opción se encuentra activada por defecto.

  12. Active la casilla de control Asociación lenta para permitir que las conexiones se asocien de forma lenta cuando se realice una operación con ellas.

    Las conexiones se disocian cuando la transacción se completa y un método de componentes finaliza, lo que ayuda a reutilizar las conexiones físicas. Esta opción se activada por defecto.

  13. Active la casilla de control Inscripción lenta de conexiones para inscribir un recurso en la transacción sólo cuando se esté utilizando en ese instante en un método.

    Esta opción se activada por defecto.

  14. Active la casilla de control Asociar a thread para asociar una conexión a un thread.

    Cuando el mismo thread necesita una conexión, se puede volver a utilizar la conexión ya asociada a ese thread y, por tanto, no se incurra en una sobrecarga a la hora de obtener una conexión del pool. Esta opción se activada por defecto.

  15. Active la casilla de control Conexiones Coincidentes para activar la coincidencia de conexión para el pool.

    Esta opción se puede desactivada si sabe que las conexiones del pool siempre serán homogéneas y, por tanto, el adaptador de recursos no necesitará buscar una coincidencia de una conexión seleccionada del pool. Esta opción se encuentra activada por defecto.

  16. En el campo Uso de la coincidencia de conexiones, introduzca el número de veces que el pool debe usar una conexión.

    La conexión se cerrará una vez reutilizada durante un número específico de veces. El valor 0 indica que el atributo no está activado. El valor por defecto es 0.

  17. Haga clic en Guardar.

    Sugerencia:

    Para volver a la configuración por defecto, haga clic en el botón Cargar Valores por Defecto y, a continuación, en Guardar.
