Informazioni fondamentali su Strumento di aggiornamento

Strumento di aggiornamento consente di gestire le installazioni dei componenti software di GlassFish Server inclusi in una singola directory del file system nel sistema in uso.

La versione Console di amministrazione di Strumento di aggiornamento consente di effettuare le operazioni riportate di seguito.

  • Installare i componenti aggiuntivi

  • Visualizzare gli aggiornamenti per i componenti installati

  • Visualizzare i componenti installati

Per aggiornare o rimuovere i componenti installati, è necessario arrestare il dominio di GlassFish Server, quindi eseguire uno dei comandi riportati di seguito.

  • as-install-parent/bin/updatetool, che consente di avviare la utility grafica Strumento di aggiornamento.

  • as-install-parent/bin/pkg, la versione da riga di comando dello strumento di aggiornamento.

Nelle sezioni seguenti vengono fornite le informazioni fondamentali su Strumento di aggiornamento.

Componente

Per componente si intende l'unità installabile gestita da Strumento di aggiornamento. Un componente è costituito da file installati come unità nel sistema all'interno di un'immagine di installazione.

Immagine di installazione

Per immagine di installazione si intende la directory che contiene i componenti software, incluso il contenuto della directory. L'immagine di installazione viene aggiunta a Strumento di aggiornamento in seguito al download e all'installazione del software di GlassFish Server. Per abilitare l'aggiunta e l'aggiornamento dei componenti software di GlassFish Server, ogni immagine di installazione include informazioni sull'installazione dei componenti.

Repository di rete

Il repository di rete contiene gli elementi seguenti:

  • componenti che possono essere aggiunti a un'immagine di installazione;

  • aggiornamenti dei componenti esistenti di un'immagine di installazione.

L'immagine di installazione è associata a uno o più repository di rete. Ogni repository di rete è rappresentato da un nome e da un URL associato. Il nome repository è univoco per ogni immagine di installazione e rappresenta sia l'organizzazione o il progetto che gestisce il repository sia un ambito di contenuti ben definito. Per ulteriori informazioni sui repository di rete preconfigurati per GlassFish Server, vedere Preconfigured Repositories for GlassFish Server in Oracle GlassFish Server Administration Guide.

Componente aggiuntivo

Per componente aggiuntivo si intende un componente dell'immagine di installazione non ancora installato.
