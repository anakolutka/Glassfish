アップグレードツール: 概要

アップグレードツールを使用すると、Application Server 8.x または Application Server 9 のインストール環境から、Application Server 9.1 インストール環境に設定情報を転送することができます。転送される情報には、配備されたアプリケーションに関するデータ、ファイルレルム、セキュリティー証明書、その他のリソースおよびサーバーの設定が含まれます。

次の表に、サポートされる Application Server のアップグレードを示します。表中の「PE」は Platform Edition を表し、「EE」は Enterprise Edition を表します。

ツールは、指定したソースサーバーインストール環境のバージョンを自動的に検出します。

注意: アップグレード処理を開始する前に、ソースサーバー (アップグレード元のサーバー) とターゲットサーバー (アップグレード先のサーバー) が停止していることを確認してください。

アップグレードに関する用語

次に、アップグレード処理に関連する重要用語について説明します。

  • ソースサーバー: 新しいバージョンにアップグレードする元のインストール環境。

  • ターゲットサーバー: アップグレード後のインストール環境。

  • ドメインルート: ドメインが作成されるディレクトリ。デフォルトでは、このディレクトリは、 asenv.conf の AS_DEF_DOMAINS_PATH で指定された場所です。

  • ドメインディレクトリ、または domain-dir: 特定のドメインに対応した (ドメインルート内の) ディレクトリ。このディレクトリには、ドメインに関連するすべての設定および他のデータが含まれます。

  • インストールルート: Application Server がインストールされるディレクトリ。

Application Server のインストール処理中に「アップグレードする」チェックボックスを選択した場合は、インストールが完了したあとに「Upgrade Wizard」画面が自動的に表示されます。アップグレード処理が開始されます。
