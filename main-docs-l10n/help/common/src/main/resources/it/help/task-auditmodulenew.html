Per creare un modulo di audit

Per audit si intende il modo in cui GlassFish Server registra gli eventi significativi, quali ad esempio gli errori o le violazioni della sicurezza. GlassFish Server registra tutti gli eventi di autenticazione nei file di log. Un log degli accessi completo fornisce una traccia sequenziale degli eventi di accesso di GlassFish Server.

In GlassFish Server è disponibile un modulo di audit predefinito semplice; per ulteriori informazioni, vedere Per usare il modulo di audit predefinito.

  1. Espandere il nodo Configurazione nell'albero di navigazione.

  2. Selezionare l'istanza da configurare:

    • Per configurare un'istanza particolare, espandere il nodo di configurazione specifico dell'istanza. Ad esempio, per l'istanza predefinita server, espandere il nodo server-config.

    • Per configurare le impostazioni predefinite per tutte le istanze, espandere il nodo default-config.

  3. Nel nodo Configurazione espandere il nodo Sicurezza.

  4. Nel nodo Sicurezza selezionare il nodo Moduli di audit.

    Viene visualizzata la pagina Moduli di audit.

  5. Nella pagina Moduli di audit fare clic su Nuovo.

    Viene visualizzata la pagina Nuovo modulo di audit.

  6. Nel campo Nome della pagina Nuovo modulo di audit digitare il nome del modulo di audit.

  7. Nel campo Nome classe digitare il nome completamente qualificato della classe che implementa il modulo.

    Il nome della classe per il modulo di audit predefinito è com.sun.enterprise.security.Audit.

  8. Nella sezione Proprietà aggiuntive, specificare le proprietà aggiuntive.

    Per aggiungere una proprietà, fare clic sul pulsante Aggiungi proprietà. Nella riga vuota visualizzata, digitare il nome della proprietà nel campo Nome e il valore della proprietà nel campo Valore.

    Per la descrizione della proprietà aggiuntiva disponibile per i moduli di audit, vedere Proprietà specifiche dei moduli di audit.

  9. Fare clic su OK.
