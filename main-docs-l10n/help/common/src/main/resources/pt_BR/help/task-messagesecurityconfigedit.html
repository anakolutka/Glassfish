Para Editar uma Configuração de Segurança de Mensagem

Para ativar a segurança de mensagens para pontos finais de Web services implantados no GlassFish Server, você deve especificar um provedor a ser usado por default no servidor. Se você ativar a segurança de mensagens de um provedor default, também será necessário permitir que os provedores sejam usados por clientes dos Web Services implantados no GlassFish Server. Informações sobre como ativar os provedores usados pelos clientes são discutidas em Para Ativar a Segurança de Mensagem para Clientes da Aplicação.

Para ativar a segurança de mensagens de chamadas de Web Services de pontos finais implantados, você deve especificar um provedor de cliente default. Se você tiver ativado um provedor cliente default para GlassFish Server, deverá assegurar que quaisquer serviços chamados de pontos de extremidades implantados no GlassFish Server estejam compativelmente configurados para segurança de camada de mensagens.

  1. Na árvore de navegação, expanda o nó Configuração.

  2. Expanda o nó Segurança.

  3. Selecione o nó Segurança de Mensagem.

    A página Configurações de Segurança de Mensagem será aberta.

  4. Na página Configurações de Segurança de Mensagem, clique no nome da Camada de Autenticação da configuração que você está editando.

    Por default, a única configuração de segurança de mensagem definida é SOAP.

    A página Editar Configuração de Segurança de Mensagem será aberta.

  5. Na página Editar Configuração de Segurança de Mensagem, da lista drop-down Provedor Default, selecione um provedor do servidor.

    O provedor do servidor default, também chamado de provedor default, identifica o provedor do servidor chamado para uma aplicação que não identifica um provedor do servidor específico. As opções disponíveis são ServerProvider, XWS_ServerProvider, ou nenhuma. Por default, nenhum provedor é selecionado, de forma que o GlassFish Server não irá chamar um provedor de segurança de mensagens no servidor. Geralmente, o ServerProvider é selecionado como provedor default.

  6. Na lista drop-down Provedor Cliente Default, selecione um provedor de cliente.

    O provedor do cliente default identifica o provedor de cliente chamado por uma aplicação que não identifica um provedor de cliente específico. As opções disponíveis são ClientProvider, XWS_ClientProvider, ou nenhuma. Por default, nenhum provedor é selecionado, de forma que o GlassFish Server não irá chamar um provedor de segurança de mensagens no cliente. Geralmente, nenhum é selecionado para o provedor de cliente default. Selecione o ClientProvider para ativar um provedor default e a política de proteção de mensagens a ser aplicada às chamadas dos Web Services que se originam de pontos finais de Web Services implantados no GlassFish Server.

  7. Clique em Salvar.

  8. Para obter instruções sobre como modificar políticas de proteção de mensagens de provedores de cliente ou servidor ativados, consulte Para Editar uma Configuração do Provedor de Segurança da Mensagem.
