Per impostare la configurazione delle applicazioni

Utilizzare la pagina Configurazione applicazioni per impostare proprietà che garantiscono l'individuazione delle modifiche apportate alle applicazioni distribuite e il ricaricamento delle classi modificate.

  1. Selezionare il nodo Dominio nell'albero di navigazione.

    Viene visualizzata la pagina Attributi dominio.

  2. Selezionare la scheda Configurazione applicazioni.

    Viene visualizzata la pagina Configurazione applicazioni.

  3. Selezionare la casella di controllo Abilitato dell'opzione Ricaricamento per abilitare il ricaricamento dinamico delle applicazioni.

    Quando è abilitato il ricaricamento dinamico non è necessario ridistribuire un'applicazione o un modulo quando se ne modifica il codice o i descrittori di distribuzione. È sufficiente copiare le pagine JSP (JavaServer Pages) o i file di classe modificati nella directory di distribuzione dell'applicazione o del modulo. Il server controlla periodicamente la presenza di eventuali modifiche nella directory di distribuzione e ridistribuisce l'applicazione, in modo automatico e dinamico, con le modifiche. Ciò risulta utile in un ambiente di sviluppo, in quanto consente di verificare rapidamente le modifiche apportate al codice. In un ambiente di produzione, tuttavia, il ricaricamento dinamico può avere un impatto negativo sulle prestazioni. Inoltre, ogni volta che si esegue un ricaricamento, le sessioni eseguite al momento del transito diventano non valide. Il client deve riavviare la sessione.

    Questa opzione è abilitata per impostazione predefinita.

  4. Nel campo Intervallo di polling ricaricamento digitare la lunghezza in secondi dell'intervallo in base al quale verificare la presenza di eventuali modifiche al codice in applicazioni e moduli e ricaricare dinamicamente tali applicazioni e moduli.

    Il valore predefinito è 2.

  5. Nel campo Timeout sessione di amministrazione digitare il numero di minuti di inattività trascorsi i quali si verifica il timeout di una sessione di Console di amministrazione.

    Il valore predefinito è 60.

  6. Selezionare la casella di controllo Abilitato dell'opzione Distribuzione automatica per abilitare la distribuzione automatica delle applicazioni.

    Se si copia un file di applicazione o modulo (JAR, WAR, RAR o EAR) in una directory specifica, quindi si abilita la distribuzione automatica, l'applicazione o il modulo viene distribuito automaticamente da GlassFish Server. Questa opzione è abilitata per impostazione predefinita.

  7. Nel campo Intervallo di polling distribuzione automatica digitare la lunghezza in secondi dell'intervallo in base al quale verificare la presenza di eventuali modifiche al codice in applicazioni e moduli e ricaricare dinamicamente tali applicazioni e moduli.

    Il valore predefinito è 2. La modifica dell'intervallo di polling non influisce sul tempo necessario per la distribuzione di un'applicazione o di un modulo.

  8. Nel campo Timeout tentativi distribuzione automatica digitare il numero di secondi durante i quali la dimensione di un file copiato parzialmente può rimanere invariata prima che si verifichi un errore.

    Il valore predefinito è 4.

  9. Nel campo Directory di distribuzione automatica digitare il percorso completo della directory da sottoporre a monitoraggio per la distribuzione automatica delle applicazioni.

    Il valore predefinito è domain-dir/autodeploy.

  10. Nell'elenco a discesa Convalida XML selezionare il tipo di convalida XML da eseguire sui descrittori di distribuzione standard e di GlassFish Server.

    Di seguito sono indicati i tipi disponibili.

    Completo

    Se la convalida XML non riesce, non riesce neanche la distribuzione. Questo è il valore predefinito.

    Analisi

    Vengono segnalati errori di convalida XML, ma la distribuzione viene eseguita.

    Nessuno

    La convalida XML non viene eseguita.

  11. Selezionare la casella di controllo Abilitato dell'opzione Verificatore per verificare i file del descrittore di distribuzione.

    Il verificatore esamina la struttura e il contenuto del file prima della distribuzione automatica. Se la verifica non riesce, la distribuzione non viene eseguita. La verifica di applicazioni di grandi dimensioni richiede spesso molto tempo. Questa opzione è disabilitata per impostazione predefinita.

  12. Selezionare la casella di controllo Abilitato dell'opzione Precompilazione per precompilare i file JSP.

    Se questa opzione è disabilitata, i file JSP vengono compilati in fase di esecuzione al primo accesso. Questa opzione è disabilitata per impostazione predefinita.

  13. Nella sezione Proprietà aggiuntive, specificare le proprietà aggiuntive.

    GlassFish Server non definisce alcuna proprietà aggiuntiva per le applicazioni.

    • Per aggiungere una proprietà, fare clic sul pulsante Aggiungi proprietà. Nella riga vuota visualizzata, digitare il nome della proprietà nel campo Nome e il valore della proprietà nel campo Valore.

    • Per modificare una proprietà, è necessario modificare il valore del campo Valore della proprietà.

    • Per eliminare una proprietà, selezionare la casella di controllo a sinistra del campo Nome corrispondente alla proprietà da eliminare, quindi fare clic sul pulsante Elimina proprietà.

  14. Fare clic su Salva.

    Suggerimento:

    Per ripristinare le impostazioni predefinite, fare clic sul pulsante Carica impostazioni predefinite, quindi su Salva.

  15. Riavviare il server.
