Introduction:

The goal of this exercise is to create a JSF page which utilizes the JMaki Charting component and create another integration point to the Administration Console. Through this exercise you will learn how to include a JMaki widget in a JSF page and add another plugin to the Administration Console.

2-C. Modify configuration file and add jsf page for new tree node
You now need to add an integration point for plugging the exercise 2 tree node to the Administration Console. Each plugin module should have 1 console-config.xml, and you can specify as many integration point as needed in this configuration file. Since you have created this file in exercise 1, all we need to do now is to edit it and add the second integration point. You want the exercise 2 node to be under the JavaOne tree node.  So, we specify the parentId to be 'j1root'. You can refer to treenode-ex1.jsf to see this. You also need to create the file treenode-ex2.jsf which will point to the JSF jMaki chart page.
 
  • Steps in NetBeans:
    • 1. Double click on Other Sources ->resources ->META-INF ->admingui -> console-config.xml 
    2. Add the following integration point to the file: 
  <integration-point 
     id="Another" 
     type="tree" 
     priority="230" 
     parentId="j1root" 
     content="treenode-ex2.jsf" 
/>

Figure: Ex2-3   Modify console-config.xml

3. Select Other Sources -> resources
4. Create an empty file named 'treenode-ex2.jsf'   (refer to 1-A on how to create empty file)
5. Edit treenode-ex2.jsf to specify the information regarding this second tree node.
<sun:treeNode
        id="second" 
        text="Exercise #2"
        url="javaone/chart.jsf"
        imageURL="resource/images/instance.gif"
        target="main" 
/>
Figure: Ex2-4  Edit of treenode-ex2.jsf

2-D   Build and Integrate this plugin module to GlassFish V3
       You have implemented all that is needed to add a JSF Page with JMaki Charting to the Administration Console.  You now need to build this module and copy the jar file to GlassFish.  When you restart the server you will be able to see this new tree node and JSF page.

Steps in NetBeans:

  1. Right click on the project "Java One 2008 Admin Console Plugin"
  2. Select Build.

Now, open a Terminal window to enter the follow command.  This is to copy over the module (jar) that was just built to GlassFish.

3. Shut down the server
% glassfish/bin/stop-domain

4. Now, Integrate the jar to GlassFish by copying it to glassfish/modules  directory
% cp  lab/javaone-plugin/target/console-javaone-plugin-1.0-SNAPSHOT.jar  v3/glassfish/modules/console-javaone-plugin-1.0-SNAPSHOT.jar

5.  Start the server again,
% glassfish/bin/asadmin start-domain 

6. Goto http://localhost:8080/admingui
You can now see both Exercise #1 and Exercise #2  nodes under the J1 TreeNode.  Click on Exercise #2 node and you can see the JMaki Chart on the right hand frame.
