Ruby アプリケーションを配備する

  1. ナビゲーションツリーで、「アプリケーション」ノードを選択します。

    「アプリケーション」ページが開きます。

  2. 「配備されているアプリケーション」の表で、「配備する」ボタンをクリックします。

    「アプリケーションまたはモジュールを配備」ページが開きます。

  3. 配備するアプリケーションが含まれるアーカイブの場所を指定します。
    • アーカイブがクライアントマシンにあるファイル、またはクライアントマシンからアクセス可能なファイルの場合は、オプションの「サーバーにアップロードされるパッケージファイル」を選択します。

      クライアントマシンとは、ブラウザを介して 管理コンソール を表示しているホストです。

      「ブラウズ」をクリックしてファイルを検索するか、またはファイルへの完全パスを入力します。

    • アーカイブがサーバーマシンにあるファイルの場合、または分割ディレクトリのパッケージ化されていないアプリケーションである場合は、「Enterprise Server からアクセス可能なローカルのパッケージファイルまたはディレクトリ」オプションをクリックします。

      サーバーマシンは、ドメイン管理サーバー (Domain Administration Server、DAS) を実行しているホストです。

      ファイルを参照するには「ファイルを参照」を、ディレクトリを参照するには「フォルダを参照」をクリックします。あるいは、ファイルまたはディレクトリへの完全パス名を入力します。JRuby アプリケーションをディレクトリから配布するのが一般的です。

  4. 「タイプ」ドロップダウンリストから、「Ruby Application」を選択します。

    「サーバーにアップロードされるパッケージファイル」または「Local Packaged File That Is Accessible From the Enterprise Server」のいずれかを指定した場合、アプリケーションのタイプは自動的に検出されます。「Directory That Is Accessible From the Enterprise Server」を指定した場合は、タイプを手動で指定する必要があります。

  5. 「アプリケーション名」フィールドで、ディレクトリ名またはファイル名のプレフィックスであるデフォルト名を使用するか、別の名前を入力します。

    ファイルのアップロードを選択した場合は、デフォルト名が表示されます。アプリケーション名は一意である必要があります。

  6. 「コンテキストルート」フィールドに、Ruby アプリケーションを識別する文字列を入力します。

    Ruby アプリケーションの URL では、コンテキストルートはポート番号の直後に指定します (http://host: port/context-root/...)。たとえば /hello のように、コンテキストルートがスラッシュで始まるようにしてください。

  7. 「状態を有効」チェックボックスを選択し、アプリケーションを有効にします。

    このオプションはデフォルトで有効になっています。このオプションが無効な場合、アプリケーションは使用できません。

  8. 「JRuby ホーム」フィールドに、JRuby コンテナがインストールされるディレクトリのパスを入力します。

    デフォルトの場所は as-install/jruby です。このフィールドに値を指定すると、Ruby コンテナに設定されている「JRuby ホーム」の値はアプリケーションの値で上書きされます。

  9. 「ランタイム」フィールドに、起動する JRuby ラインタイムの初期数を入力します。

    指定する値は、「Minimum Runtime」フィールドの値以上で、「Maximum Runtime」フィールドの値未満にする必要があります。デフォルト値は 1 です。このフィールドに値を指定すると、Ruby コンテナで設定された「ランタイム」の値はアプリケーションの値で上書きされます。

  10. 「Minimum Runtime」フィールドに、プール内の JRuby ランタイムの最小数を入力します。

    指定する値は、「ランタイム」フィールドの値以下で、「Maximum Runtime」フィールドの値未満にする必要があります。デフォルト値は 1 です。このフィールドに値を指定すると、Ruby コンテナで設定された「Minimum Runtime」の値はアプリケーションの値で上書きされます。

  11. 「Maximum Runtime」フィールドに、プール内の JRuby ランタイムの最大数を入力します。

    指定する値は、「Minimum Runtime」フィールドの値以上で、「ランタイム」フィールドの値以上にする必要があります。デフォルト値は 2 です。このフィールドに値を指定すると、Ruby コンテナで設定された「Maximum Runtime」の値はアプリケーションの値で上書きされます。

  12. 「フレームワーク」ドロップダウンリストから、アプリケーションを配備するフレームワークを選択します。

    選択できる値は、rails、merb、sinatra、および auto-detection です。デフォルトは auto-detection です。このプロパティーに auto-detection 以外の値を設定すると、通常の (時間がかかる) 自動検出プロセスが省略され、強制的に指定したフレームワークで配備を行います。配備したアプリケーションが指定したフレームワーク用に記述されていない場合は、エラーが発生します。

  13. 「Rack 環境」ドロップダウンリストから、Rails や Merb などの JRuby フレームワークを実行する環境を選択します。

    選択できる値は、production、development、および test です。デフォルトは development です。

  14. 「マルチスレッドに対して安全」ドロップダウンリストから、アプリケーションを実行するフレームワークのスレッドセーフ設定を選択します。

    このプロパティーは、自動検出されたユーザー指定の起動スクリプトを使用して起動されたアプリケーションに影響します。次の値を指定できます。

    true

    フレームワークがスレッドセーフで、プールの作成を必要としないことを指定します。

    false

    アプリケーションがアプリケーションインスタンスのプールを使用して起動され、アプリケーションの各インスタンスは一度に 1 つのスレッドからアクセスされることを指定します。

    default

    Enterprise Server は、自動的にスレッドセーフを検出しようとします。この値はデフォルトです。

    このプロパティーは、スレッドセーフを自動的に判定できない、起動中のフレームワークにのみ影響します。この値を true に設定すると、自動検出された Rails 2.1.x アプリケーションはスレッドセーフモードで起動されず、これを使用してスレッドセーフなフレームワークを強制的にプールモードで開始することもできません。

  15. 「仮想サーバー」ドロップダウンリストから、このアプリケーションに関連付ける仮想サーバーを選択します。
  16. 「説明」フィールドに、このアプリケーションの説明を入力します。
  17. 「了解 (OK)」をクリックします。
