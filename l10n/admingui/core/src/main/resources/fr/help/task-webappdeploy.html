Déployer une application Web

  1. Dans l'arborescence de navigation, développez le nœud Applications.
  2. Sous le nœud Applications, sélectionnez le nœud Applications Web.

    La page Applications Web s'ouvre.

  3. Dans le tableau Applications Web déployées, cliquez sur le bouton Déployer.

    La page Déploiement des applications/modules d'entreprise s'affiche.

  4. Dans la liste déroulante Type, sélectionnez Application Web (.war).
  5. Indiquez l'emplacement de l'archive qui contient l'application à déployer.
    • Si l'archive est un fichier stocké sur la machine client ou accessible depuis cette dernière, sélectionnez l'option Fichier empaqueté à télécharger sur le serveur.

      L'ordinateur client est l'hôte sur lequel vous visualisez la Console d’administration via un navigateur.

      Cliquez sur Parcourir pour accéder au fichier ou tapez le chemin d'accès complet au fichier.

    • Si l'archive est un fichier stocké sur la machine serveur ou une application non empaquetée dans un répertoire éclaté, sélectionnez l'option Fichier local empaqueté ou Répertoire accessible depuis le serveur d'application.

      La machine serveur est l'hôte sur lequel fonctionne le serveur d'administration du domaine.

      Saisissez le nom du chemin d'accès complet au fichier ou au répertoire. Le déploiement à partir d'un répertoire éclaté est réservé aux développeurs expérimentés et est déconseillé pour les environnements de production.

  6. Dans le champ Nom de l'application, conservez le nom par défaut, qui correspond au préfixe du nom de fichier, ou bien saisissez un autre nom.

    Le nom par défaut apparaît si vous décidez de télécharger un fichier. Le nom de l'application doit être unique.

  7. Dans le champ Racine contexte, entrez une chaîne qui identifie l'application Web.

    Dans l'URL de l'application Web, la racine contexte suit immédiatement le numéro de port (http://host: port/context-root/...). La racine contexte doit commencer par une barre oblique, par exemple /hello.

  8. Pour désactiver l'application et la rendre indisponible, désélectionnez la case à cocher Activé.

    Par défaut, l'état d'une application est Activé dès son déploiement.

  9. Cochez la case Exécuter le vérificateur activé pour vérifier la structure et le contenu du fichier avant le déploiement.

    La vérification d'applications volumineuses peut prendre un certain temps. Vérifiez le fichier si vous pensez qu'il est endommagé ou non portable.

  10. Cochez la case Précompiler les JSP activés pour compiler au préalable les fichiers JavaServer PagesTM (JSPTM).

    Si vous ne cochez pas cette case, les fichiers JSP sont compilés à l'exécution lors de votre premier accès. La compilation étant souvent une opération fastidieuse dans un environnement de production, cochez cette case.

  11. Dans le champ Bibliothèques, répertoriez les fichiers de bibliothèque (JAR) requis par l'application.
  12. Dans le champ Description, entrez une brève description de cette application Web.
  13. Cliquez sur OK.
