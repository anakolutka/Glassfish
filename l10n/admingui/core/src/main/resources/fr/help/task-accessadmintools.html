Autoriser l'accès aux outils d'administration

Seuls les utilisateurs du groupe asadmin ont accès à la Console d’administration et à l'utilitaire de ligne de commande asadmin. Pour autoriser l'accès à ces outils d'administration, ajoutez l'utilisateur au groupe asadmin dans le domaine administrateur.

  1. Dans l'arborescence de navigation, développez le nœud Configuration.
  2. Si la configuration du domaine prend en charge des clusters, sélectionnez la configuration à modifier.

    Exemples : domaines créés avec le profil Cluster ou entreprise .

  3. Développez le nœud Sécurité.
  4. Développez le nœud Domaines.
  5. Sélectionnez le nœud admin-realm.
  6. Cliquez sur le bouton Gestion utilisateurs sur la page Modification du domaine.

    Après l'installation, le nom et le mot de passe administrateur entrés lors de l'installation figurent dans un fichier nommé admin-keyfile. Par défaut, cet utilisateur appartient au groupe asadmin, qui confère le droit de modifier le Enterprise Server. N'assignez des utilisateurs à ce groupe que si vous voulez leur accorder des privilèges d'administrateur concernant le Enterprise Server.

    Si vous ajoutez des utilisateurs au domaine administrateur mais que vous les assignez à un autre groupe que asadmin, les informations utilisateur seront toujours enregistrées dans le fichier nommé admin-keyfile, mais les utilisateurs n'auront pas accès aux outils d'administration ni aux applications du domaine fichier.

  7. Cliquez sur Nouveau pour ajouter un nouvel utilisateur au domaine administrateur.
  8. Entrez l'ID de l'utilisateur, la liste des groupes et le mot de passe.

    Pour autoriser un utilisateur à modifier le Enterprise Server, ajoutez le groupe asadmin à la Liste des groupes.

  9. Cliquez sur OK pour ajouter cet utilisateur au domaine administrateur ou sur Annuler pour quitter sans sauvegarder.
