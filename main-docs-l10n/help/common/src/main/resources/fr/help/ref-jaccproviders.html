Fournisseurs JACC

La page Fournisseurs JACC permet de configurer les fournisseurs JACC (Java Authorization Contract for Containers) pour le cluster ou l'instance GlassFish Server sélectionné.

Les fournisseurs JACC peuvent être utilisés par des modules d'autorisation tiers pour des applications exécutées sur GlassFish Server. L'installation par défaut de GlassFish Server comprend deux fournisseurs JACC, nommés default et simple. Ces fournisseurs par défaut implémentent un moteur d'autorisation simple, basé sur des fichiers et conforme à la spécification JACC. Vous pouvez également créer des fournisseurs JACC personnalisés. Reportez-vous à Création d'un fournisseur JACC.

Pour plus d'informations sur l'écriture de classes d'implémentation pour un fournisseur JACC, reportez-vous à la spécification JACC sur http://jcp.org/en/jsr/detail?id=115.

Pour chaque fournisseur JACC, les informations ci-après sont fournies.

Nom

Nom du fournisseur JACC. Ce nom est utilisé pour référencer l'élément jacc-provider dans domain.xml.

Configuration de règles

Nom qualifié complet de l'élément javax.security.jacc.PolicyConfigurationFactory.provider qui implémente l'élément javax.security.jacc.PolicyConfigurationFactory propre au fournisseur.

Fournisseur de règles

Nom qualifié complet de l'élément javax.security.jacc.policy.provider qui implémente l'élément java.security.Policy.

Les fournisseurs JACC par défaut prennent également en charge la propriété suivante.

repository

Répertoire contenant le fichier de règles JACC. Pour le fournisseur JACC default de GlassFish Server le répertoire par défaut est ${com.sun.aas.instanceRoot}/generated/policy. Cette propriété n'est pas définie par défaut pour le fournisseur JACC simple de GlassFish Server.

La table Fournisseurs JACC contient également les options ci-après.

Nouveau

Bouton permettant de créer un fournisseur JACC.

Supprimer

Bouton permettant de supprimer les fournisseurs JACC sélectionnés.
