Caché de archivos HTTP

La caché del servicio HTTP almacena contenido estático para permitir que Servidor empresarial procese solicitudes de contenido estático rápidamente.

La página Caché de archivos HTTP contiene las opciones siguientes:

Activado globalmente

Si se selecciona esta opción, se activará el almacenamiento en la caché de archivos de forma global. Esta opción está desactivada de forma predeterminada.

Antigüedad máxima

Duración máxima, en segundos, para una entrada de caché válida.

Este parámetro controla el tiempo que se utiliza la información en caché después de que un archivo se haya almacenado en caché. Una entrada más antigua que la antigüedad máxima se reemplaza por una nueva entrada para el mismo archivo. Si su contenido no cambia con frecuencia, aumente este valor para mejorar el rendimiento.

La antigüedad máxima óptima depende de si los archivos existentes se modifican con regularidad. Por ejemplo, si los archivos se modifican cuatro veces al día en intervalos regulares, plantéese establecer la antigüedad máxima en 21.600 segundos (6 horas). De lo contrario, una vez que el archivo haya sido modificado, considere establecer la antigüedad máxima en el período de tiempo mayor que desee servir a la versión anterior de un archivo.

Total máximo de archivos

Número máximo de archivos que se puede almacenar en la caché de archivos.

Si el valor es demasiado grande, el servidor almacena en caché archivos que no son estrictamente necesarios, lo que consume memoria. Si el valor es demasiado pequeño, no se aprovechan las ventajas del almacenamiento en caché. Pruebe distintos valores de este atributo para determinar la solución óptima para aplicaciones específicas.

Tamaño inicial de hash

Número inicial de depósitos de hash.

El número inicial de depósitos de hash afecta al uso de la memoria y al tiempo de búsqueda.

Límite del tamaño de un archivo medio

Tamaño máximo (en bytes) de un archivo en caché que puede almacenarse como archivo con asignación de memoria

El contenido de un archivo medio se almacena en la caché asignando el archivo a la memoria virtual.

El límite óptimo es un valor ligeramente superior al tamaño de la página VM.

Tamaño de archivo medio

Sólo plataformas UNIX® y Linux: Tamaño total (en bytes) de todos los archivos almacenados en la caché como archivos con asignación de memoria.

Límite del tamaño de un archivo pequeño

Tamaño máximo de un archivo que se puede leer en la memoria.

El contenido de un archivo pequeño se almacena en la caché asignando un espacio máximo y leyendo el archivo en el espacio de pila.

El límite óptimo es un valor ligeramente inferior al tamaño de la página VM.

Tamaño de archivo pequeño

Tamaño total (en bytes) de los archivos que se van a leer en la memoria.

Almacenamiento en caché activado

Determina si el almacenamiento en caché del contenido de archivos está habilitado o deshabilitado si el tamaño del archivo es inferior al límite de tamaño de archivo medio.

ACTIVADO

Permite almacenar en caché el contenido de archivos.

DESACTIVADO

Desactiva el almacenamiento en caché del contenido de archivos.
