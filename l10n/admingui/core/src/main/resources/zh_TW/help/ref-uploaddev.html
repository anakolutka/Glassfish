部署企業應用程式/模組

使用 [部署企業應用程式/模組] 頁面，可部署應用程式。此頁面包含下列選項：

類型

正在部署的應用程式類型。此發行版本中唯一可用的應用程式類型為 Web 應用程式 (.war)]。

位置

正在部署之應用程式的歸檔位置。

下列選項指定可存取歸檔的位置，以及此歸檔是檔案還是目錄。

要上傳至伺服器的封裝檔案

歸檔是常駐在用戶端機器上的檔案，或是可從用戶端機器存取。

用戶端是您要在其上透過瀏覽器檢視管理主控台的主機。

可從應用程式伺服器存取的本機封裝檔案或目錄

歸檔是常駐在伺服器機器上的檔案，或是展開之目錄中未封裝的應用程式。

伺服器是執行應用程式伺服器網域管理伺服器的主機。

應用程式名稱

應用程式的名稱。

環境根目錄

應用程式的路徑。在應用程式的 URL 中，環境根目錄緊跟在連接埠號碼之後 (http://host:port/context-root/...)。環境根目錄的開頭必須是正斜線，例如 /hello。

虛擬伺服器

與此應用程式相關的虛擬伺服器。

啟用狀態

若選取此選項，則會啟用應用程式。此選項預設為選取。

啟用執行檢驗器

若選取此選項，則會執行檢驗器公用程式以檢查應用程式是否符合 JavaTM Platform, Enterprise Edition (Java EE 平台) 和 Enterprise Server。

啟用預編譯 JSP

若選取此選項，則會預編譯 JavaServer Pages (JSP) 檔案。若取消選取此選項，則首次存取 JSP 檔案時會於執行階段編譯這些檔案。此選項預設為取消選取。

程式庫

應用程式所需的程式庫 JAR 檔案。

說明

應用程式的簡短說明。
