Per avviare un cluster

Quando si avvia un cluster, vengono avviate tutte le istanze di GlassFish Server che non sono già in esecuzione presenti nel cluster.

Prima di iniziare

Assicurarsi che siano soddisfatti i seguenti prerequisiti:

  • tutti i nodi in cui risiedono le istanze del cluster sono abilitati per la connessione remota;

  • l'utente del server di amministrazione del dominio (DAS) può utilizzare il protocollo remoto DCOM (Distributed Component Object Model) o SSH (Secure Shell) per collegarsi all'host per qualsiasi nodo in cui risiedono le istanze del cluster.

Se uno qualsiasi di questi prerequisiti non è soddisfatto, avviare il cluster avviando singolarmente ogni istanza dalla riga di comando utilizzando il comando secondario start-local-instance(1).

  1. Selezionare il nodo Cluster nell'albero di navigazione.

    Viene visualizzata la pagina Cluster.

  2. Nella pagina Cluster selezionare la casella di controllo accanto al nome del cluster.

  3. Fare clic su Avvia cluster.

    Viene richiesto di confermare l'avvio del cluster.

  4. Confermare che si desidera avviare il cluster.

    Il cluster viene avviato. Lo stato di ogni istanza del cluster viene aggiornato nella pagina Cluster quando l'istanza viene avviata.
