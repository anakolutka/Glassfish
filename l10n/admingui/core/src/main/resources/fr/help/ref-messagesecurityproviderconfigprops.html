Propriétés propres aux configurations de fournisseur de sécurité des messages

Les propriétés suivantes sont disponibles pour la configuration d'un fournisseur de sécurité des messages.

server.config

Répertoire et nom de fichier d'un fichier XML qui contient les informations de configuration du serveur. La valeur par défaut est domain-dir/config/wss-server-config-1.0.xml .

débogage

La valeur true active le vidage des messages de débogage du fournisseur du serveur vers le journal du serveur. La valeur par défaut est false.

dynamic.username.password

La valeur true signale au fournisseur l'exécution de collecte du nom d'utilisateur et du mot de passe par le Gestionnaire de rappels pour chaque requête. Si la valeur est false, le nom d'utilisateur et le mot de passe de wsse:UsernameToken(s) sont recueillis une fois, lors de l'initialisation du module. Cette propriété s'applique uniquement à un module d'authentification client ClientAuthModule. La valeur par défaut est false.

encryption.key.alias

Spécifie la clé de chiffrement utilisée par le fournisseur. La clé est identifiée par son alias de keystore. La valeur par défaut est s1as.

signature.key.alias

Spécifie la clé de signature utilisée par le fournisseur. La clé est identifiée par son alias de keystore. La valeur par défaut est s1as.
