Nouveau service exécuteur géré

Utilisez la page Nouveau service exécuteur géré pour créer une ressource de service exécuteur géré.

Elle contient les options ci-après.

Nom JNDI

Nom unique identifiant la ressource.

Informations de contexte

Contexte de la ressource : chargeur de classe; JNDI, sécurité ou espace de travail.

Statut

Statut de la ressource. La ressource peut être activée ou désactivée. La valeur par défaut est Activé.

Priorité des threads

Priorité des threads à créer pour l'instance de service exécuteur géré. La valeur par défaut est 5.

Tâches à longue durée d'exécution

XXX.

Bloqué au bout de

Durée (en secondes) pendant laquelle une tâche peut être exécutée avant d'être considérée comme bloquée.

Taille de base

Nombre de threads à conserver dans un pool de threads. La valeur par défaut est 0.

Taille maximale du pool

Nombre maximal de threads qu'un pool peut contenir. La valeur par défaut est 2147483647.

Maintien des connexions

Durée (en secondes) pendant laquelle les threads peuvent rester inactifs lorsque le nombre de threads est supérieur à la taille de base. La valeur par défaut est 60.

Durée de vie des threads

Durée (en secondes) pendant laquelle chaque thread reste dans un pool de threads avant sa purge, que le nombre de threads soit supérieur à la taille de base ou non ou que le thread soit inactif ou non.

Capacité de la file d'attente de tâches

Taille de la file d'attente contenant les tâches soumises en attente d'exécution. La valeur par défaut est 2147483647.

Description

Description de la ressource.

Autres propriétés

Propriétés supplémentaires de la ressource. GlassFish Server ne définit aucune autre propriété pour les ressources de service exécuteur géré.
