重新部署已部署的 Web 应用程序

重新部署 Web 应用程序会在保留应用程序最初部署时的设置和其他选项的同时部署应用程序的新版本。

开始之前

确保您要重新部署的 Web 应用程序已经得到部署。

  1. 在导航树中，展开“应用程序”节点。
  2. 在“应用程序”节点下，选择“Web 应用程序”节点。

    此时将打开“Web 应用程序”页面。

  3. 在“已部署 Web 应用程序”表中，单击您要部署的应用程序行中的“重新部署”。

    此时将打开“重新部署企业应用程序/模块”页面。

  4. 如果您需要保留应用程序的活动会话并在重新部署时恢复这些会话，请选中“保持会话”复选框。

    如果无法保留或存储应用程序的任何活动会话，则在重新部署完成时将无会话可用。但是，重新部署将继续，并且记录一个警告。

  5. 指定包含要重新部署的应用程序的归档位置。
    • 如果此归档是位于客户机或可从客户机访问的文件，请选择“要上载到服务器的打包文件”选项。

      客户机是可通过浏览器查看管理控制台的主机。

      单击“浏览”以找到文件，或键入文件的完整路径。

    • 如果此归档是位于服务器计算机上的文件，或者是来自展开目录的未打包的应用程序，请选择“可从 Application Server 访问的本地打包文件或目录”选项。

      服务器计算机是运行域管理服务器 (Domain Administration Server, DAS) 的主机。

      键入文件或目录的完整路径名。从展开的目录部署适用于高级开发者，建议不要在生产环境中执行此操作。

  6. 单击“确定”。
