Modifier une application

Pour modifier une application existante, utilisez la page Modifier une application.

Celle-ci contient l'ensemble ou certaines des options suivantes, selon le type d'application.

Nom

Champ en lecture seule indiquant le nom de l'application que vous modifiez.

Le nom peut être suivi d'un identificateur de version facultatif dont il est séparé par un signe deux-points (:). Pour plus d'informations sur les versions des modules et des applications, reportez-vous à Module and Application Versions dans le manuel Oracle GlassFish Server Application Deployment Guide.

Statut

Si la case Activé est cochée, l'application est activée. Cette option est sélectionnée par défaut.

Racine du contexte

Pour une application Web, ce champ spécifie le chemin de l'application. Dans l'URL de l'application Web, la racine du contexte suit immédiatement le numéro de port (http://host:port/context-root/...). La racine du contexte doit commencer par une barre oblique, par exemple /hello.

Disponibilité

Si la case Activé est cochée, la haute disponibilité est activée pour la réalisation de points de reprise et éventuellement la passivation des sessions Web et des beans Session avec conservation de statut (SFSB). Si la valeur est False (valeur par défaut), tous les enregistrements de session Web et la réalisation de points de reprise SFSB sont désactivés pour l'application, l'application Web ou le module EJB spécifié. Si cette valeur est True, l'application ou le module spécifié est activé pour la haute disponibilité. Définissez cette option sur True seulement si la haute disponibilité est configurée et activée à des niveaux supérieurs, tels que les niveaux serveur et conteneur.

Cette option apparaît si des clusters ou des instances de serveur autonomes existent en plus de l'instance de serveur par défaut (server).

Serveurs virtuels

Serveurs virtuels associés à cette application.

L'option Serveurs virtuels apparaît si seule l'instance de serveur par défaut, server, existe. S'il existe des clusters ou d'autres instances de serveur autonomes, vous pouvez sélectionner des serveurs virtuels après le déploiement. Ouvrez la page Modifier une application, sélectionnez l'onglet Cible, puis Gérer les serveurs virtuels pour la cible de votre choix.

Java Web Start

(Pour certains types d'application) Si la case Activé est cochée, l'application utilise le logiciel Java Web Start. Il permet de déployer des applications Java à exécuter sur une JVM sans contrainte de navigateur.

Description

Brève description de l'application.

Emplacement

Emplacement de l'application déployée. Ce champ est en lecture seule.

Ordre de déploiement

Ordre de déploiement de l'application.

Les applications ayant le numéro le plus faible sont chargées en premier au démarrage du serveur. Une application ayant un ordre de déploiement de 102 est chargée avant une application ayant un ordre de déploiement de 110. Si l'ordre de déploiement n'est pas indiqué au moment où une application est déployée, l'ordre de déploiement par défaut de 100 est affecté. Si deux applications ont le même ordre de déploiement, l'application qui a été déployée en premier est chargée en premier. La spécification d'un ordre de déploiement s'avère utile si l'application a des dépendances et doit être chargée dans un certain ordre.

Bibliothèques

Fichiers JAR de bibliothèque requis par l'application. Ce champ est en lecture seule.

Modules et composants

Table affichant les noms des modules de l'application et leurs moteurs. Pour une adresse de service Web, vous pouvez sélectionner Afficher l'adresse pour obtenir la page Informations relatives à une adresse de service Web. Pour un client d'application, vous pouvez sélectionner Lancer pour afficher la page de lancement du client d'application ou Télécharger les stubs client pour télécharger les stubs client.
