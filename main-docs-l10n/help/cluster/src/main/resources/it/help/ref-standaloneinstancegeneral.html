Informazioni generali

La pagina Informazioni generali contiene le opzioni riportate di seguito.

Avvia

Pulsante che consente di avviare l'istanza. Se l'istanza è in esecuzione, il pulsante è disattivato.

Arresta

Pulsante che consente di arrestare l'istanza. Se l'istanza è arrestata, il pulsante è disattivato.

Riavvia

Pulsante che consente di riavviare l'istanza. Se l'istanza è arrestata, il pulsante è disattivato.

Visualizza file di log

Pulsante che consente di visualizzare i file di log per l'istanza.

Ruota log

Pulsante che consente di ruotare il file di log per l'istanza. Quando viene eseguita la rotazione del file di log, il file viene rinominato con un nome indicatore orario con il formato server.log_data-e-ora e viene creato un file di log vuoto per i nuovi messaggi di log. Le modifiche vengono applicate in modo dinamico. Non è necessario riavviare il server.

Recupera transazioni

Pulsante che consente di recuperare le transazioni per l'istanza.

Nome

Il nome assegnato all'istanza quando è stata creata. Questo campo è di sola lettura.

JVM

Se si fa clic su Report JVM, viene visualizzata una finestra distinta in cui sono disponibili i report relativi alla Java Virtual Machine, tra cui un report di riepilogo, un report sulla gestione della memoria e sulla garbage collection, un report sul caricamento delle classi e un dump del thread corrente.

Stato

Indica se l'istanza è in esecuzione o non è in esecuzione. Questo campo è di sola lettura.

Nodo

Il nome del nodo in cui risiede l'istanza. Se si fa clic sul nome, viene visualizzata la pagina Modifica nodo per il nodo.

Configurazione

La configurazione a cui fa riferimento l'istanza. Se si fa clic sulla configurazione, viene visualizzata la pagina Configurazioni per l'istanza.

Debug

Indica se il debug JPDA (Java Platform Debugger Architecture) (http://java.sun.com/javase/technologies/core/toolsapis/jpda/) è abilitato per l'istanza. Questo campo è di sola lettura.

Se un'istanza viene avviata da Console di amministrazione, il debug JPDA non viene abilitato per l'istanza. Per avviare un'istanza con il debug JPDA abilitato, utilizzare il comando secondario start-instance(1).

Porte HTTP

Lista separata da virgole dei numeri delle porte utilizzate dall'istanza per l'ascolto dei tipi di richiesta seguenti:

  • richieste di amministrazione;

  • richieste HTTP;

  • richieste HTTPS.

Questo campo è di sola lettura.

Porte IIOP

Lista separata da virgole dei numeri delle porte utilizzate dall'istanza per i tipi di connessione seguenti:

  • connessione IIOP sicura;

  • connessioni IIOP sicure con autenticazione del client;

  • connessioni IIOP.

Questo campo è di sola lettura.
