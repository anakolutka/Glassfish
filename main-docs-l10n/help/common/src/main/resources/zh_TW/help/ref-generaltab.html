一般資訊

GlassFish Server 在安裝時會建立一個名稱為 server 的應用程式伺服器執行處理。如有需要，您可以刪除該伺服器執行處理，並使用其他名稱建立一個新的執行處理。

每個 GlassFish Server 執行處理均有自己的 Java 組態、Java 資源、應用程式建置區域以及伺服器組態設定。變更某個應用程式伺服器執行處理並不會影響其他應用程式伺服器執行處理。一個管理網域中可有一個應用程式伺服器執行處理。

對多數使用者而言，一個應用程式伺服器執行處理已可滿足其需求。不過，依據您的環境，您可能想建立一或多個額外的應用程式伺服器執行處理。例如，在開發環境下，您可以使用不同的應用程式伺服器執行處理來測試不同的 GlassFish Server 組態，或者是比較和測試不同的應用程式建置。由於可以輕鬆地增加或刪除應用程式伺服器執行處理，因此您可以使用這些執行處理建立暫時的「沙箱」區域，在開發時進行實驗。

此外，您也可以為每個應用程式伺服器執行處理建立虛擬伺服器。在單一安裝的應用程式伺服器執行處理中，您可以提供公司或個人的網域名稱、IP 位址以及一些管理功能。對使用者而言，這就像是擁有自己的 Web 伺服器，卻不需要進行硬體與基本的伺服器維護。虛擬伺服器無法跨應用程式伺服器執行處理使用。如需有關虛擬伺服器的詳細資訊，請參閱JVM 一般設定。

在作業用的建置上，您可以針對多種用途使用虛擬伺服器來替代多個應用程式伺服器執行處理。不過，若虛擬伺服器不符合您的需求，您也可使用多個應用程式伺服器執行處理。

GlassFish Server 執行處理不會自動啟動。一旦啟動執行處理，執行處理會持續執行直到您將其停止為止。停止應用程式伺服器執行處理後，就會停止接受新連線，並等待所有未處理的連線完成。若您的機器當機或離線，伺服器就會結束，並且會遺失正在處理的任何請求。

使用「一般資訊」頁面可驗證 GlassFish Server 設定以及檢視 Java 虛擬機器資料。

「一般資訊」頁面包含以下資訊。

停止按鈕

按一下「停止」按鈕以停止 GlassFish Server。

重新啟動按鈕

按一下「重新啟動」按鈕以重新啟動 GlassFish Server。

檢視日誌檔按鈕

按一下「檢視日誌檔」按鈕，可檢視 GlassFish Server 執行處理或叢集的日誌檔。

循環日誌按鈕

按一下「循環日誌」按鈕，可循環管理伺服器 (名為 server) 的日誌檔。

復原作業事件按鈕

按一下「復原作業事件」按鈕，可復原復原作業事件頁面上管理伺服器 (名為 server) 的作業事件。

安全管理按鈕

在安全管理頁面上按一下「安全管理」按鈕，可啟用或停用安全管理。

名稱

目前伺服器的名稱。

狀態

伺服器執行處理的目前狀態。伺服器可以是已停止、已啟動或執行中。

正常執行時間

伺服器執行處理已連續執行的小時與分鐘數。這是唯讀欄位。

HTTP 負載平衡程式

負載平衡程式的名稱 (如果伺服器執行處理已啟用負載平衡程式的話)。

JVM

如果按一下「JVM 報告」，會開啟另外一個視窗並顯示 Java 虛擬機器的相關報告，包括摘要報告、記憶體管理和資源回收報告、類別載入報告以及目前繫線傾印。

組態目錄

主機電腦上包含組態檔案 (如 domain.xml) 的目錄。

安裝的版本

產品版本識別碼。

安全管理

安全管理的目前狀態。可能的值為「已啟用」與「未啟用」。

除錯

確定是否已啟用除錯功能。若已啟用除錯功能，則會同時顯示連接埠號碼。

HTTP 連接埠

目前組態用於 HTTP 請求與回應的連接埠。

IIOP 連接埠

目前組態用於 IIOP 請求與回應的連接埠。
