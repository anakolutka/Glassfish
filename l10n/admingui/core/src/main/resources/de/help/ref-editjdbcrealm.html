Die für die JDBCRealm-Klasse spezifischen Eigenschaften

Um Ihre Webseiten oder Webanwendungen zu schützen, können Sie die Sicherheit so einstellen, dass nur registrierte Benutzer darauf zugreifen können. Dies wird auch als Authentifizierungsmöglichkeit bezeichnet. Dieser Bereichstyp bezieht das Speichern von Anmeldeinformationen der Benutzer innerhalb der Datenbank mit ein. Enterprise Server verwendet die Datenbankinformationen und die aktivierte Option für den JDBC-Bereich in der Konfigurationsdatei. Folgende Eigenschaften sind für einen JDBC-Bereich erforderlich:

JAAS-Kontext

ID für das für diesen Bereich zu verwendende Anmeldemodul. Der einzig gültige Wert ist jdbcRealm.

J.N.D.I.

Der J.N.D.I.-Name für diesen Bereich. Der Standardwert ist jdbc/security.

Benutzertabelle

Tabelle mit den autorisierten Benutzern für diesen Bereich. Der Standardwert ist usertable.

Benutzernamenspalte

Name der Spalte mit den Benutzern in der Benutzertabelle. Der Standardwert ist userid.

Passwortspalte

Name der Spalte mit dem entsprechenden Benutzerpasswort in der Benutzertabelle. Der Standardwert ist password.
