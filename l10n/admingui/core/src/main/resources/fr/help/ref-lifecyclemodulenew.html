Nouveau module de cycle de vie

Utilisez la page Nouveau module de cycle de vie pour définir un nouveau module de cycle de vie.

La page Nouveau module de cycle de vie contient les options suivantes.

Nom

Nom du module de cycle de vie.

Nom de classe

Nom complet complet du fichier de classe du module de cycle de vie. La classe doit implémenter l'interface com.sun.appserv.server.LifecycleListener.

Classpath

Classpatch du module de cycle de vie. Le classpath indique là où se trouve le module de cycle de vie. La valeur par défaut est domain-dir/applications. Si le module est déjà dans ce répertoire (c'est-à-dire dans le classpath du serveur), ce champ peut être laissé vierge.

Ordre de chargement

Ordre dans lequel ce module de cycle de vie doit être chargé au démarrage. Les modules aux valeurs d'ordre de chargement à petit nombre sont chargés plus tôt. La valeur peut aller de 101 à la valeur MAXINT du système d'exploitation. Les valeurs de 1 à 100 sont réservées.

Statut

Si cette option est sélectionnée, le module de cycle de vie est activé. Cette option est activée par défaut.

Echec au chargement

Si cette option est sélectionnée, le serveur sera arrêté en cas d'échec de chargement du module de cycle de vie. Cette option est désactivée par défaut.

Serveurs virtuels

Le ou les serveur(s) virtuel(s) à associer au module de cycle de vie. Par défaut, aucun serveur virtuel n'est sélectionné.

Description

Description du module de cycle de vie.

Autres propriétés

Autres propriétés de module de cycle de vie. Le Enterprise Server ne définit aucune autre propriété pour les modules de cycle de vie.
