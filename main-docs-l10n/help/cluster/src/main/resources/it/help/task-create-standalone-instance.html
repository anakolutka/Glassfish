Per creare un'istanza standalone di GlassFish Server

Le istanze di GlassFish Server costituiscono la base per la distribuzione di un'applicazione. Un'istanza standalone non è membro di un cluster. Ogni istanza appartiene a un solo dominio e dispone di una struttura di directory, di una configurazione e di applicazioni distribuite proprie. Ogni istanza include inoltre il contenitore Web Java Platform, Enterprise Edition (Java EE) e il contenitore di tecnologia EJB (Enterprise JavaBeans). Ogni nuova istanza deve contenere un riferimento a un nodo che definisce l'host in cui risiederà l'istanza.

Nota:

Per creare un'istanza in un nodo di tipo CONFIG che rappresenta un host remoto, è necessario utilizzare il comando secondario create-local-instance(1). Per questa operazione non è possibile utilizzare Console di amministrazione.
Prima di iniziare

Assicurarsi che il nodo in cui deve risiedere l'istanza esista. Per le necessarie istruzioni, vedere Per creare un nodo.

  1. Selezionare il nodo Istanze standalone nell'albero di navigazione.

    Viene visualizzata la pagina Istanze server standalone.

  2. Nella pagina Istanze server standalone, fare clic su Nuovo.

    Viene visualizzata la pagina Nuova istanza server standalone.

  3. Nel campo Nome digitare il nome per l'istanza.

    Il nome deve soddisfare i requisiti riportati di seguito.

    • Il nome può contenere solo caratteri ASCII.

    • Il nome deve iniziare con una lettera, un numero o un carattere di sottolineatura.

    • Il nome può contenere solo i caratteri seguenti:

      • Lettere minuscole

      • Lettere maiuscole

      • Numeri

      • Trattino

      • Punto

      • Carattere di sottolineatura

    • Il nome deve essere univoco nel dominio e non deve essere il nome di un'altra istanza di GlassFish Server, di un cluster, di una configurazione denominata o di un nodo.

    • Il nome non deve essere domain, server né un'altra parola chiave riservata di GlassFish Server.

  4. Nell'elenco a discesa Nodo selezionare il nodo in cui risiederà l'istanza.

  5. Nell'elenco a discesa Configurazione selezionare la configurazione denominata che verrà utilizzata dall'istanza.

    Nota:

    Se si seleziona default-config, è necessario specificare nel passo successivo che la configurazione non è condivisa. La configurazione default-config non può essere condivisa.

  6. Specificare se la configurazione che verrà utilizzata dall'istanza è condivisa.

    • Se la configurazione non è condivisa, lasciare selezionata l'opzione Crea una copia della configurazione selezionata.

      La configurazione selezionata viene copiata al momento della creazione dell'istanza. Alla copia della configurazione viene assegnato il nome nome-istanza-config, dove nome-istanza è il nome dell'istanza che si sta creando.

    • Se la configurazione è condivisa, selezionare l'opzione Crea riferimento alla configurazione selezionata.

      L'istanza utilizzerà la configurazione denominata esistente specificata.

  7. Fare clic su OK.
