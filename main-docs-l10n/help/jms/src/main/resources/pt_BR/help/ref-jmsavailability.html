Disponibilidade de JMS

Use a página Disponibilidade do JMS para configurar o cluster do broker do the Message Queue para atender o cluster do GlassFish Server.

Cuidado:

Antes de usar a página Disponibilidade do JMS para reconfigurar um cluster existente, você deve seguir os procedimentos especiais para migrar para outro tipo de cluster do broker, conforme descrito em To Migrate Between Types of Embedded or Local Conventional Broker Clusters emOracle GlassFish Server High Availability Administration Guide. Se esses procedimentos especiais não forem seguidos, pode ocorrer perda ou corrompimento de dados e até mesmo a inutilização da sua configuração, dependendo das operações do JMS executadas no cluster existente.

A página de Disponibilidade do JMS contém as seguintes opções.

Observação:

Se quaisquer instâncias que utilizam a configuração estiverem sendo executadas, a página Disponibilidade do JMS exibirá uma advertência, definirá todas as opções como somente para leitura e desativará o botão Salvar.
Tipo de Serviço JMS

Os tipos de brokers (chamados hosts JMS no GlassFish Server) a serem usados no cluster do broker. O tipo escolhido determina quais tipos de clusters do broker estão disponíveis. Os tipos de broker são os seguintes:

Incorporado

A configuração e o ciclo de vida do broker são gerenciados pelo GlassFish Server. Cada instância clusterizada do GlassFish é atendida por um broker executado na mesma JVM da instância.

Se a opção Tipo de Serviço JMS for definida como Incorporado, apenas clusters do broker convencionais do Message Queue serão suportados. As opções para configuração do cluster convencional são exibidas.

Local

A configuração e o ciclo de vida do broker são gerenciados pelo GlassFish Server. Cada instância clusterizada do GlassFish é atendida por um broker executado em uma JVM separada no mesmo host da instância.

Se a opção Tipo de Serviço JMS estiver definida como Local, os clusters do broker convencional e aprimorado (altamente disponível) do Message Queue serão suportados. As opções para seleção do tipo de cluster e configuração do cluster são exibidas.

Remoto

O configuração e o ciclo de vida do broker e do cluster do broker são gerenciados usando as ferramentas administrativas do Message Queue.

Se a opção Tipo de Serviço JMS estiver definida como Remoto, nenhuma outra opção será exibida.

Tipo de Cluster JMS

O tipo de cluster do broker do Message Queue a ser usado para o cluster do GlassFish Server.

Se a opção Tipo de Cluster JMS estiver definida como Convencional, as seguintes opções serão exibidas.

Tipo de Armazenamento da Configuração JMS

O tipo de armazenamento de dados a ser usado para os dados de configuração do cluster convencional. Se esta opção estiver definida como Broker Mestre, um broker no cluster será designado como broker mestre e os dados de configuração serão armazenados por ele. Se esta opção estiver definida como Banco de Dados Compartilhado, os dados de configuração serão armazenados em um armazenamento de dados JDBC acessível a todos os brokers. Neste caso, as opções relacionadas ao banco de dados serão exibidas.

Tipo de Armazenamento da Mensagem JMS

O tipo de armazenamento de dados que cada broker usará para armazenar seus dados de mensagem. Se esta opção estiver definida como Arquivar, cada broker armazenará seus dados de mensagens em um armazenamento de dados baseado em arquivo. Se esta opção estiver definida como JDBC, cada broker armazenará seus dados de mensagens em um armazenamento de dados JDBC.

Se a opção Tipo de Cluster JMS estiver definida como Aprimorado (Altamente Disponível), serão exibidas as opções relacionadas ao banco de dados.

Fornecedor do Banco de Dados
URL do Banco de Dados
Usuário do Banco de Dados

O fornecedor do banco de dados, URL de acesso e o banco de dados JDBC a ser usado em qualquer uma destas situações:

  • Quando o tipo de armazenamento da configuração de um cluster convencional estiver definido como Banco de Dados Compartilhado

  • Quando o tipo de armazenamento de mensagens de um cluster convencional estiver definido como JDBC

  • Quando o tipo de cluster estiver definido como Aprimorado (Altamente Disponível)

Para obter informações sobre os fornecedores suportados e os formatos de URLs de acesso para cada fornecedor, consulte PJDBC-Based Persistence em Oracle GlassFish Server Message Queue Administration Guide.

Autenticação do Banco de Dados
Senha do Banco de Dados
Alias da Senha

As informações da senha para o usuário do banco de dados JDBC especificado em Usuário do Banco de Dados.

Se Autenticação do Banco de Dados estiver definida como Senha, a opção Senha do Banco de Dados será exibida. Se Autenticação do Banco de Dados estiver definida como Alias da Senha, a opção Alias da Senha será exibida.

Propriedades Adicionais

Propriedades adicionais para disponibilidade do JMS.

GlassFish Server não predefine nenhuma propriedade de disponibilidade de JMS adicional.
