Per modificare le impostazioni SSL per il connettore JMX

Prima di iniziare

Questo task risulta utile solo se è stata abilitata la sicurezza per il connettore JMX.

  1. Espandere il nodo Configurazione nell'albero di navigazione.

  2. Nel nodo Configurazione selezionare il nodo Servizio di amministrazione.

    Viene visualizzata la pagina Modifica connettore JMX.

  3. Selezionare la casella di controllo Abilitato dell'opzione Sicurezza per impostare la cifratura della comunicazione JMX.

    Questa opzione è disabilitata per impostazione predefinita.

  4. Fare clic sulla scheda SSL.

    Viene visualizzata la pagina SSL.

  5. Nella pagina SSL selezionare la casella di controllo Abilitato dell'opzione SSL3 per abilitare SSL3.

    Questa opzione è abilitata per impostazione predefinita.

  6. Selezionare la casella di controllo Abilitato dell'opzione TLS per abilitare TLS (Transport Layer Security).

    Questa opzione è abilitata per impostazione predefinita.

  7. Selezionare la casella di controllo Abilitazione dell'opzione Autenticazione client per richiedere l'identificazione dei client a ogni richiesta del server.

    Questa opzione è disabilitata per impostazione predefinita.

  8. Nel campo Pseudonimo certificato immettere lo pseudonimo del certificato del server nel database dei certificati o nel token PKCS#11.

    Nel certificato il formato del nome è nometoken:pseudonimo. L'inclusione della porzione tokenname: del nome in questo attributo è facoltativa.

  9. Nel campo Keystore immettere il nome del file keystore (ad esempio keystore.jks).

  10. Nel campo Algoritmo di protezione immettere il nome dell'algoritmo di gestione sicuro (ad esempio PKIX) da utilizzare per la convalida del percorso di certificazione.

  11. Nel campo Lunghezza massima certificato immettere il numero massimo di certificati intermedi non autoemessi che possono essere presenti in un percorso di certificazione.

    Questo campo viene utilizzato solo se il campo Algoritmo di protezione è impostato su PKIX. Il valore 0 indica che il percorso può contenere un solo certificato. Il valore -1 indica che la lunghezza del percorso non è vincolata (non esiste un valore massimo). L'impostazione di un valore minore di -1 comporta la restituzione di un'eccezione.

  12. Nel campo Truststore immettere il nome del file del truststore (ad esempio cacerts.jks).

  13. Nell'area Suite di cifratura specificare le suite di cifratura da utilizzare.

    Se non si aggiunge alcuna suite di cifratura, verranno utilizzate tutte le suite di cifratura disponibili.

  14. Fare clic su OK.
