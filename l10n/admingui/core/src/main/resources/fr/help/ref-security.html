Sécurité

Gestionnaire de sécurité

Si cette option est sélectionnée, le gestionnaire de sécurité est activé pour le domaine.

Une option JVM, -Djava.security.manager, est alors ajoutée au paramètre JVM de Enterprise Server. Voir Configurer les options JVM. Vous devez redémarrer le serveur pour activer la modification.

Vérifiez que vous avez bien accordé les autorisations nécessaires pour toutes les applications. Vous pouvez désactiver le gestionnaire de sécurité pour améliorer les performances.

Cette option est désactivée par défaut.

Journalisation d'audit

Si cette option est activée, le serveur chargera et exécutera tous les modules d'audit spécifiés dans le paramètre Modules d'audit. Si cette option est désactivée, le serveur n'a pas accès aux modules d'audit. Cette option est désactivée par défaut.

Domaine par défaut

Domaine actif (par défaut) utilisé par le serveur pour l'authentification. Les applications utilisent ce domaine sauf indication contraire de leur descripteur de déploiement. Tous les domaines configurés apparaissent dans la liste. La valeur par défaut est Fichier.

Principal par défaut

Spécifie le nom d'utilisateur par défaut. Le serveur utilise ce nom d'utilisateur lorsque aucun autre principal n'est fourni. Si vous saisissez une valeur dans ce champ, saisissez le mot de passe correspondant dans le champ Mot de passe principal par défaut.

Cet attribut n'est pas requis pour le fonctionnement normal du serveur.

Mot de passe principal par défaut

Mot de passe du principal par défaut spécifié dans le champ Principal. Cet attribut n'est pas requis pour le fonctionnement normal du serveur.

Modules d'audit

Liste des classes de fournisseurs de module d'audit, séparées par une virgule. Les modules répertoriés dans cette liste doivent déjà avoir été configurés. Si l'option Journalisation d'audit est activée, ce champ doit répertorier les modules d'audit. Par défaut, le serveur utilise un module d'audit nommé default. Pour plus d'informations sur la création de modules d'audit, consultez Créer un module d'audit.

Mappage principal-rôle par défaut activé

Si cette option est sélectionnée, le mappage principal-rôle par défaut est appliqué aux applications qui ne possèdent pas de mappage propre.

Classe de principal mappée

Indique une implémentation personnalisée de la classe java.security.La classe Principal utilisée dans le mappage principal-rôle par défaut.
