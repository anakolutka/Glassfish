일괄 처리 런타임을 구성하려면

  1. 탐색 트리에서 일괄 처리 작업 실행을 보려는 GlassFish Server 클러스터 또는 서버 인스턴스를 선택합니다.

    [일반 정보] 페이지가 열립니다.

  2. [일괄 처리] 탭을 선택합니다.

    [일괄 처리 작업 실행] 페이지가 열립니다.

  3. [구성] 탭을 누릅니다.

    [일괄 처리 런타임 구성] 페이지가 열립니다. 런타임은 관리 대상 실행기 서비스 및 데이터 소스를 사용하여 일괄 처리 작업을 실행합니다.

  4. [실행기 서비스 조회 이름] 드롭다운 목록에서 일괄 처리 작업에 스레드를 제공하는 데 사용할 관리 대상 실행기 서비스를 선택합니다.

    기본 관리 대상 실행기 서비스는 concurrent/__defaultManagedExecutorService입니다. 일괄 처리 작업이 일괄 처리 런타임에 제출된 후 작업 실행에 영향을 주지 않고 관리 대상 실행기 서비스를 변경할 수 있습니다.

  5. [데이터 소스 조회 이름] 드롭다운 목록에서 작업 정보를 저장하는 데 사용할 데이터 소스를 선택합니다.

    기본 데이터 소스는 jdbc/__TimerPool입니다. 첫번째 일괄 처리 작업이 실행을 위해 일괄 처리 런타임에 제출된 후에는 데이터 소스를 변경하지 마십시오. 데이터 소스를 변경해야 한다면 도메인을 정지했다가 재시작한 후 어떤 작업도 시작되거나 재시작되기 전에 변경 작업을 수행하십시오. 일단 데이터 소스가 변경되면 이전 데이터 소스에 저장된 정보에 액세스할 수 없습니다.

  6. [저장]을 누릅니다.
