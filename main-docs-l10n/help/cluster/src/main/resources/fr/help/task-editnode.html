Modification d'un noeud

La modification d'un noeud met à jour les données de configuration d'un noeud GlassFish Server. Elle permet également de modifier le type du noeud CONFIG en DCOM ou SSH, afin d'activer la communication à distance pour le noeud.

Si vous changez le type de noeud CONFIG en SSH, les valeurs par défaut sont appliquées si l'un des champs suivants n'est pas renseigné :

  • Domaine Windows

  • Utilisateur Windows

Si vous changez le type de noeud CONFIG en SSH, les valeurs par défaut sont appliquées si l'un des champs suivants n'est pas renseigné :

  • Port SSH

  • Nom utilisateur SSH

  • Fichier de clés

  1. Dans l'arborescence de navigation, développez le noeud Noeuds.

  2. Dans la liste des noeuds, sélectionnez le noeud à modifier.

    La page Modifier le noeud apparaît.

  3. Dans la liste déroulante Type, sélectionnez le type du noeud mis à jour.

    • Pour activer la prise en charge par le noeud de la communication via le protocole distant DCOM (Distributed Component Object Model), sélectionnez DCOM.

    • Pour activer la prise en charge par le noeud de la communication SSH (shell sécurisé), sélectionnez SSH.

    • Pour enlever la prise en charge de la configuration à distance à partir du noeud, sélectionnez CONFIG.

      Si vous sélectionnez CONFIG, aucune option de configuration SSH ou DCOM n'est affichée. Si le type en cours du noeud est DCOM ou SSH, et que vous sélectionnez CONFIG, les données de configuration DCOM ou SSH sont enlevées de la configuration du noeud lorsque vous enregistrez les modifications.

  4. Dans le champ Hôte du noeud, entrez le nom de l'hôte que le noeud mis à jour représentera.

    Le caractère obligatoire de ce champ dépend du type du noeud :

    • Si le type du noeud est CONFIG, le nom de l'hôte peut être omis. Le nom de l'hôte peut être déterminé lors de la création des instances résidant sur le noeud.

    • Si le type du noeud est DCOM ou SSH, le nom de l'hôte doit être spécifié. Sinon, une erreur se produit.

  5. Dans le champ Répertoire du noeud, entrez le chemin du répertoire qui contiendra les instances GlassFish Server créées sur le noeud.

    Si un chemin relatif est spécifié, il est relatif au répertoire as-install, où as-install correspond au répertoire d'installation de base du logiciel GlassFish Server sur l'hôte.

  6. Dans le champ Répertoire d'installation, entrez le chemin complet du parent du répertoire d'installation de base du logiciel GlassFish Server sur l'hôte, par exemple,/export/glassfish3.

  7. Terminez la mise à jour du noeud selon la méthode requise pour le type de noeud.

    1. Si le type de noeud est CONFIG, cliquez sur Enregistrer.

    2. Si le type de noeud est DCOM, suivez les instructions indiquées dans Achèvement de la mise à jour d'un noeud DCOM.

    3. Si le type de noeud est SSH, suivez les instructions indiquées dans Achèvement de la mise à jour d'un noeud SSH.
