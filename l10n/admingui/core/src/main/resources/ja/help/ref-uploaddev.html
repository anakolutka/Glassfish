エンタープライズアプリケーション/モジュールを配備する

アプリケーションを配備するには、「エンタープライズアプリケーション/モジュールを配備」ページを使用します。このページには次のオプションが含まれます。

形式

配備するアプリケーションの形式。このリリースで使用可能なアプリケーションの形式は「Web Application (.war)」だけです。

場所

配備するアプリケーションのアーカイブの場所。

次のオプションは、アーカイブにアクセス可能な場所とアーカイブがファイルであるかディレクトリであるかを指定します。

サーバーにアップロードされるパッケージファイル

このアーカイブは、クライアントマシンにあるファイル、またはクライアントマシンからアクセス可能なファイルです。

クライアントマシンとは、ブラウザを介して管理コンソールを表示しているホストです。

Application Server からアクセス可能なローカルのパッケージファイルまたはディレクトリ

このアーカイブは、サーバーマシンにあるファイル、または展開されたディレクトリにあるパッケージ化されていないアプリケーションです。

サーバーマシンとは、アプリケーションサーバーのドメイン管理サーバーを実行しているホストです。

アプリケーション名

アプリケーションの名前。

コンテキストルート

アプリケーションのパス。アプリケーションの URL では、コンテキストルートはポート番号の直後に続きます (http://host:port/context-root/...)。たとえば /hello のように、コンテキストルートがスラッシュで始まるようにしてください。

仮想サーバー

このアプリケーションと関連付けられている仮想サーバー。

状態

このオプションを選択すると、アプリケーションが有効になります。これは標準でオンになっています。

ベリファイアを実行

このオプションを選択すると、アプリケーションが JavaTM Platform, Enterprise Edition (Java EE platform) と Enterprise Server に適合しているかどうかを確認するベリファイアユーティリティが実行されます。

JSP をプリコンパイル

このオプションを選択すると、JavaServer Pages (JSP) ファイルがプリコンパイルされます。このオプションを選択解除すると、JSP ファイルは初めてアクセスされた実行時にコンパイルされます。このオプションは、デフォルトでは選択されていません。

ライブラリ

アプリケーションに必要なライブラリ JAR ファイル。

説明

アプリケーションの短い説明。
