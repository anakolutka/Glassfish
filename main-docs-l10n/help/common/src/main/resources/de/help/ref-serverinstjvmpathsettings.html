JVM-Pfadeinstellungen

Konfigurieren Sie auf der Seite "JVM-Pfadeinstellungen" den Pfad für die native Library. Der Pfad zur nativen Library ist eine Verkettung folgender Pfade:

  • Das Präfix für den Pfad der nativen Library

  • Der Serverpfad für die nativen Librarys (d. h. /lib

Die 64-Bit-Verzeichnisse der nativen Librarys (falls zutreffend)

Der Wert der Variablen java.library.path für die JVM, die zum Starten von GlassFish Server verwendet wird (je nach Betriebssystem verschieden)

Alle Pfade, die auf der Seite JVM-Profiler-Einstellungen angegeben sind, wenn der Profiler aktiviert ist

Das Suffix für den Pfad der nativen Library

Andere Einstellungen werden für Domains, die von einer älteren Version von GlassFish Server aktualisiert wurden, als schreibgeschützte Werte angezeigt. Sie werden in der Version v3 nicht unterstützt.

Die Seite "JVM-Pfadeinstellungen" umfasst die folgenden Optionen.

Name der Konfiguration

Der Name der Konfiguration, für die die Einstellungen auf dieser Seite angewendet werden. Dieses Feld ist schreibgeschützt.

Umgebungs-Classpath

Wenn Sie das Kontrollkästchen "Ignorieren" aktivieren, wird die Umgebungsvariable CLASSPATH ignoriert. Die Umgebungsvariable CLASSPATH ist für grundlegende Lernprogramme in der Programmierung geeignet, wird für Unternehmensumgebungen jedoch nicht empfohlen. Dies ist ein schreibgeschütztes Feld.

System-Classpath

Der Pfad zu den Klassen auf Ihrem System. Dies ist ein schreibgeschütztes Feld.

Classpath-Präfix

Vollständige Pfade zu den JAR-Dateien, die dem Classpath des Servers - jeweils einer pro Zeile - vorangestellt werden müssen. Dies ist ein schreibgeschütztes Feld.

Classpath-Suffix

Vollständige Pfade zu den JAR-Dateien, die an den Classpath - jeweils einer pro Zeile - angehängt werden müssen. Dies ist ein schreibgeschütztes Feld.

Präfix für Pfad zur nativen Library

Vollständige Pfade zu den JAR-Dateien, die dem Pfad zur nativen Library - jeweils einer pro Zeile - vorangestellt werden müssen.

Suffix für Pfad zur nativen Library

Vollständige Pfade zu den JAR-Dateien, die an den Pfad zur nativen Library - jeweils einer pro Zeile - angehängt werden müssen.
