영역을 만드는 방법

Enterprise Server는 file, certificate 및 admin-realm의 세 가지 영역으로 미리 구성됩니다. ldap, JDBC, solaris 및 사용자 정의 영역을 만들 수도 있습니다. 일반적으로 한 서버에 각 유형당 하나의 영역이 있지만 Enterprise Server에는 file 및 admin-realm의 두 파일 영역이 있습니다. 이 두 영역의 유형은 동일하지만 사용 목적은 다릅니다. 사용자 시스템의 각 가상 서버가 서로 다른 인증서 데이터베이스를 사용하도록 지정할 수도 있습니다.

  1. 탐색 트리에서 구성 노드를 선택합니다.
  2. 클러스터를 지원하도록 도메인이 구성된 경우 변경할 구성을 선택합니다.

    cluster 프로필 또는 enterprise 프로필을 통해 생성된 도메인을 예로 듭니다.

  3. 보안 노드를 확장합니다.
  4. 영역 노드를 선택합니다.
  5. 영역 페이지에서 새로 만들기를 누릅니다.

    새 영역 페이지가 표시됩니다.

  6. 이름 필드에 영역 이름을 입력합니다.
  7. 만들고 있는 영역에 클래스 이름을 지정합니다.

    다음 표에 유효 선택 항목이 나열되어 있습니다.

    영역 이름

    클래스 이름

    file

    com.sun.enterprise.security.auth.realm.file.FileRealm

    certificate

    com.sun.enterprise.security.auth.realm.certificate.CertificateRealm

    ldap

    com.sun.enterprise.security.auth.realm.ldap.LDAPRealm

    JDBC

    com.sun.enterprise.security.auth.realm.jdbc.JDBCRealm

    solaris

    com.sun.enterprise.security.auth.realm.solaris.SolarisRealm

    custom

    로그인 영역 클래스 이름
  8. 필요에 따라 추가 등록 정보를 영역에 추가합니다.
    1. 등록 정보 추가를 누릅니다.
    2. 이름 필드에 등록 정보 이름을 입력합니다.

      • 파일 영역 등록 정보에 대한 설명은 FileRealm 클래스와 관련된 등록 정보를 참조하십시오.

      • 인증서 영역 등록 정보에 대한 설명은 CertificateRealm 클래스와 관련된 등록 정보를 참조하십시오.

    3. 값 필드에 등록 정보 값을 입력합니다.
  9. 확인을 누릅니다.
