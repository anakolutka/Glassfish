Configuring Security for Listeners, Virtual Servers, and JMX Connectors

Each virtual server in the HTTP service provides network connections through one or more HTTP listeners. For general information about the HTTP service and HTTP listeners, see Chapter 13, Configuring the HTTP Service, in Sun Java System Application Server 9.1 Administration Guide.

Application Server supports CORBA (Common Object Request Broker Architecture) objects, which use the Internet Inter-Orb Protocol (IIOP) to communicate across the network. An IIOP listener accepts incoming connections from remote clients of EJB components and from other CORBA-based clients. For general information on IIOP listeners, see Chapter 15, Configuring the Object Request Broker, in Sun Java System Application Server 9.1 Administration Guide.

Use Admin Console:

  • To configure security for HTTP listeners

  • To configure security for IIOP listeners

  • To set listener security properties

  • To configure single sign-on (SSO) for a Virtual Server

  • To configure security for the Admin Service's JMX connector
