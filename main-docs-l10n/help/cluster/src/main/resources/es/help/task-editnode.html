Editar un nodo

Al editar un nodo se actualizan los datos de configuración de un nodo de GlassFish Server. Editar un nodo también permite cambiar el tipo de nodo de CONFIG a DCOM o SSH para activar la comunicación remota para el nodo.

Si cambia el tipo de nodo de CONFIG a DCOM, se aplicarán los valores por defecto si se deja vacío cualquiera de los campos siguientes:

  • Dominio de Windows

  • Usuario de Windows

Si cambia el tipo de nodo de CONFIG a SSH, se aplicarán los valores por defecto si se deja vacío cualquiera de los campos siguientes:

  • Puerto SSH

  • Nombre de Usuario SSH

  • Archivo de claves

  1. En el árbol de navegación, amplíe el nodo Nodos.

  2. En la lista de nodos, seleccione el nodo que vaya a editar.

    Se abrirá la página Editar Nodo.

  3. En la lista desplegable Tipo, seleccione el tipo de nodo actualizado.

    • Para activar el nodo para que admita comunicaciones mediante el protocolo remoto Distributed Component Object Model (DCOM), seleccione DCOM.

    • Para permitir que el nodo soporte la comunicación sobre Secure Shell (SSH), seleccione SSH.

    • Para que el nodo deje de soportar la configuración remota, seleccione CONFIG.

      Si selecciona CONFIG, no se mostrará ninguna opción para la configuración de SSH o la configuración de DCOM. Si el tipo actual del nodo es DCOM o SSH y selecciona CONFIG, se eliminarán los datos de configuración de DCOM o SSH de la configuración del nodo cuando guarde los cambios.

  4. En el campo Host de Nodo, indique el nombre del host que representará el nodo actualizado.

    Este campo es obligatorio o no en función del tipo de nodo:

    • Si el tipo de nodo es CONFIG., el nombre del host puede omitirse. El nombre del host se puede determinar al crear las instancias que residen en el nodo.

    • Si el tipo de nodo es DCOM o SSH, se debe especificar el nombre del host. De lo contrario, se produce un error.

  5. En el campo Directorio de Nodos, indique la ruta de acceso al directorio que contendrá las instancias de GlassFish Server que se crean en el nodo.

    Si especifica una ruta de acceso relativa, la ruta de acceso es relativa con respecto al directorio as-install, donde as-install es el directorio base de instalación del software GlassFish Server en el host.

  6. En el campo Directorio de Instalación, especifique la ruta de acceso completa al directorio principal del directorio base de instalación del software GlassFish Server en el host, por ejemplo /export/glassfish3.

  7. Termine la actualización del nodo tal y como lo exija el tipo de nodo.

    1. Si el tipo de nodo es CONFIG, haga clic en Guardar.

    2. Si el tipo de nodo es DCOM, siga las instrucciones de Terminar la Actualización de un Nodo DCOM.

    3. Si el tipo de nodo es SSH, siga las instrucciones de Terminar la Actualización de un Nodo SSH.
