Nueva configuración de seguridad de mensajes

Consulte la página Nueva configuración de seguridad de mensajes para crear una configuración HttpServlet para la seguridad de mensajes.

La página Nueva configuración de seguridad de mensajes contiene las siguientes opciones.

Nombre de Configuración

Nombre de la configuración a la que se aplican los valores de esta página. Este campo es de sólo lectura.

Capa de autenticación

Especifica la capa del mensaje en la que se lleva a cabo la autenticación. El valor debe ser SOAP o HttpServlet. Debido a que por defecto hay una configuración de SOAP disponible, la única opción es HttpServlet.

ID de Proveedor

Identificador único de la configuración del proveedor para esta configuración de seguridad de mensajes.

Proveedor por defecto

Si se selecciona esta opción, este proveedor será el proveedor por defecto. El proveedor por defecto se llamará para cualquier aplicación que no identifique un proveedor específico. Si el proveedor es el proveedor de cliente por defecto, el proveedor de servidor por defecto o ambos viene determinado por el tipo de proveedor.

Tipo de proveedor

Especifica el tipo de proveedor. Las opciones disponibles son:

client

Especifica que el proveedor es el proveedor de autenticación de clientes.

server

Especifica que el proveedor es el proveedor de autenticación del servidor.

client-server

Especifica que el proveedor es tanto el proveedor de autenticación del servidor como del cliente.

Nombre de clase

Nombre de la clase de implantación de Java del proveedor. Los proveedores de autenticación del cliente deben implantar la interfaz com.sun.enterprise.security.jauth.ClientAuthModule. Los proveedores del servidor deben implantar la interfaz com.sun.enterprise.security.jauth.ServerAuthModule. Los proveedores de servidor y cliente deben implantar ambas interfaces.
