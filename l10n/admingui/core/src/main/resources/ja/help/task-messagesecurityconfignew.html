メッセージセキュリティー設定を作成する

通常は、メッセージセキュリティー設定を作成する必要はありません。Enterprise Server には、SOAP 認証層用のメッセージセキュリティー設定が含まれています。デフォルトでは、この設定に対してプロバイダが有効になっていませんが、SOAP メッセージセキュリティー設定を編集してプロバイダを有効にすることができます。また、HttpServlet 認証層の設定を作成し、1 つまたは複数のプロバイダ設定を指定することもできます。

  1. ナビゲーションツリーで、「設定」ノードを展開します。
  2. 「セキュリティー」ノードを開きます。
  3. 「メッセージセキュリティー」ノードを選択します。

    「メッセージセキュリティー設定」ページが開きます。

  4. 「メッセージセキュリティー設定」ページで、「新規」をクリックします。

    「新しいメッセージセキュリティー設定」ページが開きます。

  5. 「新しいメッセージセキュリティー設定」ページで、「認証層」ドロップダウンリストから、設定を作成するメッセージ層を選択します。

    デフォルトで SOAP 設定を使用できるため、選択できるのは HttpServlet のみです。

  6. 「プロバイダ ID」フィールドに、このメッセージセキュリティー設定に対するプロバイダ設定の一意の識別子を入力します。
  7. 「デフォルトプロバイダ」チェックボックスを選択して、このプロバイダ設定がデフォルトプロバイダであることを指定します。
  8. 「プロバイダタイプ」ドロップダウンリストから、このプロバイダのタイプを選択します。

    次の値を選択できます。

    client

    プロバイダがクライアント認証プロバイダであることを指定します。

    server

    プロバイダがサーバー認証プロバイダであることを指定します。

    client-server

    プロバイダがクライアント認証プロバイダおよびサーバー認証プロバイダの両方であることを指定します。

  9. 「クラス名」フィールドに、プロバイダの Java 実装クラスの名前を入力します。

    クライアント認証プロバイダは com.sun.enterprise.security.jauth.ClientAuthModule インタフェースを実装する必要があります。サーバー側プロバイダは com.sun.enterprise.security.jauth.ServerAuthModule インタフェースを実装する必要があります。クライアントサーバープロバイダは、両方のインタフェースを実装する必要があります。

  10. (省略可能) 「追加プロパティー」セクションで、追加プロパティーを指定します。

    プロパティーを追加するには、「プロパティーを追加」ボタンをクリックします。表示される空白行で、「名前」フィールドにプロパティー名を入力し、「値」フィールドにプロパティー値を入力します。

    Enterprise Server は、メッセージセキュリティー設定の追加プロパティーを定義しません。
