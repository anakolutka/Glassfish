Editar Serviço de Contexto

Use a página Editar Serviço de Contexto para editar um recurso de serviço de contexto.

A página Editar Serviço de Contexto contém as opções a seguir.

Carregar Defaults

Botão para restaurar definições para seus valores default. Definições que não tiveram seus valores default alterados.

Nome JNDI

Um nome JNDI exclusivo que identifica o recurso. Por convenção, o nome começa com um indicador de tipo de recurso e uma barra, tais como concurrent/, jdbc/ ou jms/. O nome JNDI é um campo somente para leitura. Você pode especificar o nome JNDI ao criar um novo recurso

Nome Lógico JNDI

O nome lógico JNDI para o recurso. Este é um nome padrão do Java EE pelo qual o recurso pode ser acessado. O nome JNDI lógico também é um campo somente para leitura. Consulte Nomes Lógicos JNDI para obter mais informações.

Informações de Contexto

Os contextos a serem propagados para threads:, Classloader; JNDI, Segurança ou WorkArea. A propagação de contexto pode ser ativada ou desativada. Se a opção estiver ativada, os contextos selecionados serão propagados. Se esta opção estiver desativada, nenhum dos contextos será propagado, mesmo que eles estejam selecionados. O valor default é Ativado, com todos os quatro contextos selecionados.

Ordem de Implantação

A ordem de implantação deste recurso. Recursos com números menores são carregados primeiro na inicialização do servidor. O valor default é 100.

Descrição

Uma descrição deste recurso.

Status

Se o recurso está disponível no runtime.

Se apenas a instância do servidor default, server, existir no domínio, a disponibilidade de runtime do recurso poderá ser ativada ou desativada. A disponibilidade de runtime está ativada por default.

Se clusters ou instâncias standalone tiverem sido criados no domínio, utilize a guia Alvo para controlar a disponibilidade de runtime do recurso em seus alvos.

Propriedades Adicionais

Propriedades adicionais do recurso. O GlassFish Server não define propriedades adicionais para este tipo de recurso. Além disso, este tipo de recurso não usa atualmente propriedades adicionais.
