Informazioni sull'endpoint del servizio Web

Utilizzare la pagina Informazioni sull'endpoint del servizio Web per visualizzare le informazioni relative a un endpoint di servizio Web.

La pagina Informazioni sull'endpoint del servizio Web contiene le informazioni riportate di seguito.

Nome applicazione

Il nome dell'applicazione distribuita. Fare clic su questo collegamento o sul pulsante Indietro per tornare alla pagina Modifica applicazione.

Il nome può includere un identificativo di versione opzionale, che segue il nome ed è separato da questo mediante due punti (:). Per ulteriori informazioni sulle versioni di moduli e applicazioni, vedere Module and Application Versions in Oracle GlassFish Server Application Deployment Guide.

Tester

Il collegamento Tester è visibile solo se il tipo di distribuzione dell'applicazione è diverso da RI (ad esempio se è 109). Fare clic su questo collegamento per visualizzare una pagina che consente di fornire i parametri per eseguire il test dei metodi del servizio Web.

WSDL

WSDL (Web Services Definition Language) per l'endpoint. Fare clic su questo collegamento per visualizzare il file WSDL in una nuova scheda o finestra del browser.

Quando si fa clic sul collegamento WSDL, alcuni browser Web tentano di interpretare l'origine WSDL visualizzando quindi una pagina vuota. Per visualizzare l'origine WSDL con tali browser, utilizzare il comando del browser per la visualizzazione dell'origine nella pagina vuota.

Nome endpoint

Il nome dell'endpoint.

Nome servizio

Il nome del servizio Web.

Nome porta

Il nome della porta del servizio Web.

Tipo di distribuzione

Il tipo di distribuzione del servizio Web, ad esempio RI o 109.

Tipo di implementazione

Il tipo di distribuzione del servizio Web, ad esempio SERVLET o EJB.

Nome classe di implementazione

Il nome della classe di implementazione del servizio Web.

URI indirizzo endpoint

L'URI dell'indirizzo dell'endpoint.

Spazio di nomi

Lo spazio di nomi di destinazione associato al servizio Web.

Descrizione

La descrizione dell'endpoint, se esistente.
