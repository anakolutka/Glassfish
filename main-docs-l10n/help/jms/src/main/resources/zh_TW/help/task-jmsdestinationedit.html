編輯 JMS 目的地資源

  1. 在瀏覽樹狀結構中，展開「資源」節點。

  2. 在「資源」節點下，展開「JMS 資源」節點。

  3. 在「JMS 資源」節點下，選取「JMS 目的地資源」節點。

    將開啟「JMS 目的地資源」頁面。

  4. 在「JMS 目的地資源」頁面中，按一下您要編輯的目的地資源名稱。

    將開啟「編輯 JMS 目的地資源」頁面。

  5. 在「實體目的地名稱」欄位中，鍵入要與此資源關聯的實體目的地名稱。

    GlassFish Server 會在需要時自動建立實體目的地，並在您刪除目的地資源時將其刪除。

  6. 從「資源類型」下拉式清單中，選取「javax.jms.Topic」或「javax.jms.Queue」。

  7. 在「描述」欄位中，鍵入目的地資源的描述。

  8. 選取「啟用狀態」核取方塊以在執行階段啟用該資源。

    此選項預設為啟用。

  9. 在「其他特性」區域中，指定目的地資源的其他特性。

    GlassFish Server 未定義 JMS 目的地資源的任何其他特性。

    • 若要增加特性，請按一下「增加特性」按鈕。在出現的空白列中，於「名稱」欄位中鍵入特性名稱，然後在「值」欄位中鍵入特性值。

    • 若要修改特性，請編輯特性的「值」欄位。

    • 若要刪除特性，請選取您要刪除之特性「名稱」欄位左側的核取方塊，然後按一下「刪除特性」按鈕。

  10. 按一下「儲存」。

    提示:

    若要回到預設設定，請按一下「載入預設值」按鈕，然後按一下「儲存」。

  11. 或者，按一下「目標」標籤以移至「JMS 目的地資源目的地」頁面。在此頁面中，執行下列動作：

    1. 按一下「管理目標」以開啟「管理資源目標」頁面。

      在此頁面的「可用」欄與「已選取」欄之間移動目標。確定已在「已選取」欄中放置將建置使用資源之應用程式的一或多個目標。可用的目標包含可用的叢集與伺服器執行處理，以及預設的伺服器執行處理 server。按一下「確定」儲存變更。

    2. 選取目標的核取方塊，然後按一下「啟用」或「停用」以啟用或停用該目標的資源。
