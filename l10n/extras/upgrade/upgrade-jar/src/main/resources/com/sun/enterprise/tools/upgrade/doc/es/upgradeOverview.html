Descripción general de la herramienta de actualización

Utilice la herramienta de actualización para transferir la información de configuración de una instalación de Application Server 8.x o Application Server 9 a una instalación de Application Server 9.1. La información que se transfiere incluye datos sobre las aplicaciones implementadas, el ámbito de archivos, los certificados de seguridad y otros recursos y opciones de configuración del servidor.

La herramienta detecta automáticamente la versión de la instalación de servidor de origen especificada.

Nota: antes de iniciar el proceso de actualización, asegúrese de que tanto el servidor de origen (el servidor que está actualizando) y el servidor de destino (el servidor al que está actualizando) están detenidos.

Terminología de actualización

A continuación se muestran algunos términos importantes relacionados con el proceso de actualización:

  • Servidor de origen: instalación que se va a actualizar a la nueva versión.

  • Servidor de destino: instalación nueva a la que se va a actualizar.

  • Raíz de dominios: directorio en el que están creados los dominios. Este directorio, de forma predeterminada, es la ubicación especificada como AS_DEF_DOMAINS_PATH en asenv.conf

  • Directorio de dominio o domain-dir: es el directorio (dentro de la raíz de dominios) correspondiente a un dominio específico. En este directorio se encuentran todos los datos de configuración y de otro tipo relativos al dominio ubicado en él.

  • Root de instalación: directorio donde está instalado Application Server.

Si la casilla de verificación Actualizar se selecciona durante el proceso de instalación de Application Server, se abre automáticamente la pantalla Asistente de actualización una vez completada la instalación. El proceso de actualización se inicia.
