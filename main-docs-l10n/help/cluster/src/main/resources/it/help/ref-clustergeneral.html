Informazioni generali

La pagina Informazioni generali contiene le opzioni riportate di seguito.

Avvia cluster

Pulsante che consente di avviare il cluster. Se tutte le istanze nel cluster sono in esecuzione, il pulsante è disattivato.

Arresta cluster

Pulsante che consente di arrestare il cluster. Se tutte le istanze nel cluster sono arrestate, il pulsante è disattivato.

Esegui migrazione timer EJB

Pulsante che consente di eseguire la migrazione dei timer di tecnologia EJB (Enterprise JavaBeans) da un'istanza di GlassFish Server arrestata a un'istanza in esecuzione nel cluster. Se si fa clic su questo pulsante, viene visualizzata la pagina Esegui migrazione timer EJB. Se tutte le istanze nel cluster sono in esecuzione, il pulsante è disattivato.

Ruota log

Pulsante che consente la rotazione del file di log per il cluster. Quando viene eseguita la rotazione del file di log, il file viene rinominato con un nome indicatore orario con il formato server.log_data-e-ora e viene creato un file di log vuoto per i nuovi messaggi di log. Le modifiche vengono applicate in modo dinamico. Non è necessario riavviare il server.

Nome cluster

Il nome assegnato al cluster al momento della creazione del cluster. Questo campo è di sola lettura.

JVM

Se si fa clic su Report JVM, viene visualizzata una finestra distinta in cui sono disponibili i report relativi alla Java Virtual Machine, tra cui un report di riepilogo, un report sulla gestione della memoria e sulla garbage collection, un report sul caricamento delle classi e un dump del thread corrente.

Configurazione

La configurazione selezionata per il cluster al momento della creazione del cluster. Se si fa clic sulla configurazione, viene visualizzata la pagina Configurazioni per il cluster. Questo campo è di sola lettura.

GMS

Se questa opzione è abilitata, il servizio GMS (Group Management Service) viene avviato in ogni istanza di GlassFish Server nel cluster e nel server di amministrazione del dominio (DAS, Domain Administration Server). Il server DAS viene utilizzato in tutti i cluster per i quali è abilitato il servizio GMS. Il servizio GMS offre servizi di monitoraggio del cluster, appartenenza del cluster e comunicazione di gruppo. Questa opzione è abilitata per impostazione predefinita.

Porta multicast

Il numero della porta di comunicazione utilizzata dal servizio GMS per l'ascolto degli eventi di gruppo. Questa opzione deve specificare un numero di porta valido compreso nell'intervallo da 2048 a 49151. L'impostazione predefinita è un valore generato in modo automatico in questo intervallo.

Indirizzo multicast

L'indirizzo utilizzato dal servizio GMS per l'ascolto degli eventi di gruppo. Questa opzione deve specificare un indirizzo multicast compreso nell'intervallo da 224.0.0.0 a 239.255.255.255. L'impostazione predefinita è 228.9.XX.YY, dove XX e YY rappresentano valori indipendenti generati in modo automatico compresi tra 0 e 255.

Associa indirizzo di interfaccia

L'indirizzo IP (Internet Protocol) dell'interfaccia di rete alla quale è associato il servizio GMS. Questa opzione deve specificare l'indirizzo IP di un'interfaccia di rete locale. L'impostazione predefinita è costituita da tutti gli indirizzi pubblici dell'interfaccia di rete.

In un computer dotato di più home questa opzione consente di configurare l'interfaccia di rete utilizzata per il servizio GMS. Un computer dotato di più home possiede due o più interfacce di rete.

Per specificare un indirizzo valido per tutte le istanze di GlassFish Server nel cluster, utilizzare una proprietà di sistema per impostare l'indirizzo singolarmente per ciascuna istanza.

Ad esempio, utilizzare la pagina Proprietà di sistema del cluster per creare la proprietà di sistema GMS-BIND-INTERFACE-ADDRESS-nome-cluster. Impostare quindi l'opzione Associa indirizzo in questa pagina su ${GMS-BIND-INTERFACE-ADDRESS-nome-cluster} per specificare la proprietà di sistema. Infine, per ogni istanza nel cluster, impostare la proprietà di sistema GMS-BIND-INTERFACE-ADDRESS-nome-cluster sull'indirizzo dell'interfaccia di rete necessario nel computer dell'istanza.

Stato

Indica il numero delle istanze nel cluster in esecuzione e non in esecuzione. Questo campo è di sola lettura.
