HTTP リスナー

HTTP リスナーは、ポート番号と IP アドレスの一意の組み合わせを持つ必要があります。たとえば、IP アドレス 0.0.0.0 を指定すると、HTTP リスナーはマシン上の指定されたポートで、すべての設定済み IP アドレスを待機できます。また、HTTP リスナーはリスナーごとに一意の IP アドレスを指定できますが、同じポートを使用します。

HTTP リスナーは IP アドレスとポート番号の組み合わせなので、同じ IP アドレスと異なるポート番号 (1.1.1.1:8081 および 1.1.1.1:8082 など) を持つ複数の HTTP リスナーや、異なる IP アドレスと同じポート番号 (使用しているマシンがどちらのアドレスにも応答するように設定されている場合、1.1.1.1:8081 および 1.2.3.4:8081 など) を持つ複数の HTTP リスナーを持つことができます

ただし、HTTP リスナーに単一のポート上ですべての IP アドレスを待機する 0.0.0.0 を使用する場合は、この同じポート上に、特定の IP アドレスを待機する IP アドレスを HTTP リスナーに作成できません。たとえば、ある HTTP リスナーが 0.0.0.0:8080 (ポート 8080 上のすべての IP アドレス) を使用する場合、別の HTTP リスナーは 1.2.3.4:8080 を使用できません。

通常、Application Server が稼動するシステムでアクセスできる IP アドレスは 1 つだけであるため、HTTP リスナーは、ポートが異なる 0.0.0.0 IP アドレスを通常使用し、役割ごとに異なるポート番号を使用します。システムが複数の IP アドレスにアクセスできる場合は、各アドレスを異なる役割に使用できます。
