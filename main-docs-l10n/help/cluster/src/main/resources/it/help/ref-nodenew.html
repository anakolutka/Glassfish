Nuovo nodo

Utilizzare la pagina Nuovo nodo per creare un nodo di GlassFish Server. Un nodo rappresenta un host in cui è installato il software di GlassFish Server. È necessario un nodo per ogni host in cui risiedono le istanze di GlassFish Server.

Le opzioni contenute nella pagina Nuovo nodo dipendono dal tipo del nodo.

Opzioni per tutti i tipi di nodi

Le opzioni riportate di seguito sono disponibili per tutti i tipi di nodo.

Nome

Il nome del nodo.

Il nome deve soddisfare i requisiti riportati di seguito.

  • Il nome può contenere solo caratteri ASCII.

  • Il nome deve iniziare con una lettera, un numero o un carattere di sottolineatura.

  • Il nome può contenere solo i caratteri seguenti:

    • Lettere minuscole

    • Lettere maiuscole

    • Numeri

    • Trattino

    • Punto

    • Carattere di sottolineatura

  • Il nome deve essere univoco nel dominio e non deve essere il nome di un altro nodo, di un cluster, di una configurazione denominata o di un'istanza di GlassFish Server.

  • Il nome non deve essere domain, server né un'altra parola chiave riservata di GlassFish Server.

Tipo

Elenco a discesa che consente di specificare il tipo del nodo. Di seguito sono indicati i tipi possibili.

CONFIG

Il nodo non supporta la comunicazione remota. Se è selezionato il tipo CONFIG, non viene visualizzata alcuna opzione per la configurazione SSH o DCOM e il contenuto del campo Directory di installazione viene cancellato.

DCOM

Il nodo supporta la comunicazione sul protocollo remoto DCOM (Distributed Component Object Model). Il protocollo DCOM è disponibile solo nei sistemi Windows.

SSH

Il nodo supporta la comunicazione su SSH (Secure Shell).

Host nodo

Il nome dell'host rappresentato dal nodo. Questo campo può essere obbligatorio a seconda del tipo del nodo:

  • se il nodo è di tipo CONFIG, è possibile omettere il nome dell'host. Il nome dell'host può essere determinato al momento della creazione delle istanze che risiedono nel nodo.

  • Se il nodo è di tipo DCOM o SSH, è necessario specificare il nome dell'host. In caso contrario si verifica un errore.

Directory nodo

Il percorso della directory destinata a contenere le istanze di GlassFish Server create nel nodo. Se viene specificato un percorso relativo, il percorso è relativo rispetto alla directory as-install.

L'impostazione predefinita dipende dal tipo del nodo:

  • se il nodo è di tipo CONFIG, per il nodo non viene specificata alcuna directory per le istanze nella configurazione del server di amministrazione del dominio DAS. La directory del nodo può essere determinata al momento della creazione delle istanze che risiedono nel nodo.

  • Se il nodo è di tipo DCOM, l'impostazione predefinita è as-install\nodes, dove as-install è la directory di installazione di base del software di GlassFish Server nell'host.

  • Se il nodo è di tipo SSH, l'impostazione predefinita è as-install/nodes, dove as-install è la directory di installazione di base del software di GlassFish Server nell'host.

Directory di installazione

Il percorso completo della directory padre della directory di installazione di base del software di GlassFish Server nell'host, ad esempio /export/glassfish3.

L'impostazione predefinita dipende dal tipo del nodo:

  • se il nodo è di tipo CONFIG, per il nodo non viene specificata alcuna directory di installazione nella configurazione del server DAS. La directory di installazione può essere determinata al momento della creazione delle istanze che risiedono nel nodo.

  • Se il nodo è del tipo DCOM o SSH, l'impostazione predefinita è la directory padre della directory di installazione di base predefinita del software di GlassFish Server per il server DAS. Questa impostazione predefinita è utile solo se GlassFish Server è installato nella stessa posizione in tutti gli host.

Opzioni aggiuntive per i nodi DCOM

Le opzioni riportate di seguito sono disponibili solo se il campo Tipo è impostato su DCOM.

Installa GlassFish Server

Se si seleziona l'opzione Abilitato, il server DAS eseguirà la copia del software di GlassFish Server dall'host DAS all'host del nodo. Per copiare il software di GlassFish Server nell'host, il server DAS deve essere in grado di contattare l'host tramite DCOM.

Imponi

Se si seleziona l'opzione Abilitato, il nodo verrà creato nella configurazione DAS anche se la convalida dei parametri del nodo non riesce. Per convalidare i parametri di un nodo, il server DAS deve essere in grado di contattare l'host del nodo tramite DCOM.

Convalida DCOM

Se si seleziona l'opzione Abilitato, GlassFish Server esegue il test della connessione su DCOM sull'host del nodo prima di creare il nodo. Come parte del test GlassFish Server copia lo script denominato delete_me.bat sull'host.

Se si seleziona questa opzione, il campo Directory di test remota viene attivato per consentire di specificare la directory in cui GlassFish Server deve copiare lo script.

Directory di test remota

La cartella nell'host remoto in cui GlassFish Server deve creare il file delete_me.bat. L'utente specificato dall'opzione Nome utente Windows deve disporre dell'accesso in scrittura a questa cartella. In caso contrario si verifica un errore. L'impostazione predefinita è C:\.

Dominio Windows

Il nome del dominio Windows che contiene l'utente specificato dall'opzione Nome utente Windows. L'impostazione predefinita è il nome dell'host su cui Console di amministrazione è in esecuzione.

Nome utente Windows

L'utente nell'host del nodo che deve eseguire il processo per la connessione all'host tramite DCOM. L'impostazione predefinita è l'utente che esegue il processo DAS. Se l'opzione Host nodo è impostata su localhost-domain, l'opzione Nome utente Windows viene ignorata.

Autenticazione utente Windows

Elenco a discesa che consente di specificare la modalità di autenticazione dell'utente Windows per il login all'host del nodo. Di seguito sono riportati i metodi di autenticazione possibili.

Password

L'utente Windows viene autenticato tramite una password memorizzata nella configurazione del server DAS.

Attenzione:

La password viene memorizzata come testo in chiaro nella configurazione del server DAS. Per maggiore sicurezza, si consiglia di utilizzare un alias password.

Se si seleziona questa opzione, viene attivato il campo Password utente Windows per consentire l'immissione della password.

Alias password

L'utente Windows viene autenticato tramite un alias password. Se si seleziona questa opzione, selezionare l'alias password dall'elenco a discesa Alias password.

Password utente Windows

La password che verrà utilizzata dall'utente Windows per collegarsi all'host del nodo.

Alias password

Elenco a discesa degli alias password definiti per il dominio.

Opzioni aggiuntive per i nodi SSH

Le opzioni riportate di seguito sono disponibili solo se il campo Tipo è impostato su SSH.

Installa GlassFish Server

Se si seleziona l'opzione Abilitato, il server DAS eseguirà la copia del software di GlassFish Server dall'host DAS all'host del nodo. Per copiare il software di GlassFish Server nell'host, il server DAS deve essere in grado di contattare l'host tramite SSH.

Imponi

Se si seleziona l'opzione Abilitato, il nodo verrà creato nella configurazione DAS anche se la convalida dei parametri del nodo non riesce. Per convalidare i parametri di un nodo, il server DAS deve essere in grado di contattare l'host del nodo tramite SSH.

Porta SSH

La porta da utilizzare per le connessione SSH all'host del nodo. L'impostazione predefinita è 22. Se l'opzione Host nodo è impostata su localhost-domain, l'opzione Porta SSH viene ignorata.

Nome utente SSH

L'utente nell'host del nodo che deve eseguire il processo per la connessione all'host tramite SSH. L'impostazione predefinita è l'utente che esegue il processo DAS. Per garantire che il server DAS possa leggere il file della chiave privata SSH di questo utente, specificare l'utente che esegue il processo DAS. Se l'opzione Host nodo è impostata su localhost-domain, l'opzione Nome utente viene ignorata.

Autenticazione utente SSH

Elenco a discesa che consente di specificare la modalità di autenticazione dell'utente SSH per il login all'host del nodo. Di seguito sono riportati i metodi di autenticazione possibili.

File di chiave

L'utente SSH viene autenticato tramite un file di chiave privata SSH. Se si seleziona questa opzione, specificare il percorso assoluto del file nel campo File di chiave.

Password

L'utente SSH viene autenticato tramite una password memorizzata nella configurazione del server DAS.

Attenzione:

La password viene memorizzata come testo in chiaro nella configurazione del server DAS. Per maggiore sicurezza, si consiglia di utilizzare un file di chiave o un alias password.

Se si seleziona questa opzione, viene attivato il campo Password utente SSH per consentire l'immissione della password.

Alias password

L'utente SSH viene autenticato tramite un alias password. Se si seleziona questa opzione, selezionare l'alias password dall'elenco a discesa Alias password.

Imposta SSH

Se si seleziona l'opzione Abilitato, GlassFish Server imposta una chiave SSH sull'host del nodo. GlassFish Server imposta la chiave SSH copiando i file di chiave esistenti dall'host del server DAS o, se necessario, generando i file prima di copiarli. Questa opzione è disponibile solo se l'opzione Autenticazione utente SSH è impostata su File di chiave.

Genera chiave

Se si seleziona l'opzione Abilitato, GlassFish Server genera i file di chiave SSH prima di copiare i file nell'host locale, qualora i file non esistano. Se i file di chiave SSH esistono, GlassFish Server non li genera, anche se è selezionata l'opzione Abilitato.

Nota:

Se i file di chiave SSH non esistono, è necessario selezionare l'opzione Abilitato. In caso contrario, si verifica un errore quando si tenta di salvare le modifiche.
Password utente SSH

La password che verrà utilizzata dall'utente SSH per collegarsi all'host del nodo.

Alias password

Elenco a discesa degli alias password definiti per il dominio.

File di chiave pubblica

Il percorso assoluto del file di chiave pubblica SSH per l'utente specificato dall'opzione Nome utente SSH. Il contenuto del file di chiave pubblica viene aggiunto al file .ssh/authorized_keys dell'utente sull'host del nodo. Se il file .ssh/authorized_keys non esiste sull'host, GlassFish Server lo crea.

File di chiave

Il percorso assoluto del file di chiave privata SSH per l'utente specificato dall'opzione Nome utente SSH. Questo file viene usato per l'autenticazione al daemon sshd nell'host del nodo.

Nota:

GlassFish Server supporta anche l'autenticazione della password tramite la voce AS_ADMIN_SSHPASSWORD del password file. Il password file è specificato nell'opzione --passwordfile della utility asadmin(1).

Se il file della chiave privata SSH è protetto da una passphrase, il password file deve contenere la voce AS_ADMIN_SSHKEYPASSPHRASE.

Il percorso del file di chiave deve essere raggiungibile dal server DAS e il file deve essere leggibile per il server DAS.

L'impostazione predefinita è un file di chiave nella directory .ssh dell'utente. Se vengono trovati più file di chiave, GlassFish Server utilizza l'ordine di preferenza riportato di seguito.

  1. id_rsa

  2. id_dsa

  3. identity
