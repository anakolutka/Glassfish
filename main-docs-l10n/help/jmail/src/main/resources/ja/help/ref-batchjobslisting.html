バッチ・ジョブ実行

GlassFish Serverは、バッチ・ジョブのスケジュールと実行用にバッチ・ランタイムを提供します。バッチ・ジョブは一般的に、長期実行のバルク指向のタスクで、一連のステップが含まれます。バッチ・アプリケーションはバッチ・ランタイムにジョブを送信し、ステップの実行方法やタイミングを指示します。バッチ・ランタイムはステップを処理し、ジョブ・リポジトリのジョブに関する情報を格納します。GlassFish Serverでは、ジョブ・リポジトリはデータベースです。

バッチ・ジョブの記録された実行の詳細を表示するには、バッチ・ジョブ実行ページを使用します。

バッチ・ジョブ実行ページには、次の情報が表示されます。

インスタンス名
クラスタ名

構成が適用されるGlassFishクラスタまたはサーバー・インスタンスの名前。

バッチ・ジョブ

インスタンスまたはクラスタの記録されたバッチ・ジョブ実行のリスト。各実行について、次の情報が表示されます:

実行ID

バッチ・ジョブの実行に割り当てられたID。実行IDをクリックすると、実行のバッチ・ジョブ実行の詳細ページが表示されます。新しい実行は、ジョブが初めて開始されたとき、および既存の実行が再開されたときに作成されます。

ジョブ名

ジョブの名前。

バッチ・ステータス

バッチ・ランタイムにより設定されたとおりの実行ステータス。

終了ステータス

ジョブのジョブXMLまたはバッチ・アプリケーションにより設定されたとおりの実行ステータス。終了ステータスが明示的にオーバーライドされないかぎり、デフォルトでは、終了ステータスとバッチ・ステータスは同一です。

インスタンスID

バッチ・ジョブのインスタンスに割り当てられたID。バッチ・ジョブのインスタンスには、複数の実行を含めることができます。

開始時間

実行の開始時間。

終了時間

実行の終了時間。
