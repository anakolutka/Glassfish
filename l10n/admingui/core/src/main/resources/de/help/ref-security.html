Sicherheit

Sicherheitsmanager

Wenn Sie diese Option aktivieren, wird der Sicherheitsmanager für diese Domäne aktiviert.

Bei Aktivierung wird eine JVM-Option, -Djava.security.manager, zur JVM-Einstellung von Enterprise Server hinzugefügt. Siehe So konfigurieren Sie die JVM-Optionen. Um diese Änderung zu übernehmen, muss der Server neu gestartet werden.

Stellen Sie sicher, dass Sie über die erforderlichen Berechtigungen für alle Anwendungen verfügen. Der Sicherheitsmanager kann zum Verbessern der Leistung deaktiviert werden.

Die Option errmon ist standardmäßig deaktiviert.

Überwachungsprotokollierung

Bei Aktivierung kann der Server alle in der Einstellung "Überwachungsmodule" angegebenen Überwachungsmodule laden und ausführen. Bei Deaktivierung greift der Server nicht auf Überwachungsmodule zu. Die Option errmon ist standardmäßig deaktiviert.

Standardbereich

Der aktive (standardmäßige) Bereich, den der Server für die Authentifizierung verwendet. Die Anwendungen verwenden diesen Bereich, es sei denn, ihre Bereitstellungsbeschreibung gibt einen anderen Bereich an. Alle konfigurierten Bereiche werden in der Liste angezeigt. Der Standardwert ist "File".

Standard-Principal

Gibt den standardmäßigen Benutzernamen an. Der Server verwendet diesen Benutzernamen, wenn kein anderer Principal angegeben wird. Wenn Sie in diesem Feld einen Wert eingeben, muss auch das entsprechende Passwort im Feld "Standardmäßiges Principal-Passwort" angegeben werden.

Dieses Attribut ist für den normalen Serverbetrieb nicht erforderlich.

Standardmäßiges Principal-Passwort

Das Passwort für den Standard-Principal, der im Feld "Standard-Principal" angegeben wurde. Dieses Attribut ist für den normalen Serverbetrieb nicht erforderlich.

Überwachungsmodule

Liste mit Überwachungsmodul-Anbieterklassen, durch Kommata getrennt. Hier aufgeführte Module müssen bereits konfiguriert sein. Wenn die Überwachungsprotokollierung aktiviert ist, muss dieses Feld Überwachungsmodule auflisten. Standardmäßig verwendet der Server das Überwachungsmodul default. Weitere Informationen zum Erstellen neuer Überwachungsmodule finden Sie unter So erstellen Sie ein Überwachungsmodul.

Standardmäßige Principal/Rolle-Zuordnung aktiviert

Wenn Sie diese Option aktivieren, wird die Principal/Rolle-Zuordnung auf Anwendungen angewendet, die keine anwendungsspezifische Zuordnung aufweisen.

Zugeordnete Principal-Klasse

Gibt eine benutzerdefinierte Implementierung der java.security. an.Die Principal-Klasse, die in der standardmäßigen Principal/Rolle-Zuordnung verwendet wird.
