Update-Tool - Schlüsselkonzepte

Das Update-Tool verwaltet die Installationen der Enterprise Server-Softwarekomponenten, die in einem einzelnen Dateisystemverzeichnis auf Ihrem System enthalten sind.

Die Schlüsselkonzepte für das Update-Tool werden in den folgenden Abschnitten beschrieben.

Komponente

Eine Komponente ist die installierbare Einheit, die vom Update-Tool verwaltet wird. Eine Komponente besteht aus Dateien, die als Einheit auf Ihrem System innerhalb eines Installations-Image installiert sind.

Installations-Image

Ein Installations-Image ist das Verzeichnis, das die Softwarekomponenten enthält, einschließlich des Inhalts des Verzeichnisses. Das Installations-Image wird dem Update-Tool als Ergebnis des Herunterladens und Installierens der Enterprise Server-Software hinzugefügt. Um das Hinzufügen und Aktualisieren von Enterprise Server-Softwarekomponenten zu ermöglichen, enthält jedes Installations-Image Informationen zur Installation von Komponenten.

Netzwerk-Repository

Ein Netzwerk-Repository enthält folgende Elemente:

  • Komponenten, die einem Installations-Image hinzugefügt werden können

  • Updates von vorhandenen Komponenten eines Installations-Image

Ein Installations-Image ist mit einem oder mehreren Netzwerk-Repositorys verbunden. Jedes Netzwerk-Repository wird durch einen Namen und eine entsprechende URL dargestellt. Ein Repository-Name ist eindeutig für jedes Installations-Image und steht sowohl für die Organisation oder das Projekt, die das Repository verwalten, als auch für einen klar definierten Inhaltsbereich.

Add-On-Komponente

Eine Add-On-Komponente ist eine Komponente des Installations-Image, die noch nicht installiert ist.
