Nuevo Listener HTTP

Utilice la página Nuevo Listener HTTP para crear un nuevo listener HTTP.

Cuando se crea un listener HTTP, el separador SSL se puede editar también.

La página Nuevo Listener HTTP contiene las opciones siguientes.

Nombre de Configuración

Nombre de la configuración a la que se aplican los valores de esta página. Este campo es de sólo lectura.

Nombre

Nombre único de listener. Un nombre de listener HTTP no puede empezar con un número.

Puerto

Número de puerto en el que recibirá el listener. Los valores válidos se encuentran entre 1 y 65535. En un sistema UNIX, la creación de sockets que reciben en los puertos 1-1024 requiere privilegios de superusuario.

Estado

Estado del listener HTTP. El listener puede estar activado o desactivado. Esta opción se encuentra activada por defecto. Si el listener se encuentra desactivado, cualquier intento de establecer una conexión con él tiene como resultado una excepción de socket (java.net.ConnectException).

En las versiones anteriores a la 9.1 de GlassFish Server, un listener cuyo atributo activado se hubiera definido como false devolvía un código de respuesta 404 para cualquier solicitud que se le enviaba. Para lograr este comportamiento en la versión actual de GlassFish Server, defina el estado del listener en activado y defina cada estado de servidor virtual asociado como desactivado. Un servidor virtual incluye sus listeners asociados en la lista desplegable Listeners de Red.

Seguridad

Si se selecciona esta opción, se activa la seguridad para el listener HTTP. Esta opción se activada por defecto.

Si está configurando el admin-listener para el servidor de administración de dominio (DAS), que usa la configuración server-config, esta opción será de sólo lectura. Para activar la seguridad de este listener, debe activar primero la administración segura. Consulte la sección Administración Segura.

Listener JK

Si esta opción se encuentra seleccionada, el listener soporta el protocolo mod-jk de Apache. El protocolo mod-jk es un protocolo de red/comunicación de propiedad entre httpd de Apache (servidor web) y una instancia back-end de GlassFish Server. Si el listener soporta este protocolo, se supone que httpd maneja todo el contenido estático, mientras que cualquier solicitud de recursos dinámicos (es decir, servlets y JSP) se direccionan al listener HTTP de mod jk activado de GlassFish Server a través del protocolo mod-jk. Esta opción se activada por defecto.

Dirección

Dirección IP en la que actuará el listener de red. La dirección se puede especificar en par punteado o notación IPv6. Puede ser cualquiera (para INADDR_ANY) para recibir en todas las direcciones IP. Puede ser un nombre de host.

Servidor Virtual por Defecto

Servidor virtual que se va a asociar a este listener HTTP. Utilice la página Servidores Virtuales para definir los servidores virtuales.

Pool de Threads

Pool de threads asociado al listener HTTP. Normalmente, puede seleccionar uno de los dos pools de threads que están configurados cuando se instala GlassFish Server.

Nombre de servidor

Nombre de host que se utiliza en las URL que el servidor envía al cliente. Este nombre es el nombre de alias si el servidor utiliza un alias. Si el servidor no utiliza un alias, deje vacío este campo.

Este valor afecta a las URL que genera automáticamente el servidor; no afecta a las URL de los directorios y archivos almacenados en el servidor. Si el servidor utiliza un alias, el nombre-servidor debería ser el nombre de alias. Si se agregan dos puntos y un número de puerto, dicho puerto se usará en las URL que el servidor envía al cliente.
