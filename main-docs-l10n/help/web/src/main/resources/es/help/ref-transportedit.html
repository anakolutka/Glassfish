Editar transporte

Utilice la página Editar Transporte para modificar la configuración de un transporte existente.

La página Editar Transporte contiene las siguientes opciones.

Cargar Valores por Defecto

Botón para restaurar configuración que tiene valores por defecto a sus valores por defecto. Las configuraciones que no tienen valores por defecto no se modifican.

Nombre de Configuración

Nombre de la configuración a la que se aplican los valores de esta página. Este campo es de sólo lectura.

Nombre

Nombre del transporte. Este campo es de sólo lectura. Sólo puede especificar el nombre al crear un nuevo transporte.

Nombre de clase

Nombre de clase de la implantación del transporte. El valor por defecto es com.sun.grizzly.TCPSelectorHandler.

Manejador de Clave de Selección

El nombre de clase del manejador de clave de selección. Un manejador de clave de selección es una clase abstracta que gestiona el ciclo de vida de las claves de selección. Si no se especifica ningún valor, se utiliza una implantación por defecto.

Tipo de Buffer de Bytes

El tipo de buffer de bytes que se va a utilizar. Las opciones disponibles son HEAP y DIRECT. El valor por defecto es HEAP.

Threads de Responsable de Aceptación

Número de procesadores del equipo. Un valor de -1 indica que Grizzly calculará por su cuenta el número de threads del dispositivo de aceptación según el número de procesadores de la máquina. El valor por defecto es 1.

Para definir el número de threads de procesamiento de solicitudes, defina el valor del Tamaño Máximo del Pool de Threads del pool de threads utilizado por un listener de red que emplea este transporte.

Recuento Máximo de Conexiones

Número máximo de conexiones pendientes en un listener de red que utiliza este transporte. El valor por defecto es 4096.

Tamaño de Buffer

Tamaño de buffer, en bytes, que se proporciona para los flujos de entrada creados por un listener de red que utiliza este transporte. El valor por defecto es 8192.

Timeout de Clave Inactiva

Número de segundos después de los cuales se cancelará una clave inactiva y se cerrará el canal. El valor por defecto es 30.

Timeout de Lectura

Número de milisegundos que GlassFish Server esperará durante la fase de análisis de cabecera y cuerpo de una operación de lectura. El valor por defecto es 30000.

Timeout de Sondeo de Selector

Número de milisegundos que un selector NIO bloqueará la espera de eventos (solicitudes de usuarios). El valor por defecto es 1.000.

Timeout de Escritura

Número de milisegundos que GlassFish Server espera antes de considerar que el cliente remoto está desconectado al escribir la respuesta. El valor por defecto es 30000.

Configuración de Pantalla

Si esta opción está seleccionada, la configuración interna de Grizzly se vaciará en los logs del servidor. Esta opción puede proporcionar información útil para la depuración. Esta opción se activada por defecto.

Investigar

Si esta opción está seleccionada, la información de la respuesta/solicitud se vuelca en el log del servidor. Puede que esta opción proporcione información útil para realizar la depuración, pero reduce significativamente el rendimiento. Esta opción se activada por defecto.

TCP sin Retraso

Si esta opción está seleccionada, la opción TCP_NODELAY (también llamada algoritmo de Nagle) está activada. Esta opción se encuentra activada por defecto.
