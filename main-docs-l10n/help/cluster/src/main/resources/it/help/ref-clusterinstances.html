Istanze server in cluster

Utilizzare la pagina Istanze server in cluster per creare e gestire le istanze di GlassFish Server nel cluster corrente.

Il campo Nome cluster è un campo di sola lettura in cui viene visualizzato il nome del cluster corrente.

Nella pagina Istanze server in cluster viene visualizzata la lista delle istanze membro del cluster corrente. Per ogni istanza vengono fornite le informazioni riportate di seguito.

Nome

Il nome assegnato all'istanza quando è stata creata. Se si fa clic sul nome, viene visualizzata la pagina Informazioni generali per l'istanza.

Peso LB

Il numero intero che rappresenta il peso, o fattore ponderale, per il bilanciamento del carico dell'istanza.

Il fattore ponderale per il bilanciamento del carico determina la proporzione di tutte le richieste inviate al cluster che l'istanza dovrebbe elaborare.

Ad esempio, in un cluster a due istanze può essere necessaria un'istanza per elaborare una di quattro richieste e un'altra istanza per elaborarne tre di quattro. In questa situazione, impostare su 1 il peso dell'istanza che elabora una delle quattro richieste e su 3 il peso dell'altra istanza. Se si preferisce utilizzare valori percentuali, impostare i valori di peso delle istanze rispettivamente su 25 e 75.

Il valore di peso predefinito è 100.

Configurazione

La configurazione a cui fa riferimento l'istanza. Se si fa clic sulla configurazione, viene visualizzata la pagina Configurazioni per l'istanza.

Nodo

Il nome del nodo in cui risiede l'istanza. Se si fa clic sul nome, viene visualizzata la pagina Modifica nodo per il nodo.

Stato

Indica se l'istanza è in esecuzione, richiede il riavvio per essere aggiornata con le modifiche della configurazione o non è in esecuzione. Questo campo è di sola lettura.

La tabella Istanze server contiene inoltre le opzioni riportate di seguito.

Nuovo

Pulsante che consente di aggiungere un'istanza al cluster.

Elimina

Pulsante che consente di rimuovere una o più istanze selezionate dal cluster.

Attenzione:

Se si utilizza un cluster JMS (Java Message Service) con un broker principale, non eliminare l'istanza associata al broker principale. Se è necessario eliminare l'istanza, utilizzare il comando secondario change-master-broker(1) per assegnare il broker principale a un'altra istanza.
Avvia

Pulsante che consente di avviare una o più istanze selezionate nel cluster.

Arresta

Pulsante che consente di arrestare una o più istanze selezionate nel cluster.
