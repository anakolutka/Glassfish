Cómo volver a implementar una aplicación implementada

Al volver a implementar una aplicación se implementa una versión nueva de la aplicación y al mismo tiempo se preserva la configuración y otras opciones propias de la aplicación cuando se implementó inicialmente.

Antes de empezar

Asegúrese de que la aplicación que está volviendo a implementar ya está implementada.

  1. En el árbol de navegación, seleccione el nodo Aplicaciones.

    Se abrirá la página Aplicaciones

  2. En la tabla Aplicaciones implementadas, haga clic en Volver a implementar en la fila de la aplicación que está volviendo a implementar.

    Se abrirá la página Volver a implementar aplicaciones o Módulos.

  3. Especifique la ubicación del archivo de almacenamiento donde se encuentra la aplicación que se va a implementar.
    • Si el archivo de almacenamiento es un archivo que reside en la máquina cliente o al que se puede acceder desde la máquina cliente, seleccione la opción Archivo empaquetado que se cargará en el servidor.

      La máquina cliente es el host en el que está viendo la Consola de administración por medio de un navegador.

      Haga clic en Examinar para buscar el archivo o bien escriba la ruta completa hasta el archivo.

    • Si el archivo de almacenamiento es un archivo que reside en el equipo servidor o es una aplicación descomprimida desde un directorio desplegado, seleccione la opción Archivo empaquetado local o Directorio accesible desde Servidor empresarial.

      El equipo servidor es el host que se está ejecutando en el servidor de administración del dominio (DAS).

      Haz clic en Examinar archivos para buscar un archivo, o en Examinar carpetas para buscar un directorio. Como opción alternativa, escriba el nombre de la ruta completa del archivo o directorio. Implementar desde un directorio desplegado es para desarrolladores avanzados y no es recomendable para entornos de producción.

  4. Si necesita que se conserven las sesiones activas de la aplicación y que se restauren después, cuando se haya completado la reimplementación, seleccione la casilla de verificación Conservar sesiones.

    Si en alguna sesión activa se produce algún fallo en la preservación o restauración de la aplicación, no habrá disponible ninguna sesión cuando se haya completado la nueva implementación. Sin embargo, la reimplementación continuará y se registrará una advertencia.

  5. Seleccione la casilla de verificación Precompilar JSP para precompilar los archivos de JavaServer Pages (JSP).

    Si se deshabilita esta opción, los archivos JSP se compilarán en el tiempo de ejecución cuando se acceda a los mismos por primera vez. Esta opción se encuentra deshabilitada de forma predeterminada.

  6. Seleccione la casilla de verificación Ejecutar verificador para ejecutar el verificador antes de la reimplementación.

    Si esta opción está seleccionada y se produce un error en la verificación, la reimplementación no se llevará a cabo. El verificador examinará la estructura y contenido del archivo. La verificación de aplicaciones de gran tamaño tiende a llevar mucho tiempo. Esta opción se encuentra deshabilitada de forma predeterminada.

  7. Haga clic en Aceptar.
