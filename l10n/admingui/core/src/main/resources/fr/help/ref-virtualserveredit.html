Modification du serveur virtuel

ID

Identificateur de serveur virtuel visible en interne. Elle n'est pas visible des clients HTTP. Les noms d'hôtes visibles des clients HTTP doivent être spécifiés dans le champ Hôtes.

Hôtes

Nom(s) d'hôte de la machine sur laquelle le serveur est exécuté. Utilisez des noms d'hôte réels ou virtuels enregistrés dans le serveur DNS pour votre réseau (et, sous UNIX, dans votre fichier /etc/hosts).

État

Sélectionnez l'état désiré pour le serveur. Par défaut, les attributs sont activés.

Modules d'écoute HTTP

Ce champ est automatiquement renseigné lorsqu'un listener HTTP est créé et associé à ce serveur.

Ce champ exige que vous spécifiez un module HTTP existant. Cependant, vous ne devez pas spécifier le listener qui est utilisé par un autre serveur virtuel. Sinon, une erreur apparaît dans le journal du serveur lorsque vous redémarrez le serveur. Étant donné qu'un listener doit être associé à un serveur virtuel existant lors de sa création, tous les listeners existants sont utilisés par un autre serveur virtuel.

Module Web par défaut

Sélectionnez un module Web dans la liste déroulante. Choisissez le module Web déployé (le cas échéant) qui doit répondre à toutes les requêtes ne pouvant pas être mappées avec d'autres modules Web déployés sur le serveur virtuel. Si aucun module Web par défaut n'est spécifié, le module Web qui possède une racine contexte vide est utilisé. S'il n'existe aucun module Web avec une racine contexte vide, un module Web par défaut du système est créé et utilisé.

Fichier journal

Pour envoyer les messages de journalisation au journal de serveur par défaut, ne renseignez pas le champ Fichier journal. Sinon, saisissez le nom de chemin du fichier où les messages de journalisation provenant de ce serveur virtuel doivent apparaître.

Docroot

Chemin d'accès absolu au répertoire de documents racine pour le serveur. La valeur par défaut est rép-domaine/docroot.

SSO

Si cette option est activée, les utilisateurs doivent s'authentifier séparément auprès de chaque application sur le serveur virtuel. La valeur par défaut est Désactivé.

Journalisation des accès

Précisez si la journalisation des accès est héritée du service HTTP ou non.

Répertoire

Chemin d'accès absolu aux journaux d'accès du serveur. Le chemin par défaut est rép-domaine/logs/access.

Taille du tampon

Une valeur inférieure ou égale à zéro désactive la mise en tampon. Si ce champ n'est pas renseigné, la taille du tampon du journal d'accès est héritée du service HTTP.

Intervalle de suppression

Nombre de secondes devant s'écouler avant que le journal d'accès ne soit mis à jour. La valeur 0 désactive la mise en mémoire tampon.

Autres propriétés

Propriétés supplémentaires pour le serveur virtuel. Les propriétés suivantes sont disponibles :

sso-max-inactive-seconds

Indique le nombre de secondes au-delà duquel l'enregistrement de connexion unique d'un utilisateur peut être supprimé si aucune activité cliente n'est reçue. Étant donné que la connexion unique s'applique à plusieurs applications du même serveur virtuel, le fait d'accéder à une application maintient l'enregistrement de connexion unique actif.

La valeur par défaut est de 300 secondes (5 minutes). Des valeurs supérieures offrent une persistance de connexion unique supérieure au détriment d'une plus grande consommation de mémoire sur le serveur.

sso-reap-interval-seconds

Indique le nombre de secondes entre les purges des enregistrements de connexion unique expirés.

La valeur par défaut est 60.

allowLinking

Si la valeur est true (vrai), les ressources représentant des liens symboliques seront utilisées pour toutes les applications Web déployées sur ce serveur virtuel. Les applications Web individuelles peuvent remplacer ce paramètre en utilisant la propriété sun-web-app allowLinking du fichier sun-web.xml :

<sun-web-app>
   <property name="allowLinking" 
      value="{true|false}"/>
</sun-web-app>

La valeur par défaut est false.
