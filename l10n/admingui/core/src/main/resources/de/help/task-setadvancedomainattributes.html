So legen Sie die erweiterten Domänenattribute fest

  1. Wählen Sie in der Navigationsstruktur den Anwendungsserverknoten.
  2. Klicken Sie auf der Seite "Allgemeine Informationen" auf die Registerkarte "Erweitert".
  3. Klicken Sie auf der Seite "Anwendungskonfiguration" auf die Registerkarte "Domänenattribute".

    Die Seite "Domänenattribute" wird geöffnet.

  4. Geben Sie im Feld "Anwendungs-Root" den vollständigen Pfad zu dem Verzeichnis ein, in dem die Anwendungen bereitgestellt werden.
  5. Geben Sie im Feld "Protokoll-Root" den vollständigen Pfad des Verzeichnisses ein, in dem die Server-Protokolldateien aufbewahrt werden.
  6. Lassen Sie das Feld "Ländereinstellung" leer, um die Standardländereinstellung des Hosts zu verwenden.

    Eine Ländereinstellung ist ein Bezeichner, der eine bestimmte Kombination aus Sprache und Region angibt. Beispielsweise lautet die Ländereinstellung für Englisch (USA) en_US, die Ländereinstellung für Japanisch lautet ja_JP. Um eine nicht englische Ländereinstellung zu verwenden, muss Enterprise Server lokalisiert, d. h. in eine andere Sprache übersetzt werden.

  7. Klicken Sie auf "Speichern".
  8. Starten Sie den Server neu.
