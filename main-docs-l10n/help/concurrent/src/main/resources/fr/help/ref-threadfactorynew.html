Nouvelle fabrique de threads gérés

Utilisez la page Nouvelle fabrique de threads gérés pour créer une ressource de fabrique de threads gérés.

Elle contient les options ci-après.

Nom JNDI

Nom JNDI unique identifiant la ressource. Par convention, le nom commence par un indicateur de type de ressource et une barre oblique, comme concurrent/, jdbc/ ou jms/.

Informations de contexte

Contextes à propager vers les threads : Chargeur de classe, JNDI, Sécurité ou Zone de travail. La propagation de contexte peut être activée ou désactivée. Si elle est activée, les contextes sélectionnés sont propagés. Si elle est désactivée, aucun des contextes n'est propagé, même s'ils sont sélectionnés. La valeur par défaut est Activé, avec les quatre contextes sélectionnés.

Statut

Statut de la ressource. La ressource peut être activée ou désactivée. La valeur par défaut est Activé.

Priorité des threads

Priorité à affecter aux threads créés. La valeur par défaut est 5.

Description

Description de la ressource.

Autres propriétés

Propriétés supplémentaires de la ressource. GlassFish Server ne définit aucune autre propriété pour ce type de ressource. De plus, ce type de ressource n'utilise actuellement aucune propriété supplémentaire.

Cibles

(Cette option apparaît si des clusters ou des instances autonomes ont été créés dans le domaine.)

Clusters et instances autonomes de la ressource. Déplacez les cibles souhaitées dans la colonne Cibles sélectionnées à l'aide des boutons Ajouter et Tout ajouter. Déplacez toutes les cibles inutiles dans la colonne Cibles disponibles à l'aide des boutons Enlever et Tout enlever.
