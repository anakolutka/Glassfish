ORB

Utilizzare la pagina ORB per configurare il broker ORB (Object Request Broker).

Il broker ORB fornisce l'infrastruttura necessaria per identificare e individuare gli oggetti, gestire le connessioni, distribuire i dati e richiedere la comunicazione. I client remoti degli Enterprise Bean (moduli EJB) comunicano con GlassFish Server mediante il broker ORB.

La pagina ORB contiene le opzioni riportate di seguito.

Carica impostazioni predefinite

Pulsante per ripristinare i valori predefiniti delle impostazioni che dispongono di valori predefiniti. Le impostazioni che non dispongono di valori predefiniti non vengono modificate.

Nome configurazione

Il nome della configurazione a cui si applicano le impostazioni presenti in questa pagina. Questo campo è di sola lettura.

ID pool di thread

Il pool di thread usato dal broker ORB. Il broker ORB usa i pool di thread per rispondere alle richieste dei client remoti degli Enterprise Bean e di altri client che comunicano mediante il protocollo RMI-IIOP. Il pool di thread predefinito per il broker ORB è thread-pool-1, che utilizza le impostazioni predefinite appropriate per le richieste RMI/IIOP.

Dimensione massima frammento messaggio

La dimensione di frammento massima per i messaggi IIOP. I messaggi con dimensione superiore alla dimensione specificata vengono frammentati. Il valore predefinito è 1024 byte.

Connessioni totali

Il numero massimo di connessioni in entrata per tutti i listener IIOP. Il valore predefinito è 1024.

Autenticazione client IIOP

Se la casella di controllo Abilitato è selezionata, il server rifiuta le richieste non autenticate e inserisce un bit autenticazione richiesta nei riferimenti IOR (Interoperable Object References) inviati ai client. L'autenticazione client IIOP può essere abilitata o disabilitata. Questa opzione è disabilitata per impostazione predefinita.

Proprietà aggiuntive

Proprietà aggiuntive per il broker ORB. GlassFish Server non definisce alcuna proprietà aggiuntiva per il broker ORB.
