Novo Pool de Conexões do JDBC (etapa 2 de 2)

Use a página Novo Pool de Conexões do JDBC (Etapa 2 de 2) para criar um novo pool de conexões.

A página Novo Pool de Conexões do JDBC (Etapa 2 de 2) contém as opções a seguir.

Nome de Classe da Fonte de Dados

O nome da classe específica do fornecedor que implementa a API DataSource, ConnectionPoolDataSource ou XADataSource. Se você tiver ativado a opção Omissão e um dos tipos de recursos da fonte de dados, esse campo será preenchido automaticamente com o nome da classe associado ao tipo de recurso e com o fornecedor do banco de dados de sua escolha. Se você selecionou o tipo de recursojava.sql.Driver, esse campo será desativado.

Nome de Classe do Driver

O nome da classe do driver JDBC fornecido pelo fornecedor. Se você tiver selecionado a opção Omissão e o tipo de recurso java.sql.Driver, esse campo será automaticamente preenchido com o nome da classe associado ao driver e ao fornecedor de banco de dados de sua escolha. Se você selecionou um dos tipos de recursos de fonte de dados, esse campo é desativado.

Ping

Se essa opção for selecionada, será efetuado o ping automático no pool durante a criação ou reconfiguração, para identificar e avisar sobre valores de atributos com erros. Por default, essa opção está desativada.

Descrição

Uma breve descrição do pool de conexões.

Tamanho Inicial e Mínimo do Pool

O número mínimo de conexões no pool. Esse valor também determina o número de conexões colocadas no pool quando o pool é criado ou quando o GlassFish Server é iniciado. O valor default é 8.

Tamanho Máximo do Pool

O número máximo de conexões no pool. O valor default é 32.

Quantidade de Redimensionamento do Pool

Quando o pool aumenta e diminui em relação aos tamanhos máximo e mínimo, respectivamente, o pool é redimensionado em batches. Esse valor determina o número de conexões do batch. A definição de um valor muito alto torna demorado o processo de criação e reciclagem de conexões; um valor muito baixo compromete a eficiência. O valor default é 2.

Time-out para Ociosidade

A quantidade máxima de tempo, em segundos, que uma conexão pode permanecer inativa no pool. Após esse período, a conexão será removida do pool. O valor default é 300.

Tempo Máximo de Espera

O tempo máximo, em milissegundos, que a aplicação solicitante da conexão aguardará até obter um time-out. O valor default é 60.000.

Conexões Não Transacionais

Se essa opção for ativada, as conexões não transacionais serão retornadas.

Isolamento de Transação

Se um nível de isolamento de transação for especificado, as conexões desse pool funcionarão no nível especificado. Caso contrário, as conexões funcionarão com os níveis de isolamento default, fornecidos pelo driver JDBC. Por default, essa opção não é especificada.

Nível de Isolamento

Se a caixa de seleção Garantido for marcada, todas as conexões obtidas no pool terão o mesmo nível de isolamento. Por exemplo, se o nível de isolamento da conexão for alterado via programação (com con.setTransactionIsolation), durante a última utilização, esse mecanismo alterará o status para o nível de isolamento especificado. Por default, a opção está ativada.

Essa definição só será aplicável se um nível de isolamento de transação for especificado.

Propriedades Adicionais

Propriedades adicionais do pool de conexões JDBC. Essas propriedades podem incluir nome de banco de dados (URL), nome de usuário e senha. A tabela normalmente contém propriedades usadas pelo fornecedor do banco de dados.

Para obter a descrição de algumas das propriedades disponíveis, consulte Propriedades Específicas para os Pools de Conexões JDBC.

Se você não especificar um valor para uma propriedade, ela será removida da tabela quando você finalizar a criação do pool de conexões. Caso não deseje definir agora um valor para uma propriedade, mas deseja que a propriedade permaneça na tabela, para que o valor possa ser definido em outro momento, digite um conjunto de parênteses vazios no campo Valor:

()
