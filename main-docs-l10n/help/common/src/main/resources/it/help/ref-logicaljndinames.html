Nomi JNDI logici

Quando si crea una risorsa, si specifica un nome JNDI univoco per la risorsa. Le applicazioni utilizzano questo nome per accedere alla risorsa.

Lo standard Java EE specifica che determinate risorse predefinite devono essere rese disponibili per le applicazioni e definisce nomi JNDI specifici per queste risorse predefinite. GlassFish Server rende disponibili questi nomi attraverso l'utilizzo di nomi JNDI logici, che mappano i nomi JNDI di standard Java EE a risorse specifiche di GlassFish Server, come riportato di seguito.

java:comp/DefaultContextService

Questo nome di standard Java EE viene mappato alla risorsa concurrent/__defaultContextService.

java:comp/DefaultManagedThreadFactory

Questo nome di standard Java EE viene mappato alla risorsa concurrent/__defaultManagedThreadFactory.

java:comp/DefaultManagedExecutorService

Questo nome di standard Java EE viene mappato alla risorsa concurrent/__defaultManagedExecutorService.

java:comp/DefaultManagedScheduledExecutorService

Questo nome di standard Java EE viene mappato alla risorsa concurrent/__defaultManagedScheduledExecutorService.

java:comp/DefaultDataSource

Questo nome di standard Java EE viene mappato alla risorsa jdbc/__default.

java:comp/DefaultJMSConnectionFactory

Questo nome di standard Java EE viene mappato alla risorsa jms/__defaultConnectionFactory.
