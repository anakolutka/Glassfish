Cómo configurar las opciones de seguridad

La Consola de administración permite establecer varios valores de seguridad en todo el sistema.

  1. En el árbol de navegación, expanda el nodo Configuración.
  2. En el nodo Configuración, seleccione el nodo Seguridad.
  3. Marque la casilla de verificación Administrador de seguridad activado para activar el administrador de seguridad.

    Si se activa esta opción, se agregará una opción de JVM, -Djava.security.manager , a los valores de configuración de JVM de Servidor empresarial. Consulte Para configurar las opciones de JVM. Debe reiniciar el servidor para habilitar este cambio.

    Asegúrese de que ha otorgado los permisos correctos para todas las aplicaciones. Puede deshabilitar el administrador de seguridad para mejorar el rendimiento. Esta opción se encuentra deshabilitada de forma predeterminada.

  4. Marque la casilla de verificación Registro de auditoría activado para habilitar el registro de auditoría.

    Si esta opción está activada, el servidor cargará y ejecutará todos los módulos de auditoría seleccionados de la lista Módulos de auditoría. Si esta opción está desactivada, el servidor no accederá a los módulos de auditoría. Esta opción se encuentra deshabilitada de forma predeterminada.

  5. En la lista desplegable Dominio predeterminado, seleccione un dominio activo que el servidor utilizará para la autenticación.

    Las aplicaciones utilizarán este dominio cuando especifiquen un dominio diferente en su descriptor de implementación. Todos los dominios configurados aparecen en la lista. El valor predeterminado es file.

  6. (Opcional) Especifique la información de Principal predeterminado.
    1. En el campo Principal predeterminado, introduzca el nombre de usuario predeterminado.

      El servidor utilizará este nombre de usuario si no se proporciona otro principal.

    2. En el campo Contraseña de principal predeterminado, introduzca la contraseña del valor principal predeterminada especificada en el campo Principal predeterminado.
  7. En la lista Módulos de auditoría, seleccione los módulos que utilizará el subsistema de auditorías.

    Los módulos que aparecen en la lista tienen que estar ya configurados. Si se activa Registro de auditoría, este parámetro tiene que mostrar los módulos de auditoría. Por defecto el servidor utiliza un módulo de auditoría llamado default. Para obtener información sobre la creación de nuevos módulos de auditoría, consulte Cómo crear un módulo de auditoría.

  8. Seleccione Activar para aplicar la asignación predeterminada de principal a función para las aplicaciones que no tienen asignada una asignación específica de aplicación.
  9. La asignación principal de clase le permite personalizar la clase java.security.Clase de implementación Principal utilizada en la asignación predeterminada de principal a función.
  10. En la página Seguridad, modifique los valores según corresponda.
  11. Agregue las propiedades adicionales que desee para pasar a Java Virtual Machine (JVM).

    Las propiedades válidas dependen del tipo de dominio seleccionado en el campo Dominio predeterminado.

  12. Seleccione "Guardar".
