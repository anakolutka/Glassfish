라이프사이클 모듈을 편집하는 방법

  1. 탐색 트리에서 라이프사이클 모듈 노드를 선택합니다.

    라이프사이클 모듈 페이지가 열립니다.

  2. 라이프사이클 모듈 페이지에서 편집 중인 라이프사이클 모듈의 이름을 누릅니다.

    라이프사이클 모듈 편집 페이지가 열립니다.

  3. 클래스 이름 필드에 라이프사이클 모듈의 클래스 파일에 대한 정규화된 이름을 입력합니다.

    클래스는 com.sun.appserv.server.LifecycleListener 인터페이스를 구현해야 합니다.

  4. 클래스 경로 필드에 라이프사이클 모듈의 클래스 경로를 입력합니다.

    라이프사이클 모듈이 있는 위치를 지정하는 클래스 경로입니다. 기본값은 domain-dir/applications입니다. 모듈이 이미 이 디렉토리(즉, 서버 클래스 경로)에 있으면 이 필드를 비워둘 수 있습니다.

  5. 로드 순서 필드에서 시작할 때 이 라이프사이클 모듈을 로드할 순서를 지정하는 값을 입력합니다.

    정수 로드 순서 값이 작은 모듈이 먼저 로드됩니다. 값의 범위는 101부터 운영 체제의 MAXINT까지일 수 있습니다. 1부터 100까지의 값은 예약되어 있습니다.

  6. 라이프사이클 모듈을 활성화하려면 상태 사용 가능 확인란을 선택합니다.

    이 옵션은 기본적으로 활성화됩니다.

  7. 라이프사이클 모듈이 로드에 실패할 경우 서버를 종료하도록 지정하려면 로드 실패 시 인스턴스 시작 방지 확인란을 선택합니다.

    이 옵션은 기본적으로 비활성화됩니다.

  8. 가상 서버 드롭다운 목록에서 라이프사이클 모듈과 연결할 가상 서버를 선택합니다.

    기본적으로 가상 서버는 선택되어 있지 않습니다.

  9. 설명 필드에 라이프사이클 모듈의 설명을 입력합니다.
  10. (옵션) 추가 등록 정보 섹션에서 추가 등록 정보를 지정합니다.

    Enterprise Server에서는 라이프사이클 모듈에 추가 등록 정보를 정의하지 않습니다.

    • 등록 정보를 추가하려면 등록 정보 추가 버튼을 누릅니다. 빈 행이 표시되면 이름 필드에 등록 정보 이름을 입력하고 값 필드에 등록 정보 값을 입력합니다.
    • 등록 정보를 수정하려면 등록 정보의 값 필드를 편집합니다.
    • 등록 정보를 삭제하려면 삭제할 등록 정보의 이름 필드 왼쪽에 있는 확인란을 선택한 다음 등록 정보 삭제 버튼을 누릅니다.
  11. 저장을 누릅니다.

    참고 - 기본 설정으로 돌아가려면 기본값 로드 버튼을 누른 다음 저장을 누릅니다.
