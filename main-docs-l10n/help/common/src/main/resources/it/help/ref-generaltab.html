Informazioni generali

GlassFish Server crea un'istanza di Application Server, denominata server, al momento dell'installazione. Se necessario, è possibile eliminare l'istanza server esistente e crearne una nuova con un altro nome.

Ogni istanza di GlassFish Server dispone di configurazione Java, risorse Java, aree di distribuzione delle applicazioni e impostazioni di configurazione del server proprie. Le modifiche apportate a un'istanza di Application Server non hanno effetto sulle altre istanze di Application Server. All'interno di un dominio amministrativo può esistere una sola istanza di Application Server.

In genere una sola istanza di Application Server è sufficiente per soddisfare le esigenze di numerosi utenti, ma a seconda dell'ambiente in uso potrebbe essere necessario crearne altre. In un ambiente di sviluppo, ad esempio, è possibile usare istanze di Application Server diverse per provare varie configurazioni di GlassFish Server o per confrontare e collaudare distribuzioni di applicazioni differenti. Poiché aggiungerle o eliminarle è semplice, le istanze di Application Server possono essere utilizzate per creare aree "sandbox" temporanee da utilizzare per gli esperimenti durante la fase di sviluppo.

Inoltre, per ogni istanza di Application Server è possibile creare server virtuali. All'interno di una singola istanza di Application Server è possibile fornire a società e singole persone nomi di dominio, indirizzi IP e alcune funzionalità di amministrazione. È come se gli utenti disponessero di un Web server personale, ma senza tuttavia doversi occupare dell'hardware e della manutenzione di base. Questi server virtuali non propagano istanze di Application Server. Per ulteriori informazioni sui server virtuali, vedere Impostazioni generali JVM.

Nelle distribuzioni operative è possibile utilizzare i server virtuali invece di più istanze di Application Server in molti casi. Se tuttavia i server virtuali non soddisfano le esigenze, è possibile utilizzare più istanze di Application Server.

Un'istanza di GlassFish Server non viene avviata in modo automatico. Dopo essere stata avviata, un'istanza continua a essere eseguita finché non la si arresta. Quando si arresta un'istanza di Application Server, l'istanza smette di accettare nuove connessioni, quindi rimane in attesa del completamente di tutte le connessioni in sospeso. Se si verifica il crash del computer o il computer viene messo fuori linea, il server viene chiuso e le richieste elaborate al momento potrebbero andare perdute.

Utilizzare la pagina Informazioni generali per verificare le impostazioni di GlassFish Server e visualizzare i dati della Java Virtual Machine.

La pagina Informazioni generali contiene le informazioni riportate di seguito.

Pulsante Arresta

Fare clic sul pulsante Arresta per arrestare GlassFish Server.

Pulsante Riavvia

Fare clic sul pulsante Riavvia per riavviare GlassFish Server.

Pulsante Visualizza file di log

Fare clic sul pulsante Visualizza file di log per visualizzare i file di log per un'istanza o un cluster di GlassFish Server.

Pulsante Ruota log

Fare clic sul pulsante Ruota log per ruotare il file di log per il server di amministrazione (denominato server).

Pulsante Recupera transazione

Fare clic sul pulsante Recupera transazione per recuperare le transazioni per il server di amministrazione (denominato server) nella pagina Recupera transazioni.

Pulsante Amministrazione sicura

Fare clic sul pulsante Amministrazione sicura per abilitare o disabilitare l'amministrazione sicura nella pagina Amministrazione sicura.

Nome

Il nome del server corrente.

Stato

Lo stato corrente dell'istanza del server. Il server può avere lo stato Arrestato, Avviato o In esecuzione.

Tempo di attività

Il numero di ore e minuti durante i quali l'istanza server è stata in esecuzione in modo continuativo. Questo campo è di sola lettura.

Load balancer HTTP

Il nome del load balancer, se è abilitato per questa istanza server.

JVM

Se si fa clic su Report JVM, viene visualizzata una finestra distinta in cui sono disponibili i report relativi alla Java Virtual Machine, tra cui un report di riepilogo, un report sulla gestione della memoria e sulla garbage collection, un report sul caricamento delle classi e un dump del thread corrente.

Directory di configurazione

La directory del computer host che contiene i file di configurazione, ad esempio domain.xml.

Versione installata

L'identificativo di versione del prodotto.

Amministrazione sicura

Lo stato corrente dell'amministrazione sicura. I valori possibili sono Abilitato e Non abilitato.

Debug

Indica se la funzione di debug è abilitata. Se il debug è abilitato, viene visualizzato anche il numero di porta.

Porta/e HTTP

Le porte attualmente configurate per le richieste e le risposte HTTP.

Porta/e IIOP

Le porte attualmente configurate per le richieste e le risposte IIOP.
