Servizio HTTP

Utilizzare la pagina Servizio HTTP per specificare i criteri generali del log degli accessi per le applicazioni Web.

Il servizio HTTP fornisce le funzioni per la distribuzione delle applicazioni Web e per consentire l'accesso alle applicazioni Web distribuite da parte di client HTTP, server virtuali, pool di thread e funzionalità di configurazione di rete.

GlassFish Server usa il log degli accessi del servizio HTTP per abilitare e configurare la rotazione dei log degli accessi per i server virtuali. Questi log si trovano nella directory domain-dir/logs/access e sono denominati come riportato di seguito: virtual_server_name_access_log.yyyyMMdd-HHhmmmsss.txt

La pagina Servizio HTTP contiene le opzioni riportate di seguito.

Carica impostazioni predefinite

Pulsante per ripristinare i valori predefiniti delle impostazioni che dispongono di valori predefiniti. Le impostazioni che non dispongono di valori predefiniti non vengono modificate.

Nome configurazione

Il nome della configurazione a cui si applicano le impostazioni presenti in questa pagina. Questo campo è di sola lettura.

SSO

Indica se Single Sign-On è abilitato per impostazione predefinita per tutte le applicazioni Web in tutti i server virtuali di questo server configurate per lo stesso realm. Se questa opzione non è abilitata, Single Sign-On è disabilitato per impostazione predefinita per tutti i server virtuali e gli utenti devono eseguire l'autenticazione per ogni applicazione in ciascun server virtuale. È possibile ignorare questa impostazione per un singolo server virtuale. Single Sign-On può essere abilitato o disabilitato. Questa opzione è disabilitata per impostazione predefinita.

Log degli accessi

Se questa opzione è selezionata, il log degli accessi è abilitato per tutti gli elementi secondari dei server virtuali in cui non è specificata questa proprietà. Se non è selezionata, questa opzione disabilita il log degli accessi per tutti gli elementi secondari dei server virtuali in cui non è specificata questa proprietà. Questa opzione è disabilitata per impostazione predefinita.

Rotazione

Se questa opzione è selezionata, la rotazione dei file di log è abilitata. Questa opzione è abilitata per impostazione predefinita.

Criteri di rotazione

L'unico criterio di rotazione disponibile è time.

Intervallo di rotazione

Il numero di minuti che intercorrono tra le rotazioni del log degli accessi. Questo campo è valido solo se il criterio di rotazione è time. Il valore predefinito è 1.440 minuti (24 ore).

Suffisso rotazione

Il valore di stringa da aggiungere alla fine del nome del file di log dopo la rotazione. Il valore predefinito è yyyy-MM-dd.

Assicurarsi che il suffisso di rotazione contenga valori sufficienti per la creazione di un suffisso univoco per ogni rotazione di log. Ad esempio, se il log di accesso viene ruotato più di una volta al giorno, includere l'ora del giorno nel suffisso di rotazione.

Il suffisso univoco consente di assicurare che venga eseguita la rotazione del file di log di accesso a ogni scadenza dell'intervallo di rotazione. Se il suffisso di rotazione del nuovo file di log degli accessi è identico al suffisso di rotazione del file di log degli accessi esistente, la rotazione del file non viene eseguita.

Numero massimo di file

Il numero massimo dei file di log degli accessi ruotati da conservare. Un valore negativo indica che il numero dei file è illimitato. Il valore predefinito è -1.

Dimensione buffer

La dimensione, in byte, del buffer del log degli accessi oppure un valore minore o uguale a 0 per i log degli accessi non inseriti in un buffer. Il valore predefinito è 32768.

Intervallo di scrittura

Il numero di secondi prima della scrittura del log su disco. Il log degli accessi viene scritto quando il buffer è pieno o allo scadere dell'intervallo di tempo previsto. Se il valore è minore o uguale a 0, il buffer viene sempre scritto, anche se non è pieno. Ciò significa che ogni volta che si accede al server, i messaggi del log vengono memorizzati direttamente nel file. Il valore predefinito è 300.

Formato

Il valore di stringa che specifica le informazioni da acquisire nel log degli accessi e l'ordine in cui vengono acquisite. Di seguito è riportato il valore predefinito.

%client.name% %auth-user-name% %datetime% %request% %status% %response.length%

Nella stringa del formato è possibile includere i valori riportati di seguito.

%attribute.name%

Registra il valore (mediante la chiamata del metodo toString rispettivo) dell'attributo della richiesta con il valore name specificato.

%auth-user-name%

Nome dell'utente autorizzato

%client.name%

Nome host client

%client.dns%

DNS client

%cookie.name%

Il valore del cookie di richiesta con il valore name specificato

%cookie.value%

Il valore del primo cookie trovato nella richiesta

%datetime%

Data di sistema

%request%

Riga di richiesta HTTP completa

%status%

Stato

%response.length%

Lunghezza contenuto risposta

%header.referer%

Intestazione Referer

%header.user-agent%

Intestazione User-Agent

%http-method%

Metodo HTTP

%http-uri%

URI HTTP

%query-str%

Stringa query HTTP

%http-version%

Versione protocollo HTTP

%header.accept%

Intestazione Accept

%header.date%

Intestazione Date

%header.if-mod-since%

Intestazione If-Modified-Since

%header.auth%

Intestazione Authorization

%header.any%

Qualsiasi valore di intestazione HTTP valido definito nella RFC 2616 (anche any è un valore di intestazione valido; qui è specificato come variabile)

%session.name%

Il valore dell'attributo di sessione con nome name o NULL-SESSION-ATTRIBUTE-name se l'attributo denominato non esiste nella sessione o NULL-SESSION se la sessione non esiste

%time-taken%

Il tempo necessario per eseguire ogni richiesta in millisecondi

%vs.id%

ID server virtuale
