組態批次執行階段

  1. 在瀏覽樹狀結構中，選取您要檢視批次工作執行的 GlassFish Server 叢集或伺服器執行處理。

    此時將開啟「一般資訊」頁面。

  2. 選取「批次」頁籤。

    就會開啟「批次工作執行」頁面。

  3. 按一下「組態」頁籤。

    就會開啟「批次執行階段組態」頁面。執行階段會使用受管理執行程式服務與資料來源來執行批次工作。

  4. 從「執行程式服務查詢名稱」下拉式清單中，選取用來提供繫線給批次工作的受管理執行程式服務。

    預設受管理執行程式服務為 concurrent/__defaultManagedExecutorService。受管理執行程式服務可以在批次工作已送出至批次執行階段之後變更，且不會影響工作的執行。

  5. 從「資料來源查詢名稱」下拉式清單中，選取用來儲存工作資訊的資料來源。

    預設的資料來源為 jdbc/__TimerPool。當第一個批次工作已送出至執行的批次執行階段之後，請勿變更資料來源。如果必須變更資料來源，請停止並重新啟動網域，然後於啟動或重新啟動任何工作之前先進行變更。但是，一旦變更了資料來源，就無法繼續存取儲存於先前資料來源中的資訊。

  6. 按一下「儲存」。
