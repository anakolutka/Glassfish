To Start, Stop, and Delete Clustered Instances and Set Load Balancing Weights

  1. In the tree component, expand the Clusters node.

  2. Expand the node for the cluster that contains the server instance.

  3. Click the Instances tab to display the Clustered Server Instances page.

    On this page you can:

    • Select the checkbox for an instance and click Delete, Start, or Stop to perform the selected action on all the specified server instances.

    • Select the checkbox for an instance and choose a load balancer action from the pull-down menu.

      • Start Load Balancing enables an instance for load balancing.

      • Quiesce disables an instance for load balancing gracefully, by allowing time to complete requests that are in process. On the Load Balancer Monitoring pop-up, click Begin Quiesce Now to start quiescing the server instance or instances. Stop Instances Now stops the instance or instances without quiescing.

    • Click the name of the instance to bring up the General Information page.

    • Change the weight of the instance for the load balancer, if you are using weighted round-robin load balancing. Set the load balancer for the cluster to weighted round robin on the Cluster General Information page. The default weight is 100.

      For example, if you have two instances, and want one instance to process one request out of every four, and the other instance to process three requests out of four, set the weights to 1 and 3, or 25 and 75, if you are more comfortable working in percentages.
