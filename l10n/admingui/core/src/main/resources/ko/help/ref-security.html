보안 관리자

이 옵션을 선택하면 도메인의 보안 관리자가 활성화됩니다.

사용하도록 설정된 경우 JVM 옵션 -Djava.security.manager가 Enterprise Server의 JVM 설정에 추가됩니다. JVM 옵션을 구성하는 방법을 참조하십시오. 이 변경 사항을 활성화하려면 서버를 다시 시작해야 합니다.

모든 응용 프로그램에 올바른 권한을 부여했는지 확인하십시오. 성능 향상을 위해 보안 관리자를 비활성화할 수 있습니다.

이 옵션은 기본적으로 비활성화됩니다.

감사 로깅

이 기능이 활성화되면 서버가 감사 모듈 설정에 지정된 모든 감사 모듈을 로드하고 실행합니다. 이 기능이 비활성화되면 서버가 감사 모듈에 액세스하지 않습니다. 이 옵션은 기본적으로 비활성화됩니다.

기본 영역

서버가 인증에 사용하는 활성(기본) 영역입니다. 배포 설명자가 다른 영역을 지정하지 않는 한 응용 프로그램은 이 영역을 사용합니다. 구성된 모든 영역이 목록에 표시됩니다. 기본 값은 파일입니다.

기본 사용자

기본 사용자 이름을 지정합니다. 다른 기본값을 제공하지 않을 경우 서버는 이 사용자 이름을 사용합니다. 이 필드에 값을 입력하는 경우에는 기본 사용자 비밀번호 필드에 해당 비밀번호를 입력합니다.

일반 서버 작업의 경우 이 속성은 필수가 아닙니다.

기본 사용자 비밀번호

기본 사용자 필드에 지정한 기본 사용자의 비밀번호입니다. 일반 서버 작업의 경우 이 속성은 필수가 아닙니다.

감사 모듈

쉼표로 구분된 감사 모듈 공급자 클래스 목록입니다. 여기에 나열된 모듈은 이미 구성되어 있어야 합니다. 감사 로깅이 활성화된 경우에는 이 필드에 감사 모듈이 나열되어야 합니다. 기본적으로 서버는 이름이 default인 감사 모듈을 사용합니다. 새 감사 모듈 만들기에 대한 자세한 내용은 감사 모듈을 만드는 방법을 참조하십시오.

역할 매핑에 대한 기본 사용자 사용 가능

이 옵션을 선택하면 역할 매핑에 대한 기본 사용자가 응용 프로그램 관련 매핑이 없는 응용 프로그램에 적용됩니다.

매핑된 기본 클래스

java.security의 사용자 정의 구현을 지정합니다.역할 매핑에 대한 기본 사용자에서 사용되는 Principal 클래스입니다.
