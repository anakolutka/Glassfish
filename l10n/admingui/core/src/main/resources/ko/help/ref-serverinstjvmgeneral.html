JVM 일반 설정

JVM(Java 가상 머신)은 Enterprise Server의 필수 소프트웨어인 Java 플랫폼, Standard Edition(Java SE 플랫폼)에 포함되어 있습니다. JVM 설정이 올바르지 않으면 서버가 실행되지 않으므로 이 설정을 변경할 때 주의해야 합니다.

Java 홈

Enterprise Server가 실행하는 데 필요한 Java SE 소프트웨어의 설치 디렉토리 이름입니다. 지정하는 Java SE 소프트웨어 버전이 이 릴리스에서 지원되는지 확인하려면 Sun GlassFish Enterprise Server v3 Prelude Release Notes를 참조하십시오. 존재하지 않는 디렉토리 이름을 입력하거나 지원되지 않는 Java SE 소프트웨어 버전의 디렉토리를 입력하면 Enterprise Server가 시작되지 않습니다.

Javac 옵션

Java 프로그래밍 언어 컴파일러에 대한 명령줄 옵션입니다. EJBTM(Enterprise JavaBeansTM) 구성 요소가 배포되면 Enterprise Server에서 컴파일러를 실행합니다.

디버그

JPDA(Java Platform Debugger Architecture)로 디버깅할 수 있도록 설정합니다.

JPDA는 응용 프로그램 개발자가 사용합니다. 자세한 내용은 Sun GlassFish Enterprise Server v3 Prelude Developer’s Guide를 참조하십시오.

디버그 옵션

디버깅을 사용하도록 설정될 때 JVM에 전달되는 JPDA 옵션을 지정합니다.

RMI 컴파일 옵션

rmic 컴파일러에 대한 명령줄 옵션입니다. EJB 구성 요소가 배포되면 Enterprise Server가 rmic 컴파일러를 실행합니다.

바이트 코드 선행 프로세서

바이트 코드 선행 처리를 위한 클래스의 이름입니다. 클래스 이름을 구분하려면 쉼표를 사용합니다. 각 클래스는 com.sun.appserv.BytecodePreprocessor 인터페이스를 구현해야 합니다. 지정한 순서대로 클래스가 호출됩니다.
