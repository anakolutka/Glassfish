Enterprise-Anwendungen/Module erneut bereitstellen

Über die Seite "Enterprise-Anwendungen/Module erneut bereitstellen" können Sie eine zuvor bereitgestellte Anwendung erneut bereitstellen. Die Seite enthält die folgenden Optionen:

Name

Schreibgeschütztes Feld mit dem Namen der Anwendung, die Sie erneut bereitstellen.

Typ

Schreibgeschütztes Feld mit dem Anwendungstyp.

Sitzungen speichern

Wenn Sie das Kontrollkästchen "Sitzungen speichern" aktivieren, werden die aktiven Sitzungen der erneut bereitzustellenden Anwendung gespeichert und nach Abschluss der erneuten Bereitstellung wiederhergestellt.

Kann eine aktive Sitzung der Anwendung nicht gespeichert oder wiederhergestellt werden, ist keine der Sitzungen verfügbar, wenn die erneute Bereitstellung abgeschlossen ist. Die erneute Bereitstellung wird jedoch fortgesetzt, und eine Warnmeldung wird aufgezeichnet.

Zum Speichern von aktiven Sitzungen serialisiert Enterprise Server die Sitzungen und speichert sie im Arbeitsspeicher. Zum Wiederherstellen der Sitzungen entserialisiert das Klassenladeprogramm der erneut bereitgestellten Anwendung alle Sitzungen, die zuvor gespeichert wurden.

Ort

Der Speicherort des neuen Archivs für die erneut bereitzustellende Anwendung.

Die folgenden Optionen geben an, von wo aus auf das Archiv zugegriffen werden kann und ob das Archiv eine Datei oder ein Verzeichnis ist.

Verpackte Datei zum Hochladen auf den Server

Das Archiv ist in einer Datei, die sich auf dem Clientcomputer befindet oder auf die von dort aus zugegriffen werden kann.

Der Clientcomputer ist der Host, auf dem Sie die Administration Console über einen Browser anzeigen.

Lokale verpackte Datei oder Verzeichnis, auf die/das vom Anwendungsserver aus zugegriffen werden kann

Das Archiv ist eine Datei, die sich auf dem Servercomputer befindet, oder eine nicht verpackte Anwendung in einem aufgelösten Verzeichnis.

Der Servercomputer ist der Host, auf dem der Domain Administration Server (DAS) des Anwendungsservers ausgeführt wird.

Verwandte Aufgaben

  • So stellen Sie eine Webanwendung bereit

  • So bearbeiten Sie eine bereitgestellte Webanwendung

  • So zeigen Sie Beschreibungen für eine Webanwendung an

  • So starten Sie eine Webanwendung

  • So stellen Sie eine bereitgestellte Webanwendung erneut bereit
