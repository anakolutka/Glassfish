Configurer le service HTTP

  1. Dans l'arborescence de navigation, développez le nœud Configuration.
  2. Sous le nœud Configuration, sélectionnez le nœud Service HTTP.

    La page Service HTTP s'ouvre.

  3. Cochez la case Journalisation de l'accès activé pour activer la journalisation des accès pour tous les sous-éléments des serveurs virtuels qui ne spécifient pas cette propriété.

    Si cette option est désactivée, la journalisation des accès est désactivée pour tous les sous-éléments des serveurs virtuels qui ne spécifient pas cette propriété.

  4. Dans le champ Taille du tampon du journal d'accès, entrez la taille, en octets, du tampon du journal d'accès.

    Une valeur inférieure ou égale à zéro désactive la mise en mémoire tampon du journal d'accès. La valeur par défaut est 4096.

  5. Dans le champ Intervalle d'écriture du journal d'accès, entrez le nombre de secondes devant s'écouler avant que le journal ne soit écrit sur le disque.

    Le journal d'accès est écrit lorsque le tampon est saturé ou que l'intervalle expire. Si la valeur est inférieure ou égale à zéro, le tampon est alors toujours écrit même s'il n'est pas plein. Autrement dit, chaque fois que l'utilisateur accède au serveur, le message journal est directement enregistré dans le fichier. La valeur par défaut est 0.

  6. (Facultatif) Dans la section Autres propriétés, spécifiez d'autres propriétés.

    Les propriétés suivantes sont disponibles :

    connectionTimeout

    Indique le délai (en millisecondes) d'attente pour que les listeners HTTP présentent la ligne d'URI de requête, après avoir accepté une connexion.

    La valeur par défaut est 12,000 (12 secondes).

    maxKeepAliveRequests

    Indique le nombre maximum de requêtes HTTP pouvant être émises en pipeline jusqu'à la fermeture de la connexion par le serveur. Définissez cette propriété sur 1 pour désactiver la connexion Keep-Alive HTTP/1.0, ainsi que la connexion Keep-Alive HTTP/1.1 et le pipelining.

    La valeur par défaut est 1000.

    traceEnabled

    Si elle est définie sur true (vrai), la méthode TRACE est activée. Définissez cette propriété sur false (faux) pour protéger Enterprise Server contre les attaques de type Cross-Site Scripting.

    Pour plus d'informations sur la méthode TRACE, reportez-vous au RFC 2616 : Hypertext Transfer Protocol -- HTTP/1.1.

    La valeur par défaut est 'true' (vrai).

  7. Cliquez sur Enregistrer.
