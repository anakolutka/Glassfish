JDBC 연결 풀을 만드는 방법

관리 콘솔에서 연결 풀을 만들 때 특정 데이터베이스에 대한 연결 부분을 정의합니다.

시작하기 전에

연결 풀을 만들려면 먼저 JDBC API를 지원하는 드라이버(JDBC 드라이버)를 설치 및 통합해야 합니다. JDBC 드라이버를 통합하는 방법을 참조하십시오.

JDBC 연결 풀을 만들 때 필요한 정보는 다음과 같습니다. 계속하기 전에 이 정보를 수집합니다:

  • 데이터베이스 공급업체 이름

  • 자원 유형. 예: javax.sql.DataSource(로컬 트랜잭션만), java.sql.ConnectionPoolDataSource(로컬 트랜잭션, 성능 향상 가능), javax.sql.XADataSource(전역 트랜잭션)

  • 데이터 소스 클래스 이름

  • 필수 등록 정보: 데이터베이스 이름(URL), 사용자 이름 및 비밀번호 등

  1. 탐색 트리에서 일반 작업 노드를 선택합니다.
  2. 일반 작업 페이지에서 새 JDBC 연결 풀 만들기 버튼을 누릅니다.

    새 JDBC 연결 풀 만들기(단계 1/2) 페이지가 열립니다.

    주 – 자원 노드와 JDBC 노드를 차례로 확장한 다음 연결 풀 노드를 선택하여 JDBC 연결 풀 페이지에 액세스할 수도 있습니다. 그런 다음 새 연결 풀 페이지에서 새로 만들기를 선택하여 새 연결 풀을 만듭니다.

  3. 이름 필드에 새 연결 풀의 이름을 입력합니다.

    필수 필드입니다.

  4. 자원 유형 드롭다운 목록에서 자원 유형을 선택합니다.

    선택 항목에는 javax.sql.XADataSource(전역 트랜잭션), java.sql.ConnectionPoolDataSource(로컬 트랜잭션, 성능 향상 가능), javax.sql.DataSource(로컬 트랜잭션만)가 포함됩니다.

  5. 데이터베이스 공급업체 드롭다운 목록에서 데이터베이스 공급업체를 선택합니다.
  6. 다음을 누릅니다.

    새 JDBC 연결 풀 만들기(단계 2/2) 페이지가 열립니다.

  7. 데이터 소스 클래스 이름 필드에 DataSource 및/또는 XADataSource API를 구현하는 공급업체별 클래스를 입력합니다.
  8. 설명 필드에 연결 풀에 대한 간단한 설명을 입력합니다.
  9. 초기 및 최소 풀 크기 필드에 이 풀의 최소 연결 수를 입력합니다.

    풀 값에 따라 풀을 먼저 만들거나 Application Server를 시작할 때 풀에 있는 연결 수도 결정합니다. 기본값은 8입니다.

  10. 최대 풀 크기 필드에 이 풀의 최대 연결 수를 입력합니다.

    기본값은 32입니다.

  11. 풀 크기 조정 개수 필드에 풀 크기 조정 일괄 작업당 연결 수를 입력합니다.

    풀이 각각 최대 및 최소 풀 크기로 확장되거나 축소되는 경우 크기 일괄적으로 조정됩니다. 이 값은 일괄적으로 처리할 연결 수를 지정합니다. 이 값을 너무 크게 하면 연결 생성 및 재순환이 지체되며, 너무 작게 하면 효율성이 떨어집니다. 기본값은 2입니다.

  12. 유휴 시간 초과 필드에 풀에서 연결이 유휴 상태로 유지될 수 있는 최대 시간(초)을 입력합니다.

    이 시간이 만료되면 연결이 풀에서 제거됩니다. 기본값은 30입니다.

  13. 최대 대기 시간 필드에 연결을 요청하는 응용 프로그램이 연결 시간이 초과될 때까지 대기할 최대 시간(밀리초)을 입력합니다.

    기본값은 60,000입니다.

  14. 연결 검증 필요 확인란을 선택하여 연결 검증을 사용하도록 설정합니다.

    실패하는 경우 서버에서 연결 검증을 사용하여 다시 연결할 수 있습니다.

  15. 검증 방법 드롭다운 목록에서 검증 방법을 선택합니다.

    Application Server는 데이터베이스 연결을 자동 완결, 메타데이터 및 테이블의 세 가지 방법으로 검증할 수 있습니다.

    자동 완결 및 메타데이터

    Application Server가 con.getAutoCommit 및 con.getMetaData 메소드를 호출하여 연결을 검증합니다.

    주 – 많은 JDBC 드라이버에서 이러한 호출의 결과를 캐시하므로 이러한 호출을 사용하면 경우에 따라 신뢰할 수 없는 검증이 제공될 수 있습니다. 이러한 호출의 캐시 여부를 판단하려면 드라이버 공급업체에 확인합니다.

    테이블

    응용 프로그램이 지정된 데이터베이스 테이블을 쿼리합니다. 테이블이 반드시 필요하며 액세스할 수 있어야 하지만 행은 없어도 됩니다. 행이 많이 있는 기존 테이블이나 이미 자주 액세스하는 테이블은 사용하지 마십시오.

  16. 검증 방법 드롭다운 목록에서 테이블을 선택한 경우 테이블 이름 필드에 데이터베이스 테이블의 이름을 입력합니다.

    테이블 이름에는 영숫자, 밑줄, 대시 또는 점 문자만 사용해야 합니다.

  17. 실패 시 모든 연결 닫기 확인란을 선택하여 연결이 실패할 경우 서버에서 풀의 모든 연결을 닫고 다시 설정하게 합니다.

    이 확인란의 선택을 취소하면 개별 연결이 사용되는 경우에만 다시 설정됩니다.

  18. 구성 요소가 아닌 호출자가 풀을 사용할 수 있게 하려면 구성 요소가 아닌 호출자 허용 사용 가능 확인란을 선택합니다.

    구성 요소가 아닌 호출자의 예에는 서블릿 필터 및 라이프사이클 모듈이 있습니다.

  19. 트랜잭션이 아닌 연결로 돌아가려면 트랜잭션이 아닌 연결 사용 가능 확인란을 선택합니다.
  20. 트랜잭션 격리 수준 드롭다운 목록에서 이 연결 풀의 트랜잭션 격리 수준을 선택합니다.

    지정하지 않는 경우 연결은 JDBC 드라이버에서 제공하는 기본 격리 수준으로 실행됩니다.

  21. 격리 수준 보장 확인란은 20단계에서 트랜잭션 격리 수준을 지정한 경우에만 적용됩니다.

    보장 확인란을 선택한 경우 풀에서 가져온 모든 연결은 동일한 격리 수준을 갖게 됩니다. 예를 들어, 마지막으로 사용 시 연결의 격리 수준을 프로그래밍 방식으로 변경한 경우(예: con.setTransactionIsolation 사용) 이 메커니즘은 상태를 지정한 격리 수준으로 다시 변경합니다.

  22. (옵션) 추가 등록 정보 섹션에서 추가 등록 정보를 지정합니다.

    필요한 등록 정보의 예로는 데이터베이스 이름(URL), 사용자 이름, 비밀번호 등이 있습니다. 등록 정보는 데이터베이스 공급업체마다 다르기 때문에 자세한 내용은 공급업체의 설명서를 참조하십시오.

  23. 마침을 누릅니다.
