기본 감사 모듈을 사용하는 방법

인증 로그 항목에는 다음 정보가 포함됩니다.

  • 인증을 시도한 사용자의 이름

  • 액세스 요청을 처리한 영역

  • 요청된 웹 모듈 URI 또는 EJB 구성 요소

  • 요청의 성공 또는 실패 여부

감사 로깅의 활성화 여부에 관계없이 Enterprise Server는 거부된 모든 인증 이벤트를 기록합니다.

권한 부여 로그 항목에는 다음 정보가 포함됩니다.

  • 인증된 사용자의 이름(있는 경우)

  • 요청된 웹 URI 또는 EJB 구성 요소

  • 요청의 성공 또는 실패 여부

  1. 탐색 트리에서 구성 노드를 확장합니다.
  2. 구성할 인스턴스를 선택합니다.
    • 특정 인스턴스를 구성하려면 해당 인스턴스 구성 노드를 확장합니다. 예를 들어, 기본 인스턴스 server의 경우 server-config 노드를 확장합니다.
    • 모든 인스턴스의 기본 설정을 구성하려면 default-config 노드를 확장합니다.
  3. 구성 노드에서 보안 노드를 확장합니다.
  4. 보안 노드에서 감사 모듈 노드를 확장합니다.
  5. 감사 모듈 모드에서 default 노드를 선택합니다.

    감사 모듈 편집 페이지가 열립니다.

  6. auditOn 등록 정보 값을 true로 설정합니다.
  7. 저장을 선택합니다.
  8. 콘솔에 다시 시작해야 함이 표시되면 Enterprise Server를 다시 시작합니다.
참조

  • 감사 모듈을 만드는 방법

  • 감사 모듈을 편집하는 방법

  • 감사 모듈을 삭제하는 방법

  • 감사 로깅을 활성화하거나 비활성화하는 방법

  • 활성 감사 모듈을 설정하는 방법
