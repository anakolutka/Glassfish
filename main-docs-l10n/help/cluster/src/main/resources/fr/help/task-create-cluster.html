Création d'un cluster

Un cluster est une collection nommée d'instances GlassFish Server qui partagent les mêmes applications, ressources et données de configuration.

Un cluster fournit une haute disponibilité grâce à son évolutivité, à l'équilibrage de la charge et à une protection contre les pannes. Lorsqu'une instance ou un hôte d'un cluster est en panne, GlassFish Server détecte la défaillance, redirige les demandes de l'instance en échec vers d'autres instances du cluster et rétablit l'état de la session utilisateur. Dans la mesure où les mêmes applications et ressources se trouvent sur toutes les instances du cluster, une instance peut basculer vers n'importe quelle autre instance du cluster.

GlassFish Server vous permet d'administrer toutes les instances d'un cluster comme une unité unique à partir d'un seul hôte, que les instances résident sur le même hôte ou sur des hôtes différents.

Avant de commencer

Si vous avez l'intention d'ajouter des instances GlassFish Server au cluster que vous créez, assurez-vous que les noeuds sur lesquels les instances résideront existent déjà.

  1. Dans l'arborescence de navigation, sélectionnez le noeud Clusters.

    La page Clusters apparaît.

  2. Sur la page Clusters, cliquez sur Nouveau.

    La page Nouveau cluster apparaît.

  3. Dans le champ Nom du cluster, saisissez le nom du cluster.

    Le nom doit respecter les exigences suivantes :

    • Il ne peut contenir que des caractères ASCII.

    • Il doit commencer par une lettre, un chiffre ou un trait de soulignement.

    • Il ne peut contenir que les caractères suivants :

      • Lettres minuscules

      • Lettres majuscules

      • Chiffres

      • Trait d'union

      • Point

      • Trait de soulignement

    • Le nom doit être unique dans le domaine et ne doit pas appartenir à un autre cluster, à une configuration nommée, à une instance GlassFish Server ou à un noeud.

    • Il ne doit pas être domain, server ou tout autre mot-clé réservé par GlassFish Server.

  4. Dans la liste déroulante Configuration, sélectionnez la configuration nommée que le cluster utilisera.

    Remarque:

    Si vous sélectionnez default-config, vous devez spécifier à l'étape suivante que la configuration n'est pas partagée. La configuration default-config ne peut pas être partagée.

  5. Indiquez si la configuration que le cluster utilisera est partagée.

    • Si la configuration n'est pas partagée, laissez l'option Créer une copie de la configuration sélectionnée cochée.

      La configuration sélectionnée sera ainsi copiée lors de la création du cluster. Le nom cluster-name-config est affecté à la copie de la configuration, cluster-name correspondant au nom du cluster que vous créez.

    • Si la configuration est partagée, sélectionnez l'option Utiliser la configuration sélectionnée comme référence.

      Le cluster utilisera la configuration nommée existante indiquée.

  6. Ajout d'instances GlassFish Server au cluster

    Remarque:

    Pour créer une instance sur un noeud de type CONFIG représentant un hôte distant, vous devez utiliser la sous-commande create-local-instance(1). Vous ne pouvez pas utiliser la Console d'administration à cet effet.

    Vous pouvez également ajouter des instances une fois le cluster créé en suivant la procédure décrite dans Ajout d'une instance GlassFish Server à un cluster.

    Ajoutez chaque instance comme suit :

    1. Dans la zone Instances de serveur à créer, cliquez sur Ajouter.

      Une ligne est ajoutée à la table des instances dans le cluster.

    2. Dans le champ Nom d'instance, saisissez le nom de l'instance.

    3. Dans le champ Poids, entrez un entier qui représente la pondération de l'équilibrage de charge de l'instance.

      La pondération de l'équilibrage de charge détermine la proportion de toutes les demandes destinées au cluster que l'instance doit traiter.

      Par exemple, dans un cluster comportant deux instances, vous pouvez avoir besoin d'une instance pour le traitement de l'une des quatre demandes et d'une autre instance pour le traitement des trois demandes restantes. Dans ce cas, définissez la pondération de l'instance qui doit traiter une demande sur quatre sur 1 et celle de l'autre instance sur 3. Si vous préférez utiliser des pourcentages, définissez la pondération des instances sur 25 et 75, respectivement.

      La pondération par défaut est de 100.

    4. Dans la liste déroulante Noeud, sélectionnez le noeud sur lequel l'instance résidera.

  7. Cliquez sur OK.

    Vous êtes renvoyé à la page Clusters. Le cluster est ajouté à la table des clusters sur la page.
