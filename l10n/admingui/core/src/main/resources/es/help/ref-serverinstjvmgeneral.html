Configuración general JVM

La máquina virtual de JavaTM (JVMTM) permite ejecutar código de bytes de Java en un equipo físico. Al ajustar los valores de configuración de JVM, mejora el rendimiento y se evitan errores de asignación de memoria.

La máquina virtual de Java se incluye en la Plataforma de Java, Standard Edition (Plataforma Java SE), necesaria para Servidor empresarial. La introducción de valores incorrectos para la configuración de JVM hará que no funcione el servidor, por lo que tiene que ir con cuidado a la hora de cambiar estos valores.

Nota - Los términos “máquina virtual de Java” y “JVM” hacen referencia a una máquina virtual para la plataforma Java.

La página Configuración general de JVM contiene las siguientes opciones.

Inicio de Java

Nombre del directorio de instalación del software de Java SE, que necesita Servidor empresarial para poder ejecutarse. Para comprobar que la versión del software de JAVA SE que ha especificado es compatible con esta versión, consulte las Sun GlassFish Enterprise Server v3 Preview Release Notes . Si introduce un nombre de directorio no existente o un directorio para una versión no compatible del software de Java SE, entonces Servidor empresarial no se podrá iniciar.

Opciones de Javac

Opciones de línea de comandos para el compilador del lenguaje de programación de Java. Servidor empresarial ejecuta el compilador cuando se implementan los componentes de Enterprise JavaBeans TM (EJBTM).

Depuración

Habilita la depuración con Java Platform Debugger Architecture (JPDA).

JPDA lo utilizan los desarrolladores de aplicaciones. Para obtener más información, consulte la Sun GlassFish Enterprise Server v3 Preview Application Development Guide .

Opciones de depuración

Especifica las opciones de JPDA que enviará a JVM cuando se habilite la depuración.

Opciones de compilación RMI

Opciones de línea de comandos para el compilador rmic. Servidor empresarial ejecuta el compilador rmic cuando se implementan componentes EJB.

Preprocesador de código byte

Nombres de clases para el preprocesamiento de código byte. Utilice comas para separar los nombres de clase. Cada clase debe implementar la interfaz com.sun.appserv.BytecodePreprocessor . Las clases se llaman en el orden especificado.

Propiedades adicionales

Propiedades adicionales para JVM. Servidor empresarial no define ninguna propiedad adicional para JVM.
