JVM 경로 설정을 구성하는 방법

클래스 경로는 Java 런타임 환경이 클래스 및 기타 자원을 찾는 JAR 파일의 목록입니다.

  1. 탐색 트리에서 Application Server 노드를 선택합니다.
  2. 일반 정보 페이지에서 JVM 설정 탭을 누릅니다.
  3. JVM 일반 설정 페이지에서 경로 설정 탭을 누릅니다.

    JVM 클래스 경로 설정 페이지가 열립니다.

  4. 환경 클래스 경로 무시 확인란을 선택하여 CLASSPATH 환경 변수를 무시합니다.

    CLASSPATH 환경 변수는 프로그래밍의 기본 자습서에는 편리하지만 엔터프라이즈 환경에서는 사용하지 않는 것이 좋습니다.

  5. 시스템 클래스 경로 필드에 시스템에 있는 클래스의 경로를 입력합니다.
  6. 클래스 경로 접두어 필드에 서버의 클래스 경로 앞에 추가할 JAR 파일의 전체 경로를 입력합니다.

    여러 파일을 덧붙이려면 경로를 한 줄에 하나씩 입력합니다.

  7. 클래스 경로 접미어 필드에 서버의 클래스 경로 뒤에 추가할 JAR 파일의 전체 경로를 입력합니다.

    예를 들어 데이터베이스 드라이버의 JAR 파일을 지정할 수 있습니다. JDBC 드라이버를 통합하는 방법을 참조하십시오.

    여러 파일을 추가하려면 경로를 한 줄에 하나씩 입력합니다.

  8. 원시 라이브러리 경로 접두어 필드에 원시 라이브러리 경로 앞에 추가할 JAR 파일의 전체 경로를 입력합니다.

    여러 파일을 덧붙이려면 경로를 한 줄에 하나씩 입력합니다.

  9. 원시 라이브러리 경로 접미어 필드에 원시 라이브러리 경로에 추가할 JAR 파일의 전체 경로를 입력합니다.

    여러 파일을 추가하려면 경로를 한 줄에 하나씩 입력합니다.

    원시 라이브러리 경로는 다음 경로가 결합된 것입니다.

    • 원시 공유 라이브러리에 대한 서버의 상대 경로

    • 표준 런타임 환경 원시 라이브러리 경로

    • 쉘 환경 설정(UNIX® 시스템의 LD_LIBRARY_PATH)

    • JVM 프로필러 설정 페이지에서 지정된 모든 경로

  10. 저장을 누릅니다.
  11. 서버를 다시 시작합니다.
