Para Editar um Serviço de Executor Gerenciado

  1. Na árvore de navegação, expanda o nó Recursos.

  2. No nó Recursos, expanda o nó Recursos Concorrentes.

  3. No nó Recursos Concorrentes, selecione Serviços de Executor Gerenciado.

    A página Serviços de Executor Gerenciado é aberta.

  4. Na página Serviços de Executor Gerenciado, clique no nome do recurso que você está editando.

    A página Editar Serviço de Executor Gerenciado será aberta.

    Observação:

    O nome JNDI é um campo somente para leitura. Você pode especificar o nome JNDI ao criar um novo recurso. O nome JNDI lógico também é um campo somente para leitura.

  5. Marque a caixa de seleção Informações de Contexto Ativadas para propagar contextos a threads, e selecione os contextos que você deseja propagar.

    Esta opção é ativada e todos os contextos são selecionadas por default. Use Shift-clique ou Control-clique para escolher vários contextos. Se essa opção estiver desativada, os contextos não serão propagados, mesmo que eles estejam selecionados.

  6. Marque a caixa de seleção Status ativo para ativar o recurso.

    Por default, a opção está ativada.

  7. No campo Prioridade do Thread, digite a prioridade a ser designada aos threads criados.

    O valor default é 5.

  8. Marque a caixa de seleção Tarefas de Longa Execução Ativadas para usar o recurso de processar tarefas de longa execução.

    Se esta opção estiver ativada, tarefas de longa execução não serão relatadas como paradas. Por default, essa opção está desativada.

  9. No campo Bloqueado Após, digite o número de segundos durante os quais as tarefas podem ser executadas antes de se considerar que elas não estão respondendo.

    O valor default é 0, o que significa que as tarefas nunca são consideradas sem resposta.

  10. No campo Ordem de Implantação, digite um número para especificar a ordem de implantação do recurso.

    Recursos com números menores são carregados primeiro na inicialização do servidor. O valor default é 100.

  11. No campo Descrição, digite uma descrição curta do recurso.

    Esta descrição é um valor de string e pode incluir no máximo 250 caracteres.

  12. No campo Tamanho Núcleo, digite o número de threads a serem mantidos em um pool de threads, mesmo que eles estejam inativos.

    Quando uma nova tarefa é submetida e o número de threads em execução é menor que o tamanho do núcleo, um novo thread é criado para manipular a solicitação. O valor default é 0, o que significa que novos threads nunca são criados, a menos que a fila de tarefas esteja cheia ou que o recurso esteja utilizando o enfileiramento direto. O enfileiramento direto ocorre quando a capacidade de enfileiramento de tarefas é 0 ou quando ela é 2147483647 e o tamanho do núcleo é 0.

  13. No campo Tamanho Máximo do Pool, digite o número máximo de threads que um pool de threads pode conter.

    O valor default é 2147483647, o que significa que o pool de threads é essencialmente não vinculado e pode conter qualquer número de threads.

  14. No campo Manutenção de Funcionamento, digite o número de segundos durante os quais os threads podem permanecer inativos quando o número de threads for maior que o tamanho do núcleo.

    O valor default é 60.

  15. No campo Tempo de Vida do Thread, digite o número de segundos durante os quais os threads podem permanecer em um pool de threads antes de serem expurgados, independentemente de o número de threads ser maior do que o tamanho do núcleo ou de se os threads estão inativos.

    O valor default é 0, o que significa que os threads nunca são expurgados.

  16. No campo Capacidade da Fila de Tarefas, digite o número de tarefas submetidas que podem ser armazenadas na fila de tarefas que aguardam execução.

    O valor default é 2147483647, o que significa que a fila de tarefas é essencialmente não vinculada e pode armazenar qualquer número de tarefas submetidas.

  17. Na seção Propriedades Adicionais, não adicione propriedades.

    O GlassFish Server não define propriedades adicionais para este tipo de recurso. Além disso, este tipo de recurso não usa atualmente propriedades adicionais.

  18. Clique em Salvar.
