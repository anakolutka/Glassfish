Novo Protocolo

Utilize a página Novo Protocolo para criar um novo protocolo de rede.

Após criar o protocolo, você normalmente cria um listener de rede com o mesmo nome, e associa o listener de rede com o protocolo. Como alternativa, você pode criar um listener de rede e ter um protocolo com definições default ao mesmo tempo.

A página Novo Protocolo contém estas opções.

Nome da Configuração

O nome da configuração à qual as definições nesta página se aplicam. Este campo é somente para leitura.

Nome

O nome exclusivo do protocolo. Um nome de protocolo não pode começar com um número.

Segurança

Se essa opção for selecionada, a segurança será ativada para o protocolo. Por default, essa opção está desativada. Caso você ative a segurança, poderá usar a guia SSL da página Editar Protocolo para modificar as definições de segurança.

Status

Se essa opção for selecionada, o cache de arquivo será ativado. Por default, a opção está ativada.

Duração Máxima

A duração máxima, em segundos, de uma entrada de cache válida.

Esse parâmetro controla a quantidade de informações armazenadas no cache que será utilizada após o armazenamento de um arquivo no cache. Uma entrada mais antiga do que a duração máxima será substituída pela nova entrada do mesmo arquivo. Se o conteúdo não for alterado com frequência, aumente esse valor para melhorar o desempenho.

A duração máxima mais eficiente depende da regularidade com que os arquivos existentes são modificados. Por exemplo, se os arquivos forem modificados quatro vezes ao dia, em intervalos regulares, considere a possibilidade de definir a duração máxima em 21600 segundos (6 horas). Ou então, defina a duração máxima como sendo o tempo mais longo desejado para a exibição da versão anterior de um arquivo após a modificação.

O valor default é 30.

Tamanho Máximo do Cache

O tamanho total máximo (em bytes) do cache de arquivos, no disco. O valor default é 10485760.

Contagem Máxima de Arquivos

O número máximo de arquivos que podem ser armazenados no cache de arquivos.

Se o valor for muito alto, o servidor armazenará no cache os arquivos pouco necessários, o que consome memória. Se o valor for muito baixo, os benefícios do armazenamento no cache não será obtidos. Tente diferentes valores desse atributo para encontrar a solução mais eficiente para aplicações específicas.

O valor default é 1024.

Nome do Servidor

O nome do host a ser utilizado nas URLs que o servidor envia ao cliente. Esse nome será o nome alias se o servidor usar um alias. Se o servidor não usar alias, deixe o campo em branco.

Esse valor somente afeta os URLs gerados automaticamente pelo servidor; ele não afeta os URLs para diretórios e arquivos armazenados no servidor. Caso seu servidor use um alias, o nome do servidor deve ser o nome do alias. Se uma vírgula e um número de porta forem acrescentados, essa porta será usada nos URLs que o servidor envia para o cliente.

Servidor Virtual Default

O servidor virtual a ser associado com esse protocolo. Use a página Servidores Virtuais para definir servidores virtuais.

Porta de Redirecionamento

Valor de porta que redireciona uma solicitação para outra porta. O GlassFish Server redirecionará automaticamente a solicitação se estas duas condições existirem:

  • O listener de rede que referencia esse protocolo oferece suporte a solicitações não-SSL.

  • Uma solicitação é recebida, e a correspondente restrição de segurança exige transporte SSL.

Se uma porta de redirecionamento não estiver especificada, o GlassFish Server usará o número de porta especificado na solicitação original. Por default, nenhuma porta é especificada.

Máximo de Conexões

Especifica o número máximo de solicitações que podem colocar na fila até que a conexão seja fechada pelo servidor. O subsistema Keep-Alive consulta periodicamente as conexões ociosas. O valor default é 256.

Defina esta propriedade para 1 para desativar o keep-alive HTTP/1.0, assim como o keep-alive HTTP/1.1 e pipeline. Um valor de 0 significa que as solicitações são sempre rejeitadas. Um valor de -1 indica que não há limite para conexões keep-alive.

Time-out

A quantidade máxima de tempo, em segundos, que uma conexão pode permanecer inativa e mantida no estado keep-alive. Um valor de 0 ou menos significa que as conexões keep-alive são mantidas abertas indefinidamente. O valor default é 30 segundos.

Time-out para Upload

Se essa opção é selecionada, a conexão para um servlet que lê lentamente os bytes seja fechada após o limite do Time-out para Upload de Conexão é alcançado. Se essa opção for desativada, as conexões de servlet não são submetidas a time-out. Por default, essa opção está desativada.

Time-out para Upload de Conexão

O time-out para uploads em milissegundos. Esse campo só se aplicará se a caixa de seleção Time-out para Upload Ativado estiver marcada. O valor default é 300.000 milissegundos.

Time-out para Solicitação

O número de segundos antes que a solicitação seja submetida a time-out. Se a solicitação não for processada antes que o valor do time-out seja alcançado, a solicitação será ignorada. O valor default é 30 segundos.

Tamanho do Buffer para Envio

O tamanho máximo em bytes do buffer para envio. O valor default é 8.192 bytes.

Tamanho do Buffer de Cabeçalho

O tamanho em bytes do buffer utilizado pelos threads de processamento da solicitação para ler os dados da solicitação. O valor default é 8.192 bytes.

Tamanho Máximo de Postagem

O tamanho máximo em bytes de ações POST. O valor default é 2.097.152 bytes.

Codificação do URI

O nome do conjunto de caracteres usado para decodificar os URIs de solicitação recebidos. O valor precisa ser um nome válido de um conjunto de caracteres IANA. O valor default é UTF-8.

Versão

A versão do protocolo HTTP usado. O valor default é HTTP/1.1.

Compactação

Especifica o uso da compactação GZIP HTTP/1.1 para reduzir a largura de banda do servidor. As opções disponíveis são:

on

Compacta os dados.

off

Desativa a compactação.

force

Força a compactação dos dados em todos os casos.

O valor default off.

Tipos MIME Compactáveis

Uma lista separada por vírgulas dos tipos MIME para os quais a compactação HTTP é utilizada. O valor default é text/html,text/xml,text/plain.

Tamanho Mínimo de Compactação

O tamanho mínimo de um arquivo quando a compactação é aplicada. Esse valor deve ser definido se a Compactação estiver definida como on ou force. O valor default é 2.048 bytes.

Agentes de Não Compactação do Usuário

Uma lista separada por vírgulas de expressões regulares que coincidem com agentes do usuário ou clientes HTTP, para os quais a compactação não deve ser utilizada. Por default esse valor é uma string vazia.

Agente de Usuário Restrito

Uma lista de agentes do usuário restrito no qual a compactação HTTP é aplicada. Se nenhum agente do usuário for especificado, a compactação HTTP será aplicada em todos os agentes do usuário. Por default, nenhum agente do usuário é especificado.

Tipo de Resposta Default

Uma string que especifica o tipo de resposta default. O formato é uma string delimitada por pontos e vírgulas, consistindo no tipo de conteúdo, codificação, idioma e conjunto de caracteres. O valor default é text/html; charset=iso-8859-1.

Tipo de Resposta Forçada

Uma string que especifica o tipo de solicitação usado, caso nenhum mapeamento MIME esteja disponível que coincida com a extensão do arquivo. O formato é uma string delimitada por pontos e vírgulas, consistindo no tipo de conteúdo, codificação, idioma e conjunto de caracteres. O valor default é text/html; charset=iso-8859-1.

Adaptador

O nome da classe do adaptador de recursos estáticos. O valor default é com.sun.grizzly.tcp.StaticResourcesAdapter.

Suporte a Comet

Se esta opção for selecionada, o suporte a Comet será ativado para o listener de rede que referencia este protocolo. Por default, essa opção é desativada. Se o seu servlet ou página JSP usa a tecnologia Comet, assegure-se de que esteja inicializada quando o GlassFish Server é iniciado, ao adicionar o elemento load-on-startup em seu arquivo web.xml. Por exemplo:

<servlet>
<servlet-name>CheckIn</servlet-name>
<servlet-class>CheckInServlet</servlet-class>
<load-on-startup>0</load-on-startup>
</servlet>
Pesquisa de DNS

Se essa opção for selecionada, a pesquisa de DNS será ativada. Por default, essa opção está desativada.

Pesquisa de DNS

Se essa opção for selecionada, a pesquisa de DNS será ativada. Por default, essa opção está desativada.

Suporte RCM

Se esta opção for selecionada, o Gerenciamento de Configuração de Recursos (RCM) será ativado. Por default, essa opção está desativada.

Rastrear

Se esta opção for selecionada, a operação TRACE será ativada. Se essa opção for desativada, o GlassFish Server é menos suscetível ao ataques de scripts de site cruzado. Por default, a opção está ativada.

Passagem de Autorização

Se essa opção for selecionada, indica que o listener de rede que utiliza esse protocolo recebe tráfego de um servidor proxy em que a conexão SSL proveniente de um cliente termina no servidor proxy. Por default, essa opção está desativada.

Agrupamento

Se essa opção for selecionada, a resposta do agrupamento HTTP será ativada. Por default, a opção está ativada.

XPowered por

Se esta opção for selecionada, serão usados cabeçalhos de X-Powered-By de acordo com as especificações de Java Servlet 3.0 e Java Server Pages 2.0. Por default, a opção está ativada.

Barra Codificada

Se essa opção for selecionada, será permitido usar URIs para conter barras decodificadas. Por default, essa opção está desativada.

Suporte à Websockets

Se essa opção for selecionada, o protocolo de WebSockets é suportado. Por default, essa opção está desativada.
