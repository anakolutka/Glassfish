Para crear un pool de conexiones de conector

Antes de Empezar

Antes de crear el pool, implante el módulo del conector (adaptador de recursos) asociado al pool. Los valores que se especifican para el nuevo pool dependen del módulo de conector que se despliegue.

  1. En el árbol de navegación, amplíe el nodo Recursos.

  2. Amplíe el nodo Conectores.

  3. En el nodo Configuración, seleccione el nodo Pools de conexiones de conector.

    Se abrirá la página Pools de conexiones de conectores.

  4. En la página Pools de conexiones de conectores, haga clic en Nuevo.

    Se abrirá la página Nuevo Pool de Conexiones del Conector (paso 1 de 2).

  5. En el campo Nombre de pool, asigne un nombre único al pool.

    Especifique este nombre cuando cree un recurso de conector.

  6. En la lista desplegable Adaptador de recursos, haga una selección en la lista de adaptadores de recursos implantados (módulos de conector).

  7. En la lista desplegable Definición de Conexión, seleccione la definición de conexión para el pool de conexiones del conector.

    Las opciones de la lista dependen del adaptador de recursos seleccionado. La definición de conexión identifica un elemento connection-definition del adaptador de recursos en el archivo ra.xml.

  8. Haga clic en Siguiente.

    Se abrirá la página Nuevo Pool de Conexiones del Conector (paso 2 de 2).

  9. Active la casilla de control de activación de ping para hacer ping en el pool de conexiones durante la creación o reconfiguración para identificar los valores erróneos de sus atributos y avisar de ellos.

    Esta opción se activada por defecto.

  10. En el campo Descripción, introduzca una descripción del pool de conexiones.

  11. En el campo Tamaño de Pool Inicial y Mínimo, introduzca el número mínimo de conexiones del pool.

    El valor por defecto es 8.

  12. En el campo Tamaño de Pool Máximo, introduzca el número máximo de conexiones de este pool.

    El valor por defecto es 32.

  13. En el campo Cantidad de Cambio de Tamaño del Pool, especifique el número de conexiones que se eliminarán cuando se supere el timeout límite de inactividad del pool.

    El valor por defecto es 2.

  14. En el campo Timeout de Inactividad, indique el tiempo máximo en segundos que una conexión puede permanecer inactiva en el pool.

    El valor por defecto es 300.

  15. En el campo Tiempo de Espera Máximo, indique la cantidad de tiempo que la aplicación que solicita una conexión puede esperar para obtener un mensaje de timeout de conexión.

    El valor por defecto es 60000.

  16. Active la casilla de control Validación de conexión necesaria para activar la validación de la conexión.

    Si la casilla de control Necesaria está activada, las conexiones se validarán antes de proporcionárselas a la aplicación. Si falla la validación de un recurso, éste se destruye, y se crea y se devuelve un recurso nuevo. Esta opción se activada por defecto.

  17. Active la casilla de control Ante Cualquier Fallo Cerrar Todas las Conexiones, para que GlassFish Server cierre todas las conexiones del pool y las restablezca aunque tan sólo falle una.

    Si no activa esta casilla de control, las conexiones individuales se restablecerán sólo cuando vaya a utilizarlas. Esta opción se activada por defecto.

  18. Utilice la lista desplegable Soporte de Transacción para seleccionar el tipo de soporte de transacción para el pool de conexiones.

    El soporte de transacción seleccionado sustituye el atributo de soporte de transacción en el adaptador de recursos asociado al pool de conexiones de una forma descendente. Dicho de otro modo, puede soportar un nivel de transacción inferior o idéntico al que se especifica en el adaptador de recursos, pero no puede indicar un nivel superior.

    Las opciones siguientes están disponibles en la lista desplegable:

    • NoTransaction: indica que el adaptador de recursos no soporta transacciones JTA o transacciones locales del gestor de recursos y no implanta las interfaces XAResource o LocalTransaction.

    • LocalTransaction: indica que el adaptador de recursos soporta transacciones locales mediante la implantación de la interfaz LocalTransaction.

      Las transacciones locales se gestionan de manera interna en un gestor de recursos y no involucran a gestores de transacciones externos.

    • XATransaction: indica que el adaptador de recursos soporta transacciones JTA y transacciones locales del gestor de recursos mediante la implantación de las interfaces LocalTransaction y XAResource.

      Las transacciones XA se controlan y coordinan a través de un gestor de transacciones externo en un gestor de recursos.

  19. En la sección Propiedades adicionales, especifique propiedades adicionales del pool de conexiones de conector.

    Las propiedades disponibles dependen del adaptador de recursos para el pool de conexiones de conector.

    Para agregar una propiedad, haga clic en el botón Agregar Propiedad. En la fila en blanco que aparece, introduzca el nombre de propiedad en el campo Nombre y escriba el valor de propiedad en el campo Valor.

    Si especifica un nombre, pero no un valor para una propiedad, éste se elimina de la tabla al finalizar la creación del pool de conexiones. Si no desea definir un valor para una propiedad ahora, pero desea que la propiedad permanezca en la tabla para que el valor se pueda definir más adelante, introduzca un juego de paréntesis vacío en el campo Valor:

    ()

  20. Haga clic en Aceptar.
