HTTP サービス

「HTTP サービス」ページには、次のオプションが含まれます。

アクセスログ

選択した場合、これにより、このプロパティーが指定されていないすべての仮想サーバーサブ要素のアクセスログが有効になります。選択しない場合、このオプションにより、このプロパティーが指定されていないすべての仮想サーバーサブ要素のアクセスログが無効になります。

アクセスログバッファーのサイズ

バイト単位のアクセスログバッファーのサイズ、またはバッファーされていないアクセスログに対してはゼロ以下の値。デフォルト値は 4096 です。

アクセスログの書き込み間隔

ログがディスクに書き込まれるまでの秒数。アクセスログは、バッファーがいっぱいになるか、指定した時間が経過すると書き込まれます。値が 0 以下の場合、いっぱいになっていなくてもバッファーは常に書き込まれます。つまり、サーバーアクセスがあるたびに、ログメッセージはファイルに直接格納されます。

追加プロパティー

HTTP サービスの追加プロパティー。

次のプロパティーが使用可能です。

connectionTimeout

接続の受け入れ後、要求 URI の行が提示されるまでの HTTP リスナーの待ち時間を指定します (ミリ秒単位)。

デフォルト値は 12,000 (12 秒) です。

maxKeepAliveRequests

サーバーによって接続が閉じられるまでの、パイプラインできる HTTP 要求の最大数を指定します。HTTP/1.0 キープアライブ、HTTP/1.1 キープアライブ、パイプラインが無効化するには、このプロパティーの値を 1 に設定します。

デフォルト値は 1000 です。

traceEnabled

true に設定すると、TRACE メソッドが有効化されます。Enterprise Server がクロスサイトスクリプト攻撃の影響を受けにくくするには、このプロパティーを false に設定します。

TRACE メソッドについての詳細は、「RFC 2616: Hypertext Transfer Protocol -- HTTP/1.1」を参照してください。

デフォルト値は true です。
