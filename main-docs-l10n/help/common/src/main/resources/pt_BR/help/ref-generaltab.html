Informações Gerais

O GlassFish Server cria uma instância de servidor de aplicações chamada server no momento da instalação. É possível deletar a instância de servidor e criar uma nova instância com um nome diferente, se você preferir.

Cada instância de GlassFish Server tem sua própria configuração Java, recursos Java, áreas de implantação de aplicações e definições de configuração do servidor. As alterações em uma instância do servidor de aplicações não têm efeito em outras instâncias do servidor de aplicações. É possível ter uma instância do servidor de aplicações dentro de um domínio administrativo.

Para muitos usuários, uma instância do servidor de aplicações é suficiente para atender às suas necessidades. Entretanto, dependendo do seu ambiente, é possível criar uma ou mais instâncias do servidor de aplicações. Por exemplo, em um ambiente de desenvolvimento, é possível utilizar diferentes instâncias do servidor de aplicações para testar diferentes configurações do GlassFish Server ou para comparar e testar diferentes implantações de aplicações. Como é possível adicionar ou deletar facilmente uma instância do servidor de aplicações, você pode utilizá-las para criar áreas temporárias de "sandbox" para experimentação durante o desenvolvimento.

Além disso, para cada instância do servidor de aplicações, você pode também criar servidores virtuais. Dentro de uma única instância do servidor de aplicações instalada, você pode oferecer nomes de domínio, Endereços IP e alguns recursos de administração a empresas ou indivíduos. Para os usuários, é quase como se tivessem seu próprio servidor web, sem o hardware e a manutenção básica do servidor. Esses servidores virtuais não ultrapassam instâncias do servidor de aplicações. Para obter mais informações sobre servidores virtuais, consulte Definições Gerais de JVM.

Em implantações operacionais, para diversas finalidades, você pode usar servidores virtuais em vez de várias instâncias do servidor de aplicações. No entanto, se os servidores virtuais não atenderem às suas necessidades, você também poderá usar várias instâncias do servidor de aplicações.

Uma instância de GlassFish Server não é iniciada automaticamente. Depois que você iniciar uma instância, ela será executada até que você a interrompa. Quando você interrompe uma instância do servidor de aplicações, ela para de aceitar novas conexões e, em seguida, aguarda a conclusão de todas as conexões pendentes. Se sua máquina falhar ou ficar off-line, o servidor é fechado e quaisquer solicitações que ele estivesse atendendo poderão ser perdidas.

Use a página Informações Gerais para verificar as definições do GlassFish Server e para exibir os dados do Java Virtual Machine.

A página Informações Gerais contém as seguintes informações.

Botão Interromper

Clique no botão Interromper para interromper o GlassFish Server.

Botão Reiniciar

Clique no botão Reiniciar para iniciar novamente o GlassFish Server.

Botão Exibir Arquivos de Log.

Clique no botão Visualizar Arquivos de Log para exibir arquivos de log para um GlassFish Server instância ou cluster.

Botão Rotacionar Log

Clique no botão Rotacionar Log para realizar a rotação do arquivo de log do Servidor de Administração (com o nome server).

Botão Recuperar Transação

Clique no botão Recuperar Transação para recuperar transações para o Servidor de Administração (com o nome server) na página Recuperar Transações.

Botão Administração Segura

Clique no botão de Administração Segura para ativar ou desativar a administração segura na página Administração Segura.

Nome

O nome do servidor atual.

Status

O status atual da instância do servidor. O servidor pode estar interrompido, iniciado ou em execução.

Tempo de Funcionamento

O número de horas e minutos que a instância do servidor está em execução continuamente. Esse é somente para leitura.

Balanceador de Carga HTTP

O nome do balanceador de carga, se estiver ativado para a instância do servidor.

JVM

Se você clicar em Relatório da JVM, será aberta uma janela separada que exibe relatórios sobre a Java Virtual Machine, incluindo um relatório de resumo, um relatório de gerenciamento de memória e de coleta de erros,um relatório de carregamento de classe e um dump de threads atuais.

Diretório de Configuração

O diretório localizado na máquina host, que contém arquivos de configuração; por exemplo, domain.xml.

Versão Instalada

O identificador da versão do produto.

Administração Segura

O status atual da administração segura. Valores possíveis são Ativado ou Não-Ativado

Depuração

Identifica se a depuração está ativada. Se a depuração estiver ativada, o número de porta também será exibido.

Porta(s) HTTP

As portas configuradas atualmente para solicitações e respostas HTTP.

Porta(s) IIOP

As portas configuradas atualmente para solicitações e respostas IIOP.
