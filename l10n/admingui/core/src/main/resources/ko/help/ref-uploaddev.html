응용 프로그램 또는 모듈 배포

응용 프로그램 또는 모듈 배포 페이지에서는 응용 프로그램을 배포할 수 있습니다.

응용 프로그램 또는 모듈 배포 페이지에는 모든 응용 프로그램에 대한 다음 옵션이 포함되어 있습니다. 추가 옵션은 응용 프로그램의 위치 필드를 지정한 후에만 나타납니다.

위치

배포 중인 응용 프로그램의 아카이브 위치입니다.

다음 옵션에서는 아카이브에서 액세스할 수 있는 위치 및 아카이브가 파일 또는 디렉토리인지 여부를 지정합니다.

서버에 업로드할 패키지화된 파일

아카이브는 클라이언트 시스템에 상주하거나 클라이언트 시스템에서 액세스할 수 있는 파일에 있습니다.

클라이언트 시스템은 브라우저를 통해 관리 콘솔을 보는 호스트입니다.

Enterprise Server에서 액세스할 수 있는 패키지화된 로컬 파일 또는 디렉토리

아카이브는 서버 시스템에 상주하는 파일이거나 확장된 디렉토리의 압축 해제된 응용 프로그램입니다.

서버 시스템은 Enterprise Server 도메인 관리 서버를 실행하는 호스트입니다.

유형

응용 프로그램의 유형입니다. 사용 가능한 선택 항목은 다음과 같습니다.

  • 웹 응용 프로그램

  • 엔터프라이즈 응용 프로그램

  • 응용 프로그램 클라이언트

  • 커넥터 모듈

  • EJB Jar

  • Ruby 응용 프로그램

  • 기타

다양한 응용 프로그램 유형에 대한 추가 옵션을 다음 섹션에서 설명합니다.

  • 웹 응용 프로그램에 대한 추가 옵션

  • 엔터프라이즈 응용 프로그램에 대한 추가 옵션

  • 응용 프로그램 클라이언트에 대한 추가 옵션

  • 커넥터 모듈에 대한 추가 옵션

  • EJB Jar에 대한 추가 옵션

웹 응용 프로그램에 대한 추가 옵션

응용 프로그램 유형이 웹 응용 프로그램이면 다음 옵션이 나타납니다.

컨텍스트 루트

응용 프로그램에 대한 경로입니다. 웹 응용 프로그램의 URL에서 컨텍스트 루트 바로 앞에는 포트 번호(http://host:port /context-root/...)가 있습니다. 컨텍스트 루트는 슬래시로 시작해야 합니다(예: /hello).

응용 프로그램 이름

응용 프로그램의 이름입니다.

가상 서버

이 응용 프로그램에 연결된 가상 서버입니다.

상태

이 옵션을 활성화하면 응용 프로그램이 활성화됩니다. 이 옵션은 기본적으로 활성화됩니다.

JSP 사전 컴파일

이 옵션을 사용하도록 설정된 경우 JSPTM(JavaServer PagesTM) 파일이 사전 컴파일됩니다. 이 옵션을 사용하지 않도록 설정된 경우 JSP 파일은 런타임에 처음으로 액세스될 때 컴파일됩니다. 이 옵션은 기본적으로 비활성화됩니다.

검증자 실행

이 옵션을 사용하도록 설정된 경우 배포 전에 검증자가 실행됩니다. 검증에 실패하면 배포가 수행되지 않습니다. 검증자는 파일의 구조와 내용을 검사합니다. 큰 응용 프로그램의 검증에는 많은 시간이 걸릴 수 있습니다. 이 옵션은 기본적으로 비활성화됩니다.

라이브러리

이 모듈 또는 응용 프로그램에 관련된 라이브러리 JAR 파일의 쉼표로 구분된 목록입니다. 경로는 절대 경로이거나 상대 경로일 수 있습니다. 상대 경로는 domain-dir/lib/applibs에 상대적입니다. 경로가 절대적이면 경로는 도메인 관리 서버(DAS)가 액세스할 수 있어야 합니다. 즉, domain-dir 아래에 있어야 합니다. 라이브러리는 지정한 순서대로 응용 프로그램에 사용 가능하게 됩니다.

Jruby 홈

JRuby 컨테이너가 설치된 디렉토리의 경로입니다. 기본 위치는 as-install/jruby입니다. 이 필드에 대한 값을 지정하면 응용 프로그램 값이 Ruby 컨테이너에 대해 설정된 JRuby 홈 값을 대체합니다.

런타임

시작할 JRuby 런타임의 초기 수입니다. 값은 최소 런타임 필드의 값보다 크거나 같지만 최대 런타임 필드의 값보다 작아야 합니다. 기본값은 1입니다. 이 필드에 대한 값을 지정하면 응용 프로그램 값이 Ruby 컨테이너에 대해 설정된 런타임 값을 대체합니다.

최소 런타임

풀의 최소 JRuby 런타임 수입니다. 값은 런타임 필드의 값보다 작거나 같지만 최대 런타임 필드의 값보다 작아야 합니다. 기본값은 1입니다. 이 필드에 대한 값을 지정하면 응용 프로그램 값이 Ruby 컨테이너에 대해 설정된 최소 런타임 값을 대체합니다.

최대 런타임

풀의 최대 JRuby 런타임 수입니다. 값은 최소 런타임 필드의 값보다 크거나 같고 런타임 필드의 값보다 크거나 같아야 합니다. 기본값은 2입니다. 이 필드에 대한 값을 지정하면 응용 프로그램 값이 Ruby 컨테이너에 대해 설정된 최대 런타임 값을 대체합니다.

프레임워크

응용 프로그램을 배포할 프레임워크입니다. 사용 가능한 선택 항목은 rails, merb, sinatra 및 auto-detection입니다. 기본값은 rails입니다. 이 등록 정보를 auto-detection 이외의 값으로 설정하면 시간이 오래 걸릴 수 있는 일반 자동 감지 프로세스를 건너뛰고 지정한 프레임워크에 강제로 배포합니다. 배포된 응용 프로그램이 지정한 프레임워크용으로 작성되지 않은 경우 오류가 발생합니다.

랙 환경

Rails 또는 Merb와 같은 JRuby 프레임워크가 실행되는 환경입니다. 사용 가능한 선택 항목은 production, development 및 test입니다. 기본값은 development입니다.

MT-안전

응용 프로그램이 실행될 프레임워크에 대한 스레드 안전 설정입니다. 이 등록 정보는 자동 감지된 사용자 제공 시작 스크립트를 사용하여 시작된 응용 프로그램에 영향을 미칩니다. 사용 가능한 선택 항목은 다음과 같습니다.

true

프레임워크가 스레드에 안전하며 따라서 풀을 만들 필요가 없음을 지정합니다.

false

응용 프로그램이 응용 프로그램 인스턴스의 풀로 시작되고 한 번에 하나의 스레드가 응용 프로그램의 각 인스턴스에 액세스함을 지정합니다.

default

Enterprise Server에서 자동으로 스레드 안전을 감지하려고 시도합니다. 이 값이 기본값입니다.

이 등록 정보는 스레드 안전을 자동으로 결정할 수 없는 프레임워크가 시작될 경우에만 영향을 미칩니다. 이 값을 true로 설정하더라도 자동 감지된 Rails 2.1.x 응용 프로그램이 스레드 안전 모드에서 시작되지 않으며, 스레드 안전 프레임워크를 풀 모드에서 시작되도록 하는 데 사용할 수도 없습니다.
