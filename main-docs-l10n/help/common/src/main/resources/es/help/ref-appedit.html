Editar aplicación

Utilice la página Editar Aplicación para modificar una aplicación existente.

La página Editar Aplicación contiene algunas o todas las opciones siguientes, en función del tipo de aplicación.

Nombre

Campo de sólo lectura que muestra el nombre de la aplicación que está editando.

El nombre puede incluir un identificador de versión opcional, que va a continuación del nombre y está separado de él por dos puntos (:). Para obtener más información sobre las versiones de módulo y de aplicación, consulte el apartado sobre las versiones de módulo y de aplicación en Oracle GlassFish Server Application Deployment Guide.

Estado

Si se activa la casilla de control Activado, se activa la aplicación. Esta opción está activada por defecto.

Raíz de Contexto

Para una aplicación web, especifica la ruta de acceso a la aplicación. En la URL de la aplicación web, la raíz de contexto sigue inmediatamente al número de puerto (http://host:puertot/raíz-contexto/...). La raíz de contexto debe comenzar por una barra diagonal, por ejemplo, /hello.

Disponibilidad

Si se activa la casilla de control Activado, se activa la alta disponibilidad para las sesiones web, la aplicación de puntos de control de bean de sesión con estado (SFSB) y posiblemente para la desactivación. Si se define en false (valor por defecto), se desactiva el almacenamiento de sesiones web y puntos de control de SFSB para la aplicación, aplicación web o módulo EJB especificado. Si se define en true, la aplicación o módulo especificados se activa para alta disponibilidad. Defina esta opción como true sólo si se ha configurado y activado la alta disponibilidad en niveles superiores, como, por ejemplo, los niveles de servidor y contenedor.

Esta opción aparece si existen clusters o instancias del servidor independientes aparte de la instancia de servidor por defecto (server).

Servidores Virtuales

Servidores virtuales asociados a esta aplicación.

La opción Servidores Virtuales aparece si sólo existe la instancia de servidor por defecto, server. Si existen clusters u otras instancias de servidor independientes, puede seleccionar los servidores virtuales después del despliegue. Vaya a la página Editar Aplicación, seleccione el separador Destino y seleccione Gestionar Servidores Virtuales para el destino que desee.

Java Web Start

(Para algunos tipos de aplicaciones) Si se activa la casilla de control Activado, la aplicación empleará el software Java Web Start. Java Web Start proporciona una forma independiente de los exploradores de desplegar aplicaciones Java para que se ejecuten en una Java Virtual Machine dedicada.

Descripción

Breve descripción de la aplicación.

Ubicación

Ubicación de la aplicación desplegada. Éste es un campo de sólo lectura.

Orden de despliegue

Orden de despliegue de la aplicación.

Las aplicaciones con un número inferior se cargan en primer lugar al iniciar el servidor. Una aplicación con un orden de aplicación de 102 se carga antes que una aplicación con un orden de aplicación de 110. Si no se especifica un orden de despliegue en el momento de desplegar la aplicación, se asigna un orden de despliegue por defecto de 100. Si dos aplicaciones tienen el mismo orden de despliegue, se despliega en primer lugar la aplicación que se haya cargado primero. Especificar un orden de despliegue es útil si la aplicación tiene dependencias y se deben cargar en un orden determinado.

Bibliotecas

Archivos JAR de biblioteca que necesitará la aplicación. Éste es un campo de sólo lectura.

Módulos y Componentes

Tabla en la que figuran los nombres de los módulos de la aplicación y sus motores. Para un punto final de servicio web, puede seleccionar Ver Punto Final para visualizar la página Información de Punto Final de Servicio Web. Para un cliente de aplicación puede seleccionar Iniciar para mostrar la página de inicio del cliente de aplicaciones o Descargar Stubs de Cliente para descargar los stubs de cliente.
