編輯 JDBC 連線池進階屬性

  1. 在瀏覽樹狀結構中，展開 [資源] 節點。
  2. 在 [資源] 節點下，選取 [連線池] 節點。

    [連線池] 頁面會隨即開啟。

  3. 在 [連線池] 頁面上，按一下您要編輯的池名稱。

    [編輯連線池] 頁面會隨即開啟。

  4. 在 [編輯連線池] 頁面上，按一下 [進階] 標籤。

    [編輯連線池進階屬性] 頁面會隨即開啟。

    備註 – [名稱] 欄位是唯讀欄位。您僅能在建立新的 JDBC 連線池時指定連線池名稱。

  5. 在 [敘述逾時] 欄位中，鍵入在終止執行時間過長的查詢之前，需等待的時間長度 (以秒為單位)。

    Enterprise Server 將對建立的敘述設定 QueryTimeout。將值設定為 -1 以停用此選項。預設值為 -1。

  6. 選取 [啟用環繞 JDBC 物件] 選項，可讓應用程式取得 Statement、PreparedStatement、CallableStatement、ResultSet、DatabaseMetaData 的環繞 JDBC 物件。

    預設值為 false。

  7. 在 [驗證間隔最久時間] 欄位中，鍵入到達驗證連線間隔最長時間之前，等待的時間長度 (以秒為單位)。

    這有助於減少連線發出的驗證請求數目。將該值設定為 0 會停用連線驗證。預設值為 0。

  8. 在 [漏失逾時] 欄位中，鍵入追蹤連線池中遺漏連線的時間長度 (以秒為單位)。

    將該值設定為 0 會停用連線遺漏追蹤。預設值為 0。

  9. 選取 [啟用漏失收回] 選項，可在連線遺漏追蹤完成之後，將遺漏的連線復原至連線池。
  10. 在 [建立重新嘗試次數] 欄位中，鍵入建立新連線失敗之後，將進行的嘗試次數。

    將該值設定為 0 則不會重新嘗試建立連線。預設值為 0。

  11. 在 [重試間隔] 中，鍵入兩次嘗試建立連線之間的時間間隔 (以秒為單位)。

    預設值為 10。僅在 [建立重試嘗試] 的值大於 0 時使用此屬性。

  12. 選取 [啟用惰性關聯] 選項，可在對連線執行作業時，建立連線的惰性關聯。

    當作業事件完成以及元件方法結束時，這些連線便會取消關聯，從而可以重複使用實體連線。此選項預設為停用。

  13. 選取 [啟用惰性連線登記] 選項，即可僅在資源實際用於方法中時，才將資源登記至作業事件。
  14. 選取 [啟用與執行緒建立關聯] 選項，可建立連線與執行緒之間的關聯。

    當相同的執行緒需要其他連線時，便可以重複使用已與該執行緒建立關聯的連線，而省下從池中取得另一個連線的經常性耗用時間。

    此選項預設為停用。

  15. 選取 [啟用相符連線] 選項，可啟用池的連線比對功能。

    您可以在以下情況下停用此選項：管理員知道池中的連線一定是同質連線，因此不需要資源配接卡對從池中挑選的連線進行比對。此選項預設為停用。

  16. 在 [連線的最多使用次數] 欄位中，鍵入池重複使用連線的次數。

    當連線的重複使用次數達到指定的次數後，將關閉該連線。該選項非常有用，例如可以避免敘述錯誤。將該值設定為 0 可停用此屬性。預設值為 0。
