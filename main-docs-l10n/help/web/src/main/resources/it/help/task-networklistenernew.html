Per creare un listener di rete

Prima di iniziare

Quando si crea un listener di rete, è possibile creare contemporaneamente il protocollo che verrà utilizzato dal listener di rete oppure utilizzare un protocollo creato in precedenza. In genere tra un listener di rete e il protocollo associato viene stabilita una relazione uno-a-uno, ma non si tratta di un requisito obbligatorio.

  1. Espandere il nodo Configurazione nell'albero di navigazione.

  2. Nel nodo Configurazione espandere il nodo Configurazione di rete.

  3. Nel nodo Configurazione di rete selezionare il nodo Listener di rete.

    Viene visualizzata la pagina Listener di rete.

  4. Nella pagina Listener di rete fare clic su Nuovo.

    Viene visualizzata la pagina Nuovo listener di rete.

  5. Nel campo Nome digitare un nome per il listener di rete.

    Il nome di un listener di rete non può iniziare con un numero.

  6. Nel campo Protocollo specificare il protocollo da associare al listener di rete.

    • Selezionare Crea un nuovo protocollo per creare un nuovo protocollo con le impostazioni predefinite. Inoltre, dall'elenco a discesa Server virtuale, selezionare il server virtuale da associare al protocollo.

      Il nome predefinito del protocollo è il nome del listener di rete con l'aggiunta del suffisso -protocol, ma il valore è modificabile.

    • Selezionare Usa un protocollo esistente, quindi selezionare un protocollo dall'elenco a discesa. Per abilitare la sicurezza per il listener di rete, scegliere un protocollo in cui sia abilitata la sicurezza.

      Se si seleziona questa opzione, in genere il protocollo per questo listener è già stato creato.

    È possibile specificare il protocollo solo quando si crea il listener di rete. In seguito non sarà possibile modificare il protocollo associato. Il protocollo determina se la sicurezza è abilitata per il listener di rete.

  7. Selezionare la casella di controllo Abilitato dell'opzione Stato per abilitare il listener di rete.

    Questa opzione è abilitata per impostazione predefinita. Se il listener è disabilitato, qualsiasi tentativo di connessione al listener genera un'eccezione socket (java.net.ConnectException).

    Nelle versioni di GlassFish Server anteriori alla 9.1, in caso di listener con attributo Abilitato impostato su False, veniva restituito un codice di risposta 404 per ogni richiesta inviata al listener. Per ottenere questo funzionamento nella versione corrente di GlassFish Server, impostare lo stato del listener su Abilitato e lo stato di ciascun server virtuale associato su OFF. I listener associati a un server virtuale vengono elencati nell'elenco a discesa Listener di rete disponibile nel server.

  8. La casella di controllo di sola lettura Sicurezza abilitata indica se la sicurezza è abilitata per il protocollo utilizzato dal listener di rete.

    Se si sta configurando admin-listener per il server di amministrazione del dominio (DAS), che utilizza la configurazione server-config, questa opzione è di sola lettura. Per abilitare la sicurezza per questo listener, è necessario abilitare l'amministrazione sicura. Vedere Amministrazione sicura.

  9. Selezionare la casella di controllo Abilitato dell'opzione Listener JK per impostare nel listener il supporto del protocollo mod-jk di Apache.

    Il protocollo mod-jk è un protocollo di comunicazione/rete proprietario tra httpd (Web server) di Apache e un'istanza backend di &ProductName. Se il listener supporta il protocollo, tutto il contenuto statico deve essere gestito mediante httpd, mentre le richieste di risorse dinamiche (ovvero servlet e pagine JSP) vengono reindirizzate al listener di rete abilitato mod-jk di GlassFish Server tramite il protocollo mod-jk. Questa opzione è disabilitata per impostazione predefinita.

  10. Nel campo Porta digitare il numero della porta di ascolto del listener di rete.

    I valori validi sono compresi tra 1 e 65535. In un sistema UNIX, per creare socket in ascolto sulle porte comprese tra 1 e 1024 sono necessari i privilegi di utente privilegiato.

  11. Nel campo Indirizzo digitare l'indirizzo IP di ascolto per il listener di rete.

    L'indirizzo può essere specificato utilizzando la notazione per indirizzi IPv6 o con separazione mediante coppie di punti. È possibile specificare any (per INADDR_ANY) per impostare l'ascolto su tutti gli indirizzi IP. Può essere costituito da un nome host.

  12. Nell'elenco a discesa Pool di thread selezionare il pool di thread da associare al listener di rete.

    In genere si seleziona uno dei due pool di thread configurati durante l'installazione di GlassFish Server.

  13. Nell'elenco a discesa Trasporto selezionare il tipo di trasporto che dovrà essere utilizzato dal listener di rete.

    Per impostazione predefinita, l'unica scelta disponibile è tcp.

  14. Fare clic su OK.

Passi successivi

Se è stata abilitata la sicurezza per il protocollo utilizzato dal listener di rete, è possibile utilizzare la scheda SSL per modificare le impostazioni SSL del protocollo.
