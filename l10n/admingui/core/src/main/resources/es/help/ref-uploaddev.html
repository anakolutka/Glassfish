Implementación de aplicaciones y módulos empresariales

Utilice la página Implementación de aplicaciones y módulos empresariales para implementar una aplicación. La página contiene las siguientes opciones:

Tipo

El tipo de aplicación que está implementando. El único tipo de aplicación disponible en esta versión es la aplicación web (.war).

Ubicación

Ubicación del archivo de almacenamiento para la aplicación que está implementando.

Las siguientes opciones especifican desde donde se puede acceder al archivo de almacenamiento, y si es un archivo o directorio.

Archivo empaquetado que se cargará en el servidor

El archivo de almacenamiento se encuentra en un archivo que reside en la máquina cliente o al que se puede acceder desde la máquina cliente.

La máquina cliente es el host en el que está viendo la Consola de administración por medio de un navegador.

Directorio o archivo empaquetado local al que se puede acceder desde el servidor de aplicaciones

El archivo de almacenamiento es un archivo que reside en la máquina servidor o es una aplicación desempaquetada en un directorio de despliegue.

La máquina servidor es el host que ejecuta el servidor de administración de dominio del servidor de aplicaciones.

Nombre de la aplicación

Nombre de la aplicación.

Raíz de contexto

Ruta hacia la aplicación. En la URL de la aplicación, la raíz de contexto aparece justo después del número de puerto (http://host:port/context-root/... ). La raíz de contexto debe comenzar por una barra diagonal, por ejemplo, /hello.

Servidores virtuales

Servidores virtuales asociados a esta aplicación.

Estado activado

Si está opción está seleccionada, entonces la aplicación está activada. Esta opción está seleccionada de forma predeterminada.

Verificador de ejecución activado

Si está seleccionada está opción, se ejecutará una utilidad de verificación que comprueba si la aplicación cumple con JavaTM Platform, Enterprise Edition (plataforma de Java EE) y Servidor empresarial.

Precompilación de JSP activada

Si está seleccionada esta opción, los archivos JavaServer Pages ( JSPTM) se precompilarán. Si no esta seleccionada esta opción, los archivos JSP se compilarán en el tiempo de ejecución, cuando se acceda a los mismos por primera vez. Esta opción no aparece seleccionada de forma predeterminada.

Bibliotecas

Archivos JAR de biblioteca que necesitará la aplicación.

Descripción

Breve descripción de la aplicación.
