Introduction:

2-B. Modify your console-config.xml

You now need to add an integration point for plugging the exercise-2 tree node to the Administration Console. Each plugin module should have 1 console-config.xml, and you can specify as many integration point as needed in this configuration file. Since you have created this file in exercise-1, all you need to do now is to edit it and add the second integration point. You want the exercise-2 tree node to be under the JavaOne tree node. So, we specify the parentId to be 'j1root'. You can refer to treenode-ex1.jsf to see this. You also need to create the file treenode-ex2.jsf which will point to the JSF jMaki chart page.

Step-by-step instructions in NetBeans:
  1. Double click on Other Sources ->resources ->META-INF ->admingui -> console-config.xml
  2. Edit the file and add the following integration point 
    <integration-point 
	id="Another" 
	type="tree" 
	priority="230" 
	parentId="j1root" 
	content="treenode-ex2.jsf" 
/>


    Figure 2-3: Modify console-config.xml

2-D.  Build and Install your first GlassFish v3 plugin
The last step of this implementation is to build the project.

Note:  If you are using a command line and have 'maven 2' installed on your system you can build the project by doing the following:
%cd  <lab4520_root>/gfplugin/plugin;   mvn install;

or you can do this in NetBeans by following these steps.
    1. Right click on the project 'Java One 2008 Admin Console Plugin'
    2. Select Build.

After the above steps are finished, you will have the jar file named <lab4520_root>/gfplugin/myplugin/target/console-myplugin-1.0-SNAPSHOT.jar. This jar file is ready to be integrated into GlassFish.
You now need a Terminal to enter some commands.  Please bring up a terminal window.

The GlassFish V3 server is installed as <lab4520_root>/gfplugin/glassfish   It is already running.
The following steps show you how to integrate and test your plugin module.

Step-by-step instruction:
  1. go to your <lab4520_root> directory
    cd <lab4520_root>

  2. Shut down the server which was started in exercise-1
    gfplugin/glassfish/bin/stop-domain

  3. Integrate the jar to GlassFish by copying it to the glassfish modules directory
    %cp gfplugin/myplugin/target/console-myplugin-1.0-SNAPSHOT.jar gfplugin/glassfish/modules/console-myplugin-1.0-SNAPSHOT.jar

  4. Start the server again,
    gfplugin/glassfish/bin/asadmin  start-domain

  5. In the browser enter the following URL to bring up the Admin Console.
    http://localhost:8080/admingui/index.jsf

    Expand the JavaOne tree node which is displayed in the Navigation Tree, note that there is a new tree node "exercise #2" added under it. Click on the "exercise #2" tree node displays the charts.
