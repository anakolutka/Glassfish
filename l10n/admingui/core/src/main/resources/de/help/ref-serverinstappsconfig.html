Anwendungskonfiguration

Auf dieser Seite können Sie die Eigenschaften für die Anwendungskonfiguration festlegen, um sicherzustellen, dass Änderungen an bereitgestellten Anwendungen ermittelt und die geänderten Klassen neu geladen werden.

Neu laden

Wenn diese Option aktiviert ist, werden geänderte Klassen dynamisch neu geladen. Der Server prüft das Bereitstellungsverzeichnis regelmäßig auf Änderungen und stellt die Anwendung automatisch und dynamisch, einschließlich der Änderungen, erneut bereit. Diese Funktion ist besonders in einer Entwicklungsumgebung sehr nützlich, da Codeänderungen schnell getestet werden können. In einer Produktionsumgebung hingegen kann das dynamische erneute Laden zu einer Beeinträchtigung der Leistung führen. Darüber hinaus werden bei jedem erneuten Laden die Sitzungen ungültig, die zum Zeitpunkt des Ladevorgangs bestehen. Der Client muss die Sitzung neu starten.

Intervall für Neuladen des Abrufs

Gibt an, wie häufig der Server bereitgestellte Anwendungen auf Änderungen überprüfen soll.

Zeitüberschreitung Admin-Sitzung

Gibt in Minuten die Dauer der Inaktivität an, bevor Administration Console eine Zeitüberschreitung erfährt und Sie sich erneut anmelden müssen.

Automatisch bereitstellen

Wenn Sie diese Option aktivieren, werden die Anwendungen im Verzeichnis für automatische Bereitstellung automatisch bereitgestellt.

Abrufintervall für automatische Bereitstellung

Gibt an, wie häufig der Server das Verzeichnis für automatische Bereitstellung auf Anwendungs- oder Modularchive überprüfen soll. Das Ändern des Abrufintervalls hat keine Auswirkungen auf die Bereitstellungszeit von Anwendungen oder Modulen.

Zeitüberschreitung für Wiederholungen der automatischen Bereitstellung

Gibt in Sekunden an, wie lange die Größe einer teilweise kopierten Datei unverändert bleiben kann, bevor ein Fehler auftritt. Der Standard ist 30.

Wenn eine Datei langsam in ein Verzeichnis für automatische Bereitstellung kopiert wird, kann die Datei angezeigt werden, bevor die gesamte Datei kopiert ist. Folglich schlägt der Versuch, die Anwendung automatisch bereitzustellen, fehl. Wenn ein Versuch eine Anwendung automatisch bereitzustellen aus diesem Grund fehlschlägt, versucht Enterprise Server die Anwendung erneut bereitzustellen. Wenn die Größe der teilweise kopierten Datei für die angegebene Zeitdauer unverändert bleibt, tritt ein Fehler auf.

Verzeichnis für automatische Bereitstellung

Gibt das Verzeichnis an, in dem Sie Ihre Anwendung erstellen, sodass Sie die Dateien nicht in das standardmäßige Verzeichnis für automatische Bereitstellung kopieren müssen.

Verifier

Wenn Sie diese Option aktivieren, wird der Verifier vor der Bereitstellung ausgeführt. Der Verifier überprüft Struktur und Inhalt der Datei. Die Überprüfung großer Anwendungen ist häufig zeitaufwändig.

Vorkompilieren

Wenn Sie diese Option aktivieren, werden JavaServer PagesTM (JSPTM)-Dateien vorkompiliert. Wenn Sie diese Option deaktivieren, werden die JSP-Dateien beim ersten Zugriff zur Laufzeit kompiliert.
