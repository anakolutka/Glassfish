관리자 등록 정보를 구성하는 방법

세션 관리자는 세션을 만들고 삭제하는 방법, 세션 상태 저장 위치 및 사용 가능한 최대 세션 수를 구성할 수 있는 방법을 제공합니다.

  1. 탐색 트리에서 구성 노드를 선택합니다.
  2. 구성 노드 아래에서 웹 컨테이너 노드를 선택합니다.

    웹 컨테이너 페이지가 열립니다.

  3. 웹 컨테이너 페이지에서 관리자 등록 정보 탭을 누릅니다.
  4. 리프 간격 필드에 비활성 세션이 저장소에서 삭제되기 전까지의 시간(초)을 입력합니다.
  5. 최대 세션 필드에 허용된 최대 세션 수를 입력합니다.
  6. 세션 파일 이름 필드에 세션 데이터가 포함된 파일의 이름을 입력합니다.
  7. 세션 아이디 생성기 클래스 이름 필드에 고유한 세션 아이디를 생성하는데 필요한 사용자 정의 클래스를 지정합니다.

    이 클래스는 Enterprise Server 클래스 경로에 있어야 합니다. 서버 인스턴스당 하나의 세션 아이디 생성기 클래스만 허용됩니다. 세션 키 충돌을 방지하려면 클러스터의 모든 인스턴스에서 동일한 세션 아이디 생성기를 사용해야 합니다. 사용자 정의 세션 아이디 생성기 클래스는 com.sun.enterprise.util.uuid.UuidGenerator 인터페이스를 구현해야 합니다. 

    package com.sun.enterprise.util.uuid;

public interface UuidGenerator {

    public String generateUuid();
    public String generateUuid(Object obj);  //obj is the session object
}
  8. (옵션) 추가 등록 정보 섹션에서 추가 등록 정보를 지정합니다.
  9. 저장을 누릅니다.
참조

  • 세션 시간 초과 값을 설정하는 방법

  • 저장소 등록 정보를 구성하는 방법
