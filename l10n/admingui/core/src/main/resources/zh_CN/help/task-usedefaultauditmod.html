使用默认审计模块

验证日志条目包括以下信息：

  • 已尝试进行验证的用户的名称。

  • 处理过访问请求的领域。

  • 请求的 Web 模块 URI 或 EJB 组件。

  • 请求成功或失败。

无论是否启用审计日志记录，Enterprise Server 都将记录所有被拒绝的验证事件。

授权日志条目包括以下信息：

  • 经过验证的用户（如果有）的名称。

  • 请求的 Web URI 或 EJB 组件。

  • 请求成功或失败。

  1. 在导航树中，展开“配置”节点。
  2. 选择要配置的实例：
    • 要配置特定的实例，请展开该实例的配置节点。例如，对于默认实例 server，请展开 "server-config" 节点。
    • 要配置所有实例的默认设置，请展开 "default-config" 节点。
  3. 在“配置”节点下，展开“安全性”节点。
  4. 在“安全性”节点下，展开“审计模块”节点。
  5. 在“审计模块”节点下，选择 "default" 节点。

    将打开“编辑审计模块”页面。

  6. 将 auditOn 属性的值设为 true。
  7. 选择“保存”。
  8. 如果控制台中显示“必须重新启动”，请重新启动 Enterprise Server。
