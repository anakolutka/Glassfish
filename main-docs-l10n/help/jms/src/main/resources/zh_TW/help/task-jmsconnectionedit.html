編輯 JMS 連線工廠資源

  1. 在瀏覽樹狀結構中，展開「資源」節點。

  2. 在「資源」節點下，展開「JMS 資源」節點。

  3. 在「JMS 資源」節點下，選取「JMS 連線工廠」節點。

    將開啟「JMS 連線工廠」頁面。

  4. 在「JMS 連線工廠」頁面上，按一下您要編輯的連線工廠名稱。

    將開啟「編輯 JMS 連線工廠」頁面。

  5. 在「描述」欄位中，鍵入連線工廠的描述。

  6. 選取「啟用狀態」核取方塊以在執行階段啟用該資源。

    此選項預設為啟用。

  7. 在「集區的初始大小和最小大小」欄位中，指定集區中的最小連線數目。

    預設值為 8。

  8. 在「集區的最大大小」欄位中，指定集區中的最大連線數目。

    預設值為 32。

  9. 在「集區設定大小數量」欄位中，指定集區的閒置逾時過期時要移除的連線數目。

    預設值為 2。

  10. 在「閒置逾時」欄位中，指定連線可以在集區中閒置的最長時間 (以秒為單位)。

    預設值為 300。

  11. 在「最大等待時間」欄位中，指定請求連線的應用程式在連線逾時前等待的時間。

    預設值為 60000。

  12. 選取「一旦失敗便關閉所有連線」核取方塊，可讓 GlassFish Server 在任何一個連線失敗時，關閉集區中的所有連線並重新建立連線。

    如果未選取此核取方塊，則僅在使用個別連線時才會重新建立這些連線。

  13. 針對 JMS 連線工廠資源，指定「作業事件支援」值，如下所示：

    • 為可用於涉及使用作業事件範圍內多個資源的作業事件的資源指定 XATransaction (預設值)。

      例如，該資源加上 JDBC 資源、連線器資源或其他 JMS 連線工廠資源。該值具有最大的靈活性。組態為 XATransaction 的資源將參與兩階段的確認作業。

    • 為可用於僅涉及使用作業事件範圍內一個資源的作業事件的資源，或用做涉及多個 XA 資源的分散式作業事件中最後一個代理程式的資源指定 LocalTransaction。

      使用該值可以獲得更好的效能。組態為 LocalTransaction 的資源不會被用於分為兩個階段的確認作業。

    • 為永不參與作業事件的資源指定 NoTransaction。

      該設定在 JMS 應用程式中的使用有限。

  14. 選取「連線驗證」核取方塊以啟用連線驗證。

    此選項預設為停用。

  15. 在「其他特性」表格中，指定連線工廠的其他特性。

    如需可用特性的相關資訊，請參閱JMS 連線工廠特定的特性。

    • 若要增加特性，請按一下「增加特性」按鈕。在出現的空白列中，於「名稱」欄位中鍵入特性名稱，然後在「值」欄位中鍵入特性值。

    • 若要修改特性，請編輯特性的「值」欄位。

    • 若要刪除特性，請選取您要刪除之特性「名稱」欄位左側的核取方塊，然後按一下「刪除特性」按鈕。

  16. 或者，按一下「目標」標籤以移至「JMS 連線工廠資源目標」頁面。在此頁面中，執行下列動作：

    1. 按一下「管理目標」以開啟「管理資源目標」頁面。

      在此頁面的「可用」欄與「已選取」欄之間移動目標。確定已在「已選取」欄中放置將建置使用資源之應用程式的一或多個目標。可用的目標包含可用的叢集與伺服器執行處理，以及預設的伺服器執行處理 server。按一下「確定」儲存變更。

    2. 選取目標的核取方塊，然後按一下「啟用」或「停用」以啟用或停用該目標的資源。

  17. 按一下「儲存」。

    提示:

    若要回到預設設定，請按一下「載入預設值」按鈕，然後按一下「儲存」。
