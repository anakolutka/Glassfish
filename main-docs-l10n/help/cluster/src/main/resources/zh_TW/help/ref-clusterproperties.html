叢集特性

「叢集特性」頁面會顯示為所選取叢集設定的特性清單。這些特性可增加有關叢集的選擇性組態資訊。

「叢集名稱」欄位是唯讀欄位，會顯示所選取叢集的名稱。

對於每個特性，都會顯示以下資訊：

名稱

特性的名稱。

值

為所選取執行處理設定的特性值。

描述

提供有關特性之更多資訊的文字描述。

其他特性表也包含下列選項。

增加特性

增加特性的按鈕。按一下此按鈕即可在其他特性表中增加一列。

刪除特性

刪除一或多個所選取特性的按鈕。刪除的任何特性都會復原成其預設值或未定義 (如果未設定任何預設值)。

GlassFish Server 會定義下列叢集特性：

GMS_DISCOVERY_URI_LIST

叢集中的 GlassFish Server 執行處理位置，用於探索叢集。只有當群組管理服務 (GMS) 未使用多重播送來廣播訊息時，才需要此特性。

此特性的有效值如下：

  • 統一資源識別碼 (URI) 的逗號分隔清單。 每個 URI 必須定位一個 GlassFish Server 執行處理或 DAS。如果在相同主機上執行多個 GlassFish Server 執行處理，則需要此格式。

    清單中每個 URI 的格式如下：

    scheme://host-name-or -IP-address:port

    • scheme 為 URI 方案，它為 tcp。

    • host-name-or -IP-address 為執行處理所在主機的主機名稱或 IP 位址。

    • port 為執行處理監聽 GMS 訊息所用連接埠的連接埠號碼。必須為執行處理設定系統特性 GMS_LISTENER_PORT-clustername。如需如何設定執行處理的這個系統特性的資訊，請參閱 Oracle GlassFish Server High Availability Administration Guide 中的 Discovering a Cluster When Multicast Transport Is Unavailable。

  • 執行 DAS 或執行處理所在的 IP 位址或主機名稱的逗號分隔清單。此清單可以同時包含 IP 位址和主機名稱。叢集的執行處理正在每個主機上執行時，才能使用此格式。GMS_LISTENER_PORT 特性值在網域的每個叢集中必須是唯一的。

  • 關鍵字 generate。唯有每部主機上均有一個執行中的叢集執行處理，且 DAS 在另一部主機上執行時，才可使用此格式。相同主機上的多個執行處理不能是相同叢集的成員。GMS_LISTENER_PORT 特性值在網域的每個叢集中必須是唯一的。

GMS_LISTENER_PORT

叢集監聽來自 GMS 訊息所用之連接埠的連接埠號碼。

預設值為對 GMS_LISTENER_PORT-cluster-name 系統特性的參照。依預設，不會設定此系統特性。在此情況下，GMS 會從特性 GMS_TCPSTARTPORT 與 GMS_TCPENDPORT 定義的範圍內選取可用的連接埠。依預設，此範圍是 9090-9200。大多數情況下，使用預設值即可。

不過，如果 GMS 未使用多重播送來廣播訊息，此特性必須指定對於叢集中所有 GlassFish Server 執行處理皆有效的連接埠號碼。若要使用預設值來達成此需求，請使用系統特性為每個執行處理個別設定連接埠號碼。

例如，使用 叢集系統特性 頁面可建立 DAS 的系統特性 GMS_LISTENER_PORT-cluster-name。之後，針對叢集中的每個執行處理，將 GMS_LISTENER_PORT-cluster-name 系統特性設為執行處理監聽 GMS 訊息的連接埠號碼。叢集的 GMS_LISTENER_PORT 特性預設值會參照此系統特性。

GMS_LOOPBACK

指定執行處理是否會收到其本身的應用程式層級訊息 (由執行處理廣播到叢集)。

可能值如下：

false

執行處理不會收到其本身的訊息 (預設)。

true

執行處理可以收到其本身的訊息。使用此設定，可在執行處理是叢集中的唯一執行處理時測試該執行處理。

GMS_MULTICAST_TIME_TO_LIVE

在捨棄訊息之前，下列事件類型的多重播送訊息可經歷的重複運算或傳輸數目上限：

  • 群組探索

  • 成員活動訊號

  • 成員身份變更

為了符合建置 DAS 與叢集執行處理所在之網路的組態，請儘可能將此值設低。若要判斷系統可行的最低值，請使用 validate-multicast(1) 子命令。

值為 0 時可確保多重播送訊息絕對不會離開廣播它們的主機。

值為 1 時可避免在透過交換器或路由器連線之相同子網路上的主機之間廣播訊息。

預設值為 4，可確保在主機是透過交換器或路由器連線的網路中，將訊息成功廣播到所有叢集成員。

GMS_TCPENDPORT

如果未設定 GMS_LISTENER_PORT-cluster-name 系統特性，則會是連接埠範圍中 GMS 可選取的最大連接埠號碼。預設值為 9200。

GMS_TCPSTARTPORT

如果未設定 GMS_LISTENER_PORT-cluster-name 系統特性，則會是連接埠範圍中 GMS 可選取的最小連接埠號碼。預設值為 9090。
