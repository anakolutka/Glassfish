配置批处理运行时

  1. 在导航树中，选择要查看批处理作业执行的 GlassFish Server 集群或服务器实例。

    将打开“一般信息”页。

  2. 选择“批处理”选项卡。

    此时将打开“批处理作业执行”页。

  3. 单击 "Configuration"（配置）标签。

    此时将打开“批处理运行时配置”页。运行时使用托管执行程序服务和数据源来执行批处理作业。

  4. 从“执行程序服务查找名称”下拉列表中，选择托管执行程序服务，以用于将线程提供给批处理作业。

    默认托管执行程序服务为 concurrent/__defaultManagedExecutorService。在批处理作业已提交到批处理运行时之后，可以更改托管执行程序服务而不会影响作业的执行。

  5. 从“数据源查找名称”下拉列表中，选择用于存储作业信息的数据源。

    默认数据源为 jdbc/__TimerPool。在第一个批处理作业已提交到批处理运行时以供执行后，请勿更改数据源。如果必须更改数据源，请停止并重新启动域，然后在任何作业启动或重新启动之前进行更改。但是，一旦更改了数据源，存储在以前数据源中的信息将不可访问。

  6. 单击“保存”。
