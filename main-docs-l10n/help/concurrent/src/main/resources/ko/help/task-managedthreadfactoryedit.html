관리 대상 스레드 팩토리를 편집하는 방법

  1. 탐색 트리에서 [리소스] 노드를 확장합니다.

  2. [리소스] 노드에서 [동시 리소스] 노드를 확장합니다.

  3. [동시 리소스] 노드에서 [관리 대상 스레드 팩토리]를 선택합니다.

    [관리 대상 스레드 팩토리] 페이지가 열립니다.

  4. [관리 대상 스레드 팩토리] 페이지에서 편집할 리소스의 이름을 누릅니다.

    [관리 대상 스레드 팩토리 편집] 페이지가 열립니다.

    주:

    [JNDI 이름]은 읽기 전용 필드입니다. 새 리소스를 생성할 때만 JNDI 이름을 지정할 수 있습니다. [논리적 JNDI 이름]도 읽기 전용 필드입니다.

  5. 컨텍스트를 스레드에 전달하려면 [컨텍스트 정보 사용] 체크 박스를 선택하고 전달할 컨텍스트를 선택합니다.

    기본적으로 이 옵션은 사용으로 설정되고 모든 컨텍스트가 선택되어 있습니다. 컨텍스트를 여러 개 선택하려면 Shift 또는 Ctrl 키를 누른 채로 누르십시오. 이 옵션을 사용 안함으로 설정하면 컨텍스트가 선택된 경우에도 전달되지 않습니다.

  6. 리소스를 사용으로 설정하려면 [상태 사용] 체크 박스를 선택합니다.

    이 옵션은 기본적으로 사용으로 설정되어 있습니다.

  7. [스레드 우선순위] 필드에 생성된 스레드에 지정할 우선순위를 입력합니다.

    기본값은 5입니다.

  8. [배치 순서] 필드에 리소스의 배치 순서를 지정하는 숫자를 입력합니다.

    서버 시작 시 낮은 번호의 리소스가 먼저 로드됩니다. 기본값은 100입니다.

  9. [설명] 필드에 리소스에 대한 간단한 설명을 입력합니다.

    이 설명은 최대 250자가 포함될 수 있는 문자열 값입니다.

  10. [추가 속성] 섹션에서 속성을 추가하지 마십시오.

    GlassFish Server에서는 이 리소스 유형에 대한 추가 속성을 정의하지 않습니다. 또한 현재 이 리소스 유형은 추가 속성을 사용하지 않습니다.

  11. [저장]을 누릅니다.
