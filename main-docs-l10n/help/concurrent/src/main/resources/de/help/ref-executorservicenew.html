Neuer verwalteter ExecutorService

Auf der Seite "Neuer verwalteter ExecutorService" können Sie eine verwaltete ExecutorService-Ressource erstellen.

Die Seite "Neuer verwalteter ExecutorService" enthält die folgenden Optionen.

JNDI-Name

Ein eindeutiger JNDI-Name, der die Ressource identifiziert. Konventionsgemäß beginnt der Name mit einem Indikator des Ressourcentyps gefolgt von einem Schrägstrich. Beispiel: concurrent/, jdbc/ oder jms/.

Kontextinformation

Die an Threads zu propagierenden Kontexte: Class Loader, JNDI, Sicherheit oder WorkArea. Die Kontextpropagierung kann aktiviert oder deaktiviert werden. Ist die Option aktiviert, werden die gewählten Kontexte propagiert. Ist die Option deaktiviert, wird keiner der Kontexte propagiert, auch wenn sie ausgewählt sind. Standardmäßig ist die Option aktiviert, und alle vier Kontexte sind gewählt.

Status

Der Status der Ressource. Die Ressource kann aktiviert oder deaktiviert sein. Der Standardwert ist "Aktiviert".

Threadpriorität

Die Priorität, die erstellten Threads zugewiesen werden soll. Der Standardwert ist 5.

Tasks mit langer Ausführungszeit

Legt fest, ob die Ressource für Tasks mit langer Ausführungszeit verwendet werden soll. Ist diese Option aktiviert, werden Tasks mit langer Ausführungszeit nicht als aufgehängt gemeldet. Diese Option ist standardmäßig deaktiviert.

Blockiert nach

Die Anzahl Sekunden, die Tasks ausgeführt werden können, bevor sie als nicht reagierend eingestuft werden. Der Standardwert ist 0.

Beschreibung

Eine Beschreibung dieser Ressource.

Core-Größe

Die Anzahl von Threads, die in einem Threadpool beibehalten werden sollen. Der Standardwert ist 0.

Maximale Poolgröße

Die maximale Anzahl von Threads, die in einem Threadpool enthalten sein können. Der Standardwert ist 2147483647.

Keep-Alive

Die Anzahl Sekunden, die Threads inaktiv bleiben können, wenn die Anzahl Threads größer ist als die Kerngröße. Der Standardwert ist 60.

Threadgültigkeitsdauer

Die Anzahl Sekunden, die Threads in einem Threadpool bleiben dürfen, bevor sie gelöscht werden, unabhängig davon, ob die Anzahl Threads größer ist als die Kerngröße oder ob die Threads inaktiv sind. Der Standardwert ist 0.

Taskqueuekapazität

Die Anzahl weitergeleiteter Tasks, die in der auf Ausführung wartenden Taskqueue gespeichert werden können. Der Standardwert ist 2147483647.

Weitere Eigenschaften

Weitere Eigenschaften für die externe Ressource. In GlassFish Server werden keine weiteren Eigenschaften für diesen Ressourcentyp definiert. Darüber hinaus verwendet dieser Ressourcentyp derzeit keine weiteren Eigenschaften.

Ziele

(Diese Option wird angezeigt, wenn in der Domain Cluster oder Standalone-Instanzen erstellt wurden.)

Cluster und Standalone-Instanzen für die Ressource. Mithilfe der Schaltflächen "Hinzufügen" und "Alle hinzufügen" verschieben Sie die benötigten Ziele in die Spalte "Ausgewählte Ziele". Verschieben Sie die nicht benötigten Ziele in die Spalte "Verfügbare Ziele" mithilfe der Schaltflächen "Entfernen" und "Alle entfernen".
