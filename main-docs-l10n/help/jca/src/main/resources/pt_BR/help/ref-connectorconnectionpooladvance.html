Editar Atributos Avançados do Pool de Conexões do Conector

Use a página Editar Pool de Conexões do Conector para modificar as definições avançadas do pool de conexões do conector.

A página Editar Atributos Avançados do Pool de Conexões do Conector contém as opções a seguir.

Carregar Defaults

Botão para restaurar definições para seus valores default. Definições que não tiveram seus valores default alterados.

Validar no Máximo uma Vez

Especifica o intervalo de tempo, em segundos, entre sucessivas solicitações para validar uma conexão no máximo uma vez. O valor default é 0, o que significa que o atributo não está ativado.

Time-out para Perda de Conexão

Período em segundos para rastrear as perdas de conexão em um pool de conexões. Um valor de 0 indica que o rastreamento da perda de conexão está desativado. O valor default é 0.

Reclamação de Perda

Se esta opção está selecionada, as conexões perdidas serão restauradas no pool após a conclusão do rastreamento de perda da conexão. Por default, essa opção está desativada.

Novas Tentativas de Criação

O número de tentativas a serem feitas se houver uma falha na criação de uma nova conexão. O valor de 0 indica que nenhuma tentativa será feita para criar a conexão novamente. O valor default é 0.

Intervalo para Nova Tentativa

O intervalo de tempo, em segundos, entre duas tentativas de criar uma conexão. Esse atributo só será usado se o valor Novas Tentativas de Criação for maior que 0. O valor default é 10 segundos.

Agrupamento

Permite agrupamento de conexões para o pool. Por default, a opção está ativada.

Associação Lenta

Se esta opção está selecionada, as conexões serão associadas lentamente quando uma operação for executada nelas. As conexões serão desassociadas quando a transação for concluída e um método de componente for finalizado, o que ajuda a reutilizar as conexões físicas. Por default, essa opção está desativada.

Alocação para Conexão Lenta

Se esta opção estiver selecionada, um recurso para a transação será alocada somente quando a transação for realmente usada em um método. Por default, essa opção está desativada.

Associar a Thread

Se esta opção for selecionada, uma conexão será associada a um thread. Quando o mesmo thread for necessário a uma conexão, a conexão já associada ao thread poderá ser reutilizada, não incorrendo, portanto, na sobrecarga para obtenção de outra conexão do pool. Por default, essa opção está desativada.

Correspondência de Conexões

Se esta opção for selecionada, a correspondência de conexões será ativada para o pool. Essa opção poderá ser desativada se você souber que as conexões no pool serão sempre homogêneas e, portanto, uma conexão selecionada no pool não precisará coincidir com o adaptador de recurso. Por default, a opção está ativada.

Uso da Correspondência de Conexões

O número de vezes que uma conexão deve ser reutilizada pelo pool. Depois que uma conexão é reutilizada por um número especificado de vezes, ela será fechada. O valor de 0 indica que o atributo está desativado. O valor default é 0.
