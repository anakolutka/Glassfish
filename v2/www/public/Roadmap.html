Following is the roadmap for project GlassFish.  Note that there are now 2 releases of GlassFish.  V1_UR1 (update release) and V2.  V1_UR1 is developed from the FCS branch and is a bug fix release.    V2 is the next dot release of GlassFish and is still in the planning phase so the roadmap is tentative and subject to change.

Tentative GlassFish V1_UR1 Roadmap:

Goals for the update release are:
  • Address important issues affecting co-bundling in NetBeans and Creator
  • Address Customer escalations. 
  • Address important release noted  bugs that do not have any workaround.
  • Address critical bugs that came in after the final build. 
  • Address bugs that will generate unnecessary support calls,
         email or forum posts. For example, incorrect error messages or exception
         traces logged at an inappropriate log level.

Bugs that are under consideration for inclusion into V1_UR1 are posted on this following page (https://glassfish.dev.java.net/public/GlassFish_9.0_UR1_Potential_Bugs.html)  The bugs are review by the project lead and updated with a comment if they will be included in the release.

Per process requirments, all bugs must be fixed in the main branch and then merged into the FCS branch. 

Tentative Roadmap for GlassFish V2

Since we are still in the very early planning stages we will provide the information that was discussed at JavaOne.  Features that are under consideration for the V2 release as of JavaOne include
  • Load balancing support
  • Clustering support
  • Some scripting support
  • Performance
  • Tango support
  • Unified Test Framework
Process changes and update are taking place now to include community feedback and we hope that by mid July those processes can be put in place.  Work for the V2 release will be done on the main branch of the workspace.

Target release date for V2 is early 2007 but that is subject to change since the contents has yet to be defined.
