Configuração de Segurança de Nova Mensagem

Use a página Configuração de Segurança de Nova Mensagem para criar uma configuração HttpServlet de segurança de mensagem.

A página Configuração de Segurança de Nova Mensagem contém as seguintes opções.

Nome da Configuração

O nome da configuração à qual as definições nesta página se aplicam. Este campo é somente para leitura.

Camada de Autenticação

Especifica a camada de mensagens na qual a autenticação é realizada. O valor deve ser SOAP ou HttpServlet. Como uma configuração SOAP está disponível por default, a única opção é HttpServlet.

ID do Provedor

O identificador exclusivo da configuração do provedor desta configuração de segurança de mensagem.

Provedor Default

Se esta opção for selecionada, este provedor será o provedor default. O provedor default será chamado para qualquer aplicação que não identifique um provedor específico. Se o provedor for o provedor de cliente default, o provedor de servidor default, ou de ambos, é determinado pelo Tipo de Provedor.

Tipo de Provedor

Especifica o tipo do provedor. As opções disponíveis são:

client

Especifica que o provedor é o provedor de autenticação do cliente.

server

Especifica que o provedor é o provedor de autenticação do servidor.

client-server

Especifica que o provedor é um provedor de autenticação do cliente e do servidor.

Nome da Classe

O nome da classe de implementação Java do provedor. Provedores de autenticação do cliente devem implementar a interface com.sun.enterprise.security.jauth.ClientAuthModule. Provedores do servidor devem implementar a interface com.sun.enterprise.security.jauth.ServerAuthModule. Os provedores de cliente-servidor devem implementar as duas interfaces.
