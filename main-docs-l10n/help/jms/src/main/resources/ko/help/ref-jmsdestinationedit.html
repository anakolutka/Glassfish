JMS 대상 리소스 편집

[JMS 대상 리소스 편집] 페이지에서는 JMS 대상 리소스에 대한 설정을 수정할 수 있습니다.

[JMS 대상 리소스 편집] 페이지에는 다음 옵션이 포함되어 있습니다.

기본값 로드

기본값을 사용하는 설정을 기본값으로 복원하는 단추입니다. 기본값을 사용하지 않는 설정은 변경되지 않습니다.

JNDI 이름

대상 리소스를 식별하는 고유한 이름입니다. [JNDI 이름]은 읽기 전용 필드입니다. 새 JMS 대상 리소스를 생성할 때만 JNDI 이름을 지정할 수 있습니다.

물리적 대상 이름

이 리소스와 연관된 물리적 대상의 이름입니다.

GlassFish Server에서는 필요할 때 자동으로 물리적 대상을 생성하고 사용자가 대상 리소스를 삭제할 때 물리적 대상을 삭제합니다.

리소스 유형

대상 리소스의 유형입니다. 사용 가능한 선택 항목은 javax.jms.Topic 및 javax.jms.Queue입니다.

배치 순서

대상 리소스의 배치 순서입니다.

설명

대상 리소스에 대한 설명입니다.

상태

런타임에 리소스를 사용할 수 있는지 여부입니다.

기본 서버 인스턴스인 server가 도메인에 존재하는 경우에만 리소스의 런타임 가용성을 사용 또는 사용 안함으로 설정할 수 있습니다. 런타임 가용성은 기본적으로 사용으로 설정됩니다.

도메인에 클러스터 또는 독립형 인스턴스가 생성된 경우 [대상] 탭을 사용하여 해당 대상에서 리소스의 런타임 가용성을 제어할 수 있습니다.

추가 속성

대상 리소스에 대한 추가 속성입니다. GlassFish Server에서는 JMS 대상 리소스에 대한 추가 속성을 정의하지 않습니다.

관련 작업

  • JMS 대상 리소스를 편집하는 방법

  • JMS 대상 리소스를 삭제하는 방법
