批次執行階段組態

GlassFish Server 提供批次工作排程與執行的批次執行階段。批次工作通常是長時間執行，且包含一系列步驟的大量執行導向工作。批次應用程式會提交工作給批次執行階段，並提供步驟執行方式與步驟執行時間的指示。批次執行階段會處理步驟，並將與工作有關的資訊儲存在工作儲存區域中。在 GlassFish Server 中，工作儲存區域就是資料庫。

您可以使用「批次執行階段組態」頁面檢視及變更批次執行階段的組態。批次執行階段會使用受管理執行程式服務與資料來源來執行批次工作。受管理執行程式服務會提供繫線給工作，資料來源則會儲存工作資訊。批次執行階段組態資料會儲存在 domain.xml 內的 config 元素中。

「批次執行階段組態」頁面包含以下資訊。

執行處理名稱
叢集名稱

要套用組態之 GlassFish 叢集或伺服器執行處理的名稱。此為唯讀欄位。

執行程式服務查詢名稱

提供繫線給批次工作的受管理執行程式服務 JNDI 查詢名稱。預設受管理執行程式服務為 concurrent/__defaultManagedExecutorService。

受管理執行程式服務可以在批次工作已送出至批次執行階段之後變更，且不會影響工作的執行。

資料來源查詢名稱

儲存目前與過去批次工作相關資訊的資料來源 JNDI 查詢名稱。預設的資料來源為 jdbc/__TimerPool。

當第一個批次工作已送出至執行的批次執行階段之後，請勿變更資料來源。如果必須變更資料來源，請停止並重新啟動網域，然後於啟動或重新啟動任何工作之前先進行變更。但是，一旦變更了資料來源，就無法繼續存取儲存於先前資料來源中的資訊。
