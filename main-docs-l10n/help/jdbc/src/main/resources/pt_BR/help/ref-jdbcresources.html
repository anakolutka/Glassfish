Recursos JDBC

Um recurso de Java DataBase Connectivity (fonte de dados) fornece aplicações com todos os meios para conectar-se a um banco de dados. Em geral, o administrador cria um recurso JDBC para cada banco de dados acessado pelas aplicações implantadas em um domínio; entretanto, é possível criar mais de um recurso JDBC para um banco de dados.

As aplicações obtêm uma conexão do banco de dados de um pool de conexões, pesquisando por uma fonte de dados na árvore API de Java Naming and Directory Interface (JNDI) e solicitando uma conexão. O pool de conexões associado à fonte de dados provê a conexão com a aplicação.

Use a página Recursos JDBC para configurar os recursos JDBC.

Para cada recurso, as informações a seguir são fornecidas.

Nome JNDI

Um nome exclusivo que identifica o recurso JDBC.

Nome Lógico JNDI

O nome lógico JNDI para o recurso. Consulte Nomes Lógicos JNDI para obter mais informações.

Ativado

(Esta coluna só será exibida se a instância do servidor default, server, existir no domínio.)

Exibe uma marca de seleção se o recurso estiver ativado ou um X se o recurso estiver desativado.

Status

(Esta coluna só será exibida se os clusters ou instâncias standalone tiverem sido criadas no domínio.)

O número de clusters e instâncias independentes ao qual o recurso está direcionado e em quantos desses o recurso está ativado. Por exemplo, "Ativado em 2 de 4 Alvos" significa que o recurso está direcionado para quatro clusters e instâncias standalone e que está ativado em dois desses quatro alvos.

Pool de Conexões

O pool de conexões JDBC associado ao recurso.

Descrição

Uma breve descrição do recurso.

A tabela Recursos também contém as opções a seguir.

Novo

Botão utilizado para criar um novo recurso.

Deletar

Botão utilizado para deletar um ou mais recursos selecionados.

Ativar

Botão utilizado para ativar um ou mais recursos selecionados.

Desativar

Botão utilizado para desativar um ou mais recursos selecionados.
