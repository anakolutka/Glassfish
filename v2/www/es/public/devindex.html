Si como desarrollador o desarrolladora le interesa formar parte de la comunidad que implementó el primer servidor de aplicaciones Java EE5 compatible, el Proyecto GlassFish es el lugar perfecto para usted.

Animamos a los miembros de la comunidad GlassFish a registrarse en una Página de la gente de nuestra Wiki. También tenemos una página con un aspecto algo mejor, que usamos para detectar desarrolladores y que tratamos de mantener en expansión.

Entender el código

Desarrollar un servidor de aplicaciones es un gran desafío. Más abajo figuran los vínculos que nos ayudan a organizar el desarrollo de un proyecto de este tamaño.

  • El proyecto GlassFish se divide en módulos
  • Leer javadocs que se actualizan con cada compilación aprobada.
  • Charlar con otros desarrolladores de GlassFish sobre los detalles de la implementación en nuestras listas de correo. No olvide iniciar sesión para ver todas las listas disponibles.
  • Comprobar la lista de los propietarios de módulos
  • Consultar las tecnologías relacionadas que forman parte de GlassFish

Crearlo

La infraestructura de la compilación de GlassFish emplea maven y ant. Actualmente estamos buscando desarrolladores que ayuden a mejorar la infraestructura de la compilación para que dependa menos de ant.

  • La guía de procedimientos iniciales contiene la lista resumida de pasos necesarios para crear el servidor
  • Las instrucciones completas detallan cómo crear GlassFish

Comprobarlo

Hay varias comprobaciones diferentes disponibles en GlassFish. Quicklook comprueba la ejecución en un corto plazo de tiempo y cubre una amplia gama de funciones en el servidor de aplicaciones. Las pruebas de desarrollador también están disponibles para los diferentes módulos. Cada módulo tiene más información sobre el método de ejecución de las pruebas.

Si desea más información, consulte el Portal de calidad GlassFish.

  • Descargar una compilación nocturna o aprobada
  • Ejecutar las pruebas QuickLook
  • Unirse al programa CATfish
  • Enviar un error

Aportar código

Las aportaciones al proyecto GlassFish pueden hacerse de varias formas: solucionando errores, enviando mejoras, nuevas funciones o, incluso, módulos completos.

  • Los que desean hacer aportaciones deben firmar y remitir un Acuerdo de aportación
  • Consulte la directiva rectora para obtener más información sobre misiones y responsabilidades.
  • Todo el código enviado al espacio de trabajo debe seguir las convenciones de codificación
  • Si aún no tiene la función de desarrollador, consulte el procedimiento para enviar un parche
  • Los desarrolladores deben seguir procedimientos de confirmación al actualizar el espacio de trabajo
