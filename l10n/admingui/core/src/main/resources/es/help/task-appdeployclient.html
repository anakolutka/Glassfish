Implementar un cliente de aplicaciones

  1. En el árbol de navegación, seleccione el nodo Aplicaciones.

    Se abrirá la página Aplicaciones

  2. En la tabla Aplicaciones implementadas, haga clic en el botón Implementar.

    Se abrirá la página Implementar aplicaciones o Módulos.

  3. Especifique la ubicación del archivo de almacenamiento donde se encuentra la aplicación que se va a implementar.
    • Si el archivo de almacenamiento es un archivo que reside en la máquina cliente o al que se puede acceder desde la máquina cliente, seleccione la opción Archivo empaquetado que se cargará en el servidor.

      La máquina cliente es el host en el que está viendo la Consola de administración por medio de un navegador.

      Haga clic en Examinar para buscar el archivo o bien escriba la ruta completa hasta el archivo.

    • Si el archivo de almacenamiento es un archivo que reside en el equipo servidor o es una aplicación descomprimida desde un directorio desplegado, seleccione la opción Archivo empaquetado local o Directorio accesible desde Servidor empresarial.

      El equipo servidor es el host que se está ejecutando en el servidor de administración del dominio (DAS).

      Haz clic en Examinar archivos para buscar un archivo, o en Examinar carpetas para buscar un directorio. Como opción alternativa, escriba el nombre de la ruta completa del archivo o directorio. Implementar desde un directorio desplegado es para desarrolladores avanzados y no es recomendable para entornos de producción.

  4. En la lista desplegable Tipo, seleccione Cliente de aplicaciones.

    Si ha especificado un Archivo empaquetado para ser cargado en el servidor o un Archivo empaquetado local accesible desde Servidor empresarial,el tipo de aplicación se detectará automáticamente. Si ha especificado un Directorio accesible desde Servidor empresarial, debe especificar el tipo de manera manual.

  5. En el campo Nombre de la aplicación, puede mantener el nombre predeterminado, que es el nombre del directorio o el prefijo del nombre del archivo, o bien escribir un nombre distinto.

    El nombre predeterminado aparece si decide cargar un archivo. El nombre de la aplicación debe ser único.

  6. Marque la casilla de verificación Ejecutar verificador para ejecutar el verificador antes de la implementación.

    Si esta opción está seleccionada y se produce un error en la verificación, la implementación no se llevará a cabo. El verificador examinará la estructura y contenido del archivo. La verificación de aplicaciones de gran tamaño tiende a llevar mucho tiempo. Esta opción se encuentra deshabilitada de forma predeterminada.

  7. Seleccione la casilla de verificación Java Web Start para habilitar el acceso a Java Web Start para un módulo de cliente de aplicaciones.

    Esta opción se encuentra deshabilitada de forma predeterminada.

  8. Seleccione la casilla de verificación Forzar reimplementación para forzar la reimplementación si ya se ha implementado la aplicación.

    Si esta opción no está seleccionada y se intenta implementar una aplicación que ya se haya implementado, se producirá un error. Esta opción se encuentra deshabilitada de forma predeterminada.

  9. En el campo Descripción, introduzca una descripción de esta aplicación
  10. Haga clic en Aceptar.
