To Configure Health Checking

  1. In the tree component, select the HTTP Load Balancers node.

  2. In the right pane, click the load balancer to edit.

  3. Click the Targets tab.

    The HTTP Load Balancer Targets page appears.

  4. In the Health Check URL field, enter the URL to ping to determine whether the standalone instance or instances in the cluster are healthy.

  5. In the Health Timeout (seconds), enter the interval in seconds the health checker waits to receive a response from an instance.

    If the health checker has not received a response in this interval, the instance is considered unhealthy.

  6. The Instance Status field shows how many instances a cluster has, whether they are running, and whether they require a restart. If the target is a stand-alone instance, it shows the status of the instance.
