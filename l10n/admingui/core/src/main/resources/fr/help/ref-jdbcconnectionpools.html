Pools de connexion

Un pool de connexions JDBC contient un groupe de connexions JDBC créées lors de son enregistrement. Les pools de connexions utilisent un pilote JDBC pour créer des connexions physiques à la base de données. Votre application emprunte une connexion au pool, l'utilise, puis la renvoie à ce dernier en la fermant. Pour créer une ressource JDBC, indiquez le pool de connexions associé. Plusieurs ressources JDBC peuvent spécifier un seul pool de connexions.

Un pool de connexions JDBC est un groupe de connexions réutilisables pour une base de données particulière. La création de chaque nouvelle connexion physique exigeant du temps, le serveur conserve un pool de connexions disponibles pour accroître les performances. Lorsqu'une application demande une connexion, elle en obtient une du pool. Lorsqu'une application met fin à une connexion, cette dernière est replacée dans le pool.

Lors de la création d'un pool de connexions à l'aide de la Console d’administration, vous définissez les aspects d'une connexion à une base de données spécifique.
