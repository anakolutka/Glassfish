Cómo establecer la configuración general de JVM

La máquina virtual de Java (JVM) viene incluida en la plataforma Java, Standard Edition (plataforma Java SE), que necesita Servidor empresarial. La introducción de valores incorrectos para la configuración de JVM hará que no funcione el servidor, por lo que tiene que ir con cuidado a la hora de cambiar estos valores.

  1. En el árbol de navegación, seleccione el nodo Application Server.
  2. En la página Información general, haga clic en la ficha Configuración de JVM.

    Se abrirá la página Configuración general de JVM.

  3. En el campo Inicio de Java, introduzca el nombre del directorio de instalación del software de Java SE.

    Servidor empresarial está basado en la plataforma Java SE. Para comprobar que la versión del software de Java SE que ha especificado es compatible con esta versión, consulte Sun GlassFish Enterprise Server v3 Prelude Release Notes . Si introduce un nombre de directorio no existente o un directorio para una versión no compatible del software de Java SE, entonces Servidor empresarial no se podrá iniciar.

  4. En el campo Opciones de Javac, introduzca las opciones de línea de comandos del compilador del lenguaje de programación Java.

    Servidor empresarial ejecuta el compilador cuando se implementan los componentes de Enterprise JavaBeansTM (EJBTM).

  5. Seleccione la casilla de verificación Depuración habilitada para habilitar la depuración con Java Platform Debugger Architecture (JPDA).

    JPDA lo utilizan los desarrolladores de aplicaciones. Para obtener más información, consulte Sun GlassFish Enterprise Server v3 Prelude Developer’s Guide .

  6. En el campo Opciones de depuración, introduzca las opciones de JPDA que van a pasar a la JVM.

    Este campo sólo será aplicable si se ha habilitado la depuración en el paso anterior.

  7. En el campo Opciones de compilación RMI, introduzca las opciones de línea de comandos del compilador rmic.

    Servidor empresarial ejecuta el compilador rmic cuando se implementan componentes EJB.

  8. En el campo Preprocesador de código byte, introduzca una lista de nombres de clase.

    Utilice comas para separar los nombres de clase. Todas las clases deben implementar com.sun.appserv.Interfaz del preprocesador de código byte. Las clases se llaman en el orden especificado.

  9. Haga clic en "Guardar".
  10. Reinicie el servidor.
