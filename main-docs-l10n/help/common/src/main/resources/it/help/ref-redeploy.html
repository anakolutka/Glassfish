Ridistribuisci applicazioni o moduli

Utilizzare la pagina Ridistribuisci applicazioni o moduli per ridistribuire un'applicazione distribuita in precedenza.

La pagina Ridistribuisci applicazioni o moduli contiene le opzioni riportate di seguito.

Posizione

La posizione del nuovo archivio per l'applicazione da ridistribuire.

Le opzioni riportate di seguito consentono di specificare la posizione da cui è accessibile l'archivio e se l'archivio è un file o una directory.

File incluso in un package da caricare sul server

L'archivio è un file che risiede sul computer client o è accessibile da tale computer.

Il computer client è l'host sul quale viene visualizzata la Console di amministrazione tramite un browser.

Directory o file locale incluso in un package accessibile da GlassFish Server

L'archivio è un file che risiede sul computer server o è un'applicazione decompressa in una directory esplosa.

Il computer server è l'host sul quale è in esecuzione il server di amministrazione del dominio di GlassFish Server.

Nome applicazione

Campo di sola lettura in cui viene visualizzato il nome dell'applicazione da ridistribuire.

Il nome può includere un identificativo di versione opzionale, che segue il nome ed è separato da questo mediante due punti (:). Per ulteriori informazioni sulle versioni di moduli e applicazioni, vedere Module and Application Versions in Oracle GlassFish Server Application Deployment Guide.

Disponibilità

Se la casella di controllo Abilitato è selezionata, la funzione High Availability è abilitata per le sessioni Web e per la definizione di checkpoint e la passivazione potenziale dei bean di sessione con conservazione dello stato (SFSB). Se l'impostazione è false (predefinita), il salvataggio delle sessioni Web e la definizione di checkpoint SFSB vengono disabilitati per l'applicazione, l'applicazione Web o il modulo EJB specificato. Se l'impostazione è true, l'applicazione o il modulo specificato viene abilitato per la funzione High Availability. Impostare questa opzione su true solo se la funzione High Availability è configurata e abilitata ai livelli superiori, ad esempio a livello di server e di contenitore.

Questa opzione viene visualizzata se esistono cluster o istanze server standalone oltre all'istanza server predefinita (server).

Precompila JSP

Se questa opzione è selezionata, i file JSP (JavaServer Pages) vengono precompilati. Se questa opzione è disabilitata, i file JSP vengono compilati in fase di esecuzione al primo accesso. Questa opzione è disabilitata per impostazione predefinita.

Esegui verificatore

Se questa opzione è selezionata, i descrittori di distribuzione vengono verificati prima della distribuzione. Se la verifica non riesce, la distribuzione non viene eseguita. Il verificatore esamina la struttura e il contenuto dei descrittori di distribuzione. La verifica di applicazioni di grandi dimensioni richiede spesso molto tempo. Questa opzione è disabilitata per impostazione predefinita.

I package del verificatore devono essere installati tramite la utility Strumento di aggiornamento; in caso contrario, viene registrata un'avvertenza e l'opzione viene ignorata.

Java Web Start

Se questa opzione è selezionata, l'accesso a Java Web Start è consentito per un modulo client di applicazione. Questa opzione è disabilitata per impostazione predefinita.

Mantieni stato

Questa opzione controlla se le sessioni Web, le istanze SFSB e i timer EJB creati in modo persistente vengono conservati tra le ridistribuzioni.

Questa opzione è disabilitata per impostazione predefinita. È supportata solo nell'istanza server predefinita, denominata server. Non è supportata e viene ignorata per qualsiasi altra destinazione.

Alcuni tipi di modifiche apportate a un'applicazione tra le ridistribuzioni impediscono a questa funzione di operare in modo corretto. Ad esempio, non modificare il set di variabili di istanza nella classe bean SFSB.

Per le applicazioni Web, questa funzione è applicabile solo se nel file glassfish-web-app.xml l'attributo persistence-type dell'elemento session-manager è impostato su file.

Per le istanze di bean di sessione con conservazione dello stato, il tipo di persistenza senza High Availability viene impostato nel server (opzione Tipo di persistenza SFSB) e deve essere impostato su file, ovvero il valore predefinito e consigliato.

Se un elemento attivo quale una sessione Web, un'istanza SFSB o un timer EJB non viene preservato o ripristinato, nessuno di tali elementi sarà disponibile una volta completata la ridistribuzione. Tuttavia la ridistribuzione prosegue e viene registrata un'avvertenza.

Per preservare i dati con stato attivo, GlassFish Server li serializza e li salva nella memoria. Per ripristinare i dati, il classloader dell'applicazione appena ridistribuita deserializza i dati salvati in precedenza.

Conserva risorse con ambito applicazione

Se l'opzione è selezionata, le risorse con ambito a livello di applicazione vengono conservate e ripristinate durante la ridistribuzione. Questa opzione non è selezionata per impostazione predefinita.
