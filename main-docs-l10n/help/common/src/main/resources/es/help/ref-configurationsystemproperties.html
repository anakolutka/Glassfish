Propiedades del sistema

La página Propiedades del sistema muestra una lista de las propiedades del sistema Java definidas en la configuración con nombre seleccionada. Las propiedades del sistema Java se transfieren al iniciador de aplicaciones Java mediante la opción -D del iniciador de aplicaciones Java al iniciar GlassFish Server.

Estas propiedades definen los valores por defecto de la configuración de puerto para todas las instancias de GlassFish Server que hacen referencia a la configuración.

Para obtener más información sobre la configuración de puerto predefinida en una configuración, consulte Propiedades del sistema predefinidas.

Para cada propiedad se muestra la información siguiente:

Nombre de variable de instancia

Nombre de la propiedad del sistema.

Valor por defecto

Valor que se define para la propiedad en la configuración especificada a la que hace referencia la instancia. Este campo es de sólo lectura.

Valores de instancia

Enlace a la página Valores de instancia de la propiedad.

La tabla Propiedades adicionales también contiene las opciones siguientes.

Agregar propiedad

Botón para agregar una propiedad. Al hacer clic en este botón se agrega una fila a la tabla Propiedades adicionales.

Suprimir propiedades

Botón para suprimir una o varias propiedades seleccionadas. Cualquier propiedad que se suprima vuelve a su valor por defecto o, si no se ha definido ningún valor por defecto, se queda sin definir.

La página Propiedades del sistema también contiene las opciones siguientes:

Nombre de Configuración

Nombre de la configuración a la que se aplican los valores de esta página. Este campo es de sólo lectura.

Nueva configuración dinámica

Si esta opción está activada, los cambios realizados en la configuración se aplican a las instancias que hacen referencia a la configuración sin necesidad de reiniciar dichas instancias.

Clusters y/o instancias que utilizan esta configuración

Lista de los clusters y las instancias que hacen referencia a la configuración, con una indicación de si cada cluster o instancia se está ejecutando:

  • Si el cluster o instancia se está ejecutando, se muestra una marca de verificación (✓).

  • Si el cluster o la instancia están detenidos, se muestra un signo de exclamación (!).
