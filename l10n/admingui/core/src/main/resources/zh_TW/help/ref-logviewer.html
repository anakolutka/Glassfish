記錄檢視器

使用 [記錄檢視器] 頁面可檢視伺服器記錄，以及自訂記錄顯示方式。

[記錄檢視器] 頁面包含三個區域：

  • 搜尋條件

  • 進階搜尋

  • 搜尋結果

[搜尋條件] 區域包含下列選項。

進階搜尋

按一下此連結可開啟區域，讓您進一步縮小記錄檢視器的搜尋範圍。

時間戳記

[最近] 單選按鈕會顯示最新的訊息。[指定範圍] 單選按鈕會開啟 [從] 和 [到] 欄位，讓您只檢視某一段時間內的訊息。

對於時間值，必須採用以下語法格式 (SSS 表示毫秒)：

hh:mm:ss.SSS

例如︰

17:10:00.000

如果 [從] 欄位中的時間值遲於 [到] 欄位中的時間值，將顯示錯誤訊息。

記錄層級

選擇進行篩選的記錄層級。依預設，將顯示伺服器記錄中選取記錄層級和更高層級的所有訊息。選取標有 [請勿包含更多嚴重訊息] 的核取方塊，以僅顯示所選層級的訊息。

為確保要檢視的訊息都顯示在伺服器記錄中，請先在 [模組記錄層級] 頁面上設定適當的記錄層級。請參閱配置記錄層級。

如果選擇基於記錄層級篩選記錄訊息，則僅顯示符合指定篩選條件的訊息。不過，這種篩選不影響那些記錄到伺服器記錄中的訊息。

[進階搜尋] 區域包含下列選項。

記錄程式

用來依模組篩選的記錄程式名稱空間。如果要選擇多個名稱空間，請在按住 Shift 鍵或 Ctrl 鍵的同時以滑鼠點選。

選取較高層級的名稱空間時，也會連帶選取下層的所有名稱空間。例如，選取 [javax.enterprise.system] 的同時也就選取了該名稱空間下所有模組的記錄程式︰javax.enterprise.system.core、javax.enterprise.system.tools.admin 等等。

自訂記錄程式

特定應用程式專屬的記錄程式，每行列出一個。

如果應用程式具有多個模組，則可以檢視其中任意一個模組或者全部模組。例如，假設應用程式具有使用以下名稱的記錄程式：

com.mycompany.myapp.module1
com.mycompany.myapp.module2
com.mycompany.myapp.module3

若要檢視應用程式中所有模組的訊息，請鍵入 com.mycompany.myapp。若要僅檢視 module2 的訊息，請鍵入 com.mycompany.myapp.module2。

如果指定了一個或多個自訂記錄程式，則僅當在 [記錄程式] 區域中明確指定 Enterprise Server 模組時，才會顯示這些模組的訊息。

「名稱-值」對

執行緒的鍵值名稱和值，以檢視特定執行緒的輸出。金鑰名稱為 _ThreadID。例如︰

_ThreadID=13

假設 com.mycompany.myapp.module2 在多個執行緒中執行。若要細化記錄檢視器以僅顯示單一執行緒的輸出，請在 [自訂記錄程式] 欄位中指定該模組的記錄程式，然後在該欄位中指定執行緒 ID。

顯示

一次檢視的訊息數目。顯示的預設訊息數目為 40。其他選項包括 100、250 和 1000。

若要檢視堆疊追蹤，請取消選取 [限制過長訊息] 核取方塊。依預設，堆疊追蹤不在檢視器中顯示；若要檢視它們，請按一下訊息的 [(details)] 連結。

[搜尋結果] 表格會顯示伺服器記錄訊息。

[搜尋結果] 可顯示每則訊息的下列資訊。

條目號碼

訊息在伺服器記錄中的號碼。

記錄層級

訊息的記錄層級。

訊息

記錄訊息的簡短文字。按一下 [(details)] 連結可檢視訊息的格式化版本。

記錄程式

產生記錄訊息的記錄程式模組。

時間戳記

發出記錄訊息的日期和時間。

名稱/值對

記錄訊息的執行緒名稱和 ID。
