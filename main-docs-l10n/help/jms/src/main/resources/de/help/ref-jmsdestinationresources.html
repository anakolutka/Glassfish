JMS-Zielressourcen

Auf der Seite "JMS-Zielressourcen" können Sie JMS-Zielressourcen konfigurieren.

Für jede Zielressource werden die folgenden Informationen zur Verfügung gestellt.

JNDI-Name

Ein eindeutiger Name, der die Zielressource identifiziert.

Aktiviert

(Diese Spalte wird angezeigt, wenn nur die Standardserverinstanz server in der Domain vorhanden ist.)

Zeigt ein Häkchen an, wenn die Ressource auf dem Ziel aktiviert ist, oder ein X, wenn die Ressource deaktiviert ist.

Status

(Diese Spalte wird angezeigt, wenn in der Domain Cluster oder Standalone-Instanzen erstellt wurden.)

Die Anzahl Cluster und Standalone-Instanzen, auf die die Ressource ausgerichtet wird, und auf wie vielen dieser Ziele die Ressource aktiviert ist. Beispiel: "Auf 2 von 4 Ziel(en) aktiviert" bedeutet, dass die Ressource auf vier Cluster und Standalone-Instanzen ausgerichtet und auf zweien dieser vier Ziele aktiviert ist.

Ressourcentyp

Der Ressourcentyp der Zielressource.

Beschreibung

Eine Beschreibung der Zielressource.

Die Tabelle "Zielressourcen" enthält darüber hinaus die folgenden Optionen.

Neu

Schaltfläche zum Erstellen einer neuen Zielressource.

Löschen

Schaltfläche zum Löschen von mindestens einer ausgewählten Zielressource.

Aktivieren

Schaltfläche zum Aktivieren von mindestens einer ausgewählten Zielressource.

Deaktivieren

Schaltfläche zum Deaktivieren von mindestens einer ausgewählten Zielressource.

Verwandte Aufgaben

  • So werden JMS-Zielressourcen bearbeitet

  • So werden JMS-Zielressourcen gelöscht
