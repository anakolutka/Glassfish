Nova Sessão JavaMail

Utilize a página Nova Sessão JavaMail para criar um recurso de sessão JavaMail.

A página Nova Sessão JavaMail contém as opções a seguir.

Nome JNDI

Um nome exclusivo que identifica a sessão de correio. Use o prefixo de subcontexto de nomeação mail/para recursos do JavaMail. Por exemplo, mail/MySession. O nome deve conter somente caracteres alfanuméricos, sublinhado, traço ou ponto.

Host de Correio

O nome do host do servidor de correio default. Os métodos de conexão dos objetos Armazenamento e Transporte usam esse valor se não é fornecida uma propriedade de host específica do protocolo. O nome deve ser resolúvel para um nome de host real.

Usuário Default

O nome de usuário a ser fornecido ao conectar-se a um servidor de correio. Os métodos de conexão dos objetos Armazenamento e Transporte usam esse valor se não é fornecida uma propriedade de nome de usuário específica do protocolo. O nome de usuário deve conter somente caracteres alfanuméricos, sublinhado, traço ou ponto.

Endereço Default do Remetente

O endereço de e-mail do usuário default. O formato é

Nome do usuário@host.domain.

Descrição

Uma descrição dessa sessão de correio.

Status

O status da sessão de correio. A sessão de correio pode ser ativada ou desativada. O valor default é Ativado.

Protocolo de Armazenamento

O serviço de protocolo de armazenamento, que se conecta a um servidor de correio, recuperar mensagens e as salva em uma ou mais pastas. Os valores permitidos são imap, pop3, imaps e pop3s. O valor default é imap.

Classe do Protocolo de Armazenamento

A classe de implementação do provedor do serviço para armazenamento. Os valores permitidos são:

  • com.sun.mail.imap.IMAPStore

  • com.sun.mail.pop3.POP3Store

  • com.sun.mail.imap.IMAPSSLStore

  • com.sun.mail.pop3.POP3SSLStore

O valor default é com.sun.mail.imap.IMAPStore.

Protocolo de Transporte

O serviço de protocolo de transporte, que envia mensagens. Os valores permitidos são smtp e smtps. O valor default é smtp.

Classe do Protocolo de Transporte

A classe de implementação do provedor do serviço de transporte. Os valores permitidos são:

  • com.sun.mail.smtp.SMTPTransport

  • com.sun.mail.smtp.SMTPSSLTransport

O valor default é com.sun.mail.smtp.SMTPTransport.

Depuração

Se essa opção for selecionada, a depuração para esse recurso é ativada. Se o nível de log JavaMail estiver definido como BOM ou melhor, a saída de depuração será gerada e incluída no arquivo de log do sistema. Para obter informações sobre a definição do nível de log, consulte Para Configurar Níveis de Log.

Propriedades Adicionais

Propriedades adicionais para a sessão JavaMail. Para obter uma lista das propriedades disponíveis, consulte a documentação da API JavaMail (http://java.sun.com/products/javamail/javadocs/index.html). Propriedades adicionais devem começar com mail- e devem usar hífens, não pontos, como separadores. Por exemplo, mail-debug é correto, mas mail.debug não. O back-end do GlassFish Server converte os hífens nos pontos esperados pela JavaMail API.

Alvos

Clusters e instâncias standalone da sessão JavaMail. Mova os alvos desejados para a coluna Alvos Selecionados utilizando os botões Adicionar e Adicionar Tudo. Mova os alvos desnecessários para a coluna Alvos Disponíveis utilizando os botões Remover e Remover Tudo.

Esta opção é exibida somente se os clusters ou instâncias standalone tiverem sido criados no domínio.
