웹 응용 프로그램을 배포하는 방법

  1. 탐색 트리에서 응용 프로그램 노드를 선택합니다.
  2. 응용 프로그램 노드에서 웹 응용 프로그램 노드를 선택합니다.

    웹 응용 프로그램 페이지가 열립니다.

  3. 배포된 웹 응용 프로그램 테이블에서 배포 버튼을 누릅니다.

    엔터프라이즈 응용 프로그램/모듈 배포 페이지가 열립니다.

  4. 유형 드롭다운 목록에서 웹 응용 프로그램(.war)을 선택합니다.
  5. 배포할 응용 프로그램이 포함된 아카이브 위치를 지정합니다.
    • 아카이브가 클라이언트 시스템에 상주하거나 액세스 가능한 파일인 경우 서버에 업로드할 패키지화된 파일 옵션을 선택합니다.

      클라이언트 시스템은 브라우저를 통해 관리 콘솔을 보는 호스트입니다.

      찾아보기를 눌러 파일을 찾거나, 파일의 전체 경로를 입력합니다.

    • 아카이브가 서버 시스템에 있는 파일이거나, 확장된 디렉토리에서 압축 해제된 응용 프로그램인 경우 Application Server에서 액세스할 수 있는 패키지화된 로컬 파일 또는 디렉토리 옵션을 선택합니다.

      서버 시스템은 도메인 관리 서버(Domain Administration, DAS)를 실행하는 호스트입니다.

      파일 또는 디렉토리의 전체 경로 이름을 입력합니다. 확장된 디렉토리에서 배포하는 것은 고급 개발자 작업이며 프로덕션 환경에는 권장하지 않습니다.

  6. 응용 프로그램 이름 필드에 파일 이름의 접두어인 기본 이름을 유지하거나, 다른 이름을 입력합니다.

    파일 업로드를 선택한 경우에는 기본 이름이 나타납니다. 응용 프로그램 이름은 고유해야 합니다.

  7. 컨텍스트 루트 필드에 웹 응용 프로그램을 식별하는 문자열을 입력합니다.

    웹 응용 프로그램의 URL에서 컨텍스트 루트 바로 앞에는 포트 번호(http://host: port/context-root/...)가 있습니다. 컨텍스트 루트는 슬래시로 시작해야 합니다(예: /hello).

  8. 응용 프로그램을 사용할 수 없도록 비활성화하려면 사용 확인란의 선택을 취소합니다.

    기본적으로 응용 프로그램이 배포되는 즉시 상태는 사용 가능이 됩니다.

  9. 파일을 배포하기 전에 구조와 내용을 확인하려면 검증자 실행 사용 가능 확인란을 선택합니다.

    큰 응용 프로그램의 검증에는 많은 시간이 걸릴 수 있습니다. 손상되었거나 이동 불가능한 것으로 의심되는 경우에 파일을 검증하십시오.

  10. JSPTM(JavaServer PagesTM) 파일을 사전 컴파일하려면 JSP 사전 컴파일 사용 가능 확인란을 선택합니다.

    이 확인란을 선택하지 않으면 JSP 파일은 런타임에 처음 액세스될 때 컴파일됩니다. 컴파일에는 시간이 많이 걸릴 수 있으므로, 프로덕션 환경에서는 이 확인란을 선택하십시오.

  11. 라이브러리 필드에 응용 프로그램에서 필요한 라이브러리 JAR 파일을 나열합니다.
  12. 설명 필드에 이 웹 응용 프로그램에 대한 설명을 입력합니다.
  13. 확인을 누릅니다.
참조

  • 배포된 웹 응용 프로그램을 편집하는 방법

  • 웹 응용 프로그램의 설명자를 보는 방법

  • 웹 응용 프로그램을 시작하는 방법

  • 배포된 웹 응용 프로그램을 재배포하는 방법
