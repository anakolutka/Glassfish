JDBC-Ressourcen

Eine Java DataBase Connectivity-(JDBC-)Ressource (Datenquelle) stellt Anwendungen über die Verbindung zu einer Datenbank bereit. Normalerweise erstellt der Administrator eine JDBC-Ressource für jede Datenbank, auf die über eine in einer Domain bereitgestellte Anwendung zugegriffen wird. Allerdings können mehrere JDBC-Ressourcen für eine Datenbank erstellt werden.

Anwendungen rufen eine Datenbankverbindung aus einem Connection-Pool ab, indem sie in der Java Naming and Directory Interface-(JNDI-)API-Struktur nach einer Datenquelle suchen und dann eine Verbindung anfordern. Der mit der Datenquelle verknüpfte Connection-Pool stellt der Anwendung eine Verbindung bereit.

Auf der Seite "JDBC-Ressourcen" können Sie JDBC-Ressourcen konfigurieren.

Für jede Ressource werden die folgenden Informationen zur Verfügung gestellt.

JNDI-Name

Ein eindeutiger Name, der die JDBC-Ressource identifiziert.

Logischer JNDI-Name

Der logische JNDI-Name für die Ressource. Weitere Informationen finden Sie unter Logische JNDI-Namen.

Aktiviert

(Diese Spalte wird angezeigt, wenn nur die Standardserverinstanz server in der Domain vorhanden ist.)

Zeigt ein Häkchen an, wenn die Ressource auf dem Ziel aktiviert ist, oder ein X, wenn die Ressource deaktiviert ist.

Status

(Diese Spalte wird angezeigt, wenn in der Domain Cluster oder Standalone-Instanzen erstellt wurden.)

Die Anzahl Cluster und Standalone-Instanzen, auf die die Ressource ausgerichtet wird, und auf wie vielen dieser Ziele die Ressource aktiviert ist. Beispiel: "Auf 2 von 4 Ziel(en) aktiviert" bedeutet, dass die Ressource auf vier Cluster und Standalone-Instanzen ausgerichtet und auf zweien dieser vier Ziele aktiviert ist.

Connection-Pool

Der mit der Ressource verbundene JDBC-Connection-Pool.

Beschreibung

Eine kurze Beschreibung der Ressource.

Die Tabelle "Ressourcen" enthält darüber hinaus die folgenden Optionen.

Neu

Schaltfläche zum Erstellen einer neuen Ressource.

Löschen

Schaltfläche zum Löschen von mindestens einer ausgewählten Ressource.

Aktivieren

Schaltfläche zum Aktivieren von mindestens einer ausgewählten Ressource.

Deaktivieren

Schaltfläche zum Deaktivieren von mindestens einer ausgewählten Ressource.

Verwandte Aufgaben

  • So wird der Datenbankzugriff eingerichtet

  • So wird ein JDBC-Treiber integriert

  • So wird eine JDBC-Ressource erstellt

  • So wird eine JDBC-Ressource bearbeitet

  • So wird eine JDBC-Ressource gelöscht

  • So wird einen JDBC-Connection-Pool erstellt
