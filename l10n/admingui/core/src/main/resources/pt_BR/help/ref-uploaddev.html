Implantar Aplicativos ou Módulos

Use a página Implantar Aplicativos ou Módulos para implantar um aplicativo.

A página Implantar Aplicativos ou Módulos contém as seguintes opções para todos os aplicativos. Opções adicionais aparecem apenas após você especificar o campo Local do aplicativo.

Local

O local do arquivo compactado do aplicativo que está sendo implantado.

As seguintes opções especificam de onde o arquivo compactado é acessado e se é um arquivo ou um diretório.

Arquivo empacotado a ser carregado no servidor

O arquivo compactado reside ou é acessado no computador cliente.

O computador cliente é o host em que você visualiza o Console de Administração por meio de um navegador.

Arquivo ou diretório empacotado local que é acessado no Enterprise Server

O arquivo é um arquivo que reside no computador servidor ou é um aplicativo não empacotado, em um diretório destacado.

O computador servidor é o host que executa o servidor de administração de domínio do Enterprise Server .

Tipo

O tipo do aplicativo. As opções disponíveis são:

  • Aplicativo da Web

  • Aplicativo empresarial

  • Cliente aplicativo

  • Módulo do conector

  • EJB Jar

  • Aplicativo Ruby

  • Outro

As opções adicionais dos diversos tipos de aplicativos estão descritas nas seções a seguir.

  • Opções adicionais para um aplicativo da Web

  • Opções adicionais para um aplicativo empresarial

  • Opções adicionais para um cliente aplicativo

  • Opções adicionais para um módulo conector

  • Opções adicionais para um EJB Jar.

  • Opções adicionais para um aplicativo Ruby

Opções adicionais para um aplicativo da Web

Se o tipo do aplicativo for Aplicativo da Web, as seguintes opções aparecerão.

Raiz de Contexto

O caminho do aplicativo. Na URL do aplicativo Web, a raiz de contexto aparece imediatamente após o número da porta (http://host:port/ context-root/...). A raiz de contexto deve iniciar com uma barra; por exemplo, /hello.

Nome do Aplicativo

O nome do aplicativo.

Servidores Virtuais

Os servidores virtuais associados ao aplicativo.

Status

Se essa opção for ativada, o aplicativo será ativado. Por padrão, a opção está ativada.

Precompilar JSPs

Se esta opção for ativada, os arquivos JavaServer PagesTM ( JSPTM) serão pré-compilados. Se esta opção for desativada, os arquivos JSP serão compilados no tempo de execução, quando forem acessados pela primeira vez. Por padrão, a opção está desativada.

Executar verificador

Se esta opção estiver ativada, o verificador será executado antes da implantação. Se a verificação falhar, a implantação não será realizada O verificador examina a estrutura e o conteúdo do arquivo. A verificação de aplicativos grandes geralmente consome muito tempo. Por padrão, a opção está desativada.

Bibliotecas

Uma lista separada por vírgulas dos arquivos JAR da biblioteca, específicos deste módulo ou aplicativo. Os caminhos podem ser absolutos ou relativos. Um caminho é relativo adomain-dir/lib/applibs Se o caminho for absoluto, o caminho deve ser acessível ao servidor de administração de domínio (DAS), o que significa que ele deve estar sobdomain-dir As bibliotecas são disponibilizadas ao aplicativo na ordem em que elas são especificadas.

Descrição

Uma descrição do aplicativo.

Opções adicionais para um aplicativo Ruby

Se o tipo do aplicativo for aplicativo Ruby, aparecem estas opções.

Nome do Aplicativo

O nome do aplicativo.

Raiz de Contexto

O caminho do aplicativo. Na URL do aplicativo Web, a raiz de contexto aparece imediatamente após o número da porta (http://host:port/ context-root/...). A raiz de contexto deve iniciar com uma barra; por exemplo, /hello.

Status

Se esta opção for ativada, o aplicativo será ativado. Por padrão, a opção está ativada.

JRuby Home

O caminho para o diretório onde o recipiente JRuby está instalado. O local padrão éas-install/jruby. Se você especificar um valor para este campo, o valor do aplicativo substituirá o valor JRuby Home definido para o recipiente Ruby.

Tempo de execução

O número inicial de tempos de execução do JRuby para iniciar. O valor deve ser igual a ou maior que o valor no campo Tempo de execução mínimo, porém menor do que o valor no campo Tempo de execução máximo. O valor padrão é 1. Se você especificar um valor para este campo, o valor do aplicativo substituirá o valor Tempo de execução definido para o recipiente Ruby.

Tempo de execução mínimo

O número mínimo de tempos de execução do JRuby no pool. O valor deve ser menor que ou igual ao valor no campo Tempo de execução e menor do que o valor no campo Tempo de execução máximo. O valor padrão é 1. Se você especificar um valor para este campo, o valor do aplicativo substituirá o valor Tempo de execução mínimo definido para o recipiente Ruby.

Tempo de execução máximo

O número máximo de tempos de execução do JRuby no pool. O valor deve ser igual a ou maior que o valor no campo Tempo de execução mínimo, e igual a ou maior que o valor no campo Tempo de execução. O valor padrão é 2. Se você especificar um valor para este campo, o valor do aplicativo substituirá o valor Tempo de execução máximo definido para o recipiente Ruby.

Estrutura

A estrutura na qual o aplicativo será implantado. As opções disponíveis sãorails,merb,sinatra e autodetecção O padrão é rails. Configurar esta propriedade para um valor diferente deautodetecção ignora o processo normal e potencialmente longo da autodetecção e força a implantação na estrutura específica. Haverá erros, se o aplicativo implantado não tiver sido escrito para a estrutura específica.

Ambiente de rack

O ambiente no qual é executada a estrutura JRuby, como Rails ou Merb. As opções disponíveis sãoprodução,desenvolvimento e teste. O padrão é desenvolvimento.

MT seguro

A configuração de segmento seguro da estrutura, na qual o aplicativo será executado. Essa propriedade afeta aplicativos iniciados usando-se um script de inicialização de autodetecção, fornecido pelo usuário. As opções disponíveis são:

verdadeiro

Especifica que a estrutura é segmento seguro e, portanto, não precisa de um pool criado para ela.

falso

Especifica que o aplicativo começa com um pool de instâncias de aplicativos, e cada instância do aplicativo é acessado por um segmento de cada vez.

padrão

O Enterprise Server tenta detectar a segurança de segmentos automaticamente. Este é o valor padrão.

Essa propriedade afeta apenas estruturas que estão sendo iniciadas onde a segurança de segmentos não pode ser automaticamente determinada. A Configuração deste valor para verdadeiro não faz com que um aplicativo Rails 2.1.x de autodetecção seja iniciado no modo de segmento seguro, ou que ele possa ser usado para forçar uma estrutura de segmentos seguros a ser iniciada no modo pooled.
