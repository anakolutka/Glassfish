Informations générales

GlassFish Server crée une instance de serveur d'applications nommée server au moment de l'installation. Vous pouvez supprimer l'instance de serveur et en créer une portant un autre nom.

Chaque instance GlassFish Server a sa propre configuration Java, ainsi que ses propres ressources Java, zones de déploiement d'application et paramètres de configuration de serveur. Les modifications apportées à une instance de serveur d'applications n'ont aucun effet sur les autres instances de serveur d'applications. Vous pouvez disposer d'une seule instance de serveur d'applications dans un domaine d'administration.

Pour de nombreux utilisateurs, une instance de serveur d'applications suffit. Cependant, en fonction de votre environnement, vous pouvez avoir besoin de créer des instances de serveur d'applications supplémentaires. Par exemple, dans un environnement de développement, vous pouvez utiliser différentes instances de serveur d'applications pour tester différentes configurations GlassFish Server, ou pour comparer et tester différents déploiements d'application. Les instances de serveur d'applications étant faciles à ajouter ou à supprimer, vous pouvez les utiliser pour créer des zones de stockage temporaires (ou modèles d'environnement restreint) à tester pendant le développement.

De plus, pour chaque instance de serveur d'applications, vous pouvez également créer des serveurs virtuels. Dans une seule instance de serveur d'applications installée, vous pouvez attribuer des noms de domaine, des adresses IP et certaines fonctions d'administration à des entreprises ou des personnes. Les utilisateurs ont les mêmes avantages que s'ils disposaient de leur propre serveur Web, sans la maintenance du serveur et du matériel. Ces serveurs virtuels ne couvrent pas les instances de serveur d'applications. Pour plus d'informations sur les serveurs virtuels, reportez-vous à Paramètres généraux JVM.

Dans les déploiements opérationnels, vous pouvez, pour de nombreuses raisons, utiliser des serveurs virtuels plutôt que plusieurs instances de serveur d'applications. Cependant, si les serveurs virtuels ne répondent pas à vos besoins, vous pouvez également utiliser plusieurs instances de serveur d'applications.

Une instance GlassFish Server ne démarre pas automatiquement. Une fois que vous avez démarré une instance, elle est exécutée jusqu'à ce que vous l'arrêtiez. Lorsque vous arrêtez une instance de serveur d'applications, elle cesse d'accepter de nouvelles connexions, puis attend la fin de toutes les connexions restantes. Si votre ordinateur tombe en panne ou est mis hors ligne, le serveur est arrêté et toutes les demandes qu'il traitait risquent d'être perdues.

La page Informations générales permet de vérifier les paramètres de GlassFish Server et d'afficher les données de la JVM.

Elle contient les informations ci-après.

Bouton Arrêter

Cliquez sur le bouton Arrêter pour arrêter GlassFish Server.

Bouton Redémarrer

Cliquez sur le bouton Redémarrer pour redémarrer GlassFish Server.

Bouton Afficher les fichiers journaux

Cliquez sur le bouton Afficher les fichiers journaux pour visualiser les fichiers journaux d'un cluster ou d'une instance GlassFish Server.

Bouton Changer de journal

Cliquez sur le bouton Changer de journal pour changer le fichier journal du serveur d'administration (nommé server).

Bouton Récupérer une transaction

Cliquez sur le bouton Récupérer une transaction pour récupérer des transactions pour le serveur d'administration (nommé server) sur la page Récupérer les transactions.

Bouton Administration sécurisée

Cliquez sur le bouton Administration sécurisée pour activer ou désactiver l'administration sécurisée sur la page Administration sécurisée.

Nom

Nom du serveur en cours.

Statut

Statut en cours de l'instance de serveur. Le serveur peut être arrêté, démarré ou en cours d'exécution.

Temps d'activité

Nombre d'heures et de minutes pendant lequel l'instance de serveur a été continuellement en cours d'exécution. Cet élément est en lecture seule.

Equilibreur de charge HTTP

Nom de l'équilibreur de charge, s'il est activé pour l'instance de serveur.

JVM

Si vous cliquez sur Rapport JVM, une fenêtre distincte s'ouvre et affiche des rapports sur la Java Virtual Machine, notamment un rapport récapitulatif, un rapport sur la gestion et le nettoyage de la mémoire, un rapport de chargement de classes et le vidage des threads en cours.

Répertoire de configuration

Répertoire de l'ordinateur hôte qui contient les fichiers de configuration, tels que domain.xml.

Version installée

Identificateur de la version du produit.

Administration sécurisée

Statut en cours de l'administration sécurisée. Les valeurs possibles sont Activé et Non activé.

Déboguer

Indique si le débogage est activé. Si c'est le cas, le numéro de port est également affiché.

Port(s) HTTP

Ports actuellement configurés pour les demandes et les réponses HTTP.

Port(s) IIOP

Ports actuellement configurés pour les demandes et les réponses IIOP.
