Modification du module d'écoute HTTP

La page Modification du listener HTTP contient les options suivantes :

Nom

Identificateur unique en lecture seule du module d'écoute.

Module d'écoute

Entrez une adresse IP valide pour le serveur.

Sécurité

Disponible dans les profils Clusters et Enterprise. Crée un module d'écoute sécurisé.

Adresse réseau

Entrez l'adresse réseau 0.0.0.0 si vous souhaitez que le module écoute sur toutes les adresses IP du serveur à l'aide d'une valeur de port unique.

Port du module d'écoute

Valeur de port unique de module d'écoute si le champ Adresse réseau est 0.0.0.0, ou la valeur de port désirée si vous utilisez une autre adresse IP.

Serveur virtuel par défaut

Choisissez un serveur virtuel dans la liste déroulante.

Nom de serveur

Nom d'hôte à utiliser dans les URL envoyées par le serveur au client. Ce nom est l'alias si votre serveur en utilise un. Si votre serveur n'utilise aucun alias, laissez ce champ vide.

Port de redirection

Entrez une valeur de port pour rediriger les requêtes vers un autre port. Enterprise Server réachemine automatiquement la requête si les deux conditions suivantes sont remplies :

  • Ce listener prend en charge des requêtes non SSL.

  • Une requête pour laquelle une contrainte de sécurité correspondante requiert le transport SSL est reçue.

    Par défaut, Enterprise Server utilise le numéro de port spécifié dans la requête d'origine.

Threads accepteur

Nombre de threads accepteur. La valeur par défaut est 1.

Powered By

Désactivez la case à cocher Powered By pour désactiver l'ajout de l'en-tête X-Powered-By: Servlet/2.4 aux en-têtes de réponse HTTP générés par le servlet.

La spécification Java Servlet 2.4 définit cet en-tête, que les conteneurs peuvent ajouter aux réponses générées par le servlet. De même, la spécification 2.0 JavaServer PagesTM (JSP) définit un en-tête X-Powered-By: JSP/2.0 à ajouter (si vous le souhaitez) aux réponses utilisant la technologie JSP. L'ajout de l'en-tête X-Powered-By: JSP/2.0 est activé par défaut pour les applications Web. Ces en-têtes ont pour objectif d'aider les administrateurs à collecter des données statistiques sur l'utilisation du servlet et de la technologie JSP.

Pour plus d'informations sur l'activation et la désactivation de l'en-tête X-Powered-By sur les pages JSP, reportez-vous à l'Annexe A, Deployment Descriptor Files du Sun GlassFish Enterprise Server v3 Prelude Application Deployment Guide.

Il est possible que les environnements de production omettent volontairement la génération d'en-têtes X-Powered-By pour masquer leur technologie sous-jacente.

Blockage activé

Active le blocage pour les modules d'écoute et les ports externes.

Autres propriétés

Propriétés supplémentaires du listener.
