Para Criar uma Instância do GlassFish Server Standalone

As instâncias do GlassFish Server formam a base da implantação de uma aplicação. Uma instância standalone não é membro de um cluster. Cada instância pertence a um único domínio e tem sua própria estrutura de diretório, configuração e aplicações implantadas. Cada instância também inclui o contêiner Web Java Platform, Enterprise Edition (Java EE) e o contêiner da tecnologia Enterprise JavaBeans (tecnologia EJB). Cada instância nova deve conter uma referência a um nó que defina o host no qual a instância residirá.

Observação:

Para criar uma instância em um nó do tipo CONFIG que represente um host remoto, você deverá usar o subcomando create-local-instance(1). Não é possível utilizar o Console de Administração para este propósito.
Antes de Começar

Certifique-se de que exista o nó no qual a instância deverá residir. Consulte Para Criar um Nó para obter instruções.

  1. Na árvore de navegação, selecione o nó Instâncias Standalone.

    A página Instância de Servidor Standalone será aberta.

  2. Na página Instâncias do Servidor Standalone, clique em Novo.

    A página Nova Instância de Servidor Standalone será aberta.

  3. No campo Nome, digite um nome para a instância.

    O nome deve atender aos seguintes requisitos:

    • O nome só pode conter caracteres ASCII.

    • O nome deve começar com uma letra, um número ou um sublinhado.

    • O nome pode conter só os seguintes caracteres:

      • Letras minúsculas

      • Letras maiúsculas

      • Números

      • Hífen

      • Ponto

      • Sublinhado

    • O nome deve ser único no domínio e não deve ser o nome de outra instância, um cluster, uma configuração nomeada ou um nó do GlassFish Server.

    • O nome não deve ser domain, server, nem nenhuma outra palavra-chave que esteja reservada pelo GlassFish Server.

  4. Na lista drop-down Nó, selecione o nó no qual a instância residirá.

  5. Na lista drop-down Configuração, selecione a configuração nomeada que a instância utilizará.

    Observação:

    Se selecionar default-config, deve especificar na etapa seguinte que a configuração não é compartilhada. A configuração default-config não pode ser compartilhada.

  6. Especifique se a configuração que a instância utilizará é compartilhada.

    • Se a configuração não for compartilhada, deixe marcada a opção Fazer uma Cópia da Configuração selecionada.

      A configuração selecionada será copiada quando a instância for criada. A cópia da configuração recebe o nome de instance-name-config, em que instance-name é o nome da instância que você está criando.

    • Se a configuração for compartilhada, selecione a opção Referenciar a Configuração Selecionada.

      A instância utilizará a configuração nomeada existente especificada.

  7. Clique em OK.
