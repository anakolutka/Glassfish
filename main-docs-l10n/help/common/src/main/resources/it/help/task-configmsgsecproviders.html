Per configurare le funzioni di GlassFish Server per l'uso da parte dei provider di sicurezza messaggi

  1. Se si utilizza un token nome utente, configurare un database utente (se necessario).

    Se si utilizza un token nome utente/password, configurare in primo luogo un realm appropriato e un database utente appropriato per il realm.

    La configurazione di un database utente è illustrata in Per modificare un realm.

  2. Se necessario, gestire i certificati e le chiavi private.

Passi successivi

Una volta configurate le funzioni di GlassFish Server per l'uso da parte dei provider di sicurezza dei messaggi, è possibile abilitare i provider installati con GlassFish Server, come descritto in Per modificare una configurazione di sicurezza dei messaggi.
