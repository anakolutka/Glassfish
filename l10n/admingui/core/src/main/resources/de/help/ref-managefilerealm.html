Dateibenutzer

Verwalten Sie Benutzer des file-Bereichs über die Administration Console. Benutzer und Gruppen im file-Bereich sind in der Schlüsseldatei aufgeführt, deren Speicherort durch die Eigenschaft file angegeben wird.

Hinweis - Es ist auch möglich, diese Schritte zum Hinzufügen von Benutzern zu einem Dateibereich (einschließlich admin-realm) zu verwenden. Setzen Sie einfach den Namen des Zielbereichs an die Stelle des file-Bereichs, auf den in diesem Abschnitt verwiesen wird.

Ein Benutzer im Bereich file kann zu einer Java-Gruppe gehören, einer Benutzerkategorie, die durch gemeinsame Merkmale klassifiziert ist. So können Kunden einer eCommerce-Anwendung z. B. zur Gruppe CUSTOMER gehören, die Großkunden würden jedoch der Gruppe PREFERRED angehören. Das Kategorisieren von Benutzern in Gruppen vereinfacht die Zugangssteuerung bei einer großen Benutzeranzahl.

Nach der Installation von Enterprise Server ist der während der Installation eingegebene Administrator zunächst der einzige Benutzer. Dieser Benutzer ist standardmäßig Mitglied der Gruppe asadmin im Bereich admin-realm und damit berechtigt, Änderungen an Enterprise Server vorzunehmen. Benutzer, die dieser Gruppe zugewiesen sind, verfügen über Administratorrechte und können damit auf das asadmin-Tool und die Administration Console zugreifen.

Die Seite Dateibenutzer enthält eine Liste mit Benutzern für den ausgewählten Bereich. Für jeden Benutzer werden die folgenden Informationen zur Verfügung gestellt:

Benutzer-ID

Der Name des Benutzers.

Gruppenliste

Die Gruppen, denen der Benutzer angehört.
