Disponibilité JMS

La page Disponibilité JMS permet de configurer le cluster de brokers Message Queue pour traiter le cluster GlassFish Server.

Attention:

Pour pouvoir utiliser la page Disponibilité JMS pour reconfigurer un cluster existant, vous devez suivre les procédures spéciales de migration vers un autre type de cluster de brokers, comme indiqué dans To Migrate Between Types of Embedded or Local Conventional Broker Clusters dans le manuel Oracle GlassFish Server High Availability Administration Guide. L'échec de l'exécution de ces procédures spéciales peut entraîner une perte ou une altération des données, voire rendre votre configuration inutilisable, en fonction des opérations JMS effectuées sur le cluster existant.

La page Disponibilité JMS contient les options ci-après.

Remarque:

Si des instances utilisant la configuration sont en cours d'exécution, la page Disponibilité JMS affiche un avertissement, définit toutes les options en lecture seule et désactive le bouton Enregistrer.
Type de service JMS

Type des brokers (appelés hôtes JMS dans GlassFish Server) à utiliser dans le cluster de brokers. Le type sélectionné détermine les types de cluster de brokers disponibles. Les types de broker sont les suivants :

Imbriqué

La configuration et le cycle de vie du broker sont gérés par GlassFish Server. Chaque instance GlassFish incluse dans un cluster est traitée par un broker exécuté dans la même JVM que l'instance.

Si l'option Type de service JMS a la valeur Imbriqué, seuls les clusters de brokers Message Queue conventionnels sont pris en charge. Les options permettant de configurer le cluster conventionnel sont affichées.

Local

La configuration et le cycle de vie du broker sont gérés par GlassFish Server. Chaque instance GlassFish incluse dans un cluster est traitée par un broker exécuté dans une JVM distincte sur le même hôte que l'instance.

Si l'option Type de service JMS a la valeur Local, les clusters de brokers Message Queue conventionnels et améliorés (haute disponibilité) sont pris en charge. Les options permettant de sélectionner le type de cluster et de configurer le cluster sont affichées.

Distant

La configuration et le cycle de vie du broker et du cluster de brokers sont gérés à l'aide des outils d'administration Message Queue.

Si l'option Type de service JMS a la valeur Distant, aucune autre option n'est affichée.

Type de cluster JMS

Type de cluster de brokers Message Queue à utiliser pour le cluster GlassFish Server.

Si l'option Type de cluster JMS a la valeur Conventionnel, les options ci-après sont affichées.

Type d'emplacement de stockage de la configuration JMS

Type de banque de données à utiliser pour les données de configuration du cluster conventionnel. Si cette option a la valeur Broker maître, un broker du cluster est désigné comme broker maître et est utilisé pour stocker les données de configuration. Si cette option a la valeur Base de données partagée, les données de configuration sont stockées dans la banque de données JDBC accessible par tous les brokers. Dans ce cas, les options associées à la base de données sont affichées.

Type d'emplacement de stockage des messages JMS

Type de banque de données que chaque broker doit utiliser pour stocker ses données de message. Si cette option a la valeur Fichier, chaque broker stocke ses données de message dans une banque de données de type fichier. Si cette option a la valeur JDBC, chaque broker stocke ses données de message dans une banque de données JDBC.

Si l'option Type de cluster JMS a la valeur Amélioré (HA), les options associées à la base de données sont affichées.

Fournisseur de base de données
URL de base de données
Utilisateur de la base de données

Fournisseur, URL d'accès et utilisateur de la base de données JDBC à utiliser dans les situations suivantes :

  • Lorsque le type d'emplacement de stockage de la configuration d'un cluster conventionnel a la valeur Base de données partagée

  • Lorsque le type d'emplacement de stockage des messages d'un cluster conventionnel a la valeur JDBC

  • Lorsque le type de cluster a la valeur Amélioré (HA)

Pour plus d'informations sur les fournisseurs pris en charge et sur les formats des URL d'accès de chaque fournisseur, reportez-vous à JDBC-Based Persistence dans le manuel Oracle GlassFish Server Message Queue Administration Guide.

Authentification de base de données
Mot de passe de base de données
Alias de mot de passe

Informations sur le mot de passe de l'utilisateur de base de données JDBC indiqué dans Utilisateur de la base de données.

Si l'option Authentification de base de données a la valeur Mot de passe, l'option Mot de passe de la base de données est affichée. Si l'option Authentification de base de données a la valeur Alias de mot de passe, l'option Alias de mot de passe est affichée.

Autres propriétés

Propriétés supplémentaires de la disponibilité JMS.

GlassFish Server ne prédéfinit aucune autre propriété de disponibilité JMS.
