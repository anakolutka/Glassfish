So werden angezeigte Logdatensätze gefiltert

Bevor Sie beginnen

Stellen Sie sicher, dass Sie die Logdateien für den Domain Administration Server (DAS), eine GlassFish Server-Clusterinstanz oder eine eigenständige Instanz anzeigen. Weitere Informationen finden Sie unter den folgenden Themen:

  • So werden Logdateien für DAS angezeigt

  • So zeigen Sie Logdateien für eine GlassFish Server-Clusterinstanz an

  • So zeigen Sie Logdateien für eine Standalone-GlassFish Server-Instanz an

  1. Filtern Sie im Bereich "Suchkriterien" die Datensätze nach Content, Zeitraum und Logebene.

    1. Um nur Datensätze anzuzeigen, die einen bestimmten Text enthalten, geben Sie im Feld "Textsuche" den betreffenden Text ein.

      Bei der Suche wird zwischen Groß- und Kleinschreibung unterschieden.

    2. Um auszuwählen, wie lange Logdatensätze angezeigt werden, verwenden Sie die Option "Zeitstempel".

      • Um die aktuellsten Meldungen anzuzeigen, wählen Sie "Aktuell" (Standardeinstellung).

      • Um Meldungen aus einem bestimmten Zeitraum anzuzeigen, wählen Sie "Bestimmter Zeitraum", und geben Sie einen Datums- und Uhrzeitwert in die Felder "Von" und "Bis" ein.

        Der Wert "Datum" muss folgendes Format aufweisen:

        MM/DD/YY

        MM ist eine Ganzzahl im Bereich 1-12 für die Monatszahl; Beispiel: 1 für Januar oder 10 für Oktober. DD ist eine Ganzzahl im Bereich 1-31 für den Tag des Monats. YY sind die beiden letzten Ziffern des Jahres; Beispiel: 11 für das Jahr 2011.

        Beispiel: Um das Datum 1. Februar 2011 anzugeben, geben Sie 2/1/11 in das Feld "Datum" ein.

        Der Wert "Zeit" muss folgendes Format aufweisen:

        hh:mm:ss.SSS

        hh ist eine Ganzzahl im Bereich 0-23 zur Darstellung der Stunde. Für Zeiten am Nachmittag verwenden Sie Ganzzahlen im Bereich von 12-23; Beispiel: 14 für 2 Uhr nachmittags. mm ist eine Ganzzahl im Bereich 0-59 für die Anzahl von Minuten nach Angabe der Stunde. ss ist eine Ganzzahl im Bereich 0-59 für die Anzahl von Sekunden. SSS ist eine dreistellige Ganzzahl, gegebenenfalls mit führenden Nullen, im Bereich 000-999 für die Anzahl von Millisekunden.

        Beispiel: Um die Zeit 5:10 Uhr nachmittags anzugeben, geben Sie im Feld "Zeit" 17:10:00.000 ein.

        Wenn der Wert im Feld "Von" zeitlich hinter dem Wert in "Bis" liegt, wird eine Fehlermeldung angezeigt.

    3. Meldungen nach Logebene filtern.

      Um sicherzustellen, dass die Meldungen, die Sie anzeigen möchten, im Serverlog angezeigt werden, definieren Sie zunächst auf der Seite "Modullogebenen" die entsprechenden Logebenen. Weitere Anweisungen finden Sie in So werden Logebenen konfiguriert.

      Wenn Sie sich für eine Filterung von Logmeldungen basierend auf der Logebene entschieden haben, werden nur Meldungen angezeigt, die den angegebenen Filterkriterien entsprechen. Diese Filterung wirkt sich jedoch nicht darauf aus, welche Meldungen im Serverlog aufgezeichnet werden.

      1. Wählen Sie aus der Dropdown-Liste "Logebene" die Logebene.

      2. Um Meldungen ausschließlich auf der gewählten Ebene anzuzeigen, aktivieren Sie die Option "Keine weiteren als schwerwiegend eingestuften Meldungen aufnehmen".

        Standardmäßig werden alle im Serverlog enthaltenen Meldungen angezeigt, die der gewählten Logebene oder einer Ebene mit höherer Dringlichkeit entsprechen.

  2. Im Bereich "Erweiterte Suche" filtern Sie Datensätze entsprechend dem Logger Namespace und wählen den Informationsumfang, der angezeigt wird.

    1. Klicken Sie auf den Link "Erweiterte Suche".

      Der Bereich "Erweiterte Suche" wird in das Fenster "Log Viewer" eingefügt.

    2. Um nach Modul zu filtern, wählen Sie aus der Dropdown-Liste "Logger" mindestens einen Namespaces.

      Klicken Sie bei gedrückter Umschalt- oder Strg-Taste, um mehrere Namespaces auszuwählen.

      Die Auswahl eines Namespaces einer höheren Ebene führt zur Auswahl aller darunter angeordneten Namespaces. Beispiel: Wenn Sie javax.enterprise.system auswählen, werden auch die Logger aller Module unterhalb dieses Namespace ausgewählt, wie javax.enterprise.system.core, javax.enterprise.system.tools.admin usw.

    3. Um Meldungen von Loggern für eine bestimmte Anwendung anzuzeigen, geben Sie im Feld "Benutzerdefinierter Logger" den Namen jedes Loggers auf jeweils einer separaten Zeile ein.

      Wenn die Anwendung über mehrere Module verfügt, können Sie einige oder alle dieser Module anzeigen. Beispielsweise kann eine Anwendung über folgende Protokollfunktionen verfügen:

      com.example.myapp.module1
com.example.myapp.module2
com.example.myapp.module3

      Um Meldungen aller Module in der Anwendung anzuzeigen, geben Sie com.mycompany.myapp ein. Um nur Meldungen von module2 anzuzeigen, geben Sie com.example.myapp.module2 ein.

      Wenn Sie mindestens einen benutzerdefinierten Logger angeben, werden Meldungen von GlassFish Server-Modulen nur dann angezeigt, wenn Sie sie im Bereich "Logger" explizit aufführen.

    4. Wählen Sie aus der Dropdown-Liste "Anzeigen" die Meldungen, die angezeigt werden sollen.

      Als Standardanzeige für Meldungen wird 40 angezeigt. Darüber hinaus stehen 100, 250 und 1000 zur Auswahl.

    5. Um Stacktraces anzuzeigen, deaktivieren Sie die Option "Sehr lange Meldungen begrenzen".

      Standardmäßig werden im Viewer keine Stacktraces angezeigt. Um Stacktraces anzuzeigen, klicken Sie auf den Link (Details) für eine Meldung.

    6. Um den Bereich "Erweiterte Suche" auszublenden, klicken Sie auf den Link "Einfache Suche".

  3. Um die Instanz und die zugehörige Logdatei anzugeben, die Sie anzeigen möchten, verwenden Sie den Bereich "Suche modifizieren".

    • Wählen Sie aus der Dropdown-Liste "Instanz" die Instanz.

      Der Standardwert ist die Instanz, für die Sie anfangs festgelegt hatten, dass Logdateien angezeigt werden sollen. Wenn Sie festgelegt haben, dass Logdateien für den Domain Administration Server (DAS) angezeigt werden sollen, lautet der Standardwert, der den DAS angibt, "server".

    • Wählen Sie aus der Dropdown-Liste "Logdatei" die Logdatei.

      Der Standardname lautet server.log. Andere Optionen hängen von den Logeinstellungen für das gewählte Ziel ab.

  4. Passen Sie im Bereich "Suchergebnisse" die Anzeige von Logmeldungen an.

    Im Bereich "Suchergebnisse" werden entsprechend den Einstellungen auf den Seiten "Logging-Einstellungen" und "Logebenen" die letzten 40 Einträge des Serverlogs angezeigt.

    • Um die Meldungen unterschiedlich zu sortieren, klicken Sie auf die Pfeile in den Spalten.

    • Um eine formatierte Version einer Meldung anzuzeigen, klicken Sie auf den Link (Details) für eine Meldung.

      Es wird ein Fenster "Logeintragsdetail" angezeigt, das eine formatierte Version der Meldung enthält.

    • Um vorhergehende oder nachfolgende Einträge in der Logdatei anzuzeigen, klicken Sie auf die Schaltflächen "Einträge vor" bzw. "Einträge nach", die sich am oberen Rand der Tabelle befinden.
