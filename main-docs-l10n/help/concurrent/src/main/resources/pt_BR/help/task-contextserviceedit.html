Para Editar um Serviço de Contexto

  1. Na árvore de navegação, expanda o nó Recursos.

  2. No nó Recursos, expanda o nó Recursos Concorrentes.

  3. No nó Recursos Concorrentes, selecione Serviços de Contexto.

    O contexto Serviços página é aberta.

  4. Na página Serviços de Contexto, clique no nome do recurso que você está editando.

    A página Editar Serviço de Contexto é aberta.

    Observação:

    O nome JNDI é um campo somente para leitura. Você pode especificar o nome JNDI ao criar um novo recurso. O nome JNDI lógico também é um campo somente para leitura.

  5. Marque a caixa de seleção Informações de Contexto Ativadas para propagar contextos a threads, e selecione os contextos que você deseja propagar.

    Esta opção é ativada e todos os contextos são selecionadas por default. Use Shift-clique ou Control-clique para escolher vários contextos. Se essa opção estiver desativada, os contextos não serão propagados, mesmo que eles estejam selecionados.

  6. No campo Ordem de Implantação, digite um número para especificar a ordem de implantação do recurso.

    Recursos com números menores são carregados primeiro na inicialização do servidor. O valor default é 100.

  7. No campo Descrição, digite uma descrição curta do recurso.

    Esta descrição é um valor de string e pode incluir no máximo 250 caracteres.

  8. Marque a caixa de seleção Status ativo para ativar o recurso.

  9. Na seção Propriedades Adicionais, não adicione propriedades.

    O GlassFish Server não define propriedades adicionais para este tipo de recurso. Além disso, este tipo de recurso não usa atualmente propriedades adicionais.

  10. Clique em Salvar.
