Configurer le cache de fichiers HTTP

Le cache de fichiers HTTP stocke du contenu statique pour permettre à Enterprise Server de traiter rapidement les requêtes de contenu statique.

  1. Dans l'arborescence de navigation, développez le nœud Configuration.
  2. Sous le nœud Configuration, sélectionnez le nœud Service HTTP.
  3. Dans la page Service HTTP, sélectionnez l'onglet Cache de fichiers HTTP.
  4. Cochez la case Globalement activé pour activer la mise en cache globale des fichiers.

    Cette option est désactivée par défaut.

  5. Dans le champ Âge max., entrez la durée de vie maximale, en secondes, pour une entrée de cache valide.

    Ce paramètre détermine comment des informations en cache depuis un certain temps sont utilisées après la mise en cache d'un fichier. Une entrée antérieure à l'âge maximal est remplacée par une nouvelle entrée pour le même fichier. Si le contenu change rarement, augmentez cette valeur pour améliorer les performances.

    L'âge maximal optimal dépend de la fréquence à laquelle les fichiers existants sont modifiés. Par exemple, si les fichiers sont modifiés quatre fois par jour à intervalles réguliers, vous pouvez définir l'âge maximal sur 21600 secondes (6 heures). Sinon, pensez à définir l'âge maximal sur la période la plus longue durant laquelle vous acceptez d'utiliser la version précédente d'un fichier après sa modification.

    La valeur par défaut est 30.

  6. Dans le champ Nb max. de fichiers, entrez le nombre maximal de fichiers pouvant être stockés dans le cache de fichiers.

    Si la valeur est trop élevée, le serveur met en cache les fichiers peu utiles qui consomment de la mémoire. Si elle est trop faible, vous perdez l'avantage de la mise en cache. Testez différentes valeurs de cet attribut pour rechercher la solution optimale pour des applications spécifiques. La valeur par défaut est 1024.

  7. Dans le champ Taille initiale de hachage, entrez le nombre initial de compartiments de hachage.

    Le nombre initial de compartiments de hachage affecte l'utilisation de la mémoire et les délais de recherche. La valeur par défaut est 0.

  8. Dans le champ Limite de taille de fichier moyen, entrez la taille maximale, en octets, d'un fichier mis en cache pouvant être stocké en tant que fichier mappé en mémoire.

    Le contenu des fichiers moyens est mis en cache via le mappage des fichiers en mémoire virtuelle. La limite optimale est une valeur légèrement supérieure à la taille de la page de mémoire virtuelle. La valeur par défaut est 537,600.

  9. (Plates-formes UNIX® et Linux uniquement) Dans le champ Taille de fichier moyen, entrez la taille totale, en octets, de tous les fichiers mis en cache en tant que fichiers mappés en mémoire.

    La valeur par défaut est 10,485,760.

  10. Dans le champ Limite de taille de petit fichier, entrez la taille maximale, en octets, d'un fichier pouvant être lu en mémoire.

    Le contenu des fichiers de petite taille est mis en cache en allouant de l'espace tas et en lisant le fichier dans ce dernier. La limite optimale est une valeur légèrement inférieure à la taille de la page de mémoire virtuelle. La valeur par défaut est 2048.

  11. Dans le champ Taille de petit fichier, entrez la taille totale, en octets, des fichiers devant être lus en mémoire.

    La valeur par défaut est 1,048,576.

  12. Définissez l'option Mise en cache des fichiers activée pour activer ou désactiver la mise en cache du contenu des fichiers lorsque la taille des fichiers est inférieure à la limite de taille de fichier moyen.
    • Pour activer la mise en cache, définissez cette option sur Activé.
    • Pour désactiver la mise en cache, définissez cette option sur Désactivé.
  13. Cliquez sur Enregistrer.

    Astuce – Pour revenir aux paramètres par défaut, cliquez sur le bouton Charger valeurs par défaut, puis sur Enregistrer.
