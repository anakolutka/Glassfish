Per creare una destinazione fisica JMS

Di norma non è necessario creare destinazioni fisiche in quanto al momento della creazione di una risorsa di destinazione JMS è necessario specificare il nome di una destinazione fisica. Se necessaria, la destinazione fisica viene creata in modo automatico.

  1. Espandere il nodo Configurazione nell'albero di navigazione.

  2. Selezionare l'istanza da configurare:

    • Per configurare un'istanza particolare, selezionare il nodo di configurazione dell'istanza. Ad esempio, per l'istanza predefinita server, selezionare il nodo server-config.

    • Per configurare le impostazioni predefinite per le istanze future che utilizzano una copia di default-config, selezionare il nodo default-config.

  3. Nel nodo Configurazione espandere il nodo Java Message Service.

  4. Nel nodo Java Message Service selezionare il nodo Destinazioni fisiche.

    Viene visualizzata la pagina Destinazioni fisiche.

  5. Nella pagina Destinazioni fisiche fare clic su Nuovo.

    Viene visualizzata la pagina Nuova destinazione fisica.

  6. Nel campo Nome digitare il nome della destinazione fisica.

  7. Nell'elenco a discesa Tipo selezionare javax.jms.Topic o javax.jms.Queue.

  8. Nel campo Numero massimo di messaggi non utilizzati digitare il numero massimo di messaggi non utilizzati consentiti per la destinazione.

    Il valore -1 indica un numero illimitato di messaggi non utilizzati. Il valore predefinito è -1.

  9. Nel campo Dimensione massima messaggio digitare la dimensione massima, in byte, di ogni singolo messaggio.

    Il valore -1 indica una dimensione di messaggio illimitata. Il valore predefinito è -1.

  10. Nel campo Memoria totale massima digitare la memoria totale massima, in byte, per i messaggi non utilizzati.

    Il valore -1 indica l'assenza di limiti per la dimensione della memoria dei messaggi non utilizzati. Il valore predefinito è -1.

  11. Nell'elenco a discesa Funzionamento limite soglia selezionare il funzionamento desiderato per il broker quando viene raggiunta la soglia del limite di memoria.

    • Rifiuta i messaggi più recenti indica al broker do rifiutare i messaggi più recenti e di avvertire il client di produzione con un'eccezione se il messaggio è persistente.

      Rifiuta i messaggi più recenti è il valore predefinito.

    • Rallenta producer indica al broker di ridurre la velocità di invio dei messaggi da parte dei producer.

    • Elimina i meno recenti indica al broker di ignorare i messaggi più vecchi.

    • Elimina i messaggi con la priorità più bassa indica al broker di ignorare i messaggi con la priorità più bassa.

    Se il valore è Elimina i meno recenti o Elimina i messaggi con la priorità più bassa e Usa coda dei messaggi bloccati è impostato su True, i messaggi in eccesso vengono spostati nella coda dei messaggi bloccati.

  12. Nel campo Numero massimo di producer digitare il numero massimo di producer di messaggi che possono essere associati alla destinazione fisica.

    Una volta raggiunto questo limite, non è possibile creare nuovi producer. Il valore -1 indica un numero illimitato di producer. Il valore predefinito è 100.

  13. Nel campo Numero massimo messaggi per consegna batch digitare il numero massimo di messaggi che è possibile inviare ai consumer in un solo batch.

    Il valore -1 indica un numero illimitato di messaggi. Il valore predefinito è 1000.

  14. Nell'elenco a discesa Usa coda dei messaggi bloccati selezionare True per posizionare i messaggi bloccati nella coda dei messaggi bloccati.

    Se il valore è impostato su False, i messaggi bloccati vengono eliminati. Questa opzione è impostata su True per impostazione predefinita.

  15. Nell'elenco a discesa Abilitato dell'opzione Convalida schema XML, selezionare True per abilitare la convalida dello schema XML per la destinazione.

    Se il valore è impostato su True, prima dell'invio di un messaggio XML al broker, il runtime del client Message Queue tenterà di convalidarlo in base alle XSD (XML Schema Definition) specificate o, se non è specificata alcuna XSD, in base al DTD. Se non è possibile individuare lo schema specificato o convalidare il messaggio, quest'ultimo non viene inviato e viene restituita un'eccezione. Questa opzione è impostata su False per impostazione predefinita.

  16. Nel campo Lista URI schema XML digitare una lista separata da spazi di stringhe URI per XSD (XML Schema Definition).

    Se è abilitata la convalida dello schema XML, gli URI puntano alla posizione di una o più XSD da utilizzare per tale convalida.
