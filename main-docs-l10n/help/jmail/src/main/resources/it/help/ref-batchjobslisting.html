Esecuzioni di processi batch

GlassFish Server fornisce un ambiente runtime batch per la pianificazione e l'esecuzione di processi batch. I processi batch sono in genere task con tempi di esecuzione lunghi, orientati a blocchi, che contengono una serie di passi. Le applicazioni batch inviano i processi al runtime batch e forniscono istruzioni sulla modalità e sui tempi di esecuzione dei passi. Il runtime batch elabora i passi e memorizza le informazioni sui processi in un repository di processi. In GlassFish Server il repository di processi è un database.

Utilizzare la pagina Esecuzioni processi batch per visualizzare i dettagli sulle esecuzioni di processi batch registrate.

La pagina Esecuzioni processi batch contiene le informazioni riportate di seguito.

Nome istanza
Nome cluster

Il nome del cluster o dell'istanza server di GlassFish cui si applica la configurazione.

Processi batch

La lista delle esecuzioni di processi batch registrate per l'istanza o il cluster. Per ogni esecuzione vengono visualizzate le informazioni riportate di seguito.

ID esecuzione

L'ID assegnato all'esecuzione del processo batch. Facendo clic sull'ID esecuzione viene visualizzata la pagina Dettagli esecuzione processo batch relativa all'esecuzione. Una nuova esecuzione viene creata al primo avvio di un processo e a ogni riavvio dell'esecuzione esistente.

Nome processo

Il nome del processo.

Stato batch

Lo stato dell'esecuzione impostato dal runtime batch.

Stato di uscita

Lo stato dell'esecuzione impostato dall'XML di processo per il processo o dall'applicazione batch. Per impostazione predefinita, lo stato di uscita e lo stato del batch sono uguali a meno che lo stato di uscita non venga sostituito in modo esplicito.

ID istanza

L'ID assegnato all'istanza del processo batch. Nell'istanza di un processo batch possono essere presenti più esecuzioni.

Ora di inizio

L'ora in cui è stata avviata l'esecuzione.

Ora di fine

L'ora in cui è stata terminata l'esecuzione.
