Amministrazione sicura

Utilizzare la pagina Amministrazione sicura per abilitare o disabilitare l'amministrazione sicura.

Quando l'amministrazione sicura è abilitata:

  • il server DAS e le istanze nel dominio utilizzano i certificati SSL per cifrare i messaggi inviati;

  • il server DAS accetta i messaggi di amministrazione dai client di amministrazione remoti, quali la utility asadmin e gli ambienti IDE.

La pagina Amministrazione sicura contiene le informazioni riportate di seguito.

Abilita amministrazione sicura

Pulsante che consente di abilitare l'amministrazione sicura. Questo pulsante viene visualizzato solo quando l'amministrazione sicura non è abilitata.

Nota:

Se un utente amministrativo qualsiasi del dominio dispone di una password vuota, i tentativi di abilitare l'amministrazione sicura non riescono.
Disabilita amministrazione sicura

Pulsante che consente di disabilitare l'amministrazione sicura. Questo pulsante viene visualizzato solo quando l'amministrazione sicura è abilitata.

Alias di amministrazione

L'alias che fa riferimento al certificato SSL/TLS nel server DAS. L'alias viene utilizzato dal server DAS per identificarsi con le istanze. Il valore predefinito è s1as. Questa opzione viene visualizzata solo quando l'amministrazione sicura non è abilitata.

Alias istanze

L'alias che fa riferimento al certificato SSL/TLS nelle istanze. L'alias viene utilizzato dalle istanze per identificarsi con il server DAS. Il valore predefinito è glassfish-instance. Questa opzione viene visualizzata solo quando l'amministrazione sicura non è abilitata.
