Verwaltete Thread-Factory bearbeiten

Auf der Seite "Verwaltete Thread-Factory bearbeiten" können Sie eine verwaltete Thread-Factory-Ressource bearbeiten.

Die Seite "Verwaltete Thread-Factory bearbeiten" enthält die folgenden Optionen.

Standardeinstellungen laden

Schaltfläche zur Wiederherstellung der Standardwerte von Einstellungen, die Standardwerte haben. Einstellungen, die keine Standardwerte haben, werden nicht geändert.

JNDI-Name

Ein eindeutiger Name, der die Ressource identifiziert. Der JNDI-Name ist ein schreibgeschütztes Feld. Sie können den JNDI-Namen nur festlegen, wenn Sie eine neue Ressource erstellen.

Logischer JNDI-Name

Der logische JNDI-Name für die Ressource. Dies ist ein Java EE-Standardname, über den auf die Ressource zugegriffen werden kann. Der logische JNDI-Name ist ein schreibgeschütztes Feld. Weitere Informationen finden Sie unter Logische JNDI-Namen.

Kontextinformation

Die an Threads zu propagierenden Kontexte: Class Loader, JNDI, Sicherheit oder WorkArea. Die Kontextpropagierung kann aktiviert oder deaktiviert werden. Ist die Option aktiviert, werden die gewählten Kontexte propagiert. Ist die Option deaktiviert, wird keiner der Kontexte propagiert, auch wenn sie ausgewählt sind. Standardmäßig ist die Option aktiviert, und alle vier Kontexte sind gewählt.

Status

Gibt an, ob die Ressource zur Laufzeit verfügbar ist.

Wenn in der Domain nur die Standardserverinstanz server vorhanden ist, kann die Laufzeitverfügbarkeit der Ressource aktiviert oder deaktiviert werden. Laufzeitverfügbarkeit ist standardmäßig aktiviert.

Wurden in der Domain Cluster oder Standalone-Instanzen erstellt, können Sie die Laufzeitverfügbarkeit der Ressource auf ihren Zielen über die Registerkarte "Ziel" steuern.

Threadpriorität

Die Priorität, die erstellten Threads zugewiesen werden soll. Der Standardwert ist 5.

Deployment-Reihenfolge

Die Deployment-Reihenfolge dieser Ressource. Je niedriger die Nummer einer Ressource, desto früher wird sie beim Hochfahren des Servers geladen. Der Standardwert ist 100.

Beschreibung

Eine Beschreibung dieser Ressource.

Weitere Eigenschaften

Weitere Eigenschaften für die externe Ressource. In GlassFish Server werden keine weiteren Eigenschaften für diesen Ressourcentyp definiert. Darüber hinaus verwendet dieser Ressourcentyp derzeit keine weiteren Eigenschaften.

  • So wird eine verwaltete Thread-Factory gelöscht
