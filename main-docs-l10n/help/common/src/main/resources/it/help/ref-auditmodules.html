Moduli di audit

Utilizzare la pagina Moduli di audit per configurare i moduli di audit.

Per audit si intende il modo in cui GlassFish Server registra gli eventi significativi, quali ad esempio gli errori o le violazioni della sicurezza. GlassFish Server registra tutti gli eventi di autenticazione nei file di log. Un log degli accessi completo fornisce una traccia sequenziale degli eventi di accesso di GlassFish Server.

Per ogni modulo di audit vengono fornite le informazioni riportate di seguito.

Nome

Il nome del modulo di audit.

Nome classe

Il nome completamente qualificato della classe che implementa questo modulo. Il nome della classe per il modulo di audit predefinito è com.sun.enterprise.security.Audit.

La tabella Moduli contiene inoltre le opzioni riportate di seguito.

Nuovo

Pulsante che consente di creare un nuovo modulo di audit.

Elimina

Pulsante che consente di eliminare uno o più moduli di audit selezionati.
