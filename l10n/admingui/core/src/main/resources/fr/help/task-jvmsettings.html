Configurer les paramètres généraux du profileur JVM

La machine virtuelle JavaTM (JVM) permet d'exécuter du code binaire Java sur une machine physique. Vous pouvez régler les paramètres de JVM pour améliorer les performances et empêcher des erreurs d'allocation de mémoire de se produire.

La machine virtuelle Java est incluse dans le logiciel Java Platform Standard Edition (plate-formeJavaSE), nécessaire à l'exécution de Enterprise Server. La saisie de paramètres JVM incorrects empêchant l'exécution du serveur, vous devez être prudent(e) lors de la modification de ces paramètres.

Remarque - Les termes « Machine Java Virtual » et « JVM » désignent une machine virtuelle pour plate-forme Java.

  1. Dans l'arborescence de navigation, sélectionnez le noeud Configuration.
  2. Sous le nœud Configuration, sélectionnez le nœud Paramètres JVM.

    La page Paramètres JVM s'ouvre.

  3. Sur la page Paramètres généraux JVM, dans le champ Accueil Java, saisissez le nom du répertoire d'installation du logiciel Java SE.

    Enterprise Server fonctionne avec la plate-forme Java SE. Pour vérifier que la version du logiciel Java SE spécifié est prise en charge par cette version, consultez Sun GlassFish Enterprise Server v3 Preview Release Notes . Si vous saisissez un nom de répertoire inexistant ou le répertoire d'une version non prise en charge du logiciel Java SE, Enterprise Server ne démarrera pas.

  4. Dans le champ Options Javac, saisissez les options de ligne de commande du compilateur du langage de programmation Java.

    Enterprise Server exécute le compilateur lorsque des composants Enterprise JavaBeansTM (EJBTM) sont déployés.

  5. Cochez la case Débogage activé pour activer le débogage avec JPDA (Java Platform Debugger Architecture).

    L'architecture JPDA est utilisée par les développeurs d'applications. Pour plus d'informations, consultez le Sun GlassFish Enterprise Server v3 Preview Application Development Guide .

  6. Dans le champ Options de débogage, saisissez les options JPDA à transmettre à la JVM.

    Ce champ ne s'applique que si le débogage a été activé à l'étape précédente.

  7. Dans le champ Options de compilation RMI, saisissez les options de ligne de commande du compilateur rmic.

    Enterprise Server exécute le compilateur rmic lorsque des composants EJB sont déployés.

  8. Dans le champ Préprocesseur du bytecode, saisissez la liste des noms de classe.

    Séparez les noms de classe par des virgules. Chaque classe doit implémenter l'interface com.sun.appserv.BytecodePreprocessor . Les classes sont appelées par ordre d'énumération.

  9. (Facultatif) Dans la section Autres propriétés, spécifiez d'autres propriétés.

    Pour ajouter une propriété, cliquez sur le bouton Ajouter propriété. Sur la ligne vide qui apparaît, saisissez le nom de la propriété dans le champ Nom et la valeur de la propriété dans le champ Valeur.

    Le Enterprise Server ne définit aucune autre propriété pour la JVM.

  10. Cliquez sur Enregistrer.
  11. Redémarrez le serveur.
