Managing Standalone Server Instances

In the Standalone Server Instances page, you can do the following operations:

  • View details about the existing standalone server instances. The information includes the following:

    • Name column indicates the name of the server instance.

    • Weight column indicates the proportion of user requests forwarded to the instance by the load balancer.

    • HTTP Load Balancer column indicates the name of the load balancer that assigns the user requests to the instance.

    • Configurations column specifies the application server configurations available on the instance.

    • Node Agent specifies the name of the node agent associated with this instance.

    • Status column indicates the current lifecycle state of the server instance.

  • Create a new standalone instance by clicking New. The New Stand-Alone Server Instance page appears where you can create the new instance. For more details, see To Create a New Standalone Server Instance.

  • Start a server instance by clicking the check box next to the required server instance and clicking Start.

  • Stop a server instance by clicking the check box next to the required server instance and clicking Stop.

  • Remove a server instance by clicking the check box next to the required server instance and clicking Delete.
