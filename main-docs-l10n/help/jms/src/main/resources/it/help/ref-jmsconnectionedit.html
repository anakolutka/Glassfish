Modifica Connection Factory JMS

Utilizzare la pagina Modifica Connection Factory JMS per modificare le impostazioni per un Connection Factory JMS.

La pagina Modifica Connection Factory JMS contiene le opzioni riportate di seguito.

Carica impostazioni predefinite

Pulsante per ripristinare i valori predefiniti delle impostazioni che dispongono di valori predefiniti. Le impostazioni che non dispongono di valori predefiniti non vengono modificate.

Nome pool

Il nome univoco del connection pool del connettore creato per il Connection Factory. Questo nome corrisponde inoltre al nome JNDI del Connection Factory. Il campo del nome del pool è di sola lettura.

Tipo di risorsa

Il tipo di Connection Factory. Il campo Tipo di risorsa è di sola lettura. È possibile specificare il tipo di risorsa solo quando si crea un nuovo Connection Factory JMS.

Descrizione

La descrizione del Connection Factory.

Stato

Lo stato del Connection Factory. Il Connection Factory può essere abilitato o disabilitato. Questa opzione è abilitata per impostazione predefinita.

Dimensione pool iniziale e minima

Il numero minimo di connessioni nel pool per il Connection Factory. Il valore predefinito è 8.

Dimensione massima pool

Il numero massimo di connessioni nel pool. Il valore predefinito è 32.

Quantità di ridimensionamento pool

Il numero di connessioni da rimuovere allo scadere del timeout di inattività del pool. Il valore predefinito è 2.

Timeout inattività

Il periodo di tempo massimo, in secondi, durante il quale una connessione può rimanere inattiva nel pool. Il valore predefinito è 300.

Tempo massimo di attesa

Il periodo di tempo durante il quale l'applicazione che richiede una connessione rimarrà in attesa prima che si verifichi il timeout della connessione. Il valore predefinito è 60000.

In caso di qualsiasi errore

Se la casella di controllo Chiudi tutte le connessioni è selezionata, GlassFish Server chiude tutte le connessioni nel pool e le ristabilisce se una singola connessione non riesce. Questa opzione è disabilitata per impostazione predefinita.

Supporto transazioni

Il valore può essere uno di quelli riportati di seguito.

XATransaction

La risorsa può essere utilizzata per le transazioni che implicano l'uso di più risorse nell'ambito della transazione. Questo è il valore predefinito per un Connection Factory JMS.

Ad esempio, è possibile che in una transazione sia necessario utilizzare questa risorsa insieme a una risorsa JDBC, a una risorsa connettore o a un'altra risorsa Connection Factory JMS. Questo valore offre la massima flessibilità. Una risorsa configurata come XATransaction verrà utilizzata nelle operazioni di commit a due fasi.

LocalTransaction

La risorsa può essere utilizzata per le transazioni che implicano l'uso di una sola risorsa nell'ambito della transazione oppure come ultimo agente in una transazione distribuita che prevede l'uso di più risorse XA.

Questo valore offre prestazioni decisamente migliori. Una risorsa configurata come LocalTransaction non verrà utilizzata nelle operazioni di commit a due fasi.

NoTransaction

La risorsa non può mai essere utilizzata nelle transazioni. Questa impostazione viene utilizzata raramente nelle applicazioni JMS.

Convalida connessione

Se la casella di controllo Obbligatorio è selezionata, le connessioni vengono convalidate prima di essere fornite all'applicazione. Se la convalida di una risorsa non riesce, la risorsa viene eliminata e viene creata e restituita una nuova risorsa. Questa opzione è disabilitata per impostazione predefinita.
