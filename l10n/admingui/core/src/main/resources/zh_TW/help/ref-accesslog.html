存取記錄

[存取記錄] 頁面包含下列選項：

啟用自動重建

若選取此選項，則會啟用自動重建記錄檔。此選項預設為停用。

自動重建策略

唯一可用的自動重建策略是時間。

自動重建間隔

每兩次自動重建存取記錄的相隔分鐘數。僅當自動重建策略為 time 時，該欄位才有效。預設值為 1440 分鐘 (24 小時)。

請確保自動重建後綴包含足夠的值，以為每個記錄自動重建建立唯一的後綴。例如，如果存取記錄的自動重建間隔頻率高於每天一次，請將每日時間加入自動重建後綴。

若要確保在每次經過自動重建間隔時間後，即自動重建存取記錄檔，則需要有唯一的後綴。如果新的存取記錄檔之自動重建後綴與現有存取記錄檔的自動重建後綴相同，則不會自動重建檔案。

自動重建後綴

自動重建之後，要增加至記錄檔名稱結尾的字串值。預設值為 yyyy-MM-dd。

格式

指定要在存取記錄中擷取的資訊及擷取的順序之字串值。預設值如下：

%client.name% %auth-user-name% %datetime% %request% %status% %response.length%

格式字串中包含下列可用值：

%client.name%

用戶端主機名稱

%client.dns%

用戶端 DNS

%datetime%

系統日期

%request%

完整的 HTTP 請求行

%status%

狀態

%response.length%

回應內容長度

%header.referer%

推薦者標頭

%header.user-agent%

使用者代理程式標頭

%http-method%

HTTP 方法

%http-uri%

HTTP URI

%query-str%

HTTP 查詢字串

%http-version%

HTTP 協定版本

%header.accept%

接受標頭

%header.date%

日期標頭

%header.if-mod-since%

修改日期標頭

%header.auth%

授權標頭

%header.any%

RFC 2616 中定義的任何有效 HTTP 標頭 (any 也是有效的標頭值；此處將其指定為變數)

%auth-user-name%

授權使用者的名稱

%cookie.value%

Cookie 的值

%vs.id%

虛擬伺服器識別碼
