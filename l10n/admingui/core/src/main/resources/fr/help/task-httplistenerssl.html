Configurer un listener HTTP afin qu'il utilise SSL

Configurer le listener HTTP pour utiliser SSL permet à Enterprise Server de lier un port à un certificat de serveur.

  1. Dans l'arborescence de navigation, développez le nœud Configuration.
  2. Sous le nœud Configuration, développez le nœud Service HTTP.
  3. Sous le nœud Service HTTP, sélectionnez le nœud Modules d'écoute HTTP.
  4. Sur la page Listeners HTTP, cliquez sur le nom du listener que vous configurez.
  5. Dans la page Modification du module d'écoute HTTP, cliquez sur l'onglet SSL.
  6. Cochez la case Authentification client pour obliger les clients à s'authentifier auprès du serveur lorsqu'ils utilisent ce listener.
  7. Dans le champ Surnom du certificat, saisissez le nom d'une paire clé/certificat de serveur.
  8. Cochez la case SSL3 activé pour activer le protocole de sécurité SSL3 pour le listener.
  9. Cochez la case SSL2 activé pour activer le protocole de sécurité SSL2 pour le listener.
  10. Cochez la case TLS activé pour activer le protocole de sécurité TLS pour le listener.
  11. Dans la section Suites de chiffrement, sélectionnez les suites de chiffrement disponibles utilisées par les protocoles, puis cliquez sur le bouton Ajouter.

    Pour activer toutes les suites de chiffrement, cliquez sur le bouton Ajouter tout.

  12. Cliquez sur Enregistrer.
  13. Redémarrez le serveur.
