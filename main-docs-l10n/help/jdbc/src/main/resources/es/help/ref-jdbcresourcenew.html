Nuevo Recurso JDBC

Utilice la página Nuevo recurso JDBC para crear un nuevo recurso JDBC.

La página Nuevo recurso JDBC contiene las siguientes opciones.

Nombre JNDI

Nombre único que identifica el recurso JDBC. El nombre JNDI organiza y ubica componentes dentro de un entorno informático distribuido del mismo modo que los ficheros organizan y representan ubicaciones de libros en una biblioteca. Por consiguiente, el nombre JNDI se convierte en un importante método de acceso al recurso JDBC. Por convención, el nombre comienza por la cadena jdbc/, por ejemplo, jdbc/payrolldb. No olvide la barra diagonal. El nombre debe contener únicamente caracteres alfanuméricos, de subrayado, guiones y puntos.

Nombre de pool

Pool de conexiones asociado al recurso JDBC.

Descripción

Breve descripción del recurso JDBC.

Estado

Si esta opción está activada, el recurso está disponible en el tiempo de ejecución. Esta opción se encuentra activada por defecto.

Propiedades Adicionales

Propiedades adicionales para el recurso JDBC. GlassFish Server no define ninguna propiedad adicional para recursos JDBC.

Destinos

Clusters e instancias independientes para el recurso JDBC. Mueva los destinos seleccionados a la columna Destinos Seleccionados mediante los botones Agregar y Agregar Todos. Mueva los destinos que no necesite a la columna Destinos Disponibles mediante los botones Eliminar y Eliminar Todos.

Esta opción sólo se muestra si en el dominio se han creado clusters o instancias independientes.
