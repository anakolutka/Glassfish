重新部署已部署的应用程序

重新部署应用程序会在保留应用程序最初部署时的设置和其他选项的同时部署应用程序的新版本。

开始之前

确保要重新部署的应用程序已经部署。

  1. 在导航树中，选择“应用程序”节点。

    此时将打开“应用程序”页面。

  2. 在“已部署应用程序”表中，单击要重新部署的应用程序行中的“重新部署”。

    此时将打开“重新部署应用程序或模块”页面。

  3. 指定包含要重新部署的应用程序的归档位置。
    • 如果归档文件是位于客户机或可从客户机访问的文件，请选择“要上载到服务器的打包文件”选项。

      客户机是可通过浏览器查看管理控制台的主机。

      单击“浏览”以找到文件，或键入文件的完整路径。

    • 如果归档文件是位于服务器计算机上的文件，或者是来自展开目录的未打包的应用程序，请选择“可从 Enterprise Server 访问的本地打包文件或目录”选项。

      服务器计算机是运行域管理服务器 (Domain Administration Server, DAS) 的主机。

      单击“浏览文件”浏览至文件，或者单击“浏览文件夹”浏览至目录。此外，还可键入文件或目录的完整路径名。从展开的目录部署适用于高级开发者，建议不要在生产环境中执行此操作。

  4. 如果您需要保留应用程序的活动会话，然后在重新部署完成之后恢复这些会话，请选中“保留会话”复选框。

    如果无法保留或存储应用程序的任何活动会话，则在重新部署完成时将无会话可用。但是，重新部署将继续，并且记录一个警告。

  5. 选中“预编译 JSP”复选框，以预编译 JavaServer Pages (JSP) 文件。

    如果禁用此选项，则在运行时首次访问 JSP 文件时对其进行编译。默认情况下禁用此选项。

  6. 选中“运行验证器”复选框以在重新部署之前运行验证器。

    如果选中此选项并且验证失败，则重新部署将不会执行。验证器将检查文件的结构和内容。大型应用程序的检验通常会很耗费时间。默认情况下禁用此选项。

  7. 单击“确定”。
