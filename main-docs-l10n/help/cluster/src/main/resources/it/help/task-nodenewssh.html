Per completare la creazione di un nodo SSH

Questo task è la continuazione di Per creare un nodo per un nodo SSH.

  1. Per copiare il software di GlassFish Server dall'host del server DAS all'host del nodo, abilitare l'opzione Installa GlassFish Server.

  2. Per forzare la creazione del nodo nella configurazione del server di amministrazione del dominio (DAS) anche se la convalida dei parametri del nodo non riesce, abilitare l'opzione Imponi.

  3. Nel campo Porta SSH digitare la porta da utilizzare per le connessioni SSH all'host del nodo.

  4. Nel campo Nome utente SSH digitare il nome dell'utente nell'host del nodo che deve eseguire il processo per la connessione all'host tramite SSH.

  5. Specificare la modalità di autenticazione dell'utente SSH per il login all'host del nodo.

    • Per specificare l'autenticazione tramite un file di chiave privata SSH, attenersi alla procedura riportata di seguito.

      1. Nell'elenco a discesa Autenticazione utente SSH selezionare File di chiave.

      2. Se si desidera che GlassFish Server imposti una chiave SSH sull'host del nodo, abilitare l'opzione Imposta SSH.

        GlassFish Server imposta la chiave SSH copiando i file di chiave esistenti dall'host del server DAS o, se necessario, generando i file prima di copiarli.

        I campi Genera chiave e File di chiave pubblica vengono aggiunti alla finestra.

      3. Se si desidera che GlassFish Server generi i file di chiave SSH prima di copiare i file nell'host del nodo, abilitare l'opzione Genera chiave.

        Se i file di chiave SSH esistono, GlassFish Server non li genera, anche se si seleziona l'opzione Abilitato.

        Nota:

        Se i file di chiave SSH non esistono, è necessario selezionare l'opzione Abilitato. In caso contrario, si verifica un errore quando si tenta di salvare le modifiche.

      4. Nel campo Password utente SSH digitare la password che verrà utilizzata dall'utente SSH per collegarsi all'host del nodo.

      5. Nel campo File di chiave pubblica digitare il percorso assoluto del file di chiave pubblica SSH per l'utente specificato dall'opzione Nome utente SSH.

      6. Nel campo File di chiave digitare il percorso assoluto del file di chiave pubblica SSH per l'utente specificato dall'opzione Nome utente SSH.

        Nota:

        GlassFish Server supporta anche l'autenticazione della password tramite la voce AS_ADMIN_SSHPASSWORD del password file. Il password file è specificato nell'opzione --passwordfile della utility asadmin(1).

    • Per specificare l'autenticazione tramite una password memorizzata nella configurazione del server DAS, attenersi alla procedura riportata di seguito.

      Attenzione:

      La password viene memorizzata come testo in chiaro nella configurazione del server DAS. Per maggiore sicurezza, si consiglia di utilizzare un file di chiave o un alias password.

      1. Nell'elenco a discesa Autenticazione utente SSH selezionare Password.

      2. Nel campo Password utente SSH digitare la password che verrà utilizzata dall'utente SSH per collegarsi all'host del nodo.

    • Per specificare l'autenticazione tramite un alias password, attenersi alla procedura riportata di seguito.

      1. Nell'elenco a discesa Autenticazione utente SSH selezionare Alias password.

      2. Nell'elenco a discesa Alias password selezionare un alias esistente.

  6. Fare clic su Salva.
