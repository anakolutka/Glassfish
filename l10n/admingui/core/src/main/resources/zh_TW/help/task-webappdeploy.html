部署 Web 應用程式

  1. 在瀏覽樹狀結構中，展開 [應用程式] 節點。
  2. 在 [應用程式] 節點下，選取 [Web 應用程式] 節點。

    [Web 應用程式] 頁面會隨即開啟。

  3. 在 [已部署的 Web 應用程式] 表格中，按一下 [部署] 按鈕。

    [部署企業應用程式/模組] 頁面會隨即開啟。

  4. 從 [類型] 下拉式清單中，選取 [Web 應用程式 (.war)]。
  5. 指定包含要部署之應用程式的歸檔位置。
    • 如果歸檔是常駐在用戶端機器上或可從用戶端機器存取的檔案，請選取 [要上傳至伺服器的封裝檔案] 選項。

      用戶端是您要在其上透過瀏覽器檢視管理主控台的主機。

      按一下 [瀏覽] 以瀏覽至該檔案，或者鍵入該檔案的完整路徑。

    • 如果歸檔是常駐在伺服器機器上的檔案，或是展開的目錄中未封裝的應用程式，請選取 [可從應用程式伺服器存取的本機封裝檔案或目錄] 選項。

      伺服器機器是執行網域管理伺服器 (DAS) 的主機。

      鍵入該檔案或目錄的完整路徑名稱。從展開的目錄部署適用於進階開發者，但不建議在生產環境下進行該作業。

  6. 在 [應用程式名稱] 欄位中，保留預設名稱 (即檔案名稱的前綴)，或鍵入其他名稱。

    如果您選擇上傳檔案，則會顯示預設名稱。應用程式名稱不得重複。

  7. 在 [環境根目錄] 欄位中，輸入可識別 Web 應用程式的字串。

    在 Web 應用程式的 URL 中，環境根目錄緊跟在連接埠號碼之後 (http://host: port/context-root/...)。環境根目錄的開頭必須是正斜線，例如 /hello。

  8. 若要停用應用程式使其無法使用，請取消選取 [已啟用] 核取方塊。

    應用程式在部署後的狀態預設為 [已啟用]。

  9. 選取 [啟用執行檢驗器] 核取方塊，以驗證檔案的結構與內容，再進行部署。

    檢驗大型應用程式非常費時。如果您懷疑檔案已毀壞或不可攜式，請對其進行檢驗。

  10. 選取 [啟用預編譯 JSP] 核取方塊，以預編譯 JavaServer PagesTM (JSPTM) 檔案。

    若未選取此核取方塊，則首次存取 JSP 檔案時會在執行階段編譯這些檔案。由於編譯通常很耗時，因此，在生產環境中請選取該核取方塊。

  11. 在 [程式庫] 欄位中，將列出應用程式所需的所有程式庫 JAR 檔案。
  12. 在 [說明] 欄位中，鍵入此 Web 應用程式的說明。
  13. 按一下 [確定]。
