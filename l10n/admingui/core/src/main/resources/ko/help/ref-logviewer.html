로그 뷰어

로그 뷰어 페이지에서는 서버 로그를 보고 로그 표시를 사용자 정의할 수 있습니다.

로그 뷰어 페이지에는 다음 세 영역이 포함되어 있습니다.

  • 검색 기준

  • 고급 검색

  • 검색 결과

검색 기준 영역에는 다음 옵션이 포함되어 있습니다.

고급 검색

이 링크를 누르면 로그 뷰어를 추가적으로 조정할 수 있는 영역이 열립니다.

타임스탬프

가장 최근 라디오 버튼은 가장 최근 메시지를 표시합니다. 특정 범위 라디오 버튼은 특정 기간의 메시지만 볼 수 있는 시작 및 끝 필드를 엽니다.

시간 값의 경우 구문은 다음 형식을 따라야 합니다. 여기서 SSS는 밀리초를 나타냅니다.

hh:mm:ss.SSS

예를 들면 다음과 같습니다.

17:10:00.000

시작 값이 끝 값보다 이후일 경우 오류 메시지가 표시됩니다.

로그 수준

필터링에 대해 선택한 로그 수준입니다. 기본적으로 선택한 로그 수준과 더 심각한 수준에서 서버 로그에 표시되는 모든 메시지가 표시됩니다. 선택한 수준의 메시지만 표시하려면 "더 심각한 메시지는 포함되지 않습니다." 확인란을 선택합니다.

확인할 메시지를 서버 로그에 표시하려면 먼저 모듈 로그 수준 페이지에서 적절한 로그 수준을 설정합니다. 로그 수준을 구성하는 방법을 참조하십시오.

로그 수준을 기준으로 로그 메시지를 필터링하도록 선택한 경우 지정한 필터 기준에 맞는 메시지만 표시됩니다. 그러나 이 필터링은 서버 로그에 어떤 메시지를 기록할지에는 영향을 주지 않습니다.

고급 검색 영역에는 다음 옵션이 포함되어 있습니다.

로거

모듈별로 필터링할 로거 이름 공간입니다. Shift 키나 Ctrl 키를 사용하면 이름 공간을 여러 개 선택할 수 있습니다.

더 높은 수준의 이름 공간을 선택하면 그 아래에 있는 모든 이름 공간이 선택됩니다. 예를 들어, javax.enterprise.system을 선택하면 해당 이름 공간 아래의 모든 모듈(javax.enterprise.system.core, javax.enterprise.system.tools.admin 등)에 대한 로거도 선택됩니다.

사용자 정의 로거

한 줄당 하나씩의 특정 응용 프로그램에 대한 로거입니다.

응용 프로그램에 여러 모듈이 있을 경우 그 중 일부 또는 모두를 확인할 수 있습니다. 예를 들어 응용 프로그램에 다음과 같은 이름의 로거가 있다고 가정합니다.

com.mycompany.myapp.module1
com.mycompany.myapp.module2
com.mycompany.myapp.module3

응용 프로그램에 있는 모든 모듈의 메시지를 확인하려면 com.mycompany.myapp를 입력합니다. module2의 메시지만 확인하려면 com.mycompany.myapp.module2를 입력합니다.

사용자 정의 로거를 하나 이상 지정한 경우 해당 로거를 로거 영역에 명시적으로 지정해야만 Enterprise Server 모듈의 메시지가 표시됩니다.

이름-값 쌍

특정 스레드의 출력을 보기 위한 스레드에 대한 키 이름 및 값입니다. 키 이름은 _ThreadID입니다. 예를 들면 다음과 같습니다.

_ThreadID=13

com.mycompany.myapp.module2가 여러 스레드에서 실행되는 것으로 가정합니다. 단일 스레드의 출력만 표시하도록 로그 뷰어를 구체화하려면 사용자 정의 로거 필드에 해당 모듈의 로거를 지정한 다음 이 필드에 스레드 아이디를 지정합니다.

디스플레이

한 번에 볼 메시지의 수입니다. 표시되는 기본 메시지 수는 40입니다. 기타 선택 항목은 100, 250 및 1000입니다.

스택 추적을 보려면 “과도하게 긴 메시지 제한” 확인란을 선택 취소합니다. 기본적으로 스택 추적은 뷰어에 표시되지 않습니다. 스택 추적을 보려면 메시지의 (세부 사항) 링크를 누릅니다.

검색 결과 테이블에는 서버 로그 메시지가 표시됩니다.

각 메시지에 대해 검색 결과 페이지에는 다음 정보가 포함됩니다.

레코드 번호

서버 로그에서 메시지의 번호입니다.

로그 수준

메시지의 로그 수준입니다.

메시지

로그 메시지의 간략한 텍스트입니다. (세부 사항) 링크를 누르면 형식이 지정된 버전의 메시지를 볼 수 있습니다.

로거

로그 메시지를 생성한 로거 모듈입니다.

타임스탬프

로그 메시지가 발행된 날짜와 시간입니다.

이름/값 쌍

로그 메시지의 스레드 이름 및 ID입니다.

관련 작업

  • 로그 수준을 구성하는 방법

  • 서버 로그를 보는 방법

  • rotate-log(1)
