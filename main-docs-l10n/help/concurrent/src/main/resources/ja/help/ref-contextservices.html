コンテキスト・サービス

コンテキスト・サービスは、コンテナのコンテキストを取得し、後からアプリケーションをそのコンテキスト内で実行できるようにする動的プロキシ・オブジェクトの作成に使用されます。コンテナのコンテキストは、タスクを実行するスレッドに伝播されます。

コンテキスト・サービス・リソースを構成するには、「コンテキスト・サービス」ページを使用します。

コンテキスト・サービス・リソースごとに、次の情報が表示されます。

JNDI名

リソースを識別する一意の名前。

論理JNDI名

リソースの論理JNDI名。詳細は、「論理JNDI名」を参照してください。

有効

(この列は、ドメインにデフォルト・サーバー・インスタンスserverのみが存在する場合に表示されます。)

リソースが有効な場合はチェックマーク、リソースが無効な場合はX。

ステータス

(この列は、ドメイン内にクラスタまたはスタンドアロン・インスタンスが作成されている場合にのみ表示されます。)

リソースのターゲットであるクラスタとスタンドアロン・インスタンスの数、およびそれらのターゲットのうち、リソースが有効化されている数。たとえば、「4ターゲットの2で有効」は、リソースが4つのクラスタとスタンドアロン・インスタンスをターゲットとしており、それら4つのターゲットのうち、2つで有効化されていることを意味します。

コンテキスト情報

スレッドに伝播されたコンテナ・コンテキスト。

説明

リソースの説明。

「コンテキスト・サービス」の表には、次のオプションも表示されます。

新規

新規リソースを作成するボタン。

削除

選択した1つまたは複数のリソースを削除するボタン。

有効

選択した1つまたは複数のリソースを有効にするボタン。

無効

選択した1つまたは複数のリソースを無効にするボタン。
