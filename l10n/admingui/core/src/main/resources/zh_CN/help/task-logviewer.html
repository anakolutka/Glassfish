查看服务器日志

  1. 在导航树中，选择 "Application Server" 节点。

    将打开“常规信息”页面。

  2. 在“常规信息”页面上，单击“查看日志文件”按钮。

    此时将打开“日志查看器”窗口。

  3. 使用“搜索条件”区域自定义和过滤日志查看器。
    • 单击“搜索条件”区域中的“高级搜索”以进一步细化日志查看器的搜索条件。
    • 在“时间戳”字段中：
      • 要查看最近的消息，请选择“最近”（默认值）。
      • 如果只查看特定时间段内的消息，请选择“特定范围”并在显示的“从”字段和“到”字段中键入日期和时间值。

        对于时间值，其语法必须采用以下格式（SSS 表示毫秒）：

        hh:mm:ss.SSS

        例如：

        17:10:00.000

        如果“从”字段中的时间值迟于“到”字段中的时间值，将显示错误消息。

    • 从“日志级别”下拉式列表中，选择日志级别以根据日志级别过滤消息。选中标有“请勿包括更高级别的消息”的复选框将仅显示所选级别的消息。

      默认情况下，将显示服务器日志中选定日志级别和更高日志级别的所有消息。

      为确保要查看的消息都显示在服务器日志中，请先在“模块日志级别”页面上设置适当的日志级别。请参见配置日志级别。

      如果选择基于日志级别过滤日志消息，则将只显示符合指定过滤条件的消息。不过，这种过滤不影响将哪些消息记录到服务器日志中。

  4. 如果在“搜索条件”区域中单击了“高级搜索”，请执行以下步骤：
    1. 从“日志程序”下拉式列表中，选择一个或多个名称空间以根据模块过滤。

      通过按住 Shift 键单击或按住 Ctrl 键单击选择多个名称空间。

      选择较高级别的名称空间也就选择了该名称空间下的所有名称空间。例如，选择 javax.enterprise.system 的同时也就选择了该名称空间下所有模块的日志程序： javax.enterprise.system.core、javax.enterprise.system.tools.admin 等等。

    2. 在“自定义日志程序”字段中，键入日志程序名称（一行一个），以查看特定于特定应用程序的日志程序中的消息。

      如果应用程序具有多个模块，您可以查看任何模块，也可以查看所有模块。例如，假定应用程序具有使用以下名称的日志程序：

      com.mycompany.myapp.module1
com.mycompany.myapp.module2
com.mycompany.myapp.module3

      要查看应用程序中所有模块的消息，请键入 com.mycompany.myapp。要只查看 module2 的消息，请键入 com.mycompany.myapp.module2。

      指定一个或多个自定义日志程序时，仅当在“日志程序”区域中显式指定了 Enterprise Server 模块的消息，才会显示这些消息。

    3. 在“名称-值对”字段中，键入此线程的关键字名称和值，以查看特定线程的输出。

      关键字名称为 _ThreadID。例如：

      _ThreadID=13

      假定 com.mycompany.myapp.module2 在多个线程中运行。要细化日志查看器，使其仅显示单个线程的输出，请在“自定义日志程序”字段中指定模块的日志程序，然后在该字段中指定线程 ID。

    4. 从“显示”下拉式列表中，选择一个值，以便一次查看此数目的消息。

      所显示消息的默认数目为 40。其他选项为 100、250 和 1000。

      要查看堆栈追踪，请取消选中“限制过长的消息”复选框。默认情况下，查看器中不会显示堆栈追踪，要查看堆栈追踪，请单击消息的（详细信息） 链接。

    5. 单击“基本搜索”可以隐藏“高级选项”区域。
  5. 在“搜索结果”区域中，自定义日志消息的显示。

    “搜索结果”区域中将按照“日志设置”页面和“日志级别”页面中指定的设置显示最近的 40 条服务器日志条目。

    • 单击列中的箭头，以不同方式对消息进行排序。
    • 单击标有 (details) 的链接，以查看任何消息的格式化版本。

      将显示标有“日志条目详细资料”的窗口，该窗口包含了消息的格式化版本。

    • 单击表顶部和底部的按钮，以查看日志文件中较早或较晚的条目。
