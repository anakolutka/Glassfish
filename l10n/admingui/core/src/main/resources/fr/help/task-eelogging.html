Configurer les paramètres généraux de journalisation

  1. Dans l'arborescence de navigation, développez le nœud Configuration.
  2. Sous le nœud Configuration, sélectionnez le nœud Paramètres du .

    La page Paramètres du s'ouvre.

  3. Cochez la case Ecriture sur le journal système activée pour envoyer le résultat de la journalisation à l'utilitaire syslog en plus du journal du serveur.

    Cette fonction est disponible sur les systèmes Solaris et Linux uniquement. Cette option est désactivée par défaut.

  4. Cochez la case Journaliser sur la console pour NE SAIS PAS

    Cette option est désactivée par défaut.

  5. Dans le champ Fichier journal, saisissez un autre nom ou emplacement pour le fichier journal du serveur.

    L'emplacement par défaut est glassfish/domains/domain1/logs/server.log.

  6. Dans le champ Gestionnaire de journaux, saisissez le nom de classe absolu d'un gestionnaire de journal personnalisé.

    Un gestionnaire de journal personnalisé vous permet d'envoyer des journaux vers une destination autre que server.log ou syslog. Celui-ci doit venir en extension de la classe java.util.logging.Handler (API compatible JSR 047). Ajoutez la classe du gestionnaire au classpath du Enterprise Server de façon à installer le gestionnaire lors du démarrage du serveur.

  7. Dans le champ Limite de rotation du fichier, saisissez la taille en octets que le fichier journal du serveur doit atteindre avant sa rotation.

    Lorsque le journal du serveur atteint la taille indiquée, le Enterprise Server crée un nouveau fichier vide nommé server.log et renomme l'ancien fichier server.log_date, où date indique la date et l'heure de rotation du fichier. La valeur par défaut est 2 mégaoctets. La valeur minimale est 500 ko. Si vous spécifiez une valeur inférieure, le fichier effectuera la rotation à 500 ko. Pour désactiver la rotation du fichier journal, spécifiez la valeur 0.

  8. Dans le champ Temps limite de rotation du fichier, saisissez le nombre de minutes au bout duquel la rotation du journal du serveur doit s'effectuer.

    La valeur par défaut est 0, ce qui signifie que le fichier effectuera la rotation lorsqu'il aura atteint la taille spécifiée dans le champ Limite de rotation du fichier.

    Remarque - Si vous spécifiez une ou plusieurs minutes, le temps limite remplace la taille limite indiquée par le champ Limite de rotation du fichier.

  9. Cliquez sur Enregistrer.
  10. Arrêtez et redémarrez Enterprise Server.
