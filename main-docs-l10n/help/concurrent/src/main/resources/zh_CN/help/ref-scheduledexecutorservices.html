托管调度执行程序服务

托管的已调度执行程序服务由应用程序用于在特定时间异步执行已提交任务。任务在由容器启动和管理的线程上执行。容器的上下文将传播到执行任务的线程。

使用“托管的已调度执行程序服务”页可配置托管的已调度执行程序服务资源。

对于每个托管的已调度执行程序服务资源，将提供以下信息。

JNDI 名称

标识资源的唯一名称。

逻辑 JNDI 名称

资源的逻辑 JNDI 名称。有关详细信息，请参阅逻辑 JNDI 名称。

已启用

(如果域中只有默认服务器实例 server，则显示此列。)

如果资源已启用，则显示复选标记，如果资源已禁用，则显示 X。

状态

(如果域中已创建任何集群或独立实例，则显示此列。)

资源定向到的集群和独立实例数以及在多少个这些目标上启用了资源。例如，“在 2 个 (共 4 个) 目标上启用”表示资源定向到四个集群和独立实例，并且在这四个目标中的两个目标上启用。

上下文信息

传播到线程的容器上下文。

线程优先级

分配给线程的优先级。

说明

资源的说明。

“托管调度执行程序服务”表还包含以下选项。

新建

用于创建新资源的按钮。

删除

用于删除一个或多个所选资源的按钮。

启用

用于启用一个或多个所选资源的按钮。

禁用

用于禁用一个或多个所选资源的按钮。
