Wenn Sie als Entwickler tätig sind und Teil der Community werden möchten, die den ersten kompatiblen Java EE 5-Anwendungsserver implementiert hat, dann sind Sie bei Projekt GlassFish genau richtig!

Mitglieder der GlassFish-Community werden dazu aufgerufen, sich für eine selbstverwaltete Mitgliederseite im GlassFish-Wiki zu registrieren. Es gibt auch eine etwas schöner gestaltete Seite für die Suche nach Entwicklern, die ständig erweitert werden soll.

Verstehen des Codes

Die Entwicklung eines Anwendungsservers ist ein großes Unterfangen. Die nachstehenden Links ermöglichen es uns, ein Projekt dieser Größenordnung zu organisieren.

  • Das GlassFish-Projekt ist in Module unterteilt
  • Lesen Sie die javadocs, die mit jedem Promoted-Build aktualisiert werden
  • Tauschen Sie sich mit anderen GlassFish-Entwicklern aus, und besprechen Sie in den Mailingslisten Details zur Implementierung. Sie sollten sich in jedem Fall alle verfügbaren Mailinglisten anschauen.
  • Prüfen Sie die Liste der Modulbesitzer
  • Zeigen Sie eine Aufstellung der Technologien an, die Bestandteil von GlassFish sind

Erstellung

Die Build-Infrastruktur von GlassFish basiert auf Maven und Ant. Derzeit suchen wir Entwickler, die uns bei der Verbesserung der Build-Infrastruktur helfen, um weniger abhängig von Ant zu sein.

  • Die Kurzanleitung umfasst eine kurze Liste der Schritte, die zum Erstellen des Servers nötig sind
  • Die vollständigen Anweisungen beschreiben die Erstellung von GlassFish im Detail

Testen

In GlassFish stehen derzeit verschiedene Tests zur Verfügung. QuickLook-Tests weisen eine kurze Ausführungszeit auf und decken eine breite Palette an Features im Anwendungsserver ab. Entwicklertests für die verschiedenen Module stehen ebenfalls zur Verfügung. Jedes Modul umfasst weitere Informationen zur Ausführung dieser Tests.

Weitere Informationen finden Sie im GlassFish-Qualitätsportal.

  • Hier finden Sie einen Nightly- oder Promoted-Build für den Download
  • Führen Sie einen der verfügbaren QuickLook-Tests aus
  • Treten Sie dem CATfish-Programm bei
  • Melden Sie einen Bug

Code beitragen

Sie können auf verschiedene Weise zum Projekt GlassFish beitragen: durch Bugfixes, Erweiterungen, neue Features oder sogar ganze Module.

  • Alle Mitwirkenden müssen eine Vereinbarung für Mitwirkende unterschreiben und einsenden
  • Weitere Informationen zu Rollen und Verantwortlichkeiten finden Sie in den Projektrichtlinien
  • Sämtlicher Code, der in die Arbeitsumgebung eingecheckt wird, muss den Codierungkonventionen entsprechen
  • Wenn Sie noch nicht als Entwickler registriert sind, lesen Sie die Vorgehensweise zum Einreichen von Patches
  • Entwickler müssen sich an die Hinweise zum Arbeitsverfahren halten, wenn sie die Arbeitsumgebung aktualisieren
