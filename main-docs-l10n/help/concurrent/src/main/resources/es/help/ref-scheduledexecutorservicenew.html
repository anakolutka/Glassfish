Nuevo servicio de ejecutor programado gestionado

Utilice la página Nuevo servicio de ejecutor programado gestionado para crear un recurso del servicio de ejecutor programado gestionado.

La página Nuevo servicio de ejecutor programado gestionado contiene las siguientes opciones.

Nombre JNDI

Nombre de JNDI único que identifica el recurso. Por convención, el nombre empieza por un indicador de tipo de recurso y una barra invertida, por ejemplo, concurrent/, jdbc/ o jms/.

Información de contexto

Contextos que se propagarán a los threads: Classloader, JNDI, seguridad o área de trabajo. La propagación de contexto se puede activar o desactivar. Si está activada, se propagan los contextos seleccionados. Si está desactivada, no se propaga ninguno de los contextos, aunque estén seleccionados. El valor por defecto es Activado, con los cuatro contextos seleccionados.

Estado

Estado del recurso. El recurso se puede activar o desactivar. El valor por defecto es Activado.

Prioridad de threads

Prioridad que se asignará a los threads creados. El valor por defecto es 5.

Tareas de larga ejecución

Establece si el recurso se debe utilizar para las tareas de larga duración. Si está activada, las tareas de larga ejecución no se incluirán en el informe como paradas. Esta opción se activada por defecto.

Bloquear después

Número de segundos que se pueden ejecutar las tareas antes de que se considere que no responden. El valor por defecto es 0.

Descripción

Descripción de este recurso.

Tamaño principal

Número de threads que mantener en un pool de threads. El valor por defecto es 0.

Mantenimiento de conexiones

Número de segundos que los threads puede permanecer inactivos cuando el número de threads es mayor que el tamaño principal. El valor por defecto es 60.

Duración de threads

Número de segundos que los threads pueden permanecer en el pool de threads antes de que se depuren, independientemente de si el número de threads es mayor que el tamaño principal o si están inactivos. El valor por defecto es 0.

Propiedades Adicionales

Propiedades adicionales del recurso. GlassFish Server no define ninguna propiedad adicional para este tipo de recurso. Además, este tipo de recurso no utiliza actualmente ninguna propiedad adicional.

Destinos

(Esta opción sólo se muestra si en el dominio se ha creado algún cluster o instancia independiente).

Clusters e instancias independientes para el recurso. Mueva los destinos seleccionados a la columna Destinos Seleccionados mediante los botones Agregar y Agregar Todos. Mueva los destinos que no necesite a la columna Destinos Disponibles mediante los botones Eliminar y Eliminar Todos.
