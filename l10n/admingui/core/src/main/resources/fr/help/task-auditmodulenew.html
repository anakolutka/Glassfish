Créer un module d'audit

Les modules d'audit contrôlent la manière dont le Enterprise Server consigne les événements importants, comme les erreurs et les brèches de sécurité. Le Enterprise Server consigne tous les événements d'authentification dans des fichiers journaux. Un journal d'accès fournit un suivi séquentiel des événements d'accès du Enterprise Server.

Enterprise Server fournit un simple module d'audit par défaut ; pour plus d'informations, consultez Utiliser le module d'audit par défaut .

  1. Dans l'arborescence de navigation, développez le nœud Configuration.
  2. Sous le nœud Configuration, développez le nœud Sécurité.
  3. Sous le nœud Securité, sélectionnez le nœud Modules d'audit.

    La page Modules d'audit s'affiche.

  4. Sur la page Modules d'audit, cliquez sur Nouveau.

    La page Nouveau module d'audit s'affiche.

  5. Sur la page Nouveau module d'audit, dans le champ Nom, saisissez le nom du module d'audit.
  6. Dans le champ Nom de la classe, saisissez le nom complet de la classe qui implémente ce module.

    Le nom de classe du module d'audit par défaut est com.sun.enterprise.security.Audit.

  7. (Facultatif) Dans la section Autres propriétés, spécifiez d'autres propriétés.

    Pour ajouter une propriété, cliquez sur le bouton Ajouter propriété. Sur la ligne vide qui apparaît, saisissez le nom de la propriété dans le champ Nom et la valeur de la propriété dans le champ Valeur.

    Pour obtenir une description de la propriété supplémentaire disponible pour les modules d'audit, consultez Propriétés propres aux modules d'audit.

  8. Cliquez sur OK.
