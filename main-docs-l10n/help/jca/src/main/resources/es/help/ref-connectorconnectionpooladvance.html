Editar atributos avanzados del pool de conexiones de conector

Utilice la página Editar atributos avanzados del pool de conexiones de conector para modificar la configuración avanzada de un pool de conexiones del conector.

La página Editar atributos avanzados del pool de conexiones de conector presenta las opciones siguientes:

Cargar Valores por Defecto

Botón para restaurar configuración que tiene valores por defecto a sus valores por defecto. Las configuraciones que no tienen valores por defecto no se modifican.

Validar como máximo una vez

Especifica el intervalo de tiempo, en segundos, entre solicitudes consecutivas para validar una conexión como máximo una vez. El valor por defecto es 0, lo que significa que el atributo no está activado.

Timeout de pérdida

Duración de tiempo, en segundos, para realizar un seguimiento de las pérdidas de conexión en un pool de conexiones. El valor 0 indica que el seguimiento de pérdidas de conexión está desactivado. El valor por defecto es 0.

Recuperación de pérdida:

Si esta opción está seleccionada, las conexiones perdidas se restaurarán en el pool una vez finalizado el seguimiento de las pérdidas de conexión. Esta opción se activada por defecto.

Reintentos de creación

Número de intentos que se realizarán si se produce un fallo al crear una nueva conexión. El valor de 0 significa que no se realizará ningún intento para crear la conexión de nuevo. El valor por defecto es 0.

Intervalo de reintento

Intervalo, en segundos, entre dos intentos para crear una conexión. Este atributo se utiliza sólo si el valor de Intentos de creación es mayor que 0. El valor por defecto es 10 segundos.

Pool

Permite crear pool de conexiones en el pool. Esta opción se encuentra activada por defecto.

Asociación lenta

Si se selecciona esta opción, las conexiones se asocian de forma lenta cuando se realiza una operación en ellas. Las conexiones se disocian cuando la transacción se completa y un método de componentes finaliza, lo que ayuda a reutilizar las conexiones físicas. Esta opción se activada por defecto.

Inscripción lenta de conexiones

Si se selecciona esta opción, sólo se inscribe un recurso en la transacción cuando realmente se utiliza en un método. Esta opción se activada por defecto.

Asociar a thread

Si esta opción está seleccionada, la conexión está asociada a un thread. Cuando el mismo thread necesita una conexión, se puede volver a utilizar la conexión ya asociada a ese thread y, por tanto, no se incurra en una sobrecarga a la hora de obtener una conexión del pool. Esta opción se activada por defecto.

Conexiones coincidentes

Si selecciona esta opción, la coincidencia de conexiones está activada en el pool. Esta opción se puede desactivar si sabe que las conexiones del pool siempre serán homogéneas y, por tanto, el adaptador de recursos no necesitará buscar una coincidencia de una conexión seleccionada del pool. Esta opción se encuentra activada por defecto.

Uso de conexiones coincidentes

Número de veces que el pool debe reutilizar una conexión. La conexión se cerrará una vez reutilizada durante un número específico de veces. El valor 0 indica que el atributo no está activado. El valor por defecto es 0.
