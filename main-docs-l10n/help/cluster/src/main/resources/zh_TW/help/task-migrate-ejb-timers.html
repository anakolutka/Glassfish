遷移 EJB 計時器

如果 GlassFish Server 執行處理停止，則該執行處理上安裝的 Enterprise JavaBeans (EJB) 技術計時器也會停止執行。如果執行處理異常停止或意外地停止，則為了讓這些計時器在執行處理停止時繼續執行，您必須將計時器遷移至叢集內正在執行的伺服器執行處理。

  1. 在瀏覽樹狀結構中，展開「叢集」節點。

  2. 在瀏覽樹狀結構的「叢集」節點下，選取叢集。

    此時將開啟叢集的「一般資訊」頁面。

  3. 在「一般資訊」頁面上，按一下「遷移 EJB 計時器」。

    此時將開啟「遷移 EJB 計時器」頁面。

    備註:

    如果叢集內的所有執行處理都在執行，則會關閉「遷移 EJB 計時器」按鈕。

    為了協助您遷移計時器，「遷移 EJB 計時器」頁面包含 EJB 計時器表格與下列資訊。

    伺服器名稱

    叢集中 GlassFish Server 執行處理的清單。

    計時器計數

    每個執行處理上的永久性計時器數目。

  4. 在「遷移 EJB 計時器」頁面上，選取計時器的遷移來源執行處理和目標執行處理。

    1. 從「來源」下拉式清單中，選取要遷移的計時器所在的已停止的執行處理。

    2. 從「目的地」下拉式清單中，選取要將計時器遷移至的正在執行的執行處理。

    3. 按一下「確定」。

如果來源執行處理正在執行或目的地執行處理未執行，則管理主控台將顯示錯誤訊息。
