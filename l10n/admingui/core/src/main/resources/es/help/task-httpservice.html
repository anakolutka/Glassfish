Para configurar el servicio HTTP

  1. En el árbol de navegación, expanda el nodo Configuración.
  2. En el nodo Configuración, seleccione el nodo Servicio HTTP.

    Se abrirá la página Servicio HTTP.

  3. Marque la casilla de verificación Registro de acceso activado para habilitar el registro de acceso de todos los subelementos de servidores virtuales que no tengan especificada esta propiedad.

    Si anula la selección de la casilla de verificación Registro de acceso activado, se desactivará el registro de acceso de todos los subelementos de servidores virtuales que no tengan especificada esta propiedad.

  4. En el campo Tamaño de búfer de registro de acceso, introduzca el tamaño, en bytes, del búfer de registro de acceso.

    Un valor inferior o igual a cero deshabilita el almacenamiento en el búfer del registro de acceso. El valor predeterminado es 4096.

  5. En el campo Intervalo de escritura del registro de acceso, introduzca el número de segundos que deben transcurrir para que el registro se escriba en el disco.

    La escritura del registro de acceso tiene lugar cuando el búfer está lleno o cuando caduca el intervalo. Si el valor es igual o menor que cero, el búfer siempre se escribirá aunque no esté lleno. Esto significa que, cada vez que se accede al servidor, el mensaje de registro se almacena directamente en el archivo. El valor predeterminado es 0.

  6. (Opcional) En la sección Propiedades adicionales, especifique las propiedades adicionales.

    Están disponibles las propiedades siguientes:

    connectionTimeout

    Especifica el número de milisegundos que deben esperar las escuchas HTTP, después de aceptar una conexión, para que se presente la línea URI de solicitud.

    El valor predeterminado es 12.000 (12 segundos).

    maxKeepAliveRequests

    Especifica el número máximo de solicitudes HTTP que se pueden canalizar hasta que el servidor cierre la conexión. Establezca esta propiedad en 1 para deshabilitar el mantenimiento HTTP/1.0, así como la canalización y el mantenimiento HTTP/1.1.

    El valor predeterminado es 1.000.

    traceEnabled

    Si se establece en "true", permite el método TRACE. Establezca esta propiedad en "false" para que Servidor empresarial sea menos susceptible a ataques de secuencia de comandos entre sitios.

    Para obtener más información sobre el método TRACE, consulte RFC 2616: protocolo de transferencia de hipertexto -- HTTP/1.1.

    El valor predeterminado es Verdadero.

  7. Haga clic en "Guardar".
