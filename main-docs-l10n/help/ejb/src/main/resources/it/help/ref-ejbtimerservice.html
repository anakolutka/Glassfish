Servizio timer EJB

Utilizzare la pagina Servizio timer EJB per configurare il servizio timer EJB.

Il servizio timer è un servizio di notifica persistente e transazionale che viene fornito dal contenitore di bean enterprise e viene utilizzato per pianificare notifiche o eventi utilizzati dai bean enterprise. Tutti i bean enterprise, a eccezione dei bean di sessione con conservazione dello stato, possono ricevere notifiche dal servizio timer. I timer persistenti impostati dal servizio non vengono eliminati quando il server viene chiuso o riavviato.

La pagina Servizio timer EJB contiene le opzioni riportate di seguito.

Carica impostazioni predefinite

Pulsante per ripristinare i valori predefiniti delle impostazioni che dispongono di valori predefiniti. Le impostazioni che non dispongono di valori predefiniti non vengono modificate.

Nome configurazione

Il nome della configurazione a cui si applicano le impostazioni presenti in questa pagina. Questo campo è di sola lettura.

Intervallo minimo di consegna

Il numero minimo di millisecondi consentito prima della scadenza del timer successivo per un determinato timer. L'impostazione di questo intervallo su un valore troppo basso può causare il sovraccarico del server. Il valore predefinito è 1000.

N. massimo ripetizioni consegna

Il numero massimo di tentativi eseguiti dal servizio timer per notificare la scadenza di un timer dovuta a un'eccezione o al rollback. Il valore predefinito è 1.

Intervallo tra tentativi di consegna

L'intervallo, in millisecondi, tra i tentativi di consegna. Il valore predefinito è 5000.

Origine dati timer

Il nome JNDI della risorsa JDBC che verrà utilizzata come origine dati del timer. Se specificato, questo valore sostituisce il valore predefinito jdbc/__TimerPool per l'applicazione di sistema del servizio timer.
