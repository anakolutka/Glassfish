Sicherheit

Auf der Seite "Sicherheit" können Sie die Sicherheitseigenschaften von Enterprise Server einstellen.

Die Seite "Sicherheit" umfasst die folgenden Optionen.

Sicherheitsmanager

Wenn Sie diese Option aktivieren, wird der Sicherheitsmanager für diese Domäne aktiviert. Die Option ist standardmäßig deaktiviert.

Bei Aktivierung wird eine JVM-Option, -Djava.security.manager, zu den JVM-Einstellungen von Enterprise Server hinzugefügt. Siehe So konfigurieren Sie die JVM-Optionen. Um diese Änderung zu übernehmen, muss der Server neu gestartet werden.

Stellen Sie sicher, dass Sie über die erforderlichen Berechtigungen für alle Anwendungen verfügen. Der Sicherheitsmanager kann zum Verbessern der Leistung deaktiviert werden.

Überwachungsprotokollierung

Ist diese Option aktiviert, lädt der Server alle in der Liste "Überwachungsmodule" angegebenen Überwachungsmodule und führt sie aus. Ist die Option deaktiviert, greift der Server nicht auf Überwachungsmodule zu. Diese Option ist standardmäßig deaktiviert.

Standardbereich

Der aktive (standardmäßige) Bereich, den der Server für die Authentifizierung verwendet. Dieser Bereich wird von den Anwendungen verwendet, es sei denn, ihre Bereitstellungsbeschreibung gibt einen anderen Bereich an. Alle konfigurierten Bereiche werden in der Liste angezeigt. Der Standardwert ist file.

Standard-Principal

Gibt den standardmäßigen Benutzernamen an. Dieser Benutzername wird vom Server verwendet, wenn kein anderer Principal angegeben wird. Wenn Sie in diesem Feld einen Wert eingeben, muss auch das entsprechende Passwort im Feld "Standardmäßiges Principal-Passwort" angegeben werden.

Ein Standard-Principal ist für normale Server-Operationen nicht erforderlich.

Standardmäßiges Principal-Passwort

Das Passwort für den Standard-Principal, der im Feld "Standard-Principal" angegeben wurde. Ein Standard-Principal ist für normale Server-Operationen nicht erforderlich.

JACC

Klassenname eines konfigurierten JACC-Anbieters. Der Standardwert ist default.

Überwachungsmodule

Die Überwachungsanbietermodule, die vom Überwachungssubsystem verwendet werden, wenn die Überwachungsprotokollierung aktiviert ist. Standardmäßig verwendet der Server das Überwachungsmodul default. Weitere Informationen zum Erstellen neuer Überwachungsmodule finden Sie unter So erstellen Sie ein Überwachungsmodul.

Standardmäßige Principal/Rolle-Zuordnung aktiviert

Wenn Sie diese Option aktivieren, wird die Principal/Rolle-Zuordnung auf Anwendungen angewendet, die keine anwendungsspezifische Zuordnung aufweisen.

Zugeordnete Principal-Klasse

Legt eine benutzerdefinierte Implementierung der java.security.Principal -Klasse fest, die in der standardmäßigen Principal-to-role-Zuordnung verwendet wird, wenn die standardmäßige Principal-to-role-Zuordnung aktiviert ist.

Weitere Eigenschaften

Weitere Sicherheitseigenschaften des Servers.

Gültige Eigenschaften hängen vom Bereichstyp ab, der im Feld "Standardbereich" ausgewählt wurde, und werden üblicherweise angegeben, wenn Sie einen Bereich bearbeiten. Weitere Informationen hierzu finden Sie unter den folgenden Themen:
