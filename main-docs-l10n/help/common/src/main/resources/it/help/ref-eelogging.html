Impostazioni logger

Utilizzare la pagina Impostazioni logger per configurare il log per l'istanza o il cluster GlassFish Server selezionato.

La pagina Impostazioni logger contiene le opzioni riportate di seguito.

Carica impostazioni predefinite

Pulsante per ripristinare i valori predefiniti delle impostazioni che dispongono di valori predefiniti. Le impostazioni che non dispongono di valori predefiniti non vengono modificate.

Nome configurazione

Il nome della configurazione a cui si applicano le impostazioni presenti in questa pagina. Questo campo è di sola lettura.

Scrivi nel log di sistema

Quando questa opzione è abilitata, l'output di log viene inviato alla funzione syslog oltre che al log del server. Questa funzione è disponibile solo nei sistemi Solaris e Linux. Questa opzione è disabilitata per impostazione predefinita.

Log nella console

Quando questa opzione è abilitata, l'output di log viene inviato alla console del sistema. Questa opzione è disabilitata per impostazione predefinita.

Rotazione a cambio data

Quando questa opzione è abilitata, GlassFish Server esegue la rotazione dei file di log ogni giorno a mezzanotte. Questa opzione è disabilitata per impostazione predefinita.

Modalità multilinea

Quando questa opzione è abilitata, il corpo di un messaggio di log inizia in una nuova riga dopo l'intestazione del messaggio. Questa opzione è abilitata per impostazione predefinita.

Formato di log console

Il formato utilizzato per il collegamento alla console. Le scelte disponibili sono ULF (UniformLogFormatter) o ODL (Oracle Diagnostics Logging). Il formato predefinito è ULF.

Formato collegamento file di log

Il formato utilizzato per il collegamento al file di log del server. Le scelte disponibili sono ULF o ODL. Il formato predefinito è ODL.

Campi esclusione formato

I campi da escludere nei record di log. Tutti i campi sono inclusi per impostazione predefinita.

Limite di rotazione file

La dimensione in byte che il file di log del server deve raggiungere prima che ne venga eseguita la rotazione. Quando il log del server raggiunge la dimensione specificata, GlassFish Server crea un nuovo file vuoto denominato server.log e rinomina il file precedente server.log_data, dove data corrisponde alla data e all'ora di rotazione del file. Il valore predefinito è 2 megabyte. Il valore limite minimo è 500 KB; se si specifica un valore inferiore, il file viene ruotato quando la dimensione raggiunge 500 KB. Per disattivare la rotazione del file di log impostare il valore su 0.

Limite di tempo rotazione file

Il numero di minuti al termine dei quali deve essere eseguita la rotazione del log del server. Il valore predefinito è 0 e indica che il file viene sottoposto a rotazione quando raggiunge la dimensione specificata nel campo Limite di rotazione file. Se si specificano uno o più minuti, il limite di tempo ha la precedenza rispetto al limite di dimensione specificato nel campo Limite di rotazione file.

Frequenza flush

Il numero massimo di messaggi che possono essere scritti contemporaneamente dalla coda al log del server. Il valore predefinito è 1.

Numero massimo di file di cronologia

Il numero massimo di file di log che GlassFish Server conserva prima di eliminare il file meno recente. Se si imposta questo valore su zero (0), GlassFish Server non elimina alcun file di log meno recente.

File di log

Il nome alternativo o la posizione alternativa del file di log del server. La posizione predefinita è domain-dir/logs/server.log.

Handler di log

Il nome classe assoluto di un handler di log personalizzato. Un handler di log personalizzato consente di inviare i log a una destinazione diversa da server.log o syslog. L'handler personalizzato deve estendere la classe java.util.logging.Handler (un'interfaccia API conforme allo standard JSR 047). Inserire la classe handler nel classpath di GlassFish Server, in modo che l'handler venga installato durante l'avvio del server.
