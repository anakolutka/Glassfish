Cómo utilizar el módulo de auditoría predeterminado

Las entradas de registro de autenticación incluyen la información siguiente:

  • Nombres de usuarios que han intentado autenticarse.

  • El dominio que ha procesado la solicitud de acceso.

  • La dirección del módulo web o componente EJB solicitado.

  • Éxito o fallo de la solicitud.

Independientemente de si se activa el registro de auditoría, Servidor empresarial registra todos los sucesos de autenticación denegados.

Las entradas de registro de autorización incluyen la información siguiente:

  • Los nombres de los usuarios autenticados, si hay alguno.

  • La dirección web o componente EJB solicitado.

  • Éxito o fallo de las solicitudes.

  1. En el árbol de navegación, expanda el nodo Configuracións.
  2. Seleccione la instancia que va a configurar:
    • Para configurar una instancia específica, expanda el nodo de configuración de dicha instancia. Por ejemplo, en la instancia predeterminada server expanda el nodo server-config.
    • Para configurar los valores de configuración predeterminados de todas las instancias, expanda el nodo default-config.
  3. En el nodo Configuración, expanda el nodo Seguridad.
  4. En el nodo Seguridad, expanda el nodo Módulos de auditoría.
  5. En el nodo Módulos de auditoría, seleccione el nodo default.

    Se abrirá la página Editar módulo de auditoría.

  6. Establezcca el valor de la propiedad auditOn en true.
  7. Seleccione "Guardar".
  8. Reinicie Servidor empresarial si aparece el mensaje Reinicio necesario en la consola.
