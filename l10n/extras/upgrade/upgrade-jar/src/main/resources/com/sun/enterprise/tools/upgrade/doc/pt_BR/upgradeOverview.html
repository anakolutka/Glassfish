Upgrade Tool: visão geral

Use a Upgrade Tool para transferir as informações de configuração de uma instalação do Application Server 8.x ou Application Server 9 para uma instalação do Application Server 9.1. As informações transferidas constam de dados sobre aplicativos implantados, o reino do aplicativo, certificados de segurança e outras configurações de recursos e servidores.

A tabela a seguir mostra as atualizações suportadas do Application Server, na qual PE indica Platform Edition (edição da plataforma) e EE indica Enterprise Edition (edição corporativa).

A ferramenta detecta automaticamente a versão da instalação especificada do servidor de origem.

Observação: antes de iniciar o processo de atualização, certifique-se de que o servidor de origem (o servidor a partir do qual está realizando a atualização) e o servidor de destino (o servidor ao qual está realizando a atualização) estejam interrompidos.

Terminologia do processo de atualização

A seguir se encontram termos importantes relacionados ao processo de atualização:

  • Servidor de origem: instalação a partir da qual você está atualizando para a nova versão.

  • Servidor de destino: instalação para a qual você está atualizando.

  • Raiz de domínios: diretório no qual os domínios são criados. Por padrão, este diretório corresponde ao local especificado como AS_DEF_DOMAINS_PATH em asenv.conf

  • Diretório do domínio ou domain-dir: diretório (dentro da raiz de domínios) correspondente a um determinado domínio. Toda a configuração e outros dados pertencentes ao domínio estão neste diretório.

  • Raiz de instalação: diretório no qual o Application Server está instalado.

Se a caixa de verificação Upgrade tiver sido marcada durante o processo de instalação do Application Server, a tela Upgrade Wizard será exibida automaticamente após a conclusão da instalação. O processo de atualização se inicia.
