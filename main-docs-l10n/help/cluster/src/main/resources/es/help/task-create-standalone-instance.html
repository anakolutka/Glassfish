Crear una instancia de GlassFish Server independiente

Las instancias de GlassFish Server forman la base del despliegue de aplicaciones. Una instancia independiente no forma parte de un cluster. Cada instancia pertenece a un solo dominio y tiene su propia estructura de directorios, configuración y aplicaciones desplegadas. Cada instancia incluye también el contenedor web de la plataforma Java, Enterprise Edition (Java EE) y el contenedor de la tecnología Enterprise JavaBeans (tecnología EJB). Cada instancia nueva debe contener una referencia a un nodo que defina el host donde residirá la instancia.

Nota:

Para crear una instancia en un nodo de tipo CONFIG que representa un host remoto, debe utilizar el subcomando create-local-instance(1). No se puede utilizar Consola de administración para este fin.
Antes de Empezar

Asegúrese de que exista el nodo en el que reside la instancia. Consulte Crear un nodo para obtener instrucciones.

  1. En el árbol de navegación, seleccione el nodo Instancias Independientes.

    Se abrirá la página Instancias de Servidor Independientes.

  2. En la página Instancias de Servidor Independientes, haga clic en Nuevo.

    Se abrirá la página Nueva Instancia del Servidor Independiente.

  3. En el campo Nombre, escriba un nombre para la instancia.

    El nombre debe cumplir los siguientes requisitos:

    • El nombre sólo puede contener caracteres ASCII.

    • El nombre debe empezar por una letra, un número o un carácter de subrayado.

    • El nombre sólo puede contener los siguientes caracteres:

      • Letras en minúscula

      • Letras en mayúscula

      • Números

      • Guión

      • Punto

      • Subrayado

    • El nombre debe ser único en el dominio y no debe ser el nombre de otro nodo, cluster, configuración con nombre ni instancia de GlassFish Server.

    • El nombre no debe ser domain, server ni otra palabra clave reservada por GlassFish Server.

  4. En la lista desplegable Nodo, seleccione el nodo donde residirá la instancia.

  5. En la lista desplegable Configuración, seleccione la configuración con nombre que utilizará la instancia.

    Nota:

    Si selecciona default-config, debe especificar en el paso siguiente que la configuración no es compartida. La configuración de default-config no se puede compartir.

  6. Especifique si la configuración que usará la instancia es compartida.

    • Si la configuración no es compartida, deje seleccionada la opción Realizar una Copia de la Configuración Seleccionada.

      La configuración seleccionada se copia cuando se crea la instancia. Se asigna el nombre nombre_instancia-config a la copia de la configuración, donde nombre_instancia es el nombre de la instancia que está creando.

    • Si la configuración es compartida, seleccione la opción Hacer Referencia a la Configuración Seleccionada.

      La instancia utilizará la configuración con nombre especificada existente.

  7. Haga clic en Aceptar.
