Per avviare le istanze standalone di GlassFish Server

Prima di iniziare

Assicurarsi che siano soddisfatti i seguenti prerequisiti:

  • tutti i nodi in cui risiedono le istanze sono abilitati per la comunicazione remota;

  • l'utente del server di amministrazione del dominio (DAS) può utilizzare il protocollo remoto DCOM (Distributed Component Object Model) o SSH (Secure Shell) per collegarsi all'host per qualsiasi nodo in cui risiedono le istanze del cluster.

Se uno qualsiasi di questi prerequisiti non è soddisfatto, avviare singolarmente ogni istanza dalla riga di comando utilizzando il comando secondario start-local-instance(1).

  1. Selezionare il nodo Istanze standalone nell'albero di navigazione.

    Viene visualizzata la pagina Istanze server standalone.

  2. Per ogni istanza da avviare, selezionare la casella di controllo corrispondente nella tabella Istanze server.

  3. Fare clic su Avvia.

    Viene richiesto di confermare l'avvio delle istanze.

  4. Confermare che si desidera avviare le istanze.

    Lo stato di ogni istanza viene aggiornato nella tabella Istanze server quando l'istanza viene avviata.
