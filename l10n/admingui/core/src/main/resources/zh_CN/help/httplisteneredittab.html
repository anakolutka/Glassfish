“编辑 HTTP 侦听器”选项卡

通过编辑 HTTP 侦听器套接字，Enterprise Server 可以快速处理 HTTP 请求。

名称

侦听器的只读唯一标识符。

侦听器

为服务器输入一个有效的 IP 地址。

安全性

在“群集”配置文件和“企业”配置文件中可用。创建一个安全的侦听器。

网络地址

如果希望侦听器使用唯一端口值侦听服务器的所有 IP 地址，请输入 0.0.0.0 作为网络地址。

侦听器端口

如果“网络地址”字段为 0.0.0.0，则为一个唯一的侦听器端口值。如果使用的是其他 IP 地址，则为所需的端口值。

默认虚拟服务器

从下拉式列表中选择一个虚拟服务器。

服务器名称

要在服务器发送给客户机的 URL 中使用的主机名。如果服务器使用别名，则此名称为别名。如果服务器未使用别名，请将此字段保留为空。

重定向端口

输入一个端口值，以将请求重定向到另一个端口。如果满足以下两个条件，Enterprise Server 将自动重定向请求：

  • 此侦听器支持非 SSL 请求。

  • 接收匹配安全性约束需要 SSL 传输的请求。

    默认情况下，Enterprise Server 使用原始请求中指定的端口号。

接收器线程数

接收器线程的数量。默认值为 1。

支持者

删除选中标记，以禁用在 servlet 生成的 HTTP 响应标头中包含 X-Powered-By: Servlet/2.4 标头。

Java Servlet 2.4 规范中定义了此标头，容器可以将其添加到 Servlet 生成的响应中。与此类似，JavaServer Pages (JSP) 2.0 规范中定义了 X-Powered-By: JSP/2.0 标头，此标头可添加到使用 JSP 技术的响应中（可选）。默认情况下，将为 Web 应用程序包含 X-Powered-By: JSP/2.0 标头。这些标头的目的是帮助 Web 站点管理员收集有关 Servlet 和 JSP 技术使用方面的统计数据。

有关启用和禁用 JSP 页面的 X-Powered-By 标头的信息，请参见《GlassFish v3 Application Server Application Deployment Guide》中的附录 A，“Deployment Descriptor Files”。

根据生产环境的不同，可以省略 X-Powered-By 标头的生成，以隐藏其基本技术。

阻止

针对侦听器和外部端口启用阻止。

其他属性

为侦听器添加属性。
