Distribuisci applicazioni o moduli

Utilizzare la pagina Distribuisci applicazioni o moduli per distribuire un'applicazione.

La pagina Distribuisci applicazioni o moduli contiene le opzioni riportate di seguito per tutte le applicazioni. Le opzioni aggiuntive vengono visualizzate solo dopo aver specificato un valore nel campo Posizione per l'applicazione.

Posizione

La posizione dell'archivio per l'applicazione che si sta distribuendo.

Le opzioni riportate di seguito consentono di specificare la posizione da cui è accessibile l'archivio e se l'archivio è un file o una directory.

File incluso in un package da caricare sul server

L'archivio è un file che risiede sul computer client o è accessibile da tale computer.

Il computer client è l'host sul quale viene visualizzata la Console di amministrazione tramite un browser.

Directory o file locale incluso in un package accessibile da GlassFish Server

L'archivio è un file che risiede sul computer server o è un'applicazione decompressa in una directory esplosa.

Il computer server è l'host sul quale è in esecuzione il server di amministrazione del dominio di GlassFish Server.

Tipo

Il tipo dell'applicazione. Di seguito sono riportate le scelte disponibili.

  • Applicazione Web

  • Applicazione enterprise

  • Client applicazione

  • Modulo connettore

  • JAR EJB

  • Altro

Destinazioni

I cluster e le istanze standalone su cui distribuire l'applicazione o il modulo. Per spostare le destinazioni desiderate nella colonna Destinazioni selezionate, utilizzare i pulsanti Aggiungi e Aggiungi tutto. Per spostare le destinazioni non necessarie nella colonna Destinazioni disponibili, utilizzare i pulsanti Rimuovi e Rimuovi tutto.

Questa opzione viene visualizzata solo se nel dominio sono stati creati cluster o istanze standalone.

Nelle sezioni seguenti vengono descritte le opzioni aggiuntive per i vari tipi di applicazione.

  • Opzioni aggiuntive per un'applicazione Web

  • Opzioni aggiuntive per un'applicazione enterprise

  • Opzioni aggiuntive per un client di applicazione

  • Opzioni aggiuntive per un modulo connettore

  • Opzioni aggiuntive per un file JAR EJB

Opzioni aggiuntive per un'applicazione Web

Se il tipo di applicazione è Applicazione Web, vengono visualizzate le opzioni riportate di seguito.

Radice contesto

Il percorso dell'applicazione. Nell'URL dell'applicazione Web la radice contesto si trova subito dopo il numero di porta (http://host:porta/radice-contesto/...). La radice contesto deve iniziare con una barra, ad esempio, /hello.

Nome applicazione

Il nome dell'applicazione.

Il nome può includere un identificativo di versione opzionale, che segue il nome ed è separato da questo mediante due punti (:). L'identificativo di versione deve iniziare con una lettera o un numero. Può contenere caratteri alfanumerici più i caratteri di sottolineatura (_), lineetta (-) e punto (.). Per ulteriori informazioni sulle versioni di moduli e applicazioni, vedere Module and Application Versions in Oracle GlassFish Server Application Deployment Guide.

Server virtuali

I server virtuali associati all'applicazione.

L'opzione Server virtuali viene visualizzata se esiste solo l'istanza server predefinita server. Se esistono cluster o altre istanze server standalone, è possibile selezionare i server virtuali dopo la distribuzione. Andare alla pagina Modifica applicazione, selezionare la scheda Destinazione, quindi selezionare Gestisci server virtuali per la destinazione desiderata.

Stato

Se questa opzione è selezionata, l'applicazione è abilitata. Questa opzione è selezionata per impostazione predefinita.

Disponibilità

Se la casella di controllo Abilitato è selezionata, la funzione High Availability è abilitata per le sessioni Web e per la definizione di checkpoint e la passivazione potenziale dei bean di sessione con conservazione dello stato (SFSB). Se l'impostazione è false (predefinita), il salvataggio delle sessioni Web e la definizione di checkpoint SFSB vengono disabilitati per l'applicazione, l'applicazione Web o il modulo EJB specificato. Se l'impostazione è true, l'applicazione o il modulo specificato viene abilitato per la funzione High Availability. Impostare questa opzione su true solo se la funzione High Availability è configurata e abilitata ai livelli superiori, ad esempio a livello di server e di contenitore.

Questa opzione viene visualizzata se esistono cluster o istanze server standalone oltre all'istanza server predefinita (server).

Precompila JSP

Se questa opzione è selezionata, i file JSP (JavaServer Pages) vengono precompilati. Se questa opzione è disabilitata, i file JSP vengono compilati in fase di esecuzione al primo accesso. Questa opzione è disabilitata per impostazione predefinita.

Esegui verificatore

Se questa opzione è selezionata, i descrittori di distribuzione vengono verificati prima della distribuzione. Se la verifica non riesce, la distribuzione non viene eseguita. Il verificatore esamina la struttura e il contenuto dei descrittori di distribuzione. La verifica di applicazioni di grandi dimensioni richiede spesso molto tempo. Questa opzione è disabilitata per impostazione predefinita.

I package del verificatore devono essere installati tramite la utility Strumento di aggiornamento; in caso contrario, viene registrata un'avvertenza e l'opzione viene ignorata.

Imponi ridistribuzione

Se questa opzione è selezionata, l'applicazione viene ridistribuita se è già stata distribuita. Se questa opzione non è selezionata, il tentativo di distribuire un'applicazione già distribuita causa un errore. Questa opzione è disabilitata per impostazione predefinita.

Mantieni stato

Questa opzione controlla se le sessioni Web, le istanze SFSB e i timer EJB creati in modo persistente vengono conservati tra le ridistribuzioni.

Questa opzione è disabilitata per impostazione predefinita. È supportata solo nell'istanza server predefinita, denominata server. Non è supportata e viene ignorata per qualsiasi altra destinazione.

Alcuni tipi di modifiche apportate a un'applicazione tra le ridistribuzioni impediscono a questa funzione di operare in modo corretto. Ad esempio, non modificare il set di variabili di istanza nella classe bean SFSB.

Per le applicazioni Web, questa funzione è applicabile solo se nel file glassfish-web-app.xml l'attributo persistence-type dell'elemento session-manager è impostato su file.

Per le istanze di bean di sessione con conservazione dello stato, il tipo di persistenza senza High Availability viene impostato nel server (opzione Tipo di persistenza SFSB) e deve essere impostato su file, ovvero il valore predefinito e consigliato.

Se un elemento attivo quale una sessione Web, un'istanza SFSB o un timer EJB non viene preservato o ripristinato, nessuno di tali elementi sarà disponibile una volta completata la ridistribuzione. Tuttavia la ridistribuzione prosegue e viene registrata un'avvertenza.

Per preservare i dati con stato attivo, GlassFish Server li serializza e li salva nella memoria. Per ripristinare i dati, il classloader dell'applicazione appena ridistribuita deserializza i dati salvati in precedenza.

Conserva risorse con ambito applicazione

Se l'opzione è selezionata, le risorse con ambito a livello di applicazione vengono conservate e ripristinate durante la ridistribuzione. Questa opzione non è selezionata per impostazione predefinita.

Ordine di distribuzione

L'ordine di distribuzione dell'applicazione.

Le applicazioni con numero inferiore vengono caricate per prime all'avvio del server. Ad esempio, un'applicazione con il numero di ordine di distribuzione 102 viene caricata prima di un'applicazione con il numero di ordine di distribuzione 110. Se l'ordine di distribuzione non viene specificato al momento della distribuzione dell'applicazione, verrà assegnato l'ordine di distribuzione predefinito 100. Se due applicazioni dispongono dello stesso ordine di distribuzione, verrà caricata per prima l'applicazione che è stata distribuita per prima. La specifica dell'ordine di distribuzione è utile se l'applicazione dispone di dipendenze e deve essere caricata secondo un determinato ordine.

Librerie

Lista separata da virgole dei file JAR di libreria specifici del modulo o dell'applicazione. I percorsi possono essere assoluti o relativi. Un percorso relativo è relativo rispetto a domain-dir/lib/applibs. Se il percorso è assoluto, deve essere accessibile per il server di amministrazione del dominio (DAS), ovvero deve trovarsi in domain-dir. Le librerie vengono rese disponibili per l'applicazione secondo l'ordine in cui vengono specificate.

Descrizione

La descrizione dell'applicazione.

Se l'ordine di distribuzione non viene specificato al momento della distribuzione dell'applicazione, verrà assegnato l'ordine di distribuzione predefinito 100. Se due applicazioni dispongono dello stesso ordine di distribuzione, verrà caricata per prima l'applicazione che è stata distribuita per prima. La specifica dell'ordine di distribuzione è utile se l'applicazione dispone di dipendenze e deve essere caricata secondo un determinato ordine.

Opzioni aggiuntive per le applicazioni di tipo Altro

Se il tipo di applicazione è Altro, vengono visualizzate le opzioni riportate di seguito.

Il nome dell'applicazione.

Il nome può includere un identificativo di versione opzionale, che segue il nome ed è separato da questo mediante due punti (:). L'identificativo di versione deve iniziare con una lettera o un numero. Può contenere caratteri alfanumerici più i caratteri di sottolineatura (_), lineetta (-) e punto (.). I server virtuali associati all'applicazione.

L'opzione Server virtuali viene visualizzata se esiste solo l'istanza server predefinita server. Se esistono cluster o altre istanze server standalone, è possibile selezionare i server virtuali dopo la distribuzione. Andare alla pagina Modifica applicazione, selezionare la scheda Destinazione, quindi selezionare Gestisci server virtuali per la destinazione desiderata.

Stato

Se questa opzione è selezionata, l'applicazione è abilitata. Questa opzione è selezionata per impostazione predefinita.

Disponibilità

Se la casella di controllo Abilitato è selezionata, la funzione High Availability è abilitata per le sessioni Web e per la definizione di checkpoint e la passivazione potenziale dei bean di sessione con conservazione dello stato (SFSB). Se l'impostazione è false (predefinita), il salvataggio delle sessioni Web e la definizione di checkpoint SFSB vengono disabilitati per l'applicazione, l'applicazione Web o il modulo EJB specificato. Se l'impostazione è true, l'applicazione o il modulo specificato viene abilitato per la funzione High Availability. Impostare questa opzione su true solo se la funzione High Availability è configurata e abilitata ai livelli superiori, ad esempio a livello di server e di contenitore.

Questa opzione viene visualizzata se esistono cluster o istanze server standalone oltre all'istanza server predefinita (server).

Tipo OSGI

Se selezionata, questa opzione specifica un modulo OSGi/Java-EE ibrido. Viene visualizzata solo se il tipo selezionato è Altro.

Precompila JSP

Se questa opzione è selezionata, i file JSP (JavaServer Pages) vengono precompilati. Se questa opzione è disabilitata, i file JSP vengono compilati in fase di esecuzione al primo accesso. Questa opzione è disabilitata per impostazione predefinita.

Esegui verificatore

Se questa opzione è selezionata, i descrittori di distribuzione vengono verificati prima della distribuzione. Se la verifica non riesce, la distribuzione non viene eseguita. Il verificatore esamina la struttura e il contenuto dei descrittori di distribuzione. La verifica di applicazioni di grandi dimensioni richiede spesso molto tempo. Questa opzione è disabilitata per impostazione predefinita.

I package del verificatore devono essere installati tramite la utility Strumento di aggiornamento; in caso contrario, viene registrata un'avvertenza e l'opzione viene ignorata.

Imponi ridistribuzione

Se questa opzione è selezionata, l'applicazione viene ridistribuita se è già stata distribuita. Se questa opzione non è selezionata, il tentativo di distribuire un'applicazione già distribuita causa un errore. Questa opzione è disabilitata per impostazione predefinita.

Mantieni stato

Questa opzione controlla se le sessioni Web, le istanze SFSB e i timer EJB creati in modo persistente vengono conservati tra le ridistribuzioni.

Questa opzione è disabilitata per impostazione predefinita. È supportata solo nell'istanza server predefinita, denominata server. Non è supportata e viene ignorata per qualsiasi altra destinazione.

Alcuni tipi di modifiche apportate a un'applicazione tra le ridistribuzioni impediscono a questa funzione di operare in modo corretto. Ad esempio, non modificare il set di variabili di istanza nella classe bean SFSB.

Per le applicazioni Web, questa funzione è applicabile solo se nel file glassfish-web-app.xml l'attributo persistence-type dell'elemento session-manager è impostato su file.

Per le istanze di bean di sessione con conservazione dello stato, il tipo di persistenza senza High Availability viene impostato nel server (opzione Tipo di persistenza SFSB) e deve essere impostato su file, ovvero il valore predefinito e consigliato.

Se un elemento attivo quale una sessione Web, un'istanza SFSB o un timer EJB non viene preservato o ripristinato, nessuno di tali elementi sarà disponibile una volta completata la ridistribuzione. Tuttavia la ridistribuzione prosegue e viene registrata un'avvertenza.

Per preservare i dati con stato attivo, GlassFish Server li serializza e li salva nella memoria. Per ripristinare i dati, il classloader dell'applicazione appena ridistribuita deserializza i dati salvati in precedenza.
