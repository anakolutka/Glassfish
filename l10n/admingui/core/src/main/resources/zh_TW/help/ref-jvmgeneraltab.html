JVM 一般設定頁面

Java 虛擬機器 (JVM) 包含在 Java Platform, Standard Edition (Java SE 平台) 中，Enterprise Server 需要此軟體。錯誤的 JVM 設定會導致伺服器無法執行，因此，變更這些設定時應謹慎。

Java 首頁

Java SE 軟體的安裝目錄名稱，Enterprise Server 需要此軟體才可執行。若要驗證此發行版本是否支援所指定的 Java SE 軟體版本，請參閱「GlassFish v3 Application Server Release Notes」。若輸入不存在的目錄名稱或不受支援的 Java SE 軟體版本的目錄名稱，則 Enterprise Server 將無法啟動。

Javac 選項

Java 程式語言編譯器的指令行選項。部署 Enterprise JavaBeansTM (EJBTM) 元件後，Enterprise Server 會執行編譯器。

除錯

啟用 Java 平台除錯程式架構 (Java Platform Debugger Architecture, JPDA) 的除錯功能。

JPDA 由應用程式開發者使用。如需更多資訊，請參閱「GlassFish v3 Application Server Developer’s Guide」。

除錯選項

指定在啟用除錯功能的情況下，要傳遞至 JVM 的 JPDA 選項。

RMI 編譯選項

rmic 編譯器的指令行選項。部署 EJB 元件後，Enterprise Server 將執行 rmic 編譯器。

位元碼預處理程式

負責位元碼預處理作業的類別名稱。使用逗號分隔類別名稱。每個類別都必須實作 com.sun.appserv.BytecodePreprocessor 介面。將按指定次序呼叫這些類別。
