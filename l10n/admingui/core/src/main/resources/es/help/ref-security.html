Seguridad

Administrador de seguridad

Si se selecciona esta opción, se activará el administrador de seguridad para el dominio.

Cuando se activa, se añade una opción -Djava.security.manager a los valores de configuración de la JVM de Servidor empresarial. Consulte Para configurar las opciones de JVM. Debe reiniciar el servidor para habilitar este cambio.

Asegúrese de que ha otorgado los permisos correctos para todas las aplicaciones. Puede deshabilitar el administrador de seguridad para mejorar el rendimiento.

Esta opción se encuentra deshabilitada de forma predeterminada.

Registro de auditoría

Si se activa, el servidor cargará y ejecutará todos los módulos de auditoría especificados en el parámetro Módulos de auditoría. Si se desactiva, el servidor no accederá a los módulos de auditoría. Esta opción se encuentra deshabilitada de forma predeterminada.

Dominio predeterminado

Dominio activo (predeterminado) que utiliza el servidor para la autenticación. Las aplicaciones utilizan este dominio siempre que el descriptor de implementación no especifique un dominio diferente. Todos los dominios configurados aparecen en la lista. El valor predeterminado es file.

Principal predeterminado

Especifica el nombre de usuario predeterminado. El servidor utiliza este nombre de usuario si no se proporciona otro principal. Si introduce un valor en este campo, tiene que introducir la contraseña correspondiente en el campo Contraseña principal predeterminada.

Este atributo es necesario para el funcionamiento normal del servidor.

Contraseña principal predeterminada

Contraseña principal predeterminada especificada en el campo Principal predeterminado. Este atributo es necesario para el funcionamiento normal del servidor.

Módulos de auditoría

Lista de clases de proveedor de módulos de auditoría, delimitada por comas. Los módulos que aparecen en la lista tienen que estar ya configurados. Si está activado el Registro de auditoría, el campo mostrará los módulos de auditoría. Por defecto el servidor utiliza un módulo de auditoría llamado default. Para obtener información sobre la creación de módulos de auditoría nuevos, consulte Cómo crear un módulo de auditoría.

Asignación predeterminada de principal a función activada

Si se selecciona esta opción, se aplicará la asignación predeterminada de principal a función a las aplicaciones que no tengan ninguna asignación específica de aplicación.

Clases principales asignadas

Especifica una implementación personalizada de java.security.Clase de implementación Principal utilizada en la asignación predeterminada de principal a función.
