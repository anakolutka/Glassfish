Benutzerdefinierte Ressourcen

Eine benutzerdefinierte Ressource gibt eine benutzerdefinierte, serverweite Ressourcenobjekt-Factory an.

Auf der Seite "Benutzerdefinierte Ressourcen" können Sie benutzerdefinierte JNDI-Ressourcen konfigurieren.

Für jede benutzerdefinierte Ressource werden die folgenden Informationen zur Verfügung gestellt.

JNDI-Name

Ein eindeutiger Name, der die benutzerdefinierte Ressource identifiziert.

Aktiviert

(Diese Spalte wird angezeigt, wenn nur die Standardserverinstanz server in der Domain vorhanden ist.)

Zeigt ein Häkchen an, wenn die Ressource auf dem Ziel aktiviert ist, oder ein X, wenn die Ressource deaktiviert ist.

Status

(Diese Spalte wird angezeigt, wenn in der Domain Cluster oder Standalone-Instanzen erstellt wurden.)

Die Anzahl Cluster und Standalone-Instanzen, auf die die Ressource ausgerichtet wird, und auf wie vielen dieser Ziele die Ressource aktiviert ist. Beispiel: "Auf 2 von 4 Ziel(en) aktiviert" bedeutet, dass die Ressource auf vier Cluster und Standalone-Instanzen ausgerichtet und auf zweien dieser vier Ziele aktiviert ist.

Ressourcentyp

Der Typ der benutzerdefinierten Ressource.

Beschreibung

Eine Beschreibung der benutzerdefinierten Ressource.

Die Tabelle "Ressourcen" enthält darüber hinaus die folgenden Optionen.

Neu

Schaltfläche zum Erstellen einer neuen benutzerdefinierten Ressource.

Löschen

Schaltfläche zum Löschen von mindestens einer benutzerdefinierten Ressource.

Aktivieren

Schaltfläche zum Aktivieren von mindestens einer ausgewählten benutzerdefinierten Ressource.

Deaktivieren

Schaltfläche zum Deaktivieren von mindestens einer ausgewählten benutzerdefinierten Ressource.

Verwandte Aufgaben

  • So wird eine benutzerdefinierte JNDI-Ressource erstellt

  • So wird eine benutzerdefinierte JNDI-Ressource bearbeitet

  • So wird eine benutzerdefinierte JNDI-Ressource gelöscht
