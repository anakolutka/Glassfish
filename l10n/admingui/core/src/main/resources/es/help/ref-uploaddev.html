Implementar aplicaciones o módulos

Utilice la página Implementar aplicaciones o módulos para implementar una aplicación.

La página Implementar aplicaciones o módulos contiene las opciones siguientes para todas las aplicaciones. Únicamente aparecen más opciones en caso de que especifique el campo Ubicación para la aplicación.

Ubicación

Ubicación del archivo de almacenamiento para la aplicación que está implementando.

Las siguientes opciones especifican desde dónde se puede acceder al archivo de almacenamiento y si es un archivo o directorio.

Archivo empaquetado que se cargará en el servidor

El archivo de almacenamiento se encuentra en un archivo que reside en la máquina cliente o al que se puede acceder desde la máquina cliente.

La máquina cliente es el host en el que está viendo la Consola de administración por medio de un navegador.

Directorio o archivo empaquetado local al que se puede acceder desde Servidor empresarial

El archivo de almacenamiento es un archivo que reside en la máquina servidor o es una aplicación desempaquetada en un directorio de despliegue.

El equipo servidor es el sistema anfitrión que está ejecutando el servidor de administración del dominio de Servidor empresarial.

Tipo

Tipo de aplicación. Las opciones disponibles son:

  • Aplicación de Web

  • Aplicación empresarial

  • Cliente de aplicaciones

  • Módulo del conector

  • JAR EJB

  • Aplicación de Ruby

  • Otro

Las opciones adicionales para los distintos tipos de aplicación se describen en las siguientes secciones.

Opciones adicionales para una aplicación web

Si el tipo de aplicación corresponde a una aplicación web, aparecen las siguientes opciones.

Raíz de contexto

Ruta hacia la aplicación. En la URL de la aplicación web, la raíz de contexto aparece justo después del número de puerto (http://host:port /context-root/...). La raíz de contexto debe comenzar por una barra diagonal, por ejemplo, /hello.

Nombre de la aplicación

Nombre de la aplicación.

Servidores virtuales

Servidores virtuales asociados a esta aplicación.

Estado

Si está opción está seleccionada, entonces la aplicación está activada. Esta opción está seleccionada de forma predeterminada.

Precompilar JSP

Si está seleccionada esta opción, los archivos JavaServer PagesTM ( JSPTM) se precompilarán. Si se deshabilita esta opción, los archivos JSP se compilarán en el tiempo de ejecución cuando se acceda a los mismos por primera vez. Esta opción se encuentra deshabilitada de forma predeterminada.

Ejecutar verificador

Si esta opción está seleccionada, el verificador se ejecutará antes de la implementación. Si se produce un error en la verificación, no se realiza la implementación. El verificador examinará la estructura y contenido del archivo. La verificación de aplicaciones de gran tamaño tiende a llevar mucho tiempo. Esta opción se encuentra deshabilitada de forma predeterminada.

Forzar reimplementación

Si esta opción está seleccionada, se volverá a implementar la aplicación si está ya implementada. Si esta opción no está seleccionada y se intenta implementar una aplicación que ya se haya implementado, se producirá un error. Esta opción se encuentra deshabilitada de forma predeterminada.

Bibliotecas

Lista de archivos JAR de la biblioteca separados por comas específicos de este módulo o aplicación. Las rutas pueden ser absolutas o relativas. Una ruta relativa es relativa a domain-dir/lib/applibs. Si la ruta es absoluta, la ruta debe ser accesible para el servidor de administración del dominio (DAS), lo que implica que debe estar en domain-dir. Las bibliotecas están disponibles para la aplicación en el orden en que están especificadas.

Descripción

Una descripción de la aplicación.

Opciones adicionales para una aplicación Ruby

Si el tipo de aplicación corresponde a una aplicación Ruby, aparecen las siguientes opciones.

Nombre de la aplicación

Nombre de la aplicación.

Raíz de contexto

Ruta hacia la aplicación. En la URL de la aplicación web, la raíz de contexto aparece justo después del número de puerto (http://host:port /context-root/...). La raíz de contexto debe comenzar por una barra diagonal, por ejemplo, /hello.

Estado

Si está opción está seleccionada, entonces la aplicación está activada. Esta opción está seleccionada de forma predeterminada.

Forzar reimplementación

Si esta opción está seleccionada, se volverá a implementar la aplicación si está ya implementada. Si esta opción no está seleccionada y se intenta implementar una aplicación que ya se haya implementado, se producirá un error. Esta opción se encuentra deshabilitada de forma predeterminada.

Inicio de JRuby

Ruta hacia el directorio donde está instalado el contenedor Jruby. La ubicación predeterminada es as-install/jruby. Si especifica un valor en este campo, el valor de la aplicación reemplazará al valor de Inicio de Jruby determinado para el contenedor de Ruby.

Tiempo de ejecución

Tiempo de ejecución inicial para iniciar JRuby. El valor debe ser igual o superior al valor del campo Tiempo de ejecución mínimo, pero inferior al valor que figura en el campo Tiempo de ejecución máximo. El valor predeterminado es 1. Si especifica un valor en este campo, el valor de la aplicación reemplazará al valor de Tiempo de ejecución determinado para el contenedor de Ruby.

Tiempo de ejecución mínimo

Tiempo de ejecución mínimo de JRuby del conjunto. El valor debe ser inferior o igual al valor del campo Tiempo de ejecución e inferior al valor que figura en el campo Tiempo de ejecución máximo. El valor predeterminado es 1. Si especifica un valor en este campo, el valor de la aplicación reemplazará al valor de Tiempo mínimo de ejecución determinado para el contenedor de Ruby.

Tiempo de ejecución máximo

Tiempo de ejecución máximo de JRuby del conjunto. El valor debe ser igual o superior al valor del campo Tiempo de ejecución mínimo e igual o superior al valor que figura en el campo Tiempo de ejecución. El valor predeterminado es 2. Si especifica un valor en este campo, el valor de la aplicación reemplazará al valor de Tiempo máximo de ejecución determinado para el contenedor de Ruby.

Estructura

Estructura en la que se implementará la aplicación. Las opciones disponibles son rails, merb, sinatra y auto-detection. El valor predeterminado es rails. Si establece esta propiedad en un valor distinto a auto-detection omitirá el proceso normal y potencialmente largo de detección automática y forzará la implementación en la estructura especificada. Si la aplicación implementada no se escribe para la estructura especificada, se produce un error.

Entorno Rack

Entorno en que se ejecuta la estructura JRuby, como Rails o Merb. Las opciones disponibles son production, development y test. El valor predeterminado es development.

MT-Seguro

Configuración del subproceso seguro para la estructura en que se ejecutará la aplicación. Esta propiedad afecta a aplicaciones iniciadas mediante detección automática de una secuencia de comandos de inicio proporcionada por el usuario. Las opciones disponibles son:

true

Especifica que la estructura cuenta son subprocesos seguros y por tanto no necesita la creación de un conjunto.

false

Especifica que la aplicación se inicia con un conjunto de instancias de aplicación, y a cada instancia de la aplicación accede un subproceso al mismo tiempo.

default

Servidor empresarial trata de detectar la seguridad de los subprocesos de manera automática. Éste es el valor predeterminado.

Esta propiedad afecta únicamente a las estructuras iniciadas en lugares donde no se puede determinar automáticamente la seguridad de los subprocesos. Si configura este valor en , no se produce una detección automática de la aplicación Rails 2.1.x para que se inicie en modo de subprocesos seguros, ni se puede utilizar para forzar el inicio de una estructura de subprocesos seguros en modo agrupado.
