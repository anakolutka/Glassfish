Configurar valores de registro generales

  1. En el árbol de navegación, amplíe el nodo Configuraciones.

  2. En el nodo Configuraciones, haga clic en la configuración de instancia de servidor o cluster para el que desee definir la configuración de registrador.

    Se abrirá la página Configuración.

  3. En la página Configuración, haga clic en Configuración de Registrador.

    Se abrirá la página Configuración de Registrador del destino de configuración seleccionado.

  4. Active la casilla de control Escribir en el log del sistema activado para enviar la salida de registro a la utilidad syslog además de al log del servidor.

    Esta función sólo está disponible en sistemas Solaris y Linux. Esta opción se activada por defecto.

  5. Active la casilla Registrar en consola activada para enviar la salida de registro a la consola del sistema.

  6. Active la casilla de control Rotación en cambio de fecha para rotar los archivos log del servidor cada día a medianoche.

  7. Active la casilla de control Modo de varias líneas para formatear los mensajes de log de forma que resulten más legibles. Al cuerpo del mensaje de log se aplica sangrado y comienza en una nueva línea.

  8. En la lista desplegable Formato de Registro de Consola, seleccione el formato para el registro en la consola. La opción por defecto es ODL (Oracle Diagnostics logging). La otra opción es ULF (UniformLogFormatter).

  9. En la lista desplegable Formato de Registro de Archivo Log, seleccione el formato para el registro en el archivo log del servidor. La opción por defecto es ODL.

  10. En el campo Límite de Rotación de Archivo, introduzca el tamaño en bytes que el archivo log del servidor debe alcanzar antes de rotar.

    Cuando el log del servidor alcanza el tamaño especificado, GlassFish Server crea un nuevo archivo vacío llamado server.log y cambia el nombre del archivo antiguo server.log_date, donde date es la fecha y hora en la que se produjo la rotación del archivo. El valor por defecto es 2 megabytes. El valor mínimo para el límite es 500 kilobytes; si se especifica un valor inferior, el archivo rota cuando alcanza el tamaño de 500 KB. Para desactivar la rotación del archivo log, defina el valor en 0.

  11. En el campo Límite Temporal de Rotación de Archivo, introduzca los minutos que deben transcurrir antes de rotar el log del servidor.

    El valor por defecto es 0, lo que significa que el archivo rota cuando alcanza el tamaño especificado en el campo Límite de Rotación de Archivo.

    Nota:

    Si especifica uno o más minutos, el límite de tiempo tiene prioridad sobre el límite de tamaño especificado en el campo Límite de Rotación de Archivo.

  12. En el campo Frecuencia de Vaciado, escriba el número máximo de mensajes que se deben escribir desde la cola en el log del servidor a la vez.

    El valor por defecto es 1.

  13. En el campo Máximo de archivos de historial, escriba el número máximo de archivos log que GlassFish Server debe conservar antes de suprimir el archivo más antiguo. Si define este valor en cero (0), GlassFish Server no suprime ningún archivo log antiguo.

  14. En el campo Archivo Log, introduzca un nombre o ubicación alternativos para el archivo log del servidor.

    La ubicación por defecto es ${com.sun.aas.instanceRoot}/logs/server.log.

  15. En el campo Manejador de Logs, introduzca el nombre de clase absoluto de un manejador de logs personalizado.

    Un manejador de logs personalizado le permite enviar logs a un destino distinto a server.log o syslog. El manejador personalizado debe comprender la clase java.util.logging.ConsoleHandler (API compatible con JSR 047). Indique también la clase de manejador en la classpath GlassFish Server para que el manejador se instale durante el inicio del servidor.

  16. Haga clic en Guardar.

    Nota:

    Según la configuración para la que se activó la función de registro, puede ser necesario reiniciar el servidor.

    También puede ser necesario reiniciar el servidor si se cambia el nombre del archivo log por otro que no sea el valor por defecto.ç Si es necesario reiniciar el servidor se mostrará el mensaje Es Necesario Reiniciar en la esquina superior derecha de la Consola de administración.
