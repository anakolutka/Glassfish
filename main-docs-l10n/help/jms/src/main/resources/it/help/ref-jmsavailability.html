Disponibilità JMS

Utilizzare la pagina Disponibilità JMS per configurare il cluster broker Message Queue per il cluster GlassFish Server.

Attenzione:

Prima di utilizzare la pagina Disponibilità JMS per riconfigurare un cluster esistente, è necessario attenersi ad alcune procedure speciali per eseguire la migrazione a un altro cluster broker, come descritto in To Migrate Between Types of Embedded or Local Conventional Broker Clusters inOracle GlassFish Server High Availability Administration Guide. Se non si seguono queste procedure speciali, i dati potrebbero andare persi o danneggiarsi oppure la configurazione potrebbe divenire inutilizzabile, a seconda delle operazioni JMS eseguite sul cluster esistente.

La pagina Disponibilità JMS contiene le opzioni riportate di seguito.

Nota:

Se sono in esecuzione istanze che utilizzano la configurazione, nella pagina Disponibilità JMS viene visualizzata un'avvertenza, tutte le opzioni vengono impostate per la sola lettura e il pulsante Salva viene disabilitato.
Tipo di servizio JMS

Il tipo di broker (detti host JMS in GlassFish Server) da utilizzare nel cluster di broker. Il tipo scelto determina i tipi di cluster di broker disponibili. Di seguito sono riportati i tipi di broker.

Incorporato

La configurazione e il ciclo di vita del broker sono gestiti da GlassFish Server. Ogni istanza di GlassFish in cluster viene gestita da un broker in esecuzione nella stessa JVM dell'istanza.

Se l'opzione Tipo di servizio JMS è impostata su Incorporato, sono supportati solo cluster di broker Message Queue convenzionali. Vengono visualizzate le opzioni per configurare il cluster convenzionale.

Locale

La configurazione e il ciclo di vita del broker sono gestiti da GlassFish Server. Ogni istanza di GlassFish in cluster viene gestita da un broker in esecuzione in una JVM distinta nello stesso host dell'istanza.

Se l'opzione Tipo di servizio JMS è impostata su Locale, sono supportati i cluster di broker Message Queue sia convenzionali che avanzati (a elevata disponibilità). Vengono visualizzate le opzioni per selezionare il tipo di cluster e configurare il cluster.

Remoto

La configurazione e il ciclo di vita del broker e del cluster di broker sono gestiti mediante gli strumenti amministrativi di Message Queue.

Se l'opzione Tipo di servizio JMS è impostata su Remoto, non vengono visualizzate altre opzioni.

Tipo di cluster JMS

Il tipo di cluster di broker Message Queue da utilizzare per il cluster GlassFish Server.

Se l'opzione Tipo di cluster JMS è impostata su Convenzionale, vengono visualizzate le opzioni riportate di seguito.

Tipo di area di memorizzazione configurazione JMS

Il tipo di data store da utilizzare per i dati di configurazione del cluster convenzionale. Se questa opzione è impostata su Broker principale, un broker nel cluster viene designato come principale e memorizza i dati di configurazione. Se questa opzione è impostata su DB condiviso, i dati di configurazione vengono memorizzati in un data store JDBC a cui possono accedere tutti i broker. In questo caso vengono visualizzate le opzioni relative al database.

Tipo di area memorizzazione messaggi JMS

Il tipo di data store che ogni broker deve utilizzare per memorizzare i dati dei messaggi. Se questa opzione è impostata su File, ogni broker memorizza i dati dei messaggi in un data store basato su file. Se questa opzione è impostata su JDBC, ogni broker memorizza i dati dei messaggi in un data store JDBC.

Se l'opzione Tipo di cluster JMS è impostata su Avanzato (HA), vengono visualizzate le opzioni relative al database.

Fornitore database
URL database
Utente database

Il fornitore, l'URL di accesso e l'utente del database JDBC da utilizzare in ognuna delle situazioni seguenti:

  • quando il tipo di area di memorizzazione della configurazione di un cluster convenzionale è impostato su DB condiviso;

  • quando il tipo di area di memorizzazione dei messaggi di un cluster convenzionale è impostato su JDBC;

  • quando il tipo di cluster è impostato su Avanzato (HA).

Per informazioni sui fornitori supportati e sui formati degli URL di accesso per ciascun fornitore, vedere JDBC-Based Persistence in Oracle GlassFish Server Message Queue Administration Guide.

Autenticazione database
Password database
Alias password

Le informazioni della password per l'utente del database JDBC specificato in Utente database.

Se l'opzione Autenticazione database è impostata su Password, viene visualizzata l'opzione Password database. Se l'opzione Autenticazione database è impostata su Alias password, viene visualizzata l'opzione Alias password.

Proprietà aggiuntive

Proprietà aggiuntive per la disponibilità JMS.

GlassFish Server non definisce alcuna proprietà aggiuntiva per la disponibilità JMS.
