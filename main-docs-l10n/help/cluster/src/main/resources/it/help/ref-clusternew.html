Nuovo cluster

Utilizzare la pagina Nuovo cluster per creare un cluster.

La pagina Nuovo cluster contiene le opzioni riportate di seguito.

Nome cluster

Il nome del cluster.

Il nome deve soddisfare i requisiti riportati di seguito.

  • Il nome può contenere solo caratteri ASCII.

  • Il nome deve iniziare con una lettera, un numero o un carattere di sottolineatura.

  • Il nome può contenere solo i caratteri seguenti:

    • Lettere minuscole

    • Lettere maiuscole

    • Numeri

    • Trattino

    • Punto

    • Carattere di sottolineatura

  • Il nome deve essere univoco nel dominio e non deve essere il nome di un altro cluster, di una configurazione denominata, di un'istanza di GlassFish Server o di un nodo.

  • Il nome non deve essere domain, server né un'altra parola chiave riservata di GlassFish Server.

Configurazione

Elenco a discesa delle configurazioni denominate esistenti che il cluster può utilizzare. Un cluster richiede una configurazione denominata che definisca la configurazione di tutte le istanze che vengono aggiunte al cluster. È necessario selezionare una configurazione dalla lista. Il cluster utilizzerà la configurazione selezionata al momento della creazione del cluster.

Crea una copia della configurazione selezionata

Se questa opzione è selezionata, la configurazione selezionata viene copiata al momento della creazione del cluster. Alla copia della configurazione viene assegnato il nome nome-cluster-config, dove nome-cluster è il nome del cluster digitato nel campo Nome cluster.

Se nell'elenco a discesa Configurazione è selezionata la voce default-config, questa opzione deve essere selezionata. La configurazione default-config può essere solo copiata e non utilizzata come riferimento.

Crea riferimento alla configurazione selezionata

Se questa opzione è selezionata, il cluster utilizzerà la configurazione denominata esistente specificata.

Se nell'elenco a discesa Configurazione è selezionata la voce default-config, questa opzione non deve essere selezionata. La configurazione default-config non può essere utilizzata come riferimento, ma solo copiata.

Tipo di configurazione cluster Message Queue

Utilizzare questa opzione per specificare il tipo di cluster di broker Message Queue da utilizzare per il nuovo cluster GlassFish Server. Per impostazione predefinita, GlassFish Server utilizza un cluster di broker Message Queue convenzionale con un broker principale e un broker incorporato in ogni istanza del cluster GlassFish Server.

Se questa opzione è impostata su Personalizzato, vengono visualizzate numerose opzioni aggiuntive per configurare il cluster di broker Message Queue da utilizzare per il cluster GlassFish Server. Vedere Opzioni per i cluster di broker Message Queue personalizzati.

Istanze server da creare

Lista delle istanze di GlassFish Server da creare al momento della creazione del cluster. Per ogni istanza vengono fornite le informazioni riportate di seguito.

Nome istanza

Il nome dell'istanza.

Peso

Il numero intero che rappresenta il peso, o fattore ponderale, per il bilanciamento del carico dell'istanza.

Il fattore ponderale per il bilanciamento del carico determina la proporzione di tutte le richieste inviate al cluster che l'istanza dovrebbe elaborare.

Ad esempio, in un cluster a due istanze può essere necessaria un'istanza per elaborare una di quattro richieste e un'altra istanza per elaborarne tre di quattro. In questa situazione, impostare su 1 il peso dell'istanza che elabora una delle quattro richieste e su 3 il peso dell'altra istanza. Se si preferisce utilizzare valori percentuali, impostare i valori di peso delle istanze rispettivamente su 25 e 75.

Il valore di peso predefinito è 100.

Nodo

Elenco a discesa di nodi esistenti in cui può risiedere l'istanza. È necessario selezionare un nodo dalla lista. L'istanza risiederà nel nodo selezionato al momento della creazione del cluster.

La tabella Istanze server da creare contiene inoltre le opzioni seguenti.

Nuovo

Pulsante che consente di creare un'istanza. Se si fa clic su questo pulsante, viene aggiunta una riga alla tabella Istanze server da creare.

Elimina

Pulsante che consente di eliminare un'istanza.

Opzioni per i cluster di broker Message Queue personalizzati

Quando l'opzione Tipo di configurazione cluster Message Queue è impostata su Personalizzato, vengono visualizzate le opzioni riportate di seguito per configurare il cluster di broker Message Queue da utilizzare per il cluster GlassFish Server.

Tipo di servizio JMS

Il tipo di broker (detti host JMS in GlassFish Server) da utilizzare nel cluster di broker. Il tipo scelto determina i tipi di cluster di broker disponibili. Di seguito sono riportati i tipi di broker.

Incorporato

La configurazione e il ciclo di vita del broker sono gestiti da GlassFish Server. Ogni istanza di GlassFish in cluster viene gestita da un broker in esecuzione nella stessa JVM dell'istanza.

Se l'opzione Tipo di servizio JMS è impostata su Incorporato, sono supportati solo cluster di broker Message Queue convenzionali. Vengono visualizzate le opzioni per configurare il cluster convenzionale.

Locale

La configurazione e il ciclo di vita del broker sono gestiti da GlassFish Server. Ogni istanza di GlassFish in cluster viene gestita da un broker in esecuzione in una JVM distinta nello stesso host dell'istanza.

Se l'opzione Tipo di servizio JMS è impostata su Locale, sono supportati i cluster di broker Message Queue sia convenzionali che avanzati (a elevata disponibilità). Vengono visualizzate le opzioni per selezionare il tipo di cluster e configurare il cluster.

Remoto

La configurazione e il ciclo di vita del broker e del cluster di broker sono gestiti mediante gli strumenti amministrativi di Message Queue.

Se l'opzione Tipo di servizio JMS è impostata su Remoto, non vengono visualizzate altre opzioni.

Tipo di cluster JMS

Il tipo di cluster di broker Message Queue da utilizzare per il cluster GlassFish Server.

Se l'opzione Tipo di cluster JMS è impostata su Convenzionale, vengono visualizzate le opzioni riportate di seguito.

Tipo di area di memorizzazione configurazione JMS

Il tipo di data store da utilizzare per i dati di configurazione del cluster convenzionale. Se questa opzione è impostata su Broker principale, un broker nel cluster viene designato come principale e memorizza i dati di configurazione. Se questa opzione è impostata su DB condiviso, i dati di configurazione vengono memorizzati in un data store JDBC a cui possono accedere tutti i broker. In questo caso vengono visualizzate le opzioni relative al database.

Tipo di area memorizzazione messaggi JMS

Il tipo di data store che ogni broker deve utilizzare per memorizzare i dati dei messaggi. Se questa opzione è impostata su File, ogni broker memorizza i dati dei messaggi in un data store basato su file. Se questa opzione è impostata su JDBC, ogni broker memorizza i dati dei messaggi in un data store JDBC.

Se l'opzione Tipo di cluster JMS è impostata su Avanzato (HA), vengono visualizzate le opzioni relative al database.

Fornitore database
URL database
Utente database

Il fornitore, l'URL di accesso e l'utente del database JDBC da utilizzare in ognuna delle situazioni seguenti:

  • quando il tipo di area di memorizzazione della configurazione di un cluster convenzionale è impostato su DB condiviso;

  • quando il tipo di area di memorizzazione dei messaggi di un cluster convenzionale è impostato su JDBC;

  • quando il tipo di cluster è impostato su Avanzato (HA).

Per informazioni sui fornitori supportati e sui formati degli URL di accesso per ciascun fornitore, vedere JDBC-Based Persistence in Oracle GlassFish Server Message Queue Administration Guide.

Autenticazione database
Password database
Alias password

Le informazioni della password per l'utente del database JDBC specificato in Utente database.

Se l'opzione Autenticazione database è impostata su Password, viene visualizzata l'opzione Password database. Se l'opzione Autenticazione database è impostata su Alias password, viene visualizzata l'opzione Alias password.

Proprietà cluster JMS

Lista con una o più proprietà broker Message Queue per configurare i broker. La lista è separata da due punti (:) e ha il formato seguente:

prop1Name=prop1Value:prop2Name=prop2Value:...

Gli eventuali punti contenuti in una proprietà broker devono essere preceduti da due barre rovesciate (\\). Ad esempio, per includere la proprietà imq.system.max_count, specificare imq\\.system\\.max_count nella lista.
