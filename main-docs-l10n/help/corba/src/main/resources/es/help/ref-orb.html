ORB

Utilice la página ORB para configurar el Object Request Broker.

El ORB proporciona la infraestructura necesaria para identificar y localizar objetos, gestionar las conexiones, enviar datos y solicitar comunicación. Los clientes remotos de beans de empresa (módulos EJB) se comunican con GlassFish Server mediante ORB.

La página ORB contiene las siguientes opciones.

Cargar Valores por Defecto

Botón para restaurar configuración que tiene valores por defecto a sus valores por defecto. Las configuraciones que no tienen valores por defecto no se modifican.

Nombre de Configuración

Nombre de la configuración a la que se aplican los valores de esta página. Este campo es de sólo lectura.

ID de Pool de Threads

Pool de threads que utiliza el ORB. ORB utiliza pools de threads para responder a solicitudes de clientes remotos de beans de empresa y otros clientes que se comunican mediante RMI-IIOP. El pool de threads por defecto para ORB es thread-pool-1, que utiliza valores por defecto apropiados para las solicitudes de RMI/IIOP.

Tamaño máximo de fragmentos de mensaje

Tamaño máximo para los fragmentos de mensajes IIOP. Los mensajes más grandes que este tamaño se fragmentarán. El valor por defecto es de 1024 bytes.

Total de conexiones

Número máximo de conexiones entrantes para todos los listeners IIOP. El valor por defecto es 1024.

Autenticación de cliente IIOP

Si la casilla de control Activado está activada, el servidor rechaza las solicitudes no autenticadas e inserta un bit de autenticación necesaria en las referencias de objetos remotos interoperables (IOR) enviadas a los clientes. La autenticación de cliente IIOP se puede activar o desactivar. Esta opción se activada por defecto.

Propiedades Adicionales

Propiedades adicionales de ORB. GlassFish Server no define ninguna propiedad adicional para ORB.
