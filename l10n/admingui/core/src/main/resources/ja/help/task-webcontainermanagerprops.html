マネージャープロパティーを設定する

セッションマネージャーを使用して、セッションを作成および破棄する方法、セッション状態を格納する場所、および使用可能なセッションの最大数を設定できます。

  1. ナビゲーションツリーで、「設定」ノードを展開します。
  2. 「設定」ノードで、「Web コンテナ」ノードを選択します。

    「Web コンテナ」ページが開きます。

  3. 「Web コンテナ」ページで、「マネージャープロパティー」タブをクリックします。
  4. 「取得間隔」フィールドに、アクティブでないセッションデータがストアから削除されるまでの秒数を入力します。
  5. 「最大セッション」フィールドに、許容されるセッションの最大数を入力します。
  6. 「セッションファイル名」フィールドに、セッションデータを含むファイルの名前を入力します。
  7. 「セッション ID ジェネレータクラス名」フィールドで、一意のセッション ID を生成するカスタムクラスを指定します。

    このクラスは Enterprise Server のクラスパスに置かれている必要があります。セッション ID ジェネレータクラスはサーバーインスタンスごとに 1 つだけ許可されます。クラスタ内のすべてのインスタンスが同じセッション ID ジェネレータを使用して、セッションキーの衝突を阻止する必要があります。カスタムセッション ID ジェネレータクラスは、次の com.sun.enterprise.util.uuid.UuidGenerator インタフェースを実装する必要があります。

    package com.sun.enterprise.util.uuid;

public interface UuidGenerator {

    public String generateUuid();
    public String generateUuid(Object obj);  //obj is the session object
}
  8. (省略可能) 「追加プロパティー」セクションで、追加プロパティーを指定します。
  9. 「保存」をクリックします。
