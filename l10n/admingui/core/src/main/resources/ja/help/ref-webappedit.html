Web アプリケーション

既存の Web アプリケーションを変更するには、「Web アプリケーション」ページを使用します。このページには次のオプションが含まれます。

名前

編集する Web アプリケーションの名前を表示する読み取り専用フィールド。

コンテキストルート

Web アプリケーションのパスを指定します。Web アプリケーションの URL では、コンテキストルートはポート番号の直後に続きます (http://host:port/context-root/...)。たとえば /hello など、コンテキストルートがスラッシュで始まるようにしてください。

仮想サーバー

この Web アプリケーションと関連付けられている仮想サーバー。

説明

Web アプリケーションの説明。

状態

このオプションを選択すると、Web アプリケーションが有効になります。これは標準でオンになっています。

場所

アプリケーションの配備元アーカイブファイルの場所を表示する読み取り専用フィールド。
