Per creare una sessione JavaMail

  1. Espandere il nodo Risorse nell'albero di navigazione.

  2. Nel nodo Risorse selezionare Sessioni JavaMail.

    Viene visualizzata la pagina Sessioni JavaMail.

  3. Nella tabella Sessioni fare clic su Nuovo.

    Viene visualizzata la pagina Nuova sessione JavaMail.

  4. Nel campo Nome JNDI digitare un nome univoco per la risorsa sessione di posta.

    Utilizzare il prefisso del contesto secondario di denominazione mail/ per le risorse JavaMail. Ad esempio: mail/MySession. Il nome deve contenere solo caratteri alfanumerici, caratteri di sottolineatura, lineette e punti.

  5. Nel campo Host posta digitare il nome host del server di posta predefinito.

    Se non viene fornita una proprietà host specifica per un protocollo, i metodi di connessione degli oggetti Store e Transport utilizzano questo valore. Il nome deve essere risolvibile in un nome host effettivo.

  6. Nel campo Utente predefinito digitare il nome utente da fornire quando ci si connette a un server di posta.

    Se non viene fornita una proprietà nome utente specifica per un protocollo, i metodi di connessione degli oggetti Store e Transport utilizzano questo valore. Il nome utente deve contenere solo caratteri alfanumerici, caratteri di sottolineatura, lineette e punti.

  7. Nel campo Indirizzo mittente predefinito digitare l'indirizzo di posta elettronica dell'utente predefinito.

    Il formato da usare è nome utente@host.dominio.

  8. Nel campo Descrizione digitare una descrizione della sessione di posta.

  9. Selezionare la casella di controllo Abilitato dell'opzione Stato per abilitare la sessione di posta.

    Lo stato predefinito è Abilitato.

  10. Nel campo Protocollo di memorizzazione definire il servizio del protocollo di memorizzazione da utilizzare.

    Il servizio del protocollo di memorizzazione si connette a un server di posta, recupera i messaggi e salva i messaggi in una o più cartelle. I valori consentiti sono imap, pop3, imaps e pop3s. Il valore predefinito è imap.

  11. Nel campo Classe protocollo di memorizzazione specificare la classe di implementazione del provider di servizi per la memorizzazione.

    Di seguito sono riportati i valori consentiti.

    • com.sun.mail.imap.IMAPStore

    • com.sun.mail.pop3.POP3Store

    • com.sun.mail.imap.IMAPSSLStore

    • com.sun.mail.pop3.POP3SSLStore

    Il valore predefinito è com.sun.mail.imap.IMAPStore.

  12. Nel campo Protocollo di trasporto specificare il metodo di comunicazione di trasporto.

    Il servizio del protocollo di trasporto invia i messaggi. I valori consentiti sono smtp e smtps. Per impostazione predefinita, il protocollo di trasporto è smtp.

  13. Nel campo Classe protocollo di trasporto specificare la classe di implementazione del provider di servizi per il trasporto.

    Il valore predefinito è com.sun.mail.smtp.SMTPTransport.

  14. Selezionare la casella di controllo Debug per abilitare l'output di debug extra, compreso un trace del protocollo, per la sessione di posta.

    Se il livello di log JavaMail è impostato su FINE o FINER, l'output di debug viene generato e incluso nel file di log di sistema. Per informazioni sull'impostazione dei livelli del log, vedere Per configurare i livelli di log. Questa opzione è disabilitata per impostazione predefinita.

  15. Nella tabella Proprietà aggiuntive specificare le proprietà aggiuntive necessarie per le applicazioni, ad esempio una proprietà host o nome utente specifica del protocollo.

    Per la lista delle proprietà disponibili, consultare la documentazione dell'interfaccia API JavaMail (http://java.sun.com/products/javamail/javadocs/index.html). Le proprietà aggiuntive devono iniziare con mail- e utilizzare i trattini, e non i punti, come separatori. Ad esempio, mail-debug è una proprietà aggiuntiva corretta, mentre mail.debug non lo è. Il processo backend di GlassFish Server converte i trattini nei punti previsti dall'interfaccia API JavaMail.

    Per aggiungere una proprietà, fare clic sul pulsante Aggiungi proprietà. Nella riga vuota visualizzata, digitare il nome della proprietà nel campo Nome e il valore della proprietà nel campo Valore.

  16. Nella lista delle destinazioni disponibili selezionare la o le destinazioni per le quali configurare la sessione di posta e fare clic su Aggiungi.

    Le destinazioni possono essere cluster o istanze server standalone. Se non si seleziona una destinazione, la sessione viene configurata per l'istanza server predefinita server.

  17. Fare clic su OK.
