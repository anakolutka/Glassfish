Para editar un listener de red

  1. En el árbol de navegación, amplíe el nodo Configuración.

  2. En el nodo Configuración, amplíe el nodo Configuración de Red.

  3. En el nodo Configuración de Red, seleccione el nodo Listeners de Red.

    Se abrirá la página Listeners de Red.

  4. En la página Listeners de Red, haga clic en el nombre del listener que está editando.

    Se abrirá la página Editar Listener de Red.

  5. Seleccione el protocolo utilizado por el listener de red para cambiar su configuración. Por ejemplo, puede activar la seguridad. No puede elegir un protocolo diferente.

  6. Active o desactive la casilla de control Estado activado para cambiar el estado del listener de red.

    El listener puede estar activado o desactivado. Esta opción se encuentra activada por defecto. Si el listener se encuentra desactivado, cualquier intento de establecer una conexión con él tiene como resultado una excepción de socket (java.net.ConnectException).

  7. La casilla de control Seguridad Activada, de sólo lectura, indica si se ha activado la seguridad para el protocolo que utiliza el listener de red.

    Si está configurando el admin-listener para el servidor de administración de dominio (DAS), que usa la configuración server-config, esta opción será de sólo lectura. Para activar la seguridad de este listener, debe activar primero la administración segura. Consulte la sección Administración Segura.

  8. Active la casilla de control de activación de listener JK para especificar que el listener soporta el protocolo mod-jk de Apache.

    El protocolo mod-jk es un protocolo de red/comunicación de propiedad entre httpd de Apache (servidor web) y una instancia back-end de GlassFish Server. Si el listener soporta este protocolo, se supone que httpd maneja todo el contenido estático, mientras que cualquier solicitud de recursos dinámicos (es decir, servlets y JSP) se direccionan al listener de red con el protocolo mod jk activado de GlassFish Server a través del protocolomod-jk. Esta opción se activada por defecto.

  9. En el campo Puerto, introduzca el número de puerto en el que recibirá el listener de red.

    Los valores válidos se encuentran entre 1 y 65535. En un sistema UNIX, la creación de sockets que reciben en los puertos 1-1024 requiere privilegios de superusuario.

  10. En el campo Dirección, escriba la dirección IP en la que recibirá el listener de red.

    La dirección se puede especificar en par punteado o notación IPv6. Puede ser cualquiera (para INADDR_ANY) para recibir en todas las direcciones IP. Puede ser un nombre de host.

  11. En la lista desplegable Pool de Threads, seleccione el pool de threads que se debe asociar al listener de red.

  12. Haga clic en Guardar.

    Sugerencia:

    Para volver a la configuración por defecto, haga clic en el botón Cargar Valores por Defecto y, a continuación, en Guardar.
Pasos Siguientes

Puede utilizar las separadores SSL, HTTP y Caché de Archivos para editar la configuración del protocolo asociado al listener de red. El separador SSL únicamente tiene sentido si se ha activado la seguridad para el protocolo.
