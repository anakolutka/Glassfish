Physisches JMS-Ziel bearbeiten

Auf der Seite "Physisches JMS-Ziel bearbeiten" können Sie die Einstellungen für ein physisches JMS-Ziel ändern.

Die Seite "Physisches JMS-Ziel bearbeiten" enthält die folgenden Optionen.

Name

Der Name des physischen Ziels. Dies ist ein schreibgeschütztes Feld. Sie können den Namen nur festlegen, wenn Sie eine JMS-Zielressource oder ein physisches Ziel erstellen.

Typ

Der Typ des physischen Ziels. Dies ist ein schreibgeschütztes Feld. Sie können den Typ nur festlegen, wenn Sie eine JMS-Zielressource oder ein physisches Ziel erstellen.

Höchstzahl nicht konsumierter Nachrichten

Die für das Ziel maximal zulässige Anzahl nicht konsumierter Nachrichten. Der Wert -1 bedeutet, dass die Anzahl der nicht konsumierten Nachrichten nicht begrenzt ist. Der Standardwert ist -1.

Maximalgröße von Nachrichten

Die maximal zulässige Größe einer einzelnen Nachricht in Byte. Der Wert -1 bedeutet, dass die Nachrichtengröße unbegrenzt ist. Der Standardwert ist -1.

Maximaler Gesamtspeicher

Der maximale Gesamtspeicher für nicht konsumierte Nachrichten in Byte. Der Wert -1 bedeutet, dass die Speichergröße für nicht konsumierte Nachrichten nicht begrenzt ist. Der Standardwert ist -1.

Schwellenwertbehavior

Das Behavior, das der Broker bei Erreichen des Speicherschwellenwerts anwenden soll. Zur Auswahl stehen:

Neueste Nachrichten ablehnen

Die neuesten Nachrichten werden zurückgewiesen, und der produzierende Client wird nur dann mit einer Ausnahme benachrichtigt, wenn die Nachricht persistent ist. Dies ist der Standardwert.

Producer verlangsamen

Die Geschwindigkeit, mit der Nachrichten-Producer Nachrichten senden, wird verringert.

Throw out oldest

Die ältesten Nachrichten werden verworfen.

Throw out lowest-priority messages

Die Nachrichten mit der niedrigsten Priorität werden verworfen.

Wenn der Wert "Throw out oldest" oder "Throw out lowest-priority messages" gewählt und "Queue für nicht zugestellte Nachrichten verwenden" auf "True" gesetzt ist, werden überschüssige Nachrichten in die Queue für nicht zugestellte Nachrichten verschoben.

Höchstzahl von Producern

Die maximale Anzahl an Nachrichten-Producern, die mit diesem physischen Ziel verknüpft werden können. Bei Erreichen dieses Grenzwerts können keine neuen Producer mehr erstellt werden. Der Wert -1 bedeutet, dass die Anzahl der Producer unbegrenzt ist. Der Standardwert ist 100.

Höchstzahl der Nachrichten bei Batchzustellung

Die maximale Anzahl an Nachrichten, die Konsumenten in einem einzigen Batch zugestellt werden können. Der Wert -1 bedeutet, dass die Anzahl der Nachrichten unbegrenzt ist. Der Standardwert ist 1000.

Queue für nicht zugestellte Nachrichten verwenden

Bei aktivierter Option werden nicht zugestellte Nachrichten in die Queue für nicht zugestellte Nachrichten gestellt. Bei deaktivierter Option werden nicht zugestellte Nachrichten verworfen. Diese Option ist standardmäßig aktiviert.

XML-Schema validieren

Bei aktivierter Option versucht die Message Queue-Clientlaufzeitumgebung, eine XML-Meldung anhand der angegebenen XML-Schemadefinitionen (oder anhand der DTD, falls keine XSD angegeben ist) zu überprüfen, bevor sie an den Broker gesendet wird. Wenn das angegebene Schema nicht gefunden oder die Nachricht nicht validiert werden kann, wird die Nachricht nicht gesendet, und eine Exception wird ausgelöst. Diese Option ist standardmäßig deaktiviert.

XML-Schema - URI-Liste

Eine mit Leerzeichen getrennte Liste von URI-Zeichenfolgen einer XML-Schemadefinition (XSD). Wenn die XML-Schemaüberprüfung aktiviert ist, verweisen die URIs auf den Speicherort einer oder mehrerer XSDs, die für die XML-Schemaüberprüfung zu verwenden sind.
