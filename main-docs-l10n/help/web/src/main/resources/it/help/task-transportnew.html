Per creare un trasporto

  1. Espandere il nodo Configurazione nell'albero di navigazione.

  2. Nel nodo Configurazione espandere il nodo Configurazione di rete.

  3. Nel nodo Configurazione di rete selezionare il nodo Trasporti.

    Viene visualizzata la pagina Trasporti.

  4. Nella pagina Trasporti fare clic su Nuovo.

    Viene visualizzata la pagina Nuovo trasporto.

  5. Nel campo Nome digitare un nome univoco per il trasporto.

  6. Nel campo Nome classe digitare il nome della classe dell'implementazione del trasporto.

    Il valore predefinito è com.sun.grizzly.TCPSelectorHandler.

  7. Nel campo Handler chiavi di selezione digitare il nome della classe dell'handler delle chiavi di selezione.

    Per handler delle chiavi di selezione si intende una classe astratta che gestisce il ciclo di vita delle chiavi di selezione. Se non si specifica un valore, verrà utilizzata un'implementazione predefinita.

  8. Nell'elenco a discesa Tipo di buffer di byte selezionare il tipo di buffer di byte da utilizzare.

    Le scelte disponibili sono HEAP e DIRECT. Il valore predefinito è HEAP.

  9. Nel campo Thread processo di accettazione digitare il numero di processori presenti nel computer.

    Il valore -1 indica che Grizzly calcolerà direttamente il numero di thread del processo di accettazione, basandosi sul numero di processori del computer. Il valore predefinito è 1.

    Per impostare il numero dei thread di elaborazione delle richieste, impostare il valore Dimensione massima pool di thread del pool di thread utilizzato da un listener di rete che utilizza questo trasporto.

  10. Nel campo Numero massimo connessioni digitare il numero massimo di connessioni in sospeso su un listener di rete che utilizza questo trasporto.

    Il valore predefinito è 4096.

  11. Nel campo Dimensione buffer digitare la dimensione, in byte, del buffer da fornire per i flussi di input creati da un listener di rete che utilizza questo trasporto.

    Il valore predefinito è 8192.

  12. Nel campo Timeout chiave inattiva digitare il numero di secondi trascorsi i quali una chiave inattiva viene annullata e il canale viene chiusa.

    Il valore predefinito è 30.

  13. Nel campo Timeout di lettura digitare il numero di millisecondi di attesa di GlassFish Server durante la fase di analisi di intestazione e corpo del testo di un'operazione di lettura.

    Il valore predefinito è 30.000.

  14. Nel campo Timeout di polling del selettore digitare il numero di millisecondi di blocco di un selettore NIO in attesa di eventi (richieste di utenti).

    Il valore predefinito è 1000.

  15. Nel campo Timeout di scrittura digitare il numero di millisecondi di attesa di GlassFish Server prima che venga considerata terminata la connessione del client remoto durante la scrittura della risposta.

    Il valore predefinito è 30.000.

  16. Selezionare la casella di controllo Abilitato dell'opzione Visualizza configurazione per eseguire il flush della configurazione interna di Grizzly nei log del server.

    Questa opzione può fornire informazioni utili per il debug. Questa opzione è disabilitata per impostazione predefinita.

  17. Selezionare la casella di controllo Abilitato dell'opzione Snooping per eseguire il dump delle informazioni di richiesta/risposta nel log del server.

    Questa opzione può fornire informazioni utili per il debug, ma causa un peggioramento significativo delle prestazioni. Questa opzione è disabilitata per impostazione predefinita.

  18. Selezionare la casella di controllo TCP senza ritardo per abilitare TCP_NODELAY (noto anche come algoritmo di Nagle).

    Questa opzione è abilitata per impostazione predefinita.

  19. Fare clic su OK.
