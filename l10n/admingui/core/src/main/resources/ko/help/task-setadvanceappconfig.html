고급 응용 프로그램 구성을 설정하는 방법

응용 프로그램 구성 페이지에서는 배포된 응용 프로그램에 대한 변경 사항을 감지하고 수정된 클래스를 다시 로드하도록 도와 주는 등록 정보를 설정할 수 있습니다.

  1. 탐색 트리에서 Application Server 노드를 선택합니다.
  2. 일반 정보 페이지에서 고급 탭을 누릅니다.

    응용 프로그램 구성 페이지가 열립니다.

  3. 응용 프로그램의 동적 다시 로드를 활성화하려면 다시 로드 사용 가능 확인란을 선택합니다.

    동적 다시 로드가 활성화되면 해당 코드 또는 배포 설명자를 변경할 때 응용 프로그램 또는 모듈을 재배포할 필요가 없습니다. 이 경우 변경된 JSPTM(JavaServer PagesTM) 또는 클래스 파일을 응용 프로그램 또는 모듈의 배포 디렉토리에 복사만 하면 됩니다. 서버는 배포 디렉토리에서 변경 사항을 정기적으로 확인하고 변경 사항과 함께 동적이며 자동으로 응용 프로그램을 재배포합니다. 이 기능은 코드 변경을 빠르게 테스트할 수 있기 때문에 개발 환경에서 유용합니다. 그러나 프로덕션 환경에서는 동적 다시 로드가 성능을 저하시킬 수 있습니다. 또한 다시 로드가 수행될 때마다 해당 전송 시간의 세션이 무효화됩니다. 클라이언트가 세션을 다시 시작해야 합니다.

    이 옵션은 기본적으로 활성화됩니다.

  4. 다시 로드 폴링 간격 필드에 응용 프로그램 및 모듈에 코드 변경 사항이 있는지 확인하고 동적으로 다시 로드할 간격(초)을 입력합니다.

    기본값은 2입니다.

  5. 관리 세션 시간 초과 필드에 관리 콘솔 세션 시간 초과의 기준이 되는 비활성 시간(분)을 입력합니다.

    기본값은 60입니다.

  6. 응용 프로그램 자동 배포를 활성화하려면 자동 배포 사용 가능 확인란을 선택합니다.

    자동 배포에서는 응용 프로그램 또는 모듈 파일(JAR, WAR, RAR 또는 EAR)을 특수 디렉토리에 복사해야 합니다. 여기서 이 파일은 Application Server에 의해 자동으로 배포됩니다. 이 옵션은 기본적으로 비활성화됩니다.

  7. 자동 배포 폴링 간격 필드에 응용 프로그램 및 모듈에 코드 변경 사항이 있는지 확인하고 동적으로 다시 로드할 간격(초)을 입력합니다.

    기본값은 2입니다. 폴링 간격을 변경해도 응용 프로그램 또는 모듈 배포에 걸리는 시간은 영향을 받지 않습니다.

  8. 자동 배포 재시도 시간 초과 필드에 부분적으로 복사된 파일의 크기가 그대로 유지되어도 오류가 발생하지 않는 시간(초)을 입력합니다.

    기본값은 30입니다.

  9. 자동 배포 디렉토리 필드에 응용 프로그램의 자동 배포를 모니터링할 디렉토리의 전체 경로를 입력합니다.
  10. 배포 설명자 파일을 검증하려면 검증자 사용 가능 확인란을 선택합니다.
  11. JSP 파일을 사전 컴파일하려면 사전 컴파일 사용 가능 확인란을 선택합니다.
  12. (옵션) 추가 등록 정보 섹션에서 추가 등록 정보를 지정합니다.
  13. 저장을 누릅니다.
  14. 서버를 다시 시작합니다.
