Definições do Logger

Utilize a página Definições de Logger para configurar o log do cluster ou instância selecionada do GlassFish Server.

A página Definições de Logger contém as opções a seguir.

Nome da Configuração

O nome da configuração à qual as definições nesta página se aplicam. Este campo é somente para leitura.

Gravar no Log do Sistema

Quando ativado, a saída de log será enviada para o syslog de facilidade em adição no log do servidor. Esse recurso está disponível apenas nos sistemas Solaris e Linux. Por default, essa opção está desativada.

Log para Console

Quando essa opção é ativada, a saída de log é enviada à console do sistema.

Rotação na Alteração da Data

Quando essa opção é ativada, o GlassFish Server rotaciona arquivos de log a cada dia, à meia-noite.

Modo com Várias Linhas

Quando essa opção é ativada, as mensagens de log são formatadas de uma forma que ficam mais fáceis de serem lidas. O corpo da mensagem de log é recuado e começa em uma nova linha.

Formato de Log da Console

O formato de log para registro na console. As opções disponíveis são ULF (UniformLogFormatter) e ODL (Oracle Diagnostics Logging). O default é ODL.

Formato de Log do Arquivo de Log

O formato de log para registro no arquivo de log do servidor. As opções disponíveis são ULF e ODL. O default é ODL.

Limite de Rotação de Arquivo

O tamanho em bytes que o arquivo do log do servidor deve alcançar antes de ser rotacionado. Quando o log do servidor alcançar o tamanho especificado, o GlassFish Server cria um novo arquivo vazio denominadoserver.log e renomeia o arquivo antigo server.log_data, onde data é a data e a hora em que o arquivo foi rotacionado. O valor default é 2 megabytes. O valor mínimo para o limite é de 500 quilobytes; se você especificar um valor menor, o arquivo será rotacionado quando alcançar 500 KB. Para desativar a rotação de arquivos de log, defina o valor para 0.

Limite de Tempo para Rotação de Arquivos

O número de minutos após o qual o log do servidor deve ser rotacionado. O valor default é 0, o que significa que o arquivo é rotacionado quando ele alcança o tamanho especificado no campo Limite de Rotação de Arquivo. Se você especificar um ou mais minutos, o limite de tempo tem precedência sobre o limite de tamanho especificado pelo campo Limite de Rotação de Arquivo.

Frequência de Esvaziamento

O número máximo de mensagens a serem escritas de cada vez da fila para o log do servidor. O valor default é 1.

Máximo de Arquivos de Histórico

O número máximo de arquivos de log que o GlassFish Server mantém antes de deletar o arquivo mais antigo. Se você definir este valor como zero (0), o GlassFish Server não deletará os antigos arquivos.

Arquivo de Log

Um nome alternativo ou local do arquivo do log do servidor. O valor default é domain-dir/logs/server.log.

Handler de Log

O nome da classe absoluta de um handler de log personalizado. Um handler de log personalizado permite que você envie logs para um destino diferente de server.log ou syslog O handler personalizado deve estender a classe java.util.logging.Handler (uma API compatível com JSR 047). Coloque a classe do handler no caminho da classe do GlassFish Server, para que o handler seja instalado durante a inicialização do servidor.
