HTTP 服务

“HTTP 服务”页面包含以下选项：

访问日志记录

如果选中此项，则对所有未指定此属性的虚拟服务器子元素启用访问日志记录。如果未选中此项，则对所有未指定此属性的虚拟服务器子元素禁用访问日志记录。

访问日志缓冲区大小

访问日志缓冲区的大小（以字节为单位），或者是小于或等于零的值（对于未缓冲的访问日志）。默认值为 4096。

访问日志写入时间间隔

将日志写入磁盘之前的秒数。缓冲区已满或超过时间间隔时写入访问日志。如果该值小于或等于 0，则缓冲区始终被写入，即使缓冲区未满也是如此。这意味着每次访问服务器时都将日志消息直接存储到该文件中。

其他属性

HTTP 服务的其他属性。

以下属性可用：

connectionTimeout

指定在接受一个连接后，请求 URI 行被提交之前 HTTP 侦听器需等待的毫秒数。

默认值是 12000（12 秒）。

maxKeepAliveRequests

指定在服务器关闭连接之前可以连续处理的最大 HTTP 请求数。将此属性设置为 1，将禁用 HTTP/1.0 的保持活动功能和 HTTP/1.1 的保持活动以及连续处理功能。

默认值是 1000。

traceEnabled

将此属性设置为 true，会启用 TRACE 方法。将此属性设置为 false，会使 Enterprise Server 少受跨站点脚本攻击的影响。

有关 TRACE 方法的更多信息，请参见 RFC 2616：超文本传输协议 -- HTTP/1.1。

默认值是 true。
