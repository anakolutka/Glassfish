Configuración de tiempo de ejecución por lotes

GlassFish Server proporciona un tiempo de ejecución por lotes para la programación y ejecución de trabajos por lotes. Los trabajos por lotes suelen ser de larga ejecución, tareas orientadas a lotes que contienen varios pasos. Las aplicaciones por lotes envían trabajos al tiempo de ejecución por lotes y proporcionan instrucciones sobre cómo y cuándo ejecutar los pasos. El tiempo de ejecución por lotes procesa los pasos y almacena información sobre los trabajos en un repositorio de trabajo. En GlassFish Server, el repositorio de trabajo es una base de datos.

Utilice la página Configuración de tiempo de ejecución por lotes para ver y cambiar la configuración del tiempo de ejecución por lotes. El tiempo de ejecución por lotes utiliza un servicio de ejecutor gestionado y un origen de datos para ejecutar los trabajos por lotes. El servicio de ejecutor gestionado proporciona threads a los trabajos y el origen de datos almacena la información de los trabajos. Los datos de configuración de tiempo de ejecución por lotes se almacenan en el elemento config de domain.xml.

La página Configuración de tiempo de ejecución por lotes contiene la siguiente información.

Nombre de la instancia
Nombre del cluster

Nombre de la instancia de servidor o cluster de GlassFish al que se aplica la configuración. Éste es un campo de sólo lectura.

Nombre de la búsqueda de servicio de ejecutor

Nombre de la búsqueda de JNDI del servicio de ejecutor gestionado que proporciona threads a los trabajos por lotes. El servicio de ejecutor gestionado por defecto es concurrent/__defaultManagedExecutorService.

El servicio de ejecutor gestionado se puede cambiar una vez que el trabajo por lotes se ha ejecutado en el tiempo de ejecución por lotes sin afectar a la ejecución del trabajo.

Nombre de la búsqueda de origen de datos

Nombre de la búsqueda de JNDI del origen de datos que almacena información sobre los trabajos por lotes actuales y pasados. El origen de datos por defecto es jdbc/__TimerPool.

No cambie el origen de datos después de que se haya ejecutado el primer trabajo por lotes en el tiempo de ejecución por lotes para su ejecución. Si debe abrir el origen de datos, pare y reinicie el dominio y, a continuación, realice el cambio antes de que se inicie o reinicie cualquier trabajo. Sin embargo, una vez que el origen de datos se ha cambiado, ya no se podrá acceder a la información almacenada en el origen de datos anterior.
