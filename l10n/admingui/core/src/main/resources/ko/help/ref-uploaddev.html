엔터프라이즈 응용 프로그램/모듈 배포

엔터프라이즈 응용 프로그램/모듈 페이지를 사용하여 응용 프로그램을 배포합니다. 페이지에는 다음 옵션이 포함되어 있습니다.

유형

배포 중인 응용 프로그램의 유형입니다. 이 릴리스에서 사용할 수 있는 유일한 응용 프로그램 유형은 웹 응용 프로그램입니다(.war).

위치

배포 중인 응용 프로그램의 아카이브 위치입니다.

다음 옵션에서는 아카이브에서 액세스할 수 있는 위치 및 아카이브가 파일 또는 디렉토리인지 여부를 지정합니다.

서버에 업로드할 패키지화된 파일

아카이브는 클라이언트 시스템에 상주하거나 클라이언트 시스템에서 액세스할 수 있는 파일에 있습니다.

클라이언트 시스템은 브라우저를 통해 관리 콘솔을 보는 호스트입니다.

Application Server에서 액세스할 수 있는 패키지화된 로컬 파일 또는 디렉토리

아카이브는 서버 시스템에 상주하는 파일이거나 확장된 디렉토리의 압축 해제된 응용 프로그램입니다.

서버 시스템은 Application Server 도메인 관리 서버를 실행하는 호스트입니다.

응용 프로그램 이름

응용 프로그램의 이름입니다.

컨텍스트 루트

응용 프로그램에 대한 경로입니다. 웹 응용 프로그램의 URL에서 컨텍스트 루트 바로 앞에는 포트 번호(http://host:port/context-root/...)가 있습니다. 컨텍스트 루트는 슬래시로 시작해야 합니다(예: /hello).

가상 서버

이 응용 프로그램에 연결된 가상 서버입니다.

상태 사용 가능

이 옵션을 선택하면 응용 프로그램이 활성화됩니다. 이 옵션은 기본적으로 선택됩니다.

검증자 실행 사용 가능

이 옵션을 선택하면 검증자 유틸리티를 실행하여 응용 프로그램이 JavaTM Platform, Enterprise Edition(Java EE 플랫폼) 및 Enterprise Server를 준수하는지 확인합니다.

JSP 사전 컴파일 사용 가능

이 옵션을 선택하면 JSP(JavaServer Pages) 파일이 사전 컴파일됩니다. 이 옵션을 선택 취소하면 JSP 파일은 런타임에 처음으로 액세스될 때 컴파일됩니다. 이 옵션은 기본적으로 선택 취소됩니다.

라이브러리

응용 프로그램에 필요한 모든 라이브러리 JAR 파일입니다.

설명

응용 프로그램에 대한 간단한 설명입니다.

관련 작업

  • 웹 응용 프로그램을 배포하는 방법

  • 배포된 웹 응용 프로그램을 편집하는 방법

  • 웹 응용 프로그램의 설명자를 보는 방법

  • 웹 응용 프로그램을 시작하는 방법

  • 배포된 웹 응용 프로그램을 재배포하는 방법

관련 asadmin 명령

  • deploy(1)

  • deploydir(1)

  • redeploy(1)

  • list-applications(1)

  • list-modules(1)
