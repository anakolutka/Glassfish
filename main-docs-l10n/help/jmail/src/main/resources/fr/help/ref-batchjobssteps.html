Etapes d'exécution de traitement batch

La page Etapes d'exécution de traitement batch affiche des détails sur les étapes d'une exécution de traitement batch spécifique. Les informations suivantes apparaissent.

Nom d'instance
Nom du cluster

Nom de l'instance de serveur et du cluster GlassFish auquel s'applique la configuration.

Nom de travail

Nom du traitement batch.

ID d'exécution

ID affecté à l'exécution du traitement batch. Une exécution est créée la première fois qu'un travail est démarré et chaque fois que l'exécution existante est redémarrée.

Etapes de travail

Etapes d'une exécution de traitement batch spécifique. Les informations suivantes sont affichées pour chaque étape :

Nom d'étape

Nom de l'étape.

Statut de batch

Statut de l'étape, tel que défini par l'exécution de batch.

Statut de sortie

Statut de l'étape, tel que défini par le fichier XML du travail ou par l'application de traitement batch. Par défaut, le statut de sortie et le statut de batch sont identiques, sauf si le statut de sortie est explicitement remplacé.

Heure de début

Heure de début de l'étape.

Heure de fin

Heure de fin de l'étape.

Mesures d'étapes

Mesures de l'étape, listant le nombre d'éléments lus, écrits, validés (commit), etc.
