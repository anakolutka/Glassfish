Para Criar um Listener de Rede

Antes de Começar

Quando você cria um listener de rede, pode criar simultaneamente um protocolo que será utilizado por ele ou um protocolo criado primeiro. Normalmente, há um relacionamento um-a-um entre o listener de rede e seu protocolo associado, mas isso não é necessário.

  1. Na árvore de navegação, expanda o nó Configuração.

  2. No nó Configuração, expanda o nó Configuração da Rede.

  3. No nó Config da Rede, selecione o nó Listeners de Rede.

    A página Listeners de Rede será aberta.

  4. Na página Listeners de Rede, clique em Novo.

    A página Novo Listener de Rede será aberta.

  5. No campo Nome, digite um nome para o listener de rede.

    Um nome do listener de rede não pode começar com um número.

  6. No campo Protocolo, especifique o protocolo a ser associado ao listener de rede.

    • Selecione Criar um Novo Protocolo, para criar um novo protocolo com definições default. Também, na lista drop-down Servidor Virtual, selecione o servidor virtual a ser associado ao protocolo.

      O nome default do protocolo é o nome do listener de rede com -protocol acrescentado, mas é possível editar esse valor.

    • Selecione Usar um Protocolo Existente e, em seguida, selecione um protocolo na lista drop-down. Para ativar a segurança para o listener de rede, escolha um protocolo que tenha a segurança ativada.

      Caso você selecione esta opção, normalmente já teria criado o protocolo para esse listener.

    Você somente pode especificar um protocolo ao criar um novo listener de rede. Você não pode alterar em outro momento o protocolo associado. O protocolo determina se a segurança está ativada para o listener de rede.

  7. Marque a caixa de seleção Status Ativado para ativar o listener de rede.

    Por default, a opção está ativada. Se o listener estiver desativado, qualquer tentativa de conectar ao listener resulta em uma exceção de soquete (java.net.ConnectException).

    Nas versões do GlassFish Server anteriores a 9.1, um listener cujo atributo ativado estava definido como falso retornava um código de resposta 404, para quaisquer solicitações enviadas ao mesmo. Para obter esse comportamento na versão atual do GlassFish Server, defina o status do listener como ativado e cada estado de servidor virtual associado como OFF. Um servidor virtual lista suas listeners associadas na sua lista drop-down Listeners de Rede.

  8. A caixa de seleção somente para leitura Segurança Ativada indica se a segurança está ativada para o protocolo usado pelo listener de rede.

    Se você estiver configurando o admin-listener para o servidor de administração de domínios (DAS), que usa a configuração server-config, esta opção será somente para leitura. Para ativar a segurança deste listener, é necessário ativar a administração segura. Consulte Administração Segura.

  9. Marque a caixa de seleção Listener JK ativado para especificar que o listener suporta o protocolo mod-jk do Apache.

    O protocolo mod-jk é um protocolo proprietário de comunicação/rede entre o httpd do Apache (servidor da web) e uma instância de back-end do GlassFish Server. Se o listener oferecer suporte a esse protocolo, o httpd deverá tratar todo o conteúdo estático, enquanto as solicitações por recursos dinâmicos (ou seja, Servlets e JSPs) serão roteadas para o listener de rede ativado por mod-jk do GlassFish Server, utilizando o protocolo mod-jk. Por default, essa opção está desativada.

  10. No campo Porta, digite o número da porta do listener de rede.

    Os valores válidos são 1 a 65.535. Em sistemas UNIX, criar soquetes que fazem listening nas portas 1 - 1024 exige privilégios de superusuário.

  11. No campo Endereço, digite o endereço IP do listener de rede.

    O endereço pode estar na notação de par de pontos ou IPv6. Pode ser qualquer (para INADDR_ANY) para fazer listening em todos os endereços IP. Ele pode ser um nome de host.

  12. Na lista drop-down Pool de Threads, selecione o pool de threads a ser associado ao listener de rede.

    Normalmente, você seleciona um dos dois pool de threads que estão configurados quando o GlassFish Server é instalado.

  13. Na lista drop-down Transporte, selecione o tipo de transporte a ser usado pelo listener de rede.

    Por default, a escolha é tcp.

  14. Clique em OK.

Próximas Etapas

Se você tiver ativado a segurança para o protocolo usado pelo listener de rede, será possível usar a guia SSL para editar as definições de SSL do protocolo.
