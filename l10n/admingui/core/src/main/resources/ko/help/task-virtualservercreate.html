가상 서버를 만드는 방법

  1. 탐색 트리에서 구성 노드를 선택합니다.
  2. 구성 노드에서 HTTP 서비스 노드를 확장합니다.
  3. HTTP 서비스 노드에서 가상 서버 노드를 선택합니다.
  4. 가상 서버 페이지에서 새로 만들기를 누릅니다.

    새 가상 서버 페이지가 열립니다.

  5. 아이디 필드에 가상 서버를 내부적으로 식별하는 값을 입력합니다.

    이 값은 HTTP 클라이언트에 노출되지 않습니다. HTTP 클라이언트에 노출되는 호스트 이름은 호스트 필드에서 지정해야 합니다.

  6. 호스트 필드에 서버가 실행되고 있는 시스템의 호스트 이름 또는 IP 주소를 입력합니다.

    네트워크의 DNS 서버(UNIX® 시스템에서는 /etc/hosts 파일)에 등록된 실제 또는 가상 호스트 이름을 사용합니다.

  7. 새 가상 서버에 대한 상태 옵션을 선택합니다.

    가상 서버는 기본적으로 활성화됩니다.

  8. HTTP 수신기 필드를 비워 둡니다.

    이 필드는 HTTP 수신기를 만들고 이 서버와 연결하면 자동으로 입력됩니다.

    이 필드를 사용하려면 기존 HTTP 수신기를 지정해야 합니다. 하지만 다른 가상 서버가 사용하는 수신기를 지정하지 않아야 합니다. 그렇지 않으면 서버를 다시 시작할 때 서버 로그에 오류가 나타납니다. 수신기는 만들 때 기존 가상 서버와 연결되어야 하므로 기존 수신기는 모두 다른 가상 서버에서 사용됩니다.

  9. (옵션) 기본 웹 모듈 드롭다운 목록에서 가상 서버에 배포된 다른 웹 모듈에 매핑할 수 없는 모든 요청에 응답할 모듈을 선택합니다.

    기본 웹 모듈이 지정되지 않은 경우 빈 컨텍스트 루트가 있는 웹 모듈이 사용됩니다. 빈 컨텍스트 루트가 있는 웹 모듈이 없을 때는 시스템 기본 웹 모듈이 만들어져 사용됩니다.

  10. (옵션) 로그 파일 필드에 이 가상 서버로부터의 로깅 메시지가 나타날 파일의 경로 이름을 입력합니다.

    필드를 비워 두면 로깅 메시지는 기본 서버 로그인 domain-dir/logs/server.log에 전송됩니다.

  11. (옵션) Docroot 필드에 서버의 루트 문서 디렉토리에 대한 절대 경로를 입력합니다.

    이 필드를 비워 두면 domain-dir/docroot가 사용됩니다.

  12. 이 가상 서버에 대해 단일 사인 온을 활성화하려면 SSO 사용 가능 확인란을 선택합니다.

    이 옵션은 기본적으로 비활성화됩니다. 단일 사인 온이 비활성화되어 있는 경우 사용자는 가상 서버의 모든 응용 프로그램에 별도로 인증해야 합니다.

    Enterprise Server의 응용 프로그램에 대한 단일 사인 온은 서블릿 및 JSP 파일에서 지원됩니다. 이 기능을 통해 사용자가 응용 프로그램마다 따로 사인 온하는 것이 아니라 같은 사용자 사인 온 정보가 필요한 여러 응용 프로그램이 이 정보를 공유할 수 있습니다.

  13. 액세스 로깅을 활성화하거나 비활성화하려면 액세스 로깅 옵션을 선택합니다.

    그 외에 액세스 로깅이 HTTP 서비스에서 상속된 경우에도 지정할 수도 있습니다.

  14. 디렉토리 필드에 서버 액세스 로그의 절대 디렉토리 경로를 입력합니다.

    기본값은 domain-dir/logs/access입니다.

  15. (옵션) 버퍼 크기 필드에 버퍼 크기(바이트)를 입력합니다.

    0보다 작거나 같은 값은 버퍼링이 비활성화됩니다. 이 필드를 비워 두면 액세스 로그 버퍼 크기는 HTTP 서비스에서 상속됩니다.

  16. (옵션) 쓰기 간격 필드에 액세스 로그 업데이트 사이의 간격(초)을 입력합니다.
  17. (옵션) 추가 등록 정보 섹션에 가상 서버에 대한 추가 등록 정보를 지정합니다.

    등록 정보의 지정 여부에 관계 없이 새 서버에는 기본값으로 설정된 기본 등록 정보 docroot 및 accesslog가 있습니다. accessLoggingEnabled의 기본 설정은 개발자 및 클러스터 프로필의 경우 false, 엔터프라이즈 프로필의 경우 true입니다. true이면 이 가상 서버에 대해서만 액세스 로깅을 활성화합니다. false이면 이 가상 서버에 대해서만 액세스 로깅을 비활성화합니다.

    다음 가상 서버 등록 정보가 제공됩니다.

    sso-max-inactive-seconds

    클라이언트 작업이 수신되지 않을 경우 사용자의 단일 사인 온 레코드를 제거할 수 있는 경과 시간(초)을 지정합니다. 단일 사인 온은 같은 가상 서버에 있는 여러 응용 프로그램에 적용되므로 응용 프로그램 중 하나에 액세스하면 단일 사인 온 레코드가 활성 상태를 유지합니다.

    기본값은 300초(5분)입니다. 이 값이 높을수록 사용자에 대한 단일 사인 온 지속 시간이 길어지지만 서버의 메모리 사용량은 많아집니다.

    sso-reap-interval-seconds

    만료된 단일 사인 온 레코드의 제거 간격(초 단위)을 지정합니다.

    기본값은 60입니다.

    allowLinking

    true이면 기호 링크인 자원이 이 가상 서버에 배포된 모든 웹 응용 프로그램에 대해 작동합니다. sun-web.xml 파일의 sun-web-app 등록 정보 allowLinking을 사용하여 개별 웹 응용 프로그램이 이 설정을 대체할 수 있습니다.

    <sun-web-app>
   <property name="allowLinking" 
      value="{true|false}"/>
</sun-web-app>

    기본값은 false입니다.

  18. 확인을 누릅니다 .
