Para crear un listener de red

Antes de Empezar

Cuando se crea un listener de red, es posible crear simultáneamente un protocolo que utilizará el listener de red o utilizar un protocolo que cree previamente. Normalmente, existe una relación de uno a uno entre un listener de red y su protocolo asociado, pero no es necesario que sea así.

  1. En el árbol de navegación, amplíe el nodo Configuración.

  2. En el nodo Configuración, amplíe el nodo Configuración de Red.

  3. En el nodo Configuración de Red, seleccione el nodo Listeners de Red.

    Se abrirá la página Listeners de Red.

  4. En la página Listeners de Red, haga clic en Nuevo.

    Se abrirá la página Nuevo Listener de Red.

  5. En el campo Nombre, escriba un nombre para el listener de red.

    Un nombre de listener de red no puede empezar con un número.

  6. En el campo Protocolo, especifique el protocolo que se va a asociar al listener de red.

    • Seleccione Crear un Protocolo Nuevo para crear un nuevo protocolo con una configuración por defecto. Además, en la lista desplegable Servidor Virtual, seleccione el servidor virtual que se va a asociar al protocolo.

      El nombre por defecto del protocolo es el nombre del listener de red con la terminación -protocol agregada, pero es posible editar este valor.

    • Seleccione Usar un Protocolo Existente y, a continuación, seleccione un protocolo de la lista desplegable. Para activar la seguridad del listener de red, elija un protocolo con la seguridad activada.

      Si selecciona esta opción, por lo general habrá creado previamente el protocolo de este listener.

    Puede especificar el protocolo únicamente si ha creado el listener de red. No puede cambiar el protocolo asociado más adelante. El protocolo determina si se ha activado la seguridad para el listener de red.

  7. Active la casilla de control Estado activado para activar el listener de red.

    Esta opción se encuentra activada por defecto. Si el listener se encuentra desactivado, cualquier intento de establecer una conexión con él tiene como resultado una excepción de socket (java.net.ConnectException).

    En las versiones anteriores a la 9.1 de GlassFish Server, un listener cuyo atributo activado se hubiera definido como false devolvía un código de respuesta 404 para cualquier solicitud que se le enviaba. Para lograr este comportamiento en la versión actual de GlassFish Server, defina el estado del listener en activado y defina cada estado de servidor virtual asociado como desactivado. Un servidor virtual incluye sus listeners asociados en la lista desplegable Listeners de Red.

  8. La casilla de control Seguridad Activada, de sólo lectura, indica si se ha activado la seguridad para el protocolo que utiliza el listener de red.

    Si está configurando el admin-listener para el servidor de administración de dominio (DAS), que usa la configuración server-config, esta opción será de sólo lectura. Para activar la seguridad de este listener, debe activar primero la administración segura. Consulte la sección Administración Segura.

  9. Active la casilla de control de activación de listener JK para especificar que el listener soporta el protocolo mod-jk de Apache.

    El protocolo mod-jk es un protocolo de red/comunicación de propiedad entre httpd de Apache (servidor web) y una instancia back-end de GlassFish Server. Si el listener soporta este protocolo, se supone que httpd maneja todo el contenido estático, mientras que cualquier solicitud de recursos dinámicos (es decir, servlets y JSP) se direccionan al listener de red con el protocolo mod jk activado de GlassFish Server a través del protocolomod-jk. Esta opción se activada por defecto.

  10. En el campo Puerto, introduzca el número de puerto en el que recibirá el listener de red.

    Los valores válidos se encuentran entre 1 y 65535. En un sistema UNIX, la creación de sockets que reciben en los puertos 1-1024 requiere privilegios de superusuario.

  11. En el campo Dirección, escriba la dirección IP en la que recibirá el listener de red.

    La dirección se puede especificar en par punteado o notación IPv6. Puede ser cualquiera (para INADDR_ANY) para recibir en todas las direcciones IP. Puede ser un nombre de host.

  12. En la lista desplegable Pool de Threads, seleccione el pool de threads que se debe asociar al listener de red.

    Normalmente, puede seleccionar uno de los dos pools de threads que están configurados cuando se instala GlassFish Server.

  13. En la lista desplegable Transporte, seleccione el tipo de transporte que se debe utilizar en el listener de red.

    Por defecto, la única alternativa es tcp.

  14. Haga clic en Aceptar.

Pasos Siguientes

Si ha activado la seguridad del protocolo utilizado por el listener de red, puede usar el separador SSL para editar la configuración SSL de dicho protocolo.
