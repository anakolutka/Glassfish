Editar conector JMX

Utilice la página Editar conector JMX para editar el conector JMX compatible con JSR 360.

La página Editar conector JMX contiene las opciones siguientes.

Nombre de Configuración

Nombre de la configuración a la que se aplican los valores de esta página. Este campo es de sólo lectura.

Nombre

Nombre del conector JMX. Éste es un campo de sólo lectura.

Protocolo JMX

Nombre del protocolo que soporta este conector JMX. Éste es un campo de sólo lectura.

JMXServiceURL

URL del servicio JMX. Éste es un campo de sólo lectura.

Seguridad

Si la casilla de control Activado está activada, la comunicación de JMX será cifrada. Esta opción se activada por defecto.

Dirección

Dirección IP del servicio de nomenclatura en el que está registrado el stub del servidor del conector JMX. Éste no es el puerto del socket del servidor que permite la comunicación de JMX. Ésta es la dirección de la interfaz de red donde se ha iniciado el registro de RMI. Si el sistema dispone de varias interfaces de red, modifique este valor para que sólo se seleccione una interfaz determinada. El valor por defecto es 0.0.0.0.

Puerto

Número de puerto en el que el servicio de nomenclatura (registro de RMI) recibe las conexiones del cliente de RMI. La única función de este servicio de nomenclatura es descargar los stubs de RMI. Si el puerto por defecto está ocupado, se utilizará otro puerto que esté libre. Los valores válidos se encuentran entre 1 y 65535. En sistemas UNIX, para la creación de sockets de recepción para los puertos 1-1024 se necesitan privilegios de superusuario. El valor por defecto es 8686.

Nombre de dominio

Nombre que representa el dominio administrativo especial. Este dominio maneja toda la autenticación. El valor por defecto es admin-realm.

Propiedades Adicionales

Propiedades adicionales del conector JMX. Hay varias propiedades especificadas por defecto.
