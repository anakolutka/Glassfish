Per configurare le impostazioni del servizio GMS per un cluster

GMS (Group Management Service) è un servizio in-process che offre servizi di monitoraggio del cluster, appartenenza del cluster e comunicazione di gruppo. Il servizio GMS esegue le funzioni seguenti:

  • notifica ai moduli registrati in un'istanza di GlassFish Server l'aggiunta o la rimozione di una o più istanze membro da un cluster oppure un errore sospettato o confermato di tali istanze;

  • consente di inviare e ricevere messaggi tra i membri del cluster.

  1. Espandere il nodo Cluster nell'albero di navigazione.

  2. Selezionare il cluster per il quale configurare le impostazioni del servizio GMS.

    Viene visualizzata la pagina Informazioni generali per il cluster.

  3. Selezionare la casella di controllo Abilitato dell'opzione GMS per abilitare il servizio GMS per il cluster.

    Questa opzione è selezionata per impostazione predefinita. Per disabilitare il servizio GMS per il cluster, rimuovere il segno di spunta dalla casella di controllo GMS abilitato.

    Se abilitato, il servizio GMS viene avviato in ogni istanza server nel cluster e nel server di amministrazione del dominio (DAS). Il server DAS è attivo in ogni cluster in cui è stato abilitato il servizio GMS.

  4. Nel campo Porta multicast digitare la porta utilizzata dal servizio GMS per l'ascolto degli eventi di gruppo.

    Specificare un numero di porta UDP standard nell'intervallo compreso tra 2048 e 49151. L'impostazione predefinita è un valore generato in modo automatico in questo intervallo.

  5. Nel campo Indirizzo multicast digitare l'indirizzo utilizzato dal servizio GMS per l'ascolto degli eventi di gruppo.

    Specificare un indirizzo multicast IP di classe D. Gli indirizzi IP di classe D rientrano nell'intervallo compreso tra 224.0.0.0 e 239.255.255.255 inclusi. L'indirizzo 224.0.0.0 è riservato e non deve essere utilizzato. L'impostazione predefinita è 228.9.XX.YY, dove XX e YY rappresentano valori indipendenti generati in modo automatico compresi tra 0 e 255.

  6. Nel campo Associa indirizzo di interfaccia digitare l'indirizzo dell'interfaccia di rete per il servizio GMS se sono disponibili più interfacce di rete.

    Questa impostazione consente di configurare l'interfaccia di rete da utilizzare per il servizio GMS se sono disponibili più interfacce su uno dei computer host delle istanze del server nel cluster. Il valore deve essere un indirizzo IP di interfaccia di rete locale. Per impostazione predefinita vengono utilizzati tutti gli indirizzi di interfaccia di rete pubblici.

    Per specificare un indirizzo valido per tutte le istanze di GlassFish Server nel cluster, utilizzare una proprietà di sistema per impostare l'indirizzo singolarmente per ciascuna istanza.

    Ad esempio, creare la proprietà di sistema GMS-BIND-INTERFACE-ADDRESS-nome-cluster nel server DAS. Impostare quindi l'opzione Associa indirizzo di interfaccia su ${GMS-BIND-INTERFACE-ADDRESS-nome-cluster} per specificare la proprietà di sistema. Infine, per ogni istanza nel cluster, impostare la proprietà di sistema GMS-BIND-INTERFACE-ADDRESS-nome-cluster sull'indirizzo dell'interfaccia di rete necessario nel computer dell'istanza.

  7. Per specificare proprietà aggiuntive, selezionare la scheda Proprietà e la scheda secondaria Proprietà cluster.

    • Per aggiungere una proprietà, fare clic sul pulsante Aggiungi proprietà. Nella riga vuota visualizzata, digitare il nome della proprietà nel campo Nome e il valore della proprietà nel campo Valore.

    • Per modificare una proprietà, è necessario modificare il valore del campo Valore della proprietà.

    • Per eliminare una proprietà, selezionare la casella di controllo a sinistra del campo Nome corrispondente alla proprietà da eliminare, quindi fare clic sul pulsante Elimina proprietà.

  8. Fare clic su Salva.
