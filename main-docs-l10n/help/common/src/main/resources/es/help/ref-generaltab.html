Información General

GlassFish Server crea una instancia de servidor de aplicaciones denominada server durante la instalación. Si lo prefiere, puede suprimir esta instancia de servidor y crear una nueva instancia con un nombre distinto.

Cada instancia de GlassFish Server tiene su propia configuración Java, recursos Java, áreas de despliegue de aplicaciones y valores de configuración del servidor. Los cambios en una instancia de servidor de aplicaciones no tienen efecto en otras instancias del mismo tipo. Puede tener una instancia de servidor de aplicaciones dentro de un dominio administrativo.

Para muchos usuarios, una instancia de servidor de aplicaciones es suficiente para sus necesidades. Sin embargo, según su entorno, puede que le convenga crear una o más instancias adicionales de servidor de aplicaciones. Por ejemplo, en un entorno de desarrollo puede utilizar diferentes instancias de servidor de aplicaciones para probar distintas configuraciones de GlassFish Server o comparar y probar distintos despliegues de aplicaciones. Como puede agregar y suprimir con facilidad instancias de servidor de aplicaciones, puede utilizarlas para crear zonas "sandbox" temporales para experimentar durante los trabajos de desarrollo.

Además, para cada instancia de servidor de aplicaciones se pueden crear servidores virtuales. Dentro de una sola instancia instalada de servidor de aplicaciones puede ofrecer, tanto a compañías como individuos, nombres de dominio, direcciones IP y algunas funciones administrativas. Para los usuarios resulta similar a disponer de su propio servidor web sin la complejidad del mantenimiento básico del servidor y del hardware. Estos servidores virtuales no incluyen, a su vez, instancias de servidor de aplicaciones. Para obtener más información sobre servidores virtuales, consulte Configuración general de JVM.

En los despliegues operativos, puede utilizar los servidores virtuales en lugar de varias instancias de servidor de aplicaciones para una amplia variedad de fines. No obstante, si los servidores virtuales no se ajustan a sus necesidades, también puede usar varias instancias de servidor de aplicaciones.

Una instancia de GlassFish Server no se inicia de forma automática. Una vez que inicie la instancia, ésta se ejecutará hasta que usted la pare. Al parar una instancia de servidor de aplicaciones, dejará de aceptar nuevas conexiones y esperará a que finalicen las conexiones pendientes. Si la máquina se bloquea o se queda fuera de línea, el servidor se cierra y se pueden perder las solicitudes en curso.

Consulte la página Información General para comprobar la configuración de GlassFish Server y ver los datos de Java Virtual Machine.

La página Información General contiene la siguiente información.

Botón Parar

Haga clic en Parar para parar GlassFish Server.

Botón Reiniciar

Haga clic en Reiniciar para reiniciar GlassFish Server.

Botón Ver Archivos Log

Haga clic en el botón Ver Archivos Log para ver los archivos log para una instancia o cluster de GlassFish Server.

Botón Rotar Log

Haga clic en el botón Rotar Log para rotar el archivo log para el servidor de administración (llamado server).

Botón Recuperar Transacción

Haga clic en este botón para recuperar transacciones para el servidor de administración (llamado server) en la página Recuperar transacciones.

Botón Administración Segura

Haga clic en el botón Administración Segura para activar o desactivar la administración segura en la página Administración Segura.

Nombre

Nombre del servidor actual.

Estado

Estado actual de la instancia de servidor. El servidor puede estar parado, iniciado o en ejecución.

Tiempo de Actividad

Número de horas y minutos que la instancia de servidor ha estado continuamente en ejecución. Es de sólo lectura.

Equilibrador de Carga de HTTP

Nombre del equilibrador de carga, si está activado para la instancia del servidor.

JVM

Si hace clic en Informe de JVM, se abrirá una ventana independiente que muestra informes sobre Java Virtual Machine, incluyendo un informe resumido, un informe sobre gestión de la memoria y recolección de basura, otro informe sobre carga de clase y un volcado de thread actual.

Directorio de Configuración

Directorio de la máquina host que contiene archivos de configuración, como domain.xml.

Versión Instalada

Identificador de versión del producto.

Administración Segura

Estado actual de la administración segura. Los valores posibles son Activada y No Activada.

Depurar

Identifica si la depuración está activada. Si la depuración está activada, también aparecerá el número de puerto.

Puerto(s) HTTP

Puertos configurados actualmente para solicitudes y respuestas HTTP.

Puerto(s) IIOP

Puertos configurados actualmente para solicitudes y respuestas IIOP.
