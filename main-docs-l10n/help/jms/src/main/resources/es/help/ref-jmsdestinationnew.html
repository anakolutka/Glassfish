Nuevo Recurso de Destino JMS

Utilice la página Nuevo Recurso de Destino JMS para crear un nuevo recurso de destino JMS.

La página Nuevo Recurso de Destino JMS contiene las siguientes opciones.

Nombre JNDI

Nombre único que identifica el recurso de destino.

Es una práctica recomendada utilizar el prefijo de subcontexto de nomenclatura jms/ para recursos de JMS. Por ejemplo: jms/Queue.

Nombre de Destino Físico

Nombre del destino físico que se asocia a este recurso.

GlassFish Server creará el destino físico automáticamente cuando sea necesario y lo suprimirá cuando se suprima el recurso de destino.

Tipo de recurso

Tipo de recurso de destino. Las opciones disponibles son javax.jms.Topic y javax.jms.Queue.

Descripción

Descripción del recurso de destino.

Estado

Estado del recurso de destino. El recurso de destino se puede activar o desactivar. Esta opción se encuentra activada por defecto.

Propiedades Adicionales

Propiedades adicionales del recurso de destino. GlassFish Server no define ninguna propiedad adicional para los recursos de destino JMS.

Destinos

(Esta opción sólo se muestra si en el dominio se ha creado algún cluster o instancia independiente).

Clusters e instancias independientes para el recurso. Mueva los destinos seleccionados a la columna Destinos Seleccionados mediante los botones Agregar y Agregar Todos. Mueva los destinos que no necesite a la columna Destinos Disponibles mediante los botones Eliminar y Eliminar Todos.
