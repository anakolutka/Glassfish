Fabriques de threads gérés

Les fabriques de threads gérés permettent aux applications de créer des threads gérés sur demande. Les threads sont démarrés et gérés par le conteneur. Le contexte du conteneur est propagé vers le thread exécutant la tâche.

Utilisez la page Fabriques de threads gérés pour configurer les ressources de fabrique des threads gérée.

Pour chaque ressource de fabrique de threads gérés, les informations suivantes sont fournies.

Nom JNDI

Nom unique identifiant la ressource.

Nom JNDI logique

Nom JNDI logique pour la ressource. Pour plus d'informations, reportez-vous à Noms JNDI logiques.

Activé

(Cette colonne est affichée si seule l'instance de serveur par défaut server existe dans le domaine.)

Coche si la ressource est activée ou X si la ressource est désactivée.

Statut

(Cette colonne apparaît si des clusters ou des instances autonomes ont été créés dans le domaine.)

Nombre de clusters et d'instances autonomes sur lesquels la ressource est ciblée et nombre de cibles sur lesquelles la ressource est activée. Par exemple, Activé sur 2 cibles sur 4 signifie que la ressource est ciblée sur quatre clusters et instances autonomes et activée sur deux de ces quatre cibles.

Informations de contexte

Contextes de conteneur propagées vers les threads.

Priorité des threads

Priorité affectée aux threads.

Description

Description de la ressource.

La table Fabriques de threads gérés contient également les options suivantes :

Nouveau

Bouton permettant de créer une ressource.

Supprimer

Bouton permettant de supprimer les ressources sélectionnées.

Activer

Bouton permettant d'activer les ressources sélectionnées.

Désactiver

Bouton permettant de désactiver les ressources sélectionnées.
