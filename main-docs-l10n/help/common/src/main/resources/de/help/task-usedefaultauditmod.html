So wird das Standardauditmodul verwendet

Das Auditmodul default protokolliert Authentifizierungs- und Autorisierungsanforderungen in der Serverlogdatei. In So werden allgemeine Logging-Einstellungen konfiguriert wird beschrieben, wie das Verzeichnis der Logdatei geändert wird.

Authentifizierungslogeinträge umfassen die folgenden Informationen:

  • Die Namen von Benutzern, die einen Authentifizierungsversuch unternommen haben

  • Die Realm, in der die Zugriffsanforderung verarbeitet wurde

  • Die angeforderte URI- oder EJB-Komponente des Webmoduls

  • Erfolg oder Fehler bei der Anforderung

Unabhängig davon, ob das Audit-Logging aktiviert oder deaktiviert ist, protokolliert GlassFish Server alle abgelehnten Authentifizierungsereignisse.

Autorisierungslogeinträge umfassen die folgenden Informationen:

  • Die Namen der authentifizierten Benutzer (wenn zutreffend)

  • Die angeforderte URI- oder EJB-Webkomponente

  • Erfolg oder Fehler bei den Anforderungen

  1. Blenden Sie in der Navigationsbaumstruktur den Knoten "Konfiguration" ein.

  2. Wählen Sie die zu konfigurierende Instanz:

    • Um eine bestimmte Instanz zu konfigurieren, blenden Sie den Konfigurationsknoten der Instanz ein. Beispiel: Für die Standardinstanz server blenden Sie den Knoten server-config ein.

    • Um die Standardeinstellungen für alle Instanzen zu konfigurieren, blenden Sie den Knoten default-config ein.

  3. Wählen Sie unter dem Konfigurationsknoten den Sicherheitsknoten.

    Die Seite "Sicherheit" wird geöffnet.

  4. Wählen Sie aus der Dropdown-Liste "Auditmodule" das Auditmodul namens default.

  5. Klicken Sie auf "Speichern".
