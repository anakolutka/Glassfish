JMS 호스트를 생성하는 방법

  1. 탐색 트리에서 [구성] 노드를 확장합니다.

  2. 구성할 인스턴스를 선택합니다.

    • 특정 인스턴스를 구성하려면 해당 인스턴스의 구성 노드를 선택합니다. 예를 들어, 기본 인스턴스 server의 경우 server-config 노드를 선택합니다.

    • default-config 복사본을 사용하는 향후 인스턴스에 대해 기본 설정을 구성하려면 default-config 노드를 선택합니다.

  3. [구성] 노드에서 Java Message Service 노드를 확장합니다.

  4. Java Message Service 노드에서 [JMS 호스트] 노드를 선택합니다.

    [JMS 호스트] 페이지가 열립니다.

  5. [JMS 호스트] 페이지에서 [새로 만들기]를 누릅니다.

    [새 JMS 호스트] 페이지가 나타납니다.

  6. [이름] 필드에 호스트의 이름을 입력합니다.

    예를 들면 다음과 같습니다.

    NewJmsHost

  7. [호스트] 필드에 JMS 호스트가 실행될 시스템의 이름이나 IP 주소를 입력합니다(localhost 또는 로컬이나 원격 시스템의 이름).

  8. [포트] 필드에 JMS 서비스의 포트 번호를 입력합니다.

    사용될 JMS 서비스가 기본값이 아닌 포트에서 실행 중인 경우에만 이 필드를 변경합니다. 기본 포트는 7676입니다.

  9. [관리 사용자 이름] 및 [관리 비밀번호] 필드에 Message Queue 브로커 사용자 이름 및 비밀번호를 입력합니다.

    이는 GlassFish Server 사용자 이름 및 비밀번호와는 다릅니다. 이 필드는 메시지 대기열 중개자 값이 as-install-parent/mq/bin/imqusermgr 명령을 통해 변경된 경우에만 편집합니다. 기본값은 admin 및 admin입니다.

  10. [확인]을 누릅니다.
