So konfigurieren Sie die Sicherheitseinstellungen

Mit der Administration Console können verschiedene systemweite Sicherheitseinstellungen festgelegt werden.

  1. Erweitern Sie in der Navigationsstruktur den Konfigurationsknoten.
  2. Wählen Sie unter dem Konfigurationsknoten den Sicherheitsknoten aus.
  3. Aktivieren Sie das Kontrollkästchen "Sicherheitsmanager aktiviert", um den Sicherheitsmanager zu aktivieren.

    Bei Aktivierung wird eine JVM-Option, -Djava.security.manager, zu den JVM-Einstellungen von Enterprise Server hinzugefügt. Siehe So konfigurieren Sie die JVM-Optionen. Um diese Änderung zu übernehmen, muss der Server neu gestartet werden.

    Stellen Sie sicher, dass Sie über die erforderlichen Berechtigungen für alle Anwendungen verfügen. Der Sicherheitsmanager kann zum Verbessern der Leistung deaktiviert werden. Diese Option ist standardmäßig deaktiviert.

  4. Aktivieren Sie das Kontrollkästchen "Überwachungsprotokollierung aktiviert", um die Überwachungsprotokollierung einzuschalten.

    Ist diese Option aktiviert, lädt der Server alle in der Liste "Überwachungsmodule" ausgewählten Überwachungsmodule und führt sie aus. Ist die Option deaktiviert, greift der Server nicht auf Überwachungsmodule zu. Die Option errmon ist standardmäßig deaktiviert.

  5. Wählen Sie aus der Dropdown-Liste "Standardbereich" einen aktiven Bereich aus, den der Server zur Authentifizierung verwenden wird.

    Anwendungen verwenden diesen Bereich, sofern in ihrer Bereitstellungsbeschreibung kein anderer Bereich angegeben ist. Alle konfigurierten Bereiche werden in der Liste angezeigt. Der Standardwert ist "File".

  6. (Optional) Geben Sie Standard-Principal-Informationen an.
    1. Geben Sie in das Feld "Standard-Principal" den Standardbenutzernamen ein.

      Der Server verwendet diesen Benutzernamen, wenn kein anderer Principal angegeben wird.

    2. Geben Sie in das Feld "Standard-Principal-Passwort" das Passwort des im Feld "Standard-Principal" angegebenen Standard-Principals ein.
  7. Wählen Sie in der Liste "Überwachungsmodule" die Module aus, die vom Überwachungsuntersystem verwendet werden sollen.

    Hier aufgeführte Module müssen bereits konfiguriert sein. Wenn die Überwachungsprotokollierung aktiviert ist, müssen für diese Einstellung Module aufgeführt sein. Standardmäßig verwendet der Server das Überwachungsmodul default. Weitere Informationen zum Erstellen neuer Überwachungsmodule finden Sie unter So erstellen Sie ein Überwachungsmodul.

  8. Aktivieren Sie die Option zur Anwendung einer standardmäßigen Principal/Rolle-Zuordnung auf Anwendungen, die nicht über eine anwendungsspezifische Zuordnung verfügen.
  9. Die zugeordnete Principal-Klasse ermöglicht Ihnen eine Anpassung der java.security.Die Principal-Implementierungsklasse, die in der standardmäßigen Principal/Rolle-Zuordnung verwendet wird.
  10. Ändern Sie auf der Seite "Sicherheit" die Werte nach Bedarf.
  11. Fügen Sie bei Bedarf weitere Eigenschaften zur Übertragung an die Java Virtual Machine (JVM) hinzu.

    Zulässige Eigenschaften hängen vom Bereichstyp ab, der im Feld "Standardbereich" ausgewählt wurde.

  12. Wählen Sie "Speichern".
