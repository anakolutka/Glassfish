Asignaciones de seguridad del pool de conexiones del conector

Utilice la página Asignaciones de seguridad del pool de conexiones del conector para gestionar las asignaciones de seguridad relativas a pools de conexiones del conector.

Las asignaciones de seguridad para los pools de conexiones del conector permiten crear una asignación entre los usuarios y grupos de usuarios de Java EE y los de EIS. Una asignación de seguridad asigna un principal recibido durante el servlet o la autenticación EJB a las credenciales aceptadas por el EIS. Es posible asignar varios principales de GlassFish Server al mismo principal de backend.

Una asignación de seguridad de un pool de conexiones es diferente de una asignación de seguridad de trabajo de conectores, ya que éste asigna un principal asociado a una instancia de trabajo entrante a un principal de un dominio de seguridad del dominio de GlassFish Server. Consulte Asignaciones de Seguridad de Trabajo para obtener información sobre las asignaciones de seguridad del trabajo de los conectores.

Para cada asignación de seguridad, se proporciona la siguiente información.

Nombre

Nombre de la asignación de seguridad.

La tabla Asignaciones de seguridad también contiene las siguientes opciones.

Nuevo

Botón para crear una asignación de seguridad nueva.

Suprimir

Botón para suprimir una o más asignaciones de seguridad seleccionadas.
