To Configure Cluster Event Properties

Follow this procedure in the New Management Rule (Step 2 of 2) page.

  1. Select an event from the drop-down list. The following events are available:

    • Fail: triggers the action if the server or cluster fails.

    • Start: triggers the action if the server or cluster starts.

    • Stop: triggers the action if the server or cluster stops

  2. Select the server or cluster to which this rule applies. An asterisk (*) indicates all servers, including standalone instances and clustered instances.
  3. Enter an event description.
  4. Specify the action you want performed when the monitored data meets the condition specified in the event type.
