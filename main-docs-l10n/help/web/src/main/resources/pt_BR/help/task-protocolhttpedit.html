Para Editar Definições de HTTP de um Protocolo

  1. Na árvore de navegação, expanda o nó Configuração.

  2. No nó Configuração, expanda o nó Configuração da Rede.

  3. No nó Configuração de Rede, selecione o nó Protocolos.

    É aberta a página Protocolos.

  4. Na página Protocolos, clique no nome do protocolo a ser editado.

    A página Editar Protocolo será aberta.

  5. Na página Editar Protocolo, clique na guia HTTP.

    A página HTTP será aberta.

  6. No campo Nome do Servidor, digite o nome do host a ser usado nas URLs que o servidor envia para o cliente.

    Esse nome será o nome alias se o servidor usar um alias. Se o servidor não usar alias, deixe o campo em branco.

  7. Na lista drop-down Servidor Virtual Default, selecione o servidor virtual a ser associado ao protocolo.

    Use a página Servidores Virtuais para definir servidores virtuais.

  8. No campo Porta de Redirecionamento, digite o valor da porta a ser usado para redirecionar uma solicitação para outra porta.

    O GlassFish Server redirecionará automaticamente a solicitação se estas duas condições existirem:

    • O protocolo suporta solicitações não-SSL.

    • Uma solicitação é recebida, e a correspondente restrição de segurança exige transporte SSL.

    Se uma porta de redirecionamento não estiver especificada, o GlassFish Server usará o número de porta especificado na solicitação original.

  9. No campo Máximo de Conexões, digite o número máximo de conexões persistentes a ser mantido no modo Keep-Alive.

    O subsistema Keep-Alive consulta periodicamente as conexões ociosas.

  10. No campo Time-out, digite o máximo de tempo em segundos que uma conexão pode ser considerada inativa e mantida no estado keep-alive.

    Um valor de 0 ou menos significa que as conexões keep-alive são mantidas abertas indefinidamente. O valor default é 30 segundos.

  11. Marque a caixa de seleção Time-out para Upload Ativado para que a conexão para um servlet que lê lentamente os bytes seja fechada após o limite de Time-out para Upload de Conexão é alcançado.

    Se essa opção for desativada, as conexões de servlet não são submetidas a time-out. Por default, essa opção está desativada.

  12. No campo Time-out para Upload de Conexão, digite o time-out para uploads em milissegundos.

    Esse campo só se aplicará se a caixa de seleção Time-out para Upload Desativado estiver marcada. O valor default é 300.000 milissegundos.

  13. No campo Time-out para Solicitação, digite o número de segundos antes que uma solicitação seja submetida a time-out.

    Se a solicitação não for processada antes que o valor do time-out seja alcançado, a solicitação será ignorada. O valor default é 30 segundos.

  14. No campo Tamanho do Buffer para Envio, digite o tamanho em bytes do buffer para envio.

    O valor default é 8.192 bytes.

  15. No campo Tamanho do Buffer de Cabeçalho, digite o tamanho em bytes do buffer usado pelos threads de processamento de solicitação para ler os dados da solicitação.

    O valor default é 8.192 bytes.

  16. No campo Tamanho Máximo de Postagem, digite o tamanho máximo em bytes das ações POST.

    O valor default é 2.097.152 bytes.

  17. No campo Codificação URI, digite o nome do conjunto de caracteres usado para decodificar os URIs de solicitação recebidos.

    O valor precisa ser um nome válido de um conjunto de caracteres IANA. O valor default é UTF-8.

  18. No campo Versão, digite a versão do protocolo HTTP usado.

    O valor default é HTTP/1.1.

  19. Na lista drop-down Compactação, selecione o uso da compactação HTTP/1.1 GZIP para reduzir a largura de banda no servidor.

    As opções disponíveis são:

    on

    Compacta os dados.

    off

    Desativa a compactação.

    force

    Força a compactação dos dados em todos os casos.

    O valor default off.

  20. No campo Tipos MIME Compactáveis, digite uma lista separada por vírgulas dos tipos MIME para os quais a compactação HTTP é usada.

    O valor default é text/html,text/xml,text/plain.

  21. No campo Tamanho Mínimo de Compactação, digite o tamanho mínimo de um arquivo quando a compactação é aplicada.

    Esse valor deve ser definido se a Compactação estiver definida como on ou force. O valor default é 2.048 bytes.

  22. No campo Agentes de Não Compactação do Usuário, digite uma lista separada por vírgulas de expressões regulares que coincidem com agentes do usuário ou clientes HTTP, para os quais a compactação não deve ser usada.

    Por default esse valor é uma string vazia.

  23. No campo Agentes de Usuário Restritos, digite uma lista de agentes do usuário Restritos no qual a compactação HTTP é aplicada.

    Se nenhum agente do usuário for especificado, a compactação HTTP será aplicada em todos os agentes do usuário. Por default, nenhum agente do usuário é especificado.

  24. No campo Tipo de Resposta Default, digite uma string que especifica o tipo de resposta default.

    O formato é uma string delimitada por pontos e vírgulas, consistindo no tipo de conteúdo, codificação, idioma e conjunto de caracteres. O valor default é text/html; charset=iso-8859-1.

  25. No campo Tipo de Resposta Forçada, digite uma string que especifica o tipo de solicitação usado, caso nenhum mapeamento MIME esteja disponível que coincida com a extensão do arquivo.

    O formato é uma string delimitada por pontos e vírgulas, consistindo no tipo de conteúdo, codificação, idioma e conjunto de caracteres. O valor default é text/html; charset=iso-8859-1.

  26. No campo Adaptador, digite o nome da classe do adaptador de recursos estáticos.

    O valor default é com.sun.grizzly.tcp.StaticResourcesAdapter.

  27. Marque a caixa de seleção Suporte a Comet Ativado para ativar o suporte a Comet para o protocolo.

    Por default, essa opção está desativada.

  28. Marque a caixa de seleção Pesquisa de DNS Ativada para ativar a pesquisa de DNS.

    Por default, essa opção está desativada.

  29. Marque a caixa de seleção Suporte RCM Ativado para ativar o Gerenciamento de Configuração de Recursos (RCM).

    Por default, essa opção está desativada.

  30. Marque a caixa de seleção Rastreamento Ativado para ativar a operação TRACE.

    Se essa opção for desativada, o GlassFish Server é menos suscetível ao ataques de scripts de site cruzado. Por default, a opção está ativada.

  31. Marque a caixa de seleção Passagem de Autorização para indicar que o listener de rede que usa este protocolo recebe tráfego de um proxy de servidor em que a conexão SSL proveniente do cliente termina no servidor proxy.

    Por default, essa opção está desativada.

  32. Marque a caixa de seleção Agrupamento Ativada para ativar o agrupamento de resposta HTTP.

    Por default, a opção está ativada.

  33. Marque a caixa de seleção XPowered por Ativado para indicar ao GlassFish Server para usar os cabeçalhos X-Powered-By das especificações Java Servlet 3.0 e do Java Server Pages 2.0.

    Por default, a opção está ativada.

  34. Clique em Salvar.

    Dica:

    Para voltar às definições default, clique no botão Carregar Defaults e depois em Salvar.
