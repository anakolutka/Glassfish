Modifica risorsa JDBC

Utilizzare la pagina Modifica risorsa JDBC per modificare una risorsa JDBC esistente.

La pagina Modifica risorsa JDBC contiene le opzioni riportate di seguito.

Carica impostazioni predefinite

Pulsante per ripristinare i valori predefiniti delle impostazioni che dispongono di valori predefiniti. Le impostazioni che non dispongono di valori predefiniti non vengono modificate.

Nome JNDI

Il nome JNDI univoco che identifica la risorsa. Per convenzione, il nome inizia con un indicatore del tipo di risorsa e una barra, ad esempio concurrent/, jdbc/ o jms/. Il campo Nome JNDI è di sola lettura. È possibile specificare il nome JNDI solo quando si crea una nuova risorsa

Nome JNDI logico

Il nome JNDI logico per la risorsa. Si tratta di un nome di standard Java EE mediante il quale è possibile accedere alla risorsa. Il campo del nome JNDI logico è di sola lettura. Per ulteriori informazioni, vedere Nomi JNDI logici.

Nome pool

Il connection pool associato alla risorsa JDBC.

Ordine di distribuzione

L'ordine di distribuzione della risorsa JDBC.

Descrizione

Una breve descrizione della risorsa JDBC.

Stato

Indica se la risorsa è disponibile in fase di esecuzione.

Se nel dominio esiste solo l'istanza server predefinita server, è possibile abilitare o disabilitare la disponibilità in fase di esecuzione della risorsa. La disponibilità in fase di esecuzione è abilitata per impostazione predefinita.

Se nel dominio sono stati creati cluster o istanze standalone, utilizzare la scheda Destinazione per controllare la disponibilità in fase di esecuzione della risorsa nelle relative destinazioni.

Proprietà aggiuntive

Proprietà aggiuntive per la risorsa JDBC. GlassFish Server non definisce alcuna proprietà aggiuntiva per le risorse JDBC.
