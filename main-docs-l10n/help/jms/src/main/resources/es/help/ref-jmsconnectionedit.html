Editar fábrica de conexiones de JMS

Utilice la página Editar Fábrica de Conexiones JMS para modificar la configuración de una fábrica de conexiones JMS.

La página Editar Fábrica de Conexiones JMS contiene las siguientes opciones.

Cargar Valores por Defecto

Botón para restaurar configuración que tiene valores por defecto a sus valores por defecto. Las configuraciones que no tienen valores por defecto no se modifican.

Nombre JNDI

Nombre de JNDI único que identifica el recurso. Por convención, el nombre empieza por un indicador de tipo de recurso y una barra invertida, por ejemplo, concurrent/, jdbc/ o jms/. El nombre JNDI es un campo de sólo lectura. Sólo puede especificar el nombre JNDI al crear un nuevo recurso

Nombre de JNDI lógico

Nombre de JNDI lógico para el recurso. Se trata de un nombre estándar de Java EE a través del que se puede acceder al recurso. El nombre de JNDI lógico es un campo de sólo lectura. Consulte Nombres de JNDI lógicos para obtener más información.

Tipo de recurso

Tipo de la fábrica de conexiones. El tipo de recurso es un campo de sólo lectura. Sólo se puede especificar el tipo de recurso al crear una nueva fábrica de conexiones JMS.

Descripción

Descripción de la fábrica de conexiones.

Estado

Indica si el recurso está disponible en el tiempo de ejecución.

Si sólo existe la instancia de servidor por defecto, server en el dominio, la disponibilidad en tiempo de ejecución del recurso puede estar activada o desactivada. La disponibilidad en tiempo de ejecución está activa por defecto.

Si se ha creado algún cluster o alguna instancia independiente en el dominio, utilice el separador Destino para controlar la disponibilidad del recurso en tiempo de ejecución en sus destinos.

Tamaño de pool inicial y mínimo:

El número mínimo de conexiones del pool para la fábrica de conexiones. El valor por defecto es 8.

Tamaño de Pool Máximo

Número máximo de conexiones del pool. El valor por defecto es 32.

Cantidad de Cambio de Tamaño del Pool

Número de conexiones que se pueden eliminar cuando se agota el timeout de inactividad del pool. El valor por defecto es 2.

Timeout de Inactividad

Tiempo máximo, en segundos, que una conexión puede permanecer inactiva en el pool. El valor por defecto es 300.

Tiempo de Espera Máximo

Cantidad de tiempo que la aplicación que solicita una conexión puede esperar para obtener un mensaje de timeout de conexión. El valor por defecto es 60000.

Ante cualquier fallo

Si se activa la casilla de control Cerrar todas las conexiones, GlassFish Server cerrará todas las conexiones del pool y las restablecerá aunque tan sólo falle una. Esta opción se activada por defecto.

Soporte de Transacción

El valor puede ser cualquiera de los siguientes:

XATransaction

El recurso se puede utilizar para transacciones que suponen el uso de más de un recurso dentro de un ámbito de transacción. Éste es el valor por defecto de una fábrica de conexiones JMS.

Por ejemplo, en las transacciones puede participar este recurso más un recurso JDBC, un recurso de conector u otro recurso de fábrica de conexiones JMS. Este valor ofrece la máxima flexibilidad. Un recurso que se configure como XATransaction participará en operaciones de confirmación de dos fases.

LocalTransaction

Este recurso se puede utilizar bien para transacciones que impliquen solamente un recurso dentro del ámbito de la transacción o bien como el último agente en una transacción distribuida que involucre a más de un recurso XA.

Este valor ofrece un rendimiento mucho más óptimo. Un recurso que se configura como LocalTransaction no se utilizará en operaciones de confirmación de dos fases.

NoTransaction

Este recurso no puede participar nunca en transacciones. Esta configuración es de uso limitado en aplicaciones JMS.

Validación de la conexión

Si la casilla de control Necesaria está activada, las conexiones se validarán antes de proporcionárselas a la aplicación. Si falla la validación de un recurso, éste se destruye, y se crea y se devuelve un recurso nuevo. Esta opción se activada por defecto.

Propiedades Adicionales

Son las propiedades adicionales de la fábrica de conexiones.

Para obtener más información sobre las propiedades disponibles, consulte Propiedades específicas de fábricas de conexiones JMS.
