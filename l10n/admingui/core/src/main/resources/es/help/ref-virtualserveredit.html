Editar servidor virtual

ID

Identificador de servidor virtual visible internamente. No se muestra a los clientes HTTP. Los nombres de host que se muestran a los clientes HTTP se deben especificar en el campo Hosts.

Hosts

Nombre o nombres de host para el equipo en el que se ejecuta el servidor. Utilice nombres de host reales o virtuales que estén registrados en el servidor DNS de la red (y en un sistema UNIX, en el archivo /etc/hosts ).

Estado

Seleccione el estado deseado para el servidor. El valor predetermiando es ACTIVADO.

Escuchas HTTP

Este campo se rellena automáticamente cuando se crea una escucha HTTP y se asocia a un servidor.

El uso de este campo precisa que especifique una escucha HTTP existente. Sin embargo, no debe especificar una escucha que utilice otro servidor virtual; de lo contrario, se mostrará un error en el registro del servidor cuando reinicie el servidor. Dado que es preciso asociar una escucha a un servidor virtual cuando se crea, todas las escuchas existentes son utilizadas por otro servidor virtual.

Módulo web predeterminado

Seleccione un módulo web de la lista desplegable. Seleccione el módulo web implementado (si lo hay) que debe responder a todas las solicitudes que no pueden asignarse a otros módulos web implementados en el servidor virtual. Si no se especifica un módulo web predeterminado, se utiliza un módulo web con una raíz de contexto vacía. Si no hay ningún módulo web con una raíz de contexto vacía, se crea y utiliza un módulo web predeterminado del sistema.

Archivo de registro

Deje el campo Archivo de registro vacío para enviar mensajes al registro del servidor predeterminado. De lo contrario, introduzca el nombre de ruta del archivo en el que se guardarán los mensajes de registro de este servidor virtual.

Docroot

La ruta absoluta al directorio raíz de documentos del servidor. El valor predeterminado es domain-dir/docroot.

SSO

Si está habilitado, los usuarios deben autenticarse de forma independiente en todas las aplicaciones del servidor virtual. El valor predeterminado es Desactivado.

Registro de acceso

Especifique si el registro de acceso se ha heredado del servicio HTTP, habilitado o deshabilitado.

Directorio

La ruta de directorio absoluta a los registros de acceso del servidor. El valor predeterminado es domain-dir/logs/access.

Tamaño de búfer

Un valor igual o menor que cero desactiva el búfer. Si deja vacío el campo, el tamaño del búfer del registro de acceso se heredará del servicio HTTP.

Intervalo de escritura

Número de segundos antes de que se actualice el registro de acceso. El valor 0 deshabilita el almacenamiento en el búfer.

Propiedades adicionales

Propiedades adicionales para el servidor virtual. Están disponibles las propiedades siguientes:

sso-max-inactive-seconds

Especifica el número de segundos después de los cuales el registro de inicio de sesión único de un usuario cumple con los requisitos necesarios para depurarse si no se detecta actividad en el cliente. Dado que el inicio de sesión único se aplica en varias aplicaciones en el mismo servidor virtual, el acceso a cualquiera de las aplicaciones mantiene activo el registro de inicio de sesión único.

El valor predeterminado es 300 segundos (5 minutos). Los valores más altos proporcionan una persistencia de inicio de sesión único más larga para los usuarios a costa de un mayor consumo de memoria en el servidor.

sso-reap-interval-seconds

Especifica el número de segundos entre depuraciones de registros de inicio de sesión único caducados.

El valor predeterminado es 60.

allowLinking

Si está propiedad es true, se atenderán los recursos que son vínculos simbólicos para todas las aplicaciones web implementadas en este servidor virtual. Las aplicaciones Web individuales pueden reemplazar esta configuración utilizando la propiedad allowLinking de sun-web-app en el archivo sun-web.xml:

<sun-web-app>
   <property name="allowLinking" 
      value="{true|false}"/>
</sun-web-app>

El valor predeterminado es false.
