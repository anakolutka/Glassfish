Konfiguration der Batchlaufzeit

GlassFish Server bietet eine Batchlaufzeit für die Planung und Ausführung von Batchjobs. Batchjobs haben in der Regel eine lange Ausführungszeit und sind auf die Massenverarbeitung von Tasks ausgerichtet, die eine Reihe von Schritten enthalten. Batchanwendungen leiten Jobs an die Batchlaufzeit weiter und liefern Anweisungen, wie und wann die Schritte ausgeführt werden sollen. Die Batchlaufzeit verarbeitet die Schritte und speichert Informationen zu den Jobs in einem Job-Repository. In GlassFish Server ist das Job-Repository eine Datenbank.

Auf der Seite "Konfiguration der Batchlaufzeit" können Sie die Konfiguration der Batchlaufzeit anzeigen und ändern. Die Batchlaufzeit verwendet einen Managed Executor-Service sowie eine Datenquelle zur Ausführung von Batchjobs. Der Managed Executor-Service stellt Threads für Jobs bereit, und die Datenquelle speichert Jobinformationen. Konfigurationsdaten der Batchlaufzeit werden im Element config in der Datei domain.xml gespeichert.

Die Seite "Konfiguration der Batchlaufzeit" enthält die folgenden Informationen.

Instanzname
Clustername

Der Name des GlassFish-Clusters oder der GlassFish-Serverinstanz, auf das bzw. auf die sich die Konfiguration bezieht. Dies ist ein schreibgeschütztes Feld.

Lookup-Name des Executor-Service

Der JNDI-Lookup-Name des Managed Executor-Service, der Threads für Batchjobs bereitstellt. Der Managed Executor-Standardservice lautet concurrent/__defaultManagedExecutorService.

Der Managed Executor-Service kann nach Weiterleitung eines Batchjobs an die Batchlaufzeit geändert werden, ohne die Ausführung des Jobs zu beeinträchtigen.

Lookup-Name der Datenquelle

Der JNDI-Lookup-Name der Datenquelle, in der Informationen zu aktuellen und vergangenen Batchjobs gespeichert sind. Die Standarddatenquelle lautet jdbc/__TimerPool.

Sobald der erste Batchjob an die Batchlaufzeit zur Ausführung weitergeleitet wurde, dürfen Sie die Datenquelle nicht mehr ändern. Wenn die Datenquelle geändert werden muss, stoppen Sie die Domain, und starten Sie sie anschließend neu, und nehmen Sie dann die notwendigen Änderungen vor, bevor Sie Jobs starten bzw. neu starten. Sobald die Datenquelle jedoch geändert wurde, kann auf Informationen in der vorherigen Datenquelle nicht mehr zugegriffen werden.
