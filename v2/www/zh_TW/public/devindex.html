瞭解規定

開發應用程式伺服器是一個大型工程。下面連結可協助我們規劃這種大型專案的開發作業。

  • GlassFish 專案劃分為多個模組
  • 請參閱隨著各升級版本更新的 java 文件
  • 與郵件收信人清單上的其他 GlassFish 開發人員討論實作詳細資訊。 請務必登入以查看所有可用郵件收信人清單。
  • 查看模組所有者清單
  • 查看屬於 GlassFish 一部分的相關技術

建立

GlassFish 使用 maven 和 ant 建立基礎架構。目前我們正在尋找開發人員協助改進基礎架構的建立，以減少對 ant 的依賴。

  • 快速入門指南中提供了建立伺服器所需步驟的簡短清單
  • 完整說明中詳細介紹了如何建立 GlassFish

測試

GlassFish 目前提供數種不同測試。其中 QuickLook 測試執行時間較短，並涵蓋應用程式伺服器中廣泛的功能集。還可對不同的模組進行開發人員測試。每個模組都包含有關執行這些測試的更多資訊。

如需更多資訊，請參閱 GlassFish 品質入口網頁。

  • 下載每日更新版本或升級版本
  • 執行 QuickLook 測試
  • 加入 CATfish 計畫
  • 提交錯誤

提供程式碼

您可透過以下方式為 GlassFish 專案做出貢獻：錯誤修復、增強功能、新增功能，乃至整個模組。

  • 所有貢獻者必須簽署並繳回貢獻者協議
  • 請參閱管理政策以瞭解有關各角色和責任的更多資訊
  • 所有登入工作區的程式碼必須遵循程式碼寫作慣例
  • 如果您目前尚不具備開發人員角色，請參閱提供修補程式的程序
  • 開發人員在更新工作區時需要遵循確定程序

提供範例

我們歡迎開發人員提供可展示 GlassFish 或 Java EE 5 技術某項功能或某一方面的高品質範例。加入 GlassFish 範例專案，您就能看到自己的範例會有多大貢獻。

請查看在特定領域邀請他人提供範例網頁。此外也請查看 Java Blueprints 專案，以瞭解他人對最佳實務應用程式所做的貢獻。

所有貢獻者必須簽署並繳回貢獻者協議
