Proprietà cluster

Nella pagina Proprietà cluster viene visualizzata la lista delle proprietà impostate per i cluster selezionati. Queste proprietà aggiungono informazioni di configurazione opzionali sul cluster.

Il campo Nome cluster è un campo di sola lettura in cui viene visualizzato il nome del cluster selezionato.

Per ogni proprietà vengono visualizzate le informazioni riportate di seguito.

Nome

Il nome della proprietà.

Valore

Il valore della proprietà impostato per l'istanza selezionata.

Descrizione

Descrizione di testo che fornisce ulteriori informazioni sulla proprietà.

La tabella Proprietà aggiuntive contiene inoltre le opzioni riportate di seguito.

Aggiungi proprietà

Pulsante che consente di aggiungere una proprietà. Se si fa clic su questo pulsante, viene aggiunta una riga alla tabella Proprietà aggiuntive.

Elimina proprietà

Pulsante che consente di eliminare una o più proprietà selezionate. L'eliminazione comporta il ripristino del valore predefinito della proprietà. Se il valore predefinito non è stato impostato, la proprietà rimane non definita.

GlassFish Server definisce le proprietà di cluster riportate di seguito.

GMS_DISCOVERY_URI_LIST

Le posizioni delle istanze di GlassFish Server nel cluster da utilizzare per trovare il cluster. Questa proprietà è necessaria solo se il servizio GMS (Group Management Service) non usa la modalità multicast per la trasmissione dei messaggi.

Di seguito vengono forniti i valori validi per questa proprietà.

  • Lista separata da virgole di URI (Uniform Resource Identifier). Ogni URI deve consentire di individuare un'istanza di GlassFish Server o il server DAS. Questo formato è obbligatorio se più istanze di GlassFish Server sono in esecuzione sullo stesso host.

    Per ogni URI della lista viene utilizzato il formato seguente:

    schema://nome-host o indirizzo-IP:porta

    • schema rappresenta lo schema dell'URI, ovvero tcp.

    • nome-host o indirizzo-IP rappresenta il nome o l'indirizzo IP dell'host su cui è in esecuzione l'istanza.

    • porta rappresenta il numero della porta su cui l'istanza ascolta i messaggi provenienti dal servizio GMS. La proprietà di sistema GMS_LISTENER_PORT-nome-cluster deve essere impostata per l'istanza. Per informazioni su come impostare questa proprietà di sistema per un'istanza, vedere "Discovering a Cluster When Multicast Transport Is Unavailable" in Oracle GlassFish Server High Availability Administration Guide.

  • Lista separata da virgole di indirizzi IP o nomi host sui quali sono in esecuzione il server DAS o le istanze. La lista può contenere una combinazione di indirizzi IP e nomi host. Questo formato può essere utilizzato soltanto se una sola istanza in cluster è in esecuzione su ogni host. Il valore della proprietà GMS_LISTENER_PORT deve essere univoco per ogni cluster di un dominio.

  • Parola chiave generate. Questo formato può essere utilizzato soltanto se una sola istanza in un cluster è in esecuzione su ogni host e se il server DAS è in esecuzione su un host distinto. Più istanze nello stesso host non possono essere membri dello stesso cluster. Il valore della proprietà GMS_LISTENER_PORT deve essere univoco per ogni cluster di un dominio.

GMS_LISTENER_PORT

Il numero della porta utilizzata dal cluster per l'ascolto dei messaggi trasmessi dal servizio GMS.

Il valore predefinito è un riferimento alla proprietà di sistema GMS_LISTENER_PORT-nome-cluster. Per impostazione predefinita, questa proprietà di sistema non è impostata. In questa situazione, il servizio GMS seleziona una porta libera nell'intervallo definito dalle proprietà GMS_TCPSTARTPORT e GMS_TCPENDPORT. Per impostazione predefinita, l'intervallo è compreso tra 9090 e 9200. Il funzionamento predefinito è sufficiente nella maggior parte dei casi.

Se tuttavia il servizio GMS non utilizza la modalità multicast per la trasmissione dei messaggi, questa proprietà deve specificare un numero di porta valido per tutte le istanze di GlassFish Server del cluster. Per soddisfare questo requisito utilizzando il valore predefinito, usare una proprietà di sistema per impostare il numero di porta per ogni singola istanza.

Ad esempio, utilizzare la pagina Proprietà di sistema del cluster per creare la proprietà di sistema GMS_LISTENER_PORT-nome-cluster per il server DAS. In seguito, per ogni istanza del cluster, impostare la proprietà di sistema GMS_LISTENER_PORT-nome-cluster sul numero della porta utilizzata dall'istanza per l'ascolto dei messaggi del servizio GMS. Il valore predefinito della proprietà GMS_LISTENER_PORT per il cluster fa riferimento a questa proprietà di sistema.

GMS_LOOPBACK

Specifica se un'istanza può ricevere da se stessa i messaggi a livello di applicazione trasmessi dall'istanza al cluster.

I possibili valori sono:

false

L'istanza non può ricevere messaggi provenienti da se stessa (impostazione predefinita).

true

L'istanza può ricevere messaggi da se stessa. Utilizzare questa impostazione per sottoporre a test un'istanza quando è l'unica istanza in un cluster.

GMS_MULTICAST_TIME_TO_LIVE

Il numero massimo di iterazioni o trasmissioni che possono verificarsi per i tipi di eventi seguenti prima che un messaggio multicast venga ignorato:

  • Individuazione gruppo

  • Heartbeat membro

  • Modifiche appartenenza

Per soddisfare la configurazione della rete in cui vengono distribuite le istanze DAS e le istanze in cluster, impostare il valore minimo possibile. Per determinare il valore minimo possibile per il sistema in uso, utilizzare il comando secondario validate-multicast(1).

Se si imposta il valore 0, i messaggi multicast non lasciano mai l'host dal quale vengono trasmessi.

Il valore 1 potrebbe invece impedire la trasmissione dei messaggi tra host della stessa sottorete connessi tramite switch o router.

L'impostazione predefinita 4 garantisce che i messaggi vengano trasmessi correttamente a tutti i membri del cluster nelle reti in cui gli host sono connessi tramite switch o router.

GMS_TCPENDPORT

Il numero di porta più alto nell'intervallo all'interno del quale il servizio GMS seleziona una porta libera qualora la proprietà di sistema GMS_LISTENER_PORT-nome-cluster non sia impostata. L'impostazione predefinita è 9200.

GMS_TCPSTARTPORT

Il numero di porta più basso nell'intervallo all'interno del quale il servizio GMS seleziona una porta libera qualora la proprietà di sistema GMS_LISTENER_PORT-nome-cluster non sia impostata. L'impostazione predefinita è 9090.
