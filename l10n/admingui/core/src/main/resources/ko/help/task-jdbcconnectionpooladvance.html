JDBC 연결 풀 고급 속성을 편집하는 방법

  1. 탐색 트리에서 자원 노드를 확장합니다.
  2. 자원 노드 아래에서 연결 풀 노드를 선택합니다.

    연결 풀 페이지가 열립니다.

  3. 연결 풀 페이지에서 편집할 풀의 이름을 누릅니다.

    연결 풀 편집 페이지가 열립니다.

  4. 연결 풀 편집 페이지에서 고급 탭을 누릅니다.

    연결 풀 고급 속성 편집 페이지가 열립니다.

    주 – 이름 필드는 읽기 전용 필드입니다. 새 JDBC 연결 풀을 만들 때 연결 풀 이름만 지정할 수 있습니다.

  5. 명령문 시간 초과 필드에 비정상적으로 길게 실행되는 쿼리를 종료하기 전까지의 시간(초)을 입력합니다.

    Enterprise Server에서는 명령문을 만들 때 QueryTimeout을 설정합니다. 이 옵션을 사용하지 않으려면 값을 -1로 설정합니다. 기본값은 -1입니다.

  6. JDBC 객체 래핑 활성화를 선택하면 응용 프로그램에서는 Statement, PreparedStatement, CallableStatement, ResultSet, DatabaseMetaData에 대한 JDBC 객체를 래핑합니다.

    기본값은 false입니다.

  7. 최대 1회 검증 필드에서 연결이 최대 1회 검증되기까지의 시간(초)을 입력합니다.

    연결의 검증 요청 수를 줄이는 데 도움이 됩니다. 연결 검증을 사용하지 않으려면 값을 0으로 설정합니다. 기본값은 0입니다.

  8. 누수 시간 초과 필드에서 연결 풀의 연결 누수를 추적할 시간(초)을 식별합니다.

    연결 누수 추적을 사용하지 않으려면 값을 0으로 설정합니다. 기본값은 0입니다.

  9. 누수 연결 추적이 완료된 후 풀에 대해 누수된 연결을 복구하려면 누수 재생 활성화 옵션을 선택합니다.
  10. 만들기 재시도 횟수 필드에서 새 연결 만들기에 실패할 경우 재시도할 횟수를 입력합니다.

    연결 다시 만들기 위한 횟수를 지정하지 않으려면 값을 0으로 설정합니다. 기본값은 0입니다.

  11. 재시도 간격에서 연결을 만들기 위한 두 횟수 사이 간격(초)을 입력합니다.

    기본값은 10입니다. 이 속성은 만들기 재시도 횟수 값이 0보다 클 경우에만 사용됩니다.

  12. 작업 수행 시 연결을 느리게 연결하려면 느린 연결 활성화 옵션을 선택합니다.

    트랜잭션이 완료되고 구성 요소 메소드가 종료되면 연관 해제되어 물리적 연결의 재사용을 돕습니다. 이 옵션은 기본적으로 비활성화됩니다.

  13. 실제 메소드에서 사용할 경우에만 트랜잭션에 자원을 등록하려면 지연 연결 참여 사용 가능 옵션을 선택합니다.
  14. 연결 및 스레드 사이 연결을 만들려면 스레드와 연결 사용 가능 옵션을 선택합니다.

    다른 연결에 대해 동일한 스레드가 필요한 경우 해당 스레드와 이미 연결된 연결을 재사용할 수 있기 때문에 풀에서 다른 연결에 대한 오버헤드가 발생하지 않습니다.

    이 옵션은 기본적으로 비활성화됩니다.

  15. 풀에 대해 일치하는 연결을 활성화하려면 연결 일치 사용 가능 옵션을 선택합니다.

    이 옵션은 풀의 연결이 항상 같은 종류이며 따라서 풀에서 선택된 연결을 자원 어댑터와 일치시킬 필요가 없음을 관리자가 알고 있을 때 사용하지 않도록 설정할 수 있습니다. 이 옵션은 기본적으로 비활성화됩니다.

  16. 최대 연결 사용 수 필드에 풀에서 연결을 다시 사용할 횟수를 입력합니다.

    연결을 지정된 횟수만큼 재사용한 다음에는 해당 연결이 닫힙니다. 예를 들어 명령문 누수를 피하려고 할 때 유용합니다. 이 속성을 사용하지 않으려면 값을 0으로 설정합니다. 기본값은 0입니다.
