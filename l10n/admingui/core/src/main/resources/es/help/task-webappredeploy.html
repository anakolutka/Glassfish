Cómo volver a implementar una aplicación web implementada

Al volver a implementar una aplicación web se implementa una versión nueva de la aplicación y al mismo tiempo se preserva la configuracion y otras opciones propias de la aplicación cuando se implementó inicialmente.

Antes de empezar

Asegúrese de que la aplicación web que está volviendo a implementar ya está implementada.

  1. En el árbol de navegación, expanda el nodo Aplicaciones.
  2. En el nodo Aplicaciones, seleccione el nodo Aplicaciones web.

    Se abrirá la página Aplicaciones web.

  3. En la tabla Aplicaciones web implementadas, haga clic en Volver a implementar en la fila de la aplicación que está volviendo a implementar.

    Se abrirá la página Volver a implementar aplicaciones y módulos empresariales.

  4. Si quiere que se preserven las sesiones activas de la aplicación y que después se restauren, cuando la nueva implementación se haya completado, seleccione la casilla de verificación Conservar sesiones.

    Si en alguna sesión activa se produce algún fallo en la preservación o restauración de la aplicación, no habrá disponible ninguna sesión cuando se haya completado la nueva implementación. Sin embargo, la reimplementación continuará y se registrará una advertencia.

  5. Especifique la ubicación del archivo de almacenamiento donde se encuentra la aplicación que se va a implementar.
    • Si el archivo de almacenamiento es un archivo que reside en la máquina cliente o al que se puede acceder desde la máquina cliente, seleccione la opción Archivo empaquetado que se cargará en el servidor.

      La máquina cliente es el host en el que está viendo la Consola de administración por medio de un navegador.

      Haga clic en Examinar para buscar el archivo o bien escriba la ruta completa hasta el archivo.

    • Si el archivo de almacenamiento es un archivo que reside en el equipo servidor o es una aplicación descomprimida desde un directorio desplegado, seleccione la opción Archivo empaquetado local o Directorio accesible desde el servidor de aplicaciones.

      El equipo servidor es el host que se está ejecutando en el servidor de administración del dominio (DAS).

      Escriba el nombre de la ruta completa del archivo o directorio. Implementar desde un directorio desplegado es para desarrolladores avanzados y no es recomendable para entornos de producción.

  6. Haga clic en Aceptar.
