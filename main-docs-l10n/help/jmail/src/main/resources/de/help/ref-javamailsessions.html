JavaMail-Sessions

GlassFish Server beinhaltet die JavaMail-API. JavaMail bietet Zugriff von Java-Anwendungen auf Mailserver in Ihrem Netzwerk oder im Internet, die Internet Message Access Protocol (IMAP) oder Simple Mail Transfer Protocol (SMTP) unterstützen.

Auf der Seite "JavaMail-Sessions" können Sie JavaMail-Sessions konfigurieren.

Für jede JavaMail-Session werden die folgenden Informationen zur Verfügung gestellt.

JNDI-Name

Ein eindeutiger Name, der die JavaMail-Session identifiziert.

Aktiviert

(Diese Spalte wird angezeigt, wenn nur die Standardserverinstanz server in der Domain vorhanden ist.)

Zeigt ein Häkchen an, wenn die Ressource auf dem Ziel aktiviert ist, oder ein X, wenn die Ressource deaktiviert ist.

Status

(Diese Spalte wird angezeigt, wenn in der Domain Cluster oder Standalone-Instanzen erstellt wurden.)

Die Anzahl Cluster und Standalone-Instanzen, auf die die Ressource ausgerichtet wird, und auf wie vielen dieser Ziele die Ressource aktiviert ist. Beispiel: "Auf 2 von 4 Ziel(en) aktiviert" bedeutet, dass die Ressource auf vier Cluster und Standalone-Instanzen ausgerichtet und auf zweien dieser vier Ziele aktiviert ist.

Beschreibung

Eine Beschreibung der JavaMail-Session.

Die Tabelle "Sessions" enthält darüber hinaus die folgenden Optionen.

Neu

Schaltfläche zum Erstellen einer neuen JavaMail-Session.

Löschen

Schaltfläche zum Löschen von mindestens einer ausgewählten JavaMail-Session.

Aktivieren

Schaltfläche zum Aktivieren von mindestens einer ausgewählten JavaMail-Sessions.

Deaktivieren

Schaltfläche zum Deaktivieren von mindestens einer ausgewählten JavaMail-Sessions.

Verwandte Aufgaben

  • So wird eine JavaMail-Session erstellt

  • So wird eine JavaMail-Session bearbeitet

  • So wird eine JavaMail-Session gelöscht
