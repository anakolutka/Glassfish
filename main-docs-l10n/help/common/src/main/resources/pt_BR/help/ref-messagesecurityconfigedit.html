Editar Configuração de Segurança de Mensagem

Use a página Edit Message Security Configuration para ativar a segurança de mensagens nos provedores.

Para ativar a segurança de mensagens para pontos finais de Web services implantados no GlassFish Server, você deve especificar um provedor a ser usado por default no servidor. Se você ativar a segurança de mensagens de um provedor default, também será necessário permitir que os provedores sejam usados por clientes dos Web Services implantados no GlassFish Server. Informações sobre como ativar os provedores usados pelos clientes são discutidas em Para Ativar a Segurança de Mensagem para Clientes da Aplicação.

Para ativar a segurança de mensagens de chamadas de Web Services de pontos finais implantados, você deve especificar um provedor de cliente default. Se você tiver ativado um provedor cliente default para GlassFish Server, deverá assegurar que quaisquer serviços chamados de pontos de extremidades implantados no GlassFish Server estejam compativelmente configurados para segurança de camada de mensagens.

A página Editar Configuração de Segurança de Mensagem contém as seguintes opções.

Nome da Configuração

O nome da configuração à qual as definições nesta página se aplicam. Este campo é somente para leitura.

Camada de Autenticação

A camada de mensagens na qual a autenticação é realizada. Este é um campo somente para leitura. O valor é SOAP (da configuração de segurança de mensagem default) ou HttpServlet.

Provedor Default

O provedor do servidor default, se houver. O provedor do servidor default, também chamado de provedor default, identifica o provedor do servidor chamado para uma aplicação que não identifica um provedor do servidor específico. As opções disponíveis são ServerProvider, XWS_ServerProvider, ou nenhuma. Por default, nenhum provedor é selecionado, de forma que o GlassFish Server não irá chamar um provedor de segurança de mensagens no servidor. Geralmente, o ServerProvider é selecionado como provedor default.

Provedor Cliente Default

O provedor de cliente default, se houver. O provedor do cliente default identifica o provedor de cliente chamado por uma aplicação que não identifica um provedor de cliente específico. As opções disponíveis são ClientProvider, XWS_ClientProvider, ou nenhuma. Por default, nenhum provedor é selecionado, de forma que o GlassFish Server não irá chamar um provedor de segurança de mensagens no cliente. Geralmente, nenhum é selecionado para o provedor de cliente default. Selecione o ClientProvider para ativar um provedor default e a política de proteção de mensagens a ser aplicada às chamadas dos Web Services que se originam de pontos finais de Web Services implantados no GlassFish Server.

Comandos asadmin Relacionados

Use o comando set(1) para alterar uma configuração de segurança da mensagem.

  • Para especificar o provedor do servidor default:

    asadmin set --user admin-user --port admin-port 
server-config.security-service.message-security-config.SOAP.
default_provider=ServerProvider

  • Para especificar o provedor do cliente default:

    asadmin set --user admin-user --port admin-port 
server-config.security-service.message-security-config.SOAP.
default_client_provider=ClientProvider
