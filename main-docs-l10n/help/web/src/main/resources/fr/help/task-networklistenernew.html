Création d'un processus d'écoute réseau

Avant de commencer

Lorsque vous créez un processus d'écoute réseau, vous pouvez créer simultanément un protocole que le processus utilisera ou employer un protocole que vous avez d'abord créé. En règle générale, il existe une relation 1 à 1 entre un processus d'écoute réseau et le protocole associé, mais ce n'est pas obligatoire.

  1. Dans l'arborescence de navigation, développez le noeud Configuration.

  2. Sous le noeud Configuration, développez le noeud Configuration réseau.

  3. Sous le noeud Configuration réseau, sélectionnez le noeud Processus d'écoute réseau.

    La page Processus d'écoute réseau apparaît.

  4. Sur la page Processus d'écoute réseau, cliquez sur Nouveau.

    La page Nouveau processus d'écoute réseau apparaît.

  5. Dans le champ Nom, saisissez le nom du processus d'écoute réseau.

    Le nom d'un processus d'écoute réseau ne peut pas commencer par un chiffre.

  6. Dans le champ Protocole, spécifiez le protocole à associer au processus d'écoute réseau.

    • Sélectionnez Créer un protocole pour créer un protocole avec les paramètres par défaut. De plus, dans la liste déroulante Serveur virtuel, sélectionnez le serveur virtuel à associer au protocole.

      Le nom par défaut du protocole est celui du processus d'écoute réseau avec -protocol ajouté à la fin, mais vous pouvez modifier cette valeur.

    • Sélectionnez Utiliser un protocole existant, puis choisissez un protocole dans la liste déroulante. Pour activer la sécurité du processus d'écoute réseau, sélectionnez un protocole avec la sécurité activée.

      Si vous sélectionnez cette option, vous avez en principe créé le protocole de ce processus d'écoute au préalable.

    Vous pouvez spécifier le protocole au moment où vous créez le processus d'écoute réseau. Vous ne pouvez pas modifier le protocole associé ultérieurement. Le protocole détermine si la sécurité est activée pour le processus d'écoute réseau.

  7. Cochez la case Activé de l'option Statut pour activer le processus d'écoute réseau.

    Cette option est activée par défaut. Si le processus d'écoute est désactivé, toute tentative de connexion au processus d'écoute a pour résultat une exception de socket (java.net.ConnectException).

    Dans les versions antérieures à la version 9.1 de GlassFish Server, un processus d'écoute dont la valeur d'attribut activé était False renvoyait un code de réponse 404 pour toute demande qui lui était envoyée. Pour obtenir ce comportement dans la version en cours de GlassFish Server, activez le processus d'écoute et définissez l'état de chaque serveur virtuel associé sur OFF. Un serveur virtuel répertorie les processus d'écoute qui lui sont associés dans la liste déroulante Processus d'écoute réseau correspondante.

  8. La case en lecture seule Sécurité activée indique si la sécurité est activée pour le protocole utilisé par le processus d'écoute réseau.

    Si vous configurez admin-listener pour le serveur d'administration de domaine (DAS), qui utilise la configuration server-config, cette option est en lecture seule. Pour activer la sécurité sur ce processus d'écoute, vous devez activer l'administration sécurisée. Reportez-vous à Administration sécurisée.

  9. Cochez la case Activé de Processus d'écoute JK pour indiquer que le processus d'écoute prend en charge le protocole mod-jk Apache.

    Le protocole mod-jk est un protocole communication/réseau propriétaire entre le serveur Web httpd Apache et une instance back-end de GlassFish Server. Si le processus d'écoute prend en charge ce protocole, httpd est censé gérer tout le contenu statique, tandis que les demandes de ressource dynamique (c'est-à-dire de servlets et de JSP) sont acheminées vers le processus d'écoute réseau activé mod-jk de GlassFish Server, à l'aide du protocole mod-jk. Cette option est désactivée par défaut.

  10. Dans le champ Port, saisissez le numéro de port sur lequel le processus d'écoute réseau va écouter.

    Les valeurs autorisées sont comprises entre 1 et 65535. Sur un système UNIX, la création de sockets écoutant sur les ports 1 à 1024 requiert des privilèges de superutilisateur.

  11. Dans le champ Adresse, saisissez l'adresse IP sur laquelle le processus d'écoute réseau va écouter.

    L'adresse peut présenter une notation IPv6 ou des paires de chiffres séparées par des points. Elle peut être définie sur any (pour INADDR_ANY) pour écouter sur toutes les adresses IP. Il peut s'agir d'un nom d'hôte.

  12. Dans la liste déroulante Pool de threads, sélectionnez le pool de threads à associer au processus d'écoute réseau.

    Normalement, vous devez sélectionner l'un des deux pools de threads qui sont configurés lorsque vous installez GlassFish Server.

  13. Dans la liste déroulante Transport, sélectionnez le type de transport à utiliser par le processus d'écoute réseau.

    Par défaut, le seul choix possible est tcp.

  14. Cliquez sur OK.

Etapes suivantes

Si vous avez activé la sécurité pour le protocole du processus d'écoute réseau, vous pouvez utiliser l'onglet SSL pour modifier les paramètres SSL du protocole.
