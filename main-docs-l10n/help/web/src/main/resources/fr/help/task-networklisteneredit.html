Modification d'un processus d'écoute réseau

  1. Dans l'arborescence de navigation, développez le noeud Configuration.

  2. Sous le noeud Configuration, développez le noeud Configuration réseau.

  3. Sous le noeud Configuration réseau, sélectionnez le noeud Processus d'écoute réseau.

    La page Processus d'écoute réseau apparaît.

  4. Sur la page Processus d'écoute réseau, cliquez sur le nom du processus d'écoute réseau que vous modifiez.

    La page Modifier le processus d'écoute réseau apparaît.

  5. Sélectionnez le protocole utilisé par le processus d'écoute réseau pour modifier ses paramètres. Par exemple, vous pouvez activer la sécurité. Vous ne pouvez pas choisir un autre protocole.

  6. Cochez ou désélectionnez la case Activé de l'option Statut pour modifier le statut du processus d'écoute réseau.

    Le processus d'écoute peut être activé ou désactivé. Cette option est activée par défaut. Si le processus d'écoute est désactivé, toute tentative de connexion au processus d'écoute a pour résultat une exception de socket (java.net.ConnectException).

  7. La case en lecture seule Sécurité activée indique si la sécurité est activée pour le protocole utilisé par le processus d'écoute réseau.

    Si vous configurez admin-listener pour le serveur d'administration de domaine (DAS), qui utilise la configuration server-config, cette option est en lecture seule. Pour activer la sécurité sur ce processus d'écoute, vous devez activer l'administration sécurisée. Reportez-vous à Administration sécurisée.

  8. Cochez la case Activé de Processus d'écoute JK pour indiquer que le processus d'écoute prend en charge le protocole mod-jk Apache.

    Le protocole mod-jk est un protocole communication/réseau propriétaire entre le serveur Web httpd Apache et une instance back-end de GlassFish Server. Si le processus d'écoute prend en charge ce protocole, httpd est censé gérer tout le contenu statique, tandis que les demandes de ressource dynamique (c'est-à-dire de servlets et de JSP) sont acheminées vers le processus d'écoute réseau activé mod-jk de GlassFish Server, à l'aide du protocole mod-jk. Cette option est désactivée par défaut.

  9. Dans le champ Port, saisissez le numéro de port sur lequel le processus d'écoute réseau va écouter.

    Les valeurs autorisées sont comprises entre 1 et 65535. Sur un système UNIX, la création de sockets écoutant sur les ports 1 à 1024 requiert des privilèges de superutilisateur.

  10. Dans le champ Adresse, saisissez l'adresse IP sur laquelle le processus d'écoute réseau va écouter.

    L'adresse peut présenter une notation IPv6 ou des paires de chiffres séparées par des points. Elle peut être définie sur any (pour INADDR_ANY) pour écouter sur toutes les adresses IP. Il peut s'agir d'un nom d'hôte.

  11. Dans la liste déroulante Pool de threads, sélectionnez le pool de threads à associer au processus d'écoute réseau.

  12. Cliquez sur Enregistrer.

    Conseil:

    Pour revenir aux paramètres par défaut, cliquez sur le bouton Charger les valeurs par défaut, puis sur Enregistrer.
Etapes suivantes

Vous pouvez utiliser les onglets SSL, HTTP et Cache de fichiers pour modifier les paramètres du protocole associé au processus d'écoute réseau. L'onglet SSL n'est utile que si vous avez activé la sécurité pour le protocole.
