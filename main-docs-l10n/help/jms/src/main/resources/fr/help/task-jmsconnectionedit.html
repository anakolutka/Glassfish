Modification d'une ressource de fabrique de connexions JMS

  1. Dans l'arborescence de navigation, développez le noeud Ressources.

  2. Sous le noeud Ressources, développez le noeud Ressources JMS.

  3. Sous le noeud Ressources JMS, sélectionnez le noeud Fabriques de connexions JMS.

    La page Fabriques de connexions JMS apparaît.

  4. Sur la page Fabriques de connexions JMS, cliquez sur le nom de la fabrique de connexions à modifier.

    La page Modifier une fabrique de connexions JMS apparaît.

  5. Dans le champ Description, saisissez la description de la fabrique de connexions.

  6. Sélectionnez la case Activé du champ Statut pour activer la ressource lors de l'exécution.

    Cette option est activée par défaut.

  7. Dans le champ Taille initiale et minimale du pool, spécifiez le nombre minimal de connexions dans le pool.

    La valeur par défaut est 8.

  8. Dans le champ Taille maximale du pool, spécifiez le nombre maximal de connexions dans le pool.

    La valeur par défaut est 32.

  9. Dans le champ Valeur de redimensionnement du pool, spécifiez le nombre de connexions à enlever lorsque le délai d'inactivité du pool est dépassé.

    La valeur par défaut est 2.

  10. Dans le champ Délai d'inactivité, spécifiez la durée maximale, en secondes, pendant laquelle une connexion peut rester inactive dans le pool.

    La valeur par défaut est 300.

  11. Dans le champ Attente max., spécifiez le délai d'attente au-delà duquel la connexion est temporisée pour l'application demandant une connexion.

    La valeur par défaut est 60 000.

  12. Si la case Fermer toutes les connexions du champ Echec sur tout est cochée, GlassFish Server ferme toutes les connexions du pool et les rétablit en cas d'échec de l'une d'entre elles.

    Si vous ne sélectionnez pas cette case, les connexions individuelles ne seront rétablies que lorsqu'elles seront utilisées.

  13. Pour une ressource de fabrique de connexions JMS, spécifiez la valeur de prise en charge des transactions comme suit :

    • Choisissez XATransaction (valeur par défaut) si la ressource peut être utilisée pour les transactions impliquant l'utilisation de plusieurs ressources dans la portée d'une transaction.

      Par exemple, cette ressource plus une ressource JDBC, une ressource de connecteur ou une autre ressource de fabrique de connexions JMS. Cette valeur offre un maximum de flexibilité. Les ressources configurées comme XATransaction participent aux opérations de validation (commit) en deux phases.

    • Choisissez LocalTransaction si la ressource peut être utilisée soit pour les transactions impliquant uniquement une ressource dans la portée de la transaction, soit en tant que dernier agent dans une transaction distribuée impliquant plusieurs ressources XA.

      Cette valeur offre de meilleures performances. Les ressources configurées comme LocalTransaction ne sont pas utilisées dans les opérations de validation (commit) en deux phases.

    • Choisissez NoTransaction si la ressource ne peut participer à aucune transaction.

      Dans les applications JMS, l'utilisation de cette configuration est limitée.

  14. Cochez la case Validation de la connexion pour activer la validation de la connexion.

    Cette option est désactivée par défaut.

  15. Dans la table Autres propriétés, spécifiez les propriétés supplémentaires de la fabrique de connexions.

    Pour plus d'informations sur les propriétés disponibles, reportez-vous à Propriétés propres aux fabriques de connexions JMS.

    • Pour ajouter une propriété, cliquez sur le bouton Ajouter une propriété. Sur la ligne vide qui apparaît, saisissez le nom de la propriété dans le champ Nom et la valeur de la propriété dans le champ Valeur.

    • Pour modifier une propriété, modifiez le champ Valeur de cette propriété.

    • Pour supprimer une propriété, sélectionnez la case située à gauche du champ Nom de la propriété à supprimer, puis cliquez sur le bouton Supprimer des propriétés.

  16. Vous pouvez cliquer sur l'onglet Cibles pour accéder à la page Cibles de ressources de la fabrique de connexions JMS. Sur cette page, procédez comme suit :

    1. Cliquez sur Gérer les cibles pour ouvrir la page Gérer les cibles des ressources.

      Sur cette page, déplacez les cibles entre les colonnes Disponible et Sélectionné. Dans la colonne Sélectionné, veillez à placer les cibles sur lesquelles les applications qui utilisent la ressource seront déployées. Les cibles disponibles comprennent les instances de serveur et les clusters disponibles, ainsi que l'instance de serveur par défaut server. Cliquez sur OK pour enregistrer les modifications.

    2. Cochez la case correspondant à une cible, puis cliquez sur Activer ou sur Désactiver pour activer ou désactiver la ressource pour cette cible.

  17. Cliquez sur Enregistrer.

    Conseil:

    Pour revenir aux paramètres par défaut, cliquez sur le bouton Charger les valeurs par défaut, puis sur Enregistrer.
