Afficher le journal du serveur

  1. Dans l'arborescence de navigation, sélectionnez le nœud Application Server.

    La page Informations générales s'affiche.

  2. Sur la page Informations générales, cliquez sur le bouton Afficher les fichiers journaux.

    Une fenêtre de visionneur de journaux s'ouvre.

  3. Utilisez la zone Critères de recherche pour personnaliser et filtrer le visionneur de journaux.
    • Cliquez sur Recherche avancée dans la zone Critères de recherche pour filtrer davantage le visionneur de journaux.
    • Dans le champ Horodatage :
      • Pour afficher les messages les plus récents, sélectionnez Le plus récent (valeur par défaut).
      • Pour afficher les messages uniquement sur une période donnée, sélectionnez Plage spécifique et entrez une date et une heure dans les champs De et À qui apparaissent.

        Pour la valeur Heure, la syntaxe doit prendre la forme suivante (SSS est mis pour millisecondes) :

        hh:mm:ss.SSS

        Par exemple :

        17:10:00.000

        Si la valeur de début est postérieure à la valeur de fin, un message d'erreur apparaît.

    • Dans la liste déroulante Niveau de journalisation, sélectionnez un niveau de journalisation pour filtrer les messages par niveau de journalisation. Sélectionnez la case « Ne pas inclure des messages plus graves » pour afficher uniquement les messages du niveau choisi.

      Par défaut, tous les messages figurant dans le journal du serveur à partir du niveau de journalisation choisi sont affichés.

      Pour être sûr que les messages que vous voulez afficher apparaissent dans le journal du serveur, commencez par définir les niveaux de journalisation appropriés sur la page Niveaux de journalisation de module. Voir Configurer les niveaux de journalisation.

      Si vous choisissez de filtrer les messages du journal en fonction du niveau de journalisation, seuls les messages correspondant aux critères de filtre spécifiés s'affichent. Toutefois, ce filtrage n'affecte pas les messages consignés dans le journal du serveur.

  4. Si vous avez cliqué sur Recherche avancée dans la zone Critères de recherche, suivez ces étapes.
    1. Dans la liste déroulante , sélectionnez un ou plusieurs noms des espaces pour filtrer par module.

      Utilisez maj + clic ou Ctrl + clic pour sélectionner plusieurs espaces de noms.

      Si vous sélectionnez un espace de noms à un niveau supérieur, tous les espaces de noms situés en dessous seront également sélectionnés. Par exemple, si vous sélectionnez javax.enterprise.system, seront également sélectionnés les journaux de tous les modules situés sous cet espace de noms : javax.enterprise.system.core, javax.enterprise.system.tools.admin , etc.

    2. Dans le champ personnalisé, saisissez les noms des journals, un par ligne, pour afficher les messages des journals propres à une application particulières.

      Si l'application comprend plusieurs modules, vous pouvez les afficher tous ou seulement une partie. Prenons l'exemple d'une application comprenant les journaux suivants :

      com.mycompany.myapp.module1
com.mycompany.myapp.module2
com.mycompany.myapp.module3

      Pour afficher des messages de tous les modules dans l'application, tapez com.mycompany.myapp . Pour afficher les messages du module2 uniquement, entrez com.mycompany.myapp.module2.

      Si vous spécifiez un ou plusieurs enregistreurs personnalisés, les messages des modules du Enterprise Server n'apparaissent que si vous les avez explicitement spécifiés dans la zone Enregistreur.

    3. Dans le champ Nom-paire de valeurs, saisissez le nom-clé et la valeur de ce thread pour afficher le résutlat d'un thread spécifique.

      Le nom clé est _ThreadID. Par exemple :

      _ThreadID=13

      Supposons que com.mycompany.myapp.module2 fonctionne sur plusieurs threads. Pour affiner le visionneur de journaux et afficher uniquement le résultat d'un seul thread, spécifiez le journal de ce module dans le champ Enregistreur personnalisé, puis spécifiez l'ID thread.

    4. Dans la liste déroulante Affichage, sélectionnez une valeur dans la liste déroulante pour afficher le nombre de messages à la fois.

      Le nombre de messages affichés par défaut est de 40. Les autres choix sont 100, 250 et 1000.

      Pour afficher les suivis de pile, désélectionnez la case « Tronquer les messages trop longs ». Par défaut, les suivis de pile n'apparaissent pas dans le visionneur. Pour les afficher, cliquez sur le lien (détails) pour un message.

    5. Cliquez sur Recherche élémentaire pour masquer la zone Options avancées.
  5. Dans la zone Résultats de la recherche, vous pouvez personnaliser l'affichage des messages.

    La zone Résultats de la recherche affiche les 40 dernières entrées dans le journal du serveur, en fonction des paramètres spécifiés aux pages Paramètres de journalisation et Niveaux de journalisation.

    • Cliquez sur les flèches des colonnes pour trier les messages différemment.
    • Cliquez sur le lien intitulé (détails) pour afficher la version formatée d'un message.

      La fenêtre Détails de l'entrée de journal s'affiche avec une version formatée du message.

    • Cliquez sur les boutons situés en haut et en bas du tableau pour afficher les entrées antérieures ou postérieures du fichier journal.
