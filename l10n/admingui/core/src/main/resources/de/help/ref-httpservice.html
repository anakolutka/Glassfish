HTTP-Dienst

Die Seite "HTTP-Dienst" umfasst die folgenden Optionen:

Zugriffsprotokollierung

Bei Auswahl wird die Zugriffsprotokollierung für alle Unterelemente von virtuellen Servern aktiviert, welche diese Eigenschaft nicht angeben. Wenn diese Option nicht ausgewählt wird, wird die Zugriffsprotokollierung für alle Unterelemente von virtuellen Servern deaktiviert, welche diese Eigenschaft nicht angeben.

Größe des Zugriffsprotokollpuffers

Die Größe in Byte des Zugriffsprotokollpuffers oder ein Wert unter oder gleich Null für nicht gepufferte Zugriffsprotokolle. Der Standardwert lautet 4096.

Intervall für Schreibvorgänge für Zugriffsprotokoll

Die Anzahl der Sekunden, bevor das Protokoll auf die Festplatte geschrieben wird. Das Zugriffsprotokoll wird geschrieben, wenn der Puffer voll ist oder das Intervall abläuft. Wenn der Wert kleiner oder gleich Null ist, werden die Pufferdaten immer geschrieben, auch wenn der Puffer nicht voll ist. Dies bedeutet, dass das Meldungsprotokoll bei jedem Zugriff auf den Server direkt in der Datei gespeichert wird.

Weitere Eigenschaften

Weitere Eigenschaften für den HTTP-Dienst.

Folgende Eigenschaften stehen zur Verfügung:

connectionTimeout

Gibt die Anzahl in Millisekunden an, die der HTTP-Listener nach dem Akzeptieren einer Verbindung wartet, bis die Anforderungs-URI-Zeile gesendet wird.

Der Standardwert ist 12.000 (12 Sekunden).

maxKeepAliveRequests

Gibt die maximale Anzahl an HTTP-Anforderungen an, die in die Pipeline gestellt werden können, bis die Verbindung durch den Server geschlossen wird. Setzen Sie diese Eigenschaft auf 1, um HTTP/1.0 keep-alive, HTTP/1.1 keep-alive und pipelining zu deaktivieren.

Der Standardwert ist 1000.

traceEnabled

Wenn Sie diese Eigenschaft auf "true" setzen, wird die TRACE-Methode aktiviert. Das Setzen dieser Eigenschaft auf "false" macht Enterprise Server weniger anfällig für Cross-Site-Skriptangriffe.

Weitere Informationen zur TRACE-Methode finden Sie in RFC 2616: Hypertext Transfer Protocol -- HTTP/1.1.

Der Standardwert lautet "True".
