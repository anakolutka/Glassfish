So werden HTTP-Einstellungen für ein Protokoll bearbeitet

  1. Blenden Sie in der Navigationsbaumstruktur den Knoten "Konfiguration" ein.

  2. Blenden Sie unter dem Konfigurationsknoten den Knoten "Netzwerkkonfiguration" ein.

  3. Wählen Sie unter dem Knoten "Netzwerkkonfiguration" den Knoten "Protokolle".

    Die Seite "Protokolle" wird geöffnet.

  4. Klicken Sie auf der Seite "Protokolle" auf den Namen des zu bearbeitenden Protokolls.

    Die Seite "Protokoll bearbeiten" wird geöffnet.

  5. Klicken Sie auf der Seite "Protokoll bearbeiten" auf die Registerkarte "HTTP".

    Die Seite "HTTP" wird geöffnet.

  6. Geben Sie im Feld "Servername" den Hostnamen ein, der in den URLs verwendet wird, die der Server an den Client sendet.

    Dieser Name ist der Aliasname, falls Ihr Server einen Alias verwendet. Wenn Ihr Server keinen Alias verwendet, lassen Sie dieses Feld leer.

  7. Wählen Sie aus der Dropdown-Liste "Standard-Virtual Server" den virtuellen Server aus, der mit dem Protokoll verknüpft werden soll.

    Auf der Seite "Virtuelle Server" können Sie virtuelle Server definieren.

  8. Geben Sie in das Feld "Portumleitung" den Wert des Ports ein, der bei der Umleitung einer Anforderung an einen anderen Port verwendet werden soll.

    GlassFish Server leitet Anforderungen automatisch weiter, wenn die folgenden zwei Bedingungen erfüllt sind:

    • Dieses Protokoll unterstützt Nicht-SSL-Anforderungen.

    • Es wird eine Anforderung empfangen, für die ein übereinstimmendes Sicherheits-Constraint einen SSL-Transport erfordert.

    Wird kein Umleitungsport angegeben, verwendet GlassFish Server die in der ursprünglichen Anforderung angegebene Portnummer.

  9. Geben Sie im Feld "Maximale Verbindungen" die maximale Anzahl der permanenten Verbindungen an, die im Keep-Alive-Modus beibehalten werden sollen.

    Das Keep-Alive-Teilsystem ruft regelmäßig Verbindungen im Leerlaufstatus ab.

  10. Geben Sie im Feld "Timeout" die maximale Anzahl Sekunden an, die eine Verbindung als im Leerlauf befindlich betrachtet und im Keep-Alive-Zustand gehalten werden kann.

    Ein Wert kleiner oder gleich null (0) bedeutet, dass Keep-Alive-Verbindungen unbegrenzt offen gehalten werden. Der Standardwert ist 30 Sekunden.

  11. Aktivieren Sie das Kontrollkästchen "Timeout beim Hochladen", um die Verbindung für ein Servlet, das Byte langsam liest, nach Erreichen des Grenzwerts für das Timeout beim Hochladen der Verbindung zu schließen.

    Bei deaktivierter Option tritt für Servlet-Verbindungen kein Timeout ein. Diese Option ist standardmäßig deaktiviert.

  12. Geben Sie im Feld "Timeout beim Hochladen der Verbindung" das Timeout beim Hochladen in Millisekunden an.

    Dieses Feld steht nur zur Verfügung, wenn das Kontrollkästchen "Timeout beim Hochladen" aktiviert ist. Der Standardwert ist 300.000 Millisekunden.

  13. Geben Sie im Feld "Timeout für Anforderung" die Anzahl Sekunden ein, bevor die Anforderung wegen Timeout abgebrochen wird.

    Wenn die Anforderung nicht vor Erreichen des Timeoutwerts verarbeitet wird, wird sie ignoriert. Der Standardwert ist 30 Sekunden.

  14. Geben Sie im Feld "Größe des Sendepuffers" die Größe des Sendepuffers in Byte an.

    Der Standardwert ist 8192 Byte.

  15. Geben Sie im Feld "Headerpuffergröße" die Größe (in Byte) des Puffers an, der von den Anforderungsverarbeitungsthreads zum Lesen der Anforderungsdaten verwendet wird.

    Der Standardwert ist 8192 Byte.

  16. Geben Sie im Feld "Maximale Post-Größe" die maximale Größe von POST-Aktionen in Byte an.

    Der Standardwert ist 2.097.152 Byte.

  17. Geben Sie im Feld "URI-Codierung" den Namen des Zeichensatzes ein, der zum Decodieren der erhaltenen Anforderungs-URIs verwendet wird.

    Als Wert muss ein gültiger IANA-Zeichensatzname angegeben werden. Der Standardwert ist UTF-8.

  18. Geben Sie im Feld "Version" die Version des verwendeten HTTP-Protokolls an.

    Der Standardwert ist HTTP/1.1.

  19. Wählen Sie aus der Dropdown-Liste "Komprimierung" die HTTP/1.1 GZIP-Komprimierung, um Serverbandbreite zu sparen.

    Zur Auswahl stehen:

    On

    Komprimiert Daten.

    Off

    In der folgenden Tabelle werden diese Eigenschaften aufgelistet.

    force

    Erzwingt die Datenkomprimierung in allen Fällen.

    Der Standardwert ist "Off".

  20. Geben Sie im Feld "Komprimierbare MIME-Typen" eine durch Komma getrennte Liste von MIME-Typen ein, für die HTTP-Komprimierung verwendet wird.

    Der Standardwert ist text/html,text/xml,text/plain.

  21. Geben Sie im Feld "Mindestkomprimierungsgröße" die minimale Größe einer komprimierten Datei ein.

    Dieser Wert muss festgelegt werden, wenn das Kontrollkästchen "Komprimierung" aktiviert wird. Der Standardwert ist 2048 Byte.

  22. Geben Sie im Feld "User Agent ohne Komprimierung" eine durch Komma getrennte Liste von regulären Ausdrücken ein, die User Agents von HTTP-Clients entsprechen, für die keine Komprimierung verwendet werden soll.

    Standardmäßig handelt es sich bei diesem Wert um eine leere Zeichenfolge.

  23. Geben Sie im Feld "Beschränkte User Agents" eine Liste von beschränkten User Agents ein, auf die HTTP-Komprimierung angewendet wird.

    Wenn keine User Agents angegeben werden, wird die HTTP-Komprimierung auf alle User Agents angewendet. Standardmäßig sind keine User Agents angegeben.

  24. Geben Sie im Feld "Standardmäßiger Antworttyp" eine Zeichenfolge ein, die den standardmäßigen Antworttyp angibt.

    Das Format ist eine durch Semikolon getrennte Zeichenfolge bestehend aus Contenttyp, Codierung, Sprache und Zeichensatz. Der Standardwert ist text/html; charset=iso-8859-1.

  25. Geben Sie im Feld "Erzwungener Antworttyp" eine Zeichenfolge ein, die den zu verwendenden Antworttyp angibt, falls keine MIME-Zuordnung mit der Dateierweiterung übereinstimmt.

    Das Format ist eine durch Semikolon getrennte Zeichenfolge bestehend aus Contenttyp, Codierung, Sprache und Zeichensatz. Der Standardwert ist text/html; charset=iso-8859-1.

  26. Geben Sie im Feld "Adapter" den Klassennamen des statischen Ressourcenadapters ein.

    Der Standardwert ist com.sun.grizzly.tcp.StaticResourcesAdapter.

  27. Wählen Sie das Kontrollkästchen "Comet-Unterstützung aktiviert", um Comet-Unterstützung für das Protokoll zu aktivieren.

    Diese Option ist standardmäßig deaktiviert.

  28. Aktivieren Sie das Kontrollkästchen "DNS-Suche", um die DNS-Suche (Domain Name System) zu aktivieren.

    Diese Option ist standardmäßig deaktiviert.

  29. Aktivieren Sie das Kontrollkästchen "RCM-Unterstützung", um Resource Configuration Management (RCM) zu aktivieren.

    Diese Option ist standardmäßig deaktiviert.

  30. Aktivieren Sie das Kontrollkästchen "Trace", um den TRACE-Vorgang zu aktivieren.

    Bei deaktivierter Option ist GlassFish Server weniger anfällig für siteübergreifende Skriptangriffe. Diese Option ist standardmäßig aktiviert.

  31. Aktivieren Sie das Kontrollkästchen "Autorisierungsdurchlauf", um anzugeben, dass der Netzwerk-Listener, der dieses Protokoll verwendet, Daten von einem SSL-beendenden Proxyserver empfängt.

    Diese Option ist standardmäßig deaktiviert.

  32. Aktivieren Sie das Kontrollkästchen "Erstellen von Datenblöcken", um das Erstellen von Datenblöcken für HTTP-Antworten zu aktivieren.

    Diese Option ist standardmäßig aktiviert.

  33. Aktivieren Sie das Kontrollkästchen "XPowered-By", um GlassFish Server anzuweisen, X-Powered-By -Header entsprechend den Spezifikationen für Java Servlet 3.0 und Java Server Pages 2.0 zu verwenden.

    Diese Option ist standardmäßig aktiviert.

  34. Klicken Sie auf "Speichern".

    Tipp:

    Um zu den Standardeinstellungen zurückzukehren, klicken Sie auf die Schaltfläche "Standardwerte laden" und anschließend auf "Speichern".
