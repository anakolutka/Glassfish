HTTP 서비스를 구성하는 방법

  1. 탐색 트리에서 구성 노드를 선택합니다.
  2. 구성 노드에서 HTTP 서비스 노드를 선택합니다.

    HTTP 서비스 페이지가 열립니다.

  3. 이 등록 정보를 지정하지 않는 모든 가상 서버 하위 요소에 대해 로깅을 활성화하려면 액세스 로깅 사용 가능 확인란을 선택합니다.

    이 등록 정보를 지정하지 않는 모든 가상 서버 하위 요소에 대해 로깅을 비활성화하려면 액세스 로깅 사용 가능 확인란을 선택 취소합니다.

  4. 액세스 로그 버퍼 크기 필드에 액세스 로그 버퍼의 크기(바이트)를 입력합니다.

    값이 0보다 작거나 같으면 액세스 로그 버퍼링이 사용되지 않습니다. 기본값은 4096입니다.

  5. 액세스 로그 쓰기 간격 필드에 로그가 디스크에 기록되기 전까지의 시간(초)을 입력합니다.

    액세스 로그는 버퍼가 가득 차거나 간격이 만료되면 기록됩니다. 값이 0보다 작거나 같으면 버퍼는 가득 차지 않더라도 항상 기록됩니다. 즉, 서버에 액세스할 때마다 로그 메시지가 직접 파일에 저장됩니다. 기본값은 0입니다.

  6. (옵션) 추가 등록 정보 섹션에서 추가 등록 정보를 지정합니다.

    제공되는 등록 정보는 다음과 같습니다.

    connectionTimeout

    연결 승인 후 HTTP 수신기가 요청된 URI 라인이 나타날 때까지 기다릴 시간(밀리초 단위)을 지정합니다.

    기본값은 12,000(12초)입니다.

    maxKeepAliveRequests

    서버에서 연결을 닫을 때까지 파이프라인될 수 있는 최대 HTTP 요청 수를 지정합니다. 이 등록 정보를 1로 설정하면 HTTP/1.0 연결 유지(keep-alive)와 HTTP/1.1 연결 유지(keep-alive) 및 파이프라인 처리가 비활성화됩니다.

    기본값은 1000입니다.

    traceEnabled

    true로 설정하면 TRACE 메소드가 활성화됩니다. 이 등록 정보를 false로 설정하면 Enterprise Server가 크로스 사이트 스크립팅 공격에 덜 취약해집니다.

    TRACE 메소드에 대한 자세한 내용은 RFC 2616: Hypertext Transfer Protocol -- HTTP/1.1을 참조하십시오.

    기본값은 true입니다.

  7. 저장을 누릅니다.
