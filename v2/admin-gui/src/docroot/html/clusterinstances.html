To Start, Stop, and Delete Clustered Instances and Set Load Balancing Weights

  1. In the tree component, expand the Clusters node.
  2. Expand the node for the cluster that contains the server instance.
  3. Click the Instances tab to display the Clustered Server Instances page.

    On this page you can:

    • Select the checkbox for an instance and click Delete, Start, or Stop to perform the selected action on all the specified server instances.

    • If the cluster has load balancer, select the checkbox for an instance and choose a load balancer action from the pull-down menu.

      • Start Load Balancing enables an instance for load balancing.

      • Quiesce disables an instance for load balancing gracefully, by allowing time to complete requests that are in process. On the Load Balancer Monitoring pop-up, click Begin Quiesce Now to start quiescing the server instance or instances. Stop Instances Now stops the instance or instances without quiescing.

    • If the cluster has load balancer, change the configuration of the load balancer for an instance as follows:

      • If the load-balancing policy is weighted round-robin, change the weight of the instance for the load balancer.

        Note - You can set the load balancing policy in the Edit Load Balancer Health Checker page. For more information, see To Configure Load Balancer Health Checking.

        In the Weight field, type an integer that represents the weight of the instance.

        For example, if you have two instances, and want one instance to process one request out of every four, and the other instance to process three requests out of four, set the weights to 1 and 3, or 25 and 75, if you prefer to use percentages.

        The default weight is 100.

      • Change the time (in minutes) that this instance should take to reach a quiescent state after having been disabled.

    • Click the name of the instance to bring up the General Information page.
