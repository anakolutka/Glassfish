Écran Transférer les configurations de cluster

La mise à niveau de la configuration d'un cluster de serveur Sun Java System Application Server 8.x Enterprise Edition (EE) vers un serveur Sun Java System Application Server 9.1 requiert un fichier clinstance.conf valide. L'outil de mise à niveau suppose que le fichier clinstance.conf est un reflet exact du cluster de serveur 8.x EE. Si, pour une raison quelconque, ce fichier ne représente pas exactement la configuration réelle du cluster, la définition de celui-ci doit être modifiée manuellement avant de pouvoir effectuer la mise à niveau.

Utilisez l'écran Transférer les configurations de cluster pour spécifier les fichiers de configuration de cluster qui doivent être migrés vers la nouvelle installation.

  1. Appuyez sur le bouton Ajouter. La boîte de dialogue de sélection du fichier clinstance.conf s'affiche.
  2. Sélectionnez le fichier de cluster clinstance.conf qui contient les informations de configuration du cluster à migrer.
  3. Appuyez sur le bouton Ouvrir.
  4. Répétez la procédure jusqu'à ce que tous les fichiers de configuration de cluster devant être migrés aient été ajoutés.
  5. Appuyez ensuite sur le bouton Suivant. L'écran Résultats de la mise à niveau s'affiche, indiquant le statut de la mise à niveau.
