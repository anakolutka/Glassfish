Applicazioni

Nella pagina Applicazioni viene visualizzata la lista delle applicazioni distribuite in GlassFish Server. È possibile visualizzare e gestire le applicazioni già distribuite e distribuirne altre.

Per ogni applicazione vengono fornite le informazioni riportate di seguito.

Nome

Il nome dell'applicazione.

Il nome può includere un identificativo di versione opzionale, che segue il nome ed è separato da questo mediante due punti (:). Per ulteriori informazioni sulle versioni di moduli e applicazioni, vedere Module and Application Versions in Oracle GlassFish Server Application Deployment Guide.

Ordine di distribuzione

L'ordine di distribuzione dell'applicazione.

Le applicazioni con numero inferiore vengono caricate per prime all'avvio del server. Ad esempio, un'applicazione con il numero di ordine di distribuzione 102 viene caricata prima di un'applicazione con il numero di ordine di distribuzione 110. Se l'ordine di distribuzione non viene specificato al momento della distribuzione dell'applicazione, verrà assegnato l'ordine di distribuzione predefinito 100. Se due applicazioni dispongono dello stesso ordine di distribuzione, verrà caricata per prima l'applicazione che è stata distribuita per prima. La specifica dell'ordine di distribuzione è utile se l'applicazione dispone di dipendenze e deve essere caricata secondo un determinato ordine.

Abilitato o Stato

Il segno di spunta indica che l'applicazione è abilitata, mentre la X indica che è disabilitata, se esiste solo l'istanza server predefinita server.

Se esistono cluster o altre istanze server standalone, selezionare Abilitato in tutte le destinazioni per visualizzare le destinazioni sulle quali l'applicazione è distribuita.

Motori

I tipi di contenitori utilizzati dall'applicazione. Di seguito sono indicati i tipi di contenitori disponibili:

  • web

  • webservices

  • ejb

  • connector

  • appclient

  • weld (contenitore per Contexts and Dependency Injection per le applicazioni della piattaforma Java EE)

Azione

Collegamenti alle azioni che è possibile eseguire sul componente dopo la distribuzione: Ridistribuisci e Ricarica per tutti i componenti, Avvia per le applicazioni Web e i client applicazione e Scarica stub client per i client applicazione.

La tabella Applicazioni distribuite contiene inoltre le opzioni riportate di seguito.

Distribuisci

Pulsante che consente di distribuire un'applicazione.

Annulla distribuzione

Pulsante che consente di annullare la distribuzione di una o più applicazioni selezionate.

Abilita

Pulsante che consente di abilitare una o più applicazioni selezionate, se esiste solo l'istanza server predefinita server.

Disabilita

Pulsante che consente di disabilitare una o più applicazioni selezionate, se esiste solo l'istanza server predefinita server.

Filtro

Elenco a discesa che filtra le applicazioni in base al motore.
