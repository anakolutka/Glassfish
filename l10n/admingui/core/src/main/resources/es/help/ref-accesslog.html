Registro de acceso

La página Registro de acceso contiene las siguientes opciones:

Rotación activada

Si selecciona esta opción, la rotación del archivo de registro se activará. Esta opción se encuentra deshabilitada de forma predeterminada.

Política de rotación

La única directiva de rotación disponible es tiempo.

Intervalo de rotación

Número de minutos transcurridos entre las rotaciones del registro de acceso. Este campo sólo es válido si la directiva de rotación es "tiempo". El valor predeterminado es 1.440 minutos (24 horas).

Asegúrese de que el sufijo de rotación contiene los valores necesarios para crear un sufijo único para cada rotación de registro. Por ejemplo, si el registro de acceso rota más de una vez al día, se incluye la hora del día en el sufijo de rotación.

Es necesario un único sufijo para asegurar que el archivo de registro de acceso rota cada vez que transcurre el intervalo de rotación. Si el sufijo de rotación del nuevo archivo de registro de acceso es el mismo que el del archivo de acceso existente, el archivo no ha rotado.

Sufijo de rotación

Valor de la cadena que se agregará al final del nombre del archivo de registro tras la rotación. El valor predeterminado es aaaa-MM-dd.

Formato

Valor de la cadena que especifica la información que hay que capturar en el registro de acceso y el orden en la que hay que capturarla. El valor predeterminado es el siguiente:

%client.name% %auth-user-name% %datetime% %request% %status% %response.length%

Los valores siguientes se encuentran disponibles para su inclusión en la cadena de formato:

%client.name%

Nombre de host del cliente

%client.dns%

DNS de cliente

%datetime%

Fecha del sistema

%request%

Línea de solicitud HTTP completa

%status%

Estado

%response.length%

Longitud del contenido de respuesta

%header.referer%

Encabezado del referenciador

%header.user-agent%

Encabezado del agente del usuario

%http-method%

Método HTTP

%http-uri%

URI de HTTP

%query-str%

Cadena de consulta HTTP

%http-version%

Versión de protocolo HTTP

%header.accept%

Encabezado de aceptación

%header.date%

Encabezado de fecha

%header.if-mod-since%

Encabezado si modificados desde

%header.auth%

Encabezado de autorización

%header.any%

Cualquier valor de encabezado HTTP válido definido en RFC 2616 (cualquiera también es un valor de encabezado válido; se especifica como variable aquí)

%auth-user-name%

Nombre de usuario autorizado

%cookie.value%

Valor de una cookie

%vs.id%

Id. de servidor virtual
