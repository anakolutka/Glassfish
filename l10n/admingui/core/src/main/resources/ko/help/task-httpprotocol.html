HTTP 프로토콜을 구성하는 방법

  1. 탐색 트리에서 구성 노드를 선택합니다.
  2. 구성 노드에서 HTTP 서비스 노드를 선택합니다.
  3. HTTP 서비스 페이지에서 HTTP 프로토콜 탭을 누릅니다.

    HTTP 프로토콜 페이지가 열립니다.

  4. 버전 필드에 사용할 HTTP 프로토콜의 버전을 입력합니다.

    허용되는 값은 HTTP/1.0 및 HTTP/1.1입니다. 기본값은 HTTP/1.1입니다.

  5. 클라이언트의 DNS 항목 조회를 활성화하려면 DNS 조회 사용 가능 확인란을 선택합니다.

    이 옵션은 기본적으로 비활성화됩니다.

  6. 서버에 대한 보안을 전역적으로 비활성화하려면 SSL 사용 가능 확인란을 선택 취소합니다.

    보안이 활성화된 모든 수신기에 SSL을 사용할 수 있도록 하려면 이 확인란을 선택한 상태로 둡니다. 이 옵션은 기본적으로 활성화됩니다.

  7. 강제 유형 필드에 확장자에 일치하는 사용 가능한 MIME 매핑이 없을 때 사용할 응답 유형을 입력합니다.

    기본값은 text/html; charset=iso-8859-1입니다.

  8. 기본 유형 필드에 기본 응답 유형을 입력합니다.

    기본값은 text/html; charset=iso-8859-1입니다. 값은 내용 유형, 인코딩, 언어 및 문자 집합이 세미콜론으로 구분된 문자열입니다.

  9. 저장을 누릅니다.

    참고 – 기본 설정으로 돌아가려면 기본값 로드 버튼을 누른 다음 저장을 누릅니다.
