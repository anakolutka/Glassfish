Ajout d'une instance GlassFish Server à un cluster

Les instances GlassFish Server forment la base d'un déploiement d'application. Une instance incluse dans un cluster est un membre d'un cluster. Chacune instance appartient à un domaine unique et possède une structure de répertoire, une configuration et des applications déployées qui lui est propre. Chaque instance comprend également le conteneur Web Java Platform, Enterprise Edition (Java EE) et le conteneur Enterprise JavaBeans (technologie EJB). Chaque nouvelle instance doit contenir une référence à un noeud qui définit l'hôte sur lequel elle résidera.

Remarque:

Pour créer une instance sur un noeud de type CONFIG représentant un hôte distant, vous devez utiliser la sous-commande create-local-instance(1). Vous ne pouvez pas utiliser la Console d'administration à cet effet.
Avant de commencer

Assurez-vous que les éléments suivants existent :

  • Le cluster auquel ajouter l'instance. Pour plus d'informations, reportez-vous à Création d'un cluster.

  • Le noeud sur lequel l'instance résidera. Pour plus d'informations, reportez-vous à Création d'un noeud.

  1. Dans l'arborescence de navigation, développez le noeud Clusters.

  2. Sous le noeud Clusters développé, sélectionnez le noeud correspondant au cluster.

    La page Informations générales du cluster apparaît.

  3. Sur la page Informations générales, cliquez sur l'onglet Instances.

    La page Instances de serveur clusterisées du cluster apparaît.

  4. Sur la page Instances de serveur clusterisées, cliquez sur Nouveau.

    La page Nouvelle instance de serveur clusterisée apparaît.

  5. Dans le champ Nom, saisissez le nom de l'instance.

    Le nom doit respecter les exigences suivantes :

    • Il ne peut contenir que des caractères ASCII.

    • Il doit commencer par une lettre, un chiffre ou un trait de soulignement.

    • Il ne peut contenir que les caractères suivants :

      • Lettres minuscules

      • Lettres majuscules

      • Chiffres

      • Trait d'union

      • Point

      • Trait de soulignement

    • Le nom doit être unique dans le domaine et ne doit pas appartenir à une autre instance GlassFish Server, à une configuration nommée, à un cluster ou à un noeud.

    • Il ne doit pas être domain, server ou tout autre mot-clé réservé par GlassFish Server.

  6. Dans la liste déroulante Noeud, sélectionnez le noeud sur lequel l'instance résidera.

  7. Cliquez sur OK.
