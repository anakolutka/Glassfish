Neuer verwalteter geplanter ExecutorService

Auf der Seite "Neuer verwalteter geplanter ExecutorService" können Sie eine geplante verwaltete ExecutorService-Ressource erstellen.

Die Seite "Neuer verwalteter geplanter ExecutorService" enthält die folgenden Optionen.

JNDI-Name

Ein eindeutiger Name, der die Ressource identifiziert.

Kontextinformation

Der Kontext der Ressource: Classloader, JNDI, Security oder WorkArea.

Status

Der Status der Ressource. Die Ressource kann aktiviert oder deaktiviert sein. Der Standardwert ist "Aktiviert".

Threadpriorität

Die Priorität der Threads, die für die verwaltete geplante ExecutorService-Instanz erstellt werden müssen. Der Standardwert ist 5.

Tasks mit langer Ausführungsdauer

XXX.

Blockiert nach

Die Zeit in Sekunden, die eine Task ausgeführt werden kann, bevor sie als blockiert betrachtet wird.

Core-Größe

Die Anzahl von Threads, die in einem Threadpool beibehalten werden sollen. Der Standardwert ist 0.

Maximale Poolgröße

Die maximale Anzahl von Threads, die im Pool enthalten sein können. Der Standardwert ist 2147483647.

Keep-Alive

Die Zeit in Sekunden, die Threads inaktiv bleiben können, wenn die Anzahl von Threads größer ist als die Core-Größe. Der Standardwert ist 60.

Threadgültigkeitsdauer

Die Zeit in Sekunden, die jeder Thread in einem Threadpool bleibt, bevor er gelöscht wird, unabhängig davon, ob die Anzahl von Threads größer ist als die Core-Größe oder ob der Thread inaktiv ist.

Taskqueuekapazität

Die Größe der Queue, die weitergeleitete Tasks enthält, die auf Ausführung warten. Der Standardwert ist 2147483647.

Beschreibung

Eine Beschreibung dieser Ressource.

Weitere Eigenschaften

Weitere Eigenschaften für die Ressource. In GlassFish Server werden keine weiteren Eigenschaften für verwaltete geplante ExecutorService-Ressourcen definiert.
