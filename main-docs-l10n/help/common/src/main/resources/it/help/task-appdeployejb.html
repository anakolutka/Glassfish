Per distribuire un file JAR EJB

  1. Selezionare il nodo Applicazioni nell'albero di navigazione.

    Viene visualizzata la pagina Applicazioni.

  2. Nella tabella Applicazioni distribuite fare clic sul pulsante Distribuisci.

    Viene visualizzata la pagina Distribuisci applicazioni o moduli.

  3. Specificare la posizione dell'applicazione da distribuire.

    • Se l'applicazione è un file di archivio che si trova nel computer client o è accessibile da tale computer, selezionare l'opzione File incluso in un package da caricare sul server.

      Il computer client è l'host sul quale viene visualizzata la Console di amministrazione tramite un browser.

      Fare clic su Sfoglia per individuare il file oppure digitarne il percorso completo.

    • Se l'applicazione è un file di archivio che si trova nel computer server o è un'applicazione decompressa da una directory esplosa, selezionare l'opzione Directory o file locale incluso in un package accessibile da GlassFish Server.

      Il computer server è l'host in cui è in esecuzione il server di amministrazione del dominio (DAS).

      Fare clic su Sfoglia file per cercare un file o su Sfoglia cartelle per cercare una directory. In alternativa, digitare il percorso completo del file o della directory. La distribuzione da una directory esplosa deve essere eseguita da sviluppatori esperti ed è sconsigliata per gli ambienti di produzione.

  4. Nell'elenco a discesa Tipo selezionare JAR EJB.

    Se è stata selezionata l'opzione File incluso in un package da caricare sul server o Directory o file locale incluso in un package accessibile da GlassFish Server, il tipo di applicazione viene rilevato in modo automatico. Se è stata selezionata l'opzione Directory o file locale incluso in un package accessibile da GlassFish Server, è necessario specificare il tipo manualmente.

  5. Nel campo Nome applicazione lasciare il nome predefinito, che è il nome della directory o il prefisso del nome file, oppure digitare un altro nome.

    Il nome predefinito viene visualizzato se si è scelto di caricare un file. Il nome dell'applicazione deve essere univoco.

    Il nome può includere un identificativo di versione opzionale, che segue il nome ed è separato da questo mediante due punti (:). L'identificativo di versione deve iniziare con una lettera o un numero. Può contenere caratteri alfanumerici più i caratteri di sottolineatura (_), lineetta (-) e punto (.). Per ulteriori informazioni sulle versioni di moduli e applicazioni, vedere Module and Application Versions in Oracle GlassFish Server Application Deployment Guide.

  6. Selezionare la casella di controllo Abilitato dell'opzione Stato per abilitare l'applicazione.

    Questa opzione è selezionata per impostazione predefinita. Se l'opzione è disabilitata, l'applicazione non è disponibile.

  7. Selezionare la casella di controllo Abilitato dell'opzione Disponibilità per abilitare la definizione di checkpoint SFSB per l'applicazione.

    Se la casella di controllo Abilitato è selezionata, la funzione High Availability è abilitata per la definizione di checkpoint e potenzialmente la passivazione dei bean di sessione con conservazione dello stato (SFSB). Se l'impostazione è false (impostazione predefinita), la definizione di checkpoint SFSB viene disabilitata per il modulo EJB specificato. Se l'impostazione è true, il modulo EJB specificato viene abilitato per la funzione High Availability. Impostare questa opzione su true solo se la funzione High Availability è configurata e abilitata ai livelli superiori, ad esempio a livello di server e di contenitore.

    Questa opzione viene visualizzata se esistono cluster o istanze server standalone oltre all'istanza server predefinita (server).

  8. Selezionare la casella di controllo Esegui verificatore per verificare i descrittori di distribuzione prima della distribuzione.

    Se l'opzione è selezionata e la verifica non riesce, la distribuzione non viene eseguita. Il verificatore esamina la struttura e il contenuto dei descrittori di distribuzione. La verifica di applicazioni di grandi dimensioni richiede spesso molto tempo. Questa opzione è disabilitata per impostazione predefinita.

    I package del verificatore devono essere installati tramite la utility Strumento di aggiornamento; in caso contrario, viene registrata un'avvertenza e l'opzione viene ignorata.

  9. Selezionare la casella di controllo Compatibilità per utilizzare i requisiti di visibilità JAR di GlassFish Server v2 anziché i più rigorosi requisiti Java EE 6 implementati nelle release v3 di GlassFish Server, compresa la 4.0.

    Questa opzione è disabilitata per impostazione predefinita.

    La specifica della piattaforma Java EE 6 impone requisiti più rigorosi rispetto a quelli previsti dalla piattaforma Java EE 5 in base ai quali i file JAR possono essere visibili per diversi moduli in un file EAR. In particolare, i client di applicazione non devono avere accesso ai file JAR EJB o ad altri file JAR nel file EAR a meno che per i riferimenti non siano utilizzati i meccanismi Java SE standard (ad esempio, le estensioni) o il meccanismo libreria-directory di Java EE. La selezione di questa casella comporta la rimozione delle limitazioni Java EE 6.

  10. Selezionare la casella di controllo Imponi ridistribuzione per forzare la ridistribuzione se l'applicazione è già stata distribuita.

    Se questa opzione non è selezionata, il tentativo di distribuire un'applicazione già distribuita causa un errore. Questa opzione è disabilitata per impostazione predefinita.

  11. Selezionare la casella di controllo Mantieni stato per conservare le sessioni Web, le istanze SFSB e i timer EJB creati in modo persistente tra le ridistribuzioni.

    Questa opzione è disabilitata per impostazione predefinita. È supportata solo nell'istanza server predefinita, denominata server. Non è supportata e viene ignorata per qualsiasi altra destinazione.

    Alcuni tipi di modifiche apportate a un'applicazione tra le ridistribuzioni impediscono a questa funzione di operare in modo corretto. Ad esempio, non modificare il set di variabili di istanza nella classe bean SFSB.

    Per le applicazioni Web, questa funzione è applicabile solo se nel file glassfish-web-app.xml l'attributo persistence-type dell'elemento session-manager è impostato su file.

    Per le istanze di bean di sessione con conservazione dello stato, il tipo di persistenza senza High Availability viene impostato nel server (opzione Tipo di persistenza SFSB) e deve essere impostato su file, ovvero il valore predefinito e consigliato.

    Se un elemento attivo quale una sessione Web, un'istanza SFSB o un timer EJB non viene preservato o ripristinato, nessuno di tali elementi sarà disponibile una volta completata la ridistribuzione. Tuttavia la ridistribuzione prosegue e viene registrata un'avvertenza.

    Per preservare i dati con stato attivo, GlassFish Server li serializza e li salva nella memoria. Per ripristinare i dati, il classloader dell'applicazione appena ridistribuita deserializza i dati salvati in precedenza.

  12. Selezionare la casella di controllo Conserva risorse con ambito applicazione per preservare le risorse con ambito a livello di applicazione e ripristinarle durante la ridistribuzione.

    Questa opzione è disabilitata per impostazione predefinita.

  13. Nel campo Ordine di distribuzione digitare un numero per specificare l'ordine di distribuzione dell'applicazione.

    Le applicazioni con numero inferiore vengono caricate per prime all'avvio del server. Ad esempio, un'applicazione con il numero di ordine di distribuzione 102 viene caricata prima di un'applicazione con il numero di ordine di distribuzione 110. Se l'ordine di distribuzione non viene specificato al momento della distribuzione dell'applicazione, verrà assegnato l'ordine di distribuzione predefinito 100. Se due applicazioni dispongono dello stesso ordine di distribuzione, verrà caricata per prima l'applicazione che è stata distribuita per prima. La specifica dell'ordine di distribuzione è utile se l'applicazione dispone di dipendenze e deve essere caricata secondo un determinato ordine.

  14. Nel campo Librerie digitare una lista separata da virgole di file JAR di libreria specifici dell'applicazione in uso.

    I percorsi possono essere assoluti o relativi. Un percorso relativo è relativo rispetto a domain-dir/lib/applibs. Se il percorso è assoluto, deve essere accessibile per il server di amministrazione del dominio (DAS), ovvero deve trovarsi in domain-dir. Le librerie vengono rese disponibili per l'applicazione secondo l'ordine in cui vengono specificate.

  15. Nel campo Descrizione digitare la descrizione dell'applicazione.

  16. Per aggiungere istanze server e cluster di destinazione in cui distribuire il modulo o l'applicazione, selezionarli dalla lista Destinazioni disponibili e utilizzare il pulsante Aggiungi per spostarli nella lista Destinazioni selezionate.

    L'opzione Destinazione viene visualizzata se esistono cluster o istanze server standalone oltre all'istanza server predefinita (server). Per impostazione predefinita, la distribuzione del modulo o dell'applicazione viene eseguita nell'istanza server predefinita, denominata server. Utilizzare il pulsante Aggiungi tutto per eseguire la distribuzione in tutte le destinazioni. Utilizzare i pulsanti Rimuovi e Rimuovi tutto per rimuovere le destinazioni dalla lista Destinazioni selezionate.

  17. Fare clic su OK.
