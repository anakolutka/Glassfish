Atributos de Domínio

Use a página Atributos de Domínio para configurar atributos de domínio avançados para o GlassFish Server.

A página Atributos de Domínio contém as opções a seguir.

Carregar Console Após Inicialização do DAS

Determina quando carregar a Console de Administração:

Baseado em Uso

Carrega a Console de Administração após o Servidor de Administração de Domínios (DAS) se a console tiver sido usada nas últimas 24 horas ou se outras instâncias do servidor existirem além do DAS. Esse é o default.

Sempre

Sempre carrega a Console de Administração após o DAS. A console será carregada depois que a sequência de inicialização principal do GlassFish Server tiver sido concluída.

Nunca

Nunca carrega a Console de Administração após o DAS. A console é carregada quando é solicitado acesso a ela.

Raiz da Aplicação

O caminho completo do diretório em que residem as aplicações implantadas. O valor default é domain-dir/applications.

Raiz de Log

Identifica onde o arquivos de log do servidor são mantidos. O valor default é domain-dir/logs.

Configuração Regional

Se este campo for deixado em branco, as configurações regionais do host serão utilizadas.

Uma configuração regional é um identificador que especifica uma determinada combinação de idioma e região. Por exemplo, a área geográfica do Inglês Americano é en_US, e do Japonês é ja_JP. Para usar uma configuração regional que não seja o inglês, o GlassFish Server deve ser localizado, um processo que inclui a tradução de inglês para outro idioma.
