配置管理程式特性

階段作業管理員提供各式方法，以配置建立與銷毀階段作業的方式、階段作業狀態的儲存位置，以及可用的最大階段作業數。

  1. 在瀏覽樹狀結構中，展開 [配置] 節點。
  2. 在 [配置] 節點下，選取 [Web 容器] 節點。

    [Web 容器] 頁面會隨即開啟。

  3. 在 [Web 容器] 頁面上，按一下 [管理員特性] 標籤。
  4. 在 [清除間隔] 欄位中，鍵入在刪除儲存區內非使用中的階段作業資料之前，需等待的秒數。
  5. 在 [最大階段作業數] 欄位中，鍵入允許的最大階段作業數目。
  6. 在 [階段作業檔案名稱] 欄位中，鍵入包含階段作業資料的檔案名稱。
  7. 在 [階段作業 ID 產生器類別名稱] 欄位中，指定用於產生唯一的階段作業 ID 的自訂類別。

    此類別必須位於 Enterprise Server 類別路徑中。一個伺服器實例僅允許一個階段作業 ID 產生器類別。叢集中的所有實例必須使用相同的階段作業 ID 產生器，以避免階段作業金鑰衝突。自訂階段作業 ID 產生器類別必須實作 com.sun.enterprise.util.uuid.UuidGenerator 介面：

    package com.sun.enterprise.util.uuid;

public interface UuidGenerator {

    public String generateUuid();
    public String generateUuid(Object obj);  //obj is the session object
}
  8. (可選擇) 在 [其他特性] 區段中，指定其他特性。
  9. 按一下 [儲存]。
