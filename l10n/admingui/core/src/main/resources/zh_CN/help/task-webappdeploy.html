部署 Web 应用程序

  1. 在导航树中，展开“应用程序”节点。
  2. 在“应用程序”节点下，选择“Web 应用程序”节点。

    此时将打开“Web 应用程序”页面。

  3. 在“已部署的 Web 应用程序”表中，单击“部署”按钮。

    此时将打开“部署企业应用程序/模块”页面。

  4. 从“类型”下拉式列表中，选择“Web 应用程序”(.war)。
  5. 指定包含要部署的应用程序的归档位置。
    • 如果此归档是位于客户机上或者可从客户机访问的文件，请选择“要上载到服务器的打包文件”选项。

      客户机是可通过浏览器查看管理控制台的主机。

      单击“浏览”以找到文件，或键入文件的完整路径。

    • 如果此归档是位于服务器计算机上的文件，或者是来自展开目录的未打包的应用程序，请选择“可从 Application Server 访问的本地打包文件或目录”选项。

      服务器计算机是运行域管理服务器 (Domain Administration Server, DAS) 的主机。

      键入文件或目录的完整路径名。从展开的目录部署适用于高级开发者，建议不要在生产环境中执行此操作。

  6. 在“应用程序名称”字段中，保留默认名称（即文件名的前缀），或键入其他名称。

    如果选择上载文件，则将显示默认名称。应用程序名称必须唯一。

  7. 在“上下文根目录”字段中，输入能够标识 Web 应用程序的字符串。

    在 Web 应用程序的 URL 中，上下文根目录要紧跟着端口号 (http://host: port/context-root/...)。上下文根目录必须以正斜杠开头，例如：/hello。

  8. 要禁用应用程序以便其不可用，请取消选中“启用”复选框。

    默认情况下，只要一部署应用程序，其状态便为“启用”。

  9. 选中“启用运行验证器”复选框，以便在部署之前验证文件的结构和内容。

    大型应用程序的检验会很耗费时间。如果怀疑文件已损坏或不可移植，请检验文件。

  10. 选中“启用预编译 JSP”复选框，以对 JavaServer PagesTM (JSPTM) 文件进行预编译。

    如果未选中此复选框，则在运行时首次访问 JSP 文件时编译这些文件。由于编译通常很耗费时间，因此在生产环境中请选中此复选框。

  11. 在“库”字段中，列出应用程序所需的所有库 JAR 文件。
  12. 在“说明”字段中，键入此 Web 应用程序的说明。
  13. 单击“确定”。
