Para configurar el servicio de transacción

  1. En el árbol de navegación, amplíe el nodo Configuración.

  2. Seleccione la instancia que va a configurar:

    • Para configurar una instancia determinada, seleccione el nodo de configuración de la instancia. Por ejemplo, para la instancia por defecto server, seleccione el nodo server-config.

    • Para configurar los valores por defecto para todas las instancias, seleccione el nodo default-config.

  3. Seleccione el nodo Servicio de Transacción.

    Se abrirá la página Servicio de Transacción.

  4. En la página Servicio de Transacción, active la casilla de control Al Reiniciar para activar la recuperación de transacciones incompletas cuando se reinicie el servidor.

    Esta opción se activada por defecto.

  5. En el campo Timeout de Transacción, escriba el número de segundos que el servidor esperará antes de realizar un rollback de una transacción que no se ha completado.

    El valor 0 desactiva los timeouts de transacciones. El valor por defecto es cero, lo que supone que el servidor espera indefinidamente para que se complete la transacción.

  6. En el campo Timeout de Reintentos, introduzca el número de segundos que GlassFish Server intenta conectarse a un servidor inaccesible.

    El valor por defecto es 600 (10 minutos).

  7. En el campo Ubicación del Log de Transacciones, introduzca el directorio donde se guardan los logs del servidor.

    Los logs de transacciones se guardan en el subdirectorio tx del directorio especificado en este campo. El valor por defecto es el directorio especificado en el campo Raíz de Log de la página Atributos de Dominio, que está ubicada en el separador Avanzado del nodo GlassFish Server.

  8. En la lista desplegable Decisión Heurística, seleccione la política que debe seguirse si el resultado de una transacción distribuida no se puede determinar en caso de que otros participantes no estén disponibles.

    A menos que exista una buena razón para seleccionar Confirmar, deje la opción Decisión Heurística en Realizar Rollback (el valor por defecto). La confirmación de transacciones indeterminadas puede poner en peligro la integridad de los datos de la aplicación.

  9. En el campo Intervalo de Punto Clave, escriba el número de transacciones entre las operaciones de puntos clave.

    Las operaciones de puntos clave comprimen el archivo log de transacciones. El valor por defecto es 65536.

  10. En la sección Propiedades Adicionales, especifique las propiedades adicionales.

    Para obtener una descripción de las propiedades disponibles, consulte Propiedades específicas para el servicio de transacción.

    • Para agregar una propiedad, haga clic en el botón Agregar Propiedad. En la fila en blanco que aparece, introduzca el nombre de propiedad en el campo Nombre y escriba el valor de propiedad en el campo Valor.

    • Para modificar una propiedad, edite el campo Valor.

    • Para suprimir una propiedad, active la casilla de control a la izquierda del campo Nombre de la propiedad que está suprimiendo y, a continuación, haga clic en el botón Suprimir Propiedades.

  11. Haga clic en Guardar.

    Sugerencia:

    Para volver a la configuración por defecto, haga clic en el botón Cargar Valores por Defecto y, a continuación, en Guardar.

  12. Reinicie GlassFish Server.
