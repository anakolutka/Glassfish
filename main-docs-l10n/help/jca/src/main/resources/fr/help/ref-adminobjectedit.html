Modifier une ressource d'objet d'administration

La page Modifier une ressource d'objet d'administration permet de modifier les paramètres d'une ressource d'objet administré.

Elle contient les options ci-après.

Charger les valeurs par défaut

Bouton permettant de restaurer les valeurs par défaut des paramètres qui en ont. Les paramètres qui n'ont pas de valeurs par défaut ne sont pas modifiés.

Nom JNDI

Nom unique identifiant la ressource d'objet administré. Le champ Nom JNDI est en lecture seule. Vous ne pouvez spécifier le nom JNDI qu'à la création d'une ressource d'objet administré.

Adaptateur de ressources

Adaptateur de ressources pour la ressource d'objet administré.

Type de ressource

Type Java de la ressource d'objet administré (par exemple, javax.jms.Topic).

Nom de classe

Nom de la classe d'implémentation associée au type de ressource.

Ordre de déploiement

Ordre de déploiement de la ressource d'objet administré.

Description

Description de la ressource d'objet administré.

Statut

Indique si la ressource est disponible lors de l'exécution.

Si seule l'instance de serveur par défaut server, existe dans le domaine, la disponibilité d'exécution de la ressource peut être activée ou désactivée. La disponibilité d'exécution est activée par défaut.

Si des clusters ou des instances autonomes ont été créés dans le domaine, utilisez l'onglet Cible pour contrôler la disponibilité d'exécution de la ressource sur ses cibles.

Autres propriétés

Propriétés supplémentaires de la ressource d'objet administré. Les propriétés disponibles dépendent de l'adaptateur de ressources du pool de connexions de connecteur. Les paires nom-valeur spécifiées dans cette table peuvent être utilisées pour remplacer les valeurs par défaut des propriétés définies par le fournisseur de l'adaptateur de ressources.
