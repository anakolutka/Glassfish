가상 서버 편집

아이디

내부적으로 표시되는 가상 서버 식별자입니다. HTTP 클라이언트에는 노출되지 않습니다. HTTP 클라이언트에 노출되는 호스트 이름은 호스트 필드에서 지정해야 합니다.

호스트

서버가 실행되고 있는 시스템의 호스트 이름입니다. 네트워크의 DNS 서버(UNIX 시스템에서는 /etc/hosts 파일)에 등록된 실제 또는 가상 호스트 이름을 사용합니다.

상태

서버에 대해 원하는 상태를 선택합니다. 기본값은 ON입니다.

HTTP 수신기

이 필드는 HTTP 수신기를 만들어 이 서버와 연결할 때 자동으로 채워집니다.

이 필드를 사용하려면 기존 HTTP 수신기를 지정해야 합니다. 하지만 다른 가상 서버가 사용하는 수신기를 지정하지 않아야 합니다. 그렇지 않으면 서버를 다시 시작할 때 서버 로그에 오류가 나타납니다. 수신기는 만들 때 기존 가상 서버와 연결되어야 하므로 기존 수신기는 모두 다른 가상 서버에서 사용됩니다.

기본 웹 모듈

드롭다운 목록에서 웹 모듈을 선택합니다. 가상 서버에 배포된 다른 웹 모듈에 매핑될 수 없는 모든 요청에 응답할 배포된 웹 모듈(있는 경우)을 선택합니다. 기본 웹 모듈이 지정되지 않은 경우 빈 컨텍스트 루트가 있는 웹 모듈이 사용됩니다. 빈 컨텍스트 루트가 있는 웹 모듈이 없을 때는 시스템 기본 웹 모듈이 만들어져 사용됩니다.

로그 파일

로깅 메시지를 기본 서버 로그에 보내려면 이 로그 파일 필드를 비워 둡니다. 그렇지 않으면 이 가상 서버의 로깅 메시지가 나타날 파일의 경로 이름을 입력합니다.

Docroot

서버에 대한 루트 문서 디렉토리의 절대 경로입니다. 기본값은 domain-dir/docroot입니다.

SSO

사용하도록 설정된 경우 사용자를 가상 서버의 모든 응용 프로그램에 별도로 인증해야 합니다. 기본값은 사용 안 함입니다.

액세스 로깅

액세스 로깅이 HTTP 서비스에서 상속되는 경우 지정합니다(사용/사용 안 함).

디렉토리

서버 액세스 로그의 절대 디렉토리 경로입니다. 기본값은 domain-dir/logs/access입니다.

버퍼 크기

0보다 작거나 같은 값은 버퍼링이 비활성화됩니다. 이 필드가 비어 있는 경우 액세스 로그 버퍼 크기는 HTTP 서비스에서 상속됩니다.

쓰기 간격

액세스 로그가 업데이트되기 전까지의 시간(초)입니다. 0 값을 지정하면 버퍼링이 비활성화됩니다.

추가 등록 정보

가상 서버에 대한 추가 등록 정보입니다. 다음 등록 정보가 제공됩니다.

sso-max-inactive-seconds

클라이언트 작업이 수신되지 않을 경우 사용자의 단일 사인 온 레코드를 제거할 수 있는 경과 시간(초)을 지정합니다. 단일 사인 온은 같은 가상 서버에 있는 여러 응용 프로그램에 적용되므로 응용 프로그램 중 하나에 액세스하면 단일 사인 온 레코드가 활성 상태를 유지합니다.

기본값은 300초(5분)입니다. 이 값이 높을수록 사용자에 대한 단일 사인 온 지속 시간이 길어지지만 서버의 메모리 사용량은 많아집니다.

sso-reap-interval-seconds

만료된 단일 사인 온 레코드의 제거 간격(초 단위)을 지정합니다.

기본값은 60입니다.

allowLinking

true이면 기호 링크인 자원이 이 가상 서버에 배포된 모든 웹 응용 프로그램에 대해 작동합니다. sun-web.xml 파일의 sun-web-app 등록 정보 allowLinking을 사용하여 개별 웹 응용 프로그램이 이 설정을 대체할 수 있습니다.

<sun-web-app>
   <property name="allowLinking" 
      value="{true|false}"/>
</sun-web-app>

기본값은 false입니다.
