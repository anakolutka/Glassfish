Proveedores JACC

Utilice la página Proveedores JACC para configurar los proveedores de Java Authorization Contract for Containers (JACC) para la instancia o cluster de GlassFish Server seleccionado.

Los proveedores JACC los pueden utilizar módulos de autorización de terceros para aplicaciones que se ejecuten en GlassFish Server. La instalación por defecto de GlassFish Server incluye dos proveedores JACC denominados default y simple. Estos proveedores por defecto implantan un motor de autorización simple, basado en archivos, que cumple la especificación JACC. También puede crear sus propios proveedores JACC personalizados. Consulte Crear un proveedor JACC.

Para obtener más información sobre cómo escribir clases de implantación para un proveedor JACC, consulte la especificación JACC en http://jcp.org/en/jsr/detail?id=115.

Se proporciona la siguiente información para cada proveedor JACC.

Nombre

Nombre del proveedor JACC. Este nombre se utiliza para hacer referencia al elemento jacc-provider en domain.xml.

Configuración de políticas

Nombre de clase totalmente cualificado de javax.security.jacc.PolicyConfigurationFactory.provider que implanta el javax.security.jacc.PolicyConfigurationFactory específico del proveedor.

Proveedor de política

Nombre de clase totalmente cualificado de javax.security.jacc.policy.provider que implanta java.security.Policy.

Los proveedores JACC por defecto también soportan la siguiente propiedad.

repository

Directorio que contiene el archivo de política JACC. Para el proveedor JACC de GlassFish Server default, el directorio por defecto es ${com.sun.aas.instanceRoot}/generated/policy. Esta propiedad no está definida por defecto para el proveedor JACC de GlassFish Server simple.

La tabla Proveedores JACC también contiene las siguientes opciones.

Nuevo

Botón para crear un nuevo proveedor JACC.

Suprimir

Botón para suprimir uno o más proveedores JACC seleccionados.
