Para crear un servicio de ejecutor programado gestionado

Las aplicaciones utilizan un servicio de ejecutor programado gestionado para ejecutar tareas enviadas de forma asíncrona a determinadas horas. Las tareas se ejecutan en threads que inicia y gestiona el contenedor. El contexto del contenedor se propaga al thread que ejecuta la tarea.

En GlassFish Server, puede configurar un recurso de servicio del ejecutor programado gestionado y ponerlo a disposición de componentes de aplicación, como servlets y EJB. Las aplicaciones implantan mediante programación la programación real de tareas.

  1. En el árbol de navegación, amplíe el nodo Recursos.

  2. En el nodo Recursos, amplíe el nodo Recursos simultáneos.

  3. En el nodo Recursos simultáneos, seleccione Servicios de ejecutor programados gestionados.

    Se abrirá la página Servicios de ejecutor programados gestionados.

  4. En la página Servicios de ejecutor programados gestionados, haga clic en Nuevo.

    Se abrirá la página Nuevo servicio de ejecutor programado gestionado.

  5. En el campo Nombre de JNDI, introduzca un nombre único para identificar el recurso.

    El nombre de JNDI se registra en el servicio de nomenclatura de JNDI y lo utilizan las aplicaciones para consultar el recurso. Utilice el formato concurrent/nombre. Por ejemplo, concurrent/miServicioEjecutorProgramadoGestionado. El nombre puede contener únicamente caracteres alfanuméricos, guiones bajos, rayas y puntos.

  6. Active la casilla de control Información de contexto activada para propagar contextos a los threads y seleccione los contextos que desea propagar.

    Esta opción está activada y todos los contextos están seleccionados por defecto. Utilice la tecla Mayús o la tecla Ctrl y haga clic para seleccionar varios contextos. Si esta opción está desactivada, los contextos no se propagan, aunque estén seleccionados.

  7. Active la casilla de control Estado activado para activar el recurso.

    Esta opción está activada por defecto, lo que significa que el recurso está disponible inmediatamente después de crearlo.

  8. En el campo Prioridad de threads, introduzca la prioridad que desea asignar a los threads creados.

    El valor por defecto es 5.

  9. Active la casilla de control Tareas de larga ejecución activadas para utilizar el recurso para ejecutar tareas de larga ejecución.

    Si está activada, las tareas de larga ejecución no se incluirán en el informe como paradas. Esta opción se activada por defecto.

  10. En el campo Bloquear después de, introduzca el número de segundos que se pueden ejecutar las tareas antes de que se considere que no responden.

    El valor por defecto es 0, lo que significa que nunca se considera que las tareas no responden.

  11. En el campo Descripción, introduzca una breve descripción del recurso.

    Esta descripción es un valor de cadena y puede incluir un máximo de 250 caracteres.

  12. En el campo Tamaño principal, introduzca el número de threads que se deben mantener en un pool de threads, incluso si están inactivos.

    El valor por defecto es 0, lo que significa que se crea un thread cuando se programa la primera tarea.

  13. En el campo Mantenimiento de conexiones, introduzca el número de segundos que los threads puede permanecer inactivos cuando el número de threads es mayor que el tamaño principal.

    El valor por defecto es 60.

  14. En el campo Duración de threads, introduzca el número de segundos que los threads pueden permanecer en el pool de threads antes de que se depuren, independientemente de si el número de threads es mayor que el tamaño principal o si están inactivos.

    El valor por defecto es 0, lo que significa que los threads nunca se depuran.

  15. En la sección Propiedades adicionales, no agregue ninguna propiedad adicional.

    GlassFish Server no define ninguna propiedad adicional para este tipo de recurso. Además, este tipo de recurso no utiliza actualmente ninguna propiedad adicional.

  16. Haga clic en Aceptar.
