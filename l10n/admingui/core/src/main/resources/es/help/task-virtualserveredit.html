Para editar un servidor virtual

  1. En el árbol de navegación, expanda el nodo Configuración.
  2. En el nodo Configuración, expanda el nodo Servicio HTTP.
  3. En el nodo Servicio HTTP, seleccione el nodo Servidores virtuales.
  4. En la página Servidores virtuales, haga clic en el nombre del servidor virtual que desee editar.

    Se abrirá la página Editar servidor virtual.

  5. El campo Id. no se puede editar.
  6. En el campo Hosts, introduzca los nombres de host o las direcciones IP del equipo en el que se está ejecutando el servidor.

    Utilice nombres de host reales o virtuales que estén registrados en el servidor DNS de la red (y, en un sistema UNIX, en el archivo /etc/hosts ).

  7. Seleccione una opción de estado para el servidor virtual.

    El servidor virtual está activo de forma predeterminada.

  8. Deje el campo Escuchas HTTP vacío.

    Este campo se rellena automáticamente cuando se crea una escucha HTTP y se asocia a este servidor.

    El uso de este campo precisa que especifique una escucha HTTP existente. Sin embargo, no debe especificar una escucha que utilice otro servidor virtual; de lo contrario, se mostrará un error en el registro del servidor cuando reinicie el servidor. Dado que es preciso asociar una escucha a un servidor virtual cuando se crea, todas las escuchas existentes son utilizadas por otro servidor virtual.

  9. (Opcional) En la lista desplegable Módulo web predeterminado, seleccione el módulo que va a responder a todas las solicitudes que no se pueden asignar a otros módulos web implementados en el servidor virtual.

    Si no se especifica un módulo web predeterminado, se utiliza un módulo web con una raíz de contexto vacía. Si no hay ningún módulo web con una raíz de contexto vacía, se crea y utiliza un módulo web predeterminado del sistema.

  10. (Opcional) En el campo Archivo de registro, introduzca el nombre de ruta del archivo en el que se guardarán los mensajes de registro de este servidor virtual.

    Si se deja este campo en blanco, los mensajes de registro se enviarán al registro de servidor predeterminado, domain-dir/logs/server.log.

  11. (Opcional) En el campo Raíz de documentos, introduzca la ruta absoluta al directorio raíz de documentos del servidor.

    Si el campo se deja en blanco, se utilizará domain-dir/docroot.

  12. Marque la casilla de verificación SSO habilitado para habilitar el inicio de sesión único en este servidor virtual.

    Esta opción se encuentra deshabilitada de forma predeterminada. Si el inicio de sesión único está deshabilitado, los usuarios deben autenticar de forma independiente en cada aplicación del servidor virtual.

    Los servlets y los archivos JSP admiten el inicio de sesión único en las aplicaciones de Servidor empresarial. Esta función permite a varias aplicaciones que requieren la misma información de inicio de sesión compartir esta información de modo que el usuario no tenga que iniciar una sesión por separado para cada aplicación.

  13. Seleccione la opción Registro de acceso para habilitar o deshabilitar el registro de acceso.

    También puede especificar si el registro de acceso debe heredarse del servicio HTTP.

  14. En el campo Directorio, introduzca la ruta de directorio absoluta en los registros de acceso del servidor.

    El valor predeterminado es domain-dir/logs/access .

  15. (Opcional) En el campo Tamaño del búfer, introduzca el tamaño del búfer, en bytes.

    Un valor igual o menor que cero desactiva el búfer. Si deja vacío el campo, el tamaño del búfer del registro de acceso se heredará del servicio HTTP.

  16. (Opcional) En el campo Intervalo de escritura, introduzca el intervalo, en segundos, entre la actualización de los registros de acceso.
  17. (Opcional) En la sección Propiedades adicionales, especifique propiedades adicionales del servidor virtual.

    Independientemente de si se especifican propiedades adicionales o no, las siguientes propiedades se establecen como valores predeterminados:

    • docroot

    • accesslog

    Están disponibles las siguientes propiedades de servidores virtuales:

    sso-max-inactive-seconds

    Especifica el número de segundos después de los cuales el registro de inicio de sesión único de un usuario cumple con los requisitos necesarios para depurarse si no se detecta actividad en el cliente. Dado que el inicio de sesión único se aplica en varias aplicaciones en el mismo servidor virtual, el acceso a cualquiera de las aplicaciones mantiene activo el registro de inicio de sesión único.

    El valor predeterminado es 300 segundos (5 minutos). Los valores más altos proporcionan una persistencia de inicio de sesión único más larga para los usuarios a costa de un mayor consumo de memoria en el servidor.

    sso-reap-interval-seconds

    Especifica el número de segundos entre depuraciones de registros de inicio de sesión único caducados.

    El valor predeterminado es 60.

    allowLinking

    Si está propiedad es true, se atenderán los recursos que son vínculos simbólicos para todas las aplicaciones web implementadas en este servidor virtual. Las aplicaciones Web individuales pueden reemplazar esta configuración utilizando la propiedad allowLinking de sun-web-app en el archivo sun-web.xml:

    <sun-web-app>
   <property name="allowLinking" 
      value="{true|false}"/>
</sun-web-app>

    El valor predeterminado es false.

  18. Haga clic en "Guardar".
