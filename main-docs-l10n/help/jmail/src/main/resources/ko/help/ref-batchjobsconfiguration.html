일괄 처리 런타임 구성

GlassFish Server에서는 일괄 처리 작업의 일정 잡기 및 실행을 위해 일괄 처리 런타임을 제공합니다. 일괄 처리 작업은 대개 여러 단계가 포함된, 오랫동안 실행되는 대량 작업입니다. 일괄 처리 응용 프로그램에서는 일괄 처리 런타임에 작업을 제출하고, 각 단계를 실행하는 방법 및 시기에 대한 지침을 제공합니다. 일괄 처리 런타임은 각 단계를 처리하고 작업에 대한 정보를 작업 저장소에 저장합니다. GlassFish Server에서 작업 저장소는 데이터베이스입니다.

[일괄 처리 런타임 구성] 페이지를 사용하여 일괄 처리 런타임의 구성을 보고 변경합니다. 일괄 처리 런타임은 관리 대상 실행기 서비스 및 데이터 소스를 사용하여 일괄 처리 작업을 실행합니다. 관리 대상 실행기 서비스는 작업에 스레드를 제공하며, 데이터 소스는 작업 정보를 저장합니다. 일괄 런타임 구성 데이터는 domain.xml의 config 요소에 저장됩니다.

[일괄 처리 런타임 구성] 페이지에는 다음 정보가 포함되어 있습니다.

인스턴스 이름
클러스터 이름

구성이 적용되는 GlassFish 클러스터 또는 서버 인스턴스 이름입니다. 읽기 전용 필드입니다.

실행기 서비스 조회 이름

일괄 처리 작업에 스레드를 제공하는 관리 대상 실행기 서비스의 JNDI 조회 이름입니다. 기본 관리 대상 실행기 서비스는 concurrent/__defaultManagedExecutorService입니다.

일괄 처리 작업이 일괄 처리 런타임에 제출된 후 작업 실행에 영향을 주지 않고 관리 대상 실행기 서비스를 변경할 수 있습니다.

데이터 소스 조회 이름

현재 및 과거 일괄 처리 작업에 대한 정보를 저장하는 데이터 소스의 JNDI 조회 이름입니다. 기본 데이터 소스는 jdbc/__TimerPool입니다.

첫번째 일괄 처리 작업이 실행을 위해 일괄 처리 런타임에 제출된 후에는 데이터 소스를 변경하지 마십시오. 데이터 소스를 변경해야 한다면 도메인을 정지했다가 재시작한 후 어떤 작업도 시작되거나 재시작되기 전에 변경 작업을 수행하십시오. 일단 데이터 소스가 변경되면 이전 데이터 소스에 저장된 정보에 액세스할 수 없습니다.
