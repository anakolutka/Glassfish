Per verificare se un nodo è raggiungibile

Per creare o avviare le istanze di GlassFish Server su un nodo che rappresenta un host remoto, il server DAS deve essere in grado di contattare il nodo. Prima di tentare la creazione o l'avvio di istanze su tale nodo, verificare che il server DAS possa contattare il nodo.

Il tipo del nodo per il quale si esegue il test deve essere DCOM o SSH. Non è possibile eseguire il test su un nodo di tipo CONFIG poiché i nodi CONFIG non sono abilitati per le comunicazioni remote.

Prima di iniziare

Assicurarsi che sull'host rappresentato dal nodo sia configurato il protocollo appropriato per il tipo di nodo. Il nodo viene abilitato per le comunicazioni remote su questo protocollo.

  • Se il nodo è di tipo DCOM, è necessario configurare il protocollo DCOM (Distributed Component Object Model).

  • Se il nodo è di tipo SSH, è necessario configurare il protocollo SSH (Secure Shell).

    1. Selezionare il nodo Nodi nell'albero di navigazione.

      Viene visualizzata la pagina Nodi. Nella pagina sono elencati tutti i nodi del dominio corrente.

    2. Nella tabella dei nodi, seguire il collegamento Ping per il nodo su cui si sta eseguendo il test.

      Un messaggio che indica se il server DAS è riuscito a contattare il nodo viene aggiunto alla pagina Nodi.
