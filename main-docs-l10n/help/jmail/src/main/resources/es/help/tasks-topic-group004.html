Para configurar el tiempo de ejecución por lotes

  1. En el árbol de navegación, seleccione el cluster o la instancia de servidor de GlassFish Server para el que desea ver las ejecuciones de trabajo por lotes.

    Se abrirá la página Información General.

  2. Seleccione el separador Lote.

    Se abrirá la página Ejecuciones de trabajo por lotes.

  3. Haga clic en el separador Configuración.

    Se abrirá la página Configuración de tiempo de ejecución por lotes. El tiempo de ejecución utiliza un servicio de ejecutor gestionado y un origen de datos para ejecutar los trabajos por lotes.

  4. En la lista desplegable Nombre de la búsqueda de servicio de ejecutor, seleccione el servicio de ejecutor gestionado que desea utilizar para proporcionar threads a los trabajos por lotes.

    El servicio de ejecutor gestionado por defecto es concurrent/__defaultManagedExecutorService. El servicio de ejecutor gestionado se puede cambiar una vez que el trabajo por lotes se ha ejecutado en el tiempo de ejecución por lotes sin afectar a la ejecución del trabajo.

  5. En la lista desplegable Nombre de la búsqueda de origen de datos, seleccione el origen de datos origen que desea utilizar para almacenar información sobre el trabajo.

    El origen de datos por defecto es jdbc/__TimerPool. No cambie el origen de datos después de que se haya ejecutado el primer trabajo por lotes en el tiempo de ejecución por lotes para su ejecución. Si debe abrir el origen de datos, pare y reinicie el dominio y, a continuación, realice el cambio antes de que se inicie o reinicie cualquier trabajo. Sin embargo, una vez que el origen de datos se ha cambiado, ya no se podrá acceder a la información almacenada en el origen de datos anterior.

  6. Haga clic en Guardar.
