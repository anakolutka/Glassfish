GlassFish V3 Technology Preview Quick Start Guide

This guide describes basic steps for using the GlassFish V3 technology preview app server. You must have already installed the technology preview to use this Quick Start Guide. For download and installation instructions, see the technology preview download page.

This document contains information about the following topics:

  • Configuring Containers

  • Starting the Server

  • Deploying Applications

  • Undeploying Applications

  • Listing Deployed Applications

  • Where to Go Next

  • Getting Help

For information about conventions used in this document, see Documentation Conventions.

Note: To use the asadmin commands described below, open a command window and adjust your path to include install-dir/bin. For example:

Windows

 
set path=install-dir/bin;%path%

UNIX® sh

 
PATH=install-dir/bin:$PATH



On Unix-style systems change the asadmin script to allow execute access, for example:

chmod +x install-dir/bin/asadmin

Starting the Server

  1. Open a command window and change the default directory to install-dir. Make sure the JDK's bin directory is in your path.

  2. Run

    java -jar glassfish/lib/glassfish-10.0-SNAPSHOT.jar

Expect to see several informational messages in the console output as the server starts up, ending with a line like this

Glassfish v3 started in xxx ms

The server will run in that current process, with its output sent to the command window.

Deploying Applications

Note: The format of asadmin commands is different in V3 from the V2 format.

You deploy applications by using the asadmin command. Use

asadmin deploy --path file-path-to-the-application

You will see output showing that asadmin has converted the command into an http request to the server to request the deployment. If the deployment succeeds you will see a message like

SUCCESS : Application app-name deployed successfully

You will also see some log messages in the server's output window.

V3 will use the file name (without the file type) as the default name of the application. You can override that value using the --name option.

Undeploying Applications

To undeploy an application use

asadmin undeploy --name app-name

where <app-name> was the display during the successful deployment earlier. You should see a “success” message.

Listing Deployed Applications

Use

asadmin list-applications

Documentation Conventions

This section describes the types of conventions used throughout this guide.

General Conventions

The following general conventions are used in this guide:

  • File and directory paths are given in UNIX® format (with forward slashes separating directory names). For Windows versions, the directory paths are the same, except that backslashes are used to separate directories.

  • Installation root directories are indicated by install-dir in this document.
