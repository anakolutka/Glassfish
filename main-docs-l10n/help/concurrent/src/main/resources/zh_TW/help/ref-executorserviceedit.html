編輯受管理執行程式服務

您可以使用「編輯受管理執行程式服務」頁面來編輯受管理執行程式服務資源。

「編輯受管理執行程式服務」頁面包含下列選項。

載入預設值

用於將具有預設值的設定復原為設定值的按鈕。沒有預設值的設定則保持不變。

JNDI 名稱

可識別資源的唯一 JNDI 名稱。依慣例會是以資源類型指示符與斜線開頭的名稱，例如 concurrent/、jdbc/ 或 jms/。「JNDI 名稱」是唯讀欄位。您僅能在建立新資源時指定 JNDI 名稱

邏輯 JNDI 名稱

資源的邏輯 JNDI 名稱。這是可用來存取資源的 Java EE 標準名稱。邏輯 JNDI 名稱是唯讀欄位。請參閱 Logical JNDI Names 以瞭解詳細資訊。

環境資訊

要傳輸至繫線的環境：Classloader、JNDI、Security 或 WorkArea。您可以啟用或停用環境傳輸。如果啟用，就會傳輸選取的環境。如果停用，就不會傳輸任何環境 (包括選取的環境)。預設值為「啟用」(包括全部的四個環境)。

狀態

是否可在執行階段使用資源。

如果網域中只有預設伺服器執行處理 server 存在，可以選擇啟用或停用資源的執行階段可用性。預設會啟用執行階段可用性。

如果已在網域中建立任何叢集或獨立執行處理，可使用「目標」頁籤控制其目標之資源的執行階段可用性。

繫線優先順序

要指定給已建立繫線的優先順序。預設值為 5。

長時間執行的工作

是否應該將資源使用於長時間執行的工作。如果啟用，就不會將長時間執行的工作報告為阻塞。此選項預設為停用。

停滯前的時間

工作可以執行的秒數，超過此時間之後即將工作視為無回應。預設值為 0。

建置順序

此資源的建置順序。系統會在伺服器啟動時先載入數字較小的資源。預設值為 100。

描述

此資源的描述。

核心大小

要保留在繫線集區中的繫線數目。預設值為 0。

集區的最大大小

繫線集區可以包含的繫線上限數目。預設值為 2147483647。

持續作用時間

當繫線數目大於核心大小時，繫線可以維持閒置的秒數。預設值為 60。

繫線存留時間

繫線在被清除之前可保留在繫線集區中的秒數 (不論繫線數目是否大於核心大小或繫線是否為閒置狀態)。預設值為 0。

作業佇列容量

工作佇列中可儲存之等待執行的已提交工作數目。預設值為 2147483647。

其他特性

資源的其他特性。GlassFish Server 並未定義此資源類型的任何其他特性。此外，此資源類型目前也並未使用任何其他特性。
