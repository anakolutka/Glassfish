組態 GlassFish Server 如何復原作業事件

當伺服器當機或當資源管理員在 2 階段確認的階段期間當機時，均可能導致作業事件不完整。完成這些中斷的作業事件並將其從故障中恢復至關重要。GlassFish Server 可在伺服器啟動時從這些故障中復原並完成作業事件。

在復原之前，請確定所有相關資源均已啟動並在執行中。

如果作業事件跨伺服器進行，啟動此作業事件的伺服器會連絡其他伺服器以獲得作業事件的結果。如果無法訪問其他伺服器，則該作業事件將使用「啟發式決策」欄位來確定結果。

  1. 在瀏覽樹狀結構中，展開「組態」節點。

  2. 選取要組態的執行處理：

    • 若要組態特定的執行處理，請選取該執行處理的組態節點。例如，對於預設執行處理 server，請選取「server-config」節點。

    • 若要組態所有執行處理的預設設定，請選取「default-config」節點。

  3. 選取「作業事件服務」節點。

    「作業事件服務」頁面會隨即開啟。

  4. 在「作業事件服務」頁面上，選取「啟用在重新啟動時」核取方塊，以復原未完成的作業事件。

    依預設，不會選取「已啟用」核取方塊。

  5. 在「重試逾時」欄位中，鍵入 GlassFish Server 嘗試連線至無法連線之伺服器的秒數。

    預設值為 600 (10 分鐘)。

  6. 從「啟發式決策」下拉式清單中，選取作業事件中對於無法連線之伺服器的策略。

    除非有很好的理由選取「確認」，否則請將「啟發式決策」保留設定為「倒回」(預設值)。確認不確認的作業事件會破壞應用程式的資料完整性。

  7. 按一下「儲存」。

  8. 重新啟動 GlassFish Server。
