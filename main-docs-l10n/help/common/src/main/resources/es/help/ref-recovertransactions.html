Recuperar transacciones

Utilice la página Recuperar transacciones para recuperar transacciones para una instancia de GlassFish Server.

Si la instancia se está ejecutando, la página Recuperar transacciones sólo contiene el texto La recuperación la realiza el propio servidor en ejecución y el botón Aceptar.

Si la instancia no se está ejecutando, la página Recuperar transacciones contiene las siguientes opciones.

Nombre de instancia

Nombre que se asignó a la instancia en el momento de su creación. Este campo es de sólo lectura.

Servidor de destino

Otra instancia que puede recuperar las transacciones en la instancia actual.

Directorio de log de transacciones

Directorio en el que se almacenan los logs de transacciones si los logs se almacenan en un sistema de archivos compartidos. Esta opción no es aplicable a los logs de transacciones almacenados en una base de datos. El valor por defecto es /logs/tx en el directorio de dominio.

Los logs de transacciones se guardan en el subdirectorio tx del directorio especificado por la configuración de Ubicación de log de transacciones del servicio de transacción para la configuración del cluster. El valor por defecto de Ubicación de log de transacciones es el directorio especificado en el campo Raíz de log de la página Atributos de dominio, que está ubicada en el separador Avanzado del nodo GlassFish Server.

Subcomandos asadmin equivalentes

recover-transactions(1)
