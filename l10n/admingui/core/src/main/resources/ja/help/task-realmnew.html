レルムを作成する

Enterprise Server には、file、certificate、および admin-realm の 3 つのレルムが事前に設定されています。さらに、ldap、JDBC、solaris、またはカスタムレルムも作成できます。通常、サーバーにはタイプによってレルムが 1 つずつありますが、Enterprise Server の場合は、file および admin-realm の 2 つの file レルムがあります。これらは、異なる 2 つの目的に使用される同じタイプの 2 つのレルムです。また、システムの各仮想サーバーで異なる証明書データベースを持つこともできます。

  1. ナビゲーションツリーで、「設定」ノードを展開します。
  2. 「セキュリティー」ノードを開きます。
  3. 「レルム」ノードを選択します。

    「レルム」ページが開きます。

  4. 「レルム」ページで、「新規」をクリックします。

    「新しいレルム」ページが開きます。

  5. 「名前」フィールドで、レルムの名前を入力します。
  6. レルムのクラス名を指定するには、次のいずれかを実行します。
    • 「クラス名」ドロップダウンリストから、作成するレルムのクラス名を選択します。

      選択可能なクラス名を次の表に示します。

      レルム名

      クラス名

      file

      com.sun.enterprise.security.auth.realm.file.FileRealm

      certificate

      com.sun.enterprise.security.auth.realm.certificate.CertificateRealm

      ldap

      com.sun.enterprise.security.auth.realm.ldap.LDAPRealm

      JDBC

      com.sun.enterprise.security.auth.realm.jdbc.JDBCRealm

      solaris

      com.sun.enterprise.security.auth.realm.solaris.SolarisRealm
    • 下側のラジオボタンをオンにし、作成するカスタムレルムのクラス名を入力します。
  7. (省略可能) 「追加プロパティー」セクションで、追加プロパティーを指定します。

    プロパティーを追加するには、「プロパティーを追加」ボタンをクリックします。表示される空白行で、「名前」フィールドにプロパティー名を入力し、「値」フィールドにプロパティー値を入力します。

    有効なプロパティーはレルムのタイプによります。

    • ファイルレルムプロパティーの詳細については、「FileRealm クラスに固有のプロパティー」を参照してください。

    • 証明書レルムプロパティーの詳細については、「CertificateRealm クラスに固有のプロパティー」を参照してください。

    • JDBC レルムプロパティーの詳細については、「JDBCRealm クラスに固有のプロパティー」を参照してください。

    • LDAP レルムプロパティーの詳細については、「LDAPRealm クラスに固有のプロパティー」を参照してください。

    • Solaris レルムプロパティーの詳細については、「SolarisRealm クラスに固有のプロパティー」を参照してください。

  8. 「了解 (OK)」をクリックします。
