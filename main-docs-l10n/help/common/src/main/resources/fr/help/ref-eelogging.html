Paramètres du journaliseur

La page Paramètres du journaliseur permet de configurer la journalisation pour l'instance ou le cluster GlassFish Server sélectionné.

Elle contient les options ci-après.

Nom de configuration

Nom de la configuration à laquelle s'appliquent les paramètres de cette page. Ce champ est en lecture seule.

Ecrire dans le journal système

Lorsque cette option est activée, la sortie de journalisation est envoyée à l'utilitaire syslog en plus du journal du serveur. Cette fonctionnalité est disponible sur les systèmes Solaris et Linux uniquement. Cette option est désactivée par défaut.

Journaliser dans la console

Lorsque cette option est activée, la sortie de journalisation est envoyée à la console système.

Modification de la date de rotation

Lorsque cette option est activée, GlassFish Server change de fichier journal chaque jour à minuit.

Mode multiligne

Lorsque cette option est activée, les messages de journalisation sont mis en forme d'une manière qui rend leur lecture plus facile. Le corps des messages de journalisation est mis en retrait et commence sur une nouvelle ligne.

Format de journalisation de console

Format de journalisation dans la console. Les choix possibles sont ULF (UniformLogFormatter) et ODL (Oracle Diagnostics Logging). La valeur par défaut est ODL.

Format de journalisation LogFile

Format de journalisation dans le fichier journal du serveur. Les options disponibles sont ULF et ODL. La valeur par défaut est ODL.

Limite de rotation des fichiers

Taille en octets que le fichier journal du serveur doit atteindre avant d'être déplacé. Lorsque le journal du serveur atteint la taille indiquée, GlassFish Server crée un fichier vide nommé server.log et renomme l'ancien fichier server.log_date, où date indique la date et l'heure auxquelles le fichier est déplacé. La valeur par défaut est 2 Mo. La valeur minimale est 500 ko. Si vous spécifiez une valeur inférieure, le fichier est déplacé lorsqu'il atteint 500 ko. Pour désactiver la rotation du fichier journal, spécifiez la valeur 0.

Temps limite de rotation des fichiers

Nombre de minutes au bout duquel le journal du serveur doit être déplacé. La valeur par défaut est 0 ; cela signifie que le fichier est déplacé lorsqu'il atteint la taille spécifiée dans le champ Limite de rotation des fichiers. Si vous spécifiez une ou plusieurs minutes, le temps limite remplace la taille limite indiquée par le champ Limite de rotation des fichiers.

Fréquence de vidage

Nombre maximal de messages à écrire simultanément de la file d'attente vers le journal du serveur. La valeur par défaut est 1.

Nombre maximal de fichiers d'historique

Nombre maximal de fichiers journaux que GlassFish Server conserve avant de supprimer le fichier le plus ancien. Si vous définissez cette valeur sur zéro (0), GlassFish Server ne supprime pas les anciens fichiers journaux.

Fichier journal

Autre nom ou emplacement pour le fichier du journal du serveur. L'emplacement par défaut est domain-dir/logs/server.log.

Gestionnaire du journal

Nom de classe absolu d'un gestionnaire de journal personnalisé. Un gestionnaire de journal personnalisé permet d'envoyer des journaux vers une destination autre que server.log ou syslog. Il doit étendre la classe java.util.logging.Handler (API compatible JSR 047). Ajoutez la classe du gestionnaire au classpath de GlassFish Server de façon à installer le gestionnaire au démarrage du serveur.
