HTTP 수신기 편집 탭

HTTP 수신기 편집 소켓을 사용하면 Enterprise Server가 HTTP 요청을 빠르게 처리할 수 있습니다.

이름

수신기에 대한 읽기 전용의 고유한 식별자입니다.

수신기

서버의 유효한 IP 주소를 입력합니다.

보안

클러스터 및 엔터프라이즈 프로필에서 사용할 수 있습니다. 보안이 설정된 수신기를 만듭니다.

네트워크 주소

수신기가 고유한 포트 값을 사용하여 서버에 대한 모든 IP 주소를 수신하도록 하려면 네트워크 주소에 0.0.0.0을 입력합니다.

수신기 포트

네트워크 주소 필드가 0.0.0.0인 경우 고유한 수신기 포트 값을 입력하고, 다른 IP 주소를 사용할 경우 원하는 포트 값을 입력합니다.

기본 가상 서버

드롭다운 목록에서 가상 서버를 선택합니다.

서버 이름

서버가 클라이언트에 전송하는 URL에 사용될 호스트 이름을 입력입니다. 서버가 별칭을 사용하는 경우 이 이름은 별칭 이름입니다. 서버가 별칭을 사용하지 않는 경우에는 이 필드를 비워 두십시오.

포트 리디렉션

포트를 다른 포트로 리디렉션하는 데 사용할 포트 값을 입력합니다. 다음 두 조건이 존재하면 Enterprise Server는 자동으로 요청을 리디렉션합니다.

  • 이 수신기가 비-SSL 요청을 지원하는 경우

  • SSL 전송이 필요한 일치하는 보안 제약 조건에 대한 요청이 수신된 경우

    기본적으로 Enterprise Server는 원래 요청에 지정된 포트 번호를 사용합니다.

억셉터 스레드

억셉터 스레드 수입니다. 기본값은 1입니다.

제공

서블릿 생성 HTTP 응답 헤더에서 X-Powered-By: Servlet/2.4 헤더 포함을 비활성화하려면 확인란의 선택을 해제합니다.

Java Servlet 2.4 사양은 컨테이너가 서블릿 생성 응답에 추가할 수 있는 이 헤더를 정의합니다. 마찬가지로 JavaServer Pages(JSP) 2.0 사양은 JSP 기술을 사용하는 응답에 X-Powered-By: JSP/2.0 헤더가 추가되도록(선택 사항에 따라) 정의합니다. 웹 응용 프로그램의 경우 X-Powered-By: JSP/2.0 헤더의 포함이 기본적으로 활성화되어 있습니다. 이 헤더의 목적은 서블릿 및 JSP 기술의 사용에 대한 통계 데이터를 수집하는 웹 사이트 관리자에게 도움을 주기 위한 것입니다.

JSP 페이지에 대한 X-Powered-By 헤더의 활성화 및 비활성화에 대한 자세한 내용은 GlassFish v3 Application Server Application Deployment Guide의 부록 A, Deployment Descriptor Files를 참조하십시오.

프로덕션 환경에서는 기본 기술을 숨기기 위해 X-Powered-By 헤더의 생성을 생략할 수 있습니다.

차단

수신기 및 외부 포트에 대한 차단을 사용합니다.
