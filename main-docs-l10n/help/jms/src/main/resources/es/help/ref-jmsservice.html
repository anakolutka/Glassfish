Java Message Service

Use la página Java Message Service para configurar los valores que usará el proveedor de JMS por defecto, GlassFish Server Cola de mensajes.

Para obtener más información sobre cómo configurar el servicio JMS, consulte el Capítulo 19 sobre cómo utilizar el Java Message Service, en la Oracle GlassFish Server Application Development Guide.

La página Java Message Service contiene las opciones siguientes.

Cargar Valores por Defecto

Botón para restaurar configuración que tiene valores por defecto a sus valores por defecto. Las configuraciones que no tienen valores por defecto no se modifican.

Ping

Botón para comprobar que el servicio JMS se está ejecutando en el host JMS por defecto. Si el servicio JMS está activo y en ejecución, un mensaje informa que el ping se ha realizado correctamente.

Nombre de Configuración

Nombre de la configuración a la que se aplican los valores de esta página. Este campo es de sólo lectura.

Tipo

Tipo de acceso para el servicio JMS. Las opciones disponibles son:

EMBEDDED

Acceso al servicio JMS en el host local. El servicio JMS se inicia en la misma máquina JVM que GlassFish Server. La inicialización lenta inicia el broker por defecto embebido al acceder por primera vez a los servicios JMS, en lugar de al iniciar GlassFish Server. Éste es el tipo por defecto de servicio JMS.

LOCAL

Acceso al servicio JMS en el host local. El proveedor JMS se inicia junto con GlassFish Server.

REMOTE

Acceso al servicio JMS en otro sistema. Si selecciona REMOTE, GlassFish Server no iniciará el servicio JMS la siguiente vez que se inicie el servidor. En su lugar, el servicio JMS se inicia y se gestiona a través de Message Queue; por tanto, debe iniciar el broker de Message Queue por separado. Para obtener más información sobre cómo iniciar el broker, consulte la sección sobre cómo iniciar el broker de Oracle GlassFish Server Message Queue Administration Guide. Si selecciona este valor y está utilizando un host remoto, siga las instrucciones de la sección Para editar un host JMS para especificar el nombre del host remoto.

Timeout de Inicio

Número de segundos que GlassFish Server esperará para que se inicie el servicio JMS antes de abortar el inicio. Para un sistema lento o sobrecargado, aumente el valor por defecto. El valor por defecto es 60.

Argumentos de Inicio

Argumentos para personalizar el inicio del servicio JMS. Puede utilizar todos los argumentos disponibles a través del comando as-install-parent/mq/bin/imqbrokerd.

Volver a Conectar

Si se activa la casilla de control de activación de nueva conexión, el servicio JMS intentará volver a conectarse a un servidor de mensajes (o a la lista de direcciones de AddressList) cuando se pierda una conexión. Esta opción se encuentra activada por defecto.

Intervalo de Reconexión

Número de segundos entre los intentos de reconexión. Este intervalo se aplica a intentos con cada dirección de la AddressList y direcciones sucesivas de la lista. Si es demasiado corto, este intervalo de tiempo no dará tiempo al broker para la recuperación. Si es demasiado largo, la reconexión puede suponer un retraso inaceptable. El valor por defecto es de 5 segundos.

Intentos de Reconexión

Especifica el número de intentos de conexión (o reconexión) para cada dirección de la AddressList antes de que el tiempo de ejecución del cliente continúe con la siguiente dirección de la lista. El valor -1 indica que el número de intentos de reconexión es ilimitado (el tiempo de ejecución del cliente intenta conectarse con la primera dirección hasta que lo consigue). El valor por defecto es 3.

Host JMS por Defecto

Nombre del host JMS por defecto. El valor por defecto es default_JMS_host.

Broker Maestro

Nombre de la instancia en cluster de GlassFish Server cuyo broker Cola de mensajes asociado se va a usar como broker maestro en el cluster de broker Cola de mensajes. Deje este campo en blanco para activar la designación automática de broker maestro en GlassFish Server.

Si cambia este valor, debe reiniciar el cluster de GlassFish Server que usa la configuración.

Comportamiento de Lista de Direcciones

Orden de intentos de conexión. Las opciones disponibles son:

random

Seleccione una dirección de la lista de direcciones de forma aleatoria. Si hay muchos clientes que intentan una conexión utilizando la misma fábrica de conexiones, especifique RANDOM para evitar que todos se conecten a la misma dirección. Ésta es la opción por defecto.

priority

La reconexión siempre intenta volver a conectarse a la primera dirección de servidor de AddressList y utiliza otra solamente cuando el primer broker no está disponible.

Iteraciones de Lista de Direcciones

Número de veces que servicio JMS repite el recorrido por la lista AddressList en un esfuerzo por establecer (o restablecer) una conexión. El valor -1 indica que el número de intentos es ilimitado. El valor por defecto es 3.

Esquema MQ y Servicio MQ

Nombre de esquema de la dirección de Message Queue y el nombre del servicio de conexión de Message Queue si no se van a utilizar esquemas o servicios por defecto. Consulte Sintaxis de las Direcciones de Esquema MQ y Servicio MQ para obtener más información sobre la sintaxis.

Propiedades Adicionales

Propiedades adicionales del servicio JMS.

Para obtener más información sobre las propiedades disponibles, consulte Propiedades específicas del servicio JMS.
