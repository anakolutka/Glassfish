Guia GlassFish de Início Rápido

Este guia descreve os passos básicos para inicialização do servidor GlassFish, empacotamento e instalação (deploying) de aplicações. Este documneto também fornece informações sobre a Console de Administracao e ferramentas de linha de comando. Você deve instalar o produto para utilizar este Guia de Início Rápido. Para download e instruções de instalação, veja a página de download do GlassFish .

Este documento contém informações a respeito dos seguintes tópicos:

  • Empacotando e Instalando aplicações

  • Usando ferramentas de linha de comando

Inicializando e Verificando o servidor

Esta sessão fornece procedimentos para administradores que desejam iniciar e avaliar o servidor GlassFish:

  • Para iniciar o servidor GlassFish

  • Para confirmar que servidor está rodando

  • Para instalar a aplicação Hello usando autodeploy

  • Para Loggar na Console Administrativa

Para tarefas administrativas, o servidor GlassFish fornece ferramentas que habilitam administradores à manipularem instâncias de servidores:

  • Console Administrativa, uma interface gráfica (GUI) baseada em navegadores (browsers)

  • O utilitárioasadmin, uma ferramente de comando de linha

Os valores padrões de administração são conforme a seguir:

  • domain.name=domain1

  • instance.name=server

  • admin.user=admin

  • admin.password=adminadmin

  • admin.port=4848

  • instance.port=8080

  • orb.port=3700

  • imq.port=7676

  • https.port=8181

Estes valores são usados pelo comando ant -f setup.xml executado na instalação.  Se existir conflito em alguma porta durante o setup, edite o arquivo glassfish\setup.xml ajustando apropriadamente o(s) valore(s) da(s) porta(s) conflitante(s) e re-execute o comando ant.

para iniciar o servidor GlassFish

Inicia-se o servidor a partir da inicialização do domínio (domain). O domínio provê autenticação e administração para instância do servidor que pertence a ele. Ao iniciar o domínio, uma instância do servidor é executada para o mesmo. O domínio padrão denominado domain1 é criado automaticamente quando é instalado o servidor GlassFish.

Passos

  1. Adicione o diretório install-dir/bin/ na variável de ambiente PATH.

  2. Inicie o servidor a partir do seguinte comando: asadmin start-domain domain1.

    Quando o servidor estiver iniciado, a seguinte mensagem aparecerá:: Domain domain1 is ready to receive client requests. Additional services are being started in the background.

para intalar a aplicação Hello utilizando o auto-instalador (autodeploy)

A aplicação pré-empacotado que diz "Hello," pode ser obtido a partir do Tutorial do Java EE 5 ou pode ser baixado de http://glassfish.dev.java.net/downloads/quickstart/hello.war. A aplicação Hello está contida em um Web ARchive (WAR). Aplicações empacotadas podem estar, também, na forma de um Java™ ARchive (JAR) ou Enterprise ARchive (EAR).

Arrastando ou copiando o aplicativo pré-empacotado para o diretório autodeploy, faz com que o apicativo Hello esteja disponível para uso com o mínimo de esforço.

Passos

  1. baixe hello.war de http://glassfish.dev.java.net/downloads/quickstart/hello.war.

  2. Copie hello.war para install-dir/domains/domain1/autodeploy/.

    Dica - No Windows, você pode criar um atalho para o diretório autodeploy em seu desktop e, então, arrastar e colocar o arquivo hello.war dentro do atalho.

    Quando a instalação do aplicativo estiver finalizada, é criado um arquivo chamado hello.war_deployed no diretório autodeploy. Dependendo da velocidade do computador, o processo pode levar alguns segundos. Até que o arquivo apareça, um erro de 404-File Not Found ocorrerá se tentar acessar o endereço da aplicação Hello.

  3. Acesse a URL para visitar a página da aplicação: http://localhost:8080/hello.

    Você verá a página inicial da aplicação, que pede para você preencher seu nome:

  4. Digite o seu nome e clique em Submit.

    A aplicação irá mostrar uma resposta customizada, dando a você um olá (Hello) personalizado.

    Note - Você também pode utilizar o utilitário de linha de comando asadmin para instalar a partir de comando de linha ou via um script-shell. O comando é: asadmin deploy. Para mais informações, acesse asadmin deploy --help em comando de linha.


para se loggar na Console Administrativa

A Console Administrativa é uma interface web que simplifica uma variedade de tarefas e configurações administrativas. É normalmente utilizada para:

  • Instalação e Desinstalação de aplicações

  • Ativação e desativação de aplicações

  • Identificar quais aplicações estão rodando

  • Configurar os recursos de Java Database Connectivity (JDBC™) e outros ajustes de servidor

  • Selecionar uma visão recente dos logs emitidos

Para maiores informações sobre a Console Administrativa, veja a documentação on-line.

Passos

  1. Acesse http://localhost:4848/.

    • localhost É usando se o navegador (browser) e o servidor GlassFish estejam rodando na mesma máquina. Se o servidor GlassFish estiver rodando em outra máquina, substitua na URL localhost pelo nome da máquina a qual o servidor está rodando.

    • 4848 é o número da porta padrão da Console Administrativa. Caso seja alterado o numero da porta durante a instalação, use este numero alterado.

  2. Entre com o usuário administrativo e a senha.

    O usuário padrão é admin e a senha é adminadmin.

  3. Clique em Log In.

    Quando a Console Administrativa aparecer, será parecida como esta:

    Use os itens da árvore para configurar diversos ajustes operacionais para o seridor GlassFish. Inicie o help online a partir do botão no canto superior direito.

    Para visualizar os arquivos de log do servidor, clique Search Log Files no cabeçado de tarefas comuns (Common Tasks) no painel a direita. Quando o Visualizador de Log aparecer, informe o critério e clique em Search para mostrar as mensagens do servidor saída de sua aplicação Java EE.

Empacotando e Instalando Aplicações

Antes que aplicações possam ser instaladas, elas devem ser empacotadas em ARquivos Java (JAR), ARquivos Web (WAR) ou ARquivos Enterprise (EAR). A aplicação empacotada inclui um descritor de instalação a qual fornece ao servidor GlassFish as informações necessárias sobre as necessidades para carga da aplicação, mapeamento de sua URL, e conexão aos recursos que usa. Esta sessão descreve as tecnicas para empacotamente e instalação de aplicações conforme a seguir:

  • Usando a IDE do NetBeans para Desenvolvimento e Instalação de Aplicações

  • Para Empacorar a aplicação de Hello Usando asant

  • Para Instalar a partir da Console Administrativa

  • Para Instalar a partir do Diretório de Desenvolvimento

Para mais informações, veja "Getting Started With Web Applications" no Tutorial do Java EE 5.

Usando a IDE para Desenvolvimento e Instalação de Aplicações

IDE NetBeans™ é um ambiente integrado de desenvolvimento que pode ser usado para criar, empacotar e instalar aplicações Java EE. A IDE Netbeas está disponível para download a partir de http://www.netbeans.org. IDE NetBeans integra o ambiente de desenvolvimento com o ambinete de instalação, e automatiza muitos passos para produção de aplicações Java EE. Assim que registrar o servidor GlassFish com a IDE Netbeans, pode-se facilmente instalar aplicações, fazer mudanças, fazer re-intalação delas.

Para mais informações sobre trabalho do GlassFish com IDE NetBeans, seja a documentação em: https://glassfish.dev.java.net/public/netbeans/index.html.

A popular IDE Eclipse também pode ser usada para desenvolvimento e intalação de aplicações Java EE no GlassFish.  Veja o projeto do Pluigin GlassFish em https://glassfishplugins.dev.java.net/ que fornece um plugin para o Eclipse versão 3.1, para informação de download e um Guia de Rápido Início.


Para Empacotar a Aplicação Hello usando asant

Se a aplicação não for empacotada com a ferramentas como IDE NetBeans, pode-se usar o utilitário asant agregado ao produto. O utilitário asant é usado para automatizar tarefas repetitivas de instalação. É um script de shell que inicializa o ambiente servidor GlassFish e inicia a ferramenta Jakarta Ant. Para maiores informações sobre o utilitário asant, veja o Manual de Referência do Sun Java System Application Server Platform Edition 9 e Capítulo 3, "The asant Utility," em Sun Java System Application Server Platform Edition 9.0 Developer's Guide.

A aplicação Hello instalada recentemente já estava empacotada no arquivo WAR. Esta sessão cobre o empacotamento de componentes de aplicação dentro de outro arquivo WAR. O arquivo build.xml incluído com a aplicação Hello, contém os targets da build.

Passos

  1. Se você ainda não adicionou install-dir/bin em sua variável de ambiente PATH, faça-o agora.

  2. Vá para o diretório onnde você colocou a aplicação Hello baixada.

  3. Digite asant war.

    Este comando monta o arquivo WAR para a aplicação.


para Instalação a partir da Console Administrativa

Agora que você empacotou a aplicação Hello, pode fazer a intalação da mesma. A seguinte sessão discute como instalar a aplicação Hello utilizando a Console Administrativa. Você pode table fazer a instalação utilizando asant.

Passos

  1. Clique http://localhost:4848/asadmin.

    • localhost é utilizado se ambos browser e o servidor GlassFish estiverem rodando na mesma máquina. Se o servidor GlassFish estiver rodando em outra máquina, substitúa o nome da máquina na URL informada.

    • 4848 É o número da porta padrão da Console Administrativa. Caso tenha alterado o número da porta, utilize o número alterado.

  2. Entre com o usuário admin e sua senha.

  3. No painel a esquerda, clique no nó Applications para expandí-lo.

  4. Clique Web Applications.

  5. Se você já têm a aplicação Hello instalada, desinstale-a agora selecionando o checkbox próximo a ela e clique em Undeploy (desintalação).

  6. Para instalar a recém montada aplicação Hello, na página Web Applications, clique em deploy (instalar).

  7. Selecione o arquivo do empacotamento para opção de carga no Servidor de Aplicações e clique em Browse.

  8. Navegue para o diretório onde a aplicação Hello foi baixada e selecione hello.war.

  9. Clique em Next.

  10. Na página de Deploy Web Module, clique em OK.

    A aplicação aparecerá na lista de aplicações Web.

  11. Para verificar a instalação apropriada da aplicação, clique em Launch.

para Instalar a partir do Diretório de Desenvolvimento

Você pode instalar a aplicação diretamente a partir do diretório de desenvolvimento se a hierarquia de diretório e descritores de instalação tiverem sido criados apropriadamente. Pelo fato do processo ser complexo para aplicações não triviais, este procedimento é recomendado apenas para usuários avançados, ou para aplicações simples. Mas quando for factível, instalações diretas a partir do diretório podem aumentar a velocidade do ciclo de desenvolvimento.

A performance do processo pode ser melhorada interetivamente utilizando a Console Administrativa, ou usando o comando asadmin deploydir a partir de linha de comando ou em um script. Os passos para o comando de linha são apresentados aqui, porque a maioria dos desenvolvedores preferem autoatizar o procedimento em comandos de script.

Nota - Antes de usar asadmin no Windows, veja Para configurar o ambiente Windows para o uso de ferramentas de linha de comando.

Passos

  1. No diretório onde foi baixado o aplicativo Hello, crie um novo diretório chamado hello2/.

  2. Unzip o conteúdo do arquivo hello.war no diretório hello2/.

    Preserve a estrutura do diretório do arquivo hello.war quando descimprimí-lo. É o modelo para a estrutura de arquivo necessária para instalação direta a partir do diretório.

  3. Se voc6e já tem a aplicação instalada, desinstale-a agora usando o comando asadmin undeploy hello em linha de comando.

  4. Use o seguinte comando para instalar a aplicação:

    asadmin deploydir hello-download-dir/hello2

    Nota - Especifique o caminho completo para o diretório hello2/ directory.

    Se você não instalou o servidor GlassFish com a opção "Don't prompt" (não questione), especifique, também, a opção --user username, onde username é o nome do usuário. Você será questionado a respeito da senha de administração.

    A seguinte mensagem aparece: Command deploydir executed successfully.

  5. Verifiqye que a aplicação está rodando indo para esta URL: http://localhost:8080/hello.

Usando a Ferramenta de Linha de Comando

O servidor GlassFish oferece uma variedade de ferramentas de linha de comando para desempenhar funções administrativas, em adição ao Console Administrativo. Esta sessão expõem as ferramentas de linha de comandos disponíveis.

Para executar a ferramenta, digite o nome da ferramenta em uma janela de comandos. A tabela abaixo lista as ferramentas oir nome na primeira coluna e a descrição da mesma na segunda coluna.

Tabela 1 Ferramentas de linha de comando

Nome da ferramental

Descrição

appclient

Executa o Container de Aplicações Cliente e evoca a aplicação client empacotada no ARquivo Java (JAR).

asadmin

Ferramente de administração do servidor GlassFish para configuração do servidor.

asant

Executa o Jakarta Ant tool, desta forma, pode-se automatizar respectivamente tarefas de desenvolvimento e instalação.

asapt

Compila fontes Java com Java EE annotations. A ferramenta automaticamente evoca o comando wsimport.

asupgrade

Ferramenta de administração do servidor GlassFish para atualização do software do servidor GlassFish.

capture-schema

Extrai as informações do esquema de banco de dados e produz um arquivo de esquema que o servidor pode utilizar para o Container de Gerenciamento e Persistência (CMP).

jspc

Compila páginas JSP.

package-appclient

Empacota as bibliotecas do container da aplicação cliente e arquivos JAR.

verifier

Valida os descritores de instalacão Java EE com os DTDs..

Esta ferramenta também fornece uma interface gráfica para o usuário. Veja a GUI, especifique a opçao -u.

Alguns sistemas Windows executam um utilitário verificador de driver com o mesmo nome. Para executar o verificador do serviidor GlassFish, é necessário estar dentro do diretório install-dir/bin.

wscompile

Toma a definição das interfaces de serviço para gerar os esqueletos do lado cliente e servidor para o JAX-RPC; ou gera a linguagem de descrição de Serviços Web (WSDL) para a interface provida.

wsdeploy

Gera a implementação específica, arquivo WAR pronto para instalação para aplicações de servoços Web que usam JAX-RPC.

wsgen

Lê as classes de ponto de chegada para web service e gera todos artefatos requeridos para instalação e evocação de serviços web.

wsimport

Gera artefatos portáveis para JAX-WS , assim como interfaces de serviço de ponto de chegada (SEIs), serviços, classes de exceções mapeadas a partir de wsdl:fault e tags soap:headerfault, resposta de beans assíncrona derivada da tag wsdl:message, e tipos de valores JAX-B gerados.

para configurar o ambiente Windows para utilizar ferraentes de linha de comando

Steps

  1. A partir da jabela do Explorer ou desktop, clique com botão direito do mouse sobre Meu Computador.

  2. Escolha Propriedades para apresentar a caixa de diálogo de Propriedades do Sistema.

  3. Clique na aba Avançado.

  4. Clique Variáveis de Ambiente.

  5. Na sessão de variáveis do usuário, adicione ou atualize a variável PATH.

    • Se a variávelPATH existir:

      1. Clique Editar.

      2. Em Valor da Variável, informe o caminha para o diretório bin do servidor, separando das entradas por ponto e vírgula. Por exemplo:

        install-dir/bin;outras_entradas

    • Se a variável PATH não estiver presente:

      1. Clique em Novo.

      2. No nome da variável, digite PATH.

      3. No valor da variável, entre com o caminho o diretório bin do servidor:

        install-dir/bin

  6. Na sessão de variáveis do usuário, adicione a variável de ambiente AS_ADMIN_USER e ajuste-a com o nome do usuário de administração que você associou durante a instalação.

    Se você ajustou esta variável, não há necessidade de entrar com o nome do usuário administrativo quando rodar comandos do asadmin.

  7. Clique em OK para aceitar as mudanças e feche as janelas de comando restantes.
