Pour modifier un module d'audit

Les modules d'audit ne sont pas activés par défaut. Pour plus d'informations sur l'activation des modules d'audit, consultez Activer ou désactiver la journalisation d'audit.

  1. Développez le nœud Configurations.
  2. Sélectionnez l'instance à configurer :
    • Pour configurer une instance particulière, développez le nœud de configuration de l'instance. Par exemple, pour l'instance par défaut, server, développez le nœud server-config.
    • Pour configurer les paramètres par défaut de toutes les instances, développez le noeud default-config.
  3. Développez le nœud Sécurité.
  4. Développez le nœud Modules d'audit.
  5. Cliquez sur le nœud du module d'audit à modifier.
  6. Sur la page Modifier le module d'audit, modifiez le nom de classe, si nécessaire.
  7. Pour ajouter d'autres propriétés au module, sélectionnez le bouton Ajouter puis entrez le nom et la valeur de la propriété.

    Les propriétés valides incluent auditOn, qui détermine si cette classe d'implémentation doit être activée ou non. Les valeurs valides sont vrai et faux.

  8. Pour modifier les propriétés existantes, sélectionnez le nom ou la valeur à modifier puis entrez les modifications directement dans la zone de texte.
  9. Pour supprimer une propriété, sélectionnez la case à gauche de la propriété puis cliquez sur Supprimer des propriétés.
  10. Cliquez sur Enregistrer pour sauvegarder, ou cliquez sur le bouton Précédent du navigateur pour annuler sans sauvegarder.
