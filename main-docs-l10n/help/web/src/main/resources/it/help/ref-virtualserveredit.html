Modifica server virtuale

Utilizzare la pagina Modifica server virtuale per modificare le impostazioni di un server virtuale.

La pagina Modifica server virtuale contiene le opzioni riportate di seguito.

Carica impostazioni predefinite

Pulsante per ripristinare i valori predefiniti delle impostazioni che dispongono di valori predefiniti. Le impostazioni che non dispongono di valori predefiniti non vengono modificate.

Nome configurazione

Il nome della configurazione a cui si applicano le impostazioni presenti in questa pagina. Questo campo è di sola lettura.

ID

Identificativo del server virtuale visibile internamente. Non viene esposto ai client HTTP. I nomi host esposti ai client HTTP devono essere specificati nel campo Host.

Host

Il nome o i nomi host per il computer su cui è in esecuzione il server. Utilizzare i nomi host effettivi o virtuali registrati con il server DNS per la rete (e, in un sistema UNIX, nel file /etc/hosts). Il valore predefinito è il valore della proprietà di sistema ${com.sun.aas.hostName}.

Stato

La condizione desiderata per il server. Il valore può essere uno di quelli riportati di seguito.

On

Il server virtuale è attivo. Questo è il valore predefinito.

Off

Il server virtuale è inattivo. Ogni tentativo di accedere al server comporterà la restituzione del codice di errore 404 (risorsa non disponibile).

Disabilitato

Il server virtuale è disabilitato. Ogni tentativo di accedere al server comporterà la restituzione del codice di errore 403 (elaborazione richiesta rifiutata).

SSO

Specifica se il funzionamento Single Sign-On viene ereditato dal servizio HTTP e se è abilitato o disabilitato. Per impostazione predefinita, viene ereditato dal servizio HTTP. Se questa opzione è selezionata, Single Sign-On è abilitato per le applicazioni Web di questo server virtuale configurate per lo stesso realm. Se questa opzione non è selezionata, Single Sign-On è disabilitato per questo server virtuale e gli utenti devono eseguire l'autenticazione per ogni applicazione nel server virtuale.

Listener di rete

Il listener o i listener di rete associati a questo server.

Modulo Web predefinito

L'eventuale modulo Web distribuito che deve rispondere a tutte le richieste che non possono essere mappate ad altri moduli Web distribuiti sul server virtuale. Se il modulo Web predefinito non è specificato, viene utilizzato il modulo Web con una radice contesto vuota. Se non sono disponibili moduli Web con una radice contesto vuota, viene creato e utilizzato un modulo Web predefinito per il sistema.

File di log

Il nome percorso del file in cui verranno visualizzati i messaggi di log provenienti da questo server virtuale. Per impostazione predefinita, i messaggi di log verranno visualizzati nel file domain-dir/logs/server.log.

Docroot

Il percorso assoluto della directory di documenti radice per il server. Il valore predefinito è domain-dir/docroot.

Log degli accessi

Specifica se il log degli accessi viene ereditato dal servizio HTTP e se è abilitato o disabilitato. Per impostazione predefinita, viene ereditato dal servizio HTTP.

Directory

Il percorso assoluto della directory dei log degli accessi del server. L'impostazione predefinita è domain-dir/logs/access.

Proprietà aggiuntive

Proprietà aggiuntive per il server virtuale.

Per informazioni sulle proprietà disponibili, vedere Proprietà specifiche dei server virtuali.
