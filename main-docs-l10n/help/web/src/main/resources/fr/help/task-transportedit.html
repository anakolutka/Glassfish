Modification d'un transport

  1. Dans l'arborescence de navigation, développez le noeud Configuration.

  2. Sous le noeud Configuration, développez le noeud Configuration réseau.

  3. Sous le noeud Configuration réseau, sélectionnez le noeud Transports.

    La page Transports apparaît.

  4. Sur la page Transports, cliquez sur le nom du transport que vous modifiez.

    La page Modifier le transport apparaît.

  5. Dans le champ Nom de classe, saisissez le nom de classe de l'implémentation de transport.

    La valeur par défaut est com.sun.grizzly.TCPSelectorHandler.

  6. Dans le champ Gestionnaires de clé de sélection, saisissez le nom de classe du gestionnaire de clé de sélection.

    Nom de classe du gestionnaire de clé de sélection. Le gestionnaire de clé de sélection est une classe abstraite qui gère le cycle de vie des clés de sélection. Si vous ne spécifiez pas de valeur, une implémentation par défaut est utilisée.

  7. Dans la liste déroulante Type de tampon d'octet, sélectionnez le type ByteBuffer à utiliser.

    Les choix disponibles sont HEAP et DIRECT. La valeur par défaut est HEAP.

  8. Dans le champ Threads accepteur, saisissez le nombre de processeurs de l'ordinateur.

    La valeur -1 indique que Grizzly calculera lui-même le nombre de threads d'accepteur, en fonction du nombre de processeurs de l'ordinateur. La valeur par défaut est 1.

    Pour définir le nombre de threads de traitement des demandes, définissez la taille maximale du pool de threads utilisé par un processus d'écoute réseau qui emploie ce transport.

  9. Dans le champ Nombre max. de connexions, saisissez le nombre maximal de connexions en attente sur un processus d'écoute réseau qui utilise ce transport.

    La valeur par défaut est 4 096.

  10. Dans le champ Taille du tampon, saisissez (en octets) la taille du tampon à fournir pour les flux d'entrée créés par un processus d'écoute réseau qui utilise ce transport.

    La valeur par défaut est 8 192.

  11. Dans le champ Expiration de clé d'inactivité, entrez le nombre de secondes au-delà duquel la clé d'inactivité sera annulée et le canal fermé.

    La valeur par défaut est 30.

  12. Dans le champ Expiration de la lecture, saisissez le délai d'attente (en millisecondes) de GlassFish Server lors de la phase d'analyse de l'en-tête et du corps d'une opération de lecture.

    La valeur par défaut est 30 000.

  13. Dans le champ Expiration de l'interrogation du sélecteur, entrez la durée (en millisecondes) de blocage d'un sélecteur NIO en attente d'événements (demandes utilisateur).

    La valeur par défaut est 1000.

  14. Dans le champ Expiration de l'écriture, saisissez le délai d'attente (en millisecondes) de GlassFish Server avant qu'il ne considère que le client distant est déconnecté lors de l'écriture de la réponse.

    La valeur par défaut est 30 000.

  15. Cochez la case Activé de l'option Configuration de l'affichage pour vider la configuration interne de Grizzly vers les journaux du serveur.

    Cette option peut fournir des informations intéressantes pour le débogage. Cette option est désactivée par défaut.

  16. Cochez la case Activé de l'option Surveillance du trafic afin de vider les informations de demande/réponse vers le journal du serveur.

    Cette option peut fournir des informations intéressantes pour le débogage, mais risque de réduire considérablement les performances. Cette option est désactivée par défaut.

  17. Cochez la case Aucun délai TCP pour activer TCP_NODELAY (également appelé algorithme de Nagle).

    Cette option est activée par défaut.

  18. Cliquez sur Enregistrer.

    Conseil:

    Pour revenir aux paramètres par défaut, cliquez sur le bouton Charger les valeurs par défaut, puis sur Enregistrer.
