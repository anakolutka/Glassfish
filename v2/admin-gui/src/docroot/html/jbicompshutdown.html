To Disable or Shutdown Components

In the Developer profile, components are installed, then started, stopped, or shutdown for the one domain. In a Clusters profile, components are installed and associated with a target, then the component is enabled or disabled for that named cluster target. Consequently, starting, stopping, and shutting down components is available for the Developers profile, and enabling or disabling components is only available for the Clusters profile.

  1. In the tree view, select the JBI node and then the Components node.

    A list of Binding Component and Service Engines is displayed.

  2. In the Developer profile, click the checkbox next to all the Components and Service Engines you want to shutdown and select Shut Down from the Operations drop-down menu.

    The State column identifies that the Components and Service Engines are shutdown.

  3. In the Clusters profile, click the checkbox next to the Components and Service Engines you want to disable.

    The Properties page for the selected Components and Service Engines is displayed.

  4. In the Targets table, click the checkbox next to all the Components and Service Engines you are disabling.

  5. Click Disable.

    The Status column identifies that the Components and Service Engines are disabled. You can specify the desired target by selecting the Manage Targets button in the Targets table.

  6. Alternatively, to start a single Component and Service Engine on all targets, click the Component, select all the desired targets, then click the Shut Down button.
