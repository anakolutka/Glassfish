새 컨텍스트 서비스

[새 컨텍스트 서비스] 페이지에서는 컨텍스트 서비스 리소스를 생성할 수 있습니다.

[새 컨텍스트 서비스] 페이지에는 다음 옵션이 포함되어 있습니다.

JNDI 이름

리소스를 식별하는 고유한 JNDI 이름입니다. 관례상 이 이름은 리소스 유형 지시자 및 슬래시로 시작됩니다(예: concurrent/, jdbc/ 또는 jms/).

컨텍스트 정보

스레드로 전달할 컨텍스트입니다(Classloader, JNDI, Security 또는 WorkArea). 컨텍스트 전달은 사용 또는 사용 안함으로 설정할 수 있습니다. 사용으로 설정되면 선택된 컨텍스트가 전달됩니다. 사용 안함으로 설정되면 선택된 컨텍스트를 포함하여 어떤 컨텍스트도 전달되지 않습니다. 기본값은 [사용]이며 4가지 컨텍스트가 모두 선택되어 있습니다.

설명

해당 리소스에 대한 설명입니다.

상태

리소스의 상태입니다. 리소스는 사용 또는 사용 안함으로 설정할 수 있습니다. 기본값은 사용입니다.

추가 속성

리소스에 대한 추가 속성입니다. GlassFish Server에서는 이 리소스 유형에 대한 추가 속성을 정의하지 않습니다. 또한 현재 이 리소스 유형은 추가 속성을 사용하지 않습니다.

대상

(이 옵션은 클러스터나 독립형 인스턴스가 도메인에 생성된 경우에 표시됩니다.)

리소스에 대한 클러스터 및 독립형 인스턴스입니다. 원하는 대상을 [선택한 대상] 열로 이동하려면 [추가] 및 [모두 추가] 단추를 사용합니다. 필요하지 않은 대상을 [사용 가능한 대상] 열로 이동하려면 [제거] 및 [모두 제거] 단추를 사용합니다.
