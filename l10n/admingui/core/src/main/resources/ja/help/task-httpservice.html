HTTP サービスを設定する

  1. ナビゲーションツリーで、「設定」ノードを展開します。
  2. 「設定」ノードの下で、「HTTP サービス」ノードを選択します。

    「HTTP サービス」ページが開きます。

  3. 「アクセスログ」の「有効」チェックボックスを選択して、このプロパティーを指定していないすべての仮想サーバーのサブ要素に対するアクセスログを有効にします。

    「アクセスログ」の「有効」チェックボックスの選択を解除すると、このプロパティーを指定していないすべての仮想サーバーのサブ要素に対するアクセスログは無効になります。

  4. 「アクセスログバッファーのサイズ」フィールドに、アクセスログバッファーのサイズをバイト単位で入力します。

    ゼロ以下の値では、アクセスログのバッファーリングが無効になります。デフォルト値は 4096 です。

  5. 「アクセスログの書き込み間隔」フィールドに、ログをディスクに書き込む前の秒数を入力します。

    アクセスログは、バッファーがいっぱいになるか、指定した時間が経過すると書き込まれます。値が 0 以下の場合、いっぱいになっていなくてもバッファーは常に書き込まれます。つまり、サーバーアクセスがあるたびに、ログメッセージはファイルに直接格納されます。デフォルト値は 0 です。

  6. (省略可能) 「追加プロパティー」セクションで、追加プロパティーを指定します。

    次のプロパティーが使用可能です。

    connectionTimeout

    接続の受け入れ後、要求 URI の行が提示されるまでの HTTP リスナーの待ち時間を指定します (ミリ秒単位)。

    デフォルト値は 12,000 (12 秒) です。

    maxKeepAliveRequests

    サーバーによって接続が閉じられるまでの、パイプラインできる HTTP 要求の最大数を指定します。HTTP/1.0 キープアライブ、HTTP/1.1 キープアライブ、パイプラインが無効化するには、このプロパティーの値を 1 に設定します。

    デフォルト値は 1000 です。

    traceEnabled

    true に設定すると、TRACE メソッドが有効化されます。Enterprise Server がクロスサイトスクリプト攻撃の影響を受けにくくするには、このプロパティーを false に設定します。

    TRACE メソッドについての詳細は、「RFC 2616: Hypertext Transfer Protocol -- HTTP/1.1」を参照してください。

    デフォルト値は true です。

  7. 「保存」をクリックします。
