部署企业应用程序/模块

使用“部署企业应用程序/模块”页面来部署应用程序。此页面包含以下选项：

类型

您要部署的应用程序类型。此版本中唯一可用的应用程序类型是 Web 应用程序 (.war)。

位置

您要部署的应用程序的归档位置。

以下选项指明了可以访问此归档的位置，以及此归档是一个文件还是一个目录。

要上载到服务器的打包文件

此归档位于客户机上的文件或可从客户机访问的文件中。

客户机是可通过浏览器查看管理控制台的主机。

可从 Application Server 访问的本地打包文件或目录

此归档是位于服务器计算机上的文件，或者是展开目录中的未打包应用程序。

服务器计算机是用于运行应用服务器和域管理服务器的主机。

应用程序名称

应用程序名称。

上下文根目录

应用程序的路径。在应用程序的 URL 中，上下文根目录要紧跟着端口号 (http://host:port/context-root/...)。上下文根目录必须以正斜杠开头，例如： /hello。

虚拟服务器

与此应用程序相关联的虚拟服务器。

启用状态

如果选中此选项，则将启用应用程序。默认情况下，此选项是选中的。

启用运行检验器

如果选中了此选项，则检验器实用程序将运行以检查应用程序是否与 JavaTM Platform, Enterprise Edition（Java EE 平台）以及 Enterprise Server 兼容。

启用预编译 JSP

如果选中了此选项，将对 JavaServer Pages (JSP) 文件进行预编译。如果未选中此选项，则在运行时首次访问 JSP 文件时对其进行编译。默认情况下，此选项未选中。

库

应用程序所需的库 JAR 文件。

说明

应用程序的简要说明。
