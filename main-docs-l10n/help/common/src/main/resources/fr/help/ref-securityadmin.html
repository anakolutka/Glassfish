Administration sécurisée

Utilisez la page Administration sécurisée pour activer ou désactiver l'administration sécurisée.

Lorsque l'administration sécurisée est activée :

  • Le serveur DAS et les instances du domaine utilisent des certificats SSL pour crypter les messages qu'ils échangent.

  • Le serveur DAS accepte les messages d'administration des clients admin distants tels que l'utilitaire asadmin et les IDE.

La page Administration sécurisée contient les informations ci-après.

Activer l'administration sécurisée

Bouton permettant d'activer l'administration sécurisée. Ce bouton apparaît uniquement lorsque l'administration sécurisée est désactivée.

Remarque:

Si l'administrateur du domaine possède un mot de passe vide, les tentatives d'activation de l'administration sécurisée échouent.
Désactiver l'administration sécurisée

Bouton permettant de désactiver l'administration sécurisée. Ce bouton apparaît uniquement lorsque l'administration sécurisée est activée.

Alias d'administration

Alias qui fait référence au certificat SSL/TLS sur le serveur DAS. Le serveur DAS utilise cet alias pour s'identifier auprès des instances. La valeur par défaut est s1as. Cette option apparaît uniquement lorsque l'administration sécurisée est désactivée.

Alias d'instance

Alias qui fait référence au certificat SSL/TLS sur les instances. Les instances utilisent cet alias pour s'identifier auprès du serveur DAS. La valeur par défaut est glassfish-instance. Cette option apparaît uniquement lorsque l'administration sécurisée est désactivée.
