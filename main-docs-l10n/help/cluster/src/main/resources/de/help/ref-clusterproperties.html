Clustereigenschaften

Auf der Seite "Clustereigenschaften" werden die Eigenschaften aufgelistet, die für die ausgewählten Cluster festgelegt sind. Diese Eigenschaften fügen optionale Konfigurationsinformationen zum Cluster hinzu.

Das Feld "Clustername" ist ein schreibgeschütztes Feld, in dem der Name des ausgewählten Clusters angezeigt wird.

Für jede Instanz werden folgende Informationen angezeigt:

Name

Der Name der Eigenschaft.

Wert

Der Wert der Eigenschaft, der für die ausgewählte Instanz festgelegt ist.

Beschreibung

Eine Textbeschreibung mit weiteren Informationen zur Eigenschaft.

Die Tabelle "Weitere Eigenschaften" enthält darüber hinaus die folgenden Optionen.

Eigenschaft hinzufügen

Schaltfläche zum Hinzufügen einer Eigenschaft. Durch Klicken auf diese Schaltfläche wird eine Zeile zur Tabelle "Weitere Eigenschaften" hinzugefügt.

Eigenschaften löschen

Schaltfläche zum Löschen von mindestens einer ausgewählten Eigenschaft. Eine gelöschte Eigenschaft wird auf den Standardwert zurückgesetzt oder, falls kein Standardwert vorhanden ist, nicht definiert.

GlassFish Server definiert die folgenden Clustereigenschaften:

GMS_DISCOVERY_URI_LIST

Die Speicherorte der GlassFish Server-Instanzen im Cluster, die zur Erkennung des Clusters verwendet werden. Diese Eigenschaft ist nur erforderlich, wenn der GMS (Group Management Service) kein Multicast für die Übertragung von Nachrichten verwendet.

Gültige Werte für diese Eigenschaft sind:

  • Eine durch Komma getrennte Liste von URIs (Uniform Resource Identifiers). Jeder URI muss eine GlassFish Server-Instanz oder den DAS suchen. Dieses Format ist erforderlich, wenn mehrere GlassFish Server-Instanzen auf demselben Host ausgeführt werden.

    Das Format der einzelnen URIs in der Liste ist wie folgt:

    scheme://host-name-or -IP-address:port

    • scheme ist das URI-Schema, das tcp lautet.

    • host-name-or -IP-address ist der Hostname oder die IP-Adresse des Hosts, auf dem die Instanz ausgeführt wird.

    • Port ist die Portnummer des Ports, auf dem die Instanz auf Nachrichten von GMS horcht. Die Systemeigenschaft GMS_LISTENER_PORT-clustername muss für die Instanz festgelegt werden. Weitere Informationen zur Festlegung dieser Systemeigenschaft für eine Instanz finden Sie unter "Discovering a Cluster When Multicast Transport Is Unavailable" im Oracle GlassFish Server High Availability Administration Guide.

  • Eine durch Komma getrennte Liste mit IP-Adressen oder Namen von Hosts, auf denen der DAS oder die Instanzen ausgeführt werden. Die Liste kann eine Mischung aus IP-Adressen und Hostnamen enthalten. Dieses Format kann nur verwendet werden, wenn jeweils eine geclusterte Instanz auf den einzelnen Hosts ausgeführt wird. Der Wert der Eigenschaft GMS_LISTENER_PORT muss für jedes Cluster in einer Domain eindeutig sein.

  • Das Schlüsselwort generate. Dieses Format kann nur verwendet werden, wenn eine Instanz in einem Cluster auf jedem Host ausgeführt wird und der DAS auf einem separaten Host ausgeführt wird. Mehrere Instanzen auf demselben Host können nicht Mitglied desselben Clusters sein. Der Wert der Eigenschaft GMS_LISTENER_PORT muss für jedes Cluster in einer Domain eindeutig sein.

GMS_LISTENER_PORT

Die Portnummer des Ports, auf dem das Cluster auf Nachrichten vom GMS horcht.

Der Standardwert ist eine Referenz zur Systemeigenschaft GMS_LISTENER_PORT-cluster-name. Standardmäßig ist diese Systemeigenschaft nicht festgelegt. In diesem Fall wählt der GMS einen freien Port aus dem durch die Eigenschaften GMS_TCPSTARTPORT und GMS_TCPENDPORT definierten Bereich. Standardmäßig liegt dieser Bereich zwischen 9090 und 9200. In den meisten Fällen reicht dieses Standardbehavior aus.

Wenn der GMS jedoch nicht Multicast für Nachrichten verwendet, muss diese Eigenschaft eine Portnummer angeben, die für alle GlassFish Server-Instanzen im Cluster gilt. Damit Sie den Standardwert verwenden können, um diese Anforderung zu erfüllen, legen Sie die Portnummer individuell für jede Instanz mit einer Systemeigenschaft fest.

Beispiel: Verwenden Sie die Seite Clustersystemeigenschaften, um die Systemeigenschaft GMS_LISTENER_PORT-cluster-name für den DAS zu erstellen. Stellen Sie dann für jede Instanz im Cluster die Systemeigenschaft GMS_LISTENER_PORT-cluster-name auf die Portnummer ein, auf der die Instanz auf Nachrichten vom GMS horcht. Der Standardwert der Eigenschaft GMS_LISTENER_PORT für das Cluster referenziert diese Systemeigenschaft.

GMS_LOOPBACK

Gibt an, ob eine Instanz von sich selbst Nachrichten auf Anwendungsebene erhalten kann, die die Instanz an das Cluster sendet.

Mögliche Werte:

false

Die Instanz darf keine Nachrichten von sich selbst erhalten (Standardeinstellung).

true

Die Instanz kann Nachrichten von sich selbst erhalten. Verwenden Sie diese Einstellung zum Testen einer Instanz, wenn die Instanz die einzige Instanz in einem Cluster ist.

GMS_MULTICAST_TIME_TO_LIVE

Die maximale Anzahl der Iterationen oder Übertragungen, die bei einer Multicast-Nachricht der folgenden Ereignistypen auftreten kann, bevor die Meldung verworfen wird:

  • Gruppenerkennung

  • Mitglied-Heartbeats

  • Mitgliedschaftsänderungen

Damit die Konfiguration des Netzwerks, auf dem der DAS und die Clusterinstanzen bereitgestellt sind, übereinstimmt, setzen Sie diesen Wert so niedrig wie möglich. Um den niedrigstmöglichen Wert für das System zu ermitteln, verwenden Sie den Unterbefehl validate-multicast1.

Bei einem Wert von 0 wird sichergestellt, dass Multicast-Nachrichten auf keinen Fall den Host verlassen, von dem sie gesendet werden.

Ein Wert von 1 kann verhindern, dass Nachrichten zwischen den über einen Switch oder Router verbundenen Hosts im gleichen Subnet gesendet wird.

Der Standardwert ist 4. Dadurch wird sichergestellt, dass Nachrichten zwischen allen Cluster-Membern in Netzwerken erfolgreich gesendet werden, in denen Hosts über Switches oder Router verbunden sind.

GMS_TCPENDPORT

Die höchste Portnummer in dem Bereich, aus dem der GMS einen freien Port wählt, wenn die Systemeigenschaft GMS_LISTENER_PORT-cluster-name nicht festgelegt ist. Der Standardwert ist 9200.

GMS_TCPSTARTPORT

Die niedrigste Portnummer in dem Bereich, aus dem der GMS einen freien Port wählt, wenn die Systemeigenschaft GMS_LISTENER_PORT-cluster-name nicht festgelegt ist. Der Standardwert ist 9090.
