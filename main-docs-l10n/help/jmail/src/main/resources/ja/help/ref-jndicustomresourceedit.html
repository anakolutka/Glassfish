カスタム・リソースの編集

カスタムJNDIリソースを編集するには、「カスタム・リソースの編集」ページを使用します。

「カスタム・リソースの編集」ページには、次のオプションが含まれます。

JNDI名

リソースを識別する一意の名前。JNDI名は読取り専用フィールドです。JNDI名を指定できるのは、新規カスタム・リソースの作成時のみです。

リソース・タイプ

リソースの完全修飾タイプ定義。

ファクトリ・クラス

ファクトリ・クラスのユーザー指定の名前。このクラスはjavax.naming.spi.ObjectFactoryインタフェースを実装する必要があります。

デプロイ順序

カスタム・リソースのデプロイ順序。

説明

カスタム・リソースの説明。

ステータス

リソースが実行時に使用可能であるかどうか。

ドメインにデフォルト・サーバー・インスタンスserverのみが存在する場合、リソースの実行時可用性は有効化することも無効化することも可能です。デフォルトでは、実行時可用性は有効化されています。

ドメインにクラスタまたはスタンドアロン・インスタンスが作成されている場合は、「ターゲット」タブを使用して、ターゲットにおけるリソースの実行時可用性を制御します。

追加プロパティ

カスタム・リソースの追加プロパティ。GlassFish Serverはカスタム・リソースの追加プロパティを定義しません。
