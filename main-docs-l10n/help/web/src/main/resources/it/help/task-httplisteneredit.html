Per modificare un listener HTTP

  1. Espandere il nodo Configurazione nell'albero di navigazione.

  2. Nel nodo Configurazione espandere il nodo Servizio HTTP.

  3. Nel nodo Configurazione di rete selezionare il nodo Listener HTTP.

    Viene visualizzata la pagina Listener HTTP.

  4. Nella pagina Listener HTTP fare clic sul nome del listener HTTP da modificare.

    Viene visualizzata la pagina Modifica listener HTTP.

  5. Nel campo Porta digitare il numero della porta di ascolto che verrà utilizzata dal listener HTTP.

    I valori validi sono compresi tra 1 e 65535. In un sistema UNIX, per creare socket in ascolto sulle porte comprese tra 1 e 1024 sono necessari i privilegi di utente privilegiato.

  6. Selezionare o deselezionare la casella di controllo Abilitato dell'opzione Stato per modificare lo stato del listener HTTP.

    Il listener può essere abilitato o disabilitato. Questa opzione è abilitata per impostazione predefinita. Se il listener è disabilitato, qualsiasi tentativo di connessione al listener genera un'eccezione socket (java.net.ConnectException).

  7. Selezionare la casella di controllo Abilitato dell'opzione Sicurezza per abilitare la sicurezza per il listener HTTP.

    Questa opzione è disabilitata per impostazione predefinita.

    Se si sta configurando admin-listener per il server di amministrazione del dominio (DAS), che utilizza la configurazione server-config, questa opzione è di sola lettura. Per abilitare la sicurezza per questo listener, è necessario abilitare l'amministrazione sicura. Vedere Amministrazione sicura.

  8. Selezionare la casella di controllo Abilitato dell'opzione Listener JK per impostare nel listener il supporto del protocollo mod-jk di Apache.

    Il protocollo mod-jk è un protocollo di comunicazione/rete proprietario tra httpd (Web server) di Apache e un'istanza backend di &ProductName. Se il listener supporta il protocollo, tutto il contenuto statico deve essere gestito mediante httpd, mentre le richieste di risorse dinamiche (ovvero servlet e pagine JSP) vengono reindirizzate al listener HTTP abilitato mod-jk di GlassFish Server tramite il protocollo mod-jk. Questa opzione è disabilitata per impostazione predefinita.

  9. Nel campo Indirizzo digitare l'indirizzo IP di ascolto per il listener HTTP.

    L'indirizzo può essere specificato utilizzando la notazione per indirizzi IPv6 o con separazione mediante coppie di punti. È possibile specificare any (per INADDR_ANY) per impostare l'ascolto su tutti gli indirizzi IP. Può essere costituito da un nome host.

  10. Nell'elenco a discesa Server virtuale predefinito selezionare il server virtuale da associare a questo listener HTTP.

    Utilizzare la pagina Server virtuali per definire i server virtuali.

  11. Nell'elenco a discesa Pool di thread selezionare il pool di thread da associare al listener HTTP.

    In genere si seleziona uno dei due pool di thread configurati durante l'installazione di GlassFish Server.

  12. Nel campo Nome server, digitare il nome host da utilizzare negli URL che il server invia al client.

    Se il server utilizza un alias, questo nome corrisponde al nome alias. Se il server non utilizza un alias, lasciare vuoto questo campo.

    Questo valore ha effetto sugli URL generati in modo automatico dal server, ma non sugli URL relativi a directory e file memorizzati nel server. Se il server utilizza un alias, il nome del server deve corrispondere al nome alias. Se vengono aggiunti i due punti e il numero della porta, la porta specificata viene utilizzata negli URL che il server invia al client.

  13. Fare clic su Salva.

    Suggerimento:

    Per ripristinare le impostazioni predefinite, fare clic sul pulsante Carica impostazioni predefinite, quindi su Salva.
Passi successivi

Se per il listener HTTP è stata abilitata la sicurezza è possibile utilizzare la scheda SSL per modificare le impostazioni SSL del listener HTTP.

Un listener HTTP è un tipo di listener di rete. Per accedere alle funzioni avanzate, è possibile modificare un listener HTTP utilizzando le pagine Listener di rete.
