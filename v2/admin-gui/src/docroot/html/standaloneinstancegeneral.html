To View Server Instance Information

  1. In the Admin Console tree, expand the Stand-alone Instances node and click the desired stand-alone server instance.

    The General Information page appears. It displays the following details about the target server instance.

    • Name: Name of the standalone instance.

    • Status: Indicates if the server instance requires to be restarted or not.

    • HTTP Load Balancer: Name of the load balancer. Click Enable Load Balancing to enable the target server instance for load balancing. If you want to gracefully disable the server instance from accepting requests, click Quiesce. For more information about quiescing a server instance, see Disabling (Quiescing) a Server Instance or Cluster in Sun Java System Application Server 9.1 High Availability Administration Guide.

    • HTTP Port(s): currently configured ports for HTTP requests and responses.

    • IIOP Port(s): currently configured ports for IIOP requests and responses.

    • JVM: Click on JVM Report to display reports on the Java Virtual Machine, including a summary report, memory management and garbage collection report, class loading report, and a current thread dump.

    • Configuration Directory: directory on host machine containing configuration file, such as domain.xml.

    • Installed Version: product version identifier.

    • Debug: whether debug is enabled.

  2. Click Start Instance if the instance is in the stopped state.
  3. Click Stop Instance to stop the instance.
  4. Click View Log Files to open the server log viewer.
  5. Click Rotate Log File to rotate the log file for the instance.

    This action schedules the log file for rotation. The actual rotation takes place the next time an entry is written to the log file. The rotation happens immediately for the default server (the DAS) but is delayed for other standalone server.

  6. Click JNDI Browsing to browse the JNDI tree for a running instance.
  7. Click Recover Transactions to recover incomplete transactions.
